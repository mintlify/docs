<div id="enabling-data-privacy-integrations">
  ## Activation des intégrations de protection des données
</div>

Vous pouvez ajouter des plateformes de protection des données à votre documentation. Ajoutez le champ `integrations` à votre fichier `docs.json` avec les scripts correspondants.

```json
  "integrations": {
    "osano": "SOURCE"
  }
```

<div id="cookie-consent-and-disabling-telemetry">
  ## Consentement aux cookies et désactivation de la télémétrie
</div>

Si vous devez vérifier si un utilisateur a déjà donné son consentement aux cookies pour la conformité au RGPD, vous pouvez spécifier une key de stockage local et une value sous `cookies` :

```json
  "integrations": {
    "cookies": {
      "key": "LOCAL STORAGE KEY",
      "value": "LOCAL STORAGE VALUE"
    }
  }
```

Si ces valeurs sont définies, le stockage local sera vérifié pour déterminer si l’utilisateur a consenti aux cookies. Si ce n’est pas le cas, la télémétrie sera désactivée.

Si vous souhaitez désactiver la télémétrie pour tous les utilisateurs, vous pouvez ajouter ce qui suit à votre fichier `docs.json` :

```json
  "integrations": {
    "telemetry": {
      "enabled": false
    }
  }
```
