Ajoutez ce qui suit à votre fichier `docs.json` pour intégrer le gestionnaire de consentement aux cookies [Osano](https://www.osano.com/).

<CodeGroup>
  ```json Options d’intégration dans docs.json
  "integrations": {
      "osano": "SOURCE"
  }
  ```

  ```json Exemple
  "integrations": {
      "osano": "https://cmp.osano.com/2sUB2dqwqdkks/8dqwd-dwd86£-4a9b/osano.js"
  }
  ```
</CodeGroup>

La valeur `SOURCE` correspond à la valeur de `src` dans l’extrait de code généré par Osano. Elle commence toujours par `https://cmp.osano.com/`.

```html Code snippet from Osano
<script src="https://cmp.osano.com/placeholder/placeholder/osano.js"/>
```


<div id="troubleshooting">
  ## Dépannage
</div>

<Accordion
  title="Les pages ne se chargent pas en mode de conformité Strict"
>
  Si vos pages de documentation ne se chargent pas correctement lorsque vous utilisez le mode de conformité **Strict** d’Osano, vous devrez autoriser le domain de Mintlify pour permettre le chargement des images et autres ressources.

  <Steps>
    <Step title="Accéder à Managed Rules">
      Dans votre Dashboard Osano, allez dans **Scripts** → **Managed Rules**.
    </Step>

    <Step title="Ajouter le domain Mintlify">
      Ajoutez `.mintlify.app/` en tant que règle gérée.

      <Frame>
        <img src="/images/integrations/osano-managed-rule.png" alt="Règle gérée Osano" />
      </Frame>

      <Info>
      Cela garantit que tous les contenus servis par Mintlify (y compris les images, les feuilles de style et autres ressources de documentation) sont considérés comme essentiels et se chargeront même lorsque Osano bloque du contenu tiers non catégorisé.
      </Info>
    </Step>
  </Steps>
</Accordion>
