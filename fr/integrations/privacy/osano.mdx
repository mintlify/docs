Ajoutez ce qui suit à votre fichier `docs.json` pour intégrer le gestionnaire de consentement aux cookies [Osano](https://www.osano.com/).

<CodeGroup>
  ```json Integration options in docs.json
  "integrations": {
      "osano": "SOURCE"
  }
  ```

  ```json Exemple
  "integrations": {
      "osano": "https://cmp.osano.com/2sUB2dqwqdkks/8dqwd-dwd86£-4a9b/osano.js"
  }
  ```
</CodeGroup>

La valeur à renseigner pour `SOURCE` correspond au `src` dans l’extrait de code généré par Osano. Elle commence toujours par `https://cmp.osano.com/`.

```html Code snippet from Osano
<script src="https://cmp.osano.com/placeholder/placeholder/osano.js"/>
```


<div id="troubleshooting">
  ## Dépannage
</div>

<Accordion
  title="Les pages ne se chargent pas avec le mode de conformité Strict"
>
  Si vos pages de documentation ne se chargent pas correctement lorsque vous utilisez le mode de conformité **Strict** d’Osano, vous devrez autoriser le domain de Mintlify afin de permettre le chargement des images et autres ressources.

  <Steps>
    <Step title="Accéder à Managed Rules">
      Dans votre Dashboard Osano, allez à **Scripts** → **Managed Rules**.
    </Step>

    <Step title="Ajouter le domain Mintlify">
      Ajoutez `.mintlify.app/` comme règle gérée.

      <Frame>
        <img src="/images/integrations/osano-managed-rule.png" alt="Règle gérée Osano" />
      </Frame>

      <Info>
      Cela garantit que toutes les ressources servies par Mintlify (y compris les images, les feuilles de style et les autres ressources de documentation) sont considérées comme essentielles et se chargeront même lorsque Osano bloque du contenu tiers non catégorisé.
      </Info>
    </Step>
  </Steps>
</Accordion>
