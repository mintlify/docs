Ajoutez la gestion du consentement aux cookies [Osano](https://www.osano.com/) à votre documentation avec un [fichier JavaScript personnalisé](/fr/customize/custom-scripts#custom-javascript) dans votre référentiel.

<div id="setup">
  ## Configuration
</div>

Créez un fichier `osano.js` dans le référentiel de votre documentation avec le contenu suivant :

<CodeGroup>
  ```js Format
  var script = document.createElement("script");
  script.src = "OSANO_SCRIPT_URL";
  document.head.appendChild(script);
  ```

  ```js Example
  var script = document.createElement("script");
  script.src = "https://cmp.osano.com/your-tenant-id/your-site-id/osano.js";
  document.head.appendChild(script);
  ```
</CodeGroup>

Remplacez `OSANO_SCRIPT_URL` par l’URL de votre script Osano. Cette URL de script correspond à la valeur `src` dans l’extrait de code généré par Osano. Votre URL de script commence toujours par `https://cmp.osano.com/` et se termine par `/osano.js`.

<div id="troubleshooting">
  ## Dépannage
</div>

<Accordion title="Les pages ne se chargent pas en mode de conformité Strict">
  Si vos pages de documentation ne se chargent pas correctement lorsque vous utilisez le mode de conformité **Strict** d’Osano, vous devrez ajouter le domaine Mintlify à votre liste d’autorisation pour permettre le chargement des images et autres ressources.

  <Steps>
    <Step title="Accéder à Managed Rules">
      Dans votre Dashboard Osano, allez dans **Scripts** → **Managed Rules**.
    </Step>

    <Step title="Ajouter le domain Mintlify">
      Ajoutez `.mintlify.app/` en tant que règle gérée.

      <Frame>
        <img src="/images/integrations/osano-managed-rule.png" alt="Règle gérée Osano" />
      </Frame>

      <Info>
        Cela garantit que tous les contenus servis par Mintlify (y compris les images, les feuilles de style et autres ressources de documentation) sont considérés comme essentiels et se chargeront même lorsque Osano bloque du contenu tiers non catégorisé.
      </Info>
    </Step>
  </Steps>
</Accordion>
