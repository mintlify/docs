title: "Rédiger des prompts efficaces"
description: "Obtenez de meilleurs résultats de l'agent avec des prompts clairs et ciblés."

---

Considérez l'agent comme un assistant utile qui a besoin de votre orientation pour accomplir des tâches. Donnez-lui des instructions claires et du contexte. Des tâches plus ciblées sont plus faciles à réaliser : décomposez donc les projets complexes en plus petites étapes.

<div id="make-prompts-specific-and-outcome-focused">
  ## Formulez des prompts précis et orientés vers les résultats
</div>

Des prompts génériques comme `@mintlify Improve the onboarding page` appliquent de bonnes pratiques générales, mais ne vont pas forcément améliorer le contenu de la manière précise que vous aviez en tête.

Formulez plutôt des prompts en fonction des résultats que vous voulez que vos utilisateurs atteignent ou des problèmes qu’ils rencontrent.

- `@mintlify A lot of users have trouble installing the CLI. Review the onboarding page and update the docs so that users can easily install the CLI`
- `@mintlify Developers keep getting 401 errors when following our authentication guide. Review the auth docs and add clearer examples showing how to properly format the API key`

<div id="use-broad-prompts-for-maintenance-tasks">
  ## Utiliser des prompts généraux pour les tâches de maintenance
</div>

Utilisez des prompts généraux pour la maintenance générale de votre contenu, comme corriger les fautes de frappe, mettre à jour les redirections ou renommer une fonctionnalité dans l'ensemble de votre documentation.

- `@mintlify Find and fix all typos in the docs`
- `@mintlify change all unordered lists to use * instead of -`

<div id="specify-a-domain-name-for-multi-site-organizations">
  ## Spécifier un nom de domaine pour les organisations multisites
</div>

Si vous avez plusieurs sites de documentation, incluez le paramètre `subdomain` dans votre message pour spécifier sur quel ensemble de documentation l'agent doit travailler.

Pour trouver votre nom de domaine, regardez l'URL de votre Dashboard pour l'ensemble de documentation que vous souhaitez mettre à jour. Le nom de domaine correspond à la dernière partie après le nom de votre organisation. Par exemple, si l'URL de votre Dashboard est `https://dashboard.mintlify.com/org-name/domain-name`, votre nom de domaine est `domain-name`.

Utilisez le format `@mintlify subdomain=<your-domain-name> <your-prompt>` pour demander à l'agent de travailler sur un ensemble de documentation spécifique.

- `@mintlify subdomain=public-docs Add a new section to the quickstart about inviting collaborators based on this PR` : demande à l'agent de mettre à jour le quickstart uniquement sur le site `public-docs`.
- `@mintlify subdomain=customer-docs Update the auth docs for the new authentication method` : demande à l'agent de mettre à jour la documentation d'authentification uniquement sur le site `customer-docs`.
