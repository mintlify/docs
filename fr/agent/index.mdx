<Info>
  L'agent nécessite une [offre Pro ou Enterprise](https://mintlify.com/pricing?ref=agent).
</Info>

L'agent est un outil d'IA qui crée des pull requests (demandes de fusion) avec des modifications proposées pour votre documentation, en fonction de vos instructions. Lorsque vous envoyez une requête, l'agent :

* **Recherche** : recherche et analyse votre documentation existante, les dépôts connectés, le contexte pertinent et le web.
* **Planifie** : crée une liste de tâches structurée pour les travaux de documentation complexes.
* **Rédige** : génère ou met à jour du contenu en suivant les standards de rédaction et les bonnes pratiques.
* **Valide** : exécute les vérifications de l'Interface en ligne de commande (CLI) Mintlify pour s'assurer que la documentation se génère correctement.
* **Publie les modifications** : ouvre une pull request (demande de fusion) avec les modifications proposées pour relecture, ou pousse les modifications directement sur votre branche de déploiement, selon votre [paramètre de processus de relecture](/fr/agent/index#choose-how-the-agent-publishes-changes).

<div id="how-to-use-the-agent">
  ## Comment utiliser l'agent
</div>

Vous pouvez envoyer des prompts à l'agent depuis :

- **Slack** : mentionnez `@Mintlify` dans n'importe quel canal auquel l'agent appartient, ou envoyez un message direct à l'application. Consultez [Ajouter l'agent à Slack](/fr/agent/slack).
- **Le dashboard** : ouvrez le panneau de l'agent depuis le [dashboard Mintlify](https://app.mintlify.com) pour démarrer une nouvelle conversation.
- **Automatisations** : déclenchez l'agent selon un planning ou à partir d'un événement. Consultez [Automatisations](/fr/automations/index).

<div id="what-you-can-do-with-the-agent">
  ## Ce que vous pouvez faire avec l’agent
</div>

Utilisez l’agent pour :

* Rédiger de nouveaux contenus à partir de vos prompts, pull requests (demandes de fusion), fils Slack ou pièces jointes.
* Mettre à jour la documentation existante pour de nouvelles fonctionnalités ou des évolutions d’API.
* Traiter et intégrer des images, des schémas et d’autres fichiers à partir des pièces jointes Slack.
* Rechercher et réviser des exemples de code et des références d’API dans l’ensemble de votre documentation.
* Référencer le code source de n’importe quel dépôt sur lequel la GitHub App Mintlify est installée.
* Automatiser la maintenance de la documentation avec des automatisations.
* Répondre aux questions sur votre documentation et sur des sujets de rédaction technique.
* Traiter les retours de revue de code pour maintenir la qualité de la documentation.
* Capitaliser les connaissances issues des conversations Slack et des pull requests (demandes de fusion) avant qu’elles ne soient perdues.
* Naviguer et coordonner efficacement les modifications dans plusieurs fichiers et répertoires.

<div id="connect-your-github-account">
  ## Connectez votre compte GitHub
</div>

Par défaut, l'agent ouvre des pull requests (demandes de fusion) attribuées au bot Mintlify. Pour que les pull requests vous soient attribuées, connectez votre compte GitHub depuis la page [My profile](https://dashboard.mintlify.com/settings/account) du Dashboard.

<div id="choose-how-the-agent-publishes-changes">
  ## Choisissez comment l'agent publie les modifications
</div>

Par défaut, l'agent ouvre une pull request pour chaque modification afin que votre équipe puisse la relire avant sa mise en ligne. Si Mintlify héberge votre déploiement, l'agent pousse les modifications directement sur votre branche de déploiement pour qu'elles soient publiées immédiatement.

Pour les déploiements connectés à votre propre fournisseur Git, vous pouvez modifier la façon dont l'agent publie les modifications dans le paramètre **Review process** de la page [Agent](https://app.mintlify.com/products/agent) de votre Dashboard :

- **Create a pull request** : L'agent pousse ses commits sur une nouvelle branche et ouvre une pull request contre votre branche de déploiement. Utilisez cette option lorsque vous souhaitez qu'une personne relise les modifications avant leur mise en ligne.
- **Push to main** : L'agent pousse ses commits directement sur votre branche de déploiement, et les modifications sont publiées dès que l'agent a terminé. Utilisez cette option lorsque vous faites confiance à l'agent pour publier sans relecture, par exemple sur un déploiement de staging.

<Note>
  Si votre branche de déploiement est protégée par des règles qui bloquent les push directs, le paramètre **Review process** est masqué et l'agent ouvre toujours une pull request.
</Note>

<div id="connect-repositories-as-context">
  ## Connecter des dépôts en tant que contexte
</div>

L’agent ne peut accéder qu’aux dépôts que vous connectez via la GitHub App de Mintlify. Configurez les dépôts auxquels l’agent peut accéder dans le panneau **Settings** de l’agent ou dans les [paramètres de la GitHub App](https://github.com/apps/mintlify/installations/new).

<div id="use-ai-tools-alongside-the-agent">
  ## Utiliser des outils d'IA en complément de l'agent
</div>

L'agent fonctionne de manière asynchrone via des pull requests, mais vous pouvez aussi utiliser des outils de codage assistés par IA comme Cursor ou Claude Code en local pour des modifications rapides et itératives. Installez le [skill](/fr/ai/skillmd) Mintlify et connectez le [serveur MCP](/fr/ai/model-context-protocol) afin que votre éditeur dispose du même contexte que celui utilisé par l'agent.

Pour permettre à un agent externe de modifier directement votre contenu via des appels d'outils MCP, connectez-le au [serveur MCP Mintlify](/ai/mintlify-mcp).
