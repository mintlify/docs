L'agent peut vous aider dans de nombreuses tâches de documentation. Ces workflows présentent plusieurs façons d'intégrer l'agent à votre processus de documentation. Choisissez une approche qui correspond à la façon dont votre équipe travaille actuellement, puis adaptez-la à vos besoins spécifiques.

<div id="iterate-on-a-prompt-in-a-slack-thread">
  ## Itérer sur un prompt dans un fil Slack
</div>

Envoyez un prompt à l&#39;agent, puis continuez à le mentionner avec `@mintlify` dans le même fil pour affiner et itérer sur la pull request (demande de fusion) qu&#39;il crée.

Par exemple : `@mintlify Notre page de démarrage rapide a besoin d'une nouvelle section sur l'invitation de collaborateurs`. Puis : `@mintlify La nouvelle section devrait s'appeler "Inviter des collaborateurs"`. Et ainsi de suite pour les autres itérations.

<div id="start-with-the-agent-finish-manually">
  ## Commencer avec l'agent, terminer manuellement
</div>

Demandez à l'agent de démarrer un projet, puis récupérez la branche qu'il crée et terminez la tâche dans votre environnement local ou dans l'éditeur web. L'agent peut vous aider à démarrer, puis vous prenez le relais pour terminer la tâche.

Par exemple : `@mintlify Update the quickstart page to include information about inviting collaborators` puis basculez sur la branche pour apporter toutes les modifications supplémentaires avec la méthode de votre choix.

<div id="update-docs-when-merging-feature-changes">
  ## Mettre à jour la documentation lors de la fusion de modifications de fonctionnalités
</div>

Lorsque vous fusionnez une pull request (demande de fusion) de fonctionnalité, partagez le lien de la PR avec l&#39;agent pour mettre à jour la documentation concernée.

Par exemple : `@mintlify This PR adds a new authentication method. Update the docs to include the new auth flow: [PR link]`.

<div id="generate-release-notes-from-a-pull-request">
  ## Générer des notes de version à partir d&#39;une pull request
</div>

Fournissez à l&#39;agent une pull request (demande de fusion) spécifique pour générer des notes de version ou des mises à jour du journal des modifications à partir de l&#39;historique des commits.

Par exemple : `@mintlify Generate release notes for this PR: [PR link]`.

<div id="address-pull-request-review-feedback">
  ## Traiter les commentaires de revue sur une pull request
</div>

Partagez le lien de la pull request (demande de fusion) avec l'agent afin qu'il prenne en compte les commentaires des relecteurs et les retours de revue de code sur les pull requests de documentation.

Par exemple : `@mintlify Address the review comments on this PR: [PR link]`.

<div id="generate-code-examples">
  ## Générer des exemples de code
</div>

Demandez à l'agent de générer des exemples de code pour des fonctionnalités présentes dans l'ensemble de votre documentation ou sur des pages spécifiques.

Par exemple : `@mintlify Generate a code example to make the authentication method easier to understand`.

<div id="add-images-or-files-to-your-docs">
  ## Ajouter des images ou des fichiers à votre documentation
</div>

Joignez des fichiers ou des images directement à votre message Slack lorsque vous formulez une demande à l'agent. L'agent traite la pièce jointe et l'intègre dans la documentation.

Par exemple : `@mintlify Add this diagram to the architecture overview page` avec l'image jointe au message.

<div id="review-existing-content">
  ## Passer en revue le contenu existant
</div>

Demandez à l&#39;agent de passer en revue le contenu existant pour vérifier l&#39;exactitude technique, le style, la grammaire ou d&#39;autres aspects.

Par exemple : `@mintlify Review the API rate limiting section. We changed limits last month`.

<div id="respond-to-user-feedback">
  ## Répondre aux retours des utilisateurs
</div>

Fournissez à l'agent les retours de vos utilisateurs pour qu'il puisse mettre à jour votre documentation de manière ciblée.

Par exemple : `@mintlify Users are getting confused by step 3 in the setup guide. What might be making it unclear?`.

<div id="automate-with-the-api">
  ## Automatiser avec l&#39;API
</div>

Intégrez l&#39;agent dans vos outils d&#39;automatisation existants pour mettre automatiquement à jour la documentation lorsque le code change, déclencher des revues de contenu ou synchroniser les mises à jour de la documentation entre plusieurs dépôts.

Utilisez les endpoints de l&#39;agent pour [créer des tâches](/fr/api-reference/agent/create-agent-job), [récupérer une tâche spécifique](/fr/api-reference/agent/get-agent-job) et [récupérer toutes les tâches](/fr/api-reference/agent/get-all-jobs).

Lors de la création de tâches via l&#39;API, vous pouvez contrôler si les pull requests s&#39;ouvrent en mode brouillon à l&#39;aide du paramètre `asDraft` (dont la valeur par défaut est `true`). Définissez `asDraft: false` pour créer des pull requests non brouillon, prêtes pour une revue et une fusion immédiates dans des workflows automatisés.

Découvrez comment configurer l&#39;automatisation via l&#39;API dans le tutoriel [Mettre automatiquement à jour la documentation lorsque le code est fusionné](/fr/guides/automate-agent).
