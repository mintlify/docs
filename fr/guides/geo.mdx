Optimisez votre documentation pour les moteurs de recherche traditionnels comme pour les moteurs de réponses dopés à l’IA tels que ChatGPT, Perplexity et Google AI Overviews.

La Generative Engine Optimization (GEO) consiste à obtenir des citations par les systèmes d’IA grâce à un contenu exhaustif et des informations structurées, tandis que le SEO traditionnel vise le positionnement dans les résultats de recherche.



<div id="geo-quickstart">
  ## Démarrage rapide GEO
</div>

<div id="initial-setup">
  ### Configuration initiale
</div>

1. **Assurez-vous que votre documentation est indexée** dans vos paramètres `docs.json`
2. **Passez en revue les pages existantes** pour repérer les descriptions et les titres manquants

<div id="content-improvements">
  ### Améliorations de contenu
</div>

1. **Ajoutez des tableaux comparatifs** aux pages appropriées
2. **Passez en revue les titres** pour vous assurer qu’ils répondent aux questions courantes
3. **Améliorez le maillage interne** entre les sujets connexes
4. **Testez avec des outils d’IA** pour vérifier l’exactitude



<div id="geo-best-practices">
  ## Bonnes pratiques GEO
</div>

De manière générale, une documentation bien rédigée et bien structurée offrira de solides performances en GEO. Vous devez toujours privilégier l’écriture pour vos utilisateurs : si votre contenu répond à leurs besoins, vous serez déjà sur la bonne voie pour l’optimiser pour les outils d’IA. Créer un contenu réellement utile, plutôt que d’optimiser pour l’optimisation, est récompensé par les moteurs de recherche traditionnels comme par les moteurs de réponse d’IA.

Concentrez-vous sur :

- Un contenu aligné sur les besoins des utilisateurs plutôt que sur la simple correspondance de mots-clés
- Des informations structurées et faciles à parcourir
- Des réponses directes aux questions

<div id="format-for-clarity">
  ### Mise en forme pour plus de clarté
</div>

Ces pratiques de mise en forme aident les outils d’IA à analyser et à comprendre votre contenu :

- Ne sautez pas de niveaux de titre (H1 → H2 → H3)
- Utilisez des noms d’objets précis plutôt que « it » ou « this »
- Indiquez la langue de programmation des blocs de code
- Donnez aux images un texte alternatif descriptif
- Ajoutez des liens vers des concepts connexes pour aider l’IA à comprendre les relations

<div id="answer-questions-directly">
  ### Répondez directement aux questions
</div>

Rédigez un contenu qui traite des questions spécifiques des utilisateurs :

- Commencez les sections par l’idée principale à retenir
- Utilisez des titres descriptifs correspondant aux requêtes courantes
- Décomposez les sujets complexes en étapes numérotées



<div id="mintlify-configuration">
  ## Configuration de Mintlify
</div>

Utilisez ces fonctionnalités pour améliorer le GEO.

<div id="add-descriptive-page-metadata">
  ### Ajouter des métadonnées de page descriptives
</div>

Incluez des titres et des descriptions clairs dans votre frontmatter :

```mdx
---
title: "Guide d’authentification d’API"
description: "Guide complet pour implémenter l’authentification d’API avec des exemples de code"
---
```

<div id="configure-global-indexing-settings">
  ### Configurer les paramètres d’indexation globaux
</div>

Ajoutez ceci à votre `docs.json` :

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "Vos docs"
    }
  }
}
```

<div id="llmstxt">
  ### LLMs.txt
</div>

Les fichiers LLMs.txt aident les systèmes d’IA à comprendre la structure de votre documentation, comme les sitemaps aident les moteurs de recherche. Mintlify génère automatiquement des fichiers LLMs.txt pour vos docs. Aucune configuration n’est requise.


<div id="testing-your-documentation">
  ## Tester votre documentation
</div>

Testez différents outils d’IA avec des questions sur votre produit et votre documentation pour voir dans quelle mesure vos docs sont citées.

**Posez des questions précises aux assistants IA sur vos docs :**
- « Comment configurer l’authentification avec cette API ? »
- « Guide-moi pas à pas dans le processus d’installation »

**Vérifiez que les outils fournissent :**
- Des exemples de code corrects
- Des instructions pas à pas exactes
