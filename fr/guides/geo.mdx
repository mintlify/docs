Optimisez votre documentation pour les moteurs de recherche traditionnels et pour les moteurs de réponse alimentés par l’IA, tels que ChatGPT, Perplexity et Google AI Overviews.

L’optimisation pour les moteurs génératifs (GEO) vise à être citée par les systèmes d’IA grâce à un contenu exhaustif et des informations structurées, tandis que le SEO traditionnel cherche à améliorer le classement dans les résultats de recherche.



<div id="geo-quickstart">
  ## Démarrage rapide GEO
</div>

<div id="initial-setup">
  ### Configuration initiale
</div>

1. **Assurez-vous que votre documentation est indexée** dans vos paramètres `docs.json`
2. **Passez en revue les pages existantes** pour repérer les descriptions et titles manquants

<div id="content-improvements">
  ### Améliorations de contenu
</div>

1. **Ajoutez des tableaux comparatifs** aux pages appropriées
2. **Passez en revue les titres** pour vous assurer qu’ils répondent aux questions courantes
3. **Améliorez les liens internes** entre les sujets connexes
4. **Testez avec des outils d’IA** pour vérifier l’exactitude



<div id="geo-best-practices">
  ## Bonnes pratiques GEO
</div>

En général, une documentation bien rédigée et bien structurée offrira un bon GEO. Continuez de prioriser l’écriture pour vos utilisateurs : si votre contenu répond à leurs besoins, vous serez déjà bien avancé dans l’optimisation pour les outils d’IA. Créer un contenu réellement utile, plutôt que d’optimiser pour l’optimisation, est récompensé par les moteurs de recherche traditionnels comme par ceux basés sur l’IA.

Concentrez-vous sur :

- Un contenu aligné sur les besoins des utilisateurs plutôt que sur la simple correspondance de mots-clés
- Des informations structurées et faciles à parcourir
- Des réponses directes aux questions

<div id="format-for-clarity">
  ### Formater pour plus de clarté
</div>

Ces pratiques de formatage aident les outils d’IA à analyser et comprendre votre contenu :

- Ne sautez pas de niveaux de titre (H1 → H2 → H3)
- Utilisez des noms d’objets précis plutôt que « it » ou « this »
- Ajoutez un label aux code block avec leur langage de programmation
- Donnez aux images un texte alternatif descriptif
- Faites des liens vers des concepts associés pour aider l’IA à comprendre les relations

<div id="answer-questions-directly">
  ### Répondez directement aux questions
</div>

Rédigez un contenu qui répond à des questions précises des utilisateurs :

- Commencez les sections par l’essentiel à retenir
- Utilisez des titres descriptifs correspondant aux requêtes courantes
- Décomposez les sujets complexes en étapes numérotées



<div id="mintlify-configuration">
  ## Configuration de Mintlify
</div>

Utilisez ces fonctionnalités pour améliorer le SEO.

<div id="add-descriptive-page-metadata">
  ### Ajouter des metadata descriptives pour la page
</div>

Incluez des titres clairs et des descriptions dans votre frontmatter :

```mdx
---
title: "Guide d'authentification API"
description: "Guide complet pour implémenter l'authentification API avec des exemples de code"
---
```

<div id="configure-global-indexing-settings">
  ### Configurer les paramètres d’indexation globaux
</div>

Ajoutez ceci à votre `docs.json` :

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "Votre documentation"
    }
  }
}
```

<div id="llmstxt">
  ### LLMs.txt
</div>

Les fichiers LLMs.txt aident les systèmes d’IA à comprendre la structure de votre documentation, comme les sitemaps aident les moteurs de recherche. Mintlify génère automatiquement des fichiers LLMs.txt pour votre documentation. Aucune configuration n’est nécessaire.


<div id="testing-your-documentation">
  ## Tester votre documentation
</div>

Testez différents outils d’IA en leur posant des questions sur votre produit et votre documentation afin d’évaluer la manière dont vos docs sont citées.

**Posez des questions précises aux Assistants IA au sujet de vos docs :**
- « Comment configurer l’authentification avec cette API ? »
- « Guide-moi pas à pas dans le processus d’installation »

**Vérifiez que les outils fournissent :**
- Des exemples de code corrects
- Des instructions détaillées, étape par étape
