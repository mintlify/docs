Optimisez votre documentation pour les moteurs de recherche traditionnels et pour les moteurs de réponse alimentés par l’IA comme ChatGPT, Perplexity et Google AI Overviews.

Le Generative Engine Optimization (GEO) vise à obtenir des citations par les systèmes d’IA grâce à un contenu exhaustif et des informations structurées, tandis que le SEO traditionnel cible le classement dans les résultats de recherche.

## Démarrage rapide GEO

### Configuration initiale

1. **Vérifiez que votre documentation est indexée** dans vos paramètres `docs.json`
2. **Passez en revue les pages existantes** pour repérer les titres et descriptions manquants

### Améliorations de contenu

1. **Ajoutez des tableaux comparatifs** aux pages pertinentes
2. **Révisez les titres** pour vous assurer qu’ils répondent aux questions courantes
3. **Améliorez le maillage interne** entre les sujets connexes
4. **Testez avec des outils d’IA** pour vérifier l’exactitude

## Bonnes pratiques GEO

De manière générale, une documentation bien rédigée et bien structurée présente une bonne GEO. Vous devez toujours privilégier l’écriture pour vos utilisateurs, et si votre contenu répond à leurs besoins, vous serez bien parti pour l’optimiser pour les outils d’IA. Créer un contenu réellement utile, plutôt que d’optimiser pour l’optimisation, est valorisé à la fois par les moteurs de recherche traditionnels et par les moteurs d’IA.

Concentrez-vous sur :

- Un contenu aligné sur les besoins des utilisateurs plutôt que sur la correspondance de mots-clés
- Des informations structurées et facilement scannables
- Des réponses directes aux questions

### Mise en forme pour plus de clarté

Ces pratiques de mise en forme aident les outils d’IA à analyser et à comprendre votre contenu :

- Ne sautez pas de niveaux de titres (H1 → H2 → H3)
- Utilisez des noms d’objets explicites plutôt que « it » ou « this »
- Indiquez le langage des blocs de code
- Fournissez aux images un texte alternatif descriptif
- Créez des liens vers des concepts connexes pour aider l’IA à comprendre les relations

### Répondez directement aux questions

Rédigez un contenu qui traite des questions spécifiques des utilisateurs :

- Commencez les sections par l’essentiel à retenir
- Utilisez des titres descriptifs qui correspondent aux requêtes courantes
- Décomposez les sujets complexes en étapes numérotées

## Configuration de Mintlify

Utilisez ces fonctionnalités pour améliorer la GEO.

### Ajouter des métadonnées de page descriptives

Incluez des titres et des descriptions clairs dans votre frontmatter :

```mdx
---
title: "API authentication guide"
description: "Complete guide to implementing API authentication with code examples"
---
```

### Configurer les paramètres d’indexation globaux

Ajoutez ceci à votre `docs.json` :

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "Your docs"
    }
  }
}
```

### LLMs.txt

Les fichiers LLMs.txt aident les systèmes d’IA à comprendre la structure de votre documentation, comme les sitemaps aident les moteurs de recherche. Mintlify génère automatiquement des fichiers LLMs.txt pour vos docs. Aucune configuration n’est requise.

## Tester votre documentation

Testez divers outils d’IA avec des questions sur votre produit et votre documentation pour évaluer dans quelle mesure vos docs sont citées.

**Posez des questions précises aux assistants d’IA au sujet de vos docs :**
- « Comment configurer l’authentification avec cette API ? »
- « Guide-moi pas à pas dans le processus d’installation »

**Vérifiez que les outils fournissent :**
- Des exemples de code corrects
- Des instructions pas à pas exactes
