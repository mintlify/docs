Optimisez votre documentation pour les moteurs de recherche traditionnels et pour les moteurs de réponse alimentés par l’IA, comme ChatGPT, Perplexity et Google AI Overviews.

La Generative Engine Optimization (GEO) vise à être citée par les systèmes d’IA grâce à un contenu exhaustif et à des informations structurées, tandis que le SEO traditionnel cible le classement dans les résultats de recherche.

<div id="geo-quickstart">
  ## Démarrage rapide de GEO
</div>

<div id="initial-setup">
  ### Configuration initiale
</div>

1. **Assurez-vous que votre documentation est indexée** dans vos paramètres `docs.json`
2. **Passez en revue les pages existantes** pour repérer les descriptions et les titres manquants

<div id="content-improvements">
  ### Améliorations du contenu
</div>

1. **Ajouter des tableaux comparatifs** sur les pages concernées
2. **Vérifier les titres** pour s’assurer qu’ils répondent aux questions fréquentes
3. **Améliorer les liens internes** entre les sujets connexes
4. **Tester avec des outils d’IA** pour vérifier l’exactitude

<div id="geo-best-practices">
  ## Bonnes pratiques GEO
</div>

De manière générale, une documentation bien rédigée et bien structurée aura un bon GEO. Vous devez toujours privilégier l’écriture pour vos utilisateurs : si votre contenu répond à leurs besoins, vous serez déjà bien parti pour l’optimiser pour les outils d’IA. Créer un contenu réellement utile plutôt que d’optimiser pour l’optimisation est valorisé par les moteurs de recherche, qu’ils soient traditionnels ou basés sur l’IA.

Concentrez-vous sur :

- Un contenu aligné sur les besoins des utilisateurs plutôt que sur l’appariement de mots-clés
- Des informations structurées et faciles à parcourir
- Des réponses directes aux questions

<div id="format-for-clarity">
  ### Format pour plus de clarté
</div>

Ces pratiques de mise en forme aident les outils d’IA à analyser et à comprendre votre contenu :

- Ne sautez pas de niveaux de titre (H1 → H2 → H3)
- Utilisez des noms d’objets précis plutôt que « it » ou « this »
- Attribuez aux code blocks le langage de programmation
- Donnez aux images un texte alternatif descriptif
- Liez les concepts associés pour aider l’IA à comprendre les relations

<div id="answer-questions-directly">
  ### Répondez directement aux questions
</div>

Rédigez un objet qui répond aux questions spécifiques des utilisateurs :

- Commencez les sections par la principale idée à retenir
- Utilisez des titres descriptifs correspondant aux requêtes courantes
- Décomposez les sujets complexes en étapes numérotées

<div id="mintlify-configuration">
  ## Configuration de Mintlify
</div>

Utilisez ces fonctionnalités pour améliorer le GEO.

<div id="add-descriptive-page-metadata">
  ### Ajouter des metadata descriptives pour la page
</div>

Incluez des titres et des descriptions clairs dans votre frontmatter :

```mdx
---
title: "Guide d'authentification API"
description: "Guide complet pour implémenter l'authentification API avec des exemples de code"
---
```


<div id="configure-global-indexing-settings">
  ### Configurer les paramètres d’indexation globaux
</div>

Ajoutez ceci à votre `docs.json` :

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "Votre documentation"
    }
  }
}
```


<div id="llmstxt">
  ### llms.txt
</div>

Les fichiers llms.txt aident les systèmes d’IA à comprendre la structure de votre documentation, comme les sitemaps aident les moteurs de recherche. Mintlify génère automatiquement des fichiers llms.txt pour votre documentation. Aucune configuration n’est nécessaire.

<div id="testing-your-documentation">
  ## Tester votre documentation
</div>

Testez différents outils d’IA en leur posant des questions sur votre produit et votre documentation pour voir dans quelle mesure vos docs sont citées.

**Posez des questions précises aux assistants IA à propos de vos docs :**

- « Comment configurer l’authentification avec cette API ? »
- « Guide-moi pas à pas dans le processus d’installation »

**Vérifiez que les outils fournissent :**

- Des exemples de code corrects
- Des instructions détaillées étape par étape
