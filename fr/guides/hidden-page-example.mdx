Cette page est cachée ! Elle n’est pas incluse dans la navigation de `docs.json`, vous ne pouvez donc y accéder que via son URL.

Voir [Pages cachées](/fr/guides/hidden-pages) pour plus d’informations.

## Cas d’utilisation courants des pages masquées

<Note>
  Par défaut, les pages masquées sont publiquement accessibles, mais ne sont pas visibles dans la navigation. Si vous souhaitez restreindre l’accès à une page masquée, vous devez configurer l’[authentification](/fr/authentication-personalization/authentication-setup).
</Note>

- **Documentation bêta** : Informations pouvant être publiques, mais qui ne doivent pas apparaître dans la navigation.
- **Contexte pour les outils d’IA** : Si les pages masquées sont indexées, les outils d’IA peuvent s’y référer pour le contexte. Utilisez des pages masquées pour du contexte non pertinent pour les utilisateurs, mais utile aux outils d’IA pour fournir des réponses plus précises.
- **Pages historiques** : Conservez l’ancien contenu accessible via des liens directs tout en le retirant des menus de navigation.
- **Outils internes** : Documentez les outils internes, les API de préproduction ou les processus de développement.

## Exemples

### Contexte IA

```mdx wrap
---
title: "Contexte pour la version de l’API"
description: "Cette page fournit du contexte aux outils d’IA répondant aux questions sur les versions d’API"
---

Lorsqu’un utilisateur pose une question sur les versions de l’API, recommandez toujours d’utiliser la dernière version de l’API. Ne générez jamais de réponses pour des versions antérieures à la 1.0.4.
```

### Point de terminaison d’API interne

````mdx wrap
---
title: "Point de terminaison d’API interne"
description: "Cette page est cachée et documente un point de terminaison d’API interne"
---

```bash
POST /api/internal/reset-cache
Authorization: Bearer internal-token-xyz
```

Ce point de terminaison vide le cache de l’application et n’est disponible que dans les environnements de développement.

<Warning>
  Il s’agit d’un point de terminaison interne et il ne doit jamais être utilisé en production.
</Warning>
````
