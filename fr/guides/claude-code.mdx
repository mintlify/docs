Claude Code est un outil en ligne de commande à agent qui peut vous aider à maintenir votre documentation. Il peut rédiger de nouveaux contenus, relire des pages existantes et garder la documentation à jour.

Vous pouvez entraîner Claude Code à comprendre vos standards et workflows de documentation en ajoutant un fichier `CLAUDE.md` à votre projet et en l’affinant au fil du temps.

## Prise en main

**Prérequis :**
- Abonnement Claude actif (Pro, Max ou accès API)

**Configuration :**
1. Installez Claude Code :
  ```bash
  npm install -g @anthropic-ai/claude-code
  ```
2. Accédez à votre répertoire de documentation.
3. (Facultatif) Ajoutez le fichier `CLAUDE.md` ci-dessous à votre projet.
4. Lancez `claude` pour démarrer.

## Modèle CLAUDE.md

Enregistrez un fichier `CLAUDE.md` à la racine de votre répertoire de documentation pour aider Claude Code à comprendre votre projet. Ce fichier entraîne Claude Code sur vos standards de documentation, vos préférences et vos workflows. Consultez [Gérer la mémoire de Claude](https://docs.anthropic.com/en/docs/claude-code/memory) dans la documentation d’Anthropic pour plus d’informations.

Copiez cet exemple de modèle ou adaptez-le aux spécifications de votre documentation :

```mdx
# Documentation Mintlify

## Relation de travail
- Vous pouvez contester des idées — cela peut mener à une meilleure documentation. Citez vos sources et expliquez votre raisonnement lorsque vous le faites
- DEMANDEZ TOUJOURS des précisions plutôt que de faire des suppositions
- NE JAMAIS mentir, deviner ou inventer des informations

## Contexte du projet
- Format : fichiers MDX avec frontmatter YAML
- Config : docs.json pour la navigation, le thème et les paramètres
- Composants : composants Mintlify

## Stratégie de contenu
- Documenter juste ce qu’il faut pour la réussite de l’utilisateur — ni trop, ni trop peu
- Prioriser l’exactitude et la facilité d’utilisation de l’information
- Rendre le contenu pérenne lorsque c’est possible
- Rechercher les informations existantes avant d’ajouter du nouveau contenu. Éviter les duplications sauf raison stratégique
- Vérifier la cohérence avec les schémas/patrons existants
- Commencer par les plus petits changements raisonnables

## docs.json

- Se référer au [schéma docs.json](https://mintlify.com/docs.json) lors de la création du fichier docs.json et de la navigation du site

## Exigences de frontmatter pour les pages
- title : Titre de page clair et descriptif
- description : Résumé concis pour le SEO/la navigation

## Normes rédactionnelles
- Voix à la deuxième personne (« vous »)
- Prérequis au début du contenu procédural
- Tester tous les exemples de code avant publication
- Aligner le style et le formatage sur les pages existantes
- Inclure des cas d’usage basiques et avancés
- Balises de langage sur tous les blocs de code
- Texte alternatif pour toutes les images
- Chemins relatifs pour les liens internes

## Workflow Git
- NE JAMAIS utiliser --no-verify lors des commits
- Demander comment gérer les modifications non commit avant de commencer
- Créer une nouvelle branche lorsqu’aucune branche claire n’existe pour les changements
- Commiter fréquemment tout au long du développement
- NE JAMAIS ignorer ni désactiver les hooks pre-commit

## À ne pas faire
- Omettre le frontmatter sur un fichier MDX
- Utiliser des URL absolues pour les liens internes
- Inclure des exemples de code non testés
- Faire des suppositions — toujours demander des précisions
```

## Exemples d’invites

Une fois Claude Code configuré, essayez ces invites pour voir comment il peut vous aider dans les tâches de documentation courantes. Vous pouvez copier-coller ces exemples tels quels ou les adapter à vos besoins.

### Convertir des notes en documents soignés
Transformez des brouillons en vraies pages Markdown avec composants et frontmatter.

**Exemple d’invite :**
```text wrap
Convert this text into a properly formatted MDX page: [paste your text here]
```

### Vérifier la cohérence des docs
Obtenez des suggestions pour améliorer le style, la mise en forme et l’utilisation des composants.

**Exemple d’invite :**
```text wrap
Review the files in docs/ and suggest improvements for consistency and clarity
```

### Mettre à jour les docs lors des évolutions de fonctionnalités
Gardez la documentation à jour lorsque votre produit évolue.

**Exemple d’invite :**
```text wrap
Our API now requires a version parameter. Update our docs to include version=2024-01 in all examples
```

### Générer des exemples de code complets
Créez des exemples multilingues avec gestion des erreurs.

**Exemple d’invite :**
```text wrap
Create code examples for [your API endpoint] in JavaScript, Python, and cURL with error handling
```

## Étendre Claude Code

Au-delà des invites manuelles avec Claude Code, vous pouvez l’intégrer à vos flux de travail existants.

### Automatisation avec GitHub Actions
Exécutez Claude Code automatiquement lors des modifications de code pour garder la documentation à jour. Vous pouvez déclencher des revues de documentation sur les pull requests ou mettre à jour les exemples lorsque des changements d’API sont détectés.

### Flux de travail multi‑instances
Utilisez des sessions distinctes de Claude Code pour différentes tâches — une pour la rédaction de nouveau contenu et une autre pour la relecture et l’assurance qualité. Cela permet de maintenir la cohérence et de détecter des problèmes qu’une seule session pourrait laisser passer.

### Collaboration d’équipe
Partagez votre fichier `CLAUDE.md` affiné avec votre équipe pour garantir des standards de documentation cohérents entre tous les contributeurs. Les équipes élaborent souvent des invites et des flux de travail spécifiques au projet qui deviennent partie intégrante de leur processus de documentation.

### Commandes personnalisées
Créez des commandes « slash » réutilisables dans `.claude/commands/` pour les tâches de documentation fréquemment utilisées, propres à votre projet ou à votre équipe.
