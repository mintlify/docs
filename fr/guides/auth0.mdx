Mintlify n’est pas responsable des problèmes de sécurité pouvant découler de votre implémentation spécifique.
</Note>

<div id="overview">
  ## Présentation
</div>

Ce guide vous explique comment configurer Auth0 comme fournisseur d’authentification pour votre documentation Mintlify. À la fin, vos utilisateurs pourront se connecter à votre documentation avec leurs identifiants Auth0.

<Steps>
  <Step title="Create a Regular Web Application in Auth0">
    Connectez-vous à votre Dashboard Auth0 et accédez à **Applications** > **Applications**. Cliquez sur le bouton **Create Application**, donnez un nom à votre application (par exemple, « Mintlify »), puis sélectionnez **Regular Web Applications** comme type d’application. Cliquez ensuite sur **Create**.
    
    <Frame>![Création d’une nouvelle application dans Auth0](/images/guides/auth0/auth0-create-application.png)</Frame>
  </Step>
  <Step title="Get client information">
    Après avoir créé votre application, vous serez redirigé vers la page des paramètres de l’application. Vous y trouverez les identifiants essentiels nécessaires pour l’intégration OAuth :
    
    <Frame>![Page des paramètres du client Auth0](/images/guides/auth0/auth0clientsettings.png)</Frame>
    
    Notez les informations suivantes :
    - **Domain**: Il s’agit du domaine de votre tenant Auth0 (par exemple, `your-tenant.auth0.com`)
    - **Client ID**: L’identifiant public de votre application
    - **Client Secret**: La clé secrète de votre application (à conserver en lieu sûr)
    
    Vous aurez besoin de ces valeurs pour configurer Mintlify à l’étape suivante.
  </Step>
  <Step title="Setup Mintlify client">
    Accédez à votre Tableau de bord Mintlify et allez dans la section **Settings** > **Authentication**. Sélectionnez **OAuth** comme méthode d’authentification ; le formulaire de configuration OAuth s’affiche :
    
    <Frame>![Paramètres du client OAuth Mintlify](/images/guides/auth0/mintlifyoauthclientsettings.png)</Frame>
    
    Remplissez le formulaire avec les valeurs suivantes :
    
    - **Authorization URL**: `https://YOUR_AUTH0_DOMAIN/authorize` (remplacez `YOUR_AUTH0_DOMAIN` par votre domaine Auth0 réel indiqué à l’étape 2)
    - **Client ID**: Saisissez le Client ID de votre application Auth0
    - **Client Secret**: Saisissez le Client Secret de votre application Auth0
    - **Scopes**: Laissez vide, sauf si vous avez défini des scopes personnalisés dans Auth0
    - **Token URL**: `https://YOUR_AUTH0_DOMAIN/oauth/token` (remplacez `YOUR_AUTH0_DOMAIN` par votre domaine Auth0 réel)
    
    Après avoir renseigné ces informations, cliquez sur **Enregistrer les modifications** pour sauvegarder votre configuration OAuth.
  </Step>
  <Step title="Configure Callback URL">
    Mintlify générera une URL de redirection unique qu’Auth0 doit reconnaître pour que le flux OAuth fonctionne correctement.
    
    Copiez l’URL de redirection depuis les paramètres Authentication de votre Tableau de bord Mintlify :
    <Frame>![URL de redirection Mintlify](/images/guides/auth0/mintlifyredirecturl.png)</Frame>
    
    Revenez à la page des paramètres de votre application Auth0, faites défiler jusqu’à la section **Application URIs**, puis collez l’URL de redirection dans le champ **Allowed Callback URLs** :
    <Frame>![Configuration de l’URL de redirection Auth0](/images/guides/auth0/auth0redirecturl.png)</Frame>
    
    Cliquez sur **Save Changes** en bas de la page Auth0 pour appliquer cette configuration.
  </Step>
</Steps>
