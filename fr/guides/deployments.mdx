Votre site de documentation est automatiquement déployé lorsque vous poussez des modifications vers votre référentiel connecté. Cela nécessite que la GitHub App Mintlify soit correctement installée et connectée.

Si vos dernières modifications n’apparaissent pas sur votre site en production, vérifiez d’abord que la GitHub App est installée sur le compte ou l’organisation propriétaire de votre référentiel de documentation. Consultez la page [Dépannage GitHub](/fr/settings/github#troubleshooting) pour plus d’informations.

Si la GitHub App est connectée mais que les modifications ne se déploient toujours pas, vous pouvez déclencher manuellement une reconstruction depuis votre Dashboard.

<div id="manually-triggering-a-deployment">
  ## Déclencher manuellement un déploiement
</div>

<Steps>
  <Step title="Vérifiez que votre dernier commit s’est bien passé.">
    Assurez-vous que votre dernier commit apparaît dans votre référentiel de documentation et qu’aucune erreur ne s’est produite.
  </Step>

  <Step title="Déclenchez manuellement un déploiement.">
    Rendez-vous sur votre [Dashboard](https://dashboard.mintlify.com) et cliquez sur le bouton de déploiement.

  </Step>
</Steps>
