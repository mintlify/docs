Votre site de documentation est automatiquement déployé lorsque vous poussez des modifications vers votre dépôt connecté. Pour cela, l’application GitHub Mintlify doit être correctement installée et connectée.

Si vos dernières modifications n’apparaissent pas sur votre site en production, vérifiez d’abord que l’application GitHub est installée sur le compte ou l’organisation propriétaire de votre dépôt de documentation. Consultez la page [Dépannage GitHub](/fr/settings/github#troubleshooting) pour en savoir plus.

Si l’application GitHub est connectée mais que les modifications ne se déploient toujours pas, vous pouvez déclencher manuellement une reconstruction depuis votre tableau de bord.

<div id="manually-triggering-a-deployment">
  ## Déclencher un déploiement manuellement
</div>

<Steps>
  <Step title="Vérifiez que votre dernier commit a abouti.">
    Assurez-vous que votre dernier commit apparaît dans votre dépôt de documentation et qu’il n’a rencontré aucune erreur.
  </Step>

  <Step title="Déclenchez un déploiement manuellement.">
    Rendez-vous sur votre [tableau de bord](https://dashboard.mintlify.com) et cliquez sur le bouton de déploiement.

  </Step>
</Steps>
