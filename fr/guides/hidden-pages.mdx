Les pages masquées sont retirées de la navigation de votre site, mais restent accessibles publiquement à toute personne qui connaît leur URL.

Utilisez des pages masquées pour du contenu que vous souhaitez rendre accessible sur votre site ou référencer comme contexte pour des outils d’IA, sans qu’il soit découvrable via la navigation.

Pour le contenu nécessitant un contrôle d’accès strict, vous devez configurer l’[authentification](/fr/authentication-personalization/authentication-setup).

Si vous souhaitez masquer des pages pour certains groupes d’utilisateurs, utilisez la Personnalisation pour contrôler la [visibilité des pages](/fr/authentication-personalization/overview#page-visibility).

<div id="hiding-a-page">
  ## Masquer une page
</div>

Une page est masquée si elle n’est pas incluse dans la navigation définie dans votre `docs.json`. Pour masquer une page, retirez-la de votre structure de navigation.

Les pages masquées utilisent la même structure d’URL que les pages classiques, basée sur leur chemin de fichier. Par exemple, `guides/hidden-page.mdx` sera accessible à l’adresse `docs.yoursite.com/guides/hidden-page`.

Consultez un [exemple de page masquée](/fr/guides/hidden-page-example).

<Note>
  Certains éléments de navigation comme les barres latérales, les menus déroulants et les onglets peuvent apparaître vides ou provoquer un décalage de la mise en page sur les pages masquées.
</Note>

<div id="hiding-a-group-of-pages">
  ## Masquer un groupe de pages
</div>

Un groupe de pages est masqué si la propriété `hidden` est définie sur `true` dans votre fichier `docs.json` :

```json highlight={4}
"groups": [
  {
    "group": "Prise en main",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  },
  {
    "group": "Guides",
    "pages": [
      "guides/hidden-page.mdx",
      "guides/hidden-groups.mdx"
    ]
  }
]
```

Dans cet exemple, le groupe `Getting started` est masqué, mais le groupe `Guides` est visible.

<div id="hiding-a-tab">
  ### Masquer un onglet
</div>

Vous pouvez également masquer un onglet en ajoutant la propriété `hidden` à votre fichier `docs.json` :

```json highlight={4}
"tabs": [
  {
    "tab": "Accueil",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  }
]
```

<div id="search-seo-and-ai-indexing">
  ## Recherche, SEO et indexation par l’IA
</div>

Par défaut, les pages masquées sont exclues de l’indexation par les moteurs de recherche, de la recherche interne dans vos docs et du contexte utilisé par l’Assistant. Pour inclure les pages masquées dans les résultats de recherche et comme contexte pour l’Assistant, ajoutez la propriété `seo` à votre `docs.json` :

```json
"seo": {
    "indexing": "all"
}
```

Pour exclure une page en particulier, ajoutez `noindex: true` à ses métadonnées frontmatter.
