---

Améliorez votre documentation à partir de données quantitatives et qualitatives disponibles sur votre page d’insights.

<div id="correlate-traffic-and-satisfaction">
  ## Mettre en corrélation trafic et satisfaction
</div>

- **Trafic élevé et faibles notes de feedback** : Pages populaires avec une mauvaise expérience utilisateur. Priorisez l’amélioration de ces pages.
- **Faible trafic et notes de feedback élevées** : Documentation efficace, mais peut-être difficile à trouver. Envisagez de promouvoir ces pages.
- **Trafic élevé et notes de feedback élevées** : Les incontournables de votre documentation. Passez en revue ces pages pour trouver des idées afin d’améliorer le reste de votre documentation.

<div id="put-insights-into-action">
  ## Transformez les insights en actions
</div>

Utilisez ces approches pour hiérarchiser les améliorations de votre documentation :

- **Corrigez d’abord les problèmes à fort impact** : les pages populaires avec de faibles scores de feedback affectent le plus grand nombre d’utilisateurs.
- **Répondez aux retours des utilisateurs** : les commentaires contextuels et sur des extraits de code peuvent mettre en évidence des zones spécifiques à améliorer.
