Mintlify propose trois niveaux d'accès au Dashboard : Lecteur, Éditeur et Admin.

## Autorisations

Les actions suivantes sont limitées par rôle :

|                         | Lecteur | Éditeur | Admin |
| ----------------------- | :-----: | :-----: | :---: |
| Voir la documentation   |   ✅    |   ✅    |  ✅   |
| Modifier la documentation |   ❌    |   ✅    |  ✅   |
| Mettre à jour les rôles des utilisateurs |   ❌   |   ❌   |  ✅   |
| Supprimer des utilisateurs |   ❌   |   ❌   |  ✅   |
| Inviter des administrateurs |   ❌   |   ❌   |  ✅   |
| Gérer et mettre à jour la facturation |   ❌   |   ❌   |  ✅   |
| Mettre à jour le domain personnalisé |   ❌   |   ❌   |  ✅   |
| Mettre à jour la source Git |   ❌   |   ❌   |  ✅   |
| Supprimer l'organisation |   ❌   |   ❌   |  ✅   |

### Rôle Lecteur

Le rôle Lecteur fournit un accès en lecture seule au Dashboard. Les utilisateurs avec ce rôle peuvent consulter la documentation et les analyses, mais ne peuvent apporter aucune modification au contenu ou aux paramètres.

### Rôle Éditeur

Le rôle Éditeur permet aux utilisateurs de consulter et de modifier le contenu de la documentation. Les éditeurs peuvent apporter des modifications à la documentation, mais ne peuvent pas gérer les utilisateurs, la facturation ou les paramètres de l'organisation.

### Rôle Admin

Le rôle Admin dispose d'un accès complet à toutes les fonctionnalités du Dashboard, y compris la gestion des utilisateurs, la facturation et les paramètres de l'organisation.

## Règles d'escalade des privilèges

Lorsque le RBAC est activé, les utilisateurs ne peuvent pas inviter d'autres personnes avec des rôles qu'ils n'ont pas eux-mêmes. Par exemple :

- Un Lecteur ne peut inviter aucun utilisateur
- Un Éditeur ne peut pas inviter d'Admins
- Un Admin peut inviter des utilisateurs avec n'importe quel rôle

Vous pouvez inviter autant d'administrateurs que vous le souhaitez, mais nous recommandons de limiter l'accès administrateur aux utilisateurs qui en ont besoin.
