---
title: "Fichiers"
description: "Diffuser des ressources statiques à partir de votre documentation"
---

Mintlify diffuse automatiquement les ressources statiques depuis votre référentiel de documentation à l’emplacement approprié sur votre domain. Par exemple, si vous avez `/images/my-logo.png` dans votre référentiel, le fichier image est accessible à l’adresse `https://docs.your-project.com/images/my-logo.png`.

Cela fonctionne pour tout type de fichier que vous souhaitez mettre à disposition de vos utilisateurs, y compris les images, les PDF, les vidéos ou les schémas.

Les fichiers doivent peser moins de 20 Mo pour les images et 100 Mo pour les autres types de fichiers.

<div id="supported-file-extensions">
  ## Extensions de fichiers prises en charge
</div>

Mintlify prend en charge les extensions de fichiers statiques suivantes.

<div id="image-formats">
  ### Formats d’images
</div>

- `.jpeg`, `.jpg`, `.jfif`, `.pjpeg`, `.pjp`
- `.png`
- `.svg`, `.svgz`
- `.ico`
- `.webp`
- `.gif`, `.apng`
- `.avif`
- `.bmp`
- `.tiff`, `.tif`

<div id="video-formats">
  ### Formats vidéo
</div>

- `.mp4`
- `.webm`
- `.ogg`
- `.avi`
- `.mov`
- `.wmv`
- `.flv`
- `.mkv`
- `.m4v`

<div id="audio-formats">
  ### Formats audio
</div>

- `.mp3`

- `.aac`

<div id="document-formats">

<div id="data-formats">

<div id="file-organization">
  ## Organisation des fichiers
</div>

Organisez vos fichiers à l’aide de dossiers pour que votre référentiel reste facile à naviguer :

```text
/votre-projet
  |- docs.json
  |- images/
    |- logo.png
    |- screenshots/
      |- dashboard.png
  |- assets/
    |- whitepaper.pdf
    |- demo-video.mp4
```

Les fichiers sont servis depuis la racine de votre domain, de sorte que la structure de votre référentiel corresponde directement à la structure des URL. Dans l’exemple précédent, `assets/whitepaper.pdf` serait accessible à l’adresse `https://docs.your-project.com/assets/whitepaper.pdf`.
