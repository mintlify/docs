description: "Outils pour prévenir les liens invalides"
---

Lorsque vous modifiez le chemin d’un fichier dans votre dossier docs, l’URL de la page change également. Cela peut arriver lors de la refonte de votre documentation ou de la modification du titre de la barre latérale.

<div id="broken-links">
  ## Liens cassés
</div>

Détectez les liens cassés avec notre interface en ligne de commande (CLI). [Installez la CLI](/fr/installation) et exécutez la commande :

```bash
mint broken-links
```

L’interface en ligne de commande (CLI) détecte tous les liens relatifs inexistants dans votre documentation.

<div id="redirects">
  ## Redirections
</div>

Configurez des redirections 301 en ajoutant le champ `redirects` à votre fichier `docs.json`.

```json
"redirects": [
  {
    "source": "/chemin/source",
    "destination": "/chemin/destination"
  }
]
```

Cela redirige de manière permanente `/source/path` vers `/destination/path`, afin de ne pas perdre le référencement (SEO) antérieur de la page d’origine.

Pour faire correspondre un chemin avec un caractère générique (wildcard), utilisez `*` après un paramètre. Dans cet exemple, `/beta/:slug*` correspond à `/beta/introduction` et le redirige vers `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
