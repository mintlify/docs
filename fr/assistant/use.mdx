Les utilisateurs disposent de plusieurs moyens pour démarrer une conversation avec l'assistant. Chaque méthode ouvre un panneau de chat sur le côté droit de votre documentation. Les utilisateurs peuvent poser n'importe quelle question et l'assistant recherche une réponse dans votre documentation. Si l'assistant ne parvient pas à récupérer des informations pertinentes, il répond qu'il ne peut pas répondre à la question.

<div id="use-the-assistant-in-local-preview">
  ## Utiliser l'assistant en prévisualisation locale
</div>

L'assistant est disponible dans votre prévisualisation locale lorsque vous vous connectez avec la CLI. Cela vous permet de tester le comportement de l'assistant, les instructions personnalisées dans `Assistant.md` et les réponses pendant que vous développez votre documentation localement.

1. Si ce n'est pas déjà fait, installez la CLI avec `npm i -g mint`.
2. Exécutez `mint login` pour vous authentifier.
3. Exécutez `mint dev` pour démarrer la prévisualisation locale.

L'assistant en prévisualisation locale utilise le même contenu indexé que votre site de documentation déployé.

<div id="keyboard-shortcut">
  ## Raccourci clavier
</div>

Ouvrez le panneau de chat de l'assistant avec le raccourci clavier <kbd>Command</kbd> + <kbd>I</kbd> sur macOS et <kbd>Ctrl</kbd> + <kbd>I</kbd> sur Windows.

<div id="highlight-text">
  ## Surligner du texte
</div>

Surlignez du texte sur une page et cliquez sur le bouton contextuel **Add to assistant** pour ouvrir le panneau de chat de l'assistant et ajouter le texte surligné comme contexte. Vous pouvez ajouter plusieurs extraits de texte ou blocs de code au contexte de l'assistant.

  ## Blocs de code
</div>

<div id="file-attachments">
  ## Pièces jointes
</div>

Joignez des fichiers aux messages de l'assistant pour fournir un contexte supplémentaire.

Types de fichiers pris en charge :
- **Images** : JPEG, PNG, GIF, WebP, SVG
- **Documents** : PDF
- **Fichiers de code et texte** : JavaScript (`.js`, `.jsx`, `.mjs`, `.cjs`), TypeScript (`.ts`, `.tsx`), Python, HTML, CSS, Markdown, MDX, JSON, YAML, XML, SQL, CSV, texte brut, scripts shell (`.sh`, `.bash`, `.env`), GraphQL, TOML, Go, Rust, Ruby, Java, Kotlin, Swift, C (`.c`, `.h`), C++ (`.cpp`, `.hpp`), C#, PHP, Lua, R, Scala

Limites :
- Taille maximale du fichier : 5 Mo par pièce jointe
- Nombre maximal de pièces jointes par message : 10

<div id="urls">
  ## URLs
</div>

Ouvrez l'assistant avec un paramètre de requête URL pour créer des liens profonds qui guident les utilisateurs vers des informations spécifiques ou partager des conversations de l'assistant avec des questions préremplies.

- **Ouvrir l'assistant** : Ajoutez `?assistant=open` pour ouvrir le panneau de chat de l'assistant au chargement de la page.
  - Exemple : [https://mintlify.com/docs?assistant=open](https://mintlify.com/docs?assistant=open)
- **Ouvrir avec une requête préremplie** : Ajoutez `?assistant=VOTRE_REQUÊTE` pour ouvrir l'assistant et soumettre automatiquement une question.
  - Exemple : [https://mintlify.com/docs?assistant=explain webhooks](https://mintlify.com/docs?assistant=explain%20webhooks)

<div id="troubleshooting">
  ## Dépannage
</div>

<Accordion title="La barre de chat de l'assistant n'est pas visible">
  Si l'interface de l'assistant n'est pas visible dans certains navigateurs, vous devrez peut-être signaler un faux positif à [EasyList](https://easylist.to). Les navigateurs qui utilisent la EasyList Cookies List, comme Brave et Comet, bloquent parfois l'assistant ou d'autres éléments de l'interface. La EasyList Cookies List inclut une règle spécifique au domaine qui masque les éléments fixes sur certains domaines pour bloquer les bannières de cookies. Cette règle affecte par inadvertance des composants d'interface légitimes.

  Signalez un faux positif à [EasyList](https://github.com/easylist/easylist) pour demander la suppression de la règle. Cela résout le problème pour tous les utilisateurs une fois la liste de filtres mise à jour.

</Accordion>
