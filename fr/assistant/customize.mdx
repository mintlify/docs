---

Ajoutez un fichier `.mintlify/Assistant.md` à la racine de votre projet de documentation pour donner à l'assistant des instructions personnalisées qui façonnent ses réponses (`ASSISTANT.md` est également un nom de fichier accepté). L'assistant utilise ces instructions comme contexte au niveau du système pour chaque réponse.

<Warning>
  Placez votre fichier `Assistant.md` dans le répertoire `.mintlify/`, et non à la racine de votre projet. Si vous stockez le fichier à la racine de votre projet, vos instructions personnalisées sont accessibles publiquement via le chemin `/assistant.md`. Le répertoire `.mintlify/` n'est pas servi publiquement, donc votre fichier `Assistant.md` n'est pas accessible lorsqu'il se trouve dans ce répertoire.
  
  Si vous devez stocker votre fichier `Assistant.md` à la racine de votre projet ou dans tout autre répertoire, ajoutez-le à votre fichier [`.mintignore`](/fr/organize/mintignore) pour l'exclure de votre site de documentation.
</Warning>

Les instructions dans un fichier `Assistant.md` s'ajoutent au prompt système de l'assistant. Elles ne remplacent ni ne substituent les instructions par défaut de l'assistant.

Pour supprimer les instructions personnalisées, supprimez le fichier `Assistant.md`.

Utilisez `Assistant.md` pour :

- Ajuster la personnalité et le ton de l'assistant
- Fournir un contexte spécifique au produit
- Définir des chemins d'escalade vers le support
- Spécifier des préférences terminologiques
- Restreindre ou délimiter les domaines d'expertise de l'assistant
- Fournir des conseils spécifiques à une version

<div id="example-assistantmd-file">
  ## Exemple de fichier `Assistant.md`
</div>

```markdown .mintlify/Assistant.md
You are a helpful assistant for Acme Corp's developer platform.

## Tone
- Be concise and direct. Getting to the point helps people more than being exhaustive.
- Use technical language appropriate for software developers. Most people asking you questions are familiar with Acme's products.

## Product context
- Acme SDK v3 is the current stable release. v2 is deprecated. If people ask about v2, remind them it's deprecated but answer their questions.
- Direct billing questions to support@acme.com.

## Terminology
- Use "workspace" instead of "project" or "organization".
- Use "access token" instead of "API key".
```
