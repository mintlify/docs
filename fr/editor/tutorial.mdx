---

Ce tutoriel vous guide à travers une mise à jour complète de contenu en utilisant l'éditeur. Vous allez créer une branche, effectuer une modification, partager un aperçu pour révision et publier vos modifications.

<Steps>
  <Step title="Créer une branche">
    Une branche est une copie temporaire de votre contenu. C'est là que vous effectuez vos modifications sans affecter votre site en direct.
     
    Pensez aux branches comme à des brouillons. Vous pouvez les fusionner dans votre site en direct ou les supprimer en toute sécurité si vous choisissez de ne pas effectuer les modifications.
    
    Travailler sur une branche permet de garder vos modifications séparées de votre site en direct jusqu'à ce que vous soyez prêt à partager le contenu avec vos utilisateurs.

    1. Dans la barre d'outils de l'éditeur, cliquez sur le nom de la branche (généralement `main`).
        <Frame>
          <img src="/images/editor/create-branch-light.png" alt="Nom de la branche mis en évidence dans la barre d'outils de l'éditeur." className="block dark:hidden" />
          <img src="/images/editor/create-branch-dark.png" alt="Nom de la branche mis en évidence dans la barre d'outils de l'éditeur." className="hidden dark:block" />
        </Frame>
    1. Cliquez sur **Créer une nouvelle branche**.
    1. Saisissez un nom décrivant votre modification, comme `update-quickstart`.
    1. Cliquez sur **Créer une branche**.

    L'éditeur passe à votre nouvelle branche. Toutes les modifications que vous apportez ne se trouvent que sur la branche. Elles n'apparaîtront sur votre site en direct que lorsque vous aurez terminé le processus de publication.
  </Step>
  <Step title="Effectuer vos modifications">
    Modifiez une page existante ou créez-en une nouvelle.

    **Pour modifier une page** : Cliquez dessus dans la barre latérale de navigation pour l'ouvrir. Tapez directement dans l'éditeur pour ajouter du contenu. Appuyez sur <kbd>/</kbd> pour insérer des composants, des images ou d'autres contenus.

    **Pour créer une page** : Cliquez sur le bouton plus <Icon icon="plus" /> à côté du groupe de navigation où vous souhaitez ajouter la page, puis cliquez sur **Ajouter une page** et saisissez un nom de fichier.

    <Frame>
      <img src="/images/editor/add-page-light.png" alt="Le menu d'ajout de page ouvert à côté d'un groupe dans l'éditeur." className="block dark:hidden" />
      <img src="/images/editor/add-page-dark.png" alt="Le menu d'ajout de page ouvert à côté d'un groupe dans l'éditeur." className="hidden dark:block" />
    </Frame>

  </Step>
  <Step title="Publier votre branche">
    Pour examiner vos modifications, vous devez créer une pull request. Une pull request est une proposition de fusionner vos modifications dans la branche principale à partir de laquelle votre site est construit.

    Lorsque vous publiez votre branche, vous la rendez disponible pour révision par votre équipe. Les modifications n'apparaissent sur votre site en direct que si vous fusionnez la pull request.
    
    1. Cliquez sur **Publier** dans la barre d'outils pour ouvrir le menu de publication.
        <Frame>
          <img src="/images/editor/publish-menu-light.png" alt="Le menu de publication ouvert dans la barre d'outils de l'éditeur." className="block dark:hidden" />
          <img src="/images/editor/publish-menu-dark.png" alt="Le menu de publication ouvert dans la barre d'outils de l'éditeur." className="hidden dark:block" />
        </Frame>
    1. Optionnellement, cliquez sur n'importe quel fichier modifié dans la liste pour visualiser et comparer vos modifications avec la version publiée.
    1. Cliquez sur **Enregistrer dans la branche** pour enregistrer toutes les modifications sur votre branche.
    1. Cliquez sur **Créer une pull request**. Ajoutez un titre et une description, puis cliquez sur **Publier la pull request**.
    1. Continuez vers GitHub pour visualiser votre pull request et examiner vos modifications.

  </Step>
  <Step title="Examiner vos modifications">
    Partagez votre pull request avec votre équipe pour examiner vos modifications.
  </Step>
  <Step title="Fusionner votre pull request">
    Lorsque vos modifications sont approuvées, fusionnez votre pull request dans la branche principale. Vos modifications sont déployées sur votre site en direct.
  </Step>
</Steps>
