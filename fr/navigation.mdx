La propriété [navigation](settings#param-navigation) de `docs.json` contrôle la structure et la hiérarchie d’informations de votre documentation.

Avec une configuration de navigation adaptée, vous pouvez organiser votre contenu pour que les utilisateurs trouvent exactement ce qu’ils recherchent.

<Info>
  N’utilisez pas `api` ou `mcp` comme titres pour des éléments de navigation. Les chemins `/api` et `/mcp` sont réservés en production et entraîneront des erreurs 404 si les URL contiennent `/api` ou `/mcp`.
</Info>

## Pages

Les pages sont l’élément de navigation le plus essentiel. Elles correspondent aux fichiers MDX qui composent votre documentation.

<img
  className="block dark:hidden pointer-events-none"
  src="/images/navigation/pages-light.png"
  alt=""
/>

<img
  className="hidden dark:block pointer-events-none"
  src="/images/navigation/pages-dark.png"
  alt=""
/>

Dans l’objet `navigation`, `pages` est un tableau où chaque entrée doit indiquer le chemin vers un [fichier de page](pages).

```json
{
  "navigation": {
    "pages": [
      "settings",
      "pages",
      "navigation",
      "themes",
      "custom-domain"
    ]
  }
}
```

## Groupes

Utilisez des groupes pour organiser la navigation de votre barre latérale en sections. Les groupes peuvent être imbriqués, étiquetés avec des tags et stylisés avec des icônes.

<img
  className="block dark:hidden pointer-events-none"
  src="/images/navigation/groups-light.png"
  alt=""
/>

<img
  className="hidden dark:block pointer-events-none"
  src="/images/navigation/groups-dark.png"
  alt=""
/>

Dans l’objet `navigation`, `groups` est un tableau dont chaque entrée est un objet contenant obligatoirement un champ `group` et un champ `pages`. Les champs `icon` et `tag` sont facultatifs.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Prise en main",
        "icon": "play",
        "pages": [
          "quickstart",
          {
            "group": "Édition",
            "icon": "pencil",
            "pages": [
              "installation",
              "editor"
            ]
          }
        ]
      },
      {
        "group": "Rédaction de contenu",
        "icon": "notebook-text",
        "tag": "NEW",
        "pages": [
          "writing-content/page",
          "writing-content/text"
        ]
      }
    ]
  }
}
```

## Onglets

Les onglets créent des sections distinctes de votre documentation avec des chemins d’URL séparés. Ils affichent une barre de navigation horizontale en haut de votre documentation qui permet aux utilisateurs de passer d’une section à l’autre.

<img
  className="block dark:hidden pointer-events-none"
  src="/images/navigation/tabs-light.png"
  alt=""
/>

<img
  className="hidden dark:block pointer-events-none"
  src="/images/navigation/tabs-dark.png"
  alt=""
/>

Dans l’objet `navigation`, `tabs` est un tableau où chaque entrée est un objet qui requiert un champ `tab` et peut contenir d’autres champs de navigation tels que des groupes, des pages, des icônes ou des liens vers des pages externes.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "tab": "SDKs",
        "icon": "code",
        "pages": [
          "sdk/fetch",
          "sdk/create",
          "sdk/delete"
        ]
      },
      {
        "tab": "Blog",
        "icon": "newspaper",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

### Menus

Les menus ajoutent des éléments de navigation déroulants à un onglet. Utilisez-les pour aider les utilisateurs à accéder directement à des pages spécifiques au sein d’un onglet.

Dans l’objet `navigation`, `menu` est un tableau où chaque entrée est un objet qui requiert un champ `item` et peut contenir d’autres champs de navigation tels que des groupes, des pages, des icônes ou des liens vers des pages externes.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Developer tools",
        "icon": "square-terminal",
        "menu": [
          {
            "item": "API reference",
            "icon": "rocket",
            "groups": [
              {
                "group": "Core endpoints",
                "icon": "square-terminal",
                "pages": [
                  "api-reference/get",
                  "api-reference/post",
                  "api-reference/delete"
                ]
              }
            ]
          },
          {
            "item": "SDKs",
            "icon": "code",
            "description": "Les SDK permettent d’interagir avec l’API.",
            "pages": [
              "sdk/fetch",
              "sdk/create",
              "sdk/delete"
            ]
          }
        ]
      }
    ]
  }
}
```

## Ancres

Les ancres ajoutent des éléments de navigation persistants en haut de votre barre latérale. Utilisez-les pour structurer votre contenu, offrir un accès rapide à des ressources externes ou créer des appels à l’action bien visibles.

<img
  className="block dark:hidden pointer-events-none"
  src="/images/navigation/anchors-light.png"
/>

<img
  className="hidden dark:block pointer-events-none"
  src="/images/navigation/anchors-dark.png"
/>

Dans l’objet `navigation`, `anchors` est un tableau dont chaque entrée est un objet nécessitant un champ `anchor`, et pouvant contenir d’autres champs de navigation tels que des groupes, des pages, des icônes ou des liens vers des pages externes.

```json
{
  "navigation": {
    "anchors": [
      {
        "anchor": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "anchor": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "anchor": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

Pour les ancres qui pointent uniquement vers des liens externes, utilisez le mot-clé `global`. Les ancres dans un objet `global` doivent avoir un champ `href` et ne peuvent pas pointer vers un chemin relatif.

Les ancres globales sont particulièrement utiles pour créer des liens vers des ressources qui ne font pas partie de votre documentation, mais qui doivent rester facilement accessibles à vos utilisateurs, comme un blog ou un portail d’assistance.

```json
{
  "navigation": {
    "global":  {
      "anchors": [
        {
          "anchor": "Community",
          "icon": "house",
          "href": "https://slack.com"
        },
        {
          "anchor": "Blog",
          "icon": "pencil",
          "href": "https://mintlify.com/blog"
        }
      ]
    },
    "tabs": /*...*/
  }
}
```

## Listes déroulantes

Les listes déroulantes se trouvent dans un menu extensible en haut de la navigation de votre barre latérale. Chaque élément d’une liste déroulante pointe vers une section de votre documentation.

<img
  className="block dark:hidden pointer-events-none"
  src="/images/navigation/dropdowns-light.png"
/>

<img
  className="hidden dark:block pointer-events-none"
  src="/images/navigation/dropdowns-dark.png"
/>

Dans l’objet `navigation`, `dropdowns` est un tableau dont chaque entrée est un objet qui requiert le champ `dropdown` et peut contenir d’autres champs de navigation, tels que des groupes, des pages, des icônes ou des liens vers des pages externes.

```json
{
  "navigation": {
    "dropdowns": [
      {
        "dropdown": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"

      },
      {
        "dropdown": "Référence API",
        "icon": "square-terminal",

        ]
      },

        "dropdown": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

## OpenAPI

Intégrez des spécifications OpenAPI directement dans votre structure de navigation pour générer automatiquement la documentation de l’API. Créez des sections d’API dédiées ou placez des pages d’endpoint au sein d’autres éléments de navigation.

Définissez une spécification OpenAPI par défaut à n’importe quel niveau de votre hiérarchie de navigation. Les éléments enfants hériteront de cette spécification, sauf s’ils définissent la leur.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "API reference",
        "openapi": "/path/to/openapi-v1.json",
        "pages": [
          "overview",
          "authentication",
          "GET /users",
          "POST /users",
          {
            "group": "Products",
            "openapi": "/path/to/openapi-v2.json",
            "pages": [
              "GET /products",
              "POST /products"
            ]
          }
        ]
      }
    ]
  }
}
```

Pour en savoir plus sur la référence d’endpoints OpenAPI dans votre documentation, consultez la page [OpenAPI setup](/fr/api-playground/openapi-setup).

## Versions

Divisez votre navigation en plusieurs versions. Les versions sont sélectionnables via un menu déroulant.

<img
  className="block dark:hidden pointer-events-none"
  src="/images/navigation/versions-light.png"
/>

<img
  className="hidden dark:block pointer-events-none"
  src="/images/navigation/versions-dark.png"
/>

Dans l’objet `navigation`, `versions` est un tableau où chaque entrée est un objet qui requiert un champ `version` et peut contenir n’importe quels autres champs de navigation.

```json
{
  "navigation": {
    "versions": [
      {
        "version": "1.0.0",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["v1/overview", "v1/quickstart", "v1/development"]
          }
        ]
      },
      {
        "version": "2.0.0",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["v2/overview", "v2/quickstart", "v2/development"]
          }
        ]
      }
    ]
  }
}
```

## Imbrication

Vous pouvez combiner librement des ancres, des onglets et des menus déroulants. Les composants peuvent être imbriqués les uns dans les autres de manière interchangeable pour créer la structure de navigation souhaitée.

<CodeGroup>

```json Anchors
{
  "navigation": {
    "anchors": [
      {
        "anchor": "Anchor 1",
        "groups": [
          {
            "group": "Group 1",
            "pages": [
              "some-folder/file-1",
              "another-folder/file-2",
              "just-a-file"
            ]
          }
        ]
      },
      {
        "anchor": "Anchor 2",
        "groups": [
          {
            "group": "Group 2",
            "pages": [
              "some-other-folder/file-1",
              "various-different-folders/file-2",
              "another-file"
            ]
          }
        ]
      }
    ]
  }
}
```

## Fil d’Ariane

Appliquez un fil d’Ariane pour afficher le chemin de navigation complet en haut des pages.

```json
{
  "styling": {
    "eyebrows": "breadcrumbs"
  }
}
```
