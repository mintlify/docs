La propriété [navigation](fr/settings#param-navigation) dans `docs.json` contrôle la structure et la hiérarchie de l’information de votre documentation.

Avec une configuration de navigation adéquate, vous pouvez organiser votre contenu pour que les utilisateurs trouvent exactement ce qu’ils recherchent.

<div id="pages">
  ## Pages
</div>

Les pages sont l’élément de navigation le plus fondamental. Elles correspondent aux fichiers MDX qui composent votre documentation.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/pages-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/pages-dark.png" alt="" />

Dans l’objet `navigation`, `pages` est un tableau dans lequel chaque entrée doit référencer le chemin d’un [fichier de page](fr/pages).

```json
{
  "navigation": {
    "pages": [
      "settings",
      "pages",
      "navigation",
      "themes",
      "custom-domain"
    ]
  }
}
```

<div id="groups">
  ## Groupes
</div>

Utilisez des groupes pour organiser la navigation de votre barre latérale en sections. Les groupes peuvent être imbriqués, étiquetés avec des tags et stylisés avec des icon.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/groups-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/groups-dark.png" alt="" />

Dans l’objet `navigation`, `groups` est un tableau où chaque entrée est un objet qui requiert un champ `group` et un champ `pages`. Les champs `icon`, `tag` et `expanded` sont facultatifs.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Premiers pas",
        "icon": "play",
        "expanded": false,
        "pages": [
          "quickstart",
          {
            "group": "Édition",
            "icon": "pencil",
            "pages": [
              "installation",
              "editor"
            ]
          }
        ]
      },
      {
        "group": "Rédaction de contenu",
        "icon": "notebook-text",
        "tag": "NOUVEAU",
        "pages": [
          "writing-content/page",
          "writing-content/text"
        ]
      }
    ]
  }
}
```

<div id="default-expanded-state">
  ### État développé par défaut
</div>

Définissez `expanded: true` sur un groupe pour qu’il soit développé par défaut dans la barre latérale de navigation. C’est utile pour mettre en avant des sections importantes ou améliorer la découverte des contenus clés.

```json
{
  "group": "Prise en main",
  "expanded": true,
  "pages": ["quickstart", "installation"]
}
```

<div id="tabs">
  ## Tabs
</div>

Les Tabs créent des sections distinctes de votre documentation avec des chemins d’URL séparés. Elles créent une barre de navigation horizontale en haut de votre documentation qui permet aux utilisateurs de passer d’une section à l’autre.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/tabs-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/tabs-dark.png" alt="" />

Dans l’objet `navigation`, `tabs` est un tableau où chaque entrée est un objet qui requiert un champ `tab` et peut contenir d’autres champs de navigation tels que des groups, des pages, des icon ou des liens vers des pages externes.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Référence API",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "tab": "SDK",
        "icon": "code",
        "pages": [
          "sdk/fetch",
          "sdk/create",
          "sdk/delete"
        ]
      },
      {
        "tab": "Blog",
        "icon": "newspaper",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="menus">
  ### Menus
</div>

Les menus ajoutent des éléments de navigation déroulants à un onglet. Utilisez les menus pour aider les utilisateurs à accéder directement à des pages spécifiques au sein d’un onglet.

Dans l’objet `navigation`, `menu` est un tableau où chaque entrée est un objet qui requiert un champ `item` et peut contenir d’autres champs de navigation tels que des groupes, des pages, des icons, ou des liens vers des pages externes.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Outils de développeur",
        "icon": "square-terminal",
        "menu": [
          {
            "item": "API reference",
            "icon": "rocket",
            "groups": [
              {
                "group": "Points de terminaison principaux",
                "icon": "square-terminal",
                "pages": [
                  "api-reference/get",
                  "api-reference/post",
                  "api-reference/delete"
                ]
              }
            ]
          },
          {
            "item": "SDKs",
            "icon": "code",
            "description": "Les SDK sont utilisés pour interagir avec l'API.",
            "pages": [
              "sdk/fetch",
              "sdk/create",
              "sdk/delete"
            ]
          }
        ]
      }
    ]
  }
}
```

<div id="anchors">
  ## Ancres
</div>

Les ancres ajoutent des éléments de navigation persistants en haut de votre barre latérale. Utilisez-les pour structurer votre contenu, offrir un accès rapide à des ressources externes ou créer des appels à l’action visibles.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/anchors-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/anchors-dark.png" />

Dans l’objet `navigation`, `anchors` est un tableau où chaque entrée est un objet qui requiert un champ `anchor` et peut contenir d’autres champs de navigation, tels que des groupes, des pages, des icônes ou des liens vers des pages externes.

```json
{
  "navigation": {
    "anchors": [
      {
        "anchor": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "anchor": "Référence API",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "anchor": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

Pour les ancres qui renvoient uniquement vers des liens externes, utilisez le mot-clé `global`. Les ancres dans un objet `global` doivent comporter un champ `href` et ne peuvent pas pointer vers un chemin relatif.

Les ancres globales sont particulièrement utiles pour lier des ressources qui ne font pas partie de votre documentation, mais qui devraient rester facilement accessibles à vos utilisateurs, comme un blog ou un portail d’assistance.

```json
{
  "navigation": {
    "global":  {
      "anchors": [
        {
          "anchor": "Communauté",
          "icon": "house",
          "href": "https://slack.com"
        },
        {
          "anchor": "Blog",
          "icon": "pencil",
          "href": "https://mintlify.com/blog"
        }
      ]
    },
    "tabs": /*...*/
  }
}
```

<div id="dropdowns">
  ## Menus déroulants
</div>

Les menus déroulants se trouvent dans un menu extensible en haut de la barre latérale de votre navigation. Chaque élément d’un menu déroulant renvoie vers une section de votre documentation.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/dropdowns-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/dropdowns-dark.png" />

Dans l’objet `navigation`, `dropdowns` est un tableau dont chaque entrée est un objet qui requiert un champ `dropdown` et peut contenir d’autres champs de navigation, tels que des groupes, des pages, des icon ou des liens vers des pages externes.

```json
{
  "navigation": {
    "dropdowns": [
      {
        "dropdown": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "dropdown": "Référence API",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "dropdown": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="openapi">
  ## OpenAPI
</div>

Intégrez des spécifications OpenAPI directement dans votre structure de navigation pour générer automatiquement la documentation de votre API. Créez des sections d’API dédiées ou placez des pages d’endpoints au sein d’autres composants de navigation.

Définissez une spécification OpenAPI par défaut à n’importe quel niveau de votre hiérarchie de navigation. Les éléments enfants hériteront de cette spécification, sauf s’ils définissent la leur.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Référence API",
        "openapi": "/path/to/openapi-v1.json",
        "pages": [
          "overview",
          "authentication",
          "GET /users",
          "POST /users",
          {
            "group": "Produits",
            "openapi": "/path/to/openapi-v2.json",
            "pages": [
              "GET /products",
              "POST /products"
            ]
          }
        ]
      }
    ]
  }
}
```

Pour en savoir plus sur la façon de référencer des endpoints OpenAPI dans votre documentation, consultez la page [OpenAPI setup](/fr/api-playground/openapi-setup).

<div id="versions">
  ## Versions
</div>

Divisez votre navigation en différentes versions. Les versions peuvent être sélectionnées à partir d’un menu déroulant.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/versions-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/versions-dark.png" />

Dans l’objet `navigation`, `versions` est un tableau où chaque entrée est un objet qui doit inclure un champ `version` et peut contenir tout autre champ de navigation.

```json
{
  "navigation": {
    "versions": [
      {
        "version": "1.0.0",
        "groups": [
          {
            "group": "Premiers pas",
            "pages": ["v1/overview", "v1/quickstart", "v1/development"]
          }
        ]
      },
      {
        "version": "2.0.0",
        "groups": [
          {
            "group": "Premiers pas",
            "pages": ["v2/overview", "v2/quickstart", "v2/development"]
          }
        ]
      }
    ]
  }
}
```

Vous pouvez utiliser n’importe quelle combinaison d’ancres, d’onglets et de menus déroulants. Les composants peuvent être imbriqués les uns dans les autres de manière interchangeable pour créer la structure de navigation souhaitée.

                "some-folder/file-1",
                "another-folder/file-2",

          ]
        },
        {
          "anchor": "Anchor 2",
          "groups": [
            {
              "group": "Group 2",

<div id="breadcrumbs">
  ## Fils d’Ariane
</div>

Les fils d’Ariane affichent le chemin de navigation complet en haut des pages. Certains thèmes les activent par défaut, d’autres non. Vous pouvez contrôler leur activation pour votre site à l’aide de la propriété `styling` dans votre `docs.json`.

<CodeGroup>
  ```json Afficher les fils d’Ariane complets
  "styling": {
    "eyebrows": "breadcrumbs"
  }
  ```

  ```json Afficher uniquement la section parente
  "styling": {
    "eyebrows": "section"
  }
  ```
</CodeGroup>

<div id="interaction-configuration">
  ## Configuration des interactions
</div>

Déterminez la façon dont les utilisateurs interagissent avec les éléments de navigation à l’aide de la propriété `interaction` dans votre `docs.json`.

<div id="enable-auto-navigation-for-groups">
  ### Activer la navigation automatique pour les groupes
</div>

Lorsqu’un utilisateur déploie un groupe de navigation, certains thèmes ouvrent automatiquement la première page du groupe. Vous pouvez remplacer le comportement par défaut du thème à l’aide de l’option `drilldown`.

* Définissez `true` pour forcer la navigation automatique vers la première page lorsqu’un groupe de navigation est sélectionné.
* Définissez `false` pour empêcher la navigation et uniquement développer ou réduire le groupe lorsqu’il est sélectionné.
* Laissez la valeur non définie pour utiliser le comportement par défaut du thème.

<CodeGroup>
  ```json Force navigation
  "interaction": {
    "drilldown": true  // Forcer la navigation vers la première page lorsqu’un utilisateur déploie un menu déroulant
  }
  ```

  ```json Prevent navigation
  "interaction": {
    "drilldown": false // Ne jamais naviguer, uniquement développer ou réduire le groupe
  }
  ```
</CodeGroup>
