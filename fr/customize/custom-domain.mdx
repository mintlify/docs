Pour héberger votre documentation sur un domaine personnalisé, définissez le domaine personnalisé souhaité dans vos paramètres Mintlify et configurez vos enregistrements DNS auprès de votre fournisseur de domaine.

<Info>
  Vous souhaitez configurer un sous-répertoire personnalisé comme `mintlify.com/docs` ? Consultez la documentation sur le [sous-répertoire personnalisé](/fr/deploy/cloudflare).
</Info>

<div id="dashboard-settings">
  ## Paramètres du Dashboard
</div>

1. Accédez à votre [Dashboard](https://dashboard.mintlify.com).
2. Sélectionnez **Settings**.
3. Sélectionnez **Domain Setup**.
4. Saisissez le domaine personnalisé de votre choix. Par exemple, `docs.mintlify.com`.

<div id="configure-your-dns">
  ## Configurez votre DNS
</div>

1. Accédez aux paramètres DNS de votre domaine sur le site de votre fournisseur de domaine.
2. Créez une nouvelle entrée DNS avec les valeurs suivantes :

```bash
CNAME | docs | cname.vercel-dns.com.
```

<div id="provider-specific-settings">
  ## Paramètres spécifiques au fournisseur
</div>

<AccordionGroup>
  <Accordion title="Vérification Vercel">
    Si Vercel est votre fournisseur de domain, vous devez ajouter un enregistrement `TXT` de vérification. Ces informations s’affichent sur votre Dashboard après l’envoi de votre domain personnalisé et vous sont également envoyées par e‑mail.
  </Accordion>

  <Accordion title="Paramètres de sécurité Cloudflare">
    Si Cloudflare est votre fournisseur DNS, vous devez activer l’option de sécurité « full strict » pour le paramètre HTTPS.
  </Accordion>
</AccordionGroup>

<div id="set-a-canonical-url">
  ## Définir une URL canonique
</div>

Après avoir configuré votre DNS, définissez une URL canonique pour indiquer aux moteurs de recherche d’indexer votre domaine préféré. Une URL canonique précise aux moteurs de recherche quelle version de votre documentation est la principale. Cela améliore le SEO lorsque votre documentation est accessible via plusieurs URL et évite les problèmes de contenu dupliqué.

Ajoutez la balise meta `canonical` à votre `docs.json` :

```json
"seo": {
    "metatags": {
        "canonical": "https://www.your-custom-domain-here.com"
    }
}
```

Remplacez `https://www.your-custom-domain-here.com` par votre domain personnalisé réel. Par exemple, si votre domain personnalisé est `docs.mintlify.com`, vous utiliseriez :

```json
"seo": {
    "metatags": {
        "canonical": "https://docs.mintlify.com"
    }
}
```
