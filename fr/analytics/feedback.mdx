<Info>
  Pour collecter et consulter les retours, vous devez d’abord activer le feedback depuis la page [Modules complémentaires](https://dashboard.mintlify.com/products/addons) de votre Dashboard.
</Info>

L’onglet Feedback affiche les votes quantitatifs (pouce levé et pouce baissé) reçus par vos docs, ainsi que tout retour qualitatif fourni par les utilisateurs. Servez‑vous de ces informations pour évaluer la qualité de vos docs et apporter des améliorations.

Accédez à l’onglet Feedback en vous rendant sur la page **Analytics** de votre [Dashboard](https://dashboard.mintlify.com/products/analytics).

<div id="feedback-types">

</div>

<Note>

</Note>

L’onglet Feedback affiche des informations selon les modules complémentaires de feedback que vous avez activés.

* **Feedback sur les extraits de code uniquement** : Feedback spécifiquement sur les extraits de code.
* **Évaluation par pouces et feedback contextuel** : Vote sur la page, plus commentaires détaillés et motifs des évaluations.
* **Évaluation par pouces et feedback sur les extraits de code** : Vote sur la page, plus feedback sur les exemples de code.
* **Évaluation par pouces, contextuel, et feedback sur les extraits de code** : Système de feedback complet avec vote sur la page, commentaires détaillés et feedback sur les extraits de code.

<div id="managing-feedback">
  ## Gestion des retours
</div>

Pour les retours contextuels et sur des extraits de code, vous pouvez définir le statut d’un retour et ajouter des notes internes pour suivre l’avancement de la résolution des retours des utilisateurs.

<div id="changing-feedback-status">
  ### Modifier le statut d’un feedback
</div>

Sélectionnez le statut à côté d’un feedback pour le marquer comme **En attente**, **En cours**, **Résolu** ou **Ignoré**.

Bonnes pratiques pour définir les statuts de feedback :

* **En attente** : le feedback est en attente de revue.
* **En cours** : le feedback a été validé et est en cours de traitement.
* **Résolu** : le feedback a été résolu.
* **Ignoré** : le feedback a été ignoré car non actionnable, non pertinent ou inexact.

<div id="filtering-by-status">
  ### Filtrer par statut
</div>

Utilisez le filtre d’état pour contrôler les commentaires affichés. Désélectionnez un état pour masquer tous les commentaires ayant cet état. Par défaut, tous les commentaires sont affichés.

<div id="adding-internal-notes">
  ### Ajouter des notes internes
</div>

Cliquez sur un feedback pour ajouter une note interne. Ces notes ne sont visibles que par les personnes ayant accès à votre Dashboard.

Utilisez les notes pour ajouter des informations utiles à la collaboration, lier des tickets de support ou d’ingénierie pertinents, ou conserver toute autre information utile.

<div id="using-feedback-data">
  ## Utiliser les données de feedback
</div>

Examinez vos données de feedback pour :

* **Identifier le contenu performant** : les pages recevant le plus d’avis positifs montrent ce qui fonctionne bien dans votre documentation.
  * **Prioriser les améliorations** : les pages avec le plus d’avis négatifs indiquent quels contenus nécessitent une attention particulière.
  * **Passer à l’action** : mettez à jour la documentation sur la base des retours directs des utilisateurs.
