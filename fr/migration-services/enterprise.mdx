Vous pouvez faire appel à nos services de migration pour transférer votre documentation sur Mintlify. Ce guide couvre le processus de migration, les délais de livraison et notre approche d’assurance qualité.

<div id="what-to-expect-for-your-migration">
  ## À quoi vous attendre lors de votre migration
</div>

Une migration est un transfert ponctuel de votre documentation qui nécessite un gel du code. Nous migrons votre documentation, vos pages de référence API, votre image de marque (logos, couleurs et polices) et la structure de votre navigation. Les services de migration incluent également du conseil en design ou en architecture de l’information, ainsi que le développement de composants personnalisés, tels que des pages d’accueil ou des intégrations personnalisées.

Votre migration inclut un lien de préproduction pour suivre l’avancement et un délai de 7 jours après la livraison pour partager vos retours et résoudre tout problème lié au contenu, aux ressources ou à l’image de marque.

<div id="migration-process">
  ## Processus de migration
</div>

Le processus de migration est très collaboratif. Nous travaillons en étroite collaboration avec votre équipe pour mettre en place des solutions et des builds personnalisés adaptés à vos besoins.

<Steps>
  <Step title="Soumettre le formulaire de migration">
    Soumettez le [formulaire de migration](https://mintlify.typeform.com/pro-migrations).

    Fournissez la source de votre documentation, les éléments de marque (logos, codes couleur, polices), les spécifications OpenAPI, la date de mise en production et la méthode de livraison préférée (pull request GitHub ou fichier zip).
  </Step>

  <Step title="Appel de lancement">
    Nous examinons votre demande et organisons un appel de lancement avec notre équipe de migration pour passer en revue votre configuration actuelle, comprendre les éventuelles limitations et répondre à toutes les questions de votre équipe.
  </Step>

  <Step title="Recevoir le périmètre et le calendrier (1 à 2 jours ouvrables)">
    Nous examinons et analysons le contexte de l’appel et fournissons un calendrier de livraison estimé dans un délai de 24 à 48 heures. Nous signalons tout composant personnalisé ou toute partie de votre migration nécessitant une discussion.
  </Step>

  <Step title="Examiner l’avancement pendant la migration">
    Nous lançons la migration et partageons un lien de préproduction afin que vous puissiez suivre l’avancement en temps réel.
  </Step>

  <Step title="Recevoir la livraison finale">
    Nous livrons votre migration via pull request GitHub ou fichier zip, selon votre préférence.
  </Step>

  <Step title="Fenêtre de retours (sept jours ouvrables)">
    Soumettez tous vos retours sur le contenu, les éléments ou l’image de marque. Nous traitons tous les problèmes inclus dans le périmètre dans le même délai.
  </Step>

  <Step title="Assistance pour la mise en production">
    Nous fournissons la documentation et les conseils nécessaires pour connecter votre référentiel, configurer les redirections et paramétrer votre domaine personnalisé.
  </Step>
</Steps>

<div id="quality-assurance">
  ## Assurance qualité
</div>

Nous travaillons avec des prestataires tiers rigoureusement sélectionnés et réalisons un audit qualité interne avant la livraison. Voici ce que nous vérifions :

<Card title="Exhaustivité du contenu" icon="file-check">
  * L’ensemble de la documentation issue de vos contenus d’origine a été migré.
  * Le contenu est correctement mis en forme avec les composants Mintlify.
  * La structure de navigation correspond à vos spécifications.

<Card title="Éléments visuels" icon="palette">

<Card title="Fonctionnalité" icon="rocket">

  * Aucun problème de performance.
</Card>

<div id="after-your-migration">
  ## Après votre migration
</div>

Une fois que nous aurons effectué la migration, vous devrez suivre les étapes suivantes pour lancer votre documentation :

1. [Connectez votre référentiel GitHub et terminez la configuration](/fr/quickstart).
2. [Configurez les redirections, si nécessaire](/fr/create/redirects).
3. [Configurez votre domaine personnalisé](/fr/customize/custom-domain).
4. **Testez avant la mise en production** - Avant de basculer depuis votre ancienne documentation :
   * Passez en revue toutes les pages via votre lien de préproduction.
   * Testez la navigation et la recherche.
   * Vérifiez que toutes les images et vidéos se chargent correctement.
   * Vérifiez que les code blocks s&#39;affichent correctement.
   * Testez sur mobile et sur ordinateur.
5. **Lancez.** Votre documentation est prête à être mise en ligne.

<Note>
  **Besoin d&#39;aide ?** Notre équipe de migration est disponible pour vous accompagner dans ces étapes. Contactez-nous via votre canal Slack dédié ou [support@mintlify.com](mailto:support@mintlify.com).
</Note>

<div id="frequently-asked-questions">
  ## Questions fréquemment posées
</div>

<AccordionGroup>
  <Accordion title="Combien de temps dure une migration ?">
    Nous livrons généralement les migrations en 2 à 6 semaines, selon la complexité. Vous recevrez un calendrier précis après l&#39;appel de lancement.
  </Accordion>

  <Accordion title="Puis-je demander des modifications après la livraison ?">
    Oui. Vous disposez d&#39;une période de 7 jours après la livraison pour nous faire part de vos retours et demander des modifications au contenu, aux ressources ou à l&#39;identité visuelle. Nous traitons tous les retours entrant dans le périmètre dans le même délai et collaborons avec vous jusqu&#39;à ce que vous soyez prêt à passer en production.
  </Accordion>

  <Accordion title="Que dois-je faire une fois la migration terminée ?">
    La migration transfère votre contenu, mais vous devrez :

    * Connecter votre référentiel de documentation à votre Tableau de bord Mintlify.
    * Configurer les redirections depuis les anciennes URL de votre documentation.
    * Configurer votre domaine personnalisé.

    Nous fournissons une documentation détaillée, étape par étape, ainsi qu&#39;une assistance pour chacune de ces étapes.
  </Accordion>

  <Accordion title="Puis-je personnaliser la structure de navigation ?">
    Oui. Vous pouvez fournir une architecture de l’information personnalisée et nous la mettrons en œuvre. Nous collaborons également avec vous sur le développement de composants personnalisés si nécessaire et pris en charge par la plateforme.
  </Accordion>

  <Accordion title="Que se passe-t-il si ma documentation change pendant la migration ?">
    Comme les migrations nécessitent un gel du code, toute modification apportée à votre documentation source après la soumission ne sera pas incluse. Nous recommandons de terminer toutes les mises à jour avant de commencer la migration.

    Si cela n&#39;est pas possible, notre équipe travaillera avec vous pour planifier de courts sprints afin de migrer certaines sections ou de réserver un créneau dédié après le premier sprint pour migrer les modifications supplémentaires effectuées pendant la migration.
  </Accordion>
</AccordionGroup>
