Les migrations Enterprise sont un transfert complet de votre documentation existante vers Mintlify, piloté de bout en bout par notre équipe de migration. Cette page présente ce qui est inclus, les étapes du processus et notre approche du contrôle qualité, afin que vous sachiez à quoi vous attendre avant de commencer.

<div id="whats-included">
  ## Ce qui est inclus
</div>

Une migration est un transfert ponctuel qui nécessite un gel du code sur votre documentation source. Votre migration couvre :

- **Contenu** : toutes les pages de votre documentation source, reformatées avec les composants Mintlify.
- **Référence d'API** : pages de référence générées à partir des spécifications OpenAPI que vous fournissez.
- **Identité de marque** : logos, palettes de couleurs et polices appliqués pour correspondre à votre marque.
- **Navigation** : une architecture de l'information conçue avec votre équipe pendant la phase de cadrage.
- **Conseil** : accompagnement sur le design et l'architecture de l'information, ainsi que du travail sur des composants personnalisés comme des pages d'accueil ou des intégrations sur mesure lorsque la plateforme le permet.

Pour demander une migration, envoyez le [formulaire de migration](https://mintlify.typeform.com/ent-migrations) ou contactez votre interlocuteur Mintlify.

<div id="migration-process">
  ## Processus de migration
</div>

Le processus de migration est hautement collaboratif et piloté de bout en bout par l'équipe Mintlify. Nous travaillons en étroite collaboration avec votre équipe pour vous assurer une base solide au démarrage sur Mintlify.

<Steps>
  <Step title="Cadrage">
    Soumettez le [formulaire de migration](https://mintlify.typeform.com/ent-migrations) avec la source de votre documentation, vos ressources de marque (logos, codes couleur, polices), vos spécifications OpenAPI et votre date de mise en production.

    Nous animons une réunion de lancement pour rassembler votre contenu et déterminer votre architecture de l'information, puis nous vous fournissons un délai de livraison estimé. Nous signalerons tous les points de votre migration qui nécessitent discussion.
  </Step>

  <Step title="Reprise du contenu">
    Nous assemblons votre contenu dans un site Mintlify, effectuons un contrôle qualité du contenu et partageons un lien d'aperçu afin que vous puissiez suivre l'avancement en temps réel.
  </Step>

  <Step title="Fenêtre de retours (sept jours ouvrés)">
    Passez en revue la reprise du contenu et transmettez vos retours sur le contenu, les ressources ou l'identité de marque. Nous traitons tous les points relevant du périmètre dans le même délai.
  </Step>

  <Step title="Configurez votre tableau de bord">
    Nous livrons votre contenu migré sous forme de fichier ZIP. Ajoutez le ZIP de contenu à votre dépôt et effectuez les derniers ajustements avec votre équipe. Utilisez la [checklist de mise en production](/fr/migration-services/go-live-checklist) pour configurer votre tableau de bord.
  </Step>

  <Step title="Mise en production">
    Rattachez votre [domaine personnalisé](/fr/customize/custom-domain) pour lancer votre documentation.
  </Step>
</Steps>

<div id="quality-assurance">
  ## Contrôle qualité
</div>

Nous collaborons avec des prestataires tiers vérifiés et menons un audit qualité interne avant la livraison. Voici ce que nous vérifions :

<Card title="Complétude du contenu" icon="file-check">
  - Toute la documentation issue de votre source est présente dans votre nouvelle documentation.
  - Le contenu est correctement mis en forme avec les composants Mintlify.
  - La structure de navigation correspond à vos spécifications.
</Card>

<Card title="Éléments visuels" icon="palette">
  - Les images et vidéos s'affichent correctement.
  - Les polices personnalisées sont appliquées partout.
  - Les logos pour les modes clair et sombre s'affichent correctement.
  - Les schémas de couleur respectent les exigences de contraste dans les deux modes.
</Card>

<Card title="Fonctionnalités" icon="rocket">
  - Tous les liens fonctionnent correctement (aucun lien cassé).
  - La table des matières s'affiche correctement.
  - Les composants React fonctionnent comme prévu.
  - Aucun problème de performance.
</Card>

<Note>
  **Besoin d'aide ?** Notre équipe de migration est disponible pour vous guider à travers ces étapes. Contactez-nous via votre canal Slack dédié ou à [support@mintlify.com](mailto:support@mintlify.com).
</Note>

<div id="frequently-asked-questions">
  ## Questions fréquentes
</div>

<AccordionGroup>
  <Accordion title="Combien de temps prend une migration ?">
    Nous livrons généralement les migrations sous 2 à 6 semaines, selon la complexité. Vous recevrez un délai précis après la réunion de lancement.
  </Accordion>

  <Accordion title="Puis-je demander des modifications après la livraison ?">
    Oui. Vous disposez d'une fenêtre de retours de 7 jours après la livraison pour demander des modifications sur le contenu, les ressources ou l'identité de marque. Nous traitons tous les retours relevant du périmètre dans le même délai et vous accompagnons jusqu'à ce que vous soyez prêt à passer en production.
  </Accordion>

  <Accordion title="Que dois-je faire une fois la migration terminée ?">
    La [checklist de mise en production](/fr/migration-services/go-live-checklist) vous guide à travers chacune de ces étapes, et nous vous accompagnons tout au long du parcours.
  </Accordion>

  <Accordion title="Puis-je personnaliser la structure de navigation ?">
    Oui. Vous pouvez fournir une architecture de l'information personnalisée et nous la mettons en place pour vous.
  </Accordion>

  <Accordion title="Que se passe-t-il si ma documentation évolue pendant la migration ?">
    Les migrations nécessitant un gel de code, toute modification apportée à votre documentation source après la soumission ne sera pas reprise. Terminez toutes les mises à jour avant de démarrer la migration.

    Si ce n'est pas possible, notre équipe travaille avec vous pour mettre en place de courts sprints afin de migrer certaines sections, ou pour réserver du temps dédié après le premier sprint pour migrer les modifications différentielles apportées pendant la migration.
  </Accordion>
</AccordionGroup>
