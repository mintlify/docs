Votre offre Pro inclut des services de migration pour transférer votre documentation vers Mintlify. Ce guide présente le processus de migration, les délais de livraison et notre approche en matière d'assurance qualité.

<div id="what-to-expect-for-your-migration">
  ## À quoi vous attendre lors de votre migration
</div>

Une migration est un transfert ponctuel de votre documentation qui nécessite un gel du code. Nous migrons votre documentation, votre image de marque (logos, couleurs et polices) et votre structure de navigation.

Votre migration inclut un lien de préproduction pour suivre l’avancement et une période de 3 jours après la livraison pour nous faire part de vos retours et corriger tout problème lié au contenu, aux ressources ou à l’image de marque.

<Info>
Les services de migration **n’incluent pas** :
- La synchronisation dynamique de votre documentation
- Le conseil en design ou en architecture de l’information
- Le développement de composants personnalisés (landing pages, intégrations personnalisées, etc.)
- La configuration des redirections
- La configuration de l’API playground
- La configuration de l’analytics
</Info>

<div id="migration-process">
  ## Processus de migration
</div>

<Steps>
  <Step title="Soumettre le formulaire de migration">
    Soumettez le [formulaire de migration](https://mintlify.typeform.com/pro-migrations).
    
    Indiquez la source de votre documentation, vos éléments de marque (logos, codes couleur, polices), vos spécifications OpenAPI, la date de mise en production souhaitée et votre méthode de livraison préférée (GitHub PR ou fichier zip).
  </Step>
  
  <Step title="Recevoir la portée et le calendrier (1 à 2 jours ouvrés)">
    Nous examinons votre soumission et vous communiquons un délai de livraison estimé sous 24 à 48&nbsp;heures. Nous signalerons tout composant personnalisé ou toute partie de votre migration nécessitant une discussion.
  </Step>
  
  <Step title="Suivre l'avancement pendant la migration">
    Nous lançons la migration et partageons un lien de préproduction afin que vous puissiez suivre l'avancement en temps réel.
  </Step>
  
  <Step title="Recevoir la livraison finale">
    Nous livrons votre migration via GitHub PR ou fichier zip, selon votre préférence.
  </Step>
  
  <Step title="Période de retours (trois jours ouvrés)">
    Transmettez tout retour concernant le contenu, les éléments ou l'image de marque. Nous traitons tous les points inclus dans le périmètre dans ce même délai.
  </Step>
  
  <Step title="Assistance pour la mise en production">
    Nous fournissons la documentation et les instructions nécessaires pour connecter votre dépôt, configurer les redirections et paramétrer votre domaine personnalisé.
  </Step>
</Steps>

<div id="quality-assurance">
  ## Assurance qualité
</div>

Nous travaillons avec des prestataires tiers vérifiés et effectuons un audit qualité interne avant la livraison. Voici ce que nous contrôlons&nbsp;:

<Card title="Exhaustivité du contenu" icon="file-check">
  - L'intégralité de la documentation issue de votre source a été migrée.
  - Le contenu est correctement mis en forme avec les composants Mintlify.
  - La structure de navigation correspond à vos spécifications.
</Card>

<Card title="Éléments visuels" icon="palette">
  - Les images et les vidéos s'affichent correctement.
  - Les polices personnalisées sont appliquées de manière cohérente.
  - Les logos pour les modes clair et sombre sont utilisés de manière appropriée.
  - Les jeux de couleurs respectent les exigences de contraste dans les deux modes.
</Card>

<Card title="Fonctionnalité" icon="rocket">
  - Tous les liens fonctionnent correctement (aucun lien cassé).
  - La table des matières s'affiche correctement.
  - Les composants React fonctionnent comme prévu.
  - Aucun problème de performance.
</Card>

<div id="after-your-migration">
  ## Après votre migration
</div>

Une fois que nous aurons livré votre migration, vous devrez suivre les étapes suivantes pour lancer votre documentation :

1. [Connectez votre dépôt GitHub et effectuez la configuration](https://www.mintlify.com/docs/quickstart#install-the-github-app).
2. [Configurez des redirections si nécessaire](https://www.mintlify.com/docs/create/redirects).
3. [Configurez votre domaine personnalisé](https://www.mintlify.com/docs/customize/custom-domain).
4. **Testez avant la mise en ligne** - Avant de basculer depuis votre ancienne documentation :
   - Passez en revue toutes les pages via votre lien de préproduction.
   - Testez la navigation et la recherche.
   - Vérifiez que toutes les images et vidéos se chargent correctement.
   - Vérifiez que les blocs de code s’affichent correctement.
   - Testez sur mobile et sur ordinateur.
5. **Lancez-vous !** Votre documentation est prête à être mise en ligne !

<Note>
  **Besoin d’aide ?** Notre équipe support est disponible pour vous guider dans ces étapes. Contactez-nous via [support@mintlify.com](mailto:support@mintlify.com).
</Note>

<div id="frequently-asked-questions">
  ## Foire aux questions
</div>

<AccordionGroup>
  <Accordion title="Combien de temps dure une migration ?">
    Nous livrons généralement les migrations en 10 jours ouvrables, selon la complexité. Vous recevrez un calendrier précis dans les 24 à 48 heures suivant l'envoi de votre formulaire de migration.
  </Accordion>
  
  <Accordion title="Puis-je demander des modifications après la livraison ?">
    Oui. Vous disposez d'un délai de 3 jours après la livraison pour demander des modifications au contenu, aux ressources ou au branding. Nous traitons tous les retours inclus dans le périmètre dans le même délai.
  </Accordion>
  
  <Accordion title="Que dois-je faire une fois la migration terminée ?">
    La migration transfère votre contenu, mais vous devrez :
    - Connecter votre dépôt de documentation à votre tableau de bord Mintlify.
    - Configurer les redirections depuis les anciennes URL de votre documentation.
    - Configurer votre domaine personnalisé.
    
    Nous fournissons une documentation détaillée, étape par étape, ainsi qu'une assistance pour chacune de ces étapes.
  </Accordion>
  
  <Accordion title="Puis-je personnaliser la structure de navigation ?">
    Oui. Vous pouvez fournir une architecture de l'information personnalisée dans le formulaire de migration, et nous l'implémenterons. Cependant, le développement de composants personnalisés (comme des pages d'accueil sur mesure) n'est pas inclus dans le service de migration.
  </Accordion>
  
  <Accordion title="Que se passe-t-il si ma documentation change pendant la migration ?">
    Comme les migrations nécessitent un gel du code, toute modification apportée à votre documentation source après l'envoi ne sera pas prise en compte. Nous recommandons de finaliser toutes les mises à jour avant de commencer la migration.
  </Accordion>
</AccordionGroup>
