Les composants `<RequestExample>` et `<ResponseExample>` affichent des blocs de code dans la barre latérale droite pour créer une mise en page en deux colonnes qui garde les exemples visibles pendant que les utilisateurs font défiler votre contenu. Ces composants sont conçus pour la documentation d’API, mais fonctionnent sur toutes les pages.

Cas d’usage courants :

- Documentation d’endpoint d’API avec des exemples de requête et de réponse
- Exemples de configuration accompagnés de texte explicatif
- Extraits de code que les utilisateurs consultent en suivant des instructions
- Exemples avant/après dans des tutoriels

Sur mobile, les composants `<RequestExample>` et `<ResponseExample>` s’affichent comme des blocs de code classiques et peuvent être défilés comme le reste de la page.

## RequestExample

Utilisez `<RequestExample>` pour fixer des exemples de code dans la barre latérale droite. Ce composant fonctionne de façon similaire au composant [CodeGroup](/fr/components/code-groups), mais affiche le code dans la barre latérale plutôt qu’en ligne.

Vous pouvez inclure plusieurs blocs de code dans un seul `<RequestExample>`. Chaque bloc de code doit posséder un attribut title.

````mdx RequestExample
<RequestExample>

```bash Request
  curl --request POST \
    --url https://dog-api.kinduff.com/api/facts
```

</RequestExample>
````

## ResponseExample

Le composant `<ResponseExample>` fixe des exemples de code dans la barre latérale droite, sous tout contenu `<RequestExample>` présent sur la même page.

````mdx ResponseExample
<ResponseExample>

```json Response
{ "status": "success" }
```

</ResponseExample>
````
