Mintlify sert automatiquement les ressources statiques de votre référentiel de documentation au chemin approprié sur votre domain. Par exemple, si vous avez `/images/my-logo.png` dans votre repo, le fichier image est disponible à l’adresse `https://docs.your-project.com/images/my-logo.png`.

Cela fonctionne pour tout type de fichier que vous souhaitez mettre à disposition de vos utilisateurs, notamment des images, des PDF, des vidéos ou des schémas.

Les fichiers doivent peser moins de 20 Mo pour les images et moins de 100 Mo pour les autres types de fichiers.

<div id="file-organization">
  ## Organisation des fichiers
</div>

Organisez vos fichiers à l’aide de dossiers pour que votre référentiel reste facile à parcourir :

```text
/votre-projet
  |- docs.json
  |- images/
    |- logo.png
    |- screenshots/
      |- dashboard.png
  |- assets/
    |- whitepaper.pdf
    |- demo-video.mp4
```

Les fichiers sont servis depuis la racine de votre domaine, de sorte que la structure de votre référentiel correspond directement à la structure de l’URL. Dans l’exemple précédent, `assets/whitepaper.pdf` serait disponible à `https://docs.your-project.com/assets/whitepaper.pdf`.
