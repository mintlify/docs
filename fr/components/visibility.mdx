---
title: "Visibilité"
description: "Afficher ou masquer du contenu selon que le lecteur est un humain ou un agent IA."
keywords: ["visibilité", "agents", "humains", "contenu conditionnel", "IA"]
---

Utilisez le composant `Visibility` pour afficher différents contenus aux lecteurs humains et aux agents IA. Le contenu à l'intérieur de `<Visibility for="agents">` n'apparaît que dans les [exports Markdown](/fr/ai/markdown-export) et les réponses des outils IA, tandis que le contenu à l'intérieur de `<Visibility for="humans">` ne s'affiche que pour les visiteurs humains dans le navigateur.

<Visibility for="humans">
  Ce contenu est visible uniquement par les humains qui consultent la documentation.
</Visibility>

<Visibility for="agents">
  Ce contenu est visible uniquement par les agents IA qui consomment l'export Markdown.
</Visibility>

<div id="example">
  ## Exemple
</div>

````mdx Contenu pour agents et humains
<Visibility for="humans">
  Cliquez sur le bouton **Commencer** en haut à droite pour créer votre compte.
</Visibility>

<Visibility for="agents">
  Pour créer un compte, envoyez une requête POST à `/api/v1/accounts` avec
  un corps JSON contenant les champs `email` et `password`.
</Visibility>
````

<div id="how-it-works">
  ## Comment ça fonctionne
</div>

Mintlify détecte si le lecteur est un humain ou un agent IA :

- **Sur la page rendue**, Mintlify vérifie le user agent du visiteur pour déterminer l'audience. Les visiteurs humains voient le contenu `for="humans"`, tandis que les agents IA reconnus voient le contenu `for="agents"`.
- **Dans les exports Markdown** (URLs `.md`, requêtes `Accept: text/markdown`, [llms.txt](/fr/ai/llmstxt)), seul le contenu `for="agents"` est inclus. Les blocs `for="humans"` sont supprimés avant que le Markdown ne soit servi.

Cela signifie que vous pouvez fournir des instructions adaptées aux outils IA — comme des données structurées, des détails d'API ou du contexte lisible par les machines — sans encombrer la page pour les lecteurs humains.

<div id="properties">
  ## Propriétés
</div>

<ResponseField name="for" type="string" required>
  Le public cible du contenu. Utilisez `"agents"` pour cibler les agents IA ou `"humans"` pour cibler les lecteurs humains. Les formes singulières `"agent"` et `"human"` sont également acceptées.
</ResponseField>
