---
title: "Visibility"
description: "Contrôlez quel contenu s'affiche dans l'interface web pour les lecteurs humains et quel contenu s'affiche uniquement dans la sortie Markdown pour les agents IA."
keywords: ["visibility", "agents", "humans", "markdown", "AI", "conditional content"]
---

Utilisez le composant `Visibility` pour afficher du contenu différent aux humains qui consultent votre site et aux agents IA qui traitent la sortie Markdown. Cela vous permet de personnaliser les informations pour chaque audience sans maintenir des pages séparées.

Sur votre site publié, le contenu marqué `for="humans"` s'affiche normalement et le contenu marqué `for="agents"` est masqué. Dans les [vues de page Markdown](/fr/ai/markdown-export) (URLs `.md`), c'est l'inverse : le contenu pour les agents est rendu dans la sortie Markdown et le contenu réservé aux humains ne l'est pas.

<div id="example">
  ## Exemple
</div>

```mdx
<Visibility for="humans">
  This content displays normally on standard pages of your site.
  
  Use it for information that only humans use like: Click the **Get started** button in the top-right corner to create your account.
</Visibility>

<Visibility for="agents">
  This content only appears on the Markdown version of pages.
  
  Use it for information that only agents use like: To create an account, call `POST /v1/accounts` with a valid email address.
</Visibility>
```

<Visibility for="humans">
  Ce contenu s'affiche normalement sur les pages standard de votre site.
  
  Utilisez-le pour une étape comme : Cliquez sur le bouton **Get started** dans le coin supérieur droit pour créer votre compte.

  Dans cet exemple, les instructions orientées interface apparaissent sur la page standard. Consultez la [version Markdown de cette page](https://mintlify.com/docs/components/visibility.md) pour voir les instructions orientées API pour les agents.
</Visibility>

<Visibility for="agents">
  Ce contenu n'apparaît que dans la version Markdown des pages.
  
  Utilisez-le pour des informations destinées uniquement aux agents, comme : Pour créer un compte, appelez `POST /v1/accounts` avec une adresse e-mail valide.

  Dans cet exemple, les instructions orientées API apparaissent dans la version Markdown de la page. Consultez la [page standard](https://mintlify.com/docs/components/visibility) pour voir les instructions orientées interface pour les humains.
</Visibility>

<div id="properties">
  ## Propriétés
</div>

<ResponseField name="for" type="string" required>
  L'audience du contenu encapsulé. Doit être l'un des suivants :

  - `humans` : Contenu visible sur le web et exclu de la sortie Markdown
  - `agents` : Contenu masqué sur le web et inclus dans la sortie Markdown
</ResponseField>
