---
title: "Champs de réponse"
description: "Afficher les valeurs de réponse d’une API."
keywords: ["sortie d’API", "valeurs de retour", "structure de réponse"]
---

Le composant `<ResponseField>` est conçu pour définir les valeurs de retour d’une API. De nombreuses documentations utilisent également `<ResponseField>` sur les pages lorsque vous devez lister les types de certains éléments.

<ResponseField name="response" type="string" required>
  Exemple de champ de réponse
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Un exemple de champ de réponse
</ResponseField>
```


<div id="props">
  ## Props
</div>

<ResponseField name="name" type="string" required>
  Nom de la valeur de réponse.
</ResponseField>

<ResponseField name="type" type="string" required>
  Type attendu de la valeur de réponse — peut être n’importe quelle chaîne.
</ResponseField>

<ResponseField name="default" type="string">
  Valeur par défaut.
</ResponseField>

<ResponseField name="required" type="boolean">
  Afficher « required » à côté du nom du champ.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Indique si un champ est obsolète.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Étiquettes affichées avant le nom du champ
</ResponseField>

<ResponseField name="post" type="string[]">
  Étiquettes affichées après le nom du champ
</ResponseField>
