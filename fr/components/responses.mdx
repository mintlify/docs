---
title: "Champs de réponse"
description: "Afficher les valeurs de réponse de l'API"
---

Le composant `<ResponseField>` est conçu pour définir les valeurs de retour d'une API. De nombreuses documentations l'utilisent également sur des pages lorsque vous devez répertorier les types d'éléments.

<ResponseField name="response" type="string" required>
  Un exemple de champ de réponse
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Un exemple de champ de réponse
</ResponseField>
```

## Props

<ResponseField name="name" type="string" required>
  Le nom de la valeur de la réponse.
</ResponseField>

<ResponseField name="type" type="string" required>
  Type attendu de la valeur de la réponse — peut être n’importe quelle chaîne.
</ResponseField>

<ResponseField name="default" type="string">
  Valeur par défaut.
</ResponseField>

<ResponseField name="required" type="boolean">
  Afficher « required » à côté du nom du champ.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Indique si un champ est déprécié ou non.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Étiquettes affichées avant le nom du champ.
</ResponseField>

<ResponseField name="post" type="string[]">
  Étiquettes affichées après le nom du champ.
</ResponseField>
