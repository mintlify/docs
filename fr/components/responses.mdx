---
title: 'Champs de réponse'
description: 'Afficher les valeurs de réponse d’une API'
---

Le composant `<ResponseField>` est conçu pour définir les valeurs de retour d’une API. De nombreuses documentations utilisent également `<ResponseField>` sur les pages lorsque vous devez lister les types d’éléments.

<ResponseField name="response" type="string" required>
  Un exemple de champ de réponse
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Un exemple de champ de réponse
</ResponseField>
```


<div id="props">
  ## Props
</div>

<ResponseField name="name" type="string" required>
  Le nom de la valeur de réponse.
</ResponseField>

<ResponseField name="type" type="string" required>
  Type attendu pour la valeur de réponse — il peut s’agir de n’importe quelle chaîne.
</ResponseField>

<ResponseField name="default" type="string">
  Valeur par défaut.
</ResponseField>

<ResponseField name="required" type="boolean">
  Afficher « required » à côté du nom du champ.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Indique si un champ est obsolète.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Libellés affichés avant le nom du champ
</ResponseField>

<ResponseField name="post" type="string[]">
  Libellés affichés après le nom du champ
</ResponseField>
