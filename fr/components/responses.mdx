---
title: 'Champs de réponse'
description: 'Afficher les valeurs renvoyées par l’API'
---

Le composant `<ResponseField>` permet de définir les valeurs de retour d’une API. De nombreuses documentations l’utilisent également sur des pages lorsqu’il faut lister des types.

<ResponseField name="response" type="string" required>
  Un exemple de champ de réponse
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Un exemple de champ de réponse
</ResponseField>
```


<div id="props">
  ## Props
</div>

<ResponseField name="name" type="string" required>
  Le nom de la valeur de réponse.
</ResponseField>

<ResponseField name="type" type="string" required>
  Type attendu de la valeur de réponse — il peut s’agir de n’importe quelle chaîne.
</ResponseField>

<ResponseField name="default" type="string">
  La valeur par défaut.
</ResponseField>

<ResponseField name="required" type="boolean">
  Afficher « required » à côté du nom du champ.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Indique si un champ est « deprecated » ou non.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Libellés affichés avant le nom du champ
</ResponseField>

<ResponseField name="post" type="string[]">
  Libellés affichés après le nom du champ
</ResponseField>
