Utilisez le composant `Update` pour afficher des entrées du journal des modifications, des mises à jour de version et des notes de version avec un formatage cohérent.

<Update label="Exemple" description="v0.1.1">
  ## Exemple de mise à jour

  Vous pouvez ajouter ici tout ce que vous voulez, comme une capture d’écran, un extrait de code ou une liste de modifications.
  
  <Frame>
    <img
      className="block"

    />
  </Frame>

  ### Fonctionnalités
  - Design adaptable
  - Ancre pour chaque mise à jour
  - Entrée de flux RSS générée pour chaque mise à jour
</Update>

<div id="how-to-use">
  ## Mode d’emploi
</div>

```mdx Update example
<Update label="2024-10-11" description="v0.1.0" tags={["Mintlify"]}>
  Ceci est une mise à jour avec un libellé, une description et une étiquette.
</Update>
```

Utilisez plusieurs composants `Update` pour créer des [journaux des modifications](/fr/create/changelogs).

Les composants `Update` génèrent automatiquement des entrées de flux RSS. Les entrées contiennent uniquement du Markdown brut : les composants, les code blocks, le code en ligne et les éléments HTML bruts ne sont pas inclus.

Si votre mise à jour inclut des composants ou du HTML, utilisez la prop `rss` pour fournir un texte alternatif aux abonnés RSS.

```mdx Custom RSS content
<Update
  label="March 2025"
  description="v2.0.0"
  rss={{
    title: "Mars 2025 - Refonte majeure",
    description: "Refonte complète de l'interface utilisateur avec de nouveaux composants"
  }}
>
  ## Nouveau tableau de bord

  <Frame>
    <img src="/dashboard.png" alt="Nouvelle interface utilisateur du tableau de bord affichant les composants mis à jour." />
  </Frame>

  Découvrez notre tableau de bord repensé avec une navigation améliorée.
</Update>
```


<div id="props">
  ## Props
</div>

<ResponseField name="label" type="string" required>
  Label de la mise à jour. Apparaît à gauche de la mise à jour et crée un lien d’ancrage. Les labels doivent être uniques.
</ResponseField>

<ResponseField name="tags" type="string[]">
  Tags de la mise à jour. Affichés comme filtres dans le panneau latéral droit.
</ResponseField>

<ResponseField name="description" type="string">
  Description de la mise à jour. Apparaît sous le label et le tag.
</ResponseField>

<ResponseField name="rss" type="object">
  Les composants Update génèrent automatiquement des entrées de flux RSS. Utilisez la propriété `rss` pour créer une seule entrée de flux RSS avec un titre et une description personnalisés.
  
  Si la propriété n’est pas fournie :

  - Les mises à jour avec des intertitres Markdown créent une entrée de flux RSS distincte pour chaque intertitre.
  - Les mises à jour sans intertitres Markdown créent une seule entrée de flux RSS en utilisant le `label` comme titre.

  <Expandable title="propriétés">
    <ResponseField name="title" type="string">
      Titre personnalisé pour l’entrée de flux RSS.
    </ResponseField>

    <ResponseField name="description" type="string">
      Description personnalisée pour l’entrée de flux RSS.
    </ResponseField>
  </Expandable>
</ResponseField>
