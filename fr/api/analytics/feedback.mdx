<div id="usage">
  ## Utilisation
</div>

Utilisez cet endpoint pour exporter les retours utilisateurs collectés à partir de votre documentation. Les retours incluent les commentaires contextuels sur les pages, les votes pouces uniquement et les retours sur les extraits de code.

Parcourez les résultats à l'aide du paramètre `cursor` renvoyé dans la réponse. Continuez à effectuer des requêtes tant que `hasMore` vaut `true`.

<div id="filtering">
  ## Filtrage
</div>

Filtrez les feedbacks par :

- **Plage de dates** : utilisez `dateFrom` et `dateTo` pour limiter les résultats à une période donnée
- **Source** : filtrez par type de feedback, `code_snippet`, `contextual` ou `thumbs_only`
- **Statut** : filtrez par des valeurs de statut comme `pending`, `in_progress`, `resolved` ou `dismissed`

<div id="response-types">
  ## Types de réponse
</div>

La réponse contient différents types de commentaires en fonction de la source :

- **Commentaire contextuel** : retour au niveau de la page avec un commentaire textuel. Inclut le booléen `helpful` et éventuellement l'adresse e-mail `contact`.
- **Vote pouces uniquement** : un vote pouce levé ou pouce baissé sans commentaire textuel. Partage le même schéma que le commentaire contextuel.
- **Commentaire sur extrait de code** : retour sur un bloc de code spécifique. Inclut les champs `code`, `filename` et `lang`.
