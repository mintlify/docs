<div id="usage">
  ## Utilisation
</div>

Utilisez cet endpoint pour exporter les retours agrégés par page de documentation. Chaque entrée affiche le nombre total de pouces en l'air, pouces en bas et retours sur les extraits de code pour un chemin de page donné.

Paginez les résultats en augmentant le paramètre `limit`. Continuez à effectuer des requêtes tant que `hasMore` vaut `true`.

<div id="filtering">
  ## Filtrage
</div>

Filtrez les résultats par :

- **Plage de dates** : utilisez `dateFrom` et `dateTo` pour limiter les résultats à une période donnée
- **Source** : filtrez par type de feedback, `code_snippet` ou `contextual`
- **Statut** : filtrez par des valeurs de statut comme `pending`, `in_progress`, `resolved` ou `dismissed`

<div id="response-fields">
  ## Champs de réponse
</div>

Chaque élément du tableau `feedback` inclut :

- **path** : le chemin de la page de documentation
- **thumbsUp** : nombre de retours contextuels positifs
- **thumbsDown** : nombre de retours contextuels négatifs
- **code** : nombre de retours sur les extraits de code
- **total** : nombre total de retours pour la page
