<div id="usage">
  ## Utilisation
</div>

Utilisez ce point de terminaison pour obtenir un aperçu des retours sur les pages de votre documentation. Les résultats sont agrégés par chemin de page et triés par nombre total de retours dans l'ordre décroissant.

<div id="filtering">
  ## Filtrage
</div>

Filtrez les retours par :

- **Plage de dates** : Utilisez `dateFrom` et `dateTo` pour limiter les résultats à une période spécifique
- **Source** : Filtrez par type de retour `code_snippet` ou `contextual`
- **Statut** : Filtrez par valeurs de statut comme `pending`, `in_progress`, `resolved` ou `dismissed`

<div id="response">
  ## Réponse
</div>

Chaque entrée dans la réponse représente une seule page et comprend :

- **thumbsUp** : Nombre d'évaluations contextuelles positives
- **thumbsDown** : Nombre d'évaluations contextuelles négatives
- **code** : Nombre de retours sur des extraits de code
- **total** : Somme de tous les retours pour la page
