Vous pouvez autoriser l&#39;[agent](/fr/ai/agent) à suggérer des mises à jour de la documentation à partir de deux sources.

* **Modifications de pull request** : Surveillez les dépôts Git sélectionnés pour détecter les changements de code nécessitant des mises à jour de la documentation.
* **Conversations avec l&#39;Assistant** : Analysez les questions que les utilisateurs posent à l&#39;assistant sur votre site de documentation pour identifier les lacunes de contenu.

Lorsque l&#39;agent identifie des mises à jour potentielles de la documentation, il crée une suggestion dans votre Dashboard avec le contexte nécessaire pour créer une pull request (demande de fusion).

Utilisez les suggestions pour garder votre documentation à jour de manière proactive lorsque de nouvelles fonctionnalités sont déployées ou pour répondre aux questions fréquentes des utilisateurs.

<div id="prerequisites">
  ## Conditions requises
</div>

Avant de pouvoir utiliser les suggestions, vous devez installer la [Mintlify GitHub App](/fr/deploy/github) au sein de votre organisation. L&#39;application doit avoir accès à votre référentiel de documentation et à au moins un autre référentiel où les modifications de code nécessitent une mise à jour de la documentation.

<div id="configure-suggestions">
  ## Configurer les suggestions
</div>

Définissez quels référentiels l’agent surveille et comment vous recevez les notifications dans les paramètres de l’agent. La page des paramètres affiche toutes les organisations GitHub et tous les référentiels surveillés par l’agent.

Pour accéder aux paramètres de l’agent :

1. Cliquez sur le bouton **Ask agent** dans votre Dashboard pour ouvrir le panneau de l’agent.
2. Cliquez sur le bouton **Settings**.

<div id="monitor-repositories">
  ### Surveiller les référentiels
</div>

L’agent surveille toutes les pull requests (demandes de fusion) fusionnées, quelle que soit la branche dans laquelle elles sont fusionnées, pour chaque référentiel pour lequel vous avez activé la surveillance. Un contrôle GitHub nommé **Mintlify Autopilot** s’exécute sur les pull requests (demandes de fusion) dans les référentiels surveillés afin de les analyser et d’identifier d’éventuelles mises à jour de la documentation. Lorsque vous fusionnez une pull request qui nécessite des mises à jour de la documentation, l’agent crée une suggestion dans votre Dashboard.

<Note>
  Lorsque vous activez pour la première fois la surveillance d’un référentiel, l’agent crée des suggestions pour les pull requests fusionnées au cours des sept derniers jours. Ce traitement rétroactif n’a lieu que si aucune suggestion n’existe déjà pour ce référentiel. Il est possible que plusieurs suggestions apparaissent immédiatement après l’activation de la surveillance.
</Note>

Si vous désactivez la surveillance, l’agent cesse immédiatement de surveiller le référentiel. Toutes les suggestions existantes pour ce référentiel restent dans votre Dashboard jusqu’à ce que vous les rejetiez.

<div id="conversation-insights">
  ### Analyses des conversations
</div>

Activez les analyses des conversations pour recevoir des suggestions fondées sur les questions fréquentes que les utilisateurs posent à l’Assistant.

Lorsque cette option est activée, l’agent analyse périodiquement les conversations avec l’Assistant et génère des suggestions lorsqu’il identifie des tendances dans les questions qui révèlent une documentation manquante ou peu claire.

<div id="notifications">
  ### Notifications
</div>

Les suggestions de l&#39;agent apparaissent toujours dans votre Dashboard. Vous pouvez configurer les notifications pour recevoir des e-mails ou des messages directs Slack lorsque l&#39;agent crée de nouvelles suggestions.

<Tip>
  Si vous ne recevez pas les notifications par e-mail relatives aux suggestions, vérifiez votre dossier de courrier indésirable et ajoutez l&#39;adresse e-mail `notifications@mintlify.com` à votre liste d&#39;expéditeurs autorisés.
</Tip>

<div id="review-suggestions">
  ## Examiner les suggestions
</div>

L’agent crée des suggestions dans votre Dashboard. Chaque suggestion affiche l’intitulé de la pull request (demande de fusion) ou le sujet de conversation de l’Assistant qui a déclenché la suggestion, la date de création et les mises à jour de documentation proposées.

Le bouton **Ask agent** dans votre Dashboard affiche le nombre de suggestions en attente de votre examen.

<div id="create-pull-requests">
  ### Créer des pull requests
</div>

Ajoutez des suggestions comme contexte pour que l&#39;agent puisse créer des pull requests.

1. Cliquez sur le bouton **Ask agent** dans votre Dashboard pour ouvrir le panneau de l&#39;agent.
2. Cliquez sur **Add to chat**.
3. Envoyez le prompt à l&#39;agent pour qu&#39;il ouvre une pull request (demande de fusion).

<div id="dismiss-suggestions">
  ### Ignorer des suggestions
</div>

Si une suggestion ne nécessite pas de mise à jour de la documentation ou si vous avez déjà appliqué les modifications, ignorez-la pour la supprimer de votre Dashboard.

1. Cliquez sur le bouton **Ask agent** dans votre Dashboard pour ouvrir le panneau de l’agent.
2. Cliquez sur le bouton **Dismiss** à côté de chaque suggestion que vous souhaitez ignorer.

La suggestion est immédiatement supprimée de votre Dashboard. Vous ne pouvez pas restaurer les suggestions ignorées.
