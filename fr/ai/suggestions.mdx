Vous pouvez autoriser l&#39;[agent](/fr/ai/agent) à surveiller vos dépôts Git sélectionnés et à suggérer des mises à jour de la documentation. Lorsque l&#39;agent identifie des changements potentiellement visibles par les utilisateurs dans votre application, il prépare le contexte nécessaire pour que vous puissiez créer une pull request (demande de fusion) afin de mettre à jour votre documentation en un clic.

Utilisez les suggestions pour garder votre documentation à jour de manière proactive lorsque de nouvelles fonctionnalités ou mises à jour sont déployées.

<div id="prerequisites">
  ## Conditions requises
</div>

Avant de pouvoir utiliser les suggestions, vous devez installer la [Mintlify GitHub App](/fr/deploy/github) au sein de votre organisation. L&#39;application doit avoir accès à votre référentiel de documentation et à au moins un autre référentiel où les modifications de code nécessitent une mise à jour de la documentation.

<div id="configure-suggestions">
  ## Configurer les suggestions
</div>

Configurez les dépôts pour lesquels l&#39;agent doit proposer des suggestions dans les paramètres de l&#39;agent.

La page de configuration affiche toutes les organisations GitHub dans lesquelles vous avez installé la GitHub App Mintlify, ainsi que les dépôts que l&#39;agent surveille.

<div id="select-repositories-to-monitor">
  ### Sélectionner les référentiels à surveiller
</div>

Après avoir installé la GitHub App Mintlify au sein de vos organisations, sélectionnez les référentiels à surveiller.

1. Cliquez sur le bouton **Ask agent** dans votre Dashboard pour ouvrir le panneau de l’agent.
2. Cliquez sur le bouton **Settings**.
   <Frame>
     <img src="/images/agent/dashboard-settings-light.png" alt="Le bouton des paramètres en mode clair." className="block dark:hidden" />

     <img src="/images/agent/dashboard-settings-dark.png" alt="Le bouton des paramètres en mode sombre." className="hidden dark:block" />
   </Frame>
3. Dans la section **Monitor Repositories**, cliquez sur le nom de l’organisation qui contient les référentiels que vous souhaitez surveiller.
4. Activez le commutateur à côté de chaque référentiel que vous souhaitez surveiller, ou cliquez sur **Enable all** pour surveiller tous les référentiels de l’organisation.

L’agent surveille la branche par défaut (généralement `main`) pour chaque référentiel. Lorsqu’une pull request (demande de fusion) est fusionnée dans cette branche, l’agent analyse les modifications et crée des suggestions.

<Note>
  Lorsque vous activez pour la première fois la surveillance d’un référentiel, l’agent crée des suggestions pour les pull requests fusionnées au cours des sept derniers jours. Ce traitement rétroactif n’a lieu que si aucune suggestion n’existe déjà pour ce référentiel. Il est possible que plusieurs suggestions apparaissent immédiatement après l’activation de la surveillance.
</Note>

<div id="stop-monitoring-a-repository">
  ### Arrêter le suivi d’un référentiel
</div>

1. Cliquez sur le bouton **Ask agent** dans votre Dashboard pour ouvrir le panneau de l’agent.
2. Cliquez sur le bouton **Settings**.
   <Frame>
     <img src="/images/agent/dashboard-settings-light.png" alt="Le bouton des paramètres en mode clair." className="block dark:hidden" />

     <img src="/images/agent/dashboard-settings-dark.png" alt="Le bouton des paramètres en mode sombre." className="hidden dark:block" />
   </Frame>
3. Dans la section **Monitor Repositories**, cliquez sur le nom de l’organisation qui contient les référentiels dont vous voulez arrêter le suivi.
4. Désactivez le bouton bascule à côté du référentiel dont vous voulez arrêter le suivi, ou cliquez sur **Disable all** pour arrêter le suivi de tous les référentiels de l’organisation.

L’agent arrête immédiatement de suivre le référentiel. Toutes les suggestions existantes pour ce référentiel restent dans votre Dashboard jusqu’à ce que vous les ignoriez.

<div id="review-suggestions">
  ## Examiner les suggestions
</div>

Lorsque l’agent détecte des modifications visibles par les utilisateurs dans vos référentiels surveillés, il crée des suggestions qui apparaissent dans votre Dashboard. Chaque suggestion affiche l’intitulé de la pull request (demande de fusion), le nom du référentiel, la date de création de la suggestion et les mises à jour de documentation proposées.

Le bouton **Ask agent** dans votre Dashboard affiche le nombre de suggestions en attente de votre examen.

<div id="create-pull-requests">
  ### Créer des pull requests
</div>

Ajoutez des suggestions comme contexte pour que l&#39;agent puisse créer des pull requests.

1. Cliquez sur le bouton **Ask agent** dans votre Dashboard pour ouvrir le panneau de l&#39;agent.
2. Cliquez sur **Add to chat**.
3. Envoyez le prompt à l&#39;agent pour qu&#39;il ouvre une pull request (demande de fusion).

<div id="dismiss-suggestions">
  ### Ignorer des suggestions
</div>

Si une suggestion ne nécessite pas de mise à jour de la documentation ou si vous avez déjà appliqué les modifications, ignorez-la pour la supprimer de votre Dashboard.

1. Cliquez sur le bouton **Ask agent** dans votre Dashboard pour ouvrir le panneau de l’agent.
2. Cliquez sur le bouton **Dismiss** à côté de chaque suggestion que vous souhaitez ignorer.

La suggestion est immédiatement supprimée de votre Dashboard. Vous ne pouvez pas restaurer les suggestions ignorées.
