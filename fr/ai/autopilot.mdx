Autopilot surveille les dépôts Git que vous avez sélectionnés et suggère des mises à jour de la documentation. Autopilot signale les modifications potentiellement visibles par les utilisateurs dans votre application et prépare le contexte pour que l’agent rédige une pull request (demande de fusion) visant à mettre à jour votre documentation.

Utilisez Autopilot pour garder votre documentation à jour de manière proactive lorsque de nouvelles fonctionnalités ou mises à jour sont déployées.

<div id="prerequisites">
  ## Prérequis
</div>

Avant d&#39;utiliser autopilot, vous devez installer la [Mintlify GitHub App](/fr/deploy/github) au sein de votre organisation. L&#39;application doit avoir accès à votre référentiel de documentation et à au moins un autre référentiel où les modifications de code nécessitent des mises à jour de la documentation.

<div id="access-autopilot">
  ## Accéder à Autopilot
</div>

Accédez à la [page Autopilot](https://dashboard.mintlify.com/products/autopilot) sur votre Dashboard pour afficher les suggestions et configurer la surveillance de votre référentiel. La page Autopilot affiche les suggestions en attente qui nécessitent votre revue et donne accès aux paramètres de configuration.

<div id="configure-autopilot">
  ## Configurer autopilot
</div>

Configurez les paramètres d&#39;autopilot depuis la page autopilot en cliquant sur le bouton **Configuration**.

<Frame>
  <img src="/images/autopilot/configuration-button-light.png" alt="La page de configuration d'autopilot en mode clair." className="block dark:hidden" />

  <img src="/images/autopilot/configuration-button-dark.png" alt="La page de configuration d'autopilot en mode sombre." className="hidden dark:block" />

La page de configuration affiche toutes les organisations GitHub dans lesquelles vous avez installé la GitHub App Mintlify, ainsi que les dépôts surveillés par autopilot.

<div id="select-repositories-to-monitor">
  ### Sélectionner les référentiels à surveiller
</div>

Après avoir installé la Mintlify GitHub App sur vos organisations, vous pouvez sélectionner les référentiels à surveiller.

1. Accédez à la [page des paramètres d&#39;Autopilot](https://dashboard.mintlify.com/products/autopilot/settings).
2. Trouvez l&#39;organisation GitHub à laquelle appartient le référentiel que vous souhaitez surveiller.
3. Cliquez sur l&#39;icône d&#39;engrenage à côté du nom de l&#39;organisation.
   <Frame>
     <img src="/images/autopilot/org-config-light.png" alt="L'icône d'engrenage à côté du nom de l'organisation en mode clair." className="block dark:hidden" />

     <img src="/images/autopilot/org-config-dark.png" alt="L'icône d'engrenage à côté du nom de l'organisation en mode sombre." className="hidden dark:block" />
   </Frame>
4. Activez le bouton à bascule à côté de chaque référentiel que vous souhaitez surveiller.
5. Utilisez la barre de recherche pour filtrer les référentiels par nom.
6. Cliquez sur **Tout activer** ou **Tout désactiver** pour configurer rapidement tous les référentiels.

Autopilot surveille la branche par défaut (généralement `main`) de chaque référentiel. Lorsqu&#39;une pull request (demande de fusion) est fusionnée dans cette branche, Autopilot analyse les changements et propose des suggestions.

<div id="stop-monitoring-a-repository">
  ### Arrêter de surveiller un référentiel
</div>

1. Accédez à la [page des paramètres Autopilot](https://dashboard.mintlify.com/products/autopilot/settings).
2. Trouvez l’organisation GitHub à laquelle appartient le référentiel que vous souhaitez arrêter de surveiller.
3. Cliquez sur l’icône d’engrenage à côté du nom de l’organisation.
   <Frame>
     <img src="/images/autopilot/org-config-light.png" alt="L’icône d’engrenage à côté du nom de l’organisation en mode clair." className="block dark:hidden" />

     <img src="/images/autopilot/org-config-dark.png" alt="L’icône d’engrenage à côté du nom de l’organisation en mode sombre." className="hidden dark:block" />
   </Frame>
4. Désactivez le bouton à bascule à côté du référentiel que vous souhaitez arrêter de surveiller.
5. Utilisez la barre de recherche pour filtrer les référentiels par nom.

Autopilot arrête immédiatement de surveiller le référentiel. Toutes les suggestions existantes pour ce référentiel restent dans votre Dashboard jusqu’à ce que vous les rejetiez.

<div id="review-suggestions">
  ## Examiner les suggestions
</div>

Lorsque Autopilot détecte des modifications visibles par les utilisateurs dans vos référentiels surveillés, il crée des suggestions qui apparaissent dans votre Dashboard. Chaque suggestion affiche le titre de la pull request, le nom du référentiel, un résumé expliquant pourquoi la modification peut nécessiter une mise à jour de la documentation, ainsi que la date de création de la suggestion.

<div id="apply-suggestions">
  ### Appliquer des suggestions
</div>

Lorsque vous appliquez une suggestion, Autopilot ouvre le panneau [agent](/fr/ai/agent) avec du contexte sur la pull request (demande de fusion). L’agent reçoit l’URL de la pull request et un résumé des fichiers modifiés.

1. Accédez à la [page Autopilot](https://dashboard.mintlify.com/products/autopilot) dans votre Dashboard.
2. Cliquez sur **Apply suggestion**.
3. Utilisez l’interface de l’agent pour affiner les propositions de mise à jour de la documentation ou créer une pull request (demande de fusion).

<div id="dismiss-suggestions">
  ### Ignorer des suggestions
</div>

Si une suggestion ne nécessite pas de mise à jour de la documentation ou si vous avez déjà effectué les modifications, ignorez-la pour la retirer de votre Dashboard.

1. Accédez à la [page Autopilot](https://dashboard.mintlify.com/products/autopilot) depuis votre Dashboard.
2. Cliquez sur **Dismiss**.

La suggestion est immédiatement supprimée de votre liste affichée. Vous ne pouvez pas récupérer les suggestions ignorées.
