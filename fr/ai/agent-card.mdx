Mintlify héberge automatiquement une carte d'agent [Agent-to-Agent (A2A)](https://a2aproject.github.io/A2A/latest/) à l'adresse `/.well-known/agent-card.json`. La carte d'agent est un document JSON normalisé qui aide les agents d'IA à découvrir ce que propose votre site de documentation et où trouver des informations détaillées sur ses capacités.

Considérez la carte d'agent comme une carte pour les agents : un point de découverte unique qui pointe vers les métadonnées de votre site, l'URL de la documentation et les skills disponibles.

Consultez votre carte d'agent en ajoutant `/.well-known/agent-card.json` à l'URL de votre site de documentation. Mintlify génère la carte d'agent automatiquement et la maintient à jour. Aucune configuration n'est requise.

<div id="when-to-use-the-agent-card">
  ## Quand utiliser la carte d'agent
</div>

La carte d'agent est particulièrement utile lorsque :

- Vous souhaitez que les agents compatibles A2A découvrent votre documentation de manière programmatique.
- Vous souhaitez exposer vos fichiers `skill.md` aux agents via un point de découverte standard.
- Vous intégrez votre documentation à des frameworks d'agents conformes à la spécification A2A.

Si vos utilisateurs se connectent à votre documentation via des clients MCP, consultez [Model Context Protocol (MCP)](/fr/ai/model-context-protocol). La carte d'agent complète MCP en fournissant une couche de découverte légère qui n'exige pas l'établissement d'une session.

<div id="agent-card-structure">
  ## Structure de la carte d'agent
</div>

`GET /.well-known/agent-card.json` renvoie un document JSON conforme au schéma 0.2.0 de la carte d'agent A2A :

```json
{
  "schemaVersion": "0.2.0",
  "name": "Votre site de documentation",
  "description": "Une brève description de votre site de documentation.",
  "url": "https://your-docs-domain.com/",
  "version": "1.0.0",
  "documentationUrl": "https://your-docs-domain.com/",
  "capabilities": {
    "streaming": false
  },
  "skills": [
    {
      "id": "payments",
      "name": "Payments",
      "description": "Acceptez les paiements et gérez les abonnements.",
      "tags": [],
      "url": "https://your-docs-domain.com/.well-known/agent-skills/payments/skill.md"
    }
  ]
}
```

| Champ              | Description                                                                                                   |
| ------------------ | ------------------------------------------------------------------------------------------------------------- |
| `schemaVersion`    | La version du schéma de la carte d'agent A2A.                                                                 |
| `name`             | Le nom de votre site de documentation, issu de votre fichier `llms.txt`.                                      |
| `description`      | Une brève description de votre site de documentation, issue de votre fichier `llms.txt`.                      |
| `url`              | L'URL racine de votre site de documentation.                                                                  |
| `version`          | La version de la carte d'agent.                                                                               |
| `documentationUrl` | L'URL où les agents peuvent trouver une documentation lisible par un humain.                                  |
| `capabilities`     | Capacités d'agent prises en charge. Actuellement, signale `streaming: false`.                                 |
| `skills`           | Un tableau de skills exposées par votre documentation. Chaque entrée pointe vers un fichier `skill.md` hébergé. |

<div id="skill-entries">
  ### Entrées de skills
</div>

Chaque entrée du tableau `skills` décrit une skill que les agents peuvent récupérer :

| Champ         | Description                                                                                              |
| ------------- | -------------------------------------------------------------------------------------------------------- |
| `id`          | Un slug sûr pour les URL, dérivé du champ `name` dans le frontmatter de votre `skill.md`.                |
| `name`        | Le nom affiché de la skill.                                                                              |
| `description` | Une brève description issue du frontmatter de votre `skill.md`, tronquée à 1024 caractères.              |
| `tags`        | Une liste de tags associés à la skill. Actuellement toujours vide.                                       |
| `url`         | L'URL pour récupérer le fichier `skill.md` complet.                                                      |

Les skills proviennent de la même source que les [points de découverte de skills](/fr/ai/skillmd#skills-discovery-endpoints). Pour personnaliser les skills affichées dans votre carte d'agent, ajoutez des fichiers de skill personnalisés à votre projet. Consultez [Fichiers de skill personnalisés](/fr/ai/skillmd#custom-skill-files) pour plus d'informations.

<div id="use-the-agent-card-with-agents">
  ## Utiliser la carte d'agent avec les agents
</div>

Les agents compatibles A2A récupèrent `/.well-known/agent-card.json` pour découvrir votre documentation. À partir de la réponse, les agents peuvent :

- Identifier votre site de documentation par son nom et sa description.
- Suivre `documentationUrl` pour récupérer du contenu lisible par un humain.
- Parcourir le tableau `skills` et récupérer chaque fichier `skill.md` pour savoir ce qu'ils peuvent accomplir avec votre produit.

```bash
curl https://your-docs-domain.com/.well-known/agent-card.json
```

<Note>
  Si vous utilisez un [reverse proxy](/fr/deploy/reverse-proxy), configurez-le pour transférer le chemin `/.well-known/agent-card.json` vers votre sous-domaine Mintlify.
</Note>
