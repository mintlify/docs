La commande `mint analytics` vous donne acces aux [analytiques de votre documentation](/fr/optimize/analytics) depuis le terminal. C'est utile pour generer des rapports d'analytiques par scripts, rediriger les donnees vers d'autres outils et effectuer des verifications rapides sans ouvrir le tableau de bord.

<div id="prerequisites">

## Prerequis

</div>

Vous devez vous connecter pour utiliser les commandes d'analytiques. Executez `mint login` pour vous authentifier avec votre compte Mintlify.

Consultez [Apercu local](/fr/cli/preview#log-in-for-search-and-assistant) pour plus de details sur l'authentification.

<div id="view-key-metrics">

## Voir les metriques cles

</div>

Affichez un resume des vues, visiteurs, recherches, retours et utilisation de l'assistant :

```bash
mint analytics stats
```

Utilisez des flags pour filtrer par plage de dates, page ou source de trafic :

- `--from` : Date de debut au format `YYYY-MM-DD`. Par defaut, il y a 7 jours.
- `--to` : Date de fin au format `YYYY-MM-DD`. Par defaut, aujourd'hui.
- `--page` : Filtrer par un chemin de page specifique.
- `--humans` : Afficher uniquement le trafic humain.
- `--agents` : Afficher uniquement le trafic IA.

```bash Example analytics stats flags
mint analytics stats --from 2025-01-01 --to 2025-01-31
mint analytics stats --page /quickstart
mint analytics stats --humans
mint analytics stats --agents
```

<div id="view-search-analytics">

## Voir les analytiques de recherche

</div>

Consultez ce que vos utilisateurs recherchent, avec le nombre de resultats et les taux de clics :

```bash
mint analytics search
```

Utilisez des flags pour filtrer par chaine de requete ou page :

- `--query` : Filtrer par sous-chaine de requete de recherche.
- `--page` : Filtrer par un chemin de page specifique.

```bash Example analytics search flags
mint analytics search --query "authentication"

mint analytics search --page /api-reference
```

<div id="view-feedback">

## Voir les retours

</div>

Consultez les retours soumis par les utilisateurs sur les pages de votre documentation :

```bash
mint analytics feedback
```

Utilisez des flags pour agreger les retours par page ou filtrer par retours sur les extraits de code :

- `--type` : Agreger les retours par page ou extrait de code. Par defaut, page.
- `--page` : Filtrer par un chemin de page specifique.

```bash Example analytics feedback flags
mint analytics feedback --type page
mint analytics feedback --type code
```

<div id="view-assistant-conversations">

## Voir les conversations de l'assistant

</div>

Listez les conversations recentes et les categories de conversations de l'assistant IA :

```bash
mint analytics conversation list
mint analytics conversation buckets list
```

Chaque sortie de liste inclut un ID pour chaque entree. Passez cet ID a la commande `view` correspondante pour voir les details complets :

```bash
mint analytics conversation view <conversation-id>
mint analytics conversation buckets view <bucket-id>
```

<div id="choose-an-output-format">

## Choisir un format de sortie

</div>

Toutes les commandes d'analytiques utilisent par defaut la sortie `plain`, qui est separee par des tabulations et adaptee a la redirection vers d'autres outils. Utilisez `--format` pour changer la sortie :

```bash
mint analytics stats --format table    # Pretty-printed table with colors
mint analytics stats --format json     # Raw JSON
mint analytics stats --format graph    # Horizontal bar chart
```

Utilisez le flag `--agent` pour une sortie en JSON. La CLI detecte automatiquement les environnements d'agents lorsque la variable d'environnement `CLAUDECODE` est `1`.

<div id="set-defaults-with-config">

## Definir des valeurs par defaut avec config

</div>

Pour eviter de repeter les flags courants, definissez des valeurs par defaut avec `mint config`. La configuration est enregistree dans `~/.config/mintlify/config.json`.

Cles de configuration disponibles :

| Key | Description |
| --- | --- |
| `subdomain` | Sous-domaine par defaut pour les commandes d'analytiques. |
| `dateFrom` | Date de debut par defaut pour les requetes d'analytiques (`YYYY-MM-DD`). |
| `dateTo` | Date de fin par defaut pour les requetes d'analytiques (`YYYY-MM-DD`). |

```bash Example configs
mint config set subdomain my-docs
mint config set dateFrom 2025-01-01
```

Une fois definis, les commandes d'analytiques utilisent ces valeurs automatiquement :

```bash
# Uses the configured subdomain and dateFrom without flags
mint analytics stats
```

Pour voir ou effacer une valeur, executez :

```bash
mint config get subdomain
mint config clear subdomain
```
