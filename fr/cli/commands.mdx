<div id="global-flags">
## Options globales
</div>

Ces options sont disponibles pour toutes les commandes.

| Flag | Description |
| --- | --- |
| `--telemetry`, `-t` | Activer ou désactiver la télémétrie d'utilisation anonyme. |
| `--help`, `-h` | Afficher l'aide de la commande. |
| `--version`, `-v` | Afficher la version du CLI. Alias de `mint version`. |

<div id="mint-dev">
## `mint dev`
</div>

Démarrer un aperçu local de votre documentation.

```bash
mint dev [flags]
```

| Flag | Description |
| --- | --- |
| `--port` | Port sur lequel exécuter l'aperçu local. Par défaut `3000`. |
| `--no-open` | Ne pas ouvrir le navigateur automatiquement. |
| `--groups` | Liste de groupes d'utilisateurs séparés par des virgules à simuler pour l'aperçu. |
| `--disable-openapi` | Ignorer le traitement des fichiers OpenAPI pour améliorer les performances. |
| `--local-schema` | Autoriser les fichiers OpenAPI hébergés localement servis via HTTP. |

<div id="mint-login">
## `mint login`
</div>

S'authentifier avec votre compte Mintlify.

```bash
mint login
```

Ouvre une fenêtre de navigateur pour terminer l'authentification. Si le navigateur ne s'ouvre pas, le CLI affiche une URL à ouvrir manuellement et une invite pour coller le code d'autorisation. Les identifiants sont enregistrés dans `~/.config/mintlify/config.json`.

<div id="mint-analytics">
## `mint analytics`
</div>

Consulter les données d'analyse de votre documentation. Nécessite une authentification avec `mint login`.

```bash
mint analytics <subcommand> [flags]
```

Toutes les sous-commandes acceptent ces options partagées :

| Flag | Description |
| --- | --- |
| `--subdomain` | Sous-domaine de la documentation. Par défaut, la valeur définie avec `mint config set subdomain`. |
| `--from` | Date de début au format `YYYY-MM-DD`. Par défaut il y a 7 jours, ou la valeur définie avec `mint config set dateFrom`. |
| `--to` | Date de fin au format `YYYY-MM-DD`. Par défaut aujourd'hui, ou la valeur définie avec `mint config set dateTo`. |
| `--format` | Format de sortie : `plain` (par défaut), `table`, `json` ou `graph`. |
| `--agent` | Sortie JSON. Équivalent à `--format json`. Également activé automatiquement lorsque la variable d'environnement `CLAUDECODE` est égale à `1`. |

<div id="mint-analytics-stats">
### `mint analytics stats`
</div>

Afficher un résumé des vues, visiteurs, recherches, retours et utilisation de l'assistant.

```bash
mint analytics stats [flags]
```

| Flag | Description |
| --- | --- |
| `--page` | Filtrer par un chemin de page spécifique. |
| `--humans` | Afficher uniquement le trafic humain. |
| `--agents` | Afficher uniquement le trafic des agents IA. |

<div id="mint-analytics-search">
### `mint analytics search`
</div>

Afficher les requêtes de recherche avec les nombres de résultats et les taux de clics.

```bash
mint analytics search [flags]
```

| Flag | Description |
| --- | --- |
| `--query` | Filtrer par sous-chaîne de requête de recherche. |
| `--page` | Filtrer par page la plus cliquée. |

<div id="mint-analytics-feedback">
### `mint analytics feedback`
</div>

Afficher les retours soumis par les utilisateurs.

```bash
mint analytics feedback [flags]
```

| Flag | Description |
| --- | --- |
| `--type` | Type de retour : `page` (agrégé par page) ou `code` (retours sur les extraits de code). |
| `--page` | Filtrer par un chemin de page spécifique. |

<div id="mint-analytics-conversation">
### `mint analytics conversation`
</div>

Consulter les données de conversation de l'assistant.

```bash
mint analytics conversation <subcommand> [flags]
```

<div id="mint-analytics-conversation-list">
#### `mint analytics conversation list`
</div>

Lister les conversations de l'assistant. Chaque entrée inclut un ID de conversation.

```bash
mint analytics conversation list [flags]
```

| Flag | Description |
| --- | --- |
| `--page` | Filtrer les conversations qui font référence à une page spécifique dans les sources. |

<div id="mint-analytics-conversation-view">
#### `mint analytics conversation view <conversation-id>`
</div>

Afficher une conversation individuelle par ID. Utilisez `mint analytics conversation list` pour obtenir les IDs.

```bash
mint analytics conversation view <conversation-id>
```

<div id="mint-analytics-conversation-buckets-list">
#### `mint analytics conversation buckets list`
</div>

Lister les catégories groupées de conversations. Chaque entrée inclut un ID de catégorie.

```bash
mint analytics conversation buckets list
```

<div id="mint-analytics-conversation-buckets-view">
#### `mint analytics conversation buckets view <bucket-id>`
</div>

Afficher les conversations dans une catégorie. Utilisez `mint analytics conversation buckets list` pour obtenir les IDs.

```bash
mint analytics conversation buckets view <bucket-id>
```

<div id="mint-config">
## `mint config`
</div>

Gérer les valeurs par défaut persistantes pour les commandes du CLI. La configuration est enregistrée dans `~/.config/mintlify/config.json`.

```bash
mint config <subcommand> <key> [value]
```

| Subcommand | Description |
| --- | --- |
| `set <key> <value>` | Définir une valeur de configuration. |
| `get <key>` | Afficher une valeur de configuration. |
| `clear <key>` | Supprimer une valeur de configuration. |

<div id="configuration-keys">
### Clés de configuration
</div>

| Key | Description | Used by |
| --- | --- | --- |
| `subdomain` | Sous-domaine de documentation par défaut. | `mint analytics` |
| `dateFrom` | Date de début par défaut pour les requêtes d'analyse (`YYYY-MM-DD`). | `mint analytics` |
| `dateTo` | Date de fin par défaut pour les requêtes d'analyse (`YYYY-MM-DD`). | `mint analytics` |

<div id="mint-broken-links">
## `mint broken-links`
</div>

Vérifier les liens internes cassés dans votre documentation.

```bash
mint broken-links [flags]
```

La commande exclut les fichiers correspondant aux motifs [.mintignore](/fr/organize/mintignore). Les liens pointant vers des fichiers ignorés sont signalés comme cassés.

| Flag | Description |
| --- | --- |
| `--check-anchors` | Valider également les liens d'ancrage (par exemple, `/page#section`) par rapport aux slugs des titres. |
| `--check-external` | Vérifier également les URLs externes pour les liens cassés. |
| `--check-snippets` | Vérifier également les liens à l'intérieur des composants `<Snippet>`. |

<div id="mint-a11y">
## `mint a11y`
</div>

Vérifier les problèmes d'accessibilité dans votre documentation.

```bash
mint a11y [flags]
```

Vérifie les ratios de contraste des couleurs et le texte alternatif manquant sur les images et vidéos.

| Flag | Description |
| --- | --- |
| `--skip-contrast` | Ignorer les vérifications de contraste des couleurs. |
| `--skip-alt-text` | Ignorer les vérifications de texte alternatif manquant. |

<div id="mint-validate">
## `mint validate`
</div>

Valider la compilation de votre documentation en mode strict. Se termine avec une erreur s'il y a des avertissements ou des erreurs. Inclut la validation automatique des spécifications OpenAPI référencées dans votre `docs.json`.

```bash
mint validate [flags]
```

| Flag | Description |
| --- | --- |
| `--groups` | Liste de groupes d'utilisateurs séparés par des virgules à simuler pour la validation. |
| `--disable-openapi` | Ignorer le traitement et la validation des fichiers OpenAPI. |
| `--local-schema` | Autoriser la validation de fichiers OpenAPI hébergés localement servis via HTTP. Ne prend en charge que HTTPS en production. |

<Note>
  La commande autonome `mint openapi-check` est obsolète. Utilisez `mint validate` à la place.
</Note>

</div>

<div id="telemetry">
## Télémétrie
</div>

Le CLI collecte une télémétrie d'utilisation anonyme pour aider à améliorer Mintlify. Les données de télémétrie incluent le nom de la commande, la version du CLI, le système d'exploitation et l'architecture. Mintlify ne collecte **pas** d'informations personnellement identifiables, de contenu de projet ou de chemins de fichiers.

Par défaut, le CLI collecte des données de télémétrie. Vous pouvez vous désinscrire à tout moment en utilisant l'option `--telemetry` :

```bash
# Disable telemetry
mint --telemetry false

# Re-enable telemetry
mint --telemetry true
```

Vous pouvez également désactiver la télémétrie en définissant l'une de ces variables d'environnement :

| Variable | Value | Description |
| --- | --- | --- |
| `MINTLIFY_TELEMETRY_DISABLED` | `1` | Désactiver la télémétrie du CLI Mintlify. |
| `DO_NOT_TRACK` | `1` | Désactiver la télémétrie en utilisant le standard [Console Do Not Track](https://consoledonottrack.com/). |

Votre préférence est enregistrée dans `~/.config/mintlify/config.json` et persiste entre les sessions du CLI.
