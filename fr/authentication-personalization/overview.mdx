<Info>
  Les [offres Pro](https://mintlify.com/pricing?ref=authentication) incluent l’authentification par mot de passe.

  Les [offres Enterprise](https://mintlify.com/pricing?ref=authentication) incluent toutes les méthodes d’authentification.
</Info>

Choisissez l’une des trois approches pour gérer l’accès et adapter votre documentation en fonction des informations utilisateur et de vos besoins de sécurité :

* **Authentification complète** : protection totale de la confidentialité de tout le contenu, avec personnalisation complète.
* **Authentification partielle** : contrôle d’accès page par page, avec personnalisation complète.
* **Personnalisation** : personnalisation du contenu sans **aucune garantie de sécurité**. Tout le contenu reste accessible publiquement.

<div id="use-case-examples">
  ### Exemples de cas d’usage
</div>

**Choisissez l’authentification complète lorsque :**

* Vous créez une documentation interne contenant des informations sensibles
* Vous documentez des API propriétaires nécessitant la vérification des utilisateurs
* Vous rédigez des guides de mise en œuvre spécifiques aux clients

**Choisissez l’authentification partielle lorsque :**

* Vous proposez des guides de démarrage publics avec des fonctionnalités avancées privées
* Vous publiez une documentation open source avec des sections entreprise privées

**Choisissez la personnalisation lorsque :**

* Vous créez une documentation API publique qui affiche des exemples personnalisés selon l’utilisateur
* Vous créez des sites marketing qui personnalisent le contenu en fonction des profils utilisateurs
* Vous proposez des tutoriels publics qui s’adaptent aux préférences ou au niveau de compétence des utilisateurs

<div id="handshake-methods">
  ## Méthodes de négociation
</div>

Choisissez la méthode qui correspond le mieux à votre infrastructure existante et à vos exigences de sécurité.

| Méthode | Disponible pour | Complexité de configuration | Idéal pour |
|:-------|:-----------------|:----------------------------|:----------|
| **JWT (JSON Web Token)** | Toutes les approches | Moyenne | Parcours de connexion personnalisés, contrôle de sécurité maximal |
| **OAuth 2.0** | Toutes les approches | Élevée | Fournisseurs d’authentification tiers, sécurité d’entreprise |
| **Tableau de bord Mintlify** | Authentification uniquement | Faible | Équipes utilisant déjà le tableau de bord Mintlify |
| **Mot de passe** | Authentification uniquement | Faible | Accès partagé simple sans personnalisation |
| **Session partagée** | Personnalisation uniquement | Faible | Applications avec authentification existante basée sur des sessions |

<div id="when-to-use-each-method">
  ### Quand utiliser chaque méthode
</div>

**JWT (JSON Web Token)**: À utiliser si vous disposez d’un système d’authentification existant et souhaitez un contrôle total sur le parcours de connexion. Idéal pour une gestion utilisateur personnalisée ou lorsque vous devez découpler l’accès à la documentation de votre application principale.

**OAuth 2.0**: À utiliser si vous voulez tirer parti de fournisseurs d’authentification tiers (Google, GitHub, etc.) ou si vous avez besoin de normes de sécurité de niveau entreprise. Idéal pour les organisations utilisant déjà une infrastructure OAuth.

**Mintlify Dashboard**: À utiliser lorsque vos éditeurs de documentation sont aussi vos lecteurs de documentation. Parfait pour les équipes internes qui gèrent déjà le contenu via le Tableau de bord Mintlify.

**Mot de passe**: À utiliser pour un contrôle d’accès simple lorsque vous n’avez pas besoin de suivre les utilisateurs individuellement ni de personnaliser le contenu. Adapté pour les prestataires, les bêta‑testeurs ou des scénarios d’accès temporaire.

**Session partagée**: À utiliser si vous souhaitez une connexion transparente entre votre application et la documentation. Idéal lorsque les utilisateurs sont déjà authentifiés dans votre application principale et que vous souhaitez personnaliser leur expérience de la documentation.

<div id="content-customization">
  ## Personnalisation du contenu
</div>

Les trois approches prennent en charge des fonctionnalités de personnalisation du contenu, notamment le contenu MDX dynamique, le préremplissage de la clé API et les contrôles de visibilité des pages. Pour des instructions de mise en œuvre détaillées, consultez [Configuration de la personnalisation](/fr/authentication-personalization/personalization-setup).
