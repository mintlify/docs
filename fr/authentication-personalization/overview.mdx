  [Les offres Pro](https://mintlify.com/pricing?ref=authentication) incluent l’authentification par mot de passe.

  [Les offres Enterprise](https://mintlify.com/pricing?ref=authentication) incluent toutes les méthodes d’authentification.
</Info>

Choisissez l’une des trois approches pour gérer l’accès et personnaliser votre documentation en fonction des informations utilisateur et de vos besoins de sécurité :

* **Authentification complète** : Protection totale de la confidentialité pour l’ensemble du contenu, avec personnalisation complète.
* **Authentification partielle** : Contrôle d’accès page par page, avec personnalisation complète.
* **Personnalisation** : Personnalisation du contenu sans aucune garantie de sécurité. Tout le contenu reste publiquement accessible.

<div id="use-case-examples">
  ### Exemples de cas d’usage
</div>

**Choisissez l’authentification complète lorsque :**

* Vous créez une documentation interne d’entreprise contenant des informations sensibles
* Vous documentez des API propriétaires qui nécessitent une vérification des utilisateurs
* Vous créez des guides de mise en œuvre spécifiques à des clients

**Choisissez l’authentification partielle lorsque :**

* Vous proposez des guides de prise en main publics avec des fonctionnalités avancées privées
* Vous exploitez un produit freemium où les utilisateurs premium disposent de documentation supplémentaire
* Vous publiez une documentation open source avec des sections Enterprise privées

**Choisissez la personnalisation lorsque :**

* Vous créez une documentation d’API publique qui affiche des exemples spécifiques à chaque utilisateur
* Vous créez des sites marketing qui personnalisent le contenu en fonction des profils utilisateurs
* Vous proposez des tutoriels publics qui s’adaptent aux préférences ou au niveau de compétence des utilisateurs

<div id="handshake-methods">
  ## Méthodes d’établissement de session
</div>

Choisissez la méthode qui correspond le mieux à votre infrastructure et à vos exigences de sécurité.

| Méthode | Disponible pour | Complexité de configuration | Idéal pour |
|:-------|:-----------------|:----------------------------|:----------|
| **JWT** | Toutes les approches | Moyenne | Parcours de connexion personnalisés, contrôle maximal de la sécurité |
| **OAuth 2.0** | Toutes les approches | Élevée | Fournisseurs d’authentification tiers, sécurité de niveau entreprise |
| **Tableau de bord Mintlify** | Authentification uniquement | Faible | Équipes utilisant déjà le tableau de bord Mintlify |
| **Mot de passe** | Authentification uniquement | Faible | Accès partagé simple sans personnalisation |
| **Session partagée** | Personnalisation uniquement | Faible | Applications avec authentification par session existante |

<div id="when-to-use-each-method">
  ### Quand utiliser chaque méthode
</div>

**JWT** : À utiliser si vous disposez d’un système d’authentification existant et souhaitez un contrôle total sur le parcours de connexion. Idéal pour une gestion utilisateur personnalisée ou lorsque vous devez découpler l’accès à la documentation de votre application principale.

**OAuth 2.0** : À utiliser si vous souhaitez tirer parti de fournisseurs d’authentification tiers (Google, GitHub, etc.) ou si vous avez besoin de normes de sécurité de niveau entreprise. Idéal pour les organisations utilisant déjà une infrastructure OAuth.

**Tableau de bord Mintlify** : À utiliser lorsque vos éditeurs de documentation sont aussi vos lecteurs. Parfait pour les équipes internes qui gèrent déjà le contenu via le tableau de bord Mintlify.

**Mot de passe** : À utiliser pour un contrôle d’accès simple lorsque vous n’avez pas besoin de suivre les utilisateurs individuellement ni de personnaliser le contenu. Pratique pour les prestataires, les utilisateurs bêta ou des accès temporaires.

**Session partagée** : À utiliser lorsque vous souhaitez une connexion fluide entre votre application et la documentation. Idéal lorsque les utilisateurs sont déjà authentifiés dans votre application principale et que vous souhaitez personnaliser leur expérience de la documentation.

<div id="content-customization">
  ## Personnalisation du contenu
</div>

Les trois approches prennent en charge des fonctionnalités de personnalisation du contenu, notamment du contenu MDX dynamique, le préremplissage de clés API et le contrôle de la visibilité des pages. Pour des conseils de mise en œuvre détaillés, consultez [Configuration de la personnalisation](/fr/authentication-personalization/personalization-setup).
