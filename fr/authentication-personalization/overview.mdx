Choisissez parmi trois approches pour gérer l’accès et personnaliser votre documentation selon les informations des utilisateurs et vos besoins de sécurité :

- **Authentification complète** : Protection totale de la confidentialité pour l’ensemble du contenu, avec personnalisation complète.
- **Authentification partielle** : Contrôle d’accès page par page, avec personnalisation complète.
- **Personnalisation** : Personnalisation du contenu sans aucune garantie de sécurité. Tout le contenu reste accessible publiquement.

### Exemples d’utilisation

**Choisissez l’authentification complète lorsque :**
- Vous rédigez une documentation interne à l’entreprise contenant des informations sensibles
- Vous documentez des API propriétaires qui nécessitent une vérification des utilisateurs
- Vous créez des guides de mise en œuvre spécifiques à chaque client

**Choisissez l’authentification partielle lorsque :**
- Vous proposez des guides de démarrage publics avec des fonctionnalités avancées privées
- Vous gérez un produit freemium où les utilisateurs premium bénéficient d’une documentation supplémentaire
- Vous publiez une documentation open source avec des sections d’entreprise privées

**Choisissez la personnalisation lorsque :**
- Vous créez une documentation d’API publique qui présente des exemples spécifiques à chaque utilisateur
- Vous concevez des sites marketing qui personnalisent le contenu selon les profils utilisateurs
- Vous proposez des tutoriels publics qui s’adaptent aux préférences ou au niveau de compétence des utilisateurs

## Méthodes de handshake

Choisissez la méthode qui correspond le mieux à votre infrastructure existante et à vos exigences de sécurité.

| Méthode | Disponible pour | Complexité de configuration | Idéal pour |
|:-------|:--------------|:-----------------|:----------|
| **JWT** | Toutes les approches | Moyenne | Parcours de connexion personnalisés, contrôle maximal sur la sécurité |
| **OAuth 2.0** | Toutes les approches | Élevée | Fournisseurs d’authentification tiers, sécurité de niveau entreprise |
| **Tableau de bord Mintlify** | Authentification uniquement | Faible | Équipes utilisant déjà le tableau de bord Mintlify |
| **Mot de passe** | Authentification uniquement | Faible | Accès partagé simple sans personnalisation |
| **Session partagée** | Personnalisation uniquement | Faible | Applications avec authentification existante basée sur des sessions |

### Quand utiliser chaque méthode

**JWT** : À utiliser si vous disposez d’un système d’authentification existant et souhaitez un contrôle total sur le flux de connexion. Idéal pour une gestion utilisateur personnalisée ou lorsque vous devez découpler l’accès à la documentation de votre application principale.

**OAuth 2.0** : À utiliser si vous souhaitez tirer parti de fournisseurs d’authentification tiers (Google, GitHub, etc.) ou si vous avez besoin de normes de sécurité de niveau entreprise. Idéal pour les organisations utilisant déjà une infrastructure OAuth.

**Tableau de bord Mintlify** : À utiliser lorsque vos éditeurs de documentation sont aussi vos lecteurs de documentation. Parfait pour les équipes internes qui gèrent déjà le contenu via le tableau de bord Mintlify.

**Mot de passe** : À utiliser pour un contrôle d’accès simple lorsque vous n’avez pas besoin de suivre les utilisateurs individuellement ni de personnaliser le contenu. Adapté aux prestataires, aux bêta-testeurs ou aux scénarios d’accès temporaire.

**Session partagée** : À utiliser lorsque vous souhaitez une connexion transparente entre votre application et la documentation. Idéal lorsque les utilisateurs sont déjà authentifiés dans votre application principale et que vous voulez personnaliser leur expérience de la documentation.

## Personnalisation du contenu

Les trois approches prennent en charge des fonctionnalités de personnalisation, notamment du contenu MDX dynamique, le préremplissage de clés API et des contrôles de visibilité des pages. Pour des instructions de mise en œuvre détaillées, consultez [Configuration de la personnalisation](/fr/authentication-personalization/personalization-setup).
