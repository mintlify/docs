Après avoir terminé ce guide, vous disposerez d&#39;un site de documentation en ligne prêt à être personnalisé et mis à jour.

<div id="before-you-begin">
  ## Avant de commencer
</div>

Mintlify utilise une approche *docs-as-code* pour gérer votre documentation. Chaque page de votre site possède un fichier correspondant, stocké dans votre <Tooltip headline="Référentiel" tip="Le code source de votre documentation où tous les fichiers et leur historique sont stockés. L'éditeur web se connecte à votre référentiel de documentation pour accéder au contenu et le modifier, ou vous pouvez modifier les fichiers localement dans votre IDE préféré.">référentiel</Tooltip> de documentation. Vous pouvez utiliser votre propre référentiel Git ou un référentiel hébergé par Mintlify.

Choisissez l&#39;approche de référentiel qui vous convient le mieux.

<CardGroup cols={2}>
  <Card title="Hébergé par Mintlify" icon="cloud">
    Mintlify crée un référentiel GitHub privé connecté à votre déploiement Mintlify.

    Choisissez cette option si :

    * Vous travaillerez uniquement dans l&#39;éditeur web et ne voulez pas utiliser Git localement.
    * Vous voulez démarrer rapidement sans connecter de référentiel.
    * Vous voulez que Mintlify gère votre référentiel de documentation pour vous.
  </Card>

  <Card title="Votre propre référentiel" icon="github">
    Connectez votre propre référentiel Git à votre déploiement Mintlify.

    Choisissez cette option si :

    * Vous travaillerez localement et voulez synchroniser avec votre référentiel distant.
    * Vous avez déjà un référentiel pour votre documentation.
    * Vous voulez que votre documentation soit sous votre propre organisation GitHub.
  </Card>
</CardGroup>

<Tip>
  Vous ne savez pas quoi choisir ? Commencez avec **Hébergé par Mintlify**. Vous pourrez migrer vers votre propre référentiel à tout moment si cela correspond mieux à vos besoins.
</Tip>

<div id="deploy-your-documentation-site">
  ## Déployez votre site de documentation
</div>

<Tabs>
  <Tab title="Hébergé par Mintlify">
    <Steps>
      <Step title="Créez votre compte">
        Allez sur [mintlify.com/start](https://mintlify.com/start) et créez votre compte.
      </Step>

      <Step title="Ignorer la connexion à GitHub">
        Lorsque vous êtes invité à connecter GitHub, cliquez sur **Skip** pour utiliser un référentiel hébergé par Mintlify.
      </Step>

      <Step title="Terminer l'onboarding">
        Suivez les étapes d&#39;onboarding pour configurer votre documentation. Mintlify crée automatiquement un référentiel privé dans l&#39;organisation `mintlify-community` et configure la GitHub App pour vous.
      </Step>

      <Step title="Votre site est déployé">
        Votre documentation est immédiatement déployée et accessible à votre URL `.mintlify.app`.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Votre propre référentiel">
    <Steps>
      <Step title="Créez votre compte">
        Allez sur [mintlify.com/start](https://mintlify.com/start) et créez votre compte.
      </Step>

      <Step title="Connecter GitHub">
        Lorsque vous y êtes invité, connectez-vous à votre compte GitHub et authentifiez-vous.
      </Step>

      <Step title="Terminer l'onboarding">
        Suivez les étapes d&#39;onboarding restantes pour créer un référentiel et accéder à votre Dashboard.
      </Step>

      <Step title="Installer la GitHub App">
        Installez la [GitHub App](https://dashboard.mintlify.com/api/github/install-app) de Mintlify pour activer les déploiements automatiques. Cela permet à Mintlify de se synchroniser avec votre référentiel et de déployer automatiquement les modifications lorsque vous faites un push.
      </Step>

      <Step title="Facultatif : sélectionner un référentiel">
        Si vous souhaitez utiliser un autre référentiel que celui créé pendant l&#39;onboarding pour votre documentation, sélectionnez-le sur la page [Paramètres Git](https://dashboard.mintlify.com/settings/deployment/git-settings) de votre Dashboard. Votre référentiel de documentation doit contenir un fichier `docs.json` valide pour être déployé.
      </Step>

      <Step title="Votre site est déployé">
        Votre documentation est déployée et accessible à votre URL `.mintlify.app`. Les prochaines modifications que vous pousserez vers votre référentiel déclencheront automatiquement de nouveaux déploiements.
      </Step>
    </Steps>

    <Info>
      Pour en savoir plus sur la GitHub App et ses autorisations, consultez [GitHub](/fr/deploy/github).
    </Info>
  </Tab>
</Tabs>

<div id="view-your-deployed-site">
  ## Voir votre site déployé
</div>

Votre site de documentation est maintenant déployé à l’adresse `https://<your-project-name>.mintlify.app`.

Trouvez l’URL exacte de votre site sur la page **Overview** de votre [Tableau de bord Mintlify](https://dashboard.mintlify.com/).

  Votre site est immédiatement prêt à être consulté. Utilisez cette URL pour effectuer des tests et la partager avec votre équipe. Avant de le partager avec vos utilisateurs, vous voudrez peut-être ajouter un [domaine personnalisé](/fr/customize/custom-domain).
</Tip>

<div id="make-your-first-change">
  ## Effectuer votre première modification
</div>

<Steps>
  <Step title="Ouvrir l’éditeur web">
    Accédez à l’[éditeur web](https://dashboard.mintlify.com/editor) depuis votre Dashboard.
  </Step>

  <Step title="Modifier une page">
    Ouvrez `index.mdx` et modifiez la description dans le frontmatter :

    ```mdx
    ---
    title: "Introduction"
    description: "Votre description personnalisée ici"
    ---
    ```
  </Step>

  <Step title="Publier">
    Cliquez sur le bouton **Publish** en haut à droite de la barre d’outils de l’éditeur web.
  </Step>

  <Step title="Afficher en ligne">
    Sur la page [Overview](https://dashboard.mintlify.com/) de votre Dashboard, vous pouvez voir l’état de déploiement de votre site. Une fois le déploiement terminé, actualisez votre site de documentation pour voir vos modifications en ligne.
  </Step>
</Steps>
