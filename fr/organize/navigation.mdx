La propriété [navigation](/fr/organize/settings#param-navigation) dans `docs.json` détermine la structure et la hiérarchie de l’information de votre documentation.

Avec une configuration de navigation adéquate, vous pouvez organiser votre contenu pour que les utilisateurs trouvent exactement ce qu’ils recherchent.

<div id="pages">
  ## Pages
</div>

Les pages sont le composant de navigation le plus fondamental. Elles correspondent aux fichiers MDX qui composent votre documentation.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/pages-light.png" alt="Graphique décoratif représentant des pages." />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/pages-dark.png" alt="Graphique décoratif représentant des pages." />

Dans l’objet `navigation`, `pages` est un tableau dans lequel chaque entrée doit référencer le chemin d’accès vers un [fichier de page](/fr/organize/pages).

```json
{
  "navigation": {
    "pages": [
      "settings",
      "pages",
      "navigation",
      "themes",
      "custom-domain"
    ]
  }
}
```

<div id="groups">
  ## Groupes
</div>

Utilisez des groupes pour organiser la navigation de votre barre latérale en sections. Les groupes peuvent être imbriqués, accompagnés de tags et stylisés avec des icônes.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/groups-light.png" alt="Graphique décoratif représentant des groupes." />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/groups-dark.png" alt="Graphique décoratif représentant des groupes." />

Dans l’objet `navigation`, `groups` est un tableau où chaque entrée est un objet qui doit contenir un champ `group` et un champ `pages`. Les champs `icon`, `tag` et `expanded` sont facultatifs.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Premiers pas",
        "icon": "play",
        "expanded": false,
        "pages": [
          "quickstart",
          {
            "group": "Édition",
            "icon": "pencil",
            "pages": [
              "installation",
              "editor"
            ]
          }
        ]
      },
      {
        "group": "Rédaction de contenu",
        "icon": "notebook-text",
        "tag": "NOUVEAU",
        "pages": [
          "writing-content/page",
          "writing-content/text"
        ]
      }
    ]
  }
}
```

<div id="default-expanded-state">
  ### État développé par défaut
</div>

Définissez `expanded: true` sur un groupe pour qu’il soit développé par défaut dans la barre latérale de navigation. Cela est utile pour mettre en avant des sections importantes ou améliorer la découverte du contenu clé.

```json
{
  "group": "Prise en main",
  "expanded": true,
  "pages": ["quickstart", "installation"]
}
```

<div id="tabs">
  ## Tabs
</div>

Les Tabs créent des sections distinctes de votre documentation avec des chemins d’URL séparés. Les Tabs génèrent une barre de navigation horizontale en haut de votre documentation qui permet aux utilisateurs de passer d’une section à l’autre.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/tabs-light.png" alt="Graphique décoratif d’une navigation par onglets." />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/tabs-dark.png" alt="Graphique décoratif d’une navigation par onglets." />

Dans l’objet `navigation`, `tabs` est un tableau où chaque entrée est un objet qui requiert un champ `tab` et peut contenir d’autres champs de navigation tels que des groupes, des pages, des icons ou des liens vers des pages externes.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Référence API",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "tab": "SDK",
        "icon": "code",
        "pages": [
          "sdk/fetch",
          "sdk/create",
          "sdk/delete"
        ]
      },
      {
        "tab": "Blog",
        "icon": "newspaper",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="menus">
  ### Menus
</div>

Les menus ajoutent des éléments de navigation déroulants à un onglet. Utilisez-les pour aider les utilisateurs à accéder directement à des pages spécifiques au sein d’un onglet.

Dans l’objet `navigation`, `menu` est un tableau où chaque entrée est un objet qui requiert un champ `item` et peut contenir d’autres champs de navigation tels que des groupes, des pages, des icon, ou des liens vers des pages externes.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Outils de développeur",
        "icon": "square-terminal",
        "menu": [
          {
            "item": "API reference",
            "icon": "rocket",
            "groups": [
              {
                "group": "Points de terminaison principaux",
                "icon": "square-terminal",
                "pages": [
                  "api-reference/get",
                  "api-reference/post",
                  "api-reference/delete"
                ]
              }
            ]
          },
          {
            "item": "SDKs",
            "icon": "code",
            "description": "Les SDK sont utilisés pour interagir avec l'API.",
            "pages": [
              "sdk/fetch",
              "sdk/create",
              "sdk/delete"
            ]
          }
        ]
      }
    ]
  }
}
```

<div id="anchors">
  ## Ancres
</div>

Les ancres ajoutent des éléments de navigation persistants en haut de votre barre latérale. Utilisez-les pour structurer votre contenu, offrir un accès rapide à des ressources externes ou créer des appels à l’action bien visibles.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/anchors-light.png" alt="Image décorative d’une navigation avec ancres." />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/anchors-dark.png" alt="Image décorative d’une navigation avec ancres." />

Dans l’objet `navigation`, `anchors` est un tableau où chaque entrée est un objet qui exige un champ `anchor` et peut contenir d’autres champs de navigation tels que des groupes, des pages, des icônes ou des liens vers des pages externes.

```json
{
  "navigation": {
    "anchors": [
      {
        "anchor": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "anchor": "Référence API",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "anchor": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

Pour les ancres qui pointent uniquement vers des liens externes, utilisez le mot‑clé `global`. Les ancres dans un objet `global` doivent comporter un champ `href` et ne peuvent pas pointer vers un chemin relatif.

Les ancres globales sont particulièrement utiles pour créer des liens vers des ressources qui ne font pas partie de votre documentation mais devraient rester facilement accessibles à vos utilisateurs, comme un blog ou un portail d’assistance.

```json
{
  "navigation": {
    "global":  {
      "anchors": [
        {
          "anchor": "Communauté",
          "icon": "house",
          "href": "https://slack.com"
        },
        {
          "anchor": "Blog",
          "icon": "pencil",
          "href": "https://mintlify.com/blog"
        }
      ]
    },
    "tabs": /*...*/
  }
}
```

<div id="dropdowns">
  ## Menus déroulants
</div>

Les menus déroulants se trouvent dans un menu extensible en haut de la navigation de votre barre latérale. Chaque élément d’un menu déroulant mène à une section de votre documentation.

```json
      {
        "dropdown": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"

        "dropdown": "Référence API",
```

<div id="openapi">
  ## OpenAPI
</div>

Intégrez des spécifications OpenAPI directement dans votre structure de navigation pour générer automatiquement la documentation de votre API. Créez des sections API dédiées ou placez des pages d’endpoints au sein d’autres éléments de navigation.

Définissez une spécification OpenAPI par défaut à n’importe quel niveau de votre hiérarchie de navigation. Les éléments enfants hériteront de cette spécification, sauf s’ils définissent la leur.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Référence API",
        "openapi": "/path/to/openapi-v1.json",
        "pages": [
          "overview",
          "authentication",
          "GET /users",
          "POST /users",
          {
            "group": "Produits",
            "openapi": "/path/to/openapi-v2.json",
            "pages": [
              "GET /products",
              "POST /products"
            ]
          }
        ]
      }
    ]
  }
}
```

Pour en savoir plus sur la façon de référencer des endpoints OpenAPI dans votre documentation, consultez la page [OpenAPI setup](/fr/api-playground/openapi-setup).

<div id="versions">
  ## Versions
</div>

Divisez votre navigation en différentes versions. Les versions peuvent être sélectionnées à partir d’un menu déroulant.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/versions-light.png" alt="Illustration décorative d’un sélecteur de version." />

Dans l’objet `navigation`, `versions` est un tableau dans lequel chaque entrée est un objet qui requiert un champ `version` et peut contenir n’importe quels autres champs de navigation.

```json
{
  "navigation": {
    "versions": [
      {
        "version": "1.0.0",
        "groups": [
          {
            "group": "Premiers pas",
            "pages": ["v1/overview", "v1/quickstart", "v1/development"]
          }
        ]
      },
      {
        "version": "2.0.0",
        "groups": [
          {
            "group": "Premiers pas",
            "pages": ["v2/overview", "v2/quickstart", "v2/development"]
          }
        ]
      }
    ]
  }
}
```

<div id="breadcrumbs">
  ## Fil d’Ariane
</div>

Le fil d’Ariane affiche le chemin de navigation complet en haut des pages. Certains thèmes activent le fil d’Ariane par défaut, d’autres non. Vous pouvez contrôler son activation pour votre site à l’aide de la propriété `styling` dans votre `docs.json`.

<CodeGroup>
  ```json Display full breadcrumbs
  "styling": {
    "eyebrows": "breadcrumbs"
  }
  ```

  ```json Display parent section only
  "styling": {
    "eyebrows": "section"
  }
  ```
</CodeGroup>

<div id="interaction-configuration">
  ## Configuration des interactions
</div>

Contrôlez la manière dont les utilisateurs interagissent avec les éléments de navigation à l&#39;aide de la propriété `interaction` dans votre `docs.json`.

<div id="enable-auto-navigation-for-groups">
  ### Activer la navigation automatique pour les groupes
</div>

Lorsqu’un utilisateur ouvre un groupe de navigation, certains thèmes redirigent automatiquement vers la première page du groupe. Vous pouvez modifier le comportement par défaut d’un thème avec l’option `drilldown`.

* Définissez `true` pour forcer la navigation automatique vers la première page lorsqu’un groupe de navigation est sélectionné.
* Définissez `false` pour empêcher la navigation et uniquement développer ou réduire le groupe lorsqu’il est sélectionné.
* Laissez la valeur non définie pour utiliser le comportement par défaut du thème.

<CodeGroup>
  ```json Forcer la navigation
  "interaction": {
    "drilldown": true  // Forcer la navigation vers la première page lorsqu’un utilisateur ouvre un menu déroulant
  }
  ```

  ```json Empêcher la navigation
  "interaction": {
    "drilldown": false // Ne jamais naviguer, uniquement développer ou réduire le groupe
  }
  ```
</CodeGroup>
