<div id="prerequisites">
  ## Prérequis
</div>

- Un projet Mintlify connecté à un dépôt GitHub ou GitLab
- Pour GitHub : l'[application GitHub Mintlify](/fr/deploy/github) installée sur chaque dépôt que vous prévoyez d'utiliser dans le workflow
- Pour GitLab : un compte GitLab connecté (voir [Configuration GitLab](#gitlab-setup) ci-dessous)

<div id="create-a-custom-workflow">
  ## Créer un workflow personnalisé
</div>

<Note>
  L'agent s'exécute dans un sandbox isolé avec un accès Internet restreint. L'agent ne peut pas installer de paquets ou d'outils supplémentaires à l'exécution. Les invites qui font référence à des outils non disponibles peuvent produire des résultats inattendus ou échouer.
</Note>

1. Ouvrez la page [Workflows](https://dashboard.mintlify.com/products/workflows?tab=workflows) dans votre tableau de bord.
2. Cliquez sur **Créer un workflow personnalisé**.
3. Configurez le nom du workflow, son moment d'exécution, ses instructions et la manière dont il effectue les mises à jour.
4. Cliquez sur **Créer un workflow**.

<div id="best-practices">
  ## Bonnes pratiques
</div>

Des instructions de workflow efficaces se concentrent sur une seule tâche et définissent un résultat clair et vérifiable. Les workflows présentent une certaine variance car les agents sont non-déterministes, mais des instructions cohérentes produisent des résultats cohérents.

- **Décrivez le résultat.** Dites à l'agent à quoi devrait ressembler le résultat. Décrivez ce que signifie « terminé ».
- **Rédigez une procédure.** Donnez à l'agent un guide étape par étape à suivre.
- **Spécifiez le contexte.** Référencez les fichiers, répertoires ou dépôts que l'agent doit utiliser.
- **Gardez la portée étroite.** Divisez les tâches complexes sur plusieurs workflows plutôt que de les combiner en une seule invite.

<Tip>
  Examinez la sortie de vos workflows personnalisés pour vous assurer qu'ils fonctionnent comme prévu. Mettez à jour vos instructions pour améliorer les résultats.
</Tip>
