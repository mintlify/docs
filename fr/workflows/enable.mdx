<div id="prerequisites">
  ## Prérequis
</div>

- Un projet Mintlify connecté à un dépôt GitHub ou GitLab
- Pour GitHub : l'[application GitHub Mintlify](/fr/deploy/github) installée sur chaque dépôt que vous prévoyez d'utiliser dans le workflow
- Pour GitLab : un compte GitLab connecté (voir [Configuration GitLab](#gitlab-setup) ci-dessous)

<div id="enable-workflows">
  ## Activer les workflows
</div>

1. Ouvrez la page [Workflows](https://dashboard.mintlify.com/products/workflows?tab=workflows) dans votre tableau de bord.
2. Cliquez sur l'interrupteur à côté d'un workflow pour l'activer.
    <Frame>
      <img src="/images/workflows/workflows-dashboard-light.png" alt="Le tableau de bord des workflows." className="block dark:hidden" />
      <img src="/images/workflows/workflows-dashboard-dark.png" alt="Le tableau de bord des workflows." className="hidden dark:block" />
    </Frame>
3. Choisissez les configurations requises.
    <Frame>
      <img src="/images/workflows/workflow-settings-light.png" alt="Les options de configuration pour le workflow de mises à jour de la base de code." className="block dark:hidden" />
      <img src="/images/workflows/workflow-settings-dark.png" alt="Les options de configuration pour le workflow de mises à jour de la base de code." className="hidden dark:block" />
    </Frame>
4. Cliquez sur **Activer le workflow**.

<div id="configurations">
  ## Configurations
</div>

<div id="trigger-repositories">
  ### Dépôts de déclenchement
</div>

Pour les workflows qui s'exécutent en réponse à des événements push, spécifiez les dépôts à surveiller. Vous pouvez spécifier jusqu'à 10 dépôts par workflow.

Pour les dépôts GitHub, vous devez installer l'application GitHub Mintlify sur chaque dépôt. Ajoutez des dépôts sur la page de paramètres de l'[application GitHub](https://dashboard.mintlify.com/settings/organization/github-app).

<div id="update-mode">
  ### Mode de mise à jour
</div>

Chaque workflow a un mode par défaut pour la manière dont il effectue les mises à jour, soit en mettant directement à jour votre dépôt de contenu, soit en ouvrant une pull request.

Si vous voulez exiger des révisions avant qu'un workflow ne mette à jour votre contenu, sélectionnez **Exiger une révision**.

<Note>
  Pour les dépôts GitHub, les mises à jour automatiques nécessitent que l'application GitHub Mintlify dispose des permissions de contournement sur chaque ensemble de règles ciblant votre branche de déploiement, y compris les ensembles de règles au niveau de l'organisation et du dépôt. Consultez [Configurer l'automerge](/fr/guides/configure-automerge) pour les instructions de configuration.

  Pour les dépôts GitLab, l'automerge utilise la connexion OAuth GitLab et nécessite au moins le rôle Maintainer sur chaque projet.
</Note>

<div id="trigger">
  ### Déclencheur
</div>

Chaque workflow a une valeur par défaut pour son moment d'exécution, soit en réponse à des modifications dans votre dépôt de contenu, soit en réponse à des modifications dans un dépôt de déclenchement, soit selon un planning.

Si vous souhaitez modifier la valeur par défaut, sélectionnez un déclencheur différent.

<div id="instructions">
  ### Instructions
</div>

Ajoutez des instructions optionnelles ajoutées à l'invite de base du workflow. Utilisez ces instructions pour ajuster le style, le ton ou d'autres aspects propres à votre projet.
