<div id="prerequisites">
  ## Prérequis
</div>

- Un projet Mintlify connecté à un dépôt GitHub ou GitLab
- Pour GitHub : l'[application GitHub Mintlify](/fr/deploy/github) installée sur chaque dépôt que vous prévoyez d'utiliser dans le workflow
- Pour GitLab : un compte GitLab connecté (voir [Configuration GitLab](#gitlab-setup) ci-dessous)

<div id="enable-workflows">
  ## Activer les workflows
</div>

1. Ouvrez la page [Workflows](https://app.mintlify.com/products/workflows?tab=workflows) dans votre tableau de bord.
2. Cliquez sur l'interrupteur à côté d'un workflow pour l'activer.
    <Frame>
      <img src="/images/workflows/workflows-dashboard-light.png" alt="Le tableau de bord des workflows." className="block dark:hidden" />
      <img src="/images/workflows/workflows-dashboard-dark.png" alt="Le tableau de bord des workflows." className="hidden dark:block" />
    </Frame>

    Si le workflow peut s'exécuter avec les paramètres par défaut, il s'active immédiatement. Sinon, la fenêtre de paramètres s'ouvre pour vous permettre de choisir les configurations requises.
    <Frame>
      <img src="/images/workflows/workflow-settings-light.png" alt="Les options de configuration du workflow de mises à jour de la base de code." className="block dark:hidden" />
      <img src="/images/workflows/workflow-settings-dark.png" alt="Les options de configuration du workflow de mises à jour de la base de code." className="hidden dark:block" />
    </Frame>
3. Si la fenêtre s'ouvre, remplissez les champs requis et cliquez sur **Turn on workflow**.

    Pour modifier les paramètres d'un workflow actif, cliquez sur le bouton <Icon icon="settings" /> de paramètres à côté de celui-ci. Utilisez l'interrupteur dans l'en-tête de la fenêtre pour activer ou désactiver le workflow sans quitter la vue des paramètres.

<div id="configurations">
  ## Configurations
</div>

<div id="trigger-repositories">
  ### Dépôts de déclenchement
</div>

Pour les workflows qui s'exécutent sur des événements push, spécifiez les dépôts à surveiller. Vous pouvez spécifier jusqu'à 10 dépôts par workflow.

Pour les dépôts GitHub, vous devez installer l'application GitHub Mintlify sur chaque dépôt. Ajoutez des dépôts sur la page de paramètres de l'[application GitHub](https://app.mintlify.com/settings/organization/github-app).

<div id="update-mode">
  ### Mode de mise à jour
</div>

Chaque workflow possède un mode par défaut pour effectuer les mises à jour, soit en mettant à jour directement votre dépôt de contenu, soit en ouvrant une pull request.

Si vous souhaitez exiger une révision avant qu'un workflow ne mette à jour votre contenu, cliquez sur **Require review**.

<Note>
  Pour les dépôts GitHub, les mises à jour automatiques nécessitent que l'application GitHub Mintlify dispose de permissions de contournement sur chaque ensemble de règles ciblant votre branche de déploiement, y compris les ensembles de règles au niveau de l'organisation et du dépôt. Consultez [Configurer la fusion automatique](/fr/guides/configure-automerge) pour les instructions de configuration.

  Pour les dépôts GitLab, la fusion automatique utilise la connexion OAuth de GitLab et nécessite au moins le rôle Mainteneur sur chaque projet.
</Note>

<div id="trigger">
  ### Déclencheur
</div>

Chaque workflow a une valeur par défaut quant au moment de son exécution, soit en réponse à des changements dans votre dépôt de contenu, soit en réponse à des changements dans un dépôt de déclenchement, soit selon une planification.

Si vous souhaitez modifier la valeur par défaut, sélectionnez un autre déclencheur.

<div id="instructions">
  ### Instructions
</div>

Ajoutez des instructions facultatives qui seront ajoutées au prompt de base du workflow. Utilisez ces instructions pour ajuster le style, le ton ou d'autres aspects propres à votre projet.

<div id="target-languages">
  ### Langues cibles
</div>

Lorsque vous activez le workflow **Translate content**, sélectionnez une ou plusieurs langues que vous souhaitez maintenir synchronisées avec votre contenu source.

- Mintlify lit les [langues](/fr/organize/navigation#languages) définies dans votre `docs.json` pour identifier votre langue par défaut et présélectionne toutes les langues cibles déjà configurées.
- Vous devez sélectionner au moins une langue cible pour enregistrer le workflow.
- Vous ne pouvez pas sélectionner la langue source comme langue cible.

Vous pouvez modifier les langues cibles à tout moment en ouvrant les paramètres du workflow et en modifiant le champ **Translate to**.

<div id="run-a-translation-immediately">
  #### Exécuter une traduction immédiatement
</div>

Après avoir enregistré le workflow, cliquez sur **Translate now** pour l'exécuter à la demande en utilisant les langues cibles actuellement enregistrées. Le workflow doit être activé pour déclencher une exécution manuelle. Les exécutions manuelles n'affectent pas le déclencheur régulier du workflow.

<div id="improve-docs-from-assistant-chats">
  ## Améliorer la documentation à partir des conversations avec l'assistant
</div>

Le workflow **Improve docs from assistant chats** examine les tendances récentes dans les questions posées à l'[assistant](/fr/assistant/index) et ouvre des mises à jour ciblées de la documentation qui comblent les lacunes rencontrées par vos utilisateurs.

Utilisez ce workflow lorsque vous souhaitez que l'agent trie l'activité de l'assistant à votre place et propose des modifications qui répondent aux questions que les utilisateurs posent réellement.

Ce workflow s'exécute selon une planification. Il ne nécessite pas de dépôts de déclenchement ni de configuration de contexte supplémentaire. Activez-le depuis la page [Workflows](https://app.mintlify.com/products/workflows?tab=workflows) et choisissez si les mises à jour doivent être fusionnées automatiquement ou ouvrir une pull request pour révision.

<div id="improve-docs-from-feedback">
  ## Améliorer la documentation à partir des retours
</div>

Le workflow **Improve docs from feedback** examine les [retours sur les pages](/fr/optimize/feedback) récents et ouvre des mises à jour ciblées de la documentation basées sur les problèmes signalés par les utilisateurs.

Utilisez ce workflow lorsque vous collectez des évaluations par pouces ou des retours contextuels et que vous souhaitez que l'agent convertisse ce signal en modifications concrètes de la documentation.

Lorsqu'une exécution se termine, les détails de l'exécution dans votre tableau de bord incluent les citations spécifiques des retours utilisées par l'agent. Chaque citation renvoie au retour dans votre [tableau de bord d'analyse](https://app.mintlify.com/products/analytics/v2/feedback) pour que vous puissiez vérifier le contexte qui a motivé la mise à jour.

Ce workflow s'exécute selon une planification et nécessite que la collecte des retours soit activée sur la page [Add-ons](https://app.mintlify.com/products/addons). Il ne nécessite pas de dépôts de déclenchement ni de configuration de contexte supplémentaire.
