Les workflows exécutent l'agent automatiquement selon une planification ou en réponse à des changements dans un dépôt. Chaque workflow définit un prompt pour l'agent et un trigger qui indique quand l'exécuter. Les workflows prennent en charge à la fois les dépôts GitHub et GitLab.

Lorsqu'un workflow s'exécute, l'agent lit le contenu de votre projet et les dépôts connectés, puis suit le prompt pour effectuer les mises à jour.

<Tip>
Utilisez des workflows qui s'exécutent selon une planification pour les tâches récurrentes, comme la publication de changelogs ou l'audit des métadonnées SEO.

Utilisez des workflows qui se déclenchent sur des changements de contenu ou de code pour la maintenance réactive, comme les traductions ou la mise à jour du contenu lorsque le code source change.
</Tip>

<div id="trigger-types">
  ## Types de trigger
</div>

Les workflows prennent en charge trois types de trigger.

- **Mise à jour de contenu**—S'exécute lorsque des changements de contenu sont poussés vers votre dépôt de projet Mintlify, y compris les fusions de pull requests et les pushes directs.
- **Changement de code**—S'exécute lorsqu'une pull request est fusionnée dans un dépôt de code source connecté.
- **Planification personnalisée**—S'exécute selon une planification récurrente. Les workflows sont mis en file d'attente dans les 10 minutes suivant l'heure planifiée.

Chaque workflow a exactement un trigger. Vous ne pouvez pas combiner des types de trigger dans un seul workflow.

<div id="usage-limits">
  ## Limites d'utilisation
</div>

Les exécutions de workflows comptent dans votre utilisation de crédits. Consultez votre utilisation de crédits, configurez les dépassements et gérez votre forfait de crédits sur la page [Utilisation](https://app.mintlify.com/settings/organization/usage) de votre tableau de bord.

Chaque workflow peut s'exécuter jusqu'à 500 fois par jour. Les exécutions en échec ne comptent pas dans cette limite.
