<Info>
  Pour collecter et consulter des commentaires, vous devez d’abord activer la fonctionnalité depuis la page [Modules complémentaires](https://dashboard.mintlify.com/products/addons) de votre Dashboard.
</Info>

L’onglet Commentaires affiche le nombre de votes positifs et négatifs que vos docs ont reçus, ainsi que tout retour qualitatif fourni par les utilisateurs. Servez-vous de ces informations pour évaluer la qualité de vos docs et les améliorer.

Accédez à l’onglet Commentaires en vous rendant sur la page **Insights** de votre [Dashboard](https://dashboard.mintlify.com/products/insights).

* **Évaluation par pouces uniquement** : Vote simple (pouce levé/baissé) pour mesurer la satisfaction générale des pages.
* **Feedback sur les extraits de code uniquement** : Feedback spécifiquement sur les extraits de code.

* **Évaluation par pouces et feedback sur les extraits de code** : Vote sur la page, plus feedback sur les exemples de code.
* **Évaluation par pouces, contextuel et sur les extraits de code** : Système de feedback complet avec vote sur la page, commentaires détaillés et feedback sur les extraits de code.

<div id="managing-feedback">
  ## Gestion des retours
</div>

Pour les retours contextuels et sur des extraits de code, vous pouvez définir le statut d’un retour et ajouter des notes internes afin de suivre l’avancement de la résolution des retours des utilisateurs.

<div id="changing-feedback-status">
  ### Modifier le statut des retours
</div>

Sélectionnez le statut à côté d’un retour pour le marquer comme **En attente**, **En cours**, **Résolu** ou **Rejeté**.

Bonnes pratiques pour la définition des statuts des retours :

* **En attente** : le retour est en attente de revue.
* **En cours** : le retour a été validé et est en cours de traitement.
* **Résolu** : le retour a été résolu.
* **Rejeté** : le retour a été écarté car non exploitable, non pertinent ou inexact.

<div id="filtering-by-status">
  ### Filtrer par statut
</div>

Utilisez le filtre de statut pour contrôler les commentaires affichés. Supprimez un statut pour masquer tous les commentaires ayant ce statut. Par défaut, tous les commentaires sont affichés.

<div id="adding-internal-notes">
  ### Ajout de notes internes
</div>

Cliquez sur un commentaire pour ajouter une note interne. Ces notes ne sont visibles que par les personnes ayant accès à votre Dashboard.

Utilisez les notes pour fournir des informations destinées à la collaboration, associer des tickets de support ou d’ingénierie pertinents, ou consigner toute autre information utile.

<div id="using-feedback-data">
  ## Exploiter les données de retours
</div>

Passez en revue vos données de retours pour :

* **Identifier le contenu efficace** : Les pages qui reçoivent le plus d’avis positifs montrent ce qui fonctionne bien dans votre documentation.
  * **Prioriser les améliorations** : Les pages qui reçoivent le plus d’avis négatifs indiquent quels contenus nécessitent une attention particulière.
  * **Passer à l’action** : Mettez à jour la documentation sur la base des retours directs des utilisateurs.
