<div id="enabling-authentication">
  ## Activer l’authentification
</div>

Vous pouvez ajouter une méthode d’authentification dans votre `docs.json` pour l’activer globalement sur chaque page, ou la définir au cas par cas.

La méthode d’authentification d’une page remplacera la méthode globale si les deux sont définies.

<div id="bearer-token">
  ### Jeton d’authentification Bearer
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "bearer"
      }
    }
}
```

```mdx Page Metadata
---
title: "Titre de votre page"
authMethod: "bearer"
---
```

</CodeGroup>

<div id="api-key">
  ### Clé d’API
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "key",
        "name": "x-api-key"
      }
    }
}
```

```mdx Page Metadata
---
title: "Titre de votre page"
authMethod: "key"
---
```

</CodeGroup>

<div id="none">
  ### Aucune
</div>

La méthode d’authentification « none » est utile pour désactiver l’authentification sur un point de terminaison spécifique après avoir défini une valeur par défaut dans docs.json.

<CodeGroup>
```mdx Page Metadata
---
title: "Your page title"
authMethod: "none"
---
```
</CodeGroup>
