## Activer l’authentification

Vous pouvez ajouter une méthode d’authentification à votre `docs.json` pour l’activer globalement sur chaque page, ou la définir au cas par case.

La méthode d’authentification définie sur une page remplacera la méthode globale si les deux sont configurées.

### Jeton Bearer

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "bearer"
      }
    }
}
```

```mdx Page Metadata
---
title: "Your page title"
authMethod: "bearer"
---
```

</CodeGroup>

### Authentification Basic

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "basic"
      }
    }
}
```

```mdx Page Metadata
---
title: "Your page title"
authMethod: "basic"
---
```

</CodeGroup>

### Clé API

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "key",
        "name": "x-api-key"
      }
    }
}
```

```mdx Page Metadata
---
title: "Your page title"
authMethod: "key"
---
```

</CodeGroup>

### Aucune

La méthode d’authentification « none » est utile pour désactiver l’authentification sur un endpoint spécifique après avoir défini une valeur par défaut dans docs.json.

<CodeGroup>
```mdx Page Metadata
---
title: "Your page title"
authMethod: "none"
---
```
</CodeGroup>
