<Info>
  Les [offres Pro et Enterprise](https://mintlify.com/pricing?ref=docs-ci) incluent des contrôles CI pour les dépôts GitHub.
</Info>

Utilisez les contrôles CI pour analyser votre documentation, détecter les erreurs et afficher des avertissements avant le déploiement. Les contrôles CI sont configurés pour s’exécuter sur les commits vers la branche de déploiement configurée, ou sur les pull requests ciblant cette branche.

## Installation

Pour commencer, suivez les étapes de la page [GitHub](/fr/settings/github).

Pour les applications GitHub, vous pouvez choisir de n’accorder des autorisations qu’à un seul dépôt. Nous vous le recommandons vivement, car nous n’avons besoin d’accéder qu’au dépôt où votre documentation est hébergée.

## Configuration

Configurez les contrôles CI activés pour un déploiement en accédant à la page [Add-ons](https://dashboard.mintlify.com/products/addons) de votre tableau de bord. Activez les contrôles que vous souhaitez exécuter.

Lors de l’activation des contrôles, vous pouvez choisir de les exécuter au niveau `Warning` ou `Blocking`.

- Un contrôle de niveau `Warning` n’affichera jamais un statut d’échec, même en cas d’erreur ou de suggestions.
- Un contrôle de niveau `Blocking` affichera un statut d’échec s’il n’est pas validé ou si des modifications sont suggérées.

## Vérifications CI disponibles

### Liens brisés

Comme le [vérificateur de liens de la CLI](/fr/settings/broken-links#broken-links) fonctionne sur votre machine locale, nous vérifions automatiquement vos docs pour détecter les liens brisés.

Pour consulter les résultats de cette vérification, rendez-vous sur la page des résultats des contrôles GitHub pour un commit spécifique.

### Vale

[Vale](https://vale.sh/) est un linter rédactionnel open source, basé sur des règles, qui prend en charge une variété de types de documents, notamment Markdown et MDX.

Mintlify prend en charge l’exécution automatique de Vale dans une vérification CI et l’affichage des résultats en tant que statut de contrôle.

#### Configuration

Si vous avez un fichier `.vale.ini` à la racine du répertoire de contenu de votre déploiement, nous utilisons automatiquement ce fichier de configuration. Nous utilisons également automatiquement tous les fichiers de configuration présents dans le `stylesPath` que vous avez spécifié.

Si vous n’avez pas de fichier de configuration Vale, la configuration par défaut est chargée automatiquement.

```mdx Default vale.ini configuration expandable
# Top level styles
StylesPath = /app/styles
MinAlertLevel = suggestion
IgnoredScopes = code, tt, img, url, a
SkippedScopes = script, style, pre, figure, code

# Vocabularies
Vocab = Mintlify

# This is required since Vale doesn't officially support MDX
[formats]
mdx = md

# MDX support
[*.mdx]
BasedOnStyles = Vale
Vale.Terms = NO # Enforces really harsh capitalization rules, keep off

# `import ...`, `export ...`
# `<Component ... />`
# `<Component>...</Component>`
# `{ ... }`
TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

# Exclude:
# `<Component \n ... />`
BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})

CommentDelimiters = {/*, */}
```

<Warning>
  Pour des raisons de sécurité, nous ne pouvons pas prendre en charge un `stylesPath` absolu ni un `stylesPath` contenant des valeurs `..`. Utilisez des chemins relatifs et incluez le `stylesPath` dans votre dépôt.
</Warning>

#### Paquets

Vale prend en charge une gamme de [paquets](https://vale.sh/docs/keys/packages), qui peuvent être utilisés pour vérifier l’orthographe et le style.
Tous les paquets que vous incluez dans votre dépôt sous le `stylesPath` adéquat sont automatiquement installés et utilisés dans votre configuration Vale.

Pour les paquets qui ne figurent pas dans votre dépôt, vous pouvez spécifier n’importe quels paquets du [registre de paquets Vale](https://vale.sh/explorer) ; ils seront automatiquement téléchargés et utilisés dans votre configuration Vale.

<Warning>
  Veuillez noter que, pour des raisons de sécurité, nous ne pouvons pas prendre en charge le téléchargement automatique de paquets qui ne proviennent pas du [registre de paquets Vale](https://vale.sh/explorer).
</Warning>

#### Vale avec `MDX`

Vale ne prend pas en charge `MDX` nativement, mais l’auteur de Vale a fourni une [extension personnalisée](https://github.com/errata-ai/MDX) pour l’ajouter.

Si vous préférez ne pas utiliser cette extension, nous recommandons d’ajouter les lignes suivantes à votre fichier `.vale.ini` :

```ini
[formats]
mdx = md

[*.mdx]
CommentDelimiters = {/*, */}

TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})
```

Pour utiliser les commentaires dans le document avec Vale, utilisez des commentaires au format MDX `{/* ... */}`. Si vous utilisez le [paramètre](https://vale.sh/docs/keys/commentdelimiters) `CommentDelimiters = {/*, */}` dans votre configuration, Vale interprétera automatiquement ces commentaires lors de l’analyse. Vous pouvez ainsi facilement utiliser les fonctionnalités natives de Vale, comme ignorer des lignes ou des sections.

```mdx
{/* vale off */}

Ce texte sera ignoré par Vale

{/* vale on */}
```

Si vous choisissez de ne pas utiliser `CommentDelimiters` mais souhaitez quand même utiliser les commentaires de Vale, vous devez envelopper tout commentaire Vale dans un commentaire MDX `{/* ... */}`. Par exemple :

```mdx
{/* <!-- vale off --> */}

Ce texte sera ignoré par Vale

{/* <!-- vale on --> */}
```

Ces balises de commentaire ne sont pas prises en charge dans les composants Mintlify, mais peuvent être utilisées n’importe où au niveau racine d’un document.
