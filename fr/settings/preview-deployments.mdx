Les déploiements de prévisualisation vous permettent de voir l’apparence des modifications apportées à votre documentation avant leur fusion en production. Chaque prévisualisation génère une URL partageable qui se met automatiquement à jour lorsque vous poussez de nouveaux changements.

## Créer des déploiements de prévisualisation

Les déploiements de prévisualisation sont créés automatiquement via des pull requests ou manuellement depuis votre tableau de bord.

### Prévisualisations automatiques

Lorsque vous créez une pull request, le bot Mintlify ajoute automatiquement un lien pour afficher le déploiement de prévisualisation dans votre pull request. La prévisualisation se met à jour chaque fois que vous poussez de nouveaux commits sur la branche.

### Prévisualisations manuelles

Vous pouvez créer manuellement une prévisualisation pour n’importe quelle branche.

1. Accédez à votre [tableau de bord](https://dashboard.mintlify.com/).
2. Sélectionnez **Previews**.
3. Sélectionnez **Create preview**.
4. Saisissez le nom de votre branche.
5. Sélectionnez **Create deployment**.

## Redéployer un aperçu

Redéployez un aperçu pour actualiser le contenu ou réessayer après un échec de déploiement.

1. Sélectionnez l’aperçu depuis votre [tableau de bord](https://dashboard.mintlify.com/).
2. Sélectionnez **Redéployer**.

## Partager les déploiements de prévisualisation

Les déploiements de prévisualisation peuvent être accessibles au public ou réservés aux utilisateurs authentifiés.

**Accès public** : Les URL de prévisualisation sont publiques par défaut. Partagez un lien de prévisualisation avec toute personne qui doit examiner vos modifications.

**Accès restreint** : Activez l’authentification des déploiements de prévisualisation pour limiter l’accès aux seuls utilisateurs Mintlify authentifiés. Consultez [Configuration de l’authentification](/fr/authentication-personalization/authentication-setup) pour plus d’informations.
