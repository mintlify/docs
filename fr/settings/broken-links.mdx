Lorsque vous modifiez le chemin d’un fichier dans votre dossier de documentation, le chemin de l’URL de cette page change également. Cela peut se produire lors de la restructuration de votre documentation ou de la modification du titre de la barre latérale.

<div id="broken-links">
  ## Liens brisés
</div>

Détectez les liens brisés avec notre interface en ligne de commande (CLI). [Installez la CLI](/fr/installation) et exécutez la commande :

```bash
mint broken-links
```

L’interface en ligne de commande (CLI) détecte tout lien relatif inexistant dans votre documentation.

<div id="redirects">
  ## Redirections
</div>

Configurez des redirections 301 en ajoutant le champ `redirects` à votre fichier `docs.json`.

```json
"redirects": [
  {
    "source": "/chemin/source",
    "destination": "/chemin/destination"
  }
]
```

Cela redirige définitivement `/source/path` vers `/destination/path`, afin de ne pas perdre le SEO acquis pour la page d’origine.

Pour faire correspondre un chemin générique (wildcard), utilisez `*` après un paramètre. Dans cet exemple, `/beta/:slug*` correspond à `/beta/introduction` et le redirige vers `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
