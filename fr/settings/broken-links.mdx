Lorsque vous modifiez le chemin d’un fichier dans votre dossier de documentation, l’URL de cette page change également. Cela peut arriver lorsque vous réorganisez votre documentation ou que vous modifiez le titre de la barre latérale.

<div id="broken-links">
  ## Liens cassés
</div>

Détectez les liens cassés avec notre CLI. [Installez la CLI](/fr/installation) et exécutez la commande :

```bash
mint broken-links
```

Le CLI Mint (mint) identifie tous les liens relatifs dans votre documentation qui n’existent pas.

<div id="redirects">
  ## Redirections
</div>

Configurez des redirections 301 en ajoutant le champ `redirects` à votre fichier `docs.json`.

```json
"redirects": [
  {
    "source": "/chemin/source",
    "destination": "/chemin/destination"
  }
]
```

Cela redirige définitivement `/source/path` vers `/destination/path` afin que vous ne perdiez pas le référencement (SEO) de la page d’origine.

Pour faire correspondre un chemin générique (wildcard), utilisez `*` après un paramètre. Dans cet exemple, `/beta/:slug*` correspond à `/beta/introduction` et le redirige vers `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
