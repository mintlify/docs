Mintlify applique automatiquement de nombreuses bonnes pratiques SEO, notamment :

* Génération des balises meta
* Génération du sitemap et du fichier `robots.txt`
* Structure HTML sémantique
* Optimisation pour mobile

Vous pouvez personnaliser entièrement les balises meta de votre site en ajoutant le champ `metatags` dans votre `docs.json` ou dans le frontmatter d’une page.

<div id="global-meta-tags">
  ## Balises meta globales
</div>

Vous pouvez définir des balises meta par défaut en ajoutant le champ `metadata` à votre `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "lien vers votre image de balise méta par défaut"
    }
}
```

<div id="page-specific-meta-tags">
  ## Balises meta spécifiques à une page
</div>

Ajoutez vos balises meta aux metadata de la page comme ceci :

```mdx
---
title: 'Titre de votre page d'exemple'
'og:image': 'lien vers votre image de balise meta'
---
```

<Warning>
  Les balises meta contenant des deux-points doivent être entourées de guillemets lorsque vous les ajoutez dans la metadata de la page.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps et fichiers robots.txt
</div>

Mintlify génère automatiquement un fichier `sitemap.xml` et un fichier `robots.txt`. Vous pouvez afficher votre fichier sitemap en ajoutant `/sitemap.xml` à l’URL de votre site de documentation.

Par défaut, seules les pages répertoriées dans votre `docs.json` sont incluses. Pour inclure des liens masqués, ajoutez `seo.indexing` à votre `docs.json` :

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Plans de site personnalisés et fichiers robots.txt
</div>

Pour ajouter un fichier `sitemap.xml` ou `robots.txt` personnalisé, créez `sitemap.xml` ou `robots.txt` à la racine de votre projet. L’ajout d’un fichier personnalisé remplacera le fichier généré automatiquement portant le même nom. Si vous supprimez un fichier personnalisé, le fichier par défaut sera à nouveau utilisé.

<div id="disabling-indexing">
  ## Désactivation de l’indexation
</div>

Si vous souhaitez empêcher une page d’être indexée par les moteurs de recherche, vous pouvez inclure ce qui suit dans le [frontmatter](fr/pages) de votre page :

```
---
noindex: true
---
```

Vous pouvez également indiquer `noindex` pour toutes les pages de votre documentation en définissant le champ `metadata.robots` sur `"noindex"` dans votre `docs.json` :

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## Bonnes pratiques SEO
</div>

<AccordionGroup>
  <Accordion title="Rédigez des titres et des descriptions explicites">
    * Utilisez des titres de page clairs et descriptifs (50-60 caractères)
    * Rédigez des meta descriptions convaincantes (150-160 caractères)
    * Intégrez des mots-clés pertinents
    * Rendez chaque title de page et chaque description uniques
  </Accordion>

  <Accordion title="Optimisez la structure de votre contenu">
    * Utilisez une hiérarchie de titres appropriée (H1 → H2 → H3)
    * Écrivez d’abord pour les humains, ensuite pour les moteurs de recherche
    * Intégrez des mots-clés pertinents dans les titres et le contenu
    * Gardez les URL courtes, descriptives et organisées hiérarchiquement
    * Divisez les contenus longs avec des sous-titres et des listes
  </Accordion>

  <Accordion title="Stratégie de maillage interne">
    * Faites des liens vers des pages connexes au sein de votre documentation
    * Utilisez un texte d’ancre descriptif plutôt que « cliquez ici »
    * Créez des clusters thématiques en reliant des concepts connexes
    * Utilisez les fonctionnalités de renvoi automatique
  </Accordion>

  <Accordion title="SEO des images">
    * Utilisez des noms de fichiers descriptifs pour les images
    * Incluez toujours un texte alternatif pour l’accessibilité et le SEO
    * Optimisez la taille des fichiers image pour un chargement plus rapide
    * Utilisez des images pertinentes qui servent votre contenu
  </Accordion>
</AccordionGroup>
