Mintlify prend automatiquement en charge de nombreuses bonnes pratiques SEO, notamment :

* Génération des balises meta
* Génération du sitemap et du fichier `robots.txt`
* Structure HTML sémantique
* Optimisation pour mobile

Vous pouvez personnaliser entièrement les balises meta de votre site en ajoutant le champ `metatags` à votre `docs.json` ou au frontmatter d’une page.

<div id="global-meta-tags">
  ## Balises méta globales
</div>

Vous pouvez définir des balises méta par défaut en ajoutant le champ `metadata` à votre `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "lien vers votre image de balise méta par défaut"
    }
}
```

<div id="page-specific-meta-tags">
  ## Balises meta spécifiques à la page
</div>

Ajoutez vos balises meta aux metadata de la page comme suit :

```mdx
---
title: 'Titre de votre page d'exemple'
'og:image': 'lien vers votre image de balise meta'
---
```

<Warning>
  Les balises meta comportant des deux-points doivent être entourées de guillemets lorsque vous les ajoutez dans la metadata de la page.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps et fichiers robots.txt
</div>

Mintlify génère automatiquement un fichier `sitemap.xml` et un fichier `robots.txt`. Vous pouvez afficher votre sitemap en ajoutant `/sitemap.xml` à l’URL de votre site de documentation.

Par défaut, seules les pages référencées dans votre `docs.json` sont incluses. Pour inclure des liens masqués, ajoutez `seo.indexing` à votre `docs.json` :

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Sitemaps et fichiers robots.txt personnalisés
</div>

Pour ajouter un fichier `sitemap.xml` ou `robots.txt` personnalisé, créez un fichier `sitemap.xml` ou `robots.txt` à la racine de votre projet. L’ajout d’un fichier personnalisé remplace le fichier généré automatiquement portant le même nom. Si vous supprimez le fichier personnalisé, le fichier par défaut sera de nouveau utilisé.

<div id="disabling-indexing">
  ## Désactiver l’indexation
</div>

Pour empêcher l’indexation d’une page par les moteurs de recherche, ajoutez l’élément suivant dans le [frontmatter](fr/pages) de votre page :

```
---
noindex: true
---
```

Vous pouvez également appliquer `noindex` à toutes les pages de votre documentation en définissant le champ `metadata.robots` sur `"noindex"` dans votre `docs.json` :

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## Bonnes pratiques SEO
</div>

<AccordionGroup>
  <Accordion title="Rédigez des titres et des descriptions explicites">
    * Utilisez des titres de page clairs et descriptifs (50 à 60 caractères)
    * Rédigez des descriptions attrayantes (150 à 160 caractères)
    * Intégrez des mots-clés pertinents
    * Rendez chaque titre de page et chaque description uniques
  </Accordion>

  <Accordion title="Optimisez la structure de votre contenu">
    * Utilisez une hiérarchie de titres appropriée (H1 → H2 → H3)
    * Écrivez d’abord pour les utilisateurs, ensuite pour les moteurs de recherche
    * Intégrez des mots-clés pertinents dans les titres et le contenu
    * Gardez des URL courtes, descriptives et organisées hiérarchiquement
    * Scindez les contenus longs avec des sous-titres et des listes
  </Accordion>

  <Accordion title="Stratégie de maillage interne">
    * Faites des liens vers des pages liées au sein de votre documentation
    * Utilisez un texte d’ancrage descriptif plutôt que « cliquez ici »
    * Créez des clusters thématiques en reliant des concepts connexes
    * Utilisez les fonctionnalités de renvois automatiques
  </Accordion>

  <Accordion title="SEO pour les images">
    * Utilisez des noms de fichiers descriptifs pour les images
    * Incluez toujours un texte alternatif pour l’accessibilité et le SEO
    * Optimisez la taille des fichiers image pour un chargement plus rapide
    * Utilisez des images pertinentes qui servent votre contenu
  </Accordion>
</AccordionGroup>
