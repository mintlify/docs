<Update label="10 août - 16 août" tags={["Améliorations"]} rss={{ title: "Mises à jour hebdomadaires", description: "Optimisations des performances, intégrations LLM, améliorations de l'Éditeur web et du Terrain de jeu API" }}>
  ## Optimisations des performances et de la build

  - La transpilation MDX se fait désormais au moment du déploiement plutôt qu’à chaque chargement de page dans les environnements serverless NextJS non mis en cache, améliorant le time to first byte pour les pages non mises en cache.
  - Le hachage basé sur le contenu évite la retranspilation lorsque le MDX n’a pas changé, réduisant les temps de flux de mise à jour d’environ 50 % pour les clients avec un grand nombre de pages (les déploiements de plus de 5 minutes devraient être à peu près divisés par deux)
  - L’affichage des déploiements de prévisualisation dans le tableau de bord est désormais plus rapide grâce à l’ajout d’index de base de données et à la parallélisation des requêtes dans notre backend
  - Réduction de la taille des pages en éliminant les données `navigation` dupliquées dans la charge utile `rsc` de chaque page — les gains de performances sont les plus visibles avec un grand nombre de pages ou des structures de navigation complexes
  - Un préchargement plus agressif permet des chargements de page instantanés plus fréquents

  ## Améliorations du Terrain de jeu API et d’OpenAPI

  - La conversion OpenAPI vers MCP a été déplacée vers le backend, permettant aux serveurs MCP hébergés d’inclure des outils (attendez-vous bientôt à une documentation plus claire et à des options de configuration)
  - Ajout de la prise en charge de Ruby au Terrain de jeu API
  - Nous avons ajouté une fonctionnalité qui vous permet désormais de [spécifier des pages API en utilisant uniquement votre docs.json](/fr/api-playground/openapi-setup#auto-populate-api-pages) sans créer de nouveaux fichiers MDX.
  - Prise en charge des [pages `webhook`](/fr/api-playground/openapi-setup#webhooks) dans la navigation des docs à partir des spécifications OpenAPI
  - Contexte de modèle IA optimisé en supprimant les indications d’ancre des liens Markdown lors de la navigation vers Anthropic, OpenAI ou d’autres fournisseurs

  ## Améliorations de l’Éditeur web

  - La création/le renommage de fichiers enregistre désormais les modifications lors d’un clic à l’extérieur au lieu d’exiger l’appui sur la touche Entrée
  - Correction de la navigation entre branches où la modification de l’URL vers une branche spécifique redirigeait vers la dernière branche active au lieu de la destination prévue
  - Encodage correct en URL des titres de branches contenant `/` pour éviter les problèmes de navigation
  - Correction du raccourci de lien `Ctrl+K` dans l’Éditeur du tableau de bord monorepo qui préfixait le chemin du dépôt des docs et créait des liens cassés

  ## Analyses et intégrations LLM

  - Prise en charge de `llms.txt` et `llms-full.txt` personnalisés — à ajouter à la racine du dépôt des docs pour être servis aux endpoints `/llms.txt` et `/llms-full.txt` pour la personnalisation LLM
  - Ajout de [l’intégration analytics Hightouch](/fr/integrations/analytics/hightouch#hightouch)
  - Suivi amélioré des analytics du Menu contextuel (affichage dans le tableau de bord bientôt)
  - Ajout de tests e2e pour `llms.txt` et `llms-full.txt` afin d’assurer une diffusion correcte

  ## Améliorations des composants et du style

  - Prise en charge de classnames personnalisées dans les balises `h{1-4}` pour appliquer des styles de titres personnalisés
  - Correction des balises `h{1-4}` rendues comme composants `Heading` avec badges en mode page personnalisée
  - Ajout de sélecteurs CSS au [fil d’Ariane](/fr/navigation#breadcrumbs) pour un ciblage CSS personnalisé
  - Correction des images Open Graph étirées en analysant les dimensions pour maintenir les proportions à 56 px de hauteur
  - Correction de `VSCode` en `VS Code` dans le Menu contextuel lorsqu’il est activé
  - Correction des titres à l’intérieur des composants personnalisés apparaissant dans la table des matières aux côtés des titres sémantiques

  ## Corrections de bugs et fiabilité

  - Correction des problèmes de rendu PDF avec certains titres de pages en assainissant les caractères qui posaient des problèmes de génération
  - Résolution de l’erreur CLI `Cannot convert undefined or null to object` lors de la rencontre de fichiers JSON OpenAPI vides
  - Correction des metatags Open Graph `docs.json` personnalisés écrasés par ceux générés
  - Correction des clics sur le bouton du flux RSS lors de l’arrivée via des liens d’ancrage en utilisant origin + pathname pour les liens RSS
  - Amélioration de la vitesse de téléchargement de la CLI en supprimant les sourcemaps

  ## Améliorations techniques

  - Ajout de tests visuels au pipeline CI pour une détection plus précoce des régressions
  - Amélioration de la gestion des erreurs et des capacités de débogage
  - Couverture de tests complète pour les nouvelles fonctionnalités et les cas limites

  ## Améliorations de l'authentification

  - Accès public au niveau du groupe : rendez publics des groupes entiers de pages via `docs.json` afin d’éviter d’ajouter `public: true` sur chaque page ([en savoir plus](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level))
  - Prise en charge de [`logoutURL` dans la configuration OAuth](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3) pour supprimer les cookies en amont et finaliser la déconnexion
  - En cas d’erreurs OAuth, les utilisateurs sont redirigés vers le `logoutURL` spécifié pour relancer le flux d’authentification
  - Correction d’un bref affichage d’une erreur 500 pendant les flux OAuth/JWT avant le callback
  - Suppression automatique de `https://` des URL dans la configuration OAuth/JWT pour éviter une mauvaise configuration

  ## Améliorations du Terrain de jeu API

  - Nouveau [endpoint Search API](https://mintlify.com/docs/api-reference/assistant/search) pour vous permettre de créer des agents et des serveurs MCP par-dessus votre documentation
  - Les fichiers `openapi` et `asyncapi` sont désormais servis à leurs chemins spécifiés (p. ex. `https://mydocsurl.extension/{openapi-or-file-name}.json`)
  - Vous pouvez désormais utiliser le [champ `x-mint` dans vos fichiers OpenAPI](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension) pour remplacer des champs générés, personnaliser le préambule ou modifier les URL d’endpoint dans les extraits de code
  - [`x-mcp` devient `x-mint.mcp`](https://mintlify.com/docs/api-playground/openapi-setup#mcp) dans les configurations OpenAPI pour contrôler quelles routes sont exposées en tant qu’outils MCP

  - Ajout d’une logique de nouvelle tentative pour les exports PDF afin d’améliorer la fiabilité

  - Correction de la copie d’une page dans le presse-papiers sur Safari en spécifiant un type MIME dans `navigator.write`

  ## Nouveau schéma de configuration `docs.json`

  <Frame>
    ![capture d’écran de docs.json](/images/changelog/docsjson.png)
  </Frame>

  Nous avons introduit un nouveau schéma `docs.json` en remplacement de `mint.json`, afin de prendre en charge un versionnage multiniveau amélioré, une compréhension visuelle facilitée et une terminologie plus cohérente. Pour plus d’informations sur les changements, [consultez notre blog](https://mintlify.com/blog/refactoring-mint-json-into-docs-json).

  Mettez à niveau de `mint.json` vers `docs.json` en suivant les étapes ci-dessous :

  1. Assurez-vous que votre CLI est à jour

  ```bash
  npm i mint@latest -g
  ```

  1. Dans votre dépôt de documentation, exécutez

  ```bash
  mint upgrade
  ```

  1. Supprimez votre ancien fichier `mint.json` et poussez vos modifications
