Le fichier `docs.json` vous permet de transformer un ensemble de fichiers Markdown en un site de documentation navigable et personnalisé. Ce fichier de configuration obligatoire contrôle le style, la navigation, les intégrations, et plus encore. Considérez-le comme le plan directeur de votre documentation.

Les paramètres définis dans `docs.json` s’appliquent à l’ensemble des pages.

## Configuration de votre `docs.json`

Pour commencer, il suffit de spécifier `theme`, `name`, `colors.primary` et `navigation`. Les autres champs sont facultatifs et vous pourrez les ajouter au fur et à mesure que vos besoins en documentation évoluent.

Pour une meilleure expérience d’édition, incluez la référence de schéma en haut de votre fichier `docs.json`. Cela active l’autocomplétion, la validation et des info-bulles utiles dans la plupart des éditeurs de code :

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Your Docs",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // Votre structure de navigation
  }
  // Le reste de votre configuration
}
```

## Mise à niveau depuis `mint.json`

Si votre projet de documentation utilise le fichier obsolète `mint.json`, suivez ces étapes pour passer à `docs.json`.

<Steps>
<Step title="Installer ou mettre à jour la CLI">
Si vous n’avez pas installé la [CLI](/fr/installation), installez-la maintenant :

<CodeGroup>

```bash npm
npm i -g mint
```


```bash yarn
yarn global add mint
```


```bash pnpm
pnpm add -g mint
```

</CodeGroup>

Si la CLI est déjà installée, assurez-vous qu’elle est à jour :

```bash
mint update
```

</Step>
<Step title="Créer votre fichier docs.json">
Dans votre dépôt de documentation, exécutez :

```bash
mint upgrade
```

Cette commande créera un fichier `docs.json` à partir de votre `mint.json` existant. Passez en revue le fichier généré pour vérifier que tous les paramètres sont corrects.
</Step>
<Step title="Supprimer votre fichier mint.json">
Après avoir vérifié que votre `docs.json` est correctement configuré, vous pouvez supprimer sans risque votre ancien fichier `mint.json`.
</Step>
</Steps>
