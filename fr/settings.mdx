Le fichier `docs.json` vous permet de transformer une collection de fichiers Markdown en un site de documentation navigable et personnalisable. Ce fichier de configuration requis contrôle le style, la navigation, les intégrations, et plus encore. Considérez-le comme le plan de votre documentation.

Les paramètres de `docs.json` s’appliquent globalement à toutes les pages.

<div id="setting-up-your-docsjson">
  ## Configuration de votre `docs.json`
</div>

Pour commencer, il vous suffit de renseigner `theme`, `name`, `colors.primary` et `navigation`. Les autres champs sont facultatifs et vous pourrez les ajouter au fur et à mesure que les besoins de votre documentation évolueront.

Pour une meilleure expérience d’édition, incluez la référence de schéma en haut de votre fichier `docs.json`. Cela active l’autocomplétion, la validation et des info-bulles utiles dans la plupart des éditeurs de code :

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Votre Documentation",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // Votre structure de navigation
  }
  // Le reste de votre configuration
}
```

<div id="customization">
  ### Personnalisation
</div>

<ResponseField name="theme" required>
  Le thème de mise en page de votre site.

  L’un des suivants : `mint`, `maple`, `palm`, `willow`, `linden`, `almond`, `aspen`.

  Voir [Thèmes](fr/themes) pour plus d’informations.
</ResponseField>

<ResponseField name="name" type="string" required>
  Le nom de votre projet, de votre organisation ou de votre produit.
</ResponseField>

<ResponseField name="colors" type="object" required>
  Les couleurs utilisées dans votre documentation. Leur application varie selon les thèmes. Si vous ne définissez qu’une couleur principale, elle sera utilisée pour tous les éléments colorés.

  <Expandable title="Couleurs">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      La couleur principale de votre documentation. Généralement utilisée pour les éléments mis en avant en mode clair, avec des variations selon le thème.

      Doit être un code hexadécimal commençant par `#`.
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      La couleur utilisée pour la mise en avant en mode sombre.

      Doit être un code hexadécimal commençant par `#`.
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      La couleur utilisée pour les boutons et les états de survol en modes clair et sombre, avec des variations selon le thème.

      Doit être un code hexadécimal commençant par `#`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="description" type="string">
  Description de votre site pour le SEO et l’indexation par l’IA.
</ResponseField>

<ResponseField name="logo" type="string or object">
  Votre logo pour les modes clair et sombre.

  <Expandable title="Logo">
    <ResponseField name="light" type="string" required>
      Chemin vers votre fichier de logo pour le mode clair. Incluez l’extension du fichier. Exemple : `/logo.png`
    </ResponseField>

    <ResponseField name="dark" type="string" required>
      Chemin vers votre fichier de logo pour le mode sombre. Incluez l’extension du fichier. Exemple : `/logo-dark.png`
    </ResponseField>

    <ResponseField name="href" type="string (uri)">
      L’URL vers laquelle rediriger lors d’un clic sur le logo. Si non renseignée, le logo renverra vers votre page d’accueil. Exemple : `https://mintlify.com`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string or object">
  Chemin vers votre fichier favicon, y compris l’extension. Redimensionné automatiquement aux tailles de favicon appropriées.
  Peut être un fichier unique ou des fichiers distincts pour les modes clair et sombre. Exemple : `/favicon.png`

  <Expandable title="Favicon">
    <ResponseField name="light" type="string" required>
      Chemin vers votre fichier favicon pour le mode clair. Incluez l’extension du fichier. Exemple : `/favicon.png`

    <ResponseField name="dark" type="string" required>
      Chemin vers votre fichier favicon pour le mode sombre. Incluez l’extension du fichier. Exemple : `/favicon-dark.png`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="thumbnails" type="object">
  Personnalisation des vignettes pour les réseaux sociaux et les aperçus de page.

  <Expandable title="Vignettes">
    <ResponseField name="appearance" type="&#x22;light&#x22; | &#x22;dark&#x22;">
      Le thème visuel de vos vignettes. S’il n’est pas spécifié, les vignettes utilisent la palette de couleurs de votre site définie par le champ `colors`.
    </ResponseField>

    <ResponseField name="background" type="string">
      Image d’arrière-plan de vos vignettes. Peut être un chemin relatif ou une URL absolue.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="styling" type="object">
  Configurations de style visuel.

  <Expandable title="Stylisation">
    <ResponseField name="eyebrows" type="&#x22;section&#x22; | &#x22;breadcrumbs&#x22;">
      Le style de l’eyebrow de la page. Choisissez `section` pour afficher le nom de la section ou `breadcrumbs` pour afficher le chemin de navigation complet. Par défaut : `section`.
    </ResponseField>

    <ResponseField name="codeblocks" type="&#x22;system&#x22; | &#x22;dark&#x22; | object">
      Configuration du thème des code blocks. Par défaut : `"system"`.

      **Options simples :**

      * `"system"` : correspond au mode actuel du site (clair ou sombre)
      * `"dark"` : toujours utiliser le mode sombre

      **Configuration de thème personnalisée :**
      Utilisez un objet pour spécifier des [thèmes Shiki](https://shiki.style/themes). Choisissez un seul thème pour les deux modes ou des thèmes distincts pour les modes clair et sombre.

      <Expandable title="codeblocks">
        <ResponseField name="theme" type="string">
          Un nom de thème Shiki unique à utiliser pour les modes clair et sombre.

          ```json
          "styling": {
            "codeblocks": {
              "theme": "dracula"
            }
          }
          ```
        </ResponseField>

        <ResponseField name="themes" type="object">
          Thèmes distincts pour les modes clair et sombre.

          <Expandable title="themes">
            <ResponseField name="light" type="string" required>
              Un nom de thème Shiki pour le mode clair.
            </ResponseField>

            <ResponseField name="dark" type="string" required>
              Un nom de thème Shiki pour le mode sombre.
            </ResponseField>

            ```json
            "styling": {
              "codeblocks": {
                "themes": {
                  "light": "github-light",
                  "dark": "github-dark"
                }
              }
            }
            ```
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

</ResponseField>

    </ResponseField>

    <ResponseField name="body" type="object">

      * `view` : Affiche la page actuelle en Markdown dans un nouvel onglet.
      * `chatgpt` : Envoie le contenu de la page actuelle à ChatGPT.
      * `claude` : Envoie le contenu de la page actuelle à Claude.
      * `perplexity` : Envoie le contenu de la page actuelle à Perplexity.
      * `mcp` : Copie l’URL de votre serveur MCP dans le presse‑papiers.
      * `cursor` : Installe votre serveur MCP hébergé dans Cursor.
      * `vscode` : Installe votre serveur MCP hébergé dans VSCode.

      <img src="/images/page-context-menu.png" alt="Menu contextuel" className="rounded-xl" />

      <Note>
        Le menu contextuel est disponible uniquement sur les aperçus et les déploiements de production.
      </Note>
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="upgrading-from-mintjson">
  ## Mise à niveau depuis `mint.json`
</div>

Si votre projet de documentation utilise le fichier obsolète `mint.json`, suivez ces étapes pour passer à `docs.json`.

<Steps>
  <Step title="Installer ou mettre à jour l’interface en ligne de commande (CLI)">
    Si vous n’avez pas encore installé la [CLI](/fr/installation), installez-la maintenant :

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    Si la CLI est déjà installée, assurez-vous qu’elle est à jour :

    ```bash
    mint update
    ```
  </Step>

  <Step title="Créer votre fichier docs.json">
    Dans votre référentiel de documentation, exécutez :

    ```bash
    mint upgrade
    ```

    Cette commande va créer un fichier `docs.json` à partir de votre `mint.json` existant. Passez en revue le fichier généré pour vous assurer que tous les paramètres sont corrects.
  </Step>

  <Step title="Supprimer votre fichier mint.json">
    Après avoir vérifié que votre `docs.json` est correctement configuré, vous pouvez supprimer en toute sécurité votre ancien fichier `mint.json`.
  </Step>
</Steps>
