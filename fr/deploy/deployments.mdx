title: "Déploiements"

---

Votre site de documentation est déployé automatiquement lorsque vous poussez des modifications vers le référentiel connecté. Cela nécessite que la GitHub App Mintlify soit correctement installée et connectée.

Si vos dernières modifications n’apparaissent pas sur votre site en production, vérifiez d’abord que la GitHub App est installée sur le compte ou l’organisation propriétaire de votre référentiel de documentation. Consultez la section [GitHub troubleshooting](/fr/deploy/github#troubleshooting) pour en savoir plus.

Si la GitHub App est connectée mais que les modifications ne se déploient toujours pas, vous pouvez déclencher manuellement une reconstruction depuis le Dashboard.

<div id="manually-triggering-a-deployment">
  ## Déclencher manuellement un déploiement
</div>

<Steps>
  <Step title="Vérifiez que votre dernier commit a été effectué avec succès.">
    Assurez-vous que votre dernier commit apparaît dans votre référentiel de documentation et qu’il n’a rencontré aucune erreur.
  </Step>

  <Step title="Déclenchez manuellement un déploiement.">
    Accédez à votre [Dashboard](https://dashboard.mintlify.com) et cliquez sur le bouton de déploiement.
