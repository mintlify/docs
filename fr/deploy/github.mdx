Mintlify utilise une GitHub App pour synchroniser automatiquement votre documentation avec votre référentiel GitHub.

<div id="installing-the-github-app">
  ## Installation de l’application GitHub
</div>

Installez la GitHub App Mintlify sur le compte ou l’organisation GitHub qui détient votre référentiel de documentation. Vous devez être propriétaire de l’organisation ou disposer des autorisations d’administrateur sur un référentiel pour installer l’application. Si vous n’avez pas les autorisations nécessaires, le propriétaire du référentiel devra approuver la demande d’installation.

Installez la GitHub App Mintlify depuis votre [Dashboard](https://dashboard.mintlify.com/settings/organization/github-app).

<Tip>
Nous vous recommandons d’accorder l’accès uniquement au référentiel où votre documentation est hébergée.
</Tip>

<div id="permissions">
  ## Autorisations
</div>

Lorsque vous installez l’app GitHub, il vous sera demandé d’accorder les autorisations suivantes :

Autorisations de lecture :

* `metadata` : Informations de base du référentiel

Autorisations de lecture et d’écriture :

* `checks` : Créer des vérifications d’état sur les pull requests (demandes de fusion)
* `code` : Lire les modifications de fichiers lorsque vous faites un commit sur votre branche de documentation
* `deployments` : Générer des déploiements de prévisualisation pour les pull requests (demandes de fusion)
* `pull requests` : Créer des branches et des pull requests (demandes de fusion) depuis l’éditeur web
* `workflows` : Gérer GitHub Actions pour les déploiements de la documentation

<Info>
L’app n’accède qu’aux référentiels auxquels vous lui avez explicitement accordé l’accès. Si vous avez activé des règles de protection de branche, l’app ne peut pas pousser directement vers les branches protégées.
</Info>

<div id="managing-repository-access">
  ## Gérer l’accès aux référentiels
</div>

Lors de l’installation de notre GitHub App, vous pouvez accorder l’accès à l’ensemble de vos référentiels ou uniquement à certains d’entre eux. Nous recommandons de n’accorder l’accès qu’aux référentiels où se trouve votre documentation. Vous pouvez modifier cette sélection à tout moment dans vos [paramètres de l’application GitHub](https://github.com/apps/mintlify/installations/new).

<div id="configuring-docs-source">
  ## Configurer la source de la documentation
</div>

Modifiez l’organisation, le référentiel ou la branche à partir desquels votre documentation est générée dans la section [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) de votre Dashboard.

<div id="troubleshooting">
  ## Résolution des problèmes
</div>

<div id="github-app-connection-issues">
  ### Problèmes de connexion avec l’application GitHub
</div>

Si vous rencontrez des problèmes avec l’application GitHub, réinitialiser la connexion permet de résoudre la plupart des cas.

<Steps>
  <Step title="Désinstallez l’application Mintlify via GitHub.">
    1. Dans GitHub, allez aux [installations](https://github.com/settings/installations) et sélectionnez **Configure** à côté de l’application Mintlify. Faites défiler vers le bas et sélectionnez **Uninstall**.
    2. Allez sur [Authorized GitHub Apps](https://github.com/settings/apps/authorizations) et sélectionnez **Revoke** à côté de l’application Mintlify.
  </Step>
  <Step title="Réinstallez l’application Mintlify.">
    1. Dans votre Tableau de bord Mintlify, allez dans [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) et installez la GitHub App.
    2. Autorisez votre compte dans la section [My Profile](https://dashboard.mintlify.com/settings/account) de votre Tableau de bord.
  </Step>
</Steps>

<div id="feedback-add-ons-are-unavailable">
  ### Les modules complémentaires de feedback sont indisponibles
</div>

Si vous ne pouvez pas activer les options de suggestions d’édition ou de signalement de problèmes dans votre Tableau de bord Mintlify et que votre référentiel est public, revalidez vos paramètres Git.

<Steps>
  <Step title="Accéder aux paramètres Git">
    Accédez aux [paramètres Git](https://dashboard.mintlify.com/settings/deployment/git-settings) dans votre Tableau de bord Mintlify.
  </Step>
  <Step title="Revalider vos paramètres">
    Cliquez sur la coche verte dans le coin du cadre des paramètres Git pour revalider les paramètres de votre référentiel. Cela forcera la mise à jour de vos paramètres afin de refléter si votre référentiel est public ou privé.
