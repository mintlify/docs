---
title: "Configuration d’un monorepo"
description: "Déployez votre documentation depuis un dépôt contenant plusieurs projets"
---

Configurez Mintlify pour déployer la documentation depuis un répertoire spécifique au sein d’un monorepo. Cette configuration vous permet de maintenir la documentation aux côtés de votre code dans des dépôts qui contiennent plusieurs projets ou services.

<div id="prerequisites">
  ## Prérequis
</div>

* Accès administrateur à votre projet Mintlify.
* Fichiers de documentation organisés dans un répertoire dédié au sein de votre monorepo.
* Un fichier `docs.json` valide dans votre répertoire de documentation.

<div id="configure-monorepo-deployment">
  ## Configurer le déploiement d’un monorepo
</div>

<Steps>
<Step title="Accéder aux paramètres Git">
Accédez à [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) dans votre Dashboard.

<Frame>
<img
  className="block dark:hidden my-0 pointer-events-none"

<img

/>
</Frame>

</Step>
<Step title="Définir le chemin de votre documentation">
1. Activez l’option **Configurer en tant que monorepo**.
2. Saisissez le chemin relatif vers votre répertoire de documentation. Par exemple, si votre documentation se trouve dans le répertoire `docs`, saisissez `/docs`.
<Note>
  N’incluez pas de barre oblique finale dans le chemin.
</Note>
3. Sélectionnez **Enregistrer les modifications**.
</Step>
</Steps>
