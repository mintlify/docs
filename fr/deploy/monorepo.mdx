---

Configurez Mintlify pour déployer la documentation depuis un répertoire spécifique au sein d’un monorepo. Cette configuration vous permet de conserver la documentation aux côtés de votre code dans des référentiels contenant plusieurs projets ou services.

Une structure de monorepo typique pourrait ressembler à ceci :

```text
your-repo/
├── apps/
│   ├── web/
│   └── api/
├── docs/
│   ├── docs.json
│   └── quickstart.mdx
└── package.json
```

Dans cet exemple, vous définiriez le chemin de votre documentation sur `/docs`.

<div id="prerequisites">
  ## Prérequis
</div>

* Accès administrateur à votre projet Mintlify.
* Fichiers de documentation organisés dans un répertoire dédié au sein de votre monorepo.
* Un fichier `docs.json` valide dans votre répertoire de documentation.

<div id="configure-monorepo-deployment">
  ## Configurer le déploiement d’un monorepo
</div>

<Steps>
<Step title="Accéder aux paramètres Git">
Accédez à [Git Settings](https://app.mintlify.com/settings/deployment/git-settings) dans votre Dashboard.

</Step>
<Step title="Définir le chemin de votre documentation">
1. Activez l’option indiquant que votre `docs.json` se trouve dans un sous-répertoire. Selon la date de création de votre projet, ce contrôle est intitulé **docs.json is in a subdirectory** ou **Set up as monorepo**.
2. Dans le champ de chemin (**Path to directory containing docs.json**), saisissez le chemin relatif depuis la racine du référentiel jusqu’au répertoire contenant votre `docs.json`. Par exemple, si votre documentation se trouve dans le répertoire `docs`, saisissez `/docs`.
<Note>
  Commencez le chemin par une barre oblique initiale et n’incluez pas de barre oblique finale.
</Note>
3. Sélectionnez **Enregistrer les modifications**.
</Step>
</Steps>
