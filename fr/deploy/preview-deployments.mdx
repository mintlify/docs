<Info>
  Les aperçus de déploiement sont disponibles avec les [offres Pro et Enterprise](https://mintlify.com/pricing?ref=preview-deployments).
</Info>

Les aperçus de déploiement vous permettent de voir à quoi ressemblent les modifications de votre documentation avant la fusion en production. Chaque aperçu crée une URL partageable qui se met automatiquement à jour lorsque vous poussez de nouvelles modifications.

Par défaut, les URL d’aperçu sont publiques. Partagez un lien d’aperçu avec toute personne qui doit examiner vos modifications.

<div id="create-preview-deployments">
  ## Créer des déploiements de prévisualisation
</div>

Les déploiements de prévisualisation sont créés automatiquement via des pull requests (demandes de fusion) ou manuellement depuis votre Dashboard.

<div id="automatic-previews">
  ### Aperçus automatiques
</div>

<Note>
  Les aperçus automatiques ne sont créés que pour les pull requests (demandes de fusion) qui ciblent votre [branche de déploiement](/fr/guides/git-concepts#deployment-branch).
</Note>

Lorsque vous créez une pull request (demande de fusion), le bot Mintlify ajoute automatiquement un lien pour afficher le déploiement de prévisualisation dans votre pull request. L’aperçu est mis à jour chaque fois que vous poussez de nouveaux commits sur la branche.

<div id="manual-previews">
  ### Aperçus manuels
</div>

Vous pouvez créer manuellement un aperçu pour n’importe quelle branche.

1. Accédez à votre [Dashboard](https://dashboard.mintlify.com/).
2. Sélectionnez **Previews**.
3. Sélectionnez **Create custom preview**.
4. Entrez le nom de la branche à prévisualiser.
5. Sélectionnez **Create deployment**.

<div id="redeploy-a-preview">
  ## Redéployer un aperçu
</div>

Redéployez un aperçu pour actualiser le contenu ou réessayer après un déploiement ayant échoué.

1. Sélectionnez l’aperçu dans votre [Dashboard](https://dashboard.mintlify.com/).
2. Sélectionnez **Redéployer**.

<div id="preview-widget">
  ## Widget d’aperçu
</div>

Le widget d’aperçu apparaît sur les déploiements de prévisualisation pour vous aider à naviguer et à consulter les pages mises à jour. Le widget se présente sous la forme d’un bouton flottant dans le coin inférieur droit de votre déploiement de prévisualisation.

<Frame>
  <img src="/images/previews/widget-light.png" alt="Widget d’aperçu agrandi affichant la liste des fichiers modifiés." className="block dark:hidden" />

  <img src="/images/previews/widget-dark.png" alt="Widget d’aperçu agrandi affichant la liste des fichiers modifiés." className="hidden dark:block" />
</Frame>

1. Cliquez sur le widget pour afficher tous les fichiers ajoutés, modifiés ou supprimés dans l’aperçu.
2. Cliquez sur un fichier pour voir les modifications sur la page correspondante.
3. Utilisez la barre de recherche pour filtrer la liste des fichiers modifiés.

Le widget apparaît uniquement sur les déploiements de prévisualisation, et non sur votre site en production ni dans vos aperçus locaux.

<div id="restrict-access-to-preview-deployments">
  ## Restreindre l’accès aux déploiements de prévisualisation
</div>

Par défaut, les déploiements de prévisualisation sont accessibles publiquement à toute personne disposant de l’URL. Vous pouvez en restreindre l’accès aux membres authentifiés de votre organisation Mintlify.

1. Accédez à la section **Previews** sur la page [Modules complémentaires](https://dashboard.mintlify.com/products/addons) de votre Dashboard.
2. Activez ou désactivez l’interrupteur **Preview authentication** pour gérer l’authentification des prévisualisations.

<div id="troubleshooting-preview-deployments">
  ## Dépannage des déploiements de prévisualisation
</div>

Si votre déploiement de prévisualisation échoue, essayez les étapes de dépannage suivantes.

* **Consulter les journaux de build** : Dans votre [Dashboard](https://dashboard.mintlify.com/), allez dans **Previews** et cliquez sur la prévisualisation en échec. Les journaux de déploiement affichent les erreurs qui ont provoqué l’échec.
* **Vérifier votre configuration** :
  * Syntaxe `docs.json` non valide
  * Chemins de fichiers manquants ou incorrects référencés dans votre navigation
  * Frontmatter non valide dans les fichiers MDX
  * Liens d’images cassés ou fichiers d’images manquants
* **Valider en local** : Exécutez `mint dev` en local pour détecter les erreurs de build avant de pousser vers le référentiel.
* **Vérifier les changements récents** : Passez en revue les commits les plus récents dans votre branche pour identifier les modifications qui ont entraîné l’échec du build.
