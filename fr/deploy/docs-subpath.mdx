Hébergez votre documentation sur un sous-chemin comme `your-domain.com/docs` pour conserver votre documentation sur votre domaine principal, ce qui la rend plus facile à trouver et offre une expérience de marque cohérente à vos utilisateurs.

<div id="set-up-by-hosting-provider">
  ## Configuration selon le fournisseur d&#39;hébergement
</div>

La configuration d’un sous-chemin `/docs` varie selon votre fournisseur d’hébergement. Choisissez votre fournisseur d’hébergement et suivez les instructions de configuration.

* [Cloudflare](/fr/deploy/cloudflare) : Configurez Cloudflare Workers pour votre domaine afin de servir votre documentation au sous-chemin `/docs`.
* [AWS Route 53 and CloudFront](/fr/deploy/route53-cloudfront) : Déployez votre documentation au sous-chemin `/docs` à l’aide d’AWS avec le DNS Route 53 et le CDN CloudFront.
* [Vercel](/fr/deploy/vercel#host-from-docs) - Utilisez des réécritures pour déployer votre documentation au sous-chemin `/docs`.
* [Custom reverse proxy](/fr/deploy/reverse-proxy) - Pour d’autres plateformes d’hébergement, configurez un proxy inverse pour déployer votre documentation au sous-chemin `/docs`.

<div id="additional-configuration-for-strict-security-policies">
  ### Configuration supplémentaire pour des stratégies de sécurité strictes
</div>

Si vous faites transiter l’intégralité du trafic de votre domaine personnalisé via un proxy, vous devrez peut-être configurer les règles de votre proxy ou de vos pare-feu afin d’ajouter les [headers](/fr/deploy/csp-configuration) appropriés pour que votre documentation s’affiche correctement.
