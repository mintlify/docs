Hébergez votre documentation sur un sous-chemin comme `your-domain.com/docs` pour conserver votre documentation sur votre domaine principal, ce qui la rend plus facile à trouver et offre une expérience de marque cohérente à vos utilisateurs.

<div id="let-mintlify-manage-your-domain">
  ## Laisser Mintlify gérer votre domaine
</div>

Si votre domaine n’héberge rien d’autre, Mintlify peut servir votre documentation à votre sous-chemin sans passer par un proxy. Sur la page [Configuration du domaine personnalisé](https://app.mintlify.com/settings/deployment/custom-domain) de votre Dashboard, activez le bouton **Host at**, saisissez votre chemin de base et ajoutez votre domaine. Configurez ensuite les enregistrements DNS affichés dans votre Dashboard pour faire pointer votre domaine vers Mintlify. Consultez [Domaine personnalisé](/fr/customize/custom-domain) pour les instructions de configuration.

N’utilisez cette option que si Mintlify peut recevoir tous les chemins du domaine.

<div id="set-up-by-hosting-provider">
  ## Configuration selon le fournisseur d&#39;hébergement
</div>

Si un autre site fonctionne sur votre domaine, configurez votre fournisseur d’hébergement pour qu’il transmette votre sous-chemin à Mintlify via un proxy. Choisissez votre fournisseur d’hébergement et suivez les instructions de configuration.

- [Cloudflare](/fr/deploy/cloudflare) : configurez Cloudflare Workers pour votre domaine afin de servir votre documentation à votre sous-chemin.
- [AWS Route 53 et CloudFront](/fr/deploy/route53-cloudfront) : déployez votre documentation à votre sous-chemin avec AWS via le DNS Route 53 et le CDN CloudFront.
- [Vercel](/fr/deploy/vercel) : utilisez des réécritures pour déployer votre documentation à votre sous-chemin.
- [Proxy inverse personnalisé](/fr/deploy/reverse-proxy) : pour d’autres plateformes d’hébergement, configurez un proxy inverse pour déployer votre documentation à votre sous-chemin.

<div id="additional-configuration-for-strict-security-policies">
  ### Configuration supplémentaire pour des stratégies de sécurité strictes
</div>

Si vous faites transiter l’intégralité du trafic de votre domaine personnalisé via un proxy, vous devrez peut-être configurer les règles de votre proxy ou de vos pare-feu afin d’ajouter les [en-têtes](/fr/deploy/csp-configuration) appropriés pour que votre documentation s’affiche correctement.
