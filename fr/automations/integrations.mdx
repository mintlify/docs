Les intégrations connectent l'agent Mintlify aux outils déjà utilisés par votre équipe. L'agent peut utiliser les applications connectées lorsqu'il répond dans Slack ou exécute une automatisation. Les événements de certaines intégrations partagées peuvent aussi lancer une automatisation personnalisée.

| Où l'intégration est utilisée | Fonctionnement |
| --- | --- |
| **Agent Slack** | Utilise les intégrations partagées et les intégrations personnelles de la personne qui discute avec l'agent. |
| **Outil d'automatisation** | Fournit un contexte en lecture seule depuis les intégrations partagées sélectionnées dans **Tools**. |
| **Déclencheur d'automatisation** | Lance une automatisation personnalisée lorsqu'un événement pris en charge se produit dans une intégration partagée. |

<Note>
  L'[application Slack Mintlify](/fr/agent/slack) est distincte des intégrations de cette page. Son installation ajoute l'agent à votre espace de travail et donne à l'agent et aux exécutions d'automatisations accès au contenu Slack que l'application peut lire. Slack ne se connecte pas depuis **Integrations** et ne se sélectionne pas dans **Tools**.
</Note>

<div id="connection-scope">
  ## Portée de la connexion
</div>

- Les **intégrations partagées** sont connectées une fois pour toute l'organisation. L'agent Slack peut les utiliser, vous pouvez les ajouter comme outils d'automatisation et, si elles fournissent des événements, elles peuvent déclencher des automatisations personnalisées.
- Les **intégrations personnelles** sont connectées par personne. L'agent Slack utilise le compte personnel de la personne qui discute avec lui. Elles ne peuvent pas être ajoutées aux automatisations comme outils ou déclencheurs.

Par exemple, si deux personnes connectent des comptes Google Drive différents, chacune obtient les résultats de son propre Drive lorsqu'elle interroge l'agent dans Slack.

<div id="connect-an-integration">
  ## Connecter une intégration
</div>

Les administrateurs et les éditeurs peuvent connecter les intégrations partagées pour l'organisation. Chaque personne connecte ses propres intégrations personnelles.

### Depuis le dashboard

1. Accédez à la page [Integrations](https://app.mintlify.com/settings/organization/integrations) dans les paramètres.
2. Trouvez l'intégration et cliquez sur **Connect**.
3. Suivez le processus d'autorisation.

Vous pouvez également sélectionner une application non connectée dans le champ **Tools** d'une automatisation et l'autoriser depuis cet emplacement.

### Depuis l'agent Slack

Demandez à l'agent de connecter une application, par exemple : « Connecte mon Google Drive ». Suivez le lien d'autorisation qu'il renvoie, puis envoyez un autre message dans le même fil pour qu'il confirme et utilise la connexion.

<div id="use-integrations-with-the-slack-agent">
  ## Utiliser les intégrations avec l'agent Slack
</div>

Après avoir installé l'[application Slack Mintlify](/fr/agent/slack), l'agent peut utiliser les intégrations lorsqu'il répond aux questions ou met à jour votre documentation. Vous n'avez pas à sélectionner les intégrations pour chaque conversation.

L'agent peut utiliser :

- Les intégrations partagées connectées à votre organisation.
- Les intégrations personnelles connectées par la personne qui envoie la demande.
- Les canaux et fils Slack que l'application Mintlify est autorisée à lire.

L'agent n'accède aux données que lorsqu'elles sont pertinentes pour la demande.

<div id="use-integrations-as-automation-tools">
  ## Utiliser les intégrations comme outils d'automatisation
</div>

Les outils fournissent un contexte supplémentaire après le démarrage d'une automatisation. Seules les intégrations partagées peuvent être sélectionnées.

1. Ouvrez les paramètres de l'automatisation.
2. Dans **Tools**, sélectionnez les intégrations qu'elle peut utiliser.
3. Connectez toute intégration sélectionnée qui ne l'est pas encore.
4. Enregistrez l'automatisation.

L'intégration utilisée comme déclencheur est automatiquement incluse comme outil afin que l'agent puisse récupérer davantage de détails après la réception de l'événement.

<div id="trigger-an-automation-from-an-integration">
  ## Déclencher une automatisation depuis une intégration
</div>

1. Créez ou ouvrez une automatisation personnalisée.
2. Pour **Trigger**, sélectionnez **Integration**.
3. Sélectionnez ou connectez une intégration partagée.
4. Sélectionnez le **Trigger event** et remplissez les champs supplémentaires.
5. Enregistrez et activez l'automatisation.

Chaque automatisation possède un seul déclencheur. **Integration** remplace un déclencheur de mise à jour de contenu, de modification de code ou de calendrier personnalisé.

| Intégration | Événements |
| --- | --- |
| Confluence | Page créée ; contenu ajouté à un espace |
| Jira | Ticket créé ; ticket mis à jour ; projet créé |
| Linear | Ticket créé ; ticket mis à jour ; projet créé ; mise à jour de projet publiée |
| Notion | Page créée ; contenu de page mis à jour |
| Zendesk | Ticket créé |

Lorsque l'événement se produit, Mintlify lance l'automatisation et inclut les données de l'événement comme contexte.

<div id="permissions-and-access">
  ## Autorisations et accès
</div>

Les outils d'intégration tiers disponibles pour l'agent sont en lecture seule. L'agent peut rechercher et récupérer du contenu, mais ne peut pas créer, modifier ou supprimer des données dans l'application connectée.

L'agent hérite des accès du compte qui a autorisé la connexion. Pour les limiter, restreignez les autorisations de ce compte ou n'accordez à l'application Mintlify que l'accès à certaines ressources. Vérifiez l'écran d'autorisation pour connaître les autorisations exactes avant de connecter une application.

<div id="disconnect-an-integration">
  ## Déconnecter une intégration
</div>

1. Accédez à la page [Integrations](https://app.mintlify.com/settings/organization/integrations).
2. Cliquez sur **Configure** pour l'intégration.
3. Cliquez sur **Disable integration**.

La déconnexion d'une intégration partagée la supprime pour toute l'organisation et arrête les nouveaux événements des automatisations qui l'utilisent comme déclencheur. La déconnexion d'une intégration personnelle ne supprime que votre connexion.

<div id="troubleshoot-a-connection">
  ## Résoudre un problème de connexion
</div>

Vérifiez que le compte autorisé peut toujours accéder au contenu, déconnectez puis reconnectez l'intégration et recommencez l'autorisation. Si la connexion est destinée à l'agent Slack, envoyez un autre message dans le fil d'origine. Si l'autorisation est bloquée, demandez à votre administrateur d'applications ou d'espace de travail d'approuver Mintlify.
