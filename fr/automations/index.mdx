<Info>
  Les automatisations nécessitent une [offre Pro ou Enterprise](https://mintlify.com/pricing?ref=automations).
</Info>

Les automatisations exécutent l'agent automatiquement selon un calendrier ou en réponse à des modifications dans un dépôt. Chaque automatisation définit un prompt pour l'agent et un déclencheur indiquant quand l'exécuter. Les automatisations prennent en charge les dépôts GitHub et GitLab, avec les mêmes déclencheurs, automatisations prédéfinies et modes de mise à jour. Tout au long de ce guide, "pull request" désigne également les merge requests GitLab, sauf indication contraire.

Lorsqu'une automatisation s'exécute, l'agent lit le contenu de votre projet et tout dépôt connecté, puis suit le prompt pour effectuer les mises à jour.

<Tip>
Utilisez des automatisations planifiées pour les tâches récurrentes, comme la publication de changelogs ou combler les lacunes à partir des conversations de l'assistant.

Utilisez des automatisations déclenchées par des modifications de contenu ou de code pour la maintenance réactive, comme les traductions ou la mise à jour du contenu lorsque le code source change.
</Tip>

<div id="trigger-types">
  ## Types de déclencheurs
</div>

Les automatisations prennent en charge trois types de déclencheurs.

- **Mise à jour de contenu** : s'exécute lorsque des modifications de contenu sont poussées vers le dépôt de votre projet Mintlify, y compris les fusions de pull requests et les pushes directs.
- **Modification de code** : s'exécute lorsqu'une pull request est fusionnée dans un dépôt de code source connecté.
- **Calendrier personnalisé** : s'exécute selon un calendrier récurrent. Les automatisations sont mises en file d'attente dans les 10 minutes suivant l'heure planifiée.

Chaque automatisation possède exactement un déclencheur. Vous ne pouvez pas combiner plusieurs types de déclencheurs dans une seule automatisation.

<div id="pull-request-grouping">
  ## Regroupement des pull requests
</div>

Les automatisations regroupent les modifications associées dans une seule pull request au lieu d'en ouvrir une distincte pour chaque changement. Lorsqu'une exécution ultérieure produit d'autres modifications qui appartiennent à une pull request ouverte, l'agent les ajoute à cette pull request plutôt que d'en ouvrir une autre en double. La façon dont le regroupement se traduit dépend de l'automatisation. Consultez [Automatisations prédéfinies](/fr/automations/reference) pour connaître le comportement de chaque automatisation.

<Note>
  Les automatisations Draft changelog, Fill gaps from assistant conversations, Improve docs from user feedback, Update from support feedback et les automatisations personnalisées ne regroupent ni n'ajoutent. Chaque exécution ouvre sa propre pull request avec les modifications de cette exécution. Update from support feedback peut ouvrir jusqu'à trois pull requests par exécution, une pour chaque thème distinct qu'elle identifie.
</Note>

<div id="reviewing-automation-pull-requests">
  ## Examiner les pull requests d'automatisation
</div>

Lorsqu'une automatisation ouvre ou met à jour une pull request, Mintlify ajoute du contexte directement dans celle-ci pour que vous puissiez l'examiner sans quitter GitHub ou GitLab.

- **Titre de la PR** inclut le nom de l'automatisation qui l'a générée, afin d'identifier les PRs d'automatisation d'un coup d'œil lorsque vous parcourez la liste des PRs ouvertes.
- **Description de la PR** résume ce qui a changé et pourquoi. Chaque fois qu'une exécution ajoute de nouvelles modifications à une PR existante, la description est mise à jour pour afficher une chronologie continue de ce que chaque exécution a apporté.
- **Commentaires sur la PR** pour toute exécution qui ajoute des modifications, expliquant les changements et indiquant quelles pull requests de code source l'agent a utilisées comme contexte.
- **Relecteurs** attribués en fonction des auteurs des modifications de code source qui ont déclenché la mise à jour. S'applique aux automatisations qui utilisent des dépôts de code source comme contexte sur GitHub. GitLab ne prend pas en charge l'attribution automatique des relecteurs, attribuez-les donc manuellement sur les merge requests.

<div id="usage-limits">
  ## Limites d'utilisation
</div>

Les exécutions d'automatisations sont décomptées de votre utilisation de crédits. Les crédits consommés par une exécution dépendent de la taille de la tâche : le volume de contenu lu par l'agent, le nombre de fichiers modifiés et la durée de l'exécution. Les dépôts de contexte plus volumineux et les prompts plus larges utilisent plus de crédits que ceux dont la portée est restreinte.

Consultez votre utilisation de crédits, configurez les dépassements et gérez votre package de crédits sur la page [Usage](https://app.mintlify.com/settings/organization/usage) de votre dashboard.

Chaque automatisation peut s'exécuter jusqu'à 500 fois par jour. Les exécutions qui échouent ne comptent pas dans cette limite.
