  Analytics nécessite une [offre Pro ou Enterprise](https://mintlify.com/pricing?ref=analytics).
</Info>

La page [Analytics](https://dashboard.mintlify.com/products/analytics/v2/) dans votre Dashboard affiche des données sur les visiteurs de votre documentation, la façon dont ils interagissent avec l’Assistant, ce qu’ils recherchent et leurs commentaires. Utilisez ces informations pour identifier quelles pages sont les plus utiles pour vos utilisateurs et suivre les tendances dans le temps.


<div id="filter-by-ai-or-human-visitors">
  ## Filtrer par visiteurs IA ou humains
</div>

Filtrez vos données Analytics par source de trafic pour analyser séparément le trafic des agents IA et celui des visiteurs humains.

<Frame>
  <img src="/images/analytics/traffic-filter-light.png" alt="Le filtre de source de trafic sur la page Analytics." className="block dark:hidden" />

  <img src="/images/analytics/traffic-filter-dark.png" alt="Le filtre de source de trafic sur la page Analytics." className="hidden dark:block" />
</Frame>

Lorsque vous modifiez la source de trafic, la page Analytics n’affiche que les indicateurs pertinents pour cette source de trafic. Pour le trafic IA, la page Analytics affiche les visiteurs et les recherches Model Context Protocol (MCP). Pour le trafic humain, la page Analytics affiche les visiteurs, les vues de page, l’utilisation de l’Assistant, les recherches et les commentaires.

<div id="visitors">
  ## Visiteurs
</div>

L’onglet Visiteurs affiche un graphique en barres des visiteurs au fil du temps, une liste des pages les plus consultées et une liste des sites référents.

Analysez les statistiques de vos visiteurs pour :

* **Surveiller les tendances de trafic** : Observez les variations du trafic après des mises à jour ou la publication de nouveau contenu afin de comprendre l’impact de vos changements.
* **Identifier les pages populaires** : Appuyez-vous sur les pages populaires pour comprendre quel contenu est le plus important pour vos utilisateurs, afin de vous assurer qu’il est à jour et complet.
* **Suivre les sources de trafic** : Comprenez d’où viennent vos utilisateurs afin d’optimiser votre contenu pour le bon public cible.

<div id="agent-visitors">
  ### Visiteurs agents
</div>

Les visiteurs agents sont identifiés par adresse IP et user agent. Le nombre de visiteurs agents approxime les sources distinctes de trafic IA plutôt que les sessions ou conversations individuelles d'agents. Plusieurs requêtes provenant de la même adresse IP sont comptées comme un seul visiteur.

Lors de l'affichage du trafic des agents, **Top agents** remplace **Referrals** dans l'onglet **Visitors**. **Top agents** indique quels agents IA consultent votre documentation. Utilisez ces informations pour vous aider à déterminer :

* **Répartition des agents IA** : voyez quelles plateformes IA accèdent à votre documentation afin de savoir quels outils vos utilisateurs privilégient.
* **Opportunités d&#39;intégration** : identifiez les plateformes IA à prioriser pour l&#39;optimisation et les tests.
* **Tendances du trafic IA** : surveillez quels agents sont les plus actifs et comment leur utilisation évolue au fil du temps.

<div id="views">
  ## Vues
</div>

L’onglet Vues affiche un graphique en barres des vues de page au fil du temps, ainsi qu’une liste de vos pages les plus consultées par nombre de vues.

Analyser vos Analytics de vues de page pour :

* **Suivre l’engagement avec le contenu** : suivez le nombre total de vues de page pour comprendre l’utilisation globale de votre documentation et identifier les tendances.
* **Identifier les pages les plus consultées** : voyez quelles pages reçoivent le plus de vues afin de prioriser les mises à jour et de garantir que le contenu à fort trafic reste exact.

<div id="assistant">
  ## Assistant
</div>

L&#39;onglet Assistant affiche un graphique en barres de l&#39;utilisation de l&#39;Assistant au fil du temps, une liste de catégories et l&#39;historique des conversations.

Examinez vos Analytics de l&#39;Assistant pour :

* **Surveiller l&#39;utilisation de l&#39;Assistant** : Observez les évolutions de l&#39;utilisation de l&#39;Assistant pour comprendre comment vos utilisateurs interagissent avec votre contenu.
* **Identifier les sujets fréquents** : Utilisez les thèmes et catégories à forte fréquence pour comprendre quels sujets sont les plus importants pour vos utilisateurs. Identifiez les lacunes dans la couverture et priorisez les mises à jour de contenu.
* **Repérer où les utilisateurs rencontrent des difficultés** : Utilisez le sentiment pour identifier les sujets qui reçoivent des retours négatifs et priorisez l&#39;amélioration de ce contenu.
* **Examiner l&#39;historique des conversations** : Obtenez des données détaillées à forte intention sur la façon dont vos utilisateurs perçoivent votre produit en analysant l&#39;historique de leurs échanges avec l&#39;Assistant. Voyez quels termes ils utilisent, sur quoi ils ont besoin d&#39;aide et quelles tâches ils essaient d&#39;accomplir.

<div id="categories">
  ### Catégories
</div>

L’onglet Catégories attribue automatiquement une catégorie granulaire à chaque conversation, puis regroupe les catégories associées sous un thème plus large. La catégorisation regroupe les conversations par sujet, indépendamment de la langue.

Chaque thème et catégorie affiche :

- **Occurrence** : le nombre de conversations qu’il contient.
- **Sentiment** : le nombre de pouces levés et de pouces baissés issus des retours des utilisateurs sur les réponses de l’Assistant.

<Frame>
  <img src="/images/analytics/categories-light.png" alt="L’onglet Catégories dans la page Analytics de l’Assistant." className="block dark:hidden" />

  <img src="/images/analytics/categories-dark.png" alt="L’onglet Catégories dans la page Analytics de l’Assistant." className="hidden dark:block" />
</Frame>

Utilisez les thèmes pour identifier les sujets récurrents et les tendances dans les besoins des utilisateurs. Utilisez les catégories pour repérer des problèmes ciblés, comme une tâche spécifique sur laquelle les utilisateurs reviennent régulièrement ou une page qui génère des retours négatifs.

Cliquez sur une ligne de thème pour la déplier et afficher ses catégories. Cliquez sur une catégorie pour afficher les conversations associées.

Cliquez sur une conversation donnée pour afficher l’intégralité du fil de discussion, y compris la question de l’utilisateur, la réponse de l’Assistant et toutes les sources citées.

<div id="chat-history">
  ### Historique des conversations
</div>

L’onglet Historique des conversations affiche un enregistrement chronologique de toutes les conversations avec l’Assistant.

Cliquez sur un message pour afficher l’intégralité du fil de discussion, y compris la question de l’utilisateur, la réponse de l’Assistant et toutes les sources citées.

<div id="searches">
  ## Recherches
</div>

<Note>
  La métrique Recherches est disponible uniquement lors de l’affichage du trafic humain.
</Note>

L’onglet Recherches affiche un diagramme en barres des recherches au fil du temps ainsi que les requêtes spécifiques.

Analysez vos Analytics de recherche pour :

* **Surveiller les tendances de recherche** : Observez l’évolution des requêtes de recherche pour comprendre comment vos utilisateurs trouvent votre contenu et sur quels sujets ils souhaitent obtenir des informations.
* **Identifier les requêtes fréquentes** : Servez-vous des requêtes fréquentes pour comprendre quels sujets sont les plus importants pour vos utilisateurs. Identifiez les lacunes de couverture et priorisez les mises à jour de contenu.
* **Identifier les faibles taux de clics** : Les taux de clics (CTR) indiquent le nombre d’utilisateurs qui cliquent sur un résultat de recherche après avoir saisi une requête. Un CTR faible peut indiquer que les résultats de recherche ne sont pas pertinents par rapport aux requêtes des utilisateurs. Si vous avez des termes de recherche fréquents avec un CTR faible, envisagez d’améliorer la pertinence des résultats de recherche en ajoutant des mots-clés et en mettant à jour votre contenu.

<div id="mcp-searches">
  ## Recherches MCP
</div>

<Note>
  La métrique MCP Searches n’est disponible que lors de l’affichage du trafic de l’IA.
</Note>

L'onglet MCP Searches affiche les interactions de recherche des agents IA via le Model Context Protocol (MCP). Cette métrique comptabilise les appels à l'outil de recherche de votre serveur MCP ; elle reflète donc les agents que les utilisateurs ont connectés à votre serveur MCP et avec lesquels ils ont effectué une recherche, et non le trafic général des robots d'exploration IA sur vos pages de documentation.

Analysez les Analytics de vos recherches MCP pour :

* **Surveiller l’activité de recherche de l’IA** : Suivez à quelle fréquence les agents IA recherchent dans votre documentation et identifiez les tendances des requêtes générées par l’IA.
* **Identifier les requêtes IA courantes** : Comprenez quelles informations les agents IA recherchent afin d’optimiser votre contenu pour la consommation par l’IA.
* **Améliorer la découvrabilité pour l’IA** : Utilisez les requêtes de recherche MCP fréquentes pour repérer les lacunes de votre documentation qui empêchent les agents IA de trouver facilement des réponses.

<div id="export-analytics">
  ## Exporter les analyses
</div>

Exportez n’importe quel onglet d’analyses au format CSV pour une analyse approfondie, du reporting ou de l’archivage. Les exports sont disponibles pour les visiteurs, les vues, l’Assistant, les recherches, les sites référents et les commentaires, et respectent la source de trafic et la période actuellement sélectionnées.

1. Accédez à la page [Analytics](https://dashboard.mintlify.com/products/analytics/v2/) de votre tableau de bord.
2. Sélectionnez l’onglet que vous souhaitez exporter.
3. Si besoin, ajustez la source de trafic et la période pour restreindre les données à exporter.
4. Cliquez sur **Exporter en CSV**.
5. Mintlify vous envoie un e-mail avec un lien de téléchargement dès que l’export est prêt.

Les exports s’exécutent de façon asynchrone en arrière-plan, ce qui vous permet de continuer à travailler dans le tableau de bord pendant qu’un export volumineux est préparé. Le lien de téléchargement présent dans l’e-mail est valide pendant trois jours.

<div id="assistant-exports">
  ### Exports de l’Assistant
</div>

Les exports de l’Assistant contiennent les requêtes, les réponses, les sources et une colonne `resolutionStatus` qui indique si l’Assistant a répondu avec succès à chaque question (`answered` ou `unanswered`). Utilisez la colonne `resolutionStatus` pour repérer les lacunes de documentation révélées par les questions que l’Assistant n’a pas pu résoudre.

<Tip>
  Exemples de prompts d’analyse pour les exports de l’Assistant :

  * Répertorier toutes les requêtes pour lesquelles aucune source n’a été citée.
  * Identifier des tendances dans les interactions qui ont échoué.
  * Regrouper les requêtes sans réponse par sujet afin de prioriser les mises à jour de contenu.
</Tip>
