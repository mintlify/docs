La page [Analytics](https://dashboard.mintlify.com/products/analytics/v2/) dans votre Dashboard affiche des données sur les visiteurs de votre documentation, la façon dont ils interagissent avec l’Assistant, ce qu’ils recherchent et leurs commentaires. Utilisez ces informations pour identifier quelles pages sont les plus utiles pour vos utilisateurs et suivre les tendances dans le temps.

<div id="filter-by-ai-or-human-visitors">
  ## Filtrer par visiteurs IA ou humains
</div>

Filtrez vos données Analytics par source de trafic pour analyser séparément le trafic des agents IA et celui des visiteurs humains.

<Frame>
  <img src="/images/obama.jpg" alt="Le filtre de source de trafic sur la page Analytics." className="block dark:hidden" />

  <img src="/images/obama.jpg" alt="Le filtre de source de trafic sur la page Analytics." className="hidden dark:block" />
</Frame>

Lorsque vous modifiez la source de trafic, la page Analytics n’affiche que les indicateurs pertinents pour cette source de trafic. Pour le trafic IA, la page Analytics n’affiche que les visiteurs et les recherches Model Context Protocol (MCP). Pour le trafic humain, la page Analytics affiche les visiteurs, les analyses de l’Assistant, les recherches et les commentaires.

<div id="visitors">
  ## Visiteurs
</div>

L’onglet Visiteurs affiche un graphique en barres des visiteurs au fil du temps, une liste des pages les plus consultées et une liste des sites référents.

Analysez les statistiques de vos visiteurs pour :

* **Surveiller les tendances de trafic** : Observez les variations du trafic après des mises à jour ou la publication de nouveau contenu afin de comprendre l’impact de vos changements.
* **Identifier les pages populaires** : Appuyez-vous sur les pages populaires pour comprendre quel contenu est le plus important pour vos utilisateurs, afin de vous assurer qu’il est à jour et complet.
* **Suivre les sources de trafic** : Comprenez d’où viennent vos utilisateurs afin d’optimiser votre contenu pour le bon public cible.

<div id="top-agents">
  ### Agents principaux
</div>

Lors de l&#39;affichage du trafic IA, **Top agents** remplace **Referrals** dans l&#39;onglet **Visitors**. **Top agents** indique quels agents IA consultent votre documentation. Utilisez ces informations pour vous aider à déterminer :

* **Répartition des agents IA** : voyez quelles plateformes IA accèdent à votre documentation afin de savoir quels outils vos utilisateurs privilégient.
* **Opportunités d&#39;intégration** : identifiez les plateformes IA à prioriser pour l&#39;optimisation et les tests.
* **Tendances du trafic IA** : surveillez quels agents sont les plus actifs et comment leur utilisation évolue au fil du temps.

<div id="assistant">
  ## Assistant
</div>

L&#39;onglet Assistant affiche un graphique en barres de l&#39;utilisation de l&#39;Assistant au fil du temps, une liste de catégories et l&#39;historique des conversations.

Examinez vos Analytics de l&#39;Assistant pour :

* **Surveiller l&#39;utilisation de l&#39;Assistant** : Observez les évolutions de l&#39;utilisation de l&#39;Assistant pour comprendre comment vos utilisateurs interagissent avec votre contenu.
* **Identifier les catégories fréquentes** : Utilisez les catégories à forte fréquence pour comprendre quels sujets sont les plus importants pour vos utilisateurs. Identifiez les lacunes dans la couverture et priorisez les mises à jour de contenu.
* **Examiner l&#39;historique des conversations** : Obtenez des données détaillées à forte intention sur la façon dont vos utilisateurs perçoivent votre produit en analysant l&#39;historique de leurs échanges avec l&#39;Assistant. Voyez quels termes ils utilisent, sur quoi ils ont besoin d&#39;aide et quelles tâches ils essaient d&#39;accomplir.

<div id="categories">
  ### Catégories
</div>

L’onglet Catégories utilise des LLM pour regrouper automatiquement les conversations par sujet ou par thème.

<Frame>
  <img src="/images/obama.jpg" alt="L’onglet Catégories dans la page Analytics de l’Assistant." className="block dark:hidden" />

  <img src="/images/obama.jpg" alt="L’onglet Catégories dans la page Analytics de l’Assistant." className="hidden dark:block" />
</Frame>

Utilisez les catégories pour identifier les sujets récurrents, les tendances dans les besoins et comportements des utilisateurs, ainsi que les sections de la documentation qui pourraient nécessiter un enrichissement ou une clarification.

Cliquez sur une ligne de catégorie pour la déplier et afficher les conversations associées regroupées sous cette catégorie.

Cliquez sur une conversation donnée pour afficher l’intégralité du fil de discussion, y compris la question de l’utilisateur, la réponse de l’Assistant et toutes les sources citées.

<div id="chat-history">
  ### Historique des conversations
</div>

L’onglet Historique des conversations affiche un enregistrement chronologique de toutes les conversations avec l’Assistant.

Cliquez sur un message pour afficher l’intégralité du fil de discussion, y compris la question de l’utilisateur, la réponse de l’Assistant et toutes les sources citées.

<div id="export-queries">
  ### Exporter les requêtes
</div>

Pour une analyse plus approfondie, exportez un fichier CSV contenant vos requêtes, réponses et sources. Utilisez vos outils préférés pour analyser les données exportées.

<Tip>
  Exemples de prompts d’analyse :

  * Répertorier toutes les requêtes pour lesquelles aucune source n’a été citée.
  * Identifier des tendances dans les interactions qui ont échoué.
</Tip>

<div id="searches">
  ## Recherches
</div>

<Note>
  La métrique Recherches est disponible uniquement lors de l’affichage du trafic humain.
</Note>

L’onglet Recherches affiche un diagramme en barres des recherches au fil du temps ainsi que les requêtes spécifiques.

Analysez vos Analytics de recherche pour :

* **Surveiller les tendances de recherche** : Observez l’évolution des requêtes de recherche pour comprendre comment vos utilisateurs trouvent votre contenu et sur quels sujets ils souhaitent obtenir des informations.
* **Identifier les requêtes fréquentes** : Servez-vous des requêtes fréquentes pour comprendre quels sujets sont les plus importants pour vos utilisateurs. Identifiez les lacunes de couverture et priorisez les mises à jour de contenu.
* **Identifier les faibles taux de clics** : Les taux de clics (CTR) indiquent le nombre d’utilisateurs qui cliquent sur un résultat de recherche après avoir saisi une requête. Un CTR faible peut indiquer que les résultats de recherche ne sont pas pertinents par rapport aux requêtes des utilisateurs. Si vous avez des termes de recherche fréquents avec un CTR faible, envisagez d’améliorer la pertinence des résultats de recherche en ajoutant des mots-clés et en mettant à jour votre contenu.

<div id="mcp-searches">
  ## Recherches MCP
</div>

<Note>
  La métrique MCP Searches n’est disponible que lors de l’affichage du trafic de l’IA.
</Note>

L’onglet MCP Searches affiche les interactions de recherche des agents IA via le Model Context Protocol (MCP). Cette métrique montre comment les agents IA effectuent des requêtes et interagissent avec votre documentation.

Analysez les Analytics de vos recherches MCP pour :

* **Surveiller l’activité de recherche de l’IA** : Suivez à quelle fréquence les agents IA recherchent dans votre documentation et identifiez les tendances des requêtes générées par l’IA.
* **Identifier les requêtes IA courantes** : Comprenez quelles informations les agents IA recherchent afin d’optimiser votre contenu pour la consommation par l’IA.
* **Améliorer la découvrabilité pour l’IA** : Utilisez les requêtes de recherche MCP fréquentes pour repérer les lacunes de votre documentation qui empêchent les agents IA de trouver facilement des réponses.
