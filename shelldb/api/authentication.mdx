# API authentication

Learn how to securely authenticate with the ShellDB API.

## API key authentication

ShellDB uses API keys for authentication. Include your API key in the `Authorization` header:

```http
Authorization: Bearer sk_live_abc123def456
```

## Managing API keys

### Create API key
```bash
shelldb auth create-key --name "production-app" --permissions read,write
```

### List API keys
```bash
shelldb auth list-keys
```

### Revoke API key
```bash
shelldb auth revoke-key --key-id abc123
```

## Permissions

API keys can have different permission levels:

- `read` - Execute SELECT queries
- `write` - Execute INSERT, UPDATE, DELETE
- `schema` - Create/modify tables and indexes
- `admin` - Full database administration

## Best practices

1. Use separate keys for different environments
2. Rotate keys regularly
3. Use minimal required permissions
4. Store keys securely
5. Monitor key usage
