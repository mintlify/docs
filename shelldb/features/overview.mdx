# ShellDB features

ShellDB provides a comprehensive set of features designed for modern database needs. Whether you're building a simple application or a complex system, ShellDB has the tools you need.

## Core capabilities

### High performance
- **Fast queries**: Optimized query engine for rapid data retrieval
- **Efficient storage**: Compact data structures minimize disk usage
- **Memory optimization**: Smart caching for frequently accessed data
- **Concurrent access**: Multi-user support with proper locking

### Developer experience
- **Intuitive CLI**: Easy-to-use command-line interface
- **SQL compatibility**: Standard SQL syntax with extensions
- **Rich data types**: Support for text, numbers, dates, JSON, and more
- **ACID compliance**: Reliable transactions with full ACID properties

### Flexibility
- **Schema evolution**: Modify table structures without data loss
- **Extensible**: Plugin architecture for custom functionality
- **Cross-platform**: Works on Linux, macOS, and Windows
- **Multiple interfaces**: CLI, HTTP API, and native libraries

## Query capabilities

### Standard SQL support
- SELECT, INSERT, UPDATE, DELETE operations
- JOIN operations (INNER, LEFT, RIGHT, FULL OUTER)
- Subqueries and Common Table Expressions (CTEs)
- Window functions and aggregations

### Advanced features
- Full-text search capabilities
- JSON path queries
- Regular expression matching
- Custom functions and procedures

### Example query
```sql
WITH monthly_sales AS (
    SELECT 
        DATE_TRUNC('month', order_date) as month,
        SUM(total_amount) as sales
    FROM orders 
    WHERE order_date >= '2024-01-01'
    GROUP BY DATE_TRUNC('month', order_date)
)
SELECT 
    month,
    sales,
    LAG(sales) OVER (ORDER BY month) as prev_month,
    ROUND((sales - LAG(sales) OVER (ORDER BY month)) / 
          LAG(sales) OVER (ORDER BY month) * 100, 2) as growth_rate
FROM monthly_sales
ORDER BY month;
```

## Data types

ShellDB supports a rich set of data types:

| Category | Types | Description |
|----------|-------|-------------|
| Numeric | INTEGER, DECIMAL, FLOAT, DOUBLE | Numbers with various precision |
| Text | TEXT, VARCHAR(n), CHAR(n) | String data with optional length limits |
| Date/Time | DATE, TIME, TIMESTAMP, INTERVAL | Temporal data types |
| Boolean | BOOLEAN | True/false values |
| JSON | JSON, JSONB | Structured document data |
| Binary | BLOB, BYTEA | Raw binary data |

## Storage engine

### Architecture
- **B-tree indexes**: Efficient sorted data access
- **Hash indexes**: Fast equality lookups  
- **Full-text indexes**: Search across text fields
- **Partial indexes**: Index subsets of data

### Optimization features
- **Auto-vacuum**: Automatic space reclamation
- **Compression**: Data compression to save space
- **Write-ahead logging**: Crash recovery and replication
- **Connection pooling**: Efficient resource management
