Mintlify uses access tokens and webhooks to authenticate and sync changes between Bitbucket and Mintlify.

- Mintlify uses access tokens to pull information from Bitbucket.
- Bitbucket uses webhooks to notify Mintlify when you make changes, which enables preview deployments for pull requests.

## Set up the connection

<Steps>
  <Step title="Find your repository details">
    In your Bitbucket repository, navigate to **Repository settings** and locate your **Workspace** and **Repository slug**.
    
    The repository slug is the last part of your repository URL. For example, in `https://bitbucket.org/your-workspace/your-repo`, the repository slug is `your-repo`.
  </Step>
  <Step title="Generate an access token">
    Navigate to your **Personal settings** > **App passwords** and select **Create app password**.
    
    Configure the token with these settings:
    - **Label**: Mintlify
    - **Permissions**: 
      - **Repositories**: Read and Write
      - **Pull requests**: Read and Write
      - **Webhooks**: Read and Write
    
    Click **Create** and copy the token.

    <Note>
      App passwords are tied to your user account. If you leave the workspace, you'll need to create a new token with a different user.
    </Note>
  </Step>
  <Step title="Set up the connection">
    In the [Mintlify dashboard](https://dashboard.mintlify.com/settings/deployment/git-settings):
    1. Enter your workspace name, repository slug, and access token.
    2. If you have a monorepo and your documentation is not at the root of your repository, enable the **Set up as monorepo** toggle and enter the relative path to your docs directory.
    3. If you use a self-hosted Bitbucket instance, enable the **Set up as self-hosted** toggle and enter your Bitbucket instance's host URL (for example, `https://bitbucket.your-domain.com`). Your instance must be publicly accessible for Mintlify to reach it.
    4. Select the branch to deploy your documentation from.
    5. Click **Save Changes**.
  </Step>
</Steps>

## Create the webhook

Webhooks allow us to receive events when changes are made so that we can automatically trigger deployments.

<Steps>
  <Step title="Add new webhook">
    1. In Bitbucket, navigate to **Repository settings** > **Webhooks**.
    2. Click **Add webhook**.
  </Step>
  <Step title="Set up URL and webhook">
    Name the webhook **Mintlify**.

    In the **URL** field, enter the endpoint `https://leaves.mintlify.com/bitbucket-webhook`.
  </Step>
  <Step title="Get webtoken">
    In your Mintlify dashboard, click **Show Webtoken**. Copy the webtoken.
  </Step>
  <Step title="Configure webhook settings">
    In Bitbucket, configure the following settings:
    
    - **Status**: Active
    - **Triggers**: Select **Choose from a full list of triggers**
      - Under **Repository**: Select **Push**
      - Under **Pull Request**: Select **Created**, **Updated**, and **Merged**
    
    In the **URL** field, append the webtoken as a query parameter:
    ```
    https://leaves.mintlify.com/bitbucket-webhook?token=YOUR_WEBTOKEN
    ```
  </Step>
  <Step title="Save the webhook">
    Click **Save** to create the webhook.
  </Step>
  <Step title="Test the webhook">
    After you create the webhook, click **View requests** to see the webhook activity. Make a test push to your repository to verify the webhook is working correctly.
  </Step>
</Steps>
