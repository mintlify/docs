<Info>
  Multi-repository deployments are available on [Enterprise plans](https://mintlify.com/pricing?ref=multi-repo).
</Info>

Use a multi-repository deployment when one documentation site needs content from more than one Git repository. Each repository is configured as a source for the same Mintlify project and mounted at its own URL path.

For example, you can combine separate repositories for product docs, API reference, and SDK guides into one site:

```text
docs.example.com/product
docs.example.com/api
docs.example.com/sdks
```

## How it works

Each repository in a multi-repository deployment has its own:

- Git provider connection
- branch
- optional content directory
- URL path
- `docs.json`

During deployment, Mintlify reads each repository and combines the configured sources into one site. Each source appears under its configured URL path.

The first configured source's `docs.json` is used as the root `docs.json` for the deployment. Its site-level configuration controls the combined site, including settings such as theme, colors, logo, integrations, SEO, and other top-level fields. The other sources contribute their navigation and content under their configured URL paths.

<Note>
  Multi-repository deployments are different from [monorepo setup](/deploy/monorepo). Use monorepo setup when all docs live in one repository under a subdirectory. Use multi-repository deployments when docs are split across separate repositories.
</Note>

## Requirements

- An Enterprise plan.
- Admin access to your Mintlify project.
- A `docs.json` file in each repository source.
- Access to each repository through the same Git provider for the deployment.
- A unique URL path for each repository source.

<Warning>
  URL paths must be unique and cannot overlap. For example, do not configure one source at `docs` and another source at `docs/api`.
</Warning>

## Configure multiple repositories

<Steps>
  <Step title="Open Git settings">
    Navigate to [Git Settings](https://app.mintlify.com/settings/deployment/git-settings) in your dashboard.
  </Step>

  <Step title="Add another repository">
    Click **Add repository**.
  </Step>

  <Step title="Configure the repository source">
    Select the repository, branch, and any required provider-specific fields.

    For GitHub sources, make sure the Mintlify GitHub App has access to the repository. For GitLab sources, provide the project ID and deployment token.
  </Step>

  <Step title="Set the URL path">
    Enter a **URL path** for the repository source.

    The URL path determines where the source appears on your docs site. For example, a URL path of `api` serves that source under `/api`.
    <Note>
      You can enter the path with or without leading slashes. Mintlify normalizes the value when you save.
    </Note>
  </Step>

  <Step title="Set a content directory if needed">
    If the repository stores `docs.json` in a subdirectory, enable **docs.json is in a subdirectory** and enter the path to the directory that contains `docs.json`.
  </Step>

  <Step title="Save changes">
    Click **Save changes**. Mintlify validates the sources and deploys the combined site.
  </Step>
</Steps>

## Example repository layout

In this example, each source has its own repository and its own `docs.json`.

```text
acme/product-docs
├── docs.json
├── overview.mdx
└── guides/

acme/api-docs
├── docs.json
├── introduction.mdx
└── reference/

acme/sdk-docs
├── docs.json
├── quickstart.mdx
└── javascript/
```

Configure the sources with URL paths like this:

| Repository | URL path | Published path |
| --- | --- | --- |
| `acme/product-docs` | `product` | `/product` |
| `acme/api-docs` | `api` | `/api` |
| `acme/sdk-docs` | `sdks` | `/sdks` |

## Navigation behavior

Mintlify combines the navigation from each repository into one site navigation. Each repository source becomes a top-level product section, and its pages are served under the source's configured URL path.

Keep each source's navigation scoped to that repository. For example, pages in the API repository should reference files that live in the API repository, and pages in the SDK repository should reference files that live in the SDK repository.

The first repository source is the root source. Its `docs.json` provides the root site configuration for the deployment, while the remaining sources are merged into the site navigation under their URL paths.

## Removing a repository source

You can remove a repository source from Git settings. When only one repository source remains, Mintlify removes the source URL path and treats the remaining repository as the root source for the deployment.

## Limitations

- Each deployment should use one Git provider type for its repository sources. Use either GitHub repositories or GitLab repositories, not a mix of both.
- Each source must have a unique, non-overlapping URL path.
- Nested `navigation.products` configurations are not supported inside individual source repositories.
- Site-level configuration is taken from the primary source.
