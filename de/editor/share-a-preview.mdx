<div id="preview-deployments">
  ## Vorschau-Deployments
</div>

Jedes Mal, wenn Sie Änderungen an einem Feature-Branch speichern, erstellt Mintlify ein Vorschau-Deployment – eine temporäre URL, unter der Ihre Änderungen genau so gerendert werden, wie sie nach der Veröffentlichung aussehen werden. Verwenden Sie sie, um Ihre Arbeit zu überprüfen oder Feedback einzuholen, bevor Sie mergen.

<div id="access-your-preview">
  ### Auf Ihre Vorschau zugreifen
</div>

1. Klicken Sie auf **Veröffentlichen** in der Editor-Symbolleiste.
1. Klicken Sie im Veröffentlichungsmenü auf die Vorschau-URL. Das URL-Format lautet `organization-branch-name.mintlify.app`.

<div id="share-your-preview">
  ### Ihre Vorschau teilen
</div>

Kopieren Sie die Vorschau-URL und senden Sie sie an Prüfer. Die Vorschau wird automatisch aktualisiert, sobald Sie auf dem Branch speichern – kein erneutes Teilen erforderlich.

<div id="restrict-access-to-previews">
  ### Zugriff auf Vorschauen einschränken
</div>

Vorschau-URLs sind standardmäßig öffentlich zugänglich. Um den Zugriff auf Mitglieder Ihrer Mintlify-Organisation zu beschränken, aktivieren Sie die Vorschau-Authentifizierung auf der [Add-ons](https://dashboard.mintlify.com/products/addons)-Seite Ihres Dashboards.

<div id="share-editor-links">
  ## Editor-Links teilen
</div>

Um einen Teamkollegen einzuladen, eine bestimmte Seite auf einem Branch anzuzeigen oder zu bearbeiten, kopieren Sie die URL aus der Adressleiste Ihres Browsers und teilen Sie sie. Jeder mit Zugriff auf Ihre Mintlify-Organisation kann den Link direkt in seiner Editor-Sitzung öffnen.

Das URL-Format lautet:

```
https://dashboard.mintlify.com/{org}/{project}/editor/{branch}/~/{filepath}
```

Zum Beispiel: `https://dashboard.mintlify.com/acme/docs/editor/main/~/guides/quickstart.mdx`
