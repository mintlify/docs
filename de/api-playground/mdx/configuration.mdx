---
title: "MDX-Einrichtung"
description: "Dokumentationsseiten für deine API-Endpunkte mit `MDX` erstellen"
---

Du kannst API-Endpunkte in einzelnen `MDX`-Dateien manuell definieren, anstatt eine OpenAPI-Spezifikation zu verwenden. Diese Methode bietet Flexibilität für individuelle Inhalte, für die meisten Projekte empfehlen wir jedoch, die API-Dokumentation aus einer OpenAPI-Spezifikationsdatei zu generieren, da sie besser wartbar ist und mehr Funktionen bietet. Das Erstellen von `MDX`-Seiten für eine API kann dennoch nützlich sein, um kleine APIs zu dokumentieren oder für Prototypen.

Um Seiten für API-Endpunkte mit `MDX` zu erstellen, konfiguriere deine API-Einstellungen in `docs.json`, erstelle einzelne `MDX`-Dateien für jeden Endpunkt und verwende Komponenten wie `<ParamFields />`, um Parameter zu definieren. Aus diesen Definitionen generiert Mintlify interaktive API-Spielwiesen, Beispielanfragen und Beispielantworten.

<Steps>
  <Step title="API konfigurieren">
    Definiere in deiner `docs.json`-Datei die Basis-URL und die Authentifizierungsmethode:

    ```json
     "api": {
      "mdx": {
        "server": "https://mintlify.com/api", // string array for multiple base URLs
        "auth": {
          "method": "key",
          "name": "x-api-key" // options: bearer, basic, key.
        }
      }
    }
    ```

    Wenn du die API-Spielwiese ausblenden möchtest, verwende das Feld `display`. Du musst keine Authentifizierungsmethode angeben, wenn du die Spielwiese ausblendest.

    ```json
    "api": {
      "playground": {
        "display": "none"
      }
    }
    ```

    Eine vollständige Liste der API-Konfigurationen findest du unter [Settings](/de/settings#api-configurations).
  </Step>

  <Step title="Endpunktseiten erstellen">
    Jede API-Endpunktseite sollte eine entsprechende `MDX`-Datei haben. Lege am Anfang jeder Datei `title` und `api` fest:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    ---
    ```

    Du kannst Pfadparameter angeben, indem du den Parameternamen im Pfad mit `{}` umschließt:

    ```bash
    https://api.example.com/v1/endpoint/{userId}
    ```

    <Note>
      Wenn du in `docs.json` ein `server`-Feld konfiguriert hast, kannst du relative Pfade wie `/v1/endpoint` verwenden.
    </Note>

    Du kannst den global definierten Anzeigemodus der API-Spielwiese für eine Seite überschreiben, indem du `playground` im Frontmatter hinzufügst:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    playground: 'none'
    ---
    ```

    - `playground: 'interactive'` - Zeigt die interaktive Spielwiese an.
    - `playground: 'simple'` - Zeigt einen kopierbaren Endpunkt ohne Spielwiese an.
    - `playground: 'none'` - Blendet die Spielwiese aus.
  </Step>

  <Step title="Endpunkte zu den Docs hinzufügen">
    Füge deine Endpunktseiten der Seitenleiste hinzu, indem du die Pfade im Feld `navigation` in deiner `docs.json` einträgst. Mehr zur Struktur deiner Dokumentation findest du unter [Navigation](/de/navigation).
  </Step>
</Steps>

<div id="enabling-authentication">
  ## Authentifizierung aktivieren
</div>

Sie können Ihrer `docs.json` eine Authentifizierungsmethode hinzufügen, um sie global auf allen Seiten zu aktivieren, oder sie seitenweise festlegen.

Die Authentifizierungsmethode einer Seite überschreibt die globale Methode, wenn beide definiert sind.

<div id="bearer-token">
  ### Bearer-Token
</div>

<CodeGroup>
  ```json docs.json
  "api": {
      "mdx": {
        "auth": {
          "method": "bearer"
        }
      }
  }
  ```

  ```mdx Page Metadata
  ---
  title: "Your page title"
  authMethod: "bearer"
  ---
  ```
</CodeGroup>

<div id="none">
  ### Keine
</div>

Die Authentifizierungsmethode `none` ist nützlich, um die Authentifizierung für einen bestimmten Endpoint zu deaktivieren, nachdem ein Standard in der docs.json festgelegt wurde.

<CodeGroup>
  ```mdx Page Metadata
  ---
  title: "Your page title"
  authMethod: "none"
  ---
  ```
</CodeGroup>
