<div id="overview">
  ## Übersicht
</div>

Die API-Spielwiese ist eine interaktive Umgebung, in der Nutzer Ihre API-Endpunkte testen und erkunden können. Entwickler können API-Anfragen erstellen, sie senden und die Antworten einsehen, ohne Ihre Dokumentation zu verlassen.

<Frame>
  <img src="/images/playground/API-playground-light.png" alt="API-Spielwiese für den Endpunkt zum Auslösen eines Updates." className="block dark:hidden" />

  <img src="/images/playground/API-playground-dark.png" alt="API-Spielwiese für den Endpunkt zum Auslösen eines Updates." className="hidden dark:block" />
</Frame>

Die Spielwiese wird automatisch aus Ihrer OpenAPI-Spezifikation oder Ihrem AsyncAPI-Schema generiert, sodass Aktualisierungen Ihrer API automatisch in der Spielwiese sichtbar werden. Sie können auch manuell API-Referenzseiten erstellen, nachdem Sie eine Basis-URL und eine Authentifizierungsmethode in Ihrer `docs.json` definiert haben.

Wir empfehlen, Ihre API-Spielwiese aus einer OpenAPI-Spezifikation zu generieren. Weitere Informationen zum Erstellen Ihres OpenAPI-Dokuments finden Sie unter [OpenAPI Setup](/de/api-playground/openapi-setup).

<div id="getting-started">
  ## Erste Schritte
</div>

<Steps>
  <Step title="Füge deine OpenAPI-Spezifikationsdatei hinzu.">
    <Info>
      Stelle sicher, dass deine OpenAPI-Spezifikationsdatei gültig ist – zum Beispiel mit dem [Swagger Editor](https://editor.swagger.io/) oder der [Mint-CLI](https://www.npmjs.com/package/mint).
    </Info>

    ```bash {2}
    /your-project
      |- docs.json
      |- openapi.json
    ```
  </Step>

  <Step title="Konfiguriere `docs.json`.">
    Aktualisiere deine `docs.json`, um auf deine OpenAPI-Spezifikation zu verweisen. Füge einem beliebigen Navigationselement die Eigenschaft `openapi` hinzu, um deine Doku automatisch mit Seiten für jeden im OpenAPI-Dokument definierten Endpunkt zu befüllen.

    Dieses Beispiel erstellt für jeden in `openapi.json` definierten Endpunkt eine Seite und ordnet sie in deiner Navigation der Gruppe „API reference“ zu.

    ```json
    "navigation": {
      "groups": [
        {
          "group": "API reference",
          "openapi": "openapi.json"
        }
      ]
    }
    ```

    Um nur für bestimmte Endpunkte Seiten zu generieren, liste die Endpunkte in der Eigenschaft `pages` des Navigationselements auf.

    Dieses Beispiel generiert Seiten nur für die Endpunkte `GET /users` und `POST /users`. Um weitere Endpunktseiten zu generieren, füge zusätzliche Endpunkte zum Array `pages` hinzu.

    ```json
    "navigation": {
      "groups": [
          {
            "group": "API reference",
            "openapi": "openapi.json",
            "pages": [
              "GET /users",
              "POST /users"
            ]
          }
      ]
    }
    ```
  </Step>
</Steps>

<div id="customizing-your-playground">
  ## Anpassen Ihrer Spielwiese
</div>

Sie können Ihre API-Spielwiese anpassen, indem Sie die folgenden Eigenschaften in Ihrer `docs.json` definieren.

<ResponseField name="playground" type="object">
  Konfigurationen für die API-Spielwiese.

  <Expandable title="playground" defaultOpen="True">
    <ResponseField name="display" type="&#x22;interactive&#x22; | &#x22;simple&#x22; | &#x22;none&#x22;">
      Der Anzeigemodus der API-Spielwiese.

      - `"interactive"`: Zeigt die interaktive Spielwiese.
      - `"simple"`: Zeigt einen kopierbaren Endpoint ohne Spielwiese.
      - `"none"`: Zeigt nichts.

      Standardmäßig `interactive`.
    </ResponseField>

    <ResponseField name="proxy" type="boolean" defaultOpen="True">
      Ob API-Anfragen über einen Proxyserver geleitet werden. Standardmäßig `true`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="examples" type="object">
  Konfigurationen für die automatisch generierten API-Beispiele.

  <Expandable title="examples" defaultOpen="True">
    <ResponseField name="languages" type="array of string">
      Beispielsprachen für die automatisch generierten API-Snippets.

      Sprachen werden in der angegebenen Reihenfolge angezeigt.
    </ResponseField>

    <ResponseField name="defaults" type="&#x22;required&#x22; | &#x22;all&#x22;">
      Ob optionale Parameter in API-Beispielen angezeigt werden. Standardmäßig `all`.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="example-configuration">
  ### Beispielkonfiguration
</div>

```json
{
 "api": {
   "playground": {
     "display": "interactive"
   },
   "examples": {
     "languages": ["curl", "python", "javascript"],
     "defaults": "required"
   }
 }
}
```

Dieses Beispiel konfiguriert die API-Spielwiese als interaktiv und zeigt Beispiel-Code-Snippets für cURL, Python und JavaScript. In den Code-Snippets werden nur erforderliche Parameter angezeigt.

<div id="custom-endpoint-pages">
  ### Benutzerdefinierte Endpoint-Seiten
</div>

Wenn Sie mehr Kontrolle über Ihre API-Dokumentation benötigen, verwenden Sie die `x-mint`-Erweiterung in Ihrer OpenAPI-Spezifikation oder erstellen Sie einzelne `MDX`-Seiten für Ihre Endpunkte.

Beide Optionen ermöglichen Ihnen:

- Seitenmetadaten anzupassen
- Zusätzliche Inhalte wie Beispiele hinzuzufügen
- Das Verhalten der Spielwiese pro Seite zu steuern

Die `x-mint`-Erweiterung wird empfohlen, damit Ihre gesamte API-Dokumentation automatisch aus Ihrer OpenAPI-Spezifikation generiert und in einer Datei gepflegt wird.

Einzelne `MDX`-Seiten werden für kleine APIs oder wenn Sie Änderungen seitenweise ausprobieren möchten, empfohlen.

Weitere Informationen finden Sie unter [x-mint extension](/de/api-playground/openapi-setup#x-mint-extension) und [MDX Setup](/de/api-playground/mdx/configuration).
