---
title: "Sichtbarkeit von Seiten verwalten"
description: "Steuern, welche Endpunkte aus Ihrer OpenAPI-Spezifikation in der Navigationsstruktur Ihrer Dokumentation erscheinen"
---

Sie können festlegen, welche OpenAPI-Operationen als Dokumentationsseiten veröffentlicht werden und wie sichtbar sie in der Navigation sind. Das ist nützlich für ausschließlich intern genutzte Endpunkte, veraltete Operationen, Beta‑Funktionen oder Endpunkte, die über eine direkte URL erreichbar sein sollen, aber nicht über die Seitennavigation auffindbar.

Wenn Ihre Seiten aus einem OpenAPI-Dokument automatisch generiert werden, können Sie die Sichtbarkeit mit den Erweiterungen `x-hidden` und `x-excluded` steuern.

<div id="x-hidden">
  ## `x-hidden`
</div>

Die Erweiterung `x-hidden` erzeugt zwar eine Seite für einen Endpoint, blendet sie aber aus der Navigation aus. Die Seite ist nur erreichbar, wenn Sie die URL direkt aufrufen.

Häufige Anwendungsfälle für `x-hidden` sind:

- Endpoints, die Sie dokumentieren, aber nicht hervorheben möchten.
- Seiten, auf die Sie aus anderem Inhalt verlinken.
- Endpoints für bestimmte Nutzer.

<div id="x-excluded">
  ## `x-excluded`
</div>

Die Erweiterung `x-excluded` schließt einen Endpunkt vollständig aus Ihrer Dokumentation aus.

Häufige Anwendungsfälle für `x-excluded` sind:

- Endpunkte, die nur intern verwendet werden.
- Veraltete Endpunkte, die Sie nicht dokumentieren möchten.
- Beta-Funktionen, die noch nicht für die öffentliche Dokumentation bereit sind.

<div id="implementation">
  ## Implementierung
</div>

Fügen Sie die Erweiterung `x-hidden` oder `x-excluded` unter der HTTP-Methode in Ihrer OpenAPI-Spezifikation hinzu.

Hier sind Beispiele, wie Sie jede Eigenschaft in einem OpenAPI-Schemadokument für einen Endpoint und einen Webhook-Pfad verwenden.

```json {11, 19}
"paths": {
  "/plants": {
    "get": {
      "description": "Gibt alle Pflanzen aus dem Store zurück",
      "parameters": { /*...*/ },
      "responses": { /*...*/ }
    }
  },
  "/hidden_plants": {
    "get": {
      "x-hidden": true,
      "description": "Gibt alle eher geheimen Pflanzen aus dem Store zurück",
      "parameters": { /*...*/ },
      "responses": { /*...*/ }
    }
  },
  "/secret_plants": {
    "get": {
      "x-excluded": true,
      "description": "Gibt alle streng geheimen Pflanzen aus dem Store zurück (diesen Endpoint nicht veröffentlichen!)",
      "parameters": { /*...*/ },
      "responses": { /*...*/ }
    }
  }
},
```

```json {9, 15}
"webhooks": {
  "/plants_hook": {
    "post": {
      "description": "Webhook mit Informationen zu einer neuen, im Store hinzugefügten Pflanze",
    }
  },
  "/hidden_plants_hook": {
    "post": {
      "x-hidden": true,
      "description": "Webhook mit eher geheimen Informationen zu einer neuen, im Store hinzugefügten Pflanze"
    }
  },
  "/secret_plants_hook": {
    "post": {
      "x-excluded": true,
      "description": "Webhook mit streng geheimen Informationen zu einer neuen, im Store hinzugefügten Pflanze (diesen Endpoint nicht veröffentlichen!)"
    }
  }
}
```
