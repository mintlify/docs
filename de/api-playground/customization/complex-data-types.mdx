title: "Komplexe Datentypen"
description: "Beschreiben Sie APIs mit flexiblen Schemata, optionalen Eigenschaften und mehreren Datenformaten mithilfe der Schlüsselwörter `oneOf`, `anyOf` und `allOf`"
---

Wenn Ihre API mehrere Datenformate akzeptiert, bedingte Felder enthält oder Vererbungsmuster verwendet, helfen die Schema-Kompositions-Keywords von OpenAPI, diese flexiblen Strukturen zu dokumentieren. Mit `oneOf`, `anyOf` und `allOf` können Sie APIs beschreiben, die unterschiedliche Eingabetypen verarbeiten oder mehrere Schemata zu umfassenden Datenmodellen kombinieren.

<div id="oneof-anyof-allof-keywords">
  ## `oneOf`, `anyOf`, `allOf` Schlüsselwörter
</div>

Für komplexe Datentypen stellt OpenAPI Schlüsselwörter zum Kombinieren von Schemas bereit:

- `allOf`: Kombiniert mehrere Schemas (z. B. Objekte zusammenführen oder ein Basisschema erweitern). Entspricht einem `and`-Operator. 
- `anyOf`: Akzeptiert Daten, die einem der bereitgestellten Schemas entsprechen. Entspricht einem `or`-Operator.
- `oneOf`: Akzeptiert Daten, die genau einem der bereitgestellten Schemas entsprechen. Entspricht einem `exclusive-or`-Operator.

<Warning>Mintlify behandelt `oneOf` und `anyOf` identisch, da der praktische Unterschied die API-Nutzung nur selten beeinflusst.</Warning>

Ausführliche Spezifikationen dieser Schlüsselwörter finden Sie in der [OpenAPI-Dokumentation](https://swagger.io/docs/specification/data-models/oneof-anyof-allof-not/).

<Info>Das Schlüsselwort `not` wird derzeit nicht unterstützt.</Info>

<div id="combining-schemas-with-allof">
  ### Schemas mit `allOf` kombinieren
</div>

Wenn Sie `allOf` verwenden, führt Mintlify ein gewisses Preprocessing an Ihrem OpenAPI-Dokument durch, um komplexe Kombinationen lesbar darzustellen. Wenn Sie beispielsweise zwei Objektschemas mit `allOf` kombinieren, führt Mintlify die Eigenschaften beider zu einem einzigen Objekt zusammen. Das ist besonders nützlich, wenn Sie die wiederverwendbaren [components](https://swagger.io/docs/specification/components/) von OpenAPI nutzen.

```yaml
org_with_users:
  allOf:
    - $ref: '#/components/schemas/Org'
    - type: object
      properties:
        users:
          type: array
          description: An array containing all users in the organization
# ...
components:
  schemas:
    Org:
      type: object
      properties:
        id:
          type: string
          description: The ID of the organization
```

<ParamField body="org_with_users" type="object">
  <Expandable>
    <ParamField body="id" type="string">
      Die ID der Organisation
    </ParamField>
    <ParamField body="users" type="object[]">
      Ein Array, das alle Nutzer in der Organisation enthält
    </ParamField>
  </Expandable>
</ParamField>

<div id="providing-options-with-oneof-and-anyof">
  ### Optionen mit `oneOf` und `anyOf` bereitstellen
</div>

Wenn Sie `oneOf` oder `anyOf` verwenden, werden die Optionen in einem Tab-Container angezeigt. Geben Sie in jedem Unterschema ein `title`-Feld an, um den Optionen Namen zuzuweisen. Hier ein Beispiel, wie Sie zwei verschiedene Arten von Lieferadressen darstellen könnten:

```yaml
delivery_address:
  oneOf:
    - title: StreetAddress
      type: object
      properties:
        address_line_1:
          type: string
          description: The street address of the recipient
        # ...
    - title: POBox
      type: object
      properties:
        box_number:
          type: string
          description: The number of the PO Box
        # ...
```
