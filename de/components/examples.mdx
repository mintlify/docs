Die Komponenten `<RequestExample>` und `<ResponseExample>` zeigen Codeblöcke in der rechten Seitenleiste an und erzeugen so ein zweispaltiges Layout, das Beispiele sichtbar hält, während Nutzer durch Ihre Inhalte scrollen. Diese Komponenten sind für API-Dokumentation konzipiert, funktionieren jedoch auf allen Seiten.

Häufige Anwendungsfälle:

- API-Endpunktdokumentation mit Request- und Response-Beispielen
- Konfigurationsbeispiele neben erklärendem Text
- Codebeispiele, auf die sich Nutzer beim Befolgen von Anleitungen beziehen
- Vorher-/Nachher-Beispiele in Tutorials

Auf Mobilgeräten werden die Komponenten `<RequestExample>` und `<ResponseExample>` als reguläre Codeblöcke angezeigt und können überscrollt werden.

<div id="requestexample">
  ## RequestExample
</div>

Verwende `<RequestExample>`, um Codebeispiele in der rechten Seitenleiste anzuheften. Diese Komponente funktioniert ähnlich wie die Komponente [CodeGroup](/de/components/code-groups), zeigt den Code jedoch in der Seitenleiste statt inline an.

Du kannst mehrere Codeblöcke in einem einzigen `<RequestExample>` hinzufügen. Jeder Codeblock muss ein title-Attribut besitzen.

````mdx RequestExample
<RequestExample>

```bash Request
  curl --request POST \
    --url https://dog-api.kinduff.com/api/facts
```

</RequestExample>
````

<div id="responseexample">
  ## ResponseExample
</div>

Die Komponente `<ResponseExample>` fixiert Codebeispiele in der rechten Seitenleiste unterhalb sämtlicher `<RequestExample>`-Inhalte auf derselben Seite.

````mdx ResponseExample
<ResponseExample>

```json Response
{ "status": "success" }
```

</ResponseExample>
````
