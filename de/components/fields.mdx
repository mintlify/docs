Verwenden Sie Felder, um API-Parameter und -Antworten zu dokumentieren. Es gibt zwei Arten von Feldern: Parameterfelder und Antwortfelder.

<div id="parameter-field">
  ## Parameterfeld
</div>

Die `<ParamField>`-Komponente wird verwendet, um Parameter für Ihre APIs oder SDKs zu definieren. Das Hinzufügen eines `ParamField` fügt automatisch eine [API-Spielwiese](/de/api-playground/overview) hinzu.

<ParamField path="param" type="string" required>
  Ein Beispiel für ein Parameterfeld
</ParamField>

```mdx
<ParamField path="param" type="string" required>
  Ein Beispiel für ein Parameterfeld
</ParamField>
```

<div id="properties">
  ### Eigenschaften
</div>

<ParamField body="query, path, body, or header" type="string">
  Ob der Parameter eine Query, ein Pfad, ein Body oder ein Header ist. Danach folgt der
  Parameternamen.
</ParamField>

<ParamField body="type" type="string">
  Erwarteter Typ des Parameterwerts.

  Unterstützt `number`, `string`, `boolean`, `object`.

  Arrays können mit dem Suffix `[]` definiert werden. Zum Beispiel `string[]`.
</ParamField>

<ParamField body="required" type="boolean">
  Gibt an, ob der Parameter erforderlich ist.
</ParamField>

<ParamField body="deprecated" type="boolean">
  Gibt an, ob der Parameter veraltet ist.
</ParamField>

<ParamField body="default" type="any">
  Standardwert, der verwendet wird, wenn der Request-Wert leer ist
</ParamField>

<ParamField body="placeholder" type="string">
  Platzhaltertext für die Eingabe in der API-Spielwiese.
</ParamField>

<ParamField body="children" type="string">
  Beschreibung des Parameters (Markdown-aktiviert).
</ParamField>

<div id="response-field">
  ## Antwortfeld
</div>

Die `<ResponseField>`-Komponente definiert die Rückgabewerte einer API.

<ResponseField name="response" type="string" required>
  Beispiel für ein Antwortfeld
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  A response field example
</ResponseField>
```

<div id="properties">
  ### Eigenschaften
</div>

<ResponseField name="name" type="string" required>
  Der Name des Rückgabewerts.
</ResponseField>

<ResponseField name="type" type="string" required>
  Erwarteter Typ des Rückgabewerts. Dies kann ein beliebiger String sein.
</ResponseField>

<ResponseField name="default" type="string">
  Der Standardwert.
</ResponseField>

<ResponseField name="required" type="boolean">
  Gibt an, ob die Antwort erforderlich ist.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Gibt an, ob ein Feld veraltet ist.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Labels, die vor dem Feldnamen angezeigt werden.
</ResponseField>

<ResponseField name="post" type="string[]">
  Labels, die nach dem Feldnamen angezeigt werden.
</ResponseField>
