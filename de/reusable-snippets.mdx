Eines der Kernprinzipien der Softwareentwicklung ist DRY („Don't Repeat Yourself“), das auch für Dokumentation gilt. Wenn Sie feststellen, dass Sie denselben Inhalt an mehreren Stellen wiederholen, sollten Sie ein benutzerdefiniertes Snippet erstellen, um Ihre Inhalte konsistent zu halten.

<div id="creating-a-custom-snippet">
  ## Erstellen eines benutzerdefinierten Snippets
</div>

**Voraussetzung**: Sie müssen Ihre Snippet-Datei im Verzeichnis `snippets` erstellen, damit der Import funktioniert.

Jede Seite im Verzeichnis `snippets` wird als Snippet behandelt und nicht
als eigenständige Seite gerendert. Wenn Sie aus dem Snippet eine eigenständige Seite erstellen möchten,
importieren Sie das Snippet in eine andere Datei und verwenden Sie es als
Komponente.

<div id="default-export">
  ### Standardexport
</div>

1. Fügen Sie Ihrer Snippet-Datei Inhalte hinzu, die Sie wiederverwenden möchten. Optional können Sie Variablen hinzufügen, die beim Import des Snippets über Props
   befüllt werden können. In diesem Beispiel lautet unsere Variable word.

```typescript snippets/my-snippet.mdx
Hello world! This is my content I want to reuse across pages. 
```

2. Importieren Sie das Snippet in Ihre Zieldatei.

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import MySnippet from '/snippets/path/to/my-snippet.mdx';

## Header

Lorem impsum dolor sit amet.

<MySnippet/>

```

<div id="exporting-with-variables">
  ### Export mit Variablen
</div>

1. Optional können Sie Variablen hinzufügen, die beim Import des Snippets über Props befüllt werden können. In diesem Beispiel lautet unsere Variable word.

```typescript snippets/my-snippet.mdx
My keyword of the day is {word}.
```

2. Importieren Sie das Snippet mit der Variable in Ihre Zieldatei. Die Property wird entsprechend Ihrer Angabe befüllt.

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import MySnippet from '/snippets/path/to/my-snippet.mdx';

## Header

Lorem impsum dolor sit amet.

<MySnippet word="bananas" />

```

<div id="reusable-variables">
  ### Wiederverwendbare Variablen
</div>

1. Exportieren Sie eine Variable aus Ihrer Snippet-Datei:

```typescript snippets/path/to/custom-variables.mdx
export const myName = 'my name';

export const myObject = { fruit: 'strawberries' };
```

2. Importieren Sie das Snippet in Ihre Zieldatei und verwenden Sie die Variable:

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import { myName, myObject } from '/snippets/path/to/custom-variables.mdx';

Hello, my name is {myName} and I like {myObject.fruit}.
```

<div id="jsx-snippets">
  ### JSX-Snippets
</div>

1. Exportieren Sie eine JSX-Komponente aus Ihrer Snippet-Datei. (Weitere Informationen finden Sie unter [React components](/de/react-components)):

```js icon=square-js snippets/my-jsx-snippet.jsx
export const MyJSXSnippet = () => {
  return (
    <div>
      <h1>Hello, world!</h1>
    </div>
  )
}
```

<Note>
  Wichtig: Verwenden Sie beim Erstellen von JSX-Snippets die Pfeilfunktionssyntax (`=>`) anstelle von Funktionsdeklarationen. Das Schlüsselwort `function` wird in diesem Kontext nicht unterstützt.
</Note>

2. Importieren Sie das Snippet in Ihre Zieldatei und verwenden Sie die Komponente:

```typescript destination-file.mdx
---
title: My title
description: My Description
---

import { MyJSXSnippet } from '/snippets/my-jsx-snippet.jsx';

<MyJSXSnippet />
```
