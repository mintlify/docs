<div id="installing-the-cli">
  ## Installation der CLI
</div>

<Info>
  **Voraussetzung**: Bitte installieren Sie [Node.js](https://nodejs.org/en), bevor Sie fortfahren.
</Info>

<Steps>
  <Step title="CLI installieren.">
    Führen Sie den folgenden Befehl aus, um die [CLI](https://www.npmjs.com/package/mint) zu installieren:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>
  </Step>

  <Step title="Lokal vorab ansehen.">
    Navigieren Sie zu Ihrem Docs-Verzeichnis (in dem sich Ihre Datei `docs.json` befindet) und führen Sie den folgenden Befehl aus:

    ```bash
    mint dev
    ```

    Eine lokale Vorschau Ihrer Dokumentation ist unter `http://localhost:3000` verfügbar.
  </Step>
</Steps>

Alternativ können Sie, wenn Sie die CLI nicht global installieren möchten, ein einmaliges Skript ausführen:

```bash
npx mint dev
```

<div id="updates">
  ## Updates
</div>

Wenn Ihre lokale Vorschau nicht mit der Produktionsversion im Web übereinstimmt, aktualisieren Sie Ihre lokale CLI:

```bash
mint update
```

Wenn der Befehl `mint update` in Ihrer lokalen Version nicht verfügbar ist, installieren Sie die CLI in der neuesten Version neu:

<CodeGroup>
  ```bash npm
  npm i -g mint@latest
  ```

  ```bash pnpm
  pnpm add -g mint@latest
  ```
</CodeGroup>

<div id="custom-ports">
  ## Benutzerdefinierte Ports
</div>

Standardmäßig verwendet die CLI Port 3000. Sie können den Port mit dem Flag `--port` anpassen. Um die CLI beispielsweise auf Port 3333 auszuführen, verwenden Sie diesen Befehl:

```bash
mint dev --port 3333
```

Wenn Sie versuchen, einen bereits belegten Port zu nutzen, wird automatisch der nächste verfügbare Port verwendet:

```mdx
Port 3000 ist bereits belegt. Versuche stattdessen 3001.
```

<div id="previewing-as-a-specific-group">
  ## Vorschau als bestimmte Gruppe
</div>

Wenn Sie die Partielle Authentifizierung verwenden, um den Zugriff auf Ihre Dokumentation einzuschränken, können Sie mit dem Flag `--group [groupname]` eine Vorschau als bestimmte Authentifizierungsgruppe starten.

Wenn Sie beispielsweise eine Gruppe namens `admin` haben, können Sie als Mitglied dieser Gruppe mit folgendem Befehl eine Vorschau starten:

```bash
mint dev --group admin
```

<div id="additional-commands">
  ## Zusätzliche Befehle
</div>

Auch wenn `mint dev` der am häufigsten verwendete Befehl ist, gibt es weitere Befehle, mit denen Sie Ihre Dokumentation verwalten können.

<div id="finding-broken-links">
  ### Defekte Links finden
</div>

Die CLI unterstützt Sie beim Validieren von Referenzlinks in Ihrer Dokumentation. Verwenden Sie den folgenden Befehl, um defekte Links zu erkennen:

```bash
mint broken-links
```

<div id="checking-openapi-spec">
  ### OpenAPI-Spezifikation prüfen
</div>

Sie können die CLI verwenden, um Ihre OpenAPI-Datei mit dem folgenden Befehl auf Fehler zu prüfen:

```bash
mint openapi-check <openapiFilenameOrUrl>
```

Sie können entweder einen Dateinamen (z. B. `./openapi.yaml`) oder eine URL (z. B. `https://petstore3.swagger.io/api/v3/openapi.json`) übergeben.

<div id="renaming-files">
  ### Dateien umbenennen
</div>

Sie können Dateien umbenennen und alle Verweise darauf mit dem folgenden Befehl aktualisieren:

```bash
mint rename <oldFilename> <newFilename>
```

<div id="migrating-mdx-endpoint-pages">
  ### MDX-Endpunktseiten migrieren
</div>

Sie können MDX-Endpunktseiten mit dem folgenden Befehl in automatisch generierte Seiten aus Ihrer OpenAPI-Spezifikation migrieren:

```bash
mint migrate-mdx
```

Dieser Befehl konvertiert einzelne MDX-Endpunktseiten in automatisch generierte Seiten, die in Ihrer `docs.json` definiert sind, verschiebt MDX-Inhalte in die Erweiterung `x-mint` Ihrer OpenAPI-Spezifikation und aktualisiert Ihre Navigation. Ausführliche Informationen finden Sie unter [Migration von MDX](/de/api-playground/migrating-from-mdx).

<div id="formatting">
  ## Formatierung
</div>

Für die lokale Entwicklung empfehlen wir, Erweiterungen in deiner IDE zu nutzen, um `MDX`-Dateien zu erkennen und zu formatieren.

Wenn du Cursor, Windsurf oder VS Code verwendest, empfehlen wir die [MDX-VS-Code-Erweiterung](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) für die Syntaxhervorhebung sowie [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) für die Codeformatierung.

Wenn du JetBrains verwendest, empfehlen wir das [MDX-Plugin für IntelliJ IDEA](https://plugins.jetbrains.com/plugin/14944-mdx) für die Syntaxhervorhebung und die Einrichtung von [Prettier](https://prettier.io/docs/webstorm) für die Codeformatierung.

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

<AccordionGroup>
  <Accordion title="Fehler: Das Modul „sharp“ konnte mit der darwin-arm64-Runtime nicht geladen werden">
    Das kann an einer veralteten Node-Version liegen. Probiere Folgendes:

    1. Entferne die aktuell installierte Version der Mint-CLI: `npm uninstall -g mint`
    2. Aktualisiere auf die neueste Node.js-Version.
    3. Installiere die Mint-CLI erneut: `npm install -g mint`
  </Accordion>

  <Accordion title="Problem: Unbekannter Fehler">
    **Lösung**: Wechsle in das Root-Verzeichnis deines Geräts und lösche den Ordner `~/.mintlify`. Führe anschließend `mint dev` erneut aus.
  </Accordion>

  <Accordion title="Fehler: permission denied">
    Ursache sind fehlende Berechtigungen, um Node-Pakete global zu installieren.

    **Lösung**: Führe `sudo npm i -g mint` aus. Du wirst nach deinem Passwort gefragt – dem, mit dem du deinen Computer entsperrst.
  </Accordion>

  <Accordion title="Die lokale Vorschau sieht anders aus als meine Docs im Web">
    Das liegt wahrscheinlich an einer veralteten CLI-Version.

    **Lösung:** Führe `mint update` aus, um die neuesten Änderungen zu erhalten.
  </Accordion>

  <Accordion title="mintlify vs. mint package">
    Wenn du Probleme mit dem CLI-Paket hast, führe zuerst `npm ls -g` aus. Dieser Befehl zeigt, welche Pakete auf deinem Gerät global installiert sind.

    Wenn du npm nicht verwendest oder es nicht in der -g-Liste erscheint, nutze `which mint`, um die Installation zu finden.

    Wenn sowohl ein Paket namens `mint` als auch eines namens `mintlify` installiert ist, solltest du `mintlify` deinstallieren.

    1. Deinstalliere das alte Paket:

    ```bash
      npm uninstall -g mintlify
    ```

    2. Leere den npm-Cache:

    ```bash
      npm cache clean --force
    ```

    3. Installiere das neue Paket:

    ```bash
    npm i -g mint
    ```
  </Accordion>
</AccordionGroup>
