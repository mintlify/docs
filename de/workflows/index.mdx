<Info>
  Workflows befinden sich in der Beta-Phase. Funktionen, Verfügbarkeit und Preise können sich ändern.
</Info>

Workflows führen den Agenten automatisch nach einem Zeitplan oder als Reaktion auf Pushes in ein Repository aus. Jeder Workflow definiert einen Prompt für den Agenten und einen Auslöser für den Ausführungszeitpunkt. Workflows unterstützen sowohl GitHub- als auch GitLab-Repositories.

Wenn ein Workflow ausgeführt wird, klont der Agent alle angegebenen Repositories als Kontext und folgt dem Prompt.

<Tip>
Verwenden Sie Workflows, die nach einem Zeitplan ausgeführt werden, um wiederkehrende Aufgaben zu automatisieren, wie das Veröffentlichen von Änderungsprotokollen oder die Überprüfung von Grammatik- und Stilproblemen.

Verwenden Sie Workflows, die bei Push-Ereignissen ausgeführt werden, um reaktive Wartungsaufgaben zu automatisieren, wie das Aktualisieren von API-Referenzdokumentation oder das Identifizieren von Dokumentationsaktualisierungen, die für neue Funktionen erforderlich sind.
</Tip>

<div id="trigger-types">
  ## Auslösertypen
</div>

Workflows unterstützen zwei Auslösertypen.

- **Zeitplan (cron)** — Ausführung nach einem wiederkehrenden täglichen, wöchentlichen oder monatlichen Zeitplan. Workflows werden innerhalb von 10 Minuten nach der geplanten Zeit in die Warteschlange gestellt.
- **Push-Ereignisse** — Ausführung, wenn Änderungen in ein bestimmtes Repository oder einen bestimmten Branch gepusht werden, einschließlich Pull-Request-Merges und direkter Pushes.

Sie können Auslösertypen nicht in einem einzelnen Workflow kombinieren. Jeder Workflow hat genau einen Auslöser.

<div id="usage-limits">
  ## Nutzungsgrenzen
</div>

Jeder Workflow kann bis zu 500 Mal pro Tag ausgeführt werden. Projekte des Hobby-Plans haben ein zusätzliches Limit von insgesamt 100 Workflow-Ausführungen pro Monat. Fehlgeschlagene Ausführungen zählen nicht zu diesen Limits.
