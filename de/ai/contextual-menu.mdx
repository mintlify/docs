Das Kontextmenü bietet schnellen Zugriff auf KI-optimierte Inhalte und direkte Integrationen mit beliebten KI-Tools. Wenn Nutzende das Kontextmenü auf einer beliebigen Seite öffnen, können sie Inhalte als Kontext für KI-Tools kopieren oder Unterhaltungen in ChatGPT, Claude, Perplexity oder einem benutzerdefinierten Tool Ihrer Wahl starten – mit Ihrer Dokumentation bereits als Kontext geladen.

<div id="menu-options">
  ## Menüoptionen
</div>

Das Kontextmenü enthält mehrere vorkonfigurierte Optionen, die Sie aktivieren können, indem Sie deren Bezeichner zu Ihrer Konfiguration hinzufügen.

| Option | Bezeichner | Beschreibung |
|:--------|:------------|:-------------|
| **Seite kopieren** | `copy` | Kopiert die aktuelle Seite als Markdown, um sie als Kontext in KI-Tools einzufügen |
| **Als Markdown anzeigen** | `view` | Öffnet die aktuelle Seite als Markdown |
| **In ChatGPT öffnen** | `chatgpt` | Erstellt eine ChatGPT-Konversation mit der aktuellen Seite als Kontext |
| **In Claude öffnen** | `claude` | Erstellt eine Claude-Konversation mit der aktuellen Seite als Kontext |
| **In Perplexity öffnen** | `perplexity` | Erstellt eine Perplexity-Konversation mit der aktuellen Seite als Kontext |
| **MCP-Server-URL kopieren** | `mcp` | Kopiert Ihre MCP-Server-URL in die Zwischenablage |
| **Mit Cursor verbinden** | `cursor` | Installiert Ihren gehosteten MCP-Server in Cursor |
| **Mit VS Code verbinden** | `vscode` | Installiert Ihren gehosteten MCP-Server in VS Code |

<div id="enabling-the-contextual-menu">
  ## Aktivieren des Kontextmenüs
</div>

Fügen Sie Ihrer Datei `docs.json` das Feld `contextual` hinzu und geben Sie an, welche Optionen enthalten sein sollen.

```json
{
 "contextual": {
   "options": [
     "copy",
     "view",
     "chatgpt",
     "claude",
     "perplexity",
     "mcp",
     "cursor",
     "vscode"
   ]
 }
}
```

<div id="adding-custom-options">
  ## Benutzerdefinierte Optionen hinzufügen
</div>

Erstellen Sie benutzerdefinierte Optionen im Kontextmenü, indem Sie ein Objekt zum Array `options` hinzufügen. Jede benutzerdefinierte Option benötigt diese Eigenschaften:

<ResponseField name="title" type="string" required>
  Der Titel der Option.
</ResponseField>

<ResponseField name="description" type="string" required>
  Die Beschreibung der Option. Wird unter dem Titel angezeigt, wenn das Kontextmenü aufgeklappt ist.
</ResponseField>

<IconsRequired />

<ResponseField name="href" type="string | object" required>
  Das href der Option. Verwenden Sie eine Zeichenfolge für einfache Links oder ein Objekt für dynamische Links mit Query-Parametern.
  
  <Expandable title="href object">
    <ResponseField name="base" type="string" required>
      Die Basis-URL der Option.
    </ResponseField>

    <ResponseField name="query" type="object" required>
      Die Query-Parameter der Option.

      <Expandable title="query object">
        <ResponseField name="key" type="string" required>
          Der Schlüssel des Query-Parameters.
        </ResponseField>

        <ResponseField name="value" type="string" required>
          Der Wert des Query-Parameters. Wir ersetzen die folgenden Platzhalter durch die entsprechenden Werte:
          - Verwenden Sie `$page`, um den aktuellen Seiteninhalt in Markdown einzufügen.
          - Verwenden Sie `$path`, um den aktuellen Seitenpfad einzufügen.
          - Verwenden Sie `$mcp`, um die URL des gehosteten MCP-Servers einzufügen.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

Beispiel für eine benutzerdefinierte Option:

```json {9-14} wrap
{
    "contextual": {
        "options": [
            "copy",
            "view",
            "chatgpt",
            "claude",
            "perplexity",
            {
                "title": "Request a feature",
                "description": "Join the discussion on GitHub to request a new feature",
                "icon": "plus",
                "href": "https://github.com/orgs/mintlify/discussions/categories/feature-requests"
            }
        ]
    }
}
```
