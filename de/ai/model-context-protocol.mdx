<div id="about-mcp-servers">
  ## Über MCP-Server
</div>

Das Model Context Protocol (MCP) ist ein offenes Protokoll, das standardisierte Verbindungen zwischen KI-Anwendungen und externen Diensten wie Dokumentationen ermöglicht. Mintlify generiert aus Ihrer Dokumentation und Ihren OpenAPI-Spezifikationen einen MCP-Server und bereitet Ihre Inhalte auf das breitere KI-Ökosystem vor, in dem sich beliebige MCP-Clients (wie Claude, Cursor, Goose und andere) mit Ihrer Dokumentation und Ihren APIs verbinden können.

Ihr MCP-Server stellt Werkzeuge bereit, mit denen KI-Anwendungen Ihre Dokumentation durchsuchen und mit Ihren APIs interagieren können.

<div id="accessing-your-mcp-server">
  ## Zugriff auf Ihren MCP-Server
</div>

<Note>
  MCP-Server können nur für öffentliche Dokumentation erstellt werden. Dokumentation hinter einer Endbenutzer-Authentifizierung kann nicht zur Servererstellung herangezogen werden.
</Note>

Ihr MCP-Server wird automatisch erstellt und unter Ihrer Dokumentations-URL mit dem Pfad `/mcp` gehostet. Beispielsweise ist der MCP-Server von Mintlify unter `https://mintlify.com/docs/mcp` verfügbar.

Sie können die URL Ihres MCP-Servers in Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) anzeigen und kopieren.

<div id="configuring-your-mcp-server">
  ## Konfiguration Ihres MCP-Servers
</div>

Alle MCP-Server enthalten standardmäßig das Tool `search`, mit dem Nutzer Informationen aus Ihrer Dokumentation in anderen Tools abfragen können.

Wenn Sie einen [Pro- oder Enterprise-Plan](https://mintlify.com/pricing?ref=mcp) haben, können Sie Endpunkte aus Ihrer OpenAPI-Spezifikation als MCP-Tools bereitstellen.

Um Endpunkte als MCP-Tools bereitzustellen, verwenden Sie das Objekt `mcp` innerhalb der Erweiterung `x-mint` auf Datei- oder Endpunktebene. Der Mintlify-MCP-Server enthält beispielsweise Tools zum Erstellen von Assistenten-Chats, zum Abrufen von Statusupdates und zum Auslösen von Aktualisierungen.

MCP-Server verfolgen einen Security-first-Ansatz, bei dem API-Endpunkte nicht standardmäßig offengelegt werden. Sie müssen Endpunkte explizit aktivieren, um sie als MCP-Tools verfügbar zu machen. Legen Sie nur Endpunkte offen, die für den öffentlichen Zugriff über KI-Tools unbedenklich sind.

<ResponseField name="mcp" type="object">

  <Expandable title="MCP">

    </ResponseField>

    <ResponseField name="name" type="string">
      Der Name des MCP-Tools.
    </ResponseField>

    </ResponseField>

<div id="file-level-configuration">
  ### Konfiguration auf Dateiebene
</div>

Aktivieren Sie MCP standardmäßig für alle Endpunkte in einer OpenAPI-Spezifikationsdatei und schließen Sie anschließend gezielt Endpunkte aus:

```json
{
  "openapi": "3.1.0",
  "x-mint": {
    "mcp": {
      "enabled": true
    }
  },
  // ...
  "paths": {
    "/api/v1/users": {
      "get": {
        "x-mint": {
          "mcp": {
            "enabled": false // Deaktiviert MCP für diesen Endpunkt
          }
        },
        // ...
      }
    }
  }
}
```

<div id="endpoint-level-configuration">
  ### Konfiguration auf Endpunktebene
</div>

Aktivieren Sie MCP für bestimmte Endpunkte:

```json
{
  "paths": {
    "/api/v1/users": {
      "get": {
        "x-mint": {
          "mcp": {
            "enabled": true,
            "name": "get-users",
            "description": "Gibt eine Liste von Nutzern zurück"
          },
          // ...
        }
      }
    },
    "/api/v1/delete": {
      "delete": {
        // Kein `x-mint: mcp`, daher wird dieser Endpunkt nicht als MCP-Tool offengelegt
        // ...
      }
    }
  }
}
```

<div id="using-your-mcp-server">
  ## Verwendung Ihres MCP-Servers
</div>

Ihre Nutzer müssen Ihren MCP-Server mit ihren bevorzugten KI-Tools verbinden.

1. Machen Sie die URL Ihres MCP-Servers öffentlich zugänglich.
2. Nutzer kopieren die URL Ihres MCP-Servers und fügen sie ihren Tools hinzu.
3. Nutzer greifen über ihre Tools auf Ihre Dokumentation und API-Endpunkte zu.

So können Sie Ihre Nutzer dabei unterstützen, eine Verbindung zu Ihrem MCP-Server herzustellen:

<Tabs>
  <Tab title="Kontextmenü">
    Fügen Sie Optionen im [Kontextmenü](/de/ai/contextual-menu) hinzu, damit Ihre Nutzer von jeder Seite Ihrer Dokumentation aus eine Verbindung zu Ihrem MCP-Server herstellen können.

    | Option | Kennung | Beschreibung |
    | :----- | :------ | :----------- |
    | **MCP-Server-URL kopieren** | `mcp` | Kopiert die URL Ihres MCP-Servers in die Zwischenablage des Nutzers. |
    | **Mit Cursor verbinden** | `cursor` | Installiert Ihren MCP-Server in Cursor. |
    | **Mit VS Code verbinden** | `vscode` | Installiert Ihren MCP-Server in VS Code. |
  </Tab>

  <Tab title="Claude">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und suchen Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung für Ihre Nutzer, die Ihre MCP-Server-URL und die Schritte zur Verbindung mit Claude enthält.

        1. Navigieren Sie in den Claude-Einstellungen zur Seite [Connectors](https://claude.ai/settings/connectors).
        2. Wählen Sie **Add custom connector**.
        3. Fügen Sie den Namen und die URL Ihres MCP-Servers hinzu.
        4. Wählen Sie **Add**.
        5. Wählen Sie bei der Nutzung von Claude die Schaltfläche für Anhänge (das Plus-Symbol).
        6. Wählen Sie Ihren MCP-Server aus.
      </Step>
    </Steps>

    Weitere Details finden Sie in der [Model Context Protocol-Dokumentation](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server).
  </Tab>

  <Tab title="Claude Code">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und suchen Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung für Ihre Nutzer, die Ihre MCP-Server-URL und den Befehl zum Verbinden mit Claude Code enthält.

        ```bash
        claude mcp add --transport http <name> <url>
        ```
      </Step>
    </Steps>

    Weitere Details finden Sie in der [Claude Code-Dokumentation](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="Cursor">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und suchen Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung für Ihre Nutzer, die Ihre MCP-Server-URL und die Schritte zur Verbindung mit Cursor enthält.

        1. Verwenden Sie <kbd>Command</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd> (<kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd> unter Windows), um die Befehlspalette zu öffnen.
        2. Suchen Sie nach "Open MCP settings".
        3. Wählen Sie **Add custom MCP**. Dadurch wird die Datei `mcp.json` geöffnet.
        4. Konfigurieren Sie Ihren Server in `mcp.json`:

        ```json
        {
          "mcpServers": {
            "<your-mcp-server-name>": {
              "url": "<your-mcp-server-url>"
            }
          }
        }
        ```
      </Step>
    </Steps>

    Weitere Details finden Sie in der [Cursor-Dokumentation](https://docs.cursor.com/en/context/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="VS Code">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und suchen Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung für Ihre Nutzer, die Ihre MCP-Server-URL und die Schritte zur Verbindung mit VS Code enthält.

        1. Erstellen Sie die Datei `.vscode/mcp.json`.
        2. Konfigurieren Sie Ihren Server in `mcp.json`:

        ```json
        {
          "servers": {
            "<your-mcp-server-name>": {
              "type": "http",
              "url": "<your-mcp-server-url>"
            }
          }
        }
        ```
      </Step>
    </Steps>

    Weitere Details finden Sie in der [VS Code-Dokumentation](https://code.visualstudio.com/docs/copilot/chat/mcp-servers).
  </Tab>
</Tabs>

<div id="example-connecting-to-the-mintlify-mcp-server">
  ### Beispiel: Verbindung mit dem Mintlify MCP-Server
</div>

Stellen Sie eine Verbindung mit dem Mintlify MCP-Server her, um mit der Mintlify-API zu interagieren und unsere Dokumentation zu durchsuchen. So erhalten Sie genauere Antworten zur Verwendung von Mintlify in Ihrer lokalen Umgebung und sehen, wie Sie Ihren Nutzern helfen können, sich mit Ihrem MCP-Server zu verbinden.

<Tabs>
  <Tab title="Kontextmenü">
    Wählen Sie oben auf dieser Seite das Kontextmenü und anschließend **Mit Cursor verbinden** oder **Mit VS Code verbinden**, um den Mintlify MCP-Server mit der IDE Ihrer Wahl zu verbinden.
  </Tab>

  <Tab title="Claude">
    So verwenden Sie den Mintlify MCP-Server mit Claude:

    <Steps>
      <Step title="Mintlify MCP-Server zu Claude hinzufügen">
        1. Öffnen Sie in den Claude-Einstellungen die Seite [Connectors](https://claude.ai/settings/connectors).
        2. Wählen Sie **Add custom connector**.
        3. Fügen Sie den Mintlify MCP-Server hinzu:

        - Name: `Mintlify`
        - URL: `https://mintlify.com/docs/mcp`

        4. Wählen Sie **Add**.
      </Step>

      <Step title="Auf den MCP-Server im Chat zugreifen">
        1. Klicken Sie in Claude auf die Anhangsschaltfläche (das Plus-Symbol).
        2. Wählen Sie den Mintlify MCP-Server.
        3. Stellen Sie Claude eine Frage zu Mintlify.
      </Step>
    </Steps>

    Weitere Details finden Sie in der [Model Context Protocol-Dokumentation](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server).

    ```bash
    claude mcp add --transport http Mintlify https://mintlify.com/docs/mcp
    ```

    Testen Sie die Verbindung mit:

    ```bash
    claude mcp list
    ```

<div id="authentication">
  ## Authentifizierung
</div>

Wenn Sie einen API-Endpunkt für MCP aktivieren, berücksichtigt der Server die in Ihren OpenAPI-`securitySchemes` und `securityRequirement` definierten Authentifizierungsanforderungen. Etwaige Schlüssel werden direkt vom Tool verarbeitet und nicht von Mintlify gespeichert oder weiterverarbeitet.

Wenn ein Benutzer sein KI-Tool anweist, einen geschützten Endpunkt aufzurufen, fordert das Tool in diesem Moment die erforderlichen Authentifizierungsnachweise beim Benutzer an.

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

<AccordionGroup>
  <Accordion title="MCP-Server zeigt nur das Such-Tool">
    Wenn Ihr MCP-Server nur das Such-Tool bereitstellt, obwohl eine OpenAPI-Spezifikation vorhanden ist:

    1. Überprüfen Sie, ob Ihre OpenAPI-Spezifikation gültig und erreichbar ist.
    2. Stellen Sie sicher, dass Sie MCP für bestimmte Endpunkte ausdrücklich mit `x-mint.mcp.enabled: true` aktiviert haben.
    3. Prüfen Sie Ihre Bereitstellungs-Logs auf Fehler bei der OpenAPI-Verarbeitung.

    Wenn die OpenAPI-Verarbeitung fehlschlägt, läuft der Server zur Aufrechterhaltung der Funktionalität mit dem Such-Tool alleine weiter.
  </Accordion>

  <Accordion title="Probleme mit der Authentifizierung">
    Wenn Nutzende von Authentifizierungsproblemen berichten:

    1. Prüfen Sie, ob Ihre OpenAPI-Spezifikation geeignete Definitionen für `securitySchemes` enthält.
    2. Bestätigen Sie, dass aktivierte Endpunkte mit den angegebenen Authentifizierungsmethoden funktionieren.
  </Accordion>

  <Accordion title="Fehlende oder unklare Tool-Beschreibungen">
    Wenn KI-Tools Ihre API-Endpunkte nicht effektiv nutzen:

    1. Fügen Sie Ihren Endpunkten detaillierte `summary`- und `description`-Felder hinzu.
    2. Stellen Sie sicher, dass Parameternamen und Beschreibungen selbsterklärend sind.
    3. Verwenden Sie das MCP-Dashboard, um zu überprüfen, wie Ihre Endpunkte als Tools erscheinen.
  </Accordion>
</AccordionGroup>
