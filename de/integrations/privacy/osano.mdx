Fügen Sie Folgendes zu Ihrer `docs.json`-Datei hinzu, um den Cookie-Einwilligungsmanager [Osano](https://www.osano.com/) zu integrieren.

<CodeGroup>

```json Integrationsoptionen in docs.json
"integrations": {
    "osano": "SOURCE"
}
```

```json Beispiel
"integrations": {
    "osano": "https://cmp.osano.com/2sUB2dqwqdkks/8dqwd-dwd86£-4a9b/osano.js"
}
```

</CodeGroup>

Der Wert für `SOURCE` entspricht dem `src`-Attribut im von Osano generierten Code-Snippet. Er beginnt immer mit `https://cmp.osano.com/`.

```html Code-Snippet von Osano
<script src="https://cmp.osano.com/placeholder/placeholder/osano.js"/>
```

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

<Accordion
  title="Seiten laden nicht im Strict-Compliance-Modus"
>
  Wenn Ihre Dokumentationsseiten beim Verwenden von Osanos **Strict**-Compliance-Modus nicht korrekt laden, müssen Sie die Domain von Mintlify auf die Allowlist setzen, damit Bilder und andere Assets geladen werden.

  <Steps>
    <Step title="Zu Managed Rules navigieren">
      Gehen Sie in Ihrem Osano-Dashboard zu **Scripts** → **Managed Rules**.
    </Step>

    <Step title="Mintlify-Domain hinzufügen">
      Fügen Sie `.mintlify.app/` als Managed Rule hinzu.

      <Frame>
        <img src="/images/integrations/osano-managed-rule.png" alt="Osano Managed Rule" />
      </Frame>

      <Info>
      Dadurch wird sichergestellt, dass alle von Mintlify bereitgestellten Assets (einschließlich Bilder, Stylesheets und anderer Dokumentationsressourcen) als essenziell behandelt werden und auch dann laden, wenn Osano unkategorisierte Inhalte von Drittanbietern blockiert.
      </Info>
    </Step>
  </Steps>
</Accordion>
