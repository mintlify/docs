---
title: "Single Sign-on (SSO)"
description: "Passen Sie an, wie sich Ihr Team beim Admin-Dashboard anmelden kann"
---

<Info>
  Die SSO-Funktion ist im [Enterprise-Tarif](https://mintlify.com/pricing?ref=sso) verfügbar.
</Info>

Nutzen Sie Single Sign-on für Ihr Dashboard via SAML und OIDC. Wenn Sie Okta, Google Workspace oder Microsoft Entra verwenden, finden Sie bei uns anbieter­spezifische Anleitungen zur Einrichtung von SSO. Wenn Sie einen anderen Anbieter nutzen, [kontaktieren Sie uns](mailto:support@mintlify.com) bitte.

<div id="okta">
  ## Okta
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="Create an application">
                Unter `Applications` klicken Sie, um eine neue App-Integration mit SAML 2.0 zu erstellen.
            </Step>
            <Step title="Configure integration">
                Geben Sie Folgendes ein:
                * Single-Sign-On-URL (bereitgestellt von Mintlify)
                * Audience-URI (bereitgestellt von Mintlify)
                * NameID-Format: `EmailAddress`
                * Attribute Statements:
                    | Name | Namensformat | Wert
                    | ---- | ------------ | -----
                    | `firstName` | Basic | `user.firstName` |
                    | `lastName` | Basic | `user.lastName` |
            </Step>
            <Step title="Send us your IdP information">
                Sobald die Anwendung eingerichtet ist, wechseln Sie zum Tab „Sign-On“ und senden Sie uns die Metadata-URL.
                Wir aktivieren die Verbindung auf unserer Seite anhand dieser Informationen.
            </Step>
        </Steps>
    </Tab>
    <Tab title="OIDC">
        <Steps>
            <Step title="Create an application">
                Unter `Applications` klicken Sie, um eine neue App-Integration mit OIDC zu erstellen.
                Wählen Sie den Anwendungstyp `Web Application`.
            </Step>
            <Step title="Configure integration">
                Wählen Sie den Authorization-Code-Grant-Typ und geben Sie die von Mintlify bereitgestellte Redirect-URI ein.
            </Step>
            <Step title="Send us your IdP information">
                Sobald die Anwendung eingerichtet ist, wechseln Sie zum Tab „General“ und finden Sie die Client ID und das Client Secret.
                Bitte übermitteln Sie uns diese sicher, zusammen mit der URL Ihrer Okta-Instanz (z. B. `<your-tenant-name>.okta.com`). Sie können diese über einen Dienst wie 1Password oder SendSafely senden.
            </Step>
        </Steps>
    </Tab>
</Tabs>

<div id="google-workspace">
  ## Google Workspace
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="Anwendung erstellen">
                Unter `Web and mobile apps` wählen Sie im Dropdown `Add app` die Option `Add custom SAML app`.  
                <Frame>
                    ![](/images/gsuite-add-custom-saml-app.png)
                </Frame>              
            </Step>
            <Step title="Senden Sie uns Ihre IdP-Informationen">
                Kopieren Sie die bereitgestellte SSO-URL, Entity ID und das x509-Zertifikat und senden Sie diese an das Mintlify-Team.
                <Frame>
                    ![](/images/gsuite-saml-metadata.png)                    
                </Frame>
            </Step>
            <Step title="Integration konfigurieren">
                Geben Sie auf der Seite „Service provider details“ Folgendes ein:
                * ACS URL (bereitgestellt von Mintlify)
                * Entity ID (bereitgestellt von Mintlify)
                * Name ID format: `EMAIL`
                * Name ID: `Basic Information > Primary email`

                <Frame>
                    ![](/images/gsuite-sp-details.png)
                </Frame>

                Geben Sie auf der nächsten Seite die folgenden Attributzuweisungen ein:
                | Google Directory Attribute | App-Attribut |
                | -------------------------- | ------------ |
                | `First name` | `firstName`  |
                | `Last name` | `lastName` |

                Sobald dieser Schritt abgeschlossen ist und Nutzende der Anwendung zugewiesen wurden, informieren Sie unser Team und wir aktivieren SSO für Ihr Konto!
            </Step>
        </Steps>
    </Tab>
</Tabs>

<div id="microsoft-entra">
  ## Microsoft Entra
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="Anwendung erstellen">
                1. Wählen Sie unter „Enterprise applications“ **New application**.
                2. Wählen Sie **Create your own application** und anschließend „Integrate any other application you don't find in the gallery (Non-gallery)“.
            </Step>
            <Step title="SAML konfigurieren">
                Navigieren Sie zur Single-Sign-On-Einrichtungsseite und wählen Sie **SAML**. Geben Sie unter „Basic SAML Configuration“ Folgendes ein:
                * Identifier (Entity ID): Die von Mintlify bereitgestellte Audience-URI.
                * Reply URL (Assertion Consumer Service URL): Die von Mintlify bereitgestellte ACS-URL.
                
                Lassen Sie die übrigen Felder leer und wählen Sie **Save**.
            </Step>
            <Step title="Attribute & Claims konfigurieren">
                Bearbeiten Sie den Abschnitt „Attributes & Claims“:
                1. Wählen Sie **Unique User Identifier (Name ID)** unter „Required Claim“.
                2. Ändern Sie das Quellattribut auf `user.primaryauthoritativeemail`.
                3. Legen Sie unter „Additional claims“ die folgenden Claims an:
                    | Name | Value |
                    | ---- | ----- |
                    | `firstName` | `user.givenname` |
                    | `lastName` | `user.surname` |
            </Step>
            <Step title="IdP-Informationen an Mintlify senden">
                Sobald die Anwendung eingerichtet ist, wechseln Sie zum Abschnitt „SAML Certificates“ und senden Sie uns die App Federation Metadata URL.
                Wir aktivieren die Verbindung auf unserer Seite anhand dieser Informationen.
            </Step>
            <Step title="Benutzer zuweisen">
                Navigieren Sie in Ihrer Entra-Anwendung zu „Users and groups“ und fügen Sie die Benutzer hinzu, die Zugriff auf Ihr Dashboard erhalten sollen.
            </Step>
        </Steps>
    </Tab>
</Tabs>
