Die Eigenschaft [navigation](de/settings#param-navigation) in `docs.json` steuert die Struktur und Informationshierarchie Ihrer Dokumentation.

Mit einer passenden Navigationskonfiguration können Sie Ihre Inhalte so organisieren, dass Nutzer genau das finden, was sie suchen.

<div id="pages">
  ## Seiten
</div>

Seiten sind die grundlegendste Navigationskomponente. Seiten entsprechen den MDX-Dateien, aus denen Ihre Dokumentation besteht.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/pages-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/pages-dark.png" alt="" />

Im Objekt `navigation` ist `pages` ein Array, in dem jedes Element auf den Pfad zu einer [Seitendatei](de/pages) verweisen muss.

```json
{
  "navigation": {
    "pages": [
      "settings",
      "pages",
      "navigation",
      "themes",
      "custom-domain"
    ]
  }
}
```

<div id="groups">
  ## Gruppen
</div>

Verwenden Sie Gruppen, um die Navigation in Ihrer Seitenleiste in Abschnitte zu gliedern. Gruppen können ineinander verschachtelt, mit Tags versehen und mit Icons dargestellt werden.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/groups-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/groups-dark.png" alt="" />

Im Objekt `navigation` ist `groups` ein Array, in dem jeder Eintrag ein Objekt mit den Pflichtfeldern `group` und `pages` ist. Die Felder `icon` und `tag` sind optional.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Getting started",
        "icon": "play",
        "pages": [
          "quickstart",
          {
            "group": "Editing",
            "icon": "pencil",
            "pages": [
              "installation",
              "editor"
            ]
          }
        ]
      },
      {
        "group": "Writing content",
        "icon": "notebook-text",
        "tag": "NEW",
        "pages": [
          "writing-content/page",
          "writing-content/text"
        ]
      }
    ]
  }
}
```

<div id="tabs">
  ## Tabs
</div>

Tabs erstellen eigenständige Abschnitte Ihrer Dokumentation mit eigenen URL-Pfaden. Sie erzeugen eine horizontale Navigationsleiste am oberen Rand der Dokumentation, über die Nutzer zwischen Abschnitten wechseln können.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/tabs-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/tabs-dark.png" alt="" />

Im Objekt `navigation` ist `tabs` ein Array, in dem jeder Eintrag ein Objekt ist, das ein `tab`-Feld erfordert und weitere Navigationsfelder wie groups, pages, icons oder Links zu externen Seiten enthalten kann.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "tab": "SDKs",
        "icon": "code",
        "pages": [
          "sdk/fetch",
          "sdk/create",
          "sdk/delete"
        ]
      },
      {
        "tab": "Blog",
        "icon": "newspaper",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="menus">
  ### Menüs
</div>

Menüs fügen einem Tab Dropdown-Navigationselemente hinzu. Verwenden Sie Menüs, damit Nutzer direkt zu bestimmten Seiten innerhalb eines Tabs springen können.

Im Objekt `navigation` ist `menu` ein Array, in dem jeder Eintrag ein Objekt ist, das ein `item`-Feld erfordert und weitere Navigationsfelder wie groups, pages, icons oder Links zu externen Seiten enthalten kann.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Developer tools",
        "icon": "square-terminal",
        "menu": [
          {
            "item": "API reference",
            "icon": "rocket",
            "groups": [
              {
                "group": "Core endpoints",
                "icon": "square-terminal",
                "pages": [
                  "api-reference/get",
                  "api-reference/post",
                  "api-reference/delete"
                ]
              }
            ]
          },
          {
            "item": "SDKs",
            "icon": "code",
            "description": "SDKs werden verwendet, um mit der API zu interagieren.",
            "pages": [
              "sdk/fetch",
              "sdk/create",
              "sdk/delete"
            ]
          }
        ]
      }
    ]
  }
}
```

<div id="anchors">
  ## Anker
</div>

Anker fügen am oberen Rand deiner Seitenleiste permanente Navigationselemente hinzu. Verwende Anker, um deine Inhalte zu gliedern, schnellen Zugriff auf externe Ressourcen bereitzustellen oder auffällige Call-to-Actions zu platzieren.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/anchors-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/anchors-dark.png" />

Im Objekt `navigation` ist `anchors` ein Array, in dem jeder Eintrag ein Objekt mit einem erforderlichen Feld `anchor` ist und weitere Navigationsfelder wie Gruppen, Seiten, Icons oder Links zu externen Seiten enthalten kann.

```json
{
  "navigation": {
    "anchors": [
      {
        "anchor": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "anchor": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "anchor": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

Für Anker, die ausschließlich auf externe Links verweisen, verwende das Schlüsselwort `global`. Anker in einem `global`-Objekt müssen ein `href`-Feld haben und dürfen nicht auf einen relativen Pfad verweisen.

Globale Anker sind besonders nützlich, um auf Ressourcen zu verlinken, die nicht Teil deiner Dokumentation sind, aber für deine Nutzer leicht zugänglich sein sollten, etwa ein Blog oder ein Support-Portal.

```json
{
  "navigation": {
    "global":  {
      "anchors": [
        {
          "anchor": "Community",
          "icon": "house",
          "href": "https://slack.com"
        },
        {
          "anchor": "Blog",
          "icon": "pencil",
          "href": "https://mintlify.com/blog"
        }
      ]
    },
    "tabs": /*...*/
  }
}
```

<div id="dropdowns">
  ## Dropdowns
</div>

Dropdowns befinden sich in einem ausklappbaren Menü oben in der Sidebar-Navigation. Jeder Eintrag in einem Dropdown verweist auf einen Abschnitt der Dokumentation.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/dropdowns-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/dropdowns-dark.png" />

Im Objekt `navigation` ist `dropdowns` ein Array, in dem jeder Eintrag ein Objekt ist, das ein `dropdown`-Feld erfordert und weitere Navigationsfelder wie Gruppen, Seiten, Icons oder Links zu externen Seiten enthalten kann.

```json
{
  "navigation": {
    "dropdowns": [
      {
        "dropdown": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "dropdown": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "dropdown": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="openapi">
  ## OpenAPI
</div>

Integrieren Sie OpenAPI-Spezifikationen direkt in Ihre Navigationsstruktur, um die API-Dokumentation automatisch zu generieren. Erstellen Sie dedizierte API-Bereiche oder platzieren Sie Endpunktseiten innerhalb anderer Navigationselemente.

Legen Sie auf beliebiger Ebene Ihrer Navigationshierarchie eine Standard-OpenAPI-Spezifikation fest. Untergeordnete Elemente übernehmen diese Spezifikation, sofern sie nicht eine eigene definieren.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "API reference",
        "openapi": "/path/to/openapi-v1.json",
        "pages": [
          "overview",
          "authentication",
          "GET /users",
          "POST /users",
          {
            "group": "Products",
            "openapi": "/path/to/openapi-v2.json",
            "pages": [
              "GET /products",
              "POST /products"
            ]
          }
        ]
      }
    ]
  }
}
```

Weitere Informationen zum Verweisen auf OpenAPI-Endpunkte in Ihrer Dokumentation finden Sie unter [OpenAPI setup](/de/api-playground/openapi-setup).

<div id="versions">
  ## Versionen
</div>

Unterteilen Sie Ihre Navigation in verschiedene Versionen. Versionen können über ein Dropdown-Menü ausgewählt werden.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/versions-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/versions-dark.png" />

Im Objekt `navigation` ist `versions` ein Array, in dem jeder Eintrag ein Objekt ist, das ein `version`-Feld erfordert und beliebige weitere Navigationsfelder enthalten kann.

```json
{
  "navigation": {
    "versions": [
      {
        "version": "1.0.0",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["v1/overview", "v1/quickstart", "v1/development"]
          }
        ]
      },
      {
        "version": "2.0.0",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["v2/overview", "v2/quickstart", "v2/development"]
          }
        ]
      }
    ]
  }
}
```

<div id="interaction-configuration">
  ## Interaktionskonfiguration
</div>

Steuern Sie das Benutzerverhalten bei Navigationselementen über die Eigenschaft `interaction` in Ihrer `docs.json`.

<div id="auto-navigation-on-expand">
  ### Automatische Navigation beim Erweitern
</div>

Verschiedene Themes haben ein unterschiedliches Standardverhalten beim Klicken auf Navigationsgruppen. Sie können dieses Verhalten mit der Option `drilldown` überschreiben:

```json
{
  "interaction": {
    "drilldown": true  // Erzwingt die Navigation zur ersten Seite, wenn ein Benutzer ein Dropdown erweitert
  }
}
```

```json
{
  "interaction": {
    "drilldown": false // Nie navigieren, nur ein- oder ausklappen
  }
}
```

- Auf `true` setzen, um beim Klicken auf Navigationsgruppen automatisch zur ersten Seite zu navigieren
- Auf `false` setzen, um nicht zu navigieren und die Gruppe nur ein- oder auszuklappen
- Nicht setzen, um das Standardverhalten des Themes zu verwenden

<div id="faq">
  ## FAQ
</div>

<Accordion title="Welche Navigationsordner sollte ich vermeiden?">
  Verwenden Sie `/api` oder `/mcp` nicht als Root-Pfade (Titel oder Ordner). Die Pfade `/api` und `/mcp` sind in der Produktion reserviert und führen dazu, dass Seiten einen 404-Fehler zurückgeben, wenn ihre URLs `/api` oder `/mcp` enthalten.
</Accordion>
