Diese Schnellstartanleitung zeigt Ihnen, wie Sie Ihre Dokumentations-Website in wenigen Minuten einrichten und veröffentlichen.

Nachdem Sie diese Anleitung abgeschlossen haben, haben Sie eine live geschaltete Dokumentations-Website, die Sie anpassen und erweitern können.

<Info>
  **Voraussetzungen**: Bevor Sie beginnen, [erstellen Sie ein Konto](https://mintlify.com/start) und schließen Sie die Einführung ab.
</Info>

<div id="getting-started">
  ## Erste Schritte
</div>

Nachdem Sie den Onboarding-Prozess abgeschlossen haben, wird Ihre Dokumentationsseite automatisch unter einer eindeutigen URL mit folgendem Format bereitgestellt:

```
https://<your-project-name>.mintlify.app
```

Sie finden Ihre URL auf der Übersichtsseite Ihres [Dashboards](https://dashboard.mintlify.com/).

<Frame>
  <img src="/images/quickstart/mintlify-domain-light.png" alt="Mintlify-Domain" className="block dark:hidden" />

  <img src="/images/quickstart/mintlify-domain-dark.png" alt="Mintlify-Domain" className="hidden dark:block" />
</Frame>

Die URL Ihrer Seite ist sofort verfügbar. Verwenden Sie diese URL zum Testen und zum Teilen mit Ihrem Team, während Sie Ihre Docs-Seite einrichten.

<div id="install-the-github-app">
  ### GitHub-App installieren
</div>

Mintlify stellt eine GitHub-App bereit, die die Bereitstellung automatisiert, wenn Sie Änderungen in Ihr Repository pushen.

Installieren Sie die GitHub-App, indem Sie den Anweisungen in der Onboarding-Checkliste oder in Ihrem Dashboard folgen.

1. Navigieren Sie zu **Settings** in Ihrem Mintlify-Dashboard.
2. Wählen Sie **GitHub App** in der Seitenleiste.
3. Wählen Sie **Install GitHub App**. Dadurch wird ein neuer Tab mit der Installationsseite der GitHub-App geöffnet.
4. Wählen Sie die Organisation oder das Benutzerkonto aus, in dem Sie die App installieren möchten.
5. Wählen Sie die Repositories aus, die Sie verbinden möchten.

<Frame>
  <img src="/images/quickstart/github-app-installation-light.png" alt="Installation der GitHub-App" className="block dark:hidden" />

  <img src="/images/quickstart/github-app-installation-dark.png" alt="Installation der GitHub-App" className="hidden dark:block" />
</Frame>

<Info>
  Aktualisieren Sie die Berechtigungen der GitHub-App, wenn Sie Ihre Dokumentation in ein anderes Repository verschieben.
</Info>

<div id="authorize-your-github-account">
  ### Ihr GitHub-Konto autorisieren
</div>

1. Navigieren Sie zu **Settings** in Ihrem Mintlify-Dashboard.
2. Wählen Sie **My Profile** in der Seitenleiste.
3. Wählen Sie **Authorize GitHub account**. Dadurch wird ein neuer Tab mit der GitHub-Autorisierungsseite geöffnet.

<Info>
  Ein Admin Ihrer GitHub-Organisation muss Ihr Konto möglicherweise autorisieren – abhängig von den Einstellungen Ihrer Organisation.
</Info>

<div id="code-based-workflow">
  ## Codebasierter Workflow
</div>

Der codebasierte Workflow integriert sich in Ihre bestehende Entwicklungsumgebung und Git-Repositories. Dieser Workflow eignet sich besonders für technische Teams, die Dokumentation zusammen mit Code verwalten möchten.

<div id="install-the-cli">
  ### CLI installieren
</div>

Um lokal mit Ihrer Dokumentation zu arbeiten, installieren Sie die Command Line Interface (CLI) namens [mint](https://www.npmjs.com/package/mint), indem Sie diesen Befehl in Ihrem Terminal ausführen:

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

<Info>
  Sie benötigen Node.js auf Ihrem Rechner. Wenn bei der Installation Probleme auftreten, sehen Sie sich die Anleitung zur Fehlerbehebung an.
</Info>

<div id="edit-the-documentation">
  ### Dokumentation bearbeiten
</div>

Nachdem Sie Ihre Umgebung eingerichtet haben, können Sie mit der Bearbeitung Ihrer Dokumentationsdateien beginnen. Aktualisieren Sie beispielsweise den Titel der Einführungsseite:

1. Öffnen Sie das während des Onboardings erstellte Repository.
2. Öffnen Sie `index.mdx` und suchen Sie den Anfang der Datei:

```mdx index.mdx
---
title: "Introduction"
description: "This is the introduction to the documentation"
---
```

3. Aktualisieren Sie das Feld `title` auf `"Hello World"`.

```mdx index.mdx {2}
---
title: "Hello World"
description: "This is the introduction to the documentation"
---
```

<div id="preview-the-changes">
  ### Änderungen in der Vorschau ansehen
</div>

Um die Änderungen lokal anzuzeigen, führen Sie den folgenden Befehl aus:

```bash
mint dev
```

Ihre Vorschau ist unter `localhost:3000` verfügbar.

<div id="push-the-changes">
  ### Änderungen pushen
</div>

Wenn Sie bereit sind, Ihre Änderungen zu veröffentlichen, pushen Sie sie in Ihr Repository.

Mintlify erkennt die Änderungen automatisch, baut Ihre Dokumentation und deployt die Updates auf Ihrer Site. Überwachen Sie den Deployment-Status in der Commit-Historie Ihres GitHub-Repositories oder im [Dashboard](https://dashboard.mintlify.com).

Nach Abschluss des Deployments ist Ihr neuestes Update unter `<your-project-name>.mintlify.app` verfügbar.

<div id="web-editor-workflow">
  ## Webeditor-Workflow
</div>

Der Webeditor-Workflow bietet eine What-you-see-is-what-you-get-(WYSIWYG)-Oberfläche zum Erstellen und Bearbeiten von Dokumentationen. Dieser Workflow eignet sich am besten für Personen, die in ihrem Webbrowser ohne zusätzliche lokale Entwicklungstools arbeiten möchten.

<div id="access-the-web-editor">
  ### Auf den Webeditor zugreifen
</div>

1. Melden Sie sich in Ihrem [Dashboard](https://dashboard.mintlify.com) an.
2. Wählen Sie **Editor** in der linken Seitenleiste.

<Info>
  Wenn Sie die GitHub-App noch nicht installiert haben, werden Sie beim Öffnen des Webeditors dazu aufgefordert.
</Info>

  <img alt="Der Mintlify-Webeditor im visuellen Editor-Modus" src="/images/quickstart/web-editor-light.png" className="block dark:hidden" />

  <img alt="Der Mintlify-Webeditor im visuellen Editor-Modus" src="/images/quickstart/web-editor-dark.png" className="hidden dark:block" />
</Frame>

<div id="edit-the-documentation">
  ### Dokumentation bearbeiten
</div>

Im Webeditor können Sie in der Seitenleiste durch Ihre Dokumentationsdateien navigieren. Aktualisieren wir die Einführungsseite:

Suchen und wählen Sie `index.mdx` im Dateiexplorer aus.

Aktualisieren Sie anschließend im Editor das Titelfeld auf „Hello World“.

<Frame>
  <img alt="Bearbeiten im Webeditor" src="/images/quickstart/web-editor-editing-light.png" className="block dark:hidden" />

  <img alt="Bearbeiten im Webeditor" src="/images/quickstart/web-editor-editing-dark.png" className="hidden dark:block" />

<Tip>
  Der Editor bietet eine umfangreiche Auswahl an Formatierungstools und Komponenten. Geben Sie <kbd>/</kbd> im Editor ein, um das Befehlsmenü zu öffnen und auf diese Tools zuzugreifen.
</Tip>

<div id="publish-your-changes">
  ### Änderungen veröffentlichen
</div>

Wenn Sie mit Ihren Änderungen zufrieden sind, wählen Sie oben rechts die Schaltfläche **Publish**. Ihre Änderungen werden sofort auf Ihrer Dokumentationsseite bereitgestellt.

<Tip>
  Verwenden Sie Branches, um Änderungen über Pull Requests vor der Bereitstellung auf Ihrer Live-Website zu prüfen und zu begutachten.
</Tip>

Weitere Details zur Nutzung des Webeditors, einschließlich der Verwendung von Branches und Pull Requests zur Zusammenarbeit und Vorschau von Änderungen, finden Sie in unserer [Webeditor-Dokumentation](/de/editor).

<div id="adding-a-custom-domain">
  ## Hinzufügen einer benutzerdefinierten Domain
</div>

Während Ihre Subdomain `<your-project-name>.mintlify.app` sich gut für Tests und Entwicklung eignet, bevorzugen die meisten Teams für Produktionsdokumentation eine benutzerdefinierte Domain.

Um eine benutzerdefinierte Domain hinzuzufügen, navigieren Sie in Ihrem Dashboard zur Seite [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain).

<Frame>
  <img src="/images/quickstart/custom-domain-light.png" alt="Benutzerdefinierte Domain" className="block dark:hidden" />

  <img src="/images/quickstart/custom-domain-dark.png" alt="Benutzerdefinierte Domain" className="hidden dark:block" />
</Frame>

Geben Sie Ihre Domain ein (zum Beispiel `docs.yourcompany.com`) und folgen Sie den Anweisungen, um die DNS-Einstellungen bei Ihrem Domain-Provider zu konfigurieren.

<Table>
  | Record-Typ | Name | Wert | TTL |
  |-------------|------|-------|-----|
  | CNAME | docs (oder Subdomain) | cname.mintlify.app | 3600 |
</Table>

<Info>
  DNS-Änderungen können bis zu 48 Stunden benötigen, um zu propagieren, auch wenn sie häufig deutlich schneller wirksam werden.
</Info>

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

Wenn während der Einrichtung Probleme auftreten, prüfen Sie diese häufigen Lösungen:

<AccordionGroup>
  <Accordion title="Lokale Vorschau funktioniert nicht">
    Stellen Sie sicher, dass Node.js v19+ installiert ist und dass Sie den Befehl `mint dev` aus dem Verzeichnis ausführen, das Ihre `docs.json` enthält.
  </Accordion>

  <Accordion title="Änderungen werden auf der Live‑Site nicht angezeigt">
    Die Bereitstellung kann mehrere Minuten dauern. Prüfen Sie Ihre GitHub Actions (bei Code-Workflows) oder die Bereitstellungs-Logs im Mintlify-Dashboard, um sicherzustellen, dass keine Build-Fehler auftreten.
  </Accordion>

  <Accordion title="Benutzerdefinierte Domain wird nicht verbunden">
    Überprüfen Sie, ob Ihre DNS-Einträge korrekt konfiguriert sind, und geben Sie der DNS-Propagation ausreichend Zeit (bis zu 48 Stunden). Sie können Tools wie [DNSChecker](https://dnschecker.org) verwenden, um Ihren CNAME-Eintrag zu prüfen.
  </Accordion>
</AccordionGroup>
