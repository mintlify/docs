Optimieren Sie Ihre Dokumentation sowohl für klassische Suchmaschinen als auch für KI-gestützte Antwort-Engines wie ChatGPT, Perplexity und Google AI Overviews.

Generative Engine Optimization (GEO) zielt darauf ab, durch umfassende Inhalte und klar strukturierte Informationen von KI-Systemen zitiert zu werden, während sich klassische SEO auf die Platzierung in Suchergebnissen konzentriert.

<div id="geo-quickstart">
  ## GEO-Schnellstart
</div>

<div id="initial-setup">
  ### Ersteinrichtung
</div>

1. **Stellen Sie sicher, dass Ihre Docs indexiert werden** in den `docs.json`-Einstellungen
2. **Prüfen Sie bestehende Seiten** auf fehlende Beschreibungen und Titel

<div id="content-improvements">
  ### Inhaltsverbesserungen
</div>

1. **Fügen Sie Vergleichstabellen hinzu** auf den passenden Seiten
2. **Überarbeiten Sie Überschriften**, damit sie häufige Fragen beantworten
3. **Verbessern Sie die interne Verlinkung** zwischen verwandten Themen
4. **Testen Sie mit KI-Tools**, um die Genauigkeit zu überprüfen

<div id="geo-best-practices">
  ## GEO Best Practices
</div>

Im Allgemeinen haben gut geschriebene und gut strukturierte Dokumentationen eine starke GEO. Du solltest weiterhin für deine Nutzer schreiben und Prioritäten setzen. Wenn deine Inhalte ihre Bedürfnisse erfüllen, bist du auf einem guten Weg, für KI-Tools zu optimieren. Echte, hilfreiche Inhalte statt „Optimierung um der Optimierung willen“ werden sowohl von klassischen als auch von KI-Suchmaschinen belohnt.

Fokussiere dich auf:

- Inhalte, die auf die Bedürfnisse der Nutzer ausgerichtet sind statt auf Keyword-Matching
- Strukturierte, leicht scannbare Informationen
- Direkte Antworten auf Fragen

<div id="format-for-clarity">
  ### Format für Klarheit
</div>

Diese Formatierungspraktiken helfen KI-Tools, deine Inhalte zu analysieren und zu verstehen:

- Keine Überschriftenebenen überspringen (H1 → H2 → H3)
- Konkrete Objektnamen statt „it“ oder „this“ verwenden
- Codeblöcke mit ihrer Programmiersprache kennzeichnen
- Bildern aussagekräftigen Alternativtext geben
- Auf verwandte Konzepte verlinken, um KI beim Verständnis von Zusammenhängen zu unterstützen

<div id="answer-questions-directly">
  ### Fragen direkt beantworten
</div>

Schreibe Inhalte, die konkrete Nutzerfragen adressieren:

- Abschnitte mit der wichtigsten Erkenntnis beginnen
- Beschreibende Überschriften verwenden, die gängigen Suchanfragen entsprechen
- Komplexe Themen in nummerierte Schritte gliedern

<div id="mintlify-configuration">
  ## Mintlify-Konfiguration
</div>

Verwenden Sie diese Funktionen, um GEO zu verbessern.

<div id="add-descriptive-page-metadata">
  ### Beschreibende Seitenmetadaten hinzufügen
</div>

Fügen Sie klare Titel und Beschreibungen in Ihrem Frontmatter hinzu:

```mdx
---
title: "API authentication guide"
description: "Complete guide to implementing API authentication with code examples"
---
```

<div id="configure-global-indexing-settings">
  ### Globale Indexierungseinstellungen konfigurieren
</div>

Fügen Sie Folgendes zu Ihrer `docs.json` hinzu:

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "Your docs"
    }
  }
}
```

<div id="llmstxt">
  ### LLMs.txt
</div>

LLMs.txt-Dateien helfen KI-Systemen, die Struktur Ihrer Dokumentation zu verstehen – ähnlich wie Sitemaps Suchmaschinen helfen. Mintlify generiert LLMs.txt-Dateien für Ihre Dokumentation automatisch. Es ist keine Konfiguration erforderlich.

<div id="testing-your-documentation">
  ## Testen Ihrer Dokumentation
</div>

Testen Sie verschiedene KI-Tools mit Fragen zu Ihrem Produkt und Ihrer Dokumentation, um zu sehen, wie gut Ihre Docs zitiert werden.

**Stellen Sie KI-Assistenten gezielte Fragen zu Ihren Docs:**
- „Wie richte ich die Authentifizierung mit dieser API ein?“
- „Führen Sie mich Schritt für Schritt durch den Installationsprozess“

**Prüfen Sie, ob die Tools Folgendes liefern:**
- Korrekte Codebeispiele
- Präzise Schritt-für-Schritt-Anleitungen
