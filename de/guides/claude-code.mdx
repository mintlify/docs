Claude Code ist ein agentisches CLI-Tool, das bei der Pflege Ihrer Dokumentation hilft. Es kann neue Inhalte verfassen, bestehende Seiten überprüfen und die Docs aktuell halten.

Sie können Claude Code so trainieren, dass es Ihre Dokumentationsstandards und Workflows versteht, indem Sie Ihrem Projekt eine `CLAUDE.md`-Datei hinzufügen und diese im Laufe der Zeit verfeinern.

<div id="getting-started">
  ## Erste Schritte
</div>

**Voraussetzungen:**
- Aktives Claude-Abo (Pro, Max oder API-Zugang)

**Einrichtung:**
1. Installiere Claude Code:
  ```bash
  npm install -g @anthropic-ai/claude-code
  ```
2. Wechsle in dein Docs-Verzeichnis.
3. (Optional) Füge die untenstehende Datei `CLAUDE.md` zu deinem Projekt hinzu.
4. Starte mit `claude`.

<div id="claudemd-template">
  ## CLAUDE.md-Vorlage
</div>

Speichere eine Datei `CLAUDE.md` im Stammverzeichnis deines Docs-Ordners, damit Claude Code dein Projekt besser versteht. Diese Datei schult Claude Code in deinen Dokumentationsstandards, Präferenzen und Workflows. Siehe [Claude-Speicher verwalten](https://docs.anthropic.com/en/docs/claude-code/memory) in den Anthropic-Dokumenten für weitere Informationen.

Kopiere diese Beispielvorlage oder passe sie an die Spezifikationen deiner Docs an:

```mdx
# Mintlify-Dokumentation

## Zusammenarbeit
- Du darfst Ideen hinterfragen – das kann zu besserer Dokumentation führen. Zitiere Quellen und erkläre deine Begründung, wenn du das tust
- IMMER nachfragen, statt Annahmen zu treffen
- NIEMALS lügen, raten oder Informationen erfinden

## Projektkontext
- Format: MDX-Dateien mit YAML-Frontmatter
- Konfiguration: docs.json für Navigation, Theme und Einstellungen
- Komponenten: Mintlify-Komponenten

## Content-Strategie
- Genau so viel dokumentieren, wie für den Erfolg der Nutzer nötig ist – nicht zu viel, nicht zu wenig
- Genauigkeit und Nutzbarkeit der Informationen priorisieren
- Inhalte nach Möglichkeit langlebig gestalten
- Vor dem Hinzufügen neuer Inhalte nach bestehender Information suchen. Duplikate vermeiden, außer aus strategischen Gründen
- Bestehende Muster auf Konsistenz prüfen
- Mit den kleinstmöglichen sinnvollen Änderungen beginnen

## docs.json

- Beim Erstellen der Datei docs.json und der Seitennavigation auf das [docs.json-Schema](https://mintlify.com/docs.json) verweisen

## Frontmatter-Anforderungen für Seiten
- title: Klarer, beschreibender Seitentitel
- description: Prägnante Zusammenfassung für SEO/Navigation

## Schreibstandards
- Anrede in der zweiten Person („du“/„Sie“, bevorzugt „du“)
- Voraussetzungen zu Beginn von prozeduralen Inhalten
- Alle Codebeispiele vor der Veröffentlichung testen
- Stil und Formatierung bestehender Seiten übernehmen
- Sowohl einfache als auch fortgeschrittene Anwendungsfälle einbeziehen
- Sprach-Tags für alle Codeblöcke
- Alt-Text für alle Bilder
- Relative Pfade für interne Links

## Git-Workflow
- NIEMALS --no-verify beim Committen verwenden
- Vor Beginn klären, wie mit uncommitted Changes umzugehen ist
- Einen neuen Branch erstellen, wenn kein klarer Branch für Änderungen existiert
- Häufig während der Entwicklung committen
- NIEMALS Pre-Commit-Hooks überspringen oder deaktivieren

## Nicht tun
- Frontmatter in keiner MDX-Datei weglassen
- Keine absoluten URLs für interne Links verwenden
- Keine ungetesteten Codebeispiele einbinden
- Keine Annahmen treffen – immer um Klärung bitten
```

<div id="sample-prompts">
  ## Beispiel-Prompts
</div>

Sobald du Claude Code eingerichtet hast, probiere diese Prompts aus, um zu sehen, wie es bei typischen Dokumentationsaufgaben helfen kann. Du kannst diese Beispiele direkt kopieren und einfügen oder für deine konkreten Anforderungen anpassen.

<div id="convert-notes-to-polished-docs">
  ### Notizen in ausgearbeitete Docs umwandeln
</div>

Verwandle Rohentwürfe in korrekt formatierte Markdown-Seiten mit Komponenten und Frontmatter.

**Beispiel-Prompt:**
```text wrap
Convert this text into a properly formatted MDX page: [paste your text here]
```

<div id="review-docs-for-consistency">
  ### Docs auf Konsistenz prüfen
</div>

Erhalte Vorschläge zur Verbesserung von Stil, Formatierung und Nutzung der Komponenten.

**Beispiel-Prompt:**
```text wrap
Review the files in docs/ and suggest improvements for consistency and clarity
```

<div id="update-docs-when-features-change">
  ### Docs aktualisieren, wenn sich Features ändern
</div>

Halte die Dokumentation aktuell, wenn sich dein Produkt weiterentwickelt.

**Beispiel-Prompt:**
```text wrap
Our API now requires a version parameter. Update our docs to include version=2024-01 in all examples
```

<div id="generate-comprehensive-code-examples">
  ### Umfassende Codebeispiele generieren
</div>

Erstelle mehrsprachige Beispiele mit Fehlerbehandlung.

**Beispiel-Prompt:**
```text wrap
Create code examples for [your API endpoint] in JavaScript, Python, and cURL with error handling
```

<div id="extending-claude-code">
  ## Claude Code erweitern
</div>

Über das reine manuelle Prompten von Claude Code hinaus kannst du es in deine bestehenden Workflows integrieren.

<div id="automation-with-github-actions">
  ### Automatisierung mit GitHub Actions
</div>

Führe Claude Code automatisch aus, wenn sich Code ändert, um die Doku aktuell zu halten. Du kannst Dokumentations-Reviews bei Pull Requests auslösen oder Beispiele aktualisieren, wenn API-Änderungen erkannt werden.

<div id="multi-instance-workflows">
  ### Workflows mit mehreren Instanzen
</div>

Verwende separate Claude-Code-Sitzungen für unterschiedliche Aufgaben – eine zum Verfassen neuer Inhalte und eine weitere für Review und Qualitätssicherung. So stellst du Konsistenz sicher und findest Probleme, die einer einzelnen Sitzung entgehen könnten.

<div id="team-collaboration">
  ### Teamzusammenarbeit
</div>

Teile deine verfeinerte Datei `CLAUDE.md` mit deinem Team, um einheitliche Dokumentationsstandards über alle Beitragenden hinweg sicherzustellen. Teams entwickeln häufig projektspezifische Prompts und Workflows, die zum festen Bestandteil ihres Dokumentationsprozesses werden.

<div id="custom-commands">
  ### Benutzerdefinierte Befehle
</div>

Erstelle wiederverwendbare Slash-Befehle in `.claude/commands/` für häufig genutzte dokumentationsbezogene Aufgaben, die speziell auf dein Projekt oder Team zugeschnitten sind.
