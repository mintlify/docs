<Info>
  Vorschau-Deployments sind in den [Pro- und Enterprise-Plänen](https://mintlify.com/pricing?ref=preview-deployments) verfügbar.
</Info>

Vorschau-Deployments zeigen dir, wie sich Änderungen an deiner Dokumentation auswirken, bevor sie in die Produktion übernommen werden. Jede Vorschau erzeugt eine teilbare URL, die sich automatisch aktualisiert, sobald du neue Änderungen pushst.

<div id="creating-preview-deployments">
  ## Vorschau-Bereitstellungen erstellen
</div>

Vorschau-Bereitstellungen werden automatisch über Pull Requests oder manuell über dein Dashboard erstellt.

<div id="automatic-previews">
  ### Automatische Vorschauen
</div>

Wenn du einen Pull Request erstellst, fügt der Mintlify-Bot automatisch einen Link hinzu, um die Vorschau-Bereitstellung in deinem Pull Request aufzurufen. Die Vorschau wird jedes Mal aktualisiert, wenn du neue Commits in den Branch pushst.

<div id="manual-previews">
  ### Manuelle Vorschauen
</div>

Du kannst manuell eine Vorschau für jeden Branch erstellen.

1. Gehe zu deinem [Dashboard](https://dashboard.mintlify.com/).
2. Wähle **Previews**.
3. Wähle **Create preview**.
4. Gib deinen Branch-Namen ein.
5. Wähle **Create deployment**.

<div id="redeploying-a-preview">
  ## Vorschau neu bereitstellen
</div>

Stellen Sie eine Vorschau neu bereit, um Inhalte zu aktualisieren oder einen fehlgeschlagenen Deploy erneut zu versuchen.

1. Wählen Sie die Vorschau in Ihrem [Dashboard](https://dashboard.mintlify.com/) aus.
2. Wählen Sie **Redeploy**.

<div id="sharing-preview-deployments">
  ## Vorschau-Bereitstellungen freigeben
</div>

Vorschau-Bereitstellungen können öffentlich zugänglich sein oder auf authentifizierte Benutzer beschränkt werden.

**Öffentlicher Zugriff**: Vorschau-URLs sind standardmäßig öffentlich sichtbar. Geben Sie einen Vorschau-Link an alle weiter, die Ihre Änderungen prüfen sollen.

**Eingeschränkter Zugriff**: Aktivieren Sie die Authentifizierung für Vorschau-Bereitstellungen, um den Zugriff auf authentifizierte Mintlify-Benutzer zu beschränken. Weitere Informationen finden Sie unter [Authentication Setup](/de/authentication-personalization/authentication-setup).
