Wenn du den Pfad einer Datei in deinem docs-Ordner änderst, ändert sich auch der URL-Pfad zu dieser Seite. Das kann passieren, wenn du deine Dokumentation umstrukturierst oder den Titel der Seitenleiste änderst.

<div id="broken-links">
  ## Defekte Links
</div>

Finde defekte Links mit unserer CLI. [Installiere die CLI](/de/installation) und führe den folgenden Befehl aus:

```bash
mint broken-links
```

Die CLI erkennt alle relativen Links in deinen Docs, die nicht existieren.

<div id="redirects">
  ## Weiterleitungen
</div>

Richten Sie 301-Weiterleitungen ein, indem Sie das Feld `redirects` zu Ihrer `docs.json` hinzufügen.

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

Dies leitet `/source/path` dauerhaft nach `/destination/path` weiter, sodass keine bestehende SEO für die ursprüngliche Seite verloren geht.

Um einen Wildcard-Pfad zu matchen, verwenden Sie `*` nach einem Parameter. In diesem Beispiel matcht `/beta/:slug*` `/beta/introduction` und leitet nach `/v2/introduction` weiter.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
