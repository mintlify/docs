Mintlify übernimmt automatisch viele SEO-Best Practices, darunter:

- Generierung von Meta-Tags
- Erstellung von Sitemap und `robots.txt`
- Semantische HTML-Struktur
- Mobile Optimierung

Sie können die Meta-Tags Ihrer Website vollständig anpassen, indem Sie das Feld `metatags` zu Ihrer `docs.json` oder zum Frontmatter einer Seite hinzufügen.

<div id="global-meta-tags">
  ## Globale Meta-Tags
</div>

Sie können standardmäßige Meta-Tags festlegen, indem Sie das Feld `metadata` zu Ihrer `docs.json` hinzufügen.

```json
"seo": {
    "metatags": {
        "og:image": "link to your default meta tag image"
    }
}
```

<div id="page-specific-meta-tags">
  ## Seitenspezifische Meta-Tags
</div>

Fügen Sie Ihre Meta-Tags wie folgt in die Metadaten der Seite ein:

```mdx
---
title: 'Your Example Page Title'
'og:image': 'link to your meta tag image'
---
```

<Warning>
  Meta-Tags mit Doppelpunkten müssen in Anführungszeichen stehen, wenn Sie sie in die Seitenmetadaten einfügen.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps und robots.txt-Dateien
</div>

Mintlify erstellt automatisch eine `sitemap.xml`-Datei und eine `robots.txt`-Datei. Du kannst deine Sitemap anzeigen, indem du an die URL deiner Dokumentationsseite `/sitemap.xml` anhängst.

Standardmäßig werden nur Seiten berücksichtigt, die in deiner `docs.json` aufgeführt sind. Um versteckte Links einzuschließen, füge `seo.indexing` zu deiner `docs.json` hinzu:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Benutzerdefinierte Sitemaps und robots.txt-Dateien
</div>

Um eine eigene `sitemap.xml`- oder `robots.txt`-Datei hinzuzufügen, erstelle die jeweilige Datei im Wurzelverzeichnis deines Projekts. Eine benutzerdefinierte Datei überschreibt die automatisch generierte Datei gleichen Namens. Wenn du die benutzerdefinierte Datei löschst, wird wieder die Standarddatei verwendet.

<div id="disabling-indexing">
  ## Indizierung deaktivieren
</div>

Wenn du verhindern möchtest, dass eine Seite von Suchmaschinen indexiert wird, füge Folgendes in das [Frontmatter](de/pages) deiner Seite ein:

```
---
noindex: true
---
```

Du kannst `noindex` auch für alle Seiten in deiner Dokumentation festlegen, indem du das Feld `metadata.robots` in deiner `docs.json` auf `"noindex"` setzt:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## SEO-Best Practices
</div>

<AccordionGroup>
  <Accordion title="Aussagekräftige Titel und Beschreibungen verfassen">
    - Verwenden Sie klare, aussagekräftige Seitentitel (50–60 Zeichen)
    - Schreiben Sie ansprechende Beschreibungen (150–160 Zeichen)
    - Integrieren Sie relevante Keywords
    - Geben Sie jeder Seite einen eindeutigen Titel und eine eindeutige Beschreibung
  </Accordion>

  <Accordion title="Inhaltsstruktur optimieren">
    - Verwenden Sie eine saubere Überschriftenhierarchie (H1 → H2 → H3)
    - Schreiben Sie zuerst für Menschen, dann für Suchmaschinen
    - Platzieren Sie relevante Keywords in Überschriften und im Inhalt
    - Halten Sie URLs kurz, verständlich und hierarchisch organisiert
    - Strukturieren Sie lange Inhalte mit Zwischenüberschriften und Listen
  </Accordion>

  <Accordion title="Interne Verlinkung">
    - Verlinken Sie auf thematisch verwandte Seiten innerhalb Ihrer Dokumentation
    - Verwenden Sie aussagekräftigen Linktext statt „Hier klicken“
    - Bilden Sie Themen-Cluster, indem Sie verwandte Konzepte verknüpfen
    - Nutzen Sie die automatischen Querverweis-Funktionen
  </Accordion>

  <Accordion title="Bild-SEO">
    - Verwenden Sie aussagekräftige Dateinamen für Bilder
    - Fügen Sie stets Alt-Text für Barrierefreiheit und SEO hinzu
    - Optimieren Sie Bildgrößen für schnellere Ladezeiten
    - Setzen Sie relevante Bilder ein, die Ihren Inhalt unterstützen
  </Accordion>
</AccordionGroup>
