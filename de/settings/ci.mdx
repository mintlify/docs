<Info>
  [Pro- und Enterprise-Pläne](https://mintlify.com/pricing?ref=docs-ci) enthalten CI-Prüfungen für GitHub-Repositories.
</Info>

Verwenden Sie CI-Prüfungen, um Ihre Dokumentation zu linten, Fehler zu erkennen und Warnungen auszugeben, bevor Sie deployen. CI-Prüfungen sind so konfiguriert, dass sie bei Commits in Ihren konfigurierten Deployment-Branch oder bei Pull Requests gegen diesen Branch ausgeführt werden.

<div id="installation">
  ## Installation
</div>

Beginnen Sie, indem Sie den Schritten auf der Seite [GitHub](/de/settings/github) folgen.

Bei GitHub-Apps können Sie Berechtigungen auch nur für ein einzelnes Repository erteilen. Wir empfehlen das ausdrücklich, da wir nur Zugriff auf das Repository benötigen, in dem Ihre Dokumentation gehostet wird.

<div id="configuration">
  ## Konfiguration
</div>

Konfigurieren Sie die in der CI für eine Bereitstellung aktivierten Prüfungen, indem Sie in Ihrem Dashboard zur Seite [Add-ons](https://dashboard.mintlify.com/products/addons) navigieren. Aktivieren Sie die Prüfungen, die Sie ausführen möchten.

Beim Aktivieren der Prüfungen können Sie auswählen, ob sie auf dem Niveau `Warning` oder `Blocking` ausgeführt werden.

- Eine Prüfung auf dem Niveau `Warning` führt niemals zu einem Fehlstatus, selbst wenn Fehler oder Empfehlungen vorliegen.
- Eine Prüfung auf dem Niveau `Blocking` führt zu einem Fehlstatus, wenn sie nicht bestanden wird oder Änderungen vorgeschlagen werden.

<div id="available-ci-checks">
  ## Verfügbare CI-Checks
</div>

<div id="broken-links">
  ### Defekte Links
</div>

Ähnlich wie der [CLI-Link-Checker](/de/settings/broken-links#broken-links) auf deinem lokalen Rechner prüfen wir deine Doku automatisch auf defekte Links.

Um die Ergebnisse dieses Checks zu sehen, öffne auf GitHub die Check-Ergebnisseite für den entsprechenden Commit.

<div id="vale">
  ### Vale
</div>

[Vale](https://vale.sh/) ist ein Open-Source, regelbasierter Prosa-Linter, der verschiedene Dokumenttypen unterstützt, darunter Markdown und MDX.

Mintlify unterstützt das automatische Ausführen von Vale in einem CI-Check und zeigt die Ergebnisse als Check-Status an.

<div id="configuration">
  #### Konfiguration
</div>

Wenn du eine `.vale.ini`-Datei im Wurzelverzeichnis der Inhalte deiner Bereitstellung hast, verwenden wir diese Konfigurationsdatei automatisch. Wir verwenden außerdem automatisch alle Konfigurationsdateien in deinem angegebenen `stylesPath`.

Wenn du keine Vale-Konfigurationsdatei hast, wird die Standardkonfiguration automatisch geladen.

```mdx Default vale.ini configuration expandable
# Top level styles
StylesPath = /app/styles
MinAlertLevel = suggestion
IgnoredScopes = code, tt, img, url, a
SkippedScopes = script, style, pre, figure, code

# Vocabularies
Vocab = Mintlify

# This is required since Vale doesn't officially support MDX
[formats]
mdx = md

# MDX support
[*.mdx]
BasedOnStyles = Vale
Vale.Terms = NO # Enforces really harsh capitalization rules, keep off

# `import ...`, `export ...`
# `<Component ... />`
# `<Component>...</Component>`
# `{ ... }`
TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

# Exclude:
# `<Component \n ... />`
BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})

CommentDelimiters = {/*, */}
```

<div id="packages">
  #### Pakete
</div>

Vale unterstützt eine Reihe von [Paketen](https://vale.sh/docs/keys/packages), die zur Prüfung von Rechtschreibung und Stil verwendet werden können.
Alle Pakete, die Sie in Ihrem Repository unter dem korrekten `stylesPath` einbinden, werden automatisch installiert und in Ihrer Vale-Konfiguration verwendet.

Für Pakete, die nicht in Ihrem Repository enthalten sind, können Sie beliebige Pakete aus dem [Vale-Paketregister](https://vale.sh/explorer) angeben; diese werden automatisch heruntergeladen und in Ihrer Vale-Konfiguration verwendet.

<Warning>
  Bitte beachten Sie, dass wir aus Sicherheitsgründen das automatische Herunterladen von Paketen, die nicht aus dem [Vale-Paketregister](https://vale.sh/explorer) stammen, nicht unterstützen können.
</Warning>

<div id="vale-with-mdx">
  #### Vale mit `MDX`
</div>

Vale unterstützt `MDX` nicht nativ, aber der Autor von Vale stellt eine [benutzerdefinierte Erweiterung](https://github.com/errata-ai/MDX) zur Verfügung, um dies zu ermöglichen.

Wenn Sie diese Erweiterung nicht verwenden möchten, empfehlen wir die folgenden Zeilen in Ihrer `.vale.ini`-Datei:

```ini
[formats]
mdx = md

[*.mdx]
CommentDelimiters = {/*, */}

TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})
```

Um die Kommentare innerhalb von Dokumenten in Vale zu nutzen, verwenden Sie MDX‑Kommentare `{/* ... */}`. Wenn Sie in Ihrer Konfiguration die Einstellung `CommentDelimiters = {/*, */}` verwenden, interpretiert Vale diese Kommentare beim Linting automatisch. Das bedeutet, Sie können die integrierten Funktionen von Vale, wie das Überspringen von Zeilen oder Abschnitten, ganz einfach nutzen. Weitere Informationen finden Sie in der [Einstellung](https://vale.sh/docs/keys/commentdelimiters).

```mdx
{/* vale off */}

Dieser Text wird von Vale ignoriert

{/* vale on */}
```

Wenn Sie sich dafür entscheiden, `CommentDelimiters` nicht zu verwenden, aber weiterhin die Kommentare von Vale einsetzen möchten, müssen Sie alle Vale‑Kommentare in MDX‑Kommentare `{/* ... */}` einschließen. Zum Beispiel:

```mdx
{/* <!-- vale off --> */}

Dieser Text wird von Vale ignoriert

{/* <!-- vale on --> */}
```

Diese Kommentar-Tags werden innerhalb von Mintlify-Komponenten nicht unterstützt, können aber überall auf der Basisebene eines Dokuments verwendet werden.
