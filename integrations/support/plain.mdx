The Plain integration connects your [Plain](https://www.plain.com) workspace to Mintlify so that an AI support agent can automatically respond to customer threads using your documentation as a knowledge base. When a customer opens a thread in Plain, the agent searches your docs, drafts a response, and either resolves the thread or escalates it to your team with full context.

## How it works

When a new thread arrives in Plain:

1. Plain sends a webhook event to Mintlify.
2. The support agent searches your published documentation for relevant content.
3. If the agent finds a confident answer, it replies to the customer and resolves the thread.
4. If the agent cannot answer confidently, it escalates the thread to your team with a summary of what it found and relevant internal context.

The agent uses the same indexed documentation as the [assistant](/ai/assistant), so any content improvements you make benefit both features.

## Prerequisites

Before you begin, you need:

- A Mintlify Enterprise plan with the support agent entitlement
- A [Plain](https://www.plain.com) workspace with API access
- A GitHub connection configured in your Mintlify dashboard (required for escalation context)

## Set up the support agent

Configure the support agent from the **Support Agent** page in your [Mintlify dashboard](https://dashboard.mintlify.com).

<Steps>
  <Step title="Copy the webhook URL">
    On the support agent configuration page, copy the webhook URL. You will add this to Plain in the next step.
  </Step>
  <Step title="Add the webhook in Plain">
    In your Plain workspace settings, create a new webhook destination and paste the webhook URL from Mintlify. Set the API version to `2024-09-18`.
  </Step>
  <Step title="Add your Plain API key">
    Create an API key in your Plain workspace and paste it into the **Plain API key** field. This allows the agent to send replies and manage thread state.
  </Step>
  <Step title="Add the HMAC secret">
    Copy the request signing secret from your Plain webhook settings and paste it into the **HMAC secret** field. This verifies that incoming webhook events are from Plain.
  </Step>
  <Step title="Save and test">
    Click **Save** to store your configuration, then click **Test connection** to verify that Mintlify can communicate with your Plain workspace.
  </Step>
</Steps>

## Configure escalation context

When the agent escalates a thread, it can search your connected GitHub repositories for internal context to include in the handoff. This helps your support team understand the technical background without switching tools.

In the **Escalation context** section, select the repositories you want the agent to search during escalation. These repositories are only accessed during escalation and are not exposed to customers.

## Customize agent behavior

After saving your initial configuration, two additional settings become available.

### Agent behavior

Add custom instructions that shape how the agent responds. These instructions are appended to the agent's base system prompt and can include:

- Tone and persona guidelines
- Domain-specific decision rules
- Product terminology preferences
- Escalation criteria

```markdown Example agent behavior instructions
## Tone
Respond in a friendly, concise tone. Avoid jargon unless the customer uses it first.

## Escalation rules
Always escalate threads that mention billing, account deletion, or security incidents.
```

### Company knowledge

Add facts about your product, team, or common issues that the agent can reference when answering questions. This context supplements your documentation and helps the agent handle questions that your docs may not fully cover.

```markdown Example company knowledge
## About Acme
Acme provides developer tools for building real-time applications. Most customers are backend engineers.

## Common issues
- Connection timeouts: Usually caused by firewall rules. Direct customers to the networking guide.
- API rate limits: Free tier is limited to 100 requests per minute. Paid plans have higher limits.
```

Both fields support Markdown and have a maximum length of 8,192 characters.

## Monitor performance

Once the agent starts handling threads, switch to the **Dashboard** tab to view performance metrics:

| Metric | Description |
|---|---|
| Total threads | Total number of threads the agent has processed |
| Active | Threads currently being handled by the agent |
| Deflection rate | Percentage of completed threads resolved without human intervention |
| Escalation rate | Percentage of completed threads escalated to your team |
| Resolved | Total threads the agent resolved successfully |
| Escalated | Total threads handed off to your team |
| Errored | Threads where the agent encountered an error |
| Avg confidence | Average confidence score across agent responses |

The dashboard also shows recent threads with their current state, a preview of the initial message, and the date each thread was created.
