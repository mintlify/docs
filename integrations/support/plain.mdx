The Plain integration connects your Mintlify documentation to [Plain](https://plain.com), a customer support platform. When a customer opens a support thread in Plain, the support agent automatically researches your documentation and responds to the customer's question. If the agent can't resolve the thread, it escalates to your team with context about what it found.

<Info>
  The support agent requires the Plain support agent entitlement on your plan.
</Info>

## How the support agent works

When a new thread arrives in Plain:

1. The agent reads the customer's message and searches your documentation for relevant content.
2. If the agent finds an answer, it replies directly in the Plain thread.
3. If the agent needs more information, it asks the customer a follow-up question.
4. If the agent can't resolve the issue, it escalates the thread to your team with a summary of what it investigated.

The agent uses your documentation content, any connected GitHub repositories, and your custom instructions to formulate responses.

## Set up the Plain integration

<Steps>
  <Step title="Open the support agent page">
    Go to the [Support Agent](https://dashboard.mintlify.com/products/support-agent) page in your Mintlify dashboard.
  </Step>
  <Step title="Add the webhook URL to Plain">
    Copy the webhook URL from the configuration page and add it to your Plain workspace under **Settings > Webhooks**. Set the API version to `2024-09-18`.
  </Step>
  <Step title="Add your Plain API key">
    Create an API key in your Plain workspace and paste it into the **Plain API key** field. This allows the agent to send replies and manage thread state.
  </Step>
  <Step title="Add the HMAC secret">
    Copy the request signing secret from your Plain webhook settings and paste it into the **HMAC secret** field. This verifies that incoming webhook events are from Plain.
  </Step>
  <Step title="Test the connection">
    Click **Test connection** to verify that the agent can communicate with your Plain workspace. A successful test shows your connected workspace name.
  </Step>
</Steps>

## Configure the agent

After completing the initial setup, two additional configuration options become available.

### Escalation context

Choose GitHub repositories that the agent searches when preparing an escalation handoff for your team. This gives the agent access to source code and internal documentation beyond your public docs, which helps it provide more detailed context when escalating a thread.

Add repositories in the **Escalation context** section. You can add up to 5 repositories. The Mintlify GitHub App must be installed on each repository you select.

### Agent behavior

Customize how the agent responds by adding instructions in the **Agent behavior** field. These instructions are appended to the agent's base system prompt and control tone, persona, and decision-making rules.

Use this to define:

- The tone and style the agent should use when replying to customers
- Rules for when to escalate versus continue investigating
- Domain-specific instructions, such as how to handle billing or account questions

```markdown Example agent behavior
You are a friendly and professional support agent for Acme Corp.

Always greet the customer by name when available. Use a conversational
but professional tone.

Escalate immediately if the customer mentions:
- Billing disputes or refund requests
- Account security concerns
- Data deletion requests

Never share internal pricing details or roadmap information.
```

The agent behavior field supports up to 8,192 characters of Markdown.

### Company knowledge

Add facts about your product, team, or common issues in the **Company knowledge** field. The agent references this information when answering questions, supplementing what it finds in your documentation.

Use this to provide:

- Product details not covered in your public documentation
- Common issues and their resolutions
- Team structure or escalation paths
- Frequently asked questions and their answers

```markdown Example company knowledge
## Product

Acme Corp provides API infrastructure for developers.
The current stable release is v3.2.

## Common issues

- "API key not working" — Usually caused by using a test key in
  production. Ask the customer to verify they're using the correct
  key for their environment.
- "Webhook not firing" — Check if the customer's endpoint returns
  a 2xx response. Webhooks pause after 5 consecutive failures.

## Escalation

For billing questions, escalate to the billing team.
For enterprise plan inquiries, escalate to the sales team.
```

The company knowledge field supports up to 8,192 characters of Markdown.

## Monitor performance

Switch to the **Dashboard** tab on the support agent page to view metrics about how the agent handles threads.

The dashboard shows:

| Metric | Description |
|--------|-------------|
| **Total threads** | Total number of threads the agent has handled. |
| **Active** | Threads currently being investigated or answered. |
| **Resolved** | Threads the agent resolved without escalation. |
| **Escalated** | Threads handed off to your team. |
| **Deflection rate** | Percentage of threads resolved without escalation. |
| **Escalation rate** | Percentage of threads escalated to your team. |
| **Avg confidence** | The agent's average confidence score across all threads. |

The dashboard also shows recent threads with their current state and a preview of the initial message.
