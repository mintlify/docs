The Plain support agent uses your documentation to automatically respond to customer support threads in [Plain](https://www.plain.com). When a customer opens a thread, the agent researches your docs, drafts a response, and either resolves the conversation or escalates it to your team.

<Info>
  The support agent requires an [Enterprise plan](https://mintlify.com/pricing).
</Info>

## How it works

When the agent receives a new support thread from Plain:

1. It analyzes the customer's question.
2. It searches your documentation for relevant answers.
3. It responds directly in the Plain thread.
4. If it can't confidently answer, it escalates to your team.

The dashboard tracks metrics like deflection rate, escalation rate, and average confidence so you can monitor performance.

## Prerequisites

- A [Plain](https://www.plain.com) workspace
- An Enterprise Mintlify plan with the support agent enabled
- Admin access to your Mintlify dashboard

## Set up the Plain integration

<Steps>
  <Step title="Copy the webhook URL">
    Go to **Support Agent** in your Mintlify dashboard and copy the webhook URL shown at the top of the configuration page.
  </Step>

  <Step title="Add the webhook in Plain">
    In your Plain workspace settings, add a new webhook destination using the URL from the previous step. Set the API version to `2024-09-18`.
  </Step>

  <Step title="Create a Plain API key">
    In your Plain workspace, create an API key. This allows the agent to send replies and escalations on your behalf.
  </Step>

  <Step title="Copy the HMAC secret">
    In your Plain webhook settings, copy the request signing secret. This verifies that webhook events are genuinely from Plain.
  </Step>

  <Step title="Save your credentials">
    Back in the Mintlify dashboard, paste your Plain API key and HMAC secret into the configuration form, then click **Save**.
  </Step>

  <Step title="Test the connection">
    Click **Test connection** to verify that your credentials are valid and the integration is working.
  </Step>
</Steps>

## Customize the agent

After connecting Plain, two customization fields appear in the configuration page. Use these to control how the agent responds and what context it has access to beyond your documentation.

### Agent behavior

The **Agent behavior** field lets you add instructions that shape how the agent responds. These instructions are appended to the agent's base system prompt, so they apply to every conversation.

Use this field to define:

- **Tone and persona** — whether the agent should be formal, casual, concise, or detailed.
- **Decision rules** — when to escalate instead of answering, what topics to avoid, or how to handle specific question types.
- **Domain-specific behavior** — product-specific instructions that apply across all support threads.

```markdown Example
## Tone
Respond in a friendly, concise tone. Keep answers under 3 paragraphs.

## Domain-specific behavior
When users ask about pricing, always direct them to the sales team.
When users report a bug, ask for their account ID and browser version before troubleshooting.
```

### Company knowledge

The **Company knowledge** field provides context that the agent can reference when answering questions. Use it for information that isn't in your documentation, such as internal policies, common issue resolutions, or product details.

```markdown Example
## About Acme Corp
We build developer tools for API management. Our customers are primarily backend engineers.

## Common issues
- "API key not working": Usually caused by expired keys. Direct users to Settings > API Keys to regenerate.
- "Slow response times": Check if the user is on the free plan, which has lower rate limits.

## Internal policies
- Refunds: We offer refunds within 30 days. Direct refund requests to billing@acme.com.
- Downtime: If there's a known outage, link to status.acme.com.
```

Both fields support Markdown formatting and have a maximum length of 8,192 characters.

## Monitor performance

Switch to the **Dashboard** tab to view support agent metrics:

| Metric | Description |
| --- | --- |
| **Total threads** | Number of support threads the agent has handled. |
| **Active** | Threads currently being processed. |
| **Deflection rate** | Percentage of resolved threads out of all completed threads. |
| **Escalation rate** | Percentage of threads escalated to your team. |
| **Resolved** | Threads the agent resolved without human intervention. |
| **Escalated** | Threads passed to your team because the agent wasn't confident enough. |
| **Avg confidence** | The agent's average confidence score across all threads. |

The dashboard also shows recent threads with their status and a preview of the initial message.
