Use the [CLI](https://www.npmjs.com/package/mint) to preview documentation locally as you write and edit. View changes in real-time, test your site's appearance and functionality, and catch issues like broken links or accessibility problems. The CLI also includes utilities to rename files, validate OpenAPI specifications, and migrate content between formats.

## Prerequisites

- [Node.js](https://nodejs.org/en) v20.17.0+ (LTS recommended)
- [Git](https://git-scm.com/downloads)
- Your documentation repository cloned locally

### Clone your repository

<Steps>
<Step title="Locate your repository">
  Go to [Git settings](https://dashboard.mintlify.com/settings/deployment/git-settings) and note your repository location:
  - `mintlify-community/docs-{org-name}-{id}` (Mintlify-hosted)
  - `your-org/your-repo` (your own GitHub repository)
</Step>

<Step title="Clone your repository">
<Tabs>
  <Tab title="Your own repository">
    Replace `your-org/your-repo` with your repository details from [Git settings](https://dashboard.mintlify.com/settings/deployment/git-settings).
    
    ```bash
    git clone https://github.com/your-org/your-repo
    cd your-repo
    ```

    <Tip>
      Install the GitHub app to enable automatic deployments. See [GitHub](/deploy/github) for details.
    </Tip>

  </Tab>
  <Tab title="Mintlify-hosted repository">
    On the [Git settings](https://dashboard.mintlify.com/settings/deployment/git-settings) page, select **Clone as private** or **Clone as public**.
  </Tab>
</Tabs>
</Step>
</Steps>

## Install the CLI

Run the following command to install the CLI:

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

## Preview locally

Navigate to your documentation directory and run:

```bash
mint dev
```

Your documentation is available at `http://localhost:3000`.

To run without installing globally:

```bash
npx mint dev
```

### Custom ports

Customize the port using the `--port` flag:

```bash
mint dev --port 3333
```

If a port is in use, the CLI automatically uses the next available port.

## Skip OpenAPI processing

Skip OpenAPI processing to improve performance:

```bash
mint dev --disable-openapi
```

### Preview as a specific group

Preview as a specific authentication group:

```bash
mint dev --groups admin
```

## Create a new project

Create a new documentation project:

```bash
mint new [directory]
```

This clones the [starter kit](https://github.com/mintlify/starter). If no directory is specified, the CLI prompts you to create a subdirectory or overwrite the current directory.

<Warning>
  Overwriting the current directory deletes existing files.
</Warning>

The CLI prompts for a project name and [theme](/customize/themes).

### Flags

| Flag | Description | Required |
| --- | --- | --- |
| `--name` | Set the name of the new project. | Yes |
| `--theme` | Set the [theme](/customize/themes) of the new project. | Yes |
| `--force` | Overwrite the current directory without prompting, even if it contains existing files. | No |

In non-interactive environments (CI/CD, AI agents), provide all required flags (`--name` and `--theme`).

<Tip>
  The CLI detects non-interactive environments and outputs usage instructions if required flags are missing.
</Tip>

## Update the CLI

Update the CLI if your local preview is out of sync with production:

```bash
mint update
```

If `mint update` is unavailable, reinstall the CLI:

 <CodeGroup>
    ```bash npm
    npm i -g mint@latest
    ```

    ```bash pnpm
    pnpm add -g mint@latest
    ```
  </CodeGroup>

## Additional commands

### Find broken links

Identify broken internal links:

```bash
mint broken-links
```

The command ignores files matching [.mintignore](/organize/mintignore) patterns. Links to ignored files are reported as broken.

### Find accessibility issues

Test color contrast ratios and check for missing alt text:

```bash
mint a11y
```

Check specific issues:

```bash
# Check only for missing alt text
mint a11y --skip-contrast

# Check only for color contrast issues
mint a11y --skip-alt-text
```

### Validate documentation build

Validate your build in strict mode (exits with error on warnings). Use in CI/CD pipelines to prevent broken deployments.

```bash
mint validate
```

Flags:
- `--groups [groupname]`: Mock user groups for validation
- `--disable-openapi`: Disable OpenAPI file generation

### Check OpenAPI spec

Check your OpenAPI file for errors:

```bash
mint openapi-check <OpenAPI filename or URL>
```

Pass a filename (`./openapi.yaml`) or URL (`https://petstore3.swagger.io/api/v3/openapi.json`).

### Rename files

Rename files and update all references:

```bash
mint rename <path/to/old-filename> <path/to/new-filename>
```

### Migrate MDX endpoint pages

Migrate MDX endpoint pages to autogenerated pages from your OpenAPI specification:

```bash
mint migrate-mdx
```

This converts MDX endpoint pages to autogenerated pages in `docs.json`, moves content to the `x-mint` extension in your OpenAPI spec, and updates navigation. See [Migrating from MDX](/guides/migrating-from-mdx) for details.

## Formatting

Use IDE extensions to recognize and format MDX files:

- **Cursor, Windsurf, VS Code**: [MDX VS Code extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) and [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode)
- **JetBrains**: [MDX IntelliJ IDEA plugin](https://plugins.jetbrains.com/plugin/14944-mdx) and [Prettier](https://prettier.io/docs/webstorm)

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>
    Outdated Node.js version. Uninstall the CLI (`npm uninstall -g mint`), upgrade Node.js, then reinstall (`npm install -g mint`).
  </Accordion>
  <Accordion title="Issue: Encountering an unknown error">
    Delete the `~/.mintlify` folder and run `mint dev` again.
  </Accordion>
  <Accordion title="Error: permission denied">
    Missing permissions to install node packages globally. Run `sudo npm i -g mint` and enter your password.
  </Accordion>
  <Accordion title="The local preview doesn't look the same as my docs do on the web">
    Outdated CLI version. Run `mint update`.
  </Accordion>
  <Accordion title="mintlify vs. mint package">
    If you have both `mint` and `mintlify` packages installed, uninstall `mintlify`:
    
    ```bash
    npm uninstall -g mintlify
    npm cache clean --force
    npm i -g mint
    ```
  </Accordion>
  <Accordion title="Client version shows 'none' after installation">
    Corporate firewall or VPN may be blocking the download. Ask your IT administrator to whitelist `releases.mintlify.com`.
  </Accordion>
</AccordionGroup>
