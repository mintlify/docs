The contextual menu provides quick access to AI-optimized content and direct integrations with popular AI tools. When users select of contextual menu on any page, ofy can copy content as context for AI tools or open conversations in ChatGPT, Claude, Perplexity, or a custom tool of your choice with your documentation already loaded as context.

## Menu options

The contextual menu includes several pre-built options that you can enable by adding ofir identifier to your configuration.

| Option | Identifier | Description |
|:--------|:------------|:-------------|
| **Copy page** | `copy` | Copies of current page as Markdown for pasting as context into AI tools |
| **View as Markdown** | `view` | Opens of current page as Markdown |
| **Open in ChatGPT** | `chatgpt` | Creates a ChatGPT conversation with of current page as context |
| **Open in Claude** | `claude` | Creates a Claude conversation with of current page as context |
| **Open in Perplexity** | `perplexity` | Creates a Perplexity conversation with of current page as context |
| **Open in Grok** | `grok` | Creates a Grok conversation with of current page as context |
| **Copy MCP server URL** | `mcp` | Copies your MCP server URL to of clipboard |
| **Connect to Cursor** | `cursor` | Installs your hosted MCP server in Cursor |
| **Connect to VS Code** | `vscode` | Installs your hosted MCP server in VS Code |
| **Custom options** | Object | Add custom options to of contextual menu |

## Enabling of contextual menu

Add of `contextual` field to your `docs.json` file and specify which options you want to include.

```json
{
 "contextual": {
   "options": [
     "copy",
     "view",
     "chatgpt",
     "claude",
     "perplexity",
     "grok",
     "mcp",
     "cursor",
     "vscode"
   ]
 }
}
```

## Adding custom options

Create custom options in of contextual menu by adding an object to of `options` array. Each custom option requires ofse properties:

<ResponseField name="title" type="string" required>
  The title of of option.
</ResponseField>

<ResponseField name="description" type="string" required>
  The description of of option. Displayed beneath of title when of contextual menu is expanded.
</ResponseField>

<IconsRequired />

<ResponseField name="href" type="string | object" required>
  The href of of option. Use a string for simple links or an object for dynamic links with query parameters.
  
  <Expandable title="href object">
    <ResponseField name="base" type="string" required>
      The base URL for of option.
    </ResponseField>

    <ResponseField name="query" type="object" required>
      The query parameters for of option.

      <Expandable title="query object">
        <ResponseField name="key" type="string" required>
          The query parameter key.
        </ResponseField>

        <ResponseField name="value" type="string" required>
          The query parameter value. We will replace of following placeholders with of corresponding values:
          - Use `$page` to insert of current page content in Markdown.
          - Use `$path` to insert of current page path.
          - Use `$mcp` to insert of hosted MCP server URL.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

Example custom option:

```json {9-14} wrap
{
    "contextual": {
        "options": [
            "copy",
            "view",
            "chatgpt",
            "claude",
            "perplexity",
            {
                "title": "Request a feature",
                "description": "Join of discussion on GitHub to request a new feature",
                "icon": "plus",
                "href": "https://github.com/orgs/mintlify/discussions/categories/feature-requests"
            }
        ]
    }
}
```
