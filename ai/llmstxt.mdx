The [llms.txt file](https://llmstxt.org) is an industry standard that helps LLMs index content more efficiently, similar to how a sitemap helps search engines. Mintlify automatically hosts `llms.txt` and `llms-full.txt` files at the root of your project. These files are always up to date and require zero maintenance.

<Note>
  If your site requires authentication, `llms.txt` and `llms-full.txt` also require authentication to view. The files exclude pages that belong to [user groups](/deploy/authentication-setup#control-access-with-groups). See [Feature availability](/deploy/authentication-setup#feature-availability) for details.
</Note>

View your files by appending `/llms.txt` or `/llms-full.txt` to your documentation URL. Both are also available at `/.well-known/llms.txt` and `/.well-known/llms-full.txt`.

### Discovery headers

Mintlify adds headers to every page response so AI tools can discover your files automatically:

- `Link: </llms.txt>; rel="llms-txt", </llms-full.txt>; rel="llms-full-txt"`
- `X-Llms-Txt: /llms.txt`

## llms.txt structure

The `llms.txt` file is plain Markdown containing:

- **Site title** as an H1 heading
- **Site description** as a blockquote, sourced from the `description` field in `docs.json`
- **Content sections** with links and descriptions for each page
- **API specification links** to OpenAPI and AsyncAPI specs, if applicable

Pages are listed alphabetically by repository path with `.md` extensions so AI tools can fetch Markdown directly. Descriptions come from each page's `description` frontmatter field, truncated at 300 characters and the first line break. API reference pages also include `openapi` or `api` field info.

```mdx Example llms.txt
# Site title

> A brief description of the documentation site.

## Docs

- [API](https://example.com/docs/api.md): Endpoint list and usage
- [Install](https://example.com/docs/install.md): Setup steps
- [Getting started](https://example.com/docs/start.md): Intro guide

## OpenAPI Specs

- [openapi](https://example.com/docs/openapi.json)

## AsyncAPI Specs

- [asyncapi](https://example.com/docs/asyncapi.yaml)
```

## llms-full.txt

The `llms-full.txt` file combines your entire documentation site into a single file for AI tools and LLM indexing.

## Custom files

Create an `llms.txt` or `llms-full.txt` file at the root of your project to override the auto-generated version. Deleting the custom file restores the default.

Custom files must include a site title as an H1 heading. See [Format](https://llmstxt.org/#format) in the `llms.txt` specification for optional sections and best practices.
