Mintlify automatically hosts an [Agent-to-Agent (A2A)](https://a2aproject.github.io/A2A/latest/) agent card at `/.well-known/agent-card.json`. The agent card is a standardized JSON document that helps AI agents discover what your documentation site offers and where to find detailed capability information.

Think of the agent card as a map for agents: a single discovery endpoint that points to your site's metadata, documentation URL, and available skills.

View your agent card by appending `/.well-known/agent-card.json` to your documentation site's URL. Mintlify generates the agent card automatically and keeps it up to date. No configuration is required.

## When to use the agent card

The agent card is most useful when:

- You want A2A-compatible agents to programmatically discover your documentation.
- You want to expose your `skill.md` files to agents through a standard discovery endpoint.
- You are integrating your documentation with agent frameworks that follow the A2A specification.

If your users connect to your documentation through MCP clients, see [Model Context Protocol (MCP)](/ai/model-context-protocol). The agent card complements MCP by providing a lightweight discovery layer that does not require establishing a session.

## Agent card structure

`GET /.well-known/agent-card.json` returns a JSON document that follows the A2A agent card 0.2.0 schema:

```json
{
  "schemaVersion": "0.2.0",
  "name": "Your documentation site",
  "description": "A brief description of your documentation site.",
  "url": "https://your-docs-domain.com/",
  "version": "1.0.0",
  "documentationUrl": "https://your-docs-domain.com/",
  "capabilities": {
    "streaming": false
  },
  "skills": [
    {
      "id": "payments",
      "name": "Payments",
      "description": "Accept payments and manage subscriptions.",
      "tags": [],
      "url": "https://your-docs-domain.com/.well-known/agent-skills/payments/skill.md"
    }
  ]
}
```

| Field              | Description                                                                                                  |
| ------------------ | ------------------------------------------------------------------------------------------------------------ |
| `schemaVersion`    | The A2A agent card schema version.                                                                           |
| `name`             | The name of your documentation site, sourced from your `llms.txt` file.                                      |
| `description`      | A brief description of your documentation site, sourced from your `llms.txt` file.                           |
| `url`              | The root URL of your documentation site.                                                                     |
| `version`          | The agent card version.                                                                                      |
| `documentationUrl` | The URL where agents can find human-readable documentation.                                                  |
| `capabilities`     | Supported agent capabilities. Currently reports `streaming: false`.                                          |
| `skills`           | An array of skills exposed by your documentation. Each entry links to a hosted `skill.md` file.              |

### Skill entries

Each entry in the `skills` array describes a single skill that agents can fetch:

| Field         | Description                                                                                           |
| ------------- | ----------------------------------------------------------------------------------------------------- |
| `id`          | A URL-safe slug derived from the `name` in your `skill.md` frontmatter.                               |
| `name`        | The display name of the skill.                                                                        |
| `description` | A brief description from your `skill.md` frontmatter, truncated to 1024 characters.                   |
| `tags`        | A list of tags associated with the skill. Currently always empty.                                     |
| `url`         | The URL to fetch the full `skill.md` file.                                                            |

Skills come from the same source as the [skills discovery endpoints](/ai/skillmd#skills-discovery-endpoints). To customize the skills that appear in your agent card, add custom skill files to your project. See [Custom skill files](/ai/skillmd#custom-skill-files) for more information.

## Use the agent card with agents

A2A-compatible agents fetch `/.well-known/agent-card.json` to discover your documentation. From the response, agents can:

- Identify your documentation site by name and description.
- Follow the `documentationUrl` to retrieve human-readable content.
- Iterate over the `skills` array and fetch each `skill.md` file to learn what they can accomplish with your product.

```bash
curl https://your-docs-domain.com/.well-known/agent-card.json
```

<Note>
  If you use a [reverse proxy](/deploy/reverse-proxy), configure it to forward the `/.well-known/agent-card.json` path to your Mintlify subdomain.
</Note>
