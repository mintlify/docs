<Info>
  Autopilot is available on [Pro and Custom plans](https://mintlify.com/pricing?ref=autopilot) for anyone with access to your dashboard.
</Info>

Autopilot monitors your connected GitHub repositories and identifies documentation gaps based on code changes. When changes occur in your repositories, Autopilot analyzes them and creates pull request suggestions to keep your documentation up to date.

Use Autopilot to:

- Automatically detect when code changes require documentation updates
- Receive AI-powered suggestions for new documentation content
- Keep your docs in sync with your codebase

## Access Autopilot

Navigate to the [Autopilot page](https://dashboard.mintlify.com/products/autopilot) in your dashboard to view suggestions and configure repository monitoring.

## Configure repository monitoring

Connect GitHub repositories to Autopilot so it can monitor for documentation gaps.

1. Navigate to the [Autopilot page](https://dashboard.mintlify.com/products/autopilot) in your dashboard.
2. Click the **Configuration** button.
3. In the **Connect a new repository** section, select:
   - **Organization**: The GitHub organization that owns the repository.
   - **Repository**: The repository to monitor.
   - **Branch**: The branch to monitor (defaults to `main`).
4. Click **Connect** to add the repository.

<Note>
  Autopilot can only access repositories available through your GitHub App installation. Configure repository access in your [GitHub App settings](https://github.com/apps/mintlify/installations/new).
</Note>

## View connected repositories

The **Connected repositories** section displays all repositories that Autopilot is currently monitoring. Each entry shows:

- The repository owner and name
- The monitored branch

## Remove a repository

To stop monitoring a repository:

1. Navigate to the Autopilot **Configuration** page.
2. Find the repository in the **Connected repositories** section.
3. Click the remove button next to the repository.

## Review suggestions

Autopilot displays documentation suggestions on the main Autopilot page. Each suggestion includes:

- A description of the documentation gap
- The related code changes that triggered the suggestion
- A proposed pull request to address the gap

Review suggestions and approve or dismiss them based on your documentation needs.
