The `Columns` component lets you group multiple `Card` components together. It's most often used to put cards in a grid, by specifying the number of grid columns.

<Columns cols={2}>
  <Card title="Get started" icon="rocket">
    Set up your project with our quickstart guide.
  </Card>
  <Card title="API reference" icon="code">
    Explore endpoints, parameters, and examples for your API.
  </Card>
</Columns>

```mdx Columns example
<Columns cols={2}>
  <Card title="Get started">
    Set up your project with our quickstart guide.
  </Card>
  <Card title="API reference">
    Explore endpoints, parameters, and examples for your API.
  </Card>
</Columns>
```

## Layout options

The `cols` property controls how many columns appear per row. You can use values from 1 to 4.

### Two columns

<Columns cols={2}>
  <Card title="First card" icon="square">
    Content for the first card.
  </Card>
  <Card title="Second card" icon="circle">
    Content for the second card.
  </Card>
</Columns>


## Responsive behavior

Columns automatically adjust to smaller screen sizes:

- **Desktop**: Displays the number of columns you specify
- **Tablet**: Reduces to 2 columns when space is limited
- **Mobile**: Stacks cards vertically in a single column

This ensures your content remains readable on all devices without additional configuration.

### Feature showcase

Use three columns to highlight key features.

<Columns cols={3}>
  <Card title="Fast" icon="zap">
    Built for speed and performance.
  </Card>
  <Card title="Secure" icon="shield">
    Enterprise-grade security.
  </Card>
  <Card title="Scalable" icon="trending-up">
    Grows with your needs.
  </Card>
</Columns>

<Columns cols={2}>
  <Card title="Getting started" icon="rocket" href="/quickstart">
    Set up your project in minutes.

  <Card title="API reference" icon="code" href="/api-reference">
    Explore all available endpoints.

  <Card title="Guides" icon="book-open" href="/guides">
