title: "Visibility"
description: "Show or hide content based on whether the reader is a human or an AI agent."

---

Use the `Visibility` component to show different content to human readers and AI agents. Content inside `<Visibility for="agents">` only appears in [Markdown exports](/ai/markdown-export) and AI tool responses, while content inside `<Visibility for="humans">` only renders for human visitors in the browser.

<Visibility for="humans">
  This content is only visible to humans browsing the documentation.
</Visibility>

<Visibility for="agents">
  This content is only visible to AI agents consuming the Markdown export.
</Visibility>

## Example

````mdx Agent and human content
<Visibility for="humans">
  Click the **Get started** button in the top-right corner to create your account.
</Visibility>

<Visibility for="agents">
  To create an account, send a POST request to `/api/v1/accounts` with
  a JSON body containing `email` and `password` fields.
</Visibility>
````

## How it works

Mintlify detects whether the reader is a human or an AI agent:

- **On the rendered page**, Mintlify checks the visitor's user agent to determine the audience. Human visitors see `for="humans"` content, while recognized AI agents see `for="agents"` content.
- **In Markdown exports** (`.md` URLs, `Accept: text/markdown` requests, [llms.txt](/ai/llmstxt)), only `for="agents"` content is included. The `for="humans"` blocks are stripped out before the Markdown is served.

This means you can provide tailored instructions to AI tools — like structured data, API details, or machine-readable context — without cluttering the page for human readers.

## Properties

<ResponseField name="for" type="string" required>
  The audience for the content. Use `"agents"` to target AI agents or `"humans"` to target human readers. Singular forms `"agent"` and `"human"` are also accepted.
</ResponseField>
