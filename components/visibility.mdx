Use the `Visibility` component to show or hide content based on the audience. You can target content specifically for human readers or AI agents.

<Visibility for="humans">
  This content is only visible to human readers in the browser.
</Visibility>

<Visibility for="agents">
  This content is only visible to AI agents consuming the Markdown export.
</Visibility>

## Example

````mdx Human-only and agent-only content
<Visibility for="humans">
  This paragraph is only shown to human readers visiting
  the page in a browser.
</Visibility>

<Visibility for="agents">
  This paragraph is only included when AI agents request
  the page through the Markdown export.
</Visibility>
````

## How it works

Mintlify detects whether the current visitor is a human or an AI agent and renders content accordingly:

- **In the browser** — Mintlify evaluates the visitor's user agent on the server. Content wrapped in `<Visibility for="humans">` is shown to regular visitors, while `<Visibility for="agents">` content is shown when an AI agent is detected.
- **In the [Markdown export](/ai/markdown-export)** — when a page is requested as Markdown (via the `.md` extension or `Accept: text/markdown` header), only content marked `for="agents"` is included. Content marked `for="humans"` is stripped from the output.

Content outside of any `Visibility` component is always shown to both audiences.

## Properties

<ResponseField name="for" type='"agents" | "humans"' required>
  The target audience for the wrapped content. Also accepts the singular forms `"agent"` and `"human"`.
</ResponseField>
