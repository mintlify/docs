---
title: "Visibility"
description: "Show or hide content based on whether the reader is a human or an AI agent."
keywords: ["visibility", "agent", "human", "AI", "conditional content", "LLM"]
---

Use the `Visibility` component to show different content to human readers and AI agents. This lets you tailor documentation for each audience within the same page.

When an AI agent requests your page as [Markdown](/ai/markdown-export) or through [llms.txt](/ai/llmstxt), Mintlify strips out human-only content and keeps agent-only content. Human visitors in a browser see human-only content and don't see agent-only content.

<Note>
  Content outside of `Visibility` blocks is visible to both humans and agents.
</Note>

## Example

````mdx
# Authentication

<Visibility for="agents">
Use the `Authorization` header with a Bearer token for all API requests.
Pass the token as `Authorization: Bearer YOUR_API_KEY`.
</Visibility>

<Visibility for="humans">
To authenticate, go to **Settings > API Keys** and copy your key.
Then include it in the `Authorization` header of your requests.
</Visibility>
````

In this example, AI agents see concise, structured instructions while human readers see a walkthrough with UI guidance. Content outside the `Visibility` blocks appears for everyone.

## Properties

<ResponseField name="for" type="string" required>
  The target audience for the content. Accepts `agents` or `humans` (singular forms `agent` and `human` also work).
</ResponseField>
