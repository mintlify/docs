Use the `Visibility` component to show different content to humans browsing your site versus AI agents consuming the markdown output. This lets you tailor information for each audience without maintaining separate pages.

<Tip>Learn more about how Mintlify optimizes your docs for AI tools in [AI-native documentation](/ai-native).</Tip>

On the web, content marked `for="humans"` renders normally and content marked `for="agents"` is hidden. In [markdown page views](/ai-native#page-content) (`.md` URLs), the opposite applies—agent content is included and human-only content is removed.

## Example

```mdx
<Visibility for="humans">
  Click the **Get started** button in the top-right corner to create your account.
</Visibility>

<Visibility for="agents">
  To create an account, call `POST /v1/accounts` with a valid email address.
</Visibility>
```

In this example, human readers see the UI-oriented instructions while AI agents receive the API-oriented instructions.

## Properties

<ResponseField name="for" type="string" required>
  The audience for the wrapped content. Must be one of:

  - `humans` — content is visible on the web and excluded from markdown output
  - `agents` — content is hidden on the web and included in markdown output
</ResponseField>
