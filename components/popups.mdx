Use popups to display supplementary content in a floating overlay when a user clicks a trigger element. Popups are useful for definitions, previews, or contextual details without navigating away from the current page.

<Popup>
  <PopupTrigger>
    Click me to see a popup
  </PopupTrigger>
  <PopupContent>
    This content appears in a floating popup overlay.
  </PopupContent>
</Popup>

```mdx Popup example
<Popup>
  <PopupTrigger>
    Click me to see a popup
  </PopupTrigger>
  <PopupContent>
    This content appears in a floating popup overlay.
  </PopupContent>
</Popup>
```

## Usage

The `Popup` component has two required children:

- `PopupTrigger` — the clickable element that opens the popup.
- `PopupContent` — the content displayed inside the floating overlay.

```mdx Popup with rich content
<Popup>
  <PopupTrigger>
    <Badge>API Key</Badge>
  </PopupTrigger>
  <PopupContent>
    An API key is a unique identifier used to authenticate requests to your API.
    You can generate one from your [dashboard](/api-playground/overview).
  </PopupContent>
</Popup>
```

## Components

### Popup

The wrapper component that manages the open/closed state of the popup.

### PopupTrigger

The element that a user clicks to toggle the popup. You can use plain text or nest other components inside it.

### PopupContent

The floating content area that appears when the popup is triggered. Supports Markdown and other MDX components.
