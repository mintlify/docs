## Synchronized selections

Tabs automatically synchronize with other tabs and [code groups](/components/code-groups) on the same page when their labels match. When you select a tab, all other tabs and code groups with the same label will update to match your selection.

<Tabs>
  <Tab title="JavaScript">
    This is JavaScript content in the first tab group.
  </Tab>
  <Tab title="Python">
    This is Python content in the first tab group.
  </Tab>
</Tabs>

<Tabs>
  <Tab title="JavaScript">
    This is JavaScript content in the second tab group. When you select JavaScript above, this tab automatically becomes active too.
  </Tab>
  <Tab title="Python">
    This is Python content in the second tab group.
  </Tab>
</Tabs>

This synchronization works across all tabs and code groups on the page, making it easy for users to see consistent examples in their preferred language or framework without manually switching each component.

Language preferences from code groups persist across page reloads, so users don't need to reselect their preferred language when navigating your documentation.

## Tab properties

<ResponseField name="title" type="string" required>
  The title of the tab. Short titles are easier to navigate. Tabs with matching titles will synchronize their selections across the page.
</ResponseField>

<ResponseField name="icon" type="string">
  A [Font Awesome](https://fontawesome.com/icons) icon, [Lucide](https://lucide.dev/icons) icon, URL to an icon, or relative path to an icon.
</ResponseField>

<ResponseField name="iconType" type="string">
  For Font Awesome icons only: One of `regular`, `solid`, `light`, `thin`, `sharp-solid`, `duotone`, `brands`.
</ResponseField>

## Tabs properties

<ResponseField name="borderBottom" type="boolean">
  Adds a subtle bottom border and padding to the tabs container. Useful for visually separating tabbed content from the rest of the page, especially when tabs contain content of varying lengths.
</ResponseField>
