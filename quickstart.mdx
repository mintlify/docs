#### Configure branch protection rules

For production deployments, set up branch protection rules in your GitHub repository:

1. Navigate to your repository **Settings** > **Branches**.
2. Add a branch protection rule for your main branch.
3. Enable the following settings:
   - **Require pull request reviews before merging** (at least 1 approval)
   - **Require status checks to pass before merging**
   - **Require branches to be up to date before merging**
   - **Include administrators** (optional but recommended)

<CodeGroup>

```yaml .github/workflows/mintlify-check.yml
name: Mintlify Documentation Check

on:
  pull_request:
    branches: [main]
    paths:
      - '**.mdx'
      - 'docs.json'
      - 'mint.json'

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          
      - name: Install Mintlify CLI
        run: npm install -g mint
        
      - name: Check documentation build
        run: mint build --check
```

```yaml .github/CODEOWNERS
# Documentation team owns all MDX files
*.mdx @your-org/docs-team

# Configuration files require admin approval
docs.json @your-org/admin-team
mint.json @your-org/admin-team
```

</CodeGroup>

This ensures all documentation changes are reviewed before deployment and automated checks validate your documentation builds successfully.

## Error handling

Implement comprehensive error handling for production applications:

```typescript
try {
  const response = await client.users.get(userId);
  return response.data;
} catch (error) {
  if (error instanceof APIError) {
    switch (error.statusCode) {
      case 401:
        console.error('Authentication failed:', error.message);
        break;
      case 404:
        console.error('User not found:', error.message);
        break;
      case 429:
        console.error('Rate limit exceeded:', error.message);
        // Implement exponential backoff
        break;
      default:
        console.error('API error:', error.message);
    }
  }
  throw error;
}
```

<Accordion title="Common error codes and solutions">
  - **401 Unauthorized**: Check your API key is valid and has not expired
  - **403 Forbidden**: Verify your API key has the required permissions
  - **404 Not Found**: Confirm the resource ID exists
  - **429 Too Many Requests**: Implement rate limiting and exponential backoff
  - **500 Internal Server Error**: Retry the request with exponential backoff
</Accordion>

### Deploy changes with CI/CD integration

When you are ready to publish your changes, follow a production-ready deployment workflow:

<Steps>
  <Step title="Create a feature branch">
    Create a new branch for your changes:
    
    ```bash
    git checkout -b docs/add-getting-started-guide
    ```
  </Step>
  
  <Step title="Commit your changes">
    Commit with descriptive messages following conventional commits:
    
    ```bash
    git add .
    git commit -m "docs: add comprehensive getting started guide with code examples"
    ```
  </Step>
  
  <Step title="Push and create a pull request">
    Push your branch and create a pull request:
    
    ```bash
    git push origin docs/add-getting-started-guide
    ```
    
    Mintlify automatically generates a preview deployment for your PR.
  </Step>
  
  <Step title="Review the preview deployment">
    Check the preview URL in your PR comments to verify changes before merging.
  </Step>
  
  <Step title="Merge to production">
    After approval, merge your PR. Mintlify automatically deploys to production.
  </Step>
</Steps>

Mintlify automatically detects the changes, builds your documentation, and deploys the updates to your site. Monitor the deployment status in your GitHub repository commit history or the [dashboard](https://dashboard.mintlify.com).

After the deployment completes, your latest update will be available at `<your-project-name>.mintlify.app`.

### DNS configuration

Configure your DNS records based on your hosting setup:

<Tabs>
  <Tab title="Standard CNAME">
    For most subdomain configurations:
    
    <Table>
      | Record Type | Name                | Value                | TTL  |
      | ----------- | ------------------- | -------------------- | ---- |
      | CNAME       | docs (or subdomain) | cname.vercel-dns.com | 3600 |
    </Table>
  </Tab>
  
  <Tab title="Apex Domain">
    For root domain configurations (requires ALIAS or ANAME support):
    
    <Table>
      | Record Type | Name | Value                | TTL  |
      | ----------- | ---- | -------------------- | ---- |
      | ALIAS       | @    | cname.vercel-dns.com | 3600 |
    </Table>
    
    <Warning>
      Not all DNS providers support ALIAS records. Check with your provider or use a subdomain instead.
    </Warning>
  </Tab>
  
  <Tab title="Multiple Domains">
    For serving documentation on multiple domains:
    
    ```json docs.json
    {
      "domains": [
        "docs.yourcompany.com",
        "documentation.yourcompany.com"
      ],
      "redirects": [
        {
          "source": "/old-path/:slug",
          "destination": "/new-path/:slug",
          "permanent": true
        }
      ]
    }
    ```
  </Tab>
</Tabs>

<Info>
  DNS changes can take up to 48 hours to propagate, though changes often complete much sooner. Use [DNSChecker](https://dnschecker.org) to verify propagation status.
</Info>

### SSL certificate provisioning

Mintlify automatically provisions and renews SSL certificates for your custom domain:

1. After DNS records are configured and verified, SSL provisioning begins automatically
2. Certificate issuance typically completes within 10-15 minutes
3. Certificates auto-renew 30 days before expiration
4. All traffic is automatically redirected from HTTP to HTTPS

<Accordion title="Troubleshooting SSL issues">
  If SSL provisioning fails:
  
  - **Verify DNS records**: Ensure CNAME points to `cname.vercel-dns.com`
  - **Check CAA records**: If you have CAA records, add `letsencrypt.org` as an authorized CA
  - **Remove conflicting records**: Delete any existing A or AAAA records for the subdomain
  - **Wait for propagation**: DNS changes must fully propagate before SSL can be issued
  
  Contact support if issues persist after 24 hours.
</Accordion>

### Configure domain security headers

Add security headers to your custom domain for enhanced protection:

```json docs.json
{
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "X-Frame-Options",
          "value": "SAMEORIGIN"
        },
        {
          "key": "X-Content-Type-Options",
          "value": "nosniff"
        },
        {
          "key": "Referrer-Policy",
          "value": "strict-origin-when-cross-origin"
        },
        {
          "key": "Permissions-Policy",
          "value": "camera=(), microphone=(), geolocation=()"
        },
        {
          "key": "Strict-Transport-Security",
          "value": "max-age=31536000; includeSubDomains"
        }
      ]
    }
  ]
}
```

These headers protect against common web vulnerabilities and enforce HTTPS usage.

## Troubleshooting

If you encounter issues during the setup process, check these common troubleshooting solutions:

<AccordionGroup>
  <Accordion title="Local preview not working">
    Make sure you have Node.js v20.17.0 or higher installed and that you run the `mint dev` command from the directory containing your `docs.json` file.
  </Accordion>
  <Accordion title="Changes not reflecting on live site">
    Deployment can take up to a few minutes. Check your GitHub Actions (for code-based workflow) or deployment logs in the Mintlify dashboard to ensure there are no build errors.
  </Accordion>
  <Accordion title="Custom domain not connecting">
