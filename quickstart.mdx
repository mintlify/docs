## Your documentation site

After onboarding, your site deploys automatically to:

```
https://<your-project-name>.mintlify.app
```

Find your URL on the [dashboard](https://dashboard.mintlify.com/) Overview page.

## Connect GitHub

1. Navigate to **Settings** > **GitHub App** in your dashboard.
2. Select **Install GitHub App** and choose your repositories.

<Info>
  Update the GitHub App permissions if you move your documentation to a different repository.
</Info>

## Code-based workflow

### Install the CLI

<Info>
  Requires [Node.js](https://nodejs.org/en) v20.17.0+ through v24.
</Info>

<CodeGroup>

```bash npm
npm i -g mint
```

```bash pnpm
pnpm add -g mint
```

</CodeGroup>

### Create and edit

```bash
mint new        # Create a new project
mint dev        # Preview at localhost:3000
```

Edit your MDX files, then push to GitHub. Mintlify automatically deploys your changes.

## Web editor workflow

1. Open **Editor** in your [dashboard](https://dashboard.mintlify.com).
2. Select a file and make your edits.
3. Select **Publish** to deploy.

<Tip>
  Type <kbd>/</kbd> in the editor to access formatting tools and components.
</Tip>

## Add a custom domain

1. Go to [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain) in your dashboard.
2. Enter your domain and configure DNS:

| Record Type | Name                | Value                | TTL  |
| ----------- | ------------------- | -------------------- | ---- |
| CNAME       | docs (or subdomain) | cname.vercel-dns.com | 3600 |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Local preview not working">
    Ensure Node.js v20.17.0+ is installed and run `mint dev` from the directory containing `docs.json`.
  </Accordion>
  <Accordion title="Changes not reflecting on live site">
    Deployment takes a few minutes. Check your dashboard for build errors.
  </Accordion>
  <Accordion title="Custom domain not connecting">
    Verify DNS records and allow up to 48 hours for propagation. Use [DNSChecker](https://dnschecker.org) to verify.
  </Accordion>
</AccordionGroup>
