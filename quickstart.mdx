## Your documentation site

After onboarding, your site deploys to `https://<your-project-name>.mintlify.app`. Find your URL on the [dashboard](https://dashboard.mintlify.com/) Overview page.

## Choose your workflow

<Tabs>
  <Tab title="Web editor">
    The web editor provides a visual interface for editing documentation directly in your browser.

    1. Log in to your [dashboard](https://dashboard.mintlify.com).
    2. Select **Editor** on the left sidebar.
    3. Select a file from the sidebar to edit it.
    4. Select **Publish** to deploy your changes.

    <Tip>
      Type <kbd>/</kbd> in the editor to open the command menu and access formatting tools.
    </Tip>

    For more details, see the [web editor documentation](/editor).
  </Tab>
  <Tab title="Code-based">
    The code-based workflow integrates with your local development environment and Git repositories.

    **Install the CLI** ([Node.js](https://nodejs.org/en) v20.17.0+ required):

    <CodeGroup>
    ```bash npm
    npm i -g mint
    ```

    ```bash pnpm
    pnpm add -g mint
    ```
    </CodeGroup>

    **Create and preview a project:**

    ```bash
    mint new
    mint dev
    ```

    Your preview is available at `localhost:3000`.

    **Deploy changes:**

    Push to your repository. Mintlify automatically builds and deploys your updates.
  </Tab>
</Tabs>

## Connect GitHub

Install the GitHub App to automate deployments:

1. Navigate to **Settings** > **GitHub App** in your dashboard.
2. Select **Install GitHub App**.
3. Choose your organization and repositories to connect.

## Add a custom domain

Navigate to [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain) in your dashboard and add a CNAME record with your domain provider:

| Record Type | Name                | Value                | TTL  |
| ----------- | ------------------- | -------------------- | ---- |
| CNAME       | docs (or subdomain) | cname.vercel-dns.com | 3600 |
