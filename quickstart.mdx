</Info>

## Your documentation site

After onboarding, your site deploys to `https://<your-project-name>.mintlify.app`. Find your URL in the [dashboard](https://dashboard.mintlify.com/).

## Choose your workflow

<Tabs>
  <Tab title="Web editor">
    The web editor provides a visual interface for editing documentation directly in your browser.

    1. Go to your [dashboard](https://dashboard.mintlify.com) and select **Editor**.
    2. Select a file from the sidebar to edit.
    3. Make your changes and select **Publish**.

    <Tip>
      Type <kbd>/</kbd> in the editor to access formatting tools and components.
    </Tip>

    For more details, see the [web editor documentation](/editor).
  </Tab>
  <Tab title="Code-based">
    The code-based workflow lets you edit documentation locally and deploy via Git.

    **Install the CLI**

    <Info>
      Requires [Node.js](https://nodejs.org/en) v20.17.0 or higher.
    </Info>

    ```bash
    npm i -g mint
    ```

    **Create and preview a project**

    ```bash
    mint new
    mint dev
    ```

    Your preview is available at `localhost:3000`.

    **Deploy changes**

    Push to your repository. Mintlify automatically builds and deploys your site.
  </Tab>
</Tabs>

## Connect GitHub

Install the GitHub App to automate deployments:

1. Go to **Settings** > **GitHub App** in your dashboard.
2. Select **Install GitHub App** and choose your repositories.

## Add a custom domain

1. Go to [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain) in your dashboard.
2. Enter your domain and configure DNS with your provider:

| Record Type | Name                | Value                | TTL  |
| ----------- | ------------------- | -------------------- | ---- |
| CNAME       | docs (or subdomain) | cname.vercel-dns.com | 3600 |

<Info>
  DNS changes can take up to 48 hours to propagate.
</Info>
