Set up and deploy your documentation site in minutes.

<Info>
  **Prerequisites**: [Create an account](https://mintlify.com/start) and complete onboarding.
</Info>

## Getting started

After onboarding, your site automatically deploys to `https://<your-project-name>.mintlify.app`. Find your URL on the Overview page of your [dashboard](https://dashboard.mintlify.com/).

### Install the GitHub App

The GitHub App automates deployment when you push changes.

1. In your dashboard, go to **Settings** \> **GitHub App**.
2. Select **Install GitHub App** and choose the organization and repositories to connect.

<Info>
  Update the GitHub App permissions if you move your docs to a different repository.
</Info>

### Authorize your GitHub account

In your dashboard, go to **Settings** \> **My Profile** and select **Authorize GitHub account**.

<Info>
  An admin for your GitHub organization may need to authorize your account.
</Info>

## Code-based workflow

Best for technical teams managing docs alongside code.

1. Install the [mint](https://www.npmjs.com/package/mint) CLI (requires Node.js):
   ```bash
   npm i -g mint
   ```
2. Edit your docs files locally (for example, update `title` in `index.mdx`).
3. Preview changes with `mint dev` at `localhost:3000`.
4. Push to your repository. Mintlify automatically builds and deploys. Monitor status in GitHub or the [dashboard](https://dashboard.mintlify.com).

## Web editor workflow

Best for people who prefer a visual interface without local tools.

1. Log in to your [dashboard](https://dashboard.mintlify.com) and select **Editor**.
2. Find and edit files in the sidebar. Type <kbd>/</kbd> to open the command menu for components.
3. Select **Publish** to deploy your changes.

<Tip>
  Use branches to preview and review changes via pull requests before deploying. See the [web editor documentation](/editor) for details.
</Tip>

## Adding a custom domain

For production, add a custom domain on the [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain) page. Configure your DNS with a CNAME record:

<Table>
  | Record Type | Name                | Value              | TTL  |
  | ----------- | ------------------- | ------------------ | ---- |
  | CNAME       | docs (or subdomain) | cname.mintlify.app | 3600 |
</Table>

<Info>
  DNS changes can take up to 48 hours to propagate.
</Info>

## Troubleshooting

If you encounter issues during the setup process, check these common troubleshooting solutions:

<AccordionGroup>
  <Accordion title="Local preview not working">
    Make sure you have Node.js v19\+ installed and that you run the `mint dev` command from the directory containing your `docs.json` file.
  </Accordion>

  <Accordion title="Changes not reflecting on live site">
    Deployment can take upwards to a few minutes. Check your GitHub Actions (for code-based workflow) or deployment logs in the Mintlify dashboard to ensure there are no build errors.
  </Accordion>

  <Accordion title="Custom domain not connecting">
    Verify that your DNS records are set up correctly and allow sufficient time for DNS propagation (up to 48 hours). You can use tools like [DNSChecker](https://dnschecker.org) to verify your CNAME record.
  </Accordion>
</AccordionGroup>
