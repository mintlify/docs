Related topics display links to relevant pages at the bottom of each documentation page. These suggestions help readers discover content they might not find through navigation alone.

Related topics supports two modes:

- **Automatic**: Uses search to find semantically related pages. You can override suggestions on individual pages with frontmatter.
- **Manual**: Only shows related pages that you specify in frontmatter.

## Enable related topics

Related topics are enabled by default. To configure or disable them:

<Steps>
  <Step title="Open the Add-ons page">
    Navigate to the [Add-ons](https://dashboard.mintlify.com/products/addons) page in your dashboard.
  </Step>
  <Step title="Configure related topics">
    Toggle **Show related topics** to enable or disable the feature. When enabled, choose between **Automatic** and **Manual** mode.
  </Step>
  <Step title="Set the number of topics">
    Use the **Number of topics** control to set the maximum number of related pages to display. The default is 5 and the maximum is 10.
  </Step>
</Steps>

## Automatic mode

In automatic mode, Mintlify uses search to find pages related to the current page's title. Results are filtered by language and version when applicable, and the current page is excluded from suggestions.

To override automatic suggestions on a specific page, add a `relatedPages` field to the page's frontmatter. When frontmatter is present, it replaces the automatic suggestions for that page.

## Manual mode

In manual mode, related topics only appear on pages where you define them in frontmatter. Pages without a `relatedPages` field show no suggestions.

## Set related pages in frontmatter

Use the `relatedPages` frontmatter field to specify which pages appear as related topics. This works in both automatic mode (as an override) and manual mode.

### Link to internal pages

Reference pages by their path. Mintlify resolves the page title and icon automatically from your navigation.

```yaml
---
title: "Authentication"
relatedPages:
  - /quickstart
  - /api-playground/overview
---
```

### Link to external pages

Use full URLs to link to pages outside your documentation.

```yaml
---
title: "Authentication"
relatedPages:
  - https://github.com/your-org/your-repo
---
```

### Set a custom title

Use a key-value format to set a custom display title for a related page.

```yaml
---
title: "Authentication"
relatedPages:
  - Getting started quickly: /quickstart
  - API reference: /api-playground/overview
  - Source code: https://github.com/your-org/your-repo
---
```

### Disable related topics on a page

Set `relatedPages` to `false` to hide suggestions on a specific page, even when related topics are enabled globally.

```yaml
---
title: "Changelog"
relatedPages: false
---
```
