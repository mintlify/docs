Related pages displays a list of contextually relevant page suggestions at the bottom of each documentation page. This helps readers discover related content and navigate your docs more effectively.

<Info>
  To enable related pages, go to the [Add-ons](https://dashboard.mintlify.com/products/addons) page in your dashboard.
</Info>

## How it works

When enabled, Mintlify automatically displays a "Related topics" section below the page content. You can choose between two modes:

- **Automatic** — Mintlify uses search-based matching to find semantically related pages. You can override automatic suggestions on individual pages using frontmatter.
- **Manual** — Only pages with explicit `relatedPages` frontmatter display related pages. No automatic suggestions are generated.

## Enable related pages

<Steps>
  <Step title="Open the Add-ons page">
    Go to the [Add-ons](https://dashboard.mintlify.com/products/addons) page in your dashboard.
  </Step>
  <Step title="Enable related pages">
    Toggle the related pages feature on.
  </Step>
  <Step title="Choose a mode">
    Select **Automatic** for search-based suggestions, or **Manual** to only show pages you specify via frontmatter.
  </Step>
  <Step title="Set a limit">
    Configure the maximum number of related pages to display. The default is 5, and the maximum is 10.
  </Step>
</Steps>

## Configure related pages per page

Use the `relatedPages` frontmatter property to control related pages on individual pages. This property works in both automatic and manual modes.

### Specify related pages manually

List the pages you want to display as related pages in the frontmatter. Each entry can be a page path or an external URL.

```yaml
---
title: "Authentication"
relatedPages:
  - /api-reference/tokens
  - /guides/security
  - https://example.com/external-resource
---
```

### Set custom titles

Provide a custom title for a related page by using the `Title: path` format.

```yaml
---
title: "Authentication"
relatedPages:
  - Managing API tokens: /api-reference/tokens
  - Security best practices: /guides/security
---
```

For internal pages, if you don't provide a custom title, the page title from navigation is used. For external URLs without a custom title, the URL itself is displayed.

### Disable related pages on a specific page

To hide related pages on a page where they would otherwise appear, set `relatedPages` to `false`.

```yaml
---
title: "Changelog"
relatedPages: false
---
```

<Note>
  When using automatic mode, frontmatter-specified related pages override the automatic suggestions for that page.
</Note>

## Limits

- The maximum number of related pages per page is 10, regardless of the limit set in the dashboard.
- Manually specified related pages in frontmatter are also subject to this limit.
