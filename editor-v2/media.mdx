## Supported file types

- **Images**: PNG, JPG, JPEG, GIF, WebP (max 20 MB)
- **Videos**: MP4, WebM (max 20 MB)
- **Fonts**: TrueType (TTF), WOFF, WOFF2
- **Other**: PDF, ICO (favicons)

## Add media to a page

### Drag and drop

Drag and drop media from your computer onto a page. In visual mode, the image or video inserts inline. In Markdown mode, the editor inserts the path at your cursor position. Images save to the root of your `images/` folder, and videos save to the root of your `videos/` folder.

### Slash command

Type <kbd>/image</kbd> to open the image menu or <kbd>/video</kbd> to open the video menu. Upload new media or select from existing files in your repository.

### Embed

Type <kbd>/youtube</kbd> to embed a YouTube video or <kbd>/loom</kbd> to embed a Loom video. Paste the video URL and the editor generates an iframe.

For other embeds, type <kbd>/embed</kbd> and switch to **Manual embed** to write custom HTML.

## Format images

After inserting an image in visual mode, hover over it to reveal the image toolbar.

- **Light/dark variants**: Upload separate images for light and dark mode. The editor displays the correct image automatically without needing a `<Frame>` wrapper.
- **Alignment**: Set the image alignment to left, center, right, or full-width.
- **Frame**: Wrap the image in a `<Frame>` component to add a border and background.
- **Settings**: Replace the image or update its alt text.

## Best practices

- **Use descriptive names**: Name files clearly, like `api-dashboard-light.png` instead of `img1.png`.
- **Add alt text**: Provide descriptive alt text for accessibility and SEO.
- **Use appropriate formats**: PNG for graphics with transparency, JPG for photos. SVG files must be added to your repository via Git—they cannot be uploaded through the editor.
- **Light and dark variants**: When content looks different in light and dark mode, use the image toolbar to set both variants rather than wrapping in a `<Frame>`.
