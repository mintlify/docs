# Revalidating Your Git Connection

If you recently changed your GitHub repository's visibility (private → public or public → private) and features like **Suggest Edits** have stopped working, your git connection is likely stale. Mintlify cached the previous visibility state and needs to re-verify its access.

## How to fix it

1. Go to your [Git Settings page](https://app.mintlify.com/settings/deployment/git-settings)
2. Press **`Cmd + Shift + Enter`** (Mac) or **`Ctrl + Shift + Enter`** (Windows)

This keyboard shortcut revalidates your git connection. Mintlify will re-check the repository's visibility and permissions, which restores affected features.

## Why this happens

When a repository's privacy setting changes, Mintlify's cached understanding of the repo's permissions becomes outdated. Features like Suggest Edits rely on Mintlify having confirmed read access — if the cache hasn't caught up to the new visibility state, these features appear broken even though the repo itself is fine.

After revalidating, allow a minute for changes to propagate, then refresh your docs site.
