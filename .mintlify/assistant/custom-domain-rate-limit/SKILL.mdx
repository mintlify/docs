# Custom Domain Setup: \_acme-challenge Value Not Appearing

If you've entered your custom domain in the Mintlify dashboard but the `_acme-challenge` TXT record value never loads — the field is blank, shows a spinner, or the value keeps changing — this is almost always a **Let's Encrypt rate limit**.

## Why this happens

Let's Encrypt limits how many certificate requests can be made for the same domain in a short window. Each time you delete and re-add your custom domain in the dashboard, or retry a failed validation, it counts as a new attempt. After several failed attempts, Let's Encrypt imposes a cooldown before a new challenge token can be generated.

## What to do

1. **Wait 24–72 hours** before retrying. The rate limit clears on its own.
2. **Do not delete and re-add the domain** — this resets the attempt counter and can extend the wait. Leave the domain entry as-is in the dashboard.
3. **Do not keep refreshing or clicking retry** — additional attempts may extend the cooldown further.
4. Once the cooldown clears, the `_acme-challenge` TXT value will appear automatically. Add it to your DNS and complete validation as normal.

## If you've already been retrying repeatedly

The cooldown can extend up to **7 days** if the rate limit has been hit multiple times. The best path forward is to leave the domain entry in place and wait. Contact support if the value hasn't appeared after a week.
