[Loops](https://loops.so) builds software for teams that expect clarity and speed, shipping updates on a daily basis to thousands of monthly users. To keep documentation aligned with that pace, they turned to Mintlify. The [Mintlify agent](/agent/index) allows them to update content with AI directly from Slack, while the documentation as code framework fits naturally into an engineering workflow and keeps everything accurate as the product evolves.

## The Mintlify agent helps the team move quickly and improve support

Slack plays a key role in the team's overall workflow. They often handle support questions in Slack, and [Mintlify's agent Slackbot](/agent/slack) lets them generate documentation pull requests directly from those conversations. This tightens the loop between customer insight and content updates and has made the process "a lot, lot easier."

Strong documentation also reduces support load. The team watches for patterns in customer questions and treats repeated issues as signals that a page needs improvement. They check [Mintlify analytics](/optimize/analytics) regularly to see what people are reading most and where they might be getting stuck.

## Documentation as code

Mintlify's documentation as code framework keeps the process familiar to Loops' infrastructure as code workflow. Git version control system is the ground truth for everything.

Docs live in a Git repo, reviewers use pull requests, and updates publish the moment a commit is merged. Mintlify's Github app automatically configures CI checks for spellchecks, broken links, and accessibility on the site.

## One platform for guides, concepts, and API reference

The Loops documentation site is entirely built on Mintlify. The team uses it to organize onboarding content, conceptual explanations, troubleshooting guides, and their full API reference.

Mintlify's auto-generated [OpenAPI documentation](/api-playground/openapi-setup) is especially impactful: generating examples in seven languages, providing an interactive API playground, and updating everything whenever API changes ship. This eliminates a large amount of manual writing and maintenance.

## Documentation as part of the product and sales journey

Loops treats documentation as a core part of the customer experience. Their marketing site is intentionally concise and directs visitors straight to the docs. Many prospects land on documentation pages through search, so every page is written to stand alone and clearly communicate its purpose.

For existing customers, well-structured docs help them understand what Loops can do and discover features they may not yet be using.

> With Mintlify, we can focus on writing clear docs instead of maintaining a docs engine. Updates ship through GitHub, everything stays organized, and the site always reflects the latest version of our product.
>
> — Dan Rowden, Developer Experience Engineer at Loops

## The outcome

Loops runs a documentation system that supports product education, onboarding, and self-serve adoption and Mintlify provides the structure, automation, and workflow alignment that make this possible. The result is a fast and approachable docs experience that matches the quality of the Loops platform and helps customers get value quickly.
