# API reference

The Ian ACME API allows you to programmatically interact with your workspace, projects, tasks, and more.

## Authentication

All API requests require authentication using an API key.

### Get your API key

1. Navigate to **Settings** > **API**
2. Click **Generate New Key**
3. Copy and store your key securely

<Callout type="warning">
Keep your API keys secure. Never commit them to version control or share them publicly.
</Callout>

### Making authenticated requests

Include your API key in the Authorization header:

```bash
curl https://api.ian-acme.com/v1/projects \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Base URL

All API requests should be made to:

```
https://api.ian-acme.com/v1
```

## Rate limits

- **Free tier** - 100 requests per hour
- **Pro tier** - 1,000 requests per hour
- **Enterprise** - Custom limits available

## Core resources

### Projects

Manage your projects programmatically.

#### List projects

```bash
GET /projects
```

Response:

```json
{
  "data": [
    {
      "id": "proj_123",
      "name": "Q1 Marketing Campaign",
      "description": "Marketing initiatives for Q1",
      "created_at": "2026-01-01T00:00:00Z",
      "status": "active"
    }
  ]
}
```

#### Create a project

```bash
POST /projects
```

Request body:

```json
{
  "name": "New Project",
  "description": "Project description",
  "visibility": "private"
}
```

### Tasks

Create and manage tasks within projects.

#### Create a task

```bash
POST /tasks
```

Request body:

```json
{
  "title": "Complete documentation",
  "description": "Write API documentation",
  "project_id": "proj_123",
  "assignee_id": "user_456",
  "due_date": "2026-03-01",
  "priority": "high"
}
```

#### Update a task

```bash
PATCH /tasks/{task_id}
```

Request body:

```json
{
  "status": "completed",
  "progress": 100
}
```

### Teams

Manage team members and permissions.

#### List team members

```bash
GET /teams/{team_id}/members
```

#### Add a team member

```bash
POST /teams/{team_id}/members
```

Request body:

```json
{
  "email": "user@example.com",
  "role": "member"
}
```

## Webhooks

Subscribe to events in your workspace.

### Available events

- `project.created`
- `project.updated`
- `project.deleted`
- `task.created`
- `task.updated`
- `task.completed`
- `member.added`
- `member.removed`

### Configure webhooks

1. Go to **Settings** > **Webhooks**
2. Click **Add Webhook**
3. Enter your endpoint URL
4. Select events to subscribe to

### Webhook payload

```json
{
  "event": "task.completed",
  "timestamp": "2026-02-09T10:30:00Z",
  "data": {
    "task_id": "task_789",
    "project_id": "proj_123",
    "completed_by": "user_456"
  }
}
```

## SDKs

Official SDKs are available for popular languages:

- [JavaScript/TypeScript](https://github.com/ian-acme/sdk-js)
- [Python](https://github.com/ian-acme/sdk-python)
- [Ruby](https://github.com/ian-acme/sdk-ruby)
- [Go](https://github.com/ian-acme/sdk-go)

### JavaScript example

```javascript
import { IanAcme } from '@ian-acme/sdk';

const client = new IanAcme({
  apiKey: process.env.IAN_ACME_API_KEY
});

// Create a task
const task = await client.tasks.create({
  title: 'New task',
  projectId: 'proj_123',
  assigneeId: 'user_456'
});
```

## Error handling

The API uses standard HTTP status codes:

- `200` - Success
- `400` - Bad request
- `401` - Unauthorized
- `403` - Forbidden
- `404` - Not found
- `429` - Rate limit exceeded
- `500` - Server error

Error response format:

```json
{
  "error": {
    "code": "invalid_request",
    "message": "Project ID is required"
  }
}
```
