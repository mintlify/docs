## Writing principles

- **Be concise.** Get to the point quickly.
- **Clarity over cleverness.** Be simple, direct, and avoid jargon.
- **Use active voice.** Write "Create a configuration file" not "A configuration file should be created."
- **Be skimmable.** Use headlines, bullet points, and short paragraphs.
- **Write in second person.** Address your reader directly.

## Common mistakes

- **Spelling and grammar errors** reduce credibility.
- **Inconsistent terminology** confuses users.
- **Product-centric language** assumes too much context.
- **Colloquialisms** hurt clarity and localization.

## Enforce style

Use established style guides:

- [Microsoft Style Guide](https://learn.microsoft.com/en-us/style-guide/welcome/)
- [Google Developer Documentation Style Guide](https://developers.google.com/style)

Automate with [CI checks](/deploy/ci) or linters like [Vale](https://vale.sh).
