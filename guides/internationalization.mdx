Add multi-language support to your documentation with a language switcher dropdown. Users can select their preferred language, and you can maintain separate content and navigation for each language.

## File structure

Create a separate directory for each language using [ISO 639-1 language codes](https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes). Keep the same file names and directory structure across all languages.

<Expandable title="supported language codes">
- `ar` - Arabic
- `cs` - Czech
- `cn` or `zh-Hans` - Chinese (Simplified)
- `zh-Hant` - Chinese (Traditional)
- `de` - German
- `en` - English
- `es` - Spanish
- `fr` - French
- `he` - Hebrew
- `hi` - Hindi
- `id` - Indonesian
- `it` - Italian
- `jp` - Japanese
- `ko` - Korean
- `lv` - Latvian
- `nl` - Dutch
- `no` - Norwegian
- `pl` - Polish
- `pt` or `pt-BR` - Portuguese
- `ro` - Romanian
- `ru` - Russian
- `sv` - Swedish
- `tr` - Turkish
- `ua` - Ukrainian
- `uz` - Uzbek
- `vi` - Vietnamese
</Expandable>

```text Example file structure
docs/
├── index.mdx                    # English (default)
├── quickstart.mdx
├── fr/
│   ├── index.mdx               # French
│   ├── quickstart.mdx
├── es/
│   ├── index.mdx               # Spanish
│   ├── quickstart.mdx
└── zh/
    ├── index.mdx               # Chinese
    └── quickstart.mdx
```



## Add the language switcher

Configure the `languages` array in your `docs.json` navigation to enable the language switcher dropdown.

```json docs.json
{
  "navigation": {
    "languages": [
      {
        "language": "en",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["index", "quickstart"]
          }
        ]
      },
      {
        "language": "es",
        "groups": [
          {
            "group": "Comenzando",
            "pages": ["es/index", "es/quickstart"]
          }
        ]
      }
    ]
  }
}
```

Each language entry requires a `language` code and its own navigation structure with paths to translated files. You can customize the navigation for each language independently.

<Tip>
  Translate navigation labels like group or tab names to match the language of the content. This creates a fully localized experience for your users.
</Tip>

### Add global navigation

Use the `global` object to add navigation elements that appear across all languages.

```json docs.json
{
  "navigation": {
    "global": {
      "anchors": [
        {
          "anchor": "Documentation",
          "href": "https://example.com/docs"
        },
        {
          "anchor": "Blog",
          "href": "https://example.com/blog"
        }
      ]
    },
    "languages": [
      // Language-specific navigation
    ]
  }
}
```

## Translation workflow

1. Update source content in your primary language.
2. Translate changed content.
3. Update translated files.
4. Verify navigation and links work.

### Translate images

Store translated images in language-specific directories and reference them using relative paths.

```
images/
├── dashboard.png          # English version
├── fr/
│   └── dashboard.png     # French version
└── es/
    └── dashboard.png     # Spanish version
```

```mdx es/index.mdx
![Captura de pantalla del panel de control](/images/es/dashboard.png)
```

## SEO optimization

### Translate page metadata

Include translated metadata in each file's frontmatter:

```mdx fr/index.mdx
---
title: "Commencer"
description: "Apprenez à commencer avec notre produit."
keywords: ["démarrage", "tutoriel", "guide"]
---
```

## Best practices

- **Maintain content parity** - Keep the same content structure and quality across all languages.
- **Create a translation glossary** - Document technical terms to ensure consistency.
- **Use locale-specific formats** - Adapt dates (MM/DD/YYYY vs DD/MM/YYYY), numbers (1,000.00 vs 1.000,00), and currency symbols for each language.
- **Test layout differences** - Some languages require more space than English. Verify navigation, code blocks, and tables display properly.
- **Use UTF-8 encoding** - Ensure your development environment supports all characters and alphabets.
