## What you will build

A configuration that allows a GitHub App to automatically create and merge pull requests without requiring manual approvals or status checks that apply to human contributors.

This is useful when you have automation tools (like CI/CD bots or documentation agents) that need to merge changes directly to your `main` branch.

## Prerequisites

- Repository admin access
- A GitHub App installed on the repository
- An existing ruleset targeting the `main` branch

## Steps

### Navigate to rulesets

Go to **Settings** > **Rules** > **Rulesets** in your repository.

### Select the ruleset

Click on the ruleset that targets your `main` branch.

### Add the GitHub App to the bypass list

1. Under **Bypass list**, click **Add bypass**.
2. Search for and select the installed GitHub App.
3. Set the bypass mode to **Always**.

This allows the app to create PRs and merge them without satisfying the review and status check requirements that apply to everyone else.

### Save

Click **Save changes**.

## Verify the setup

1. **As a human**: Open a PR targeting `main`. Confirm that it still requires approval and passing status checks before the merge button is enabled.
2. **As the GitHub App**: Have the app create a PR and attempt to merge it. Confirm that the merge succeeds without requiring approval.

## Troubleshooting

| Problem | Cause | Fix |
|---|---|---|
| App can't merge | App is not on the bypass list | Add the app under **Bypass list** with **Always** mode |
| App can't merge | Multiple rulesets target `main` | The app must be on the bypass list of **every** active ruleset targeting `main` |
| Humans can merge without approval | Enforcement is set to **Disabled** or **Evaluate** | Set enforcement to **Active** |
