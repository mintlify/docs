## Navigate files

Browse your documentation files in the **Navigation** tab of the left panel.

- Click navigation items to expand or collapse them.
- Click files to open them in the editor.
- Click the search icon or press <kbd>Cmd</kbd> + <kbd>P</kbd> (macOS) or <kbd>Ctrl</kbd> + <kbd>P</kbd> (Windows) to search for files by name.

## Create new pages

1. Click the **+** button in the navigation element where you want to add a page.
1. Click **Add a page**.
1. Enter a file name. The editor adds the `.mdx` extension automatically.

### Visual mode

Edit content with real-time previews that show how the content looks when it is published.

- **Add text**: Type in the editor to see how the text appears when published.
- **Format text**: Use the toolbar to bold, italicize, or apply other formatting to text.
- **Add components**: Press <kbd>/</kbd> to open the component menu and select components.
- **Add images**: Use the image component from the <kbd>/</kbd> menu.
- **Insert links**: Select text and press <kbd>Cmd</kbd> + <kbd>K</kbd>.

See [Components](/components) documentation for the complete list of available components.

### Markdown mode

Edit the MDX source code.

- **Direct MDX editing**: Write MDX and Markdown syntax for precise control over content.
- **Syntax highlighting**: Code is highlighted for readability.
- **Component properties**: Set component properties and configurations.
- **Frontmatter**: Edit page metadata at the top of the file.

See [Format text](/create/text) and [Format code](/create/code) for more information on MDX syntax.

## Configure pages

Configure how pages appear in your documentation by right-clicking a page and selecting **Configure**.

### Content and navigation

- **Title**: The main heading for your page. Appears in navigation, browser tabs, and search results.
- **Sidebar title**: Display different text in the sidebar navigation than the page title. Use this when the full title is too long for the sidebar.
- **Description**: A brief summary of the page. Appears beneath the title, in search results, and in SEO meta tags.
- **Icon**: Add a visual marker next to the page title in the sidebar navigation.
- **OG Image URL**: Set a custom preview image for social media shares.
- **External URL**: Link to an external site from the page. Use this to link to external resources from your sidebar navigation.

### Page layout

- **Mode** - Control how the page displays:
  - `default` - Standard page layout with table of contents
  - `wide` - Full-width layout for content that needs more horizontal space
  - `custom` - Set a specific maximum width for the content area
  - `center` - Centered content with constrained width for better readability
  - `frame` - Embed external content in an iframe

### Visibility

- **Hide from navigation** - Remove the page from your site's navigation sidebar. The page remains accessible via direct URL.

### SEO and social

- **Keywords** - Help users find this page in search. Add relevant terms.
- **OpenGraph image** - Set a custom preview image for social media shares.

See [Pages reference](/organize/pages) for complete frontmatter options.

## Managing files

Manage files using right-click actions or drag and drop:

| Action | How to | Result |
|--------|--------|--------|
| **Rename** | Right-click → Rename → Enter new name → <kbd>Enter</kbd> | File paths in navigation and cross-references update automatically |
| **Move** | Drag file to destination folder | File path updates automatically |
| **Duplicate** | Right-click → Duplicate | Creates copy with numbered suffix (use as starting point for similar pages) |
| **Delete** | Right-click → Delete → Confirm | File removed from repository and cannot be recovered |
