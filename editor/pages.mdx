## Navigating files

### File tree

Browse your documentation files in the **Files** tab of the left panel:

- Expand folders to see their contents
- Click a file to open it in the editor
- Use folder icons to expand or collapse directories

### File search

Quickly find files using the file search:

1. Press <kbd>Cmd</kbd> + <kbd>P</kbd> (macOS) or <kbd>Ctrl</kbd> + <kbd>P</kbd> (Windows)
2. Type the file name or path
3. Select the file from the results

### File tree vs navigation tree

- **File tree**: Shows all files in your repository
- **Navigation tree**: Shows pages included in your site's navigation

Not all files appear in navigation. Use the navigation tree to add files to your site's sidebar.

## Creating new pages

Create documentation pages from the file tree:

1. Click the **+** button in the Files tab
2. Select **New file**
3. Choose the file location
4. Enter a file name (`.mdx` extension is added automatically)
5. Press <kbd>Enter</kbd> to create

### File types

Choose the file type when creating files:

- **.mdx**: MDX files with support for React components (default)
- **.md**: Standard Markdown files
- **Other**: Create other file types like JSON or CSS

## Editing content

### Visual mode

Edit content visually with real-time preview:

- **Type directly**: Edit text as it will appear
- **Format text**: Use the toolbar to bold, italicize, and format text
- **Add components**: Press <kbd>/</kbd> to open the component menu
- **Insert links**: Select text and press <kbd>Cmd</kbd>/<kbd>Ctrl</kbd> + <kbd>K</kbd>
- **Add images**: Use the image component from the <kbd>/</kbd> menu

Available components include callouts, code blocks, tabs, accordions, and more. See the [Components documentation](/components) for the complete list.

### Markdown mode

Edit the raw MDX source:

- **Direct MDX editing**: Write MDX and Markdown syntax directly
- **Syntax highlighting**: Code is highlighted for readability
- **Component properties**: Set component props and advanced configurations
- **Frontmatter**: Edit page metadata at the top of the file

See [Format text](/create/text) and [Format code](/create/code) for MDX syntax reference.

### Switching modes

Switch between visual and Markdown mode using the toggle in the toolbar. Your changes are preserved when switching.

## Configure pages

Configure how pages appear in your documentation by opening the page configuration sheet.

**Access page settings:**
- Right-click a file in the file tree or navigation tree
- Select **Configure**

The configuration sheet opens as a side panel. Press <kbd>Enter</kbd> to save changes or <kbd>Escape</kbd> to close without saving.

### Content and navigation

- **Title** - The main heading for your page. Appears in navigation, browser tabs, and search results.
- **Sidebar title** - Show different text in navigation than the page title. Use this when the full title is too long for the sidebar.
- **Description** - Brief summary of the page (1-2 sentences). Shown in search results and used for SEO.
- **Icon** - Add a visual marker next to the page in navigation. Choose from the [Lucide icon library](https://lucide.dev).

### Page layout

- **Mode** - Control how the page displays:
  - `default` - Standard page layout with table of contents
  - `wide` - Full-width layout for content that needs more horizontal space
  - `custom` - Set a specific maximum width for the content area
  - `center` - Centered content with constrained width for better readability
  - `frame` - Embed external content in an iframe

### Visibility

- **Hide from navigation** - Remove the page from your site's navigation sidebar. The page remains accessible via direct URL.

### SEO and social

- **Keywords** - Help users find this page in search. Add relevant terms.
- **OpenGraph image** - Set a custom preview image for social media shares.

See [Pages reference](/organize/pages) for complete frontmatter options.

## Managing files

Manage files using right-click actions or drag and drop:

| Action | How to | Result |
|--------|--------|--------|
| **Rename** | Right-click → Rename → Enter new name → <kbd>Enter</kbd> | File paths in navigation and cross-references update automatically |
| **Move** | Drag file to destination folder | File path updates automatically |
| **Duplicate** | Right-click → Duplicate | Creates copy with numbered suffix (use as starting point for similar pages) |
| **Delete** | Right-click → Delete → Confirm | File removed from repository and cannot be recovered |
