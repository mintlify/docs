Find solutions to common issues you might encounter while using the web editor.

## Publishing problems

<AccordionGroup>
  <Accordion title="Changes not appearing after publishing">
    **Possible causes:**
    - Deployment is still in progress
    - Browser caching issues
    - Build or deployment errors

    **Solutions:**
    1. Check deployment status in your Mintlify Dashboard
    2. Wait 2-3 minutes for deployment to complete  
    3. Hard refresh your browser (<kbd>Ctrl</kbd> + <kbd>F5</kbd> or <kbd>Cmd</kbd> + <kbd>Shift</kbd> + <kbd>R</kbd>)
    4. Clear your browser cache and cookies for your documentation site
    5. Try viewing your site in an incognito/private window
  </Accordion>

  <Accordion title="Permission errors when publishing">
    **Possible causes:**
    - Insufficient permissions to the Git repository
    - Authentication issues with your Git provider
    - Expired or revoked access tokens

    **Solutions:**
    1. Verify you have write access to the repository in GitHub/GitLab
    2. Check if your Git integration is properly configured in settings
    3. Re-authorize the Mintlify app in your Git provider
    4. Contact your repository administrator to verify permissions
    5. Review the [Editor Permissions](/advanced/dashboard/permissions) documentation
  </Accordion>

  <Accordion title="Pull request creation fails">
    **Possible causes:**
    - Branch protection rules preventing direct pushes
    - Conflicts with existing pull requests
    - Repository permissions issues

    **Solutions:**
    1. Check if there's already an open pull request for your branch
    2. Ensure your branch is up to date with the main branch
    3. Verify repository permissions and branch protection settings
    4. Try creating the pull request manually in GitHub/GitLab
  </Accordion>
</AccordionGroup>

## Editor interface issues

<AccordionGroup>
  <Accordion title="Editor loading slowly or not at all">
    **Possible causes:**
    - Network connectivity problems
    - Large documentation repositories
    - Browser compatibility issues

    **Solutions:**
    1. Check your internet connection stability
    2. Try refreshing the page and waiting longer
    3. Clear browser cache and cookies
    4. Try a different browser (Chrome, Firefox, Safari, Edge)
    5. Disable browser extensions that might interfere
    6. Contact support if the issue persists
  </Accordion>

  <Accordion title="Files not loading or showing errors">
    **Possible causes:**
    - Invalid MDX syntax in files
    - Missing or corrupted files
    - Large file sizes causing timeouts

    **Solutions:**
    1. Check the file syntax for MDX formatting errors
    2. Verify the file exists in your repository
    3. Try opening a different file to isolate the issue
    4. Check for files over 1MB that might cause loading issues
    5. View the file directly in GitHub/GitLab to verify content
  </Accordion>

  <Accordion title="Mode switching not working">
    **Possible causes:**
    - Complex MDX content that can't render in visual mode
    - JavaScript errors in the editor
    - Browser compatibility issues

    **Solutions:**
    1. Try refreshing the page and switching modes again
    2. Check browser console for JavaScript errors
    3. Simplify complex MDX content that might be causing issues
    4. Clear browser cache and try again
  </Accordion>
</AccordionGroup>

## Content and formatting issues

<AccordionGroup>
  <Accordion title="Components not rendering correctly">
    **Possible causes:**
    - Invalid component syntax
    - Missing required properties
    - Unsupported component configurations

    **Solutions:**
    1. Check component syntax against the documentation
    2. Verify all required properties are included
    3. Use the `/` menu in visual mode to insert components correctly
    4. Compare with working examples in other files
    5. Switch to Markdown mode to inspect the raw MDX
  </Accordion>

  <Accordion title="Images not displaying">
    **Possible causes:**
    - Incorrect image paths
    - Missing image files
    - Large image file sizes

    **Solutions:**
    1. Verify image files exist in your repository
    2. Check that image paths are correct (relative to your docs root)
    3. Ensure image files are properly committed to your repository
    4. Try using absolute URLs for external images
    5. Optimize large images to reduce file size
  </Accordion>

  <Accordion title="Formatting looks different than expected">
    **Possible causes:**
    - Differences between visual mode and published output
    - Custom CSS overrides
    - Theme-specific styling issues

    **Solutions:**
    1. Preview your changes on the actual documentation site
    2. Check if custom CSS is affecting your content
    3. Compare with similar content that renders correctly
    4. Use Markdown mode for more precise formatting control
  </Accordion>
</AccordionGroup>

## Performance and saving issues

<AccordionGroup>
  <Accordion title="Changes not saving">
    **Possible causes:**
    - Network connectivity issues
    - Repository write permission problems
    - Large content changes causing timeouts

    **Solutions:**
    1. Check your internet connection
    2. Try saving smaller chunks of content
    3. Verify you have repository write permissions
    4. Copy your content to avoid losing work, then refresh and try again
    5. Check browser console for error messages
  </Accordion>

  <Accordion title="Editor running slowly">
    **Possible causes:**
    - Large files or repositories
    - Browser memory issues
    - Multiple tabs or applications running

    **Solutions:**
    1. Close other browser tabs to free up memory
    2. Work with smaller files when possible
    3. Refresh the editor periodically during long editing sessions
    4. Use a computer with adequate RAM and processing power
    5. Consider breaking large files into smaller, focused pages
  </Accordion>
</AccordionGroup>

## Getting additional help

If these solutions don't resolve your issue:

1. **Check the status page**: Look for known issues or outages
2. **Contact support**: Include specific error messages and steps to reproduce
3. **Community forums**: Search for similar issues or ask questions
4. **Documentation**: Review related documentation for additional context

When reporting issues, include:
- Your browser and version
- Steps to reproduce the problem  
- Any error messages displayed
- Screenshots of the issue if applicable
