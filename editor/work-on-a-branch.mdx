Branches let you work on documentation updates without touching your live site. Your live site builds from your deployment branch (usually `main`). Changes on other branches don't go live until you merge them.

## Create a branch

1. Click the branch name in the editor toolbar.
1. Click **Create new branch**.
1. If you have pending changes, choose whether to bring them to the new branch or leave them on the current branch.
1. Enter a name and click **Create branch**.

<Tip>
  Type a new branch name in the search field and press <kbd>Enter</kbd> to create it directly without clicking through the menu.
</Tip>

## Switch branches

1. Click the branch name in the toolbar.
1. Search for or scroll to the branch you want.
1. Click the branch to switch to it.

The branch switcher organizes branches into two sections:

- **Synced in editor**: Branches ready to use immediately.
- **Not synced in editor**: Branches that exist in your repository but haven't loaded in the editor yet. Selecting one starts a sync.

Each branch shows a status badge:

| Badge | Meaning |
|-------|---------|
| **Default** | Your deployment branch |
| **Current** | The branch you are on now |
| **Syncing** | The branch is loading into the editor |
| **Failed** | The branch failed to sync |

<Warning>
  Switching branches while you have unpublished changes will prompt you to bring those changes to the new branch or leave them behind. Changes left behind remain on your original branch.
</Warning>

## After you publish

When you're ready to merge your branch into your deployment branch, click **Create pull request** in the publish popover. Your team reviews it in GitHub or GitLab. Once approved, click **Merge and publish** in the editor to merge and deploy in one step.

For the full set of publish options, see [Branching and publishing](/editor/branching-and-publishing).

## Resolve conflicts

Conflicts occur when your branch and the deployment branch have incompatible changes to the same files — for example, when you and a teammate edited the same lines on different branches.

The editor displays a warning when conflicts block publishing. Follow the prompts to choose which version of each conflicting section to keep.
