## Disable a workflow

1. Go to the [Workflows](https://dashboard.mintlify.com/products/workflows?tab=workflows) page in your dashboard.
2. Click the toggle beside a workflow to disable it.

When you re-enable a scheduled workflow, Mintlify recalculates the next run time from the current time.

## Delete a custom workflow

Deleting a workflow is permanent and removes its configuration and run history.

1. Go to the [Workflows](https://dashboard.mintlify.com/products/workflows?tab=workflows) page in your dashboard.
2. Click the <Icon icon="gear" /> settings button beside a workflow.
3. Click **Delete**.

## View run history

Each workflow keeps a log of past runs, including the status and a summary of changes made.

1. Go to the [Workflows](https://dashboard.mintlify.com/products/workflows) page in your dashboard.
2. Click a workflow.
3. Use the dropdown menus to filter by specific workflows or status.

## Slack notifications

When a workflow opens a pull request, Mintlify can post a notification to Slack so your team can review the change without leaving the workspace.

Each notification includes:

- The workflow name and current status (open, merged, closed, or failed).
- The pull request title, linked to the PR.
- A short summary of the changes the agent made.
- For push-triggered workflows, a link to the pull request in the trigger repository that started the run, labeled with the repository name.

### Review pull requests from Slack

While a workflow pull request is open, the notification includes **Accept** and **Close** buttons. Use them to merge or close the PR directly from Slack without opening the dashboard.

- **Accept** — Merges the pull request using the same automerge rules as the dashboard. The Slack message updates to show the merged status.
- **Close** — Closes the pull request without merging. The Slack message updates to show the closed status.

Only members of the same Mintlify organization can use the buttons. After the PR is merged or closed, the buttons disappear from the notification.

<Note>
  Merging from Slack requires the same Git permissions as merging from the dashboard. For GitHub, the Mintlify GitHub App must have bypass permissions on every ruleset targeting your deploy branch. For GitLab, the connected user must have at least the Maintainer role. See [Configure automerge](/guides/configure-automerge) for details.
</Note>
