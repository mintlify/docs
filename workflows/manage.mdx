## Run a scheduled workflow manually

Run a scheduled workflow on demand without waiting for its next scheduled time. Manual runs are useful for testing prompt changes, backfilling missed runs, or kicking off a one-off task between scheduled executions.

Manual runs are only available for active workflows that use a schedule (cron) trigger. To run a push-triggered workflow, push a change to the configured repository or branch.

1. Go to the [Workflows](https://dashboard.mintlify.com/products/workflows?tab=workflows) page in your dashboard.
2. Click the <Icon icon="gear" /> settings button beside a scheduled workflow.
3. Click **Run now**.

A toast confirms that the run has been queued. Click **View runs** in the toast to open the runs page and track its progress.

<Tip>
  If **Run now** is disabled, the workflow is paused. Re-enable it from the Workflows page, then try again.
</Tip>

## Disable a workflow

1. Go to the [Workflows](https://dashboard.mintlify.com/products/workflows?tab=workflows) page in your dashboard.
2. Click the toggle beside a workflow to disable it.

When you re-enable a scheduled workflow, Mintlify recalculates the next run time from the current time.

## Delete a custom workflow

Deleting a workflow is permanent and removes its configuration and run history.

1. Go to the [Workflows](https://dashboard.mintlify.com/products/workflows?tab=workflows) page in your dashboard.
2. Click the <Icon icon="gear" /> settings button beside a workflow.
3. Click **Delete**.

## View run history

Each workflow keeps a log of past runs, including the status and a summary of changes made.

1. Go to the [Workflows](https://dashboard.mintlify.com/products/workflows) page in your dashboard.
2. Click a workflow.
3. Use the dropdown menus to filter by specific workflows or status.
