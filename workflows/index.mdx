Workflows run the agent automatically on a schedule or in response to repository changes. Each workflow has a prompt and a single trigger, and supports GitHub and GitLab.

When a workflow runs, the agent reads your project content and any connected repositories, then follows the prompt to make updates.

<Tip>
Use scheduled workflows for recurring tasks (changelogs, SEO audits). Use content or code change triggers for reactive maintenance (translations, syncing docs with source code).
</Tip>

## Trigger types

- **Content update** — Runs on pushes and PR merges in your Mintlify project repository.
- **Code change** — Runs when a PR merges in a connected source code repository.
- **Custom schedule** — Runs on a recurring schedule (queues within 10 minutes).

## Usage limits

Workflow runs count toward your credit usage. Manage credits on the [Usage](https://app.mintlify.com/settings/organization/usage) page. Each workflow can run up to 500 times per day; failed runs don't count.
