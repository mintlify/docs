Mintlify includes predefined workflows for the most common documentation maintenance tasks. Each workflow has a base prompt and a default configuration. You can customize the trigger, update mode, and instructions from the [Workflows](https://app.mintlify.com/products/workflows?tab=workflows) page in your dashboard.

## Self-updating content workflows

Self-updating content workflows make updates based on context that you define so your content remains accurate and up to date.

### Sync content with code

Updates your content when source code changes. You must specify at least one source repository to watch for changes.

- **Default trigger**: Code change. Runs when a pull request merges in a source repository
- **Default update mode**: Require review. Always opens a pull request
- **Context repositories**: Not supported
- **Custom instructions**: Supported

<Note>
  This workflow always opens a pull request for review. You cannot set the update mode to automatic.
</Note>

### Changelog

Drafts a changelog entry from recent product updates on a recurring schedule. You can specify a custom schedule or use the default schedule of every Friday at 3 PM UTC.

- **Default trigger**: Custom schedule
- **Default update mode**: Automatic
- **Context repositories**: Supported. Add source code repositories for the agent to reference
- **Custom instructions**: Supported

<Tip>
  Add context repositories pointing to your product's source code so the agent has more detail about what changed.
</Tip>

### Improve content from assistant conversations

Reviews recent [assistant](/assistant/index) question trends and updates content based on information gaps.

- **Default trigger**: Custom schedule. Runs every Monday at 9 AM UTC
- **Default update mode**: Automatic
- **Context repositories**: Not supported
- **Custom instructions**: Supported

### Improve content from user feedback

Reviews recent page [feedback](/optimize/feedback) and updates content based on valid user feedback.

- **Default trigger**: Custom schedule. Runs every Monday at 9 AM UTC
- **Default update mode**: Automatic
- **Context repositories**: Not supported
- **Custom instructions**: Supported

**Required configuration:** You must enable feedback collection on the [Add-ons](https://app.mintlify.com/products/addons) page of your dashboard.

## Maintenance workflows

Maintenance workflows automate routine tasks to improve the quality and consistency of your documentation.

Any enabled maintenance workflows run on self-updating content workflows. If changes are needed, the agent commits the maintenance changes to the self-updating pull request. For example, if you enable the **translate content** workflow, the agent adds translations for any updates made with the **sync content with code** workflow to the same pull request.

### Translate content

Keeps your selected languages in sync whenever content changes. You must specify at least one target language. Mintlify reads the [languages](/organize/navigation#languages) defined in your `docs.json` and preselects any already configured target languages.

To disable translations for a language, you must remove all files in the language directory. You cannot disable translations for a language that still has published content in that language.

- **Default trigger**: Content update. Runs when you push content changes
- **Default update mode**: Automatic
- **Context repositories**: Not supported
- **Custom instructions**: Supported

### Fix broken links

Finds and fixes broken internal and external links whenever content changes.

- **Default trigger**: Content update. Runs when you push content changes
- **Default update mode**: Automatic
- **Context repositories**: Not supported
- **Custom instructions**: Supported

### Audit SEO metadata

Audits titles, meta descriptions, headings, and canonical tags whenever content changes.

- **Default trigger**: Content update. Runs when you push content changes
- **Default update mode**: Automatic
- **Context repositories**: Not supported
- **Custom instructions**: Supported

### Check grammar

Finds and fixes typos, spelling mistakes, and grammar errors whenever content changes.

- **Default trigger**: Content update. Runs when you push content changes
- **Default update mode**: Automatic
- **Context repositories**: Not supported
- **Custom instructions**: Supported

### Enforce writing style

Applies your style guide's voice, tone, and rules to content whenever it changes.

- **Default trigger**: Content update. Runs when you push content changes
- **Default update mode**: Automatic
- **Context repositories**: Not supported
- **Custom instructions**: Supported

<Tip>
  Add instructions to specify the style rules you want the agent to enforce.
</Tip>
