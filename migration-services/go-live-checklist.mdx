---
title: "Go-live checklist"
description: "A checklist to verify your Mintlify site is ready before going live."
---

This is a comprehensive list of configurations to set up and settings to validate before going live. Feel free to skip steps if they're not applicable.

### Core Setup

- **Connect GitHub docs repo to Mintlify.** Install the [GitHub app](/deploy/github). 
  - If the GitHub app is already installed, you will need to uninstall and re-install it. To verify this step works, make a change to the docs locally, push to GitHub, and check that the Mintlify docs site redeploys on the dashboard home page.
  - Determine if you need to configure a [multi-repository structure](/deploy/multi-repo) to build a Mintlify site from multiple repos.
- **Invite team members.** Invite the emails of all team members that will need access to Mintlify in **Settings \> Workspace \> Members**. See the guide to member [permissions](/dashboard/roles).

### Security 

- Review your dashboard access. 
  - Set up [dashboard SSO](https://www.mintlify.com/docs/dashboard/sso) for your team to edit. 
  - Set up [dashboard access policies](https://www.mintlify.com/docs/dashboard/network-access).
  - Set up [SCIM provisioning](https://www.mintlify.com/docs/dashboard/scim). 
- Review your Mintlify site access. 
  - Set up [authentication](https://www.mintlify.com/docs/deploy/authentication-setup). 
  - Review content to ensure that pages are tagged with proper groups or are marked public. 

### Content Validation

- **Verify your information architecture.** Is this the proper navigation structure? Are there missing sections? See [content types](/guides/content-types) for reference.
- [**Set up redirects, if required**](https://www.mintlify.com/docs/create/redirects).

### Dashboard Features

- **Navigate to the Assistant tab.** Turn it on, then [configure assistant deflection and search domains](/assistant/configure) and [add assistant skills](/assistant/skills).
- **Enable relevant add-ons.** Go to **Settings \> Add-ons**. Enable relevant add-ons such as agent feedback, [CI checks](/deploy/ci), related topics, and more.
- **Set up the Slack agent.** This lets you make changes to the docs directly in Slack. See [Add the agent to Slack](/agent/slack#add-the-agent-to-slack).
- **Set up automations.** See [how to enable an automation](/automations/manage#enable-an-automation).
- **Install relevant analytics plugins.** See the [list of analytics integrations](/integrations/analytics/overview).

### Go Live

- **Attach your custom domain.** See the [custom domain guide](/customize/custom-domain).
