[llms.txt 文件](https://llmstxt.org) 是一种行业标准，可帮助 LLM 更高效地索引内容，类似于站点地图之于搜索引擎。AI 工具可以使用该文件理解你的文档结构，并定位与用户查询相关的内容。

Mintlify 会在你的项目根目录自动托管一个 `llms.txt` 文件，用于列出文档中的所有可用页面。该文件始终保持最新且无需维护。你也可以选择在项目根目录添加自定义的 `llms.txt` 文件。

在你的文档站点 URL 末尾追加 `/llms.txt` 以查看 `llms.txt`。

<PreviewButton href="https://mintlify.com/docs/llms.txt">打开此站点的 llms.txt。</PreviewButton>

## llms.txt 结构

`llms.txt` 是一个纯 Markdown 文件，包含：

- 以 H1 作为的**站点标题**。
- **结构化的内容分区**，附带链接以及文档中每个页面的描述。

```mdx Example llms.txt
# Example product docs

## Guides
- [Getting started](https://example.com/docs/start): Intro guide
- [Install](https://example.com/docs/install): Setup steps

## Reference
- [API](https://example.com/docs/api): Endpoint list and usage
```

这种结构化方式便于大模型高效地从全局理解你的文档，并为用户查询定位相关内容，从而提升 AI 辅助文档搜索的准确性与速度。

## llms-full.txt

`llms-full.txt` 文件将你的整个文档站点汇总为一个文件，供 AI 工具用作上下文，并会被大模型流量索引。

Mintlify 会在你的项目根目录自动托管一个 `llms-full.txt` 文件。可在文档站点的 URL 末尾追加 `/llms-full.txt` 来查看你的 `llms-full.txt`。

<PreviewButton href="https://mintlify.com/docs/llms-full.txt">打开本网站的 llms-full.txt</PreviewButton>

## 自定义文件

要添加自定义的 `llms.txt` 或 `llms-full.txt` 文件，请在项目根目录创建对应的 `llms.txt` 或 `llms-full.txt`。添加自定义文件会覆盖同名的自动生成文件。若删除自定义文件，将恢复使用默认文件。

自定义的 `llms.txt` 或 `llms-full.txt` 必须包含以 H1 标题形式呈现的站点标题。其他内容为可选。有关可选部分与最佳实践的更多信息，请参阅 `llms.txt` 规范中的[格式](https://llmstxt.org/#format)。
