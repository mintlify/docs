<Info>
  [Pro 方案](https://mintlify.com/pricing?ref=authentication) 包含密码认证。

  [Enterprise 方案](https://mintlify.com/pricing?ref=authentication) 覆盖所有认证方式。
</Info>

根据用户信息和安全需求，从以下三种方式中选择，用于管理访问并自定义文档：

* **完全认证**：为全部内容提供完整的隐私保护，并支持全面的内容定制。
* **部分认证**：按页面进行访问控制，并支持全面的内容定制。
* **个性化**：可进行内容定制，但**不提供任何安全保证**。所有内容仍向公众公开。

<div id="use-case-examples">
  ### 用例示例
</div>

**在以下情况下选择完全认证：**

* 搭建包含敏感信息的公司内部文档
* 编写需要用户身份验证的专有 API 文档
* 创建面向特定客户的实施指南

**在以下情况下选择部分认证：**

* 提供公开的入门指南，同时将高级功能设为私有
* 发布开源文档，并将企业版部分设为私有

**在以下情况下选择个性化：**

* 创建公开的 API 文档，展示用户特定的示例
* 搭建可基于用户画像定制内容的营销网站
* 提供可根据用户偏好或技能水平自适应的公开教程

<div id="handshake-methods">
  ## 握手方式
</div>

选择最符合你现有基础设施与安全要求的方式。

| 方法 | 适用范围 | 设置复杂度 | 最适合 |
|:-------|:--------------|:-----------------|:----------|
| **JWT（JSON Web Token）** | 所有方案 | 中等 | 自定义登录流程，最大化安全控制 |
| **OAuth 2.0** | 所有方案 | 高 | 第三方认证提供商、企业级安全 |
| **Mintlify 控制台** | 仅限认证 | 低 | 已经使用 Mintlify 控制台的团队 |
| **密码** | 仅限认证 | 低 | 无个性化的简单共享访问 |
| **共享会话** | 仅限个性化 | 低 | 已有会话式认证的应用 |

<div id="when-to-use-each-method">
  ### 何时使用每种方法
</div>

**JWT（JSON Web Token）**：当你已有认证系统并希望完全掌控登录流程时使用。适用于自定义用户管理，或当你需要将文档访问与主应用解耦时。

**OAuth 2.0**：当你想借助第三方认证提供商（Google、GitHub 等）或需要企业级安全标准时使用。最适合已在使用 OAuth 基础设施的组织。

**Mintlify 控制台**：当你的文档编辑者同时也是文档读者时使用。非常适合已通过 Mintlify 控制台管理内容的内部团队。

**密码**：当你不需要跟踪个体用户或个性化内容时，可用于简单的访问控制。适用于外部合作方、内测用户或临时访问场景。

**共享会话**：当你希望在应用与文档之间实现无缝登录时使用。适用于用户已在主应用中完成认证，且你希望个性化其文档体验的情况。

<div id="content-customization">
  ## 内容自定义
</div>

这三种方式都支持内容个性化功能，包括动态 MDX content、API key 预填充和页面可见性控制。有关详细的实现指南，请参见[个性化设置](/zh/authentication-personalization/personalization-setup)。
