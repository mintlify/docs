`<RequestExample>` 和 `<ResponseExample>` 组件会在右侧边栏展示代码块，形成双栏布局，确保用户滚动阅读内容时示例始终可见。这些组件专为 API 文档设计，但适用于所有页面。

常见用法：

- 含请求与响应示例的 API 端点文档
- 与说明性文字并排的配置示例
- 供用户在按照指引操作时参考的代码示例
- 教程中的前后对比示例

在移动设备上，`<RequestExample>` 和 `<ResponseExample>` 组件会以常规代码块显示，可直接滑动越过。

## RequestExample

使用 `<RequestExample>` 将代码示例固定在右侧边栏。该组件的工作方式与 [CodeGroup](/zh/components/code-groups) 组件类似，但会将代码显示在侧边栏中，而非内联展示。

你可以在单个 `<RequestExample>` 中包含多个代码块。每个代码块都必须带有 title 属性。

````mdx RequestExample
<RequestExample>

```bash Request
  curl --request POST \
    --url https://dog-api.kinduff.com/api/facts
```

</RequestExample>
````

## ResponseExample

`<ResponseExample>` 组件会将代码示例固定在右侧边栏，并显示在同一页面中任意 `<RequestExample>` 内容的下方。

````mdx ResponseExample
<ResponseExample>

```json Response
{ "status": "success" }
```

</ResponseExample>
````
