本快速上手指南将演示如何在几分钟内设置并部署你的文档站点。

完成本指南后，你将拥有一个可在线访问的文档站点，随时可进行自定义与扩展。

<Info>
  **前提条件**：开始前，请[创建账户](https://mintlify.com/start)并完成入门流程。
</Info>

## 入门

完成引导流程后，你的文档站点会自动部署到一个唯一的 URL，格式如下：

```
https://<your-project-name>.mintlify.app
```

在你的[控制台](https://dashboard.mintlify.com/)的 Overview 页面找到该 URL。

<Frame>
  <img src="/images/quickstart/mintlify-domain-light.png" alt="Mintlify 域名" className="block dark:hidden" />

  <img src="/images/quickstart/mintlify-domain-dark.png" alt="Mintlify 域名" className="hidden dark:block" />
</Frame>

你的网站 URL 会立即可用。在你设置文档站点期间，使用该 URL 进行测试并与团队分享。

### 安装 GitHub 应用

Mintlify 提供一个 GitHub 应用，可在你将更改推送到代码仓库时自动触发部署。

按照引导清单或控制台中的说明安装 GitHub 应用。

1. 在 Mintlify 控制台进入 **Settings**。
2. 在侧边栏选择 **GitHub App**。
3. 选择 **Install GitHub App**。将会在新标签页打开 GitHub 应用安装页面。
4. 选择你要安装该应用的组织或个人账户。
5. 选择你想要连接的代码仓库。

<Frame>
  <img src="/images/quickstart/github-app-installation-light.png" alt="GitHub 应用安装" className="block dark:hidden" />

  <img src="/images/quickstart/github-app-installation-dark.png" alt="GitHub 应用安装" className="hidden dark:block" />
</Frame>

<Info>
  如果你将文档迁移到其他代码仓库，请更新 GitHub 应用的权限。
</Info>

### 授权你的 GitHub 账户

1. 在 Mintlify 控制台进入 **Settings**。
2. 在侧边栏选择 **My Profile**。
3. 选择 **Authorize GitHub account**。将会在新标签页打开 GitHub 授权页面。

<Info>
  视组织设置而定，你的 GitHub 组织管理员可能需要为你的账户进行授权。
</Info>

## 基于代码的工作流

基于代码的工作流可与您现有的开发环境和 Git 仓库集成。该工作流最适合希望将文档与代码一体化管理的技术团队。

### 安装 CLI

要在本地处理文档，请在终端运行以下命令安装命令行界面（CLI）[mint](https://www.npmjs.com/package/mint)：

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

<Info>
  需要在本机安装 Node.js。如果遇到安装问题，请查看故障排查指南。
</Info>

### 编辑文档

完成环境准备后，即可开始编辑文档文件。例如，更新介绍页面的标题：

1. 打开在入门引导期间创建的仓库。
2. 打开 `index.mdx` 并定位到文件顶部：

```mdx index.mdx
---
title: "Introduction"
description: "This is the introduction to the documentation"
---
```

3. 将 `title` 字段更新为 `"Hello World"`。

```mdx index.mdx {2}
---
title: "Hello World"
description: "This is the introduction to the documentation"
---
```

### 预览更改

要在本地预览更改，请运行以下命令：

```bash
mint dev
```

预览可在 `localhost:3000` 访问。

  <img src="/images/quickstart/mintlify-dev-light.png" alt="Mintlify Dev" className="block dark:hidden" />

### 推送更改

准备发布更改时，将其推送到仓库。

Mintlify 会自动检测更改、构建文档，并将更新部署到站点。可在 GitHub 仓库的提交历史或[控制台](https://dashboard.mintlify.com)中查看部署状态。

部署完成后，最新更新将可在 `<your-project-name>.mintlify.app` 访问。

<Card title="跳转到添加自定义域名" icon="arrow-down" href="zh/#adding-a-custom-domain" horizontal>
  也可以选择跳过网页编辑器工作流，直接跳转到添加自定义域名。
</Card>

## 网页编辑器（Web editor）工作流

网页编辑器工作流提供所见即所得（WYSIWYG）界面，用于创建和编辑文档。该工作流适合希望直接在浏览器中工作、无需额外本地开发工具的用户。

### 访问网页编辑器

1. 登录你的[控制台](https://dashboard.mintlify.com)。
2. 在左侧边栏选择 **Editor**。

<Info>
  如果你尚未安装 GitHub App，打开网页编辑器时会提示你安装该应用。
</Info>

<Frame>
  <img alt="Mintlify 网页编辑器的可视化编辑模式" src="/images/quickstart/web-editor-light.png" className="block dark:hidden" />

  <img alt="Mintlify 网页编辑器的可视化编辑模式" src="/images/quickstart/web-editor-dark.png" className="hidden dark:block" />
</Frame>

### 编辑文档

在网页编辑器中，你可以通过侧边栏浏览文档文件。让我们更新介绍页：

在文件浏览器中找到并选择 `index.mdx`。

然后在编辑器中，将标题字段更新为“Hello World”。

<Frame>
  <img alt="在网页编辑器中进行编辑" src="/images/quickstart/web-editor-editing-light.png" className="block dark:hidden" />

  <img alt="在网页编辑器中进行编辑" src="/images/quickstart/web-editor-editing-dark.png" className="hidden dark:block" />
</Frame>

<Tip>
  编辑器提供丰富的格式化工具和组件。在编辑器中输入 <kbd>/</kbd> 可打开命令菜单以使用这些工具。
</Tip>

### 发布你的更改

当你对编辑结果满意时，点击右上角的 **Publish** 按钮。你的更改会立即部署到文档站点。

<Tip>
  使用分支，通过拉取请求在部署到线上站点前预览并评审更改。
</Tip>

有关网页编辑器的更多用法（包括使用分支和拉取请求进行协作与预览更改），请参见我们的[网页编辑器文档](/zh/editor)。

## 添加自定义域名

虽然你的 `<your-project-name>.mintlify.app` 子域适合用于测试和开发，但多数团队更倾向于在生产环境的文档中使用自定义域名。

要添加自定义域名，请在仪表盘中前往 [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain) 页面。

<Frame>
  <img src="/images/quickstart/custom-domain-light.png" alt="Custom Domain" className="block dark:hidden" />

  <img src="/images/quickstart/custom-domain-dark.png" alt="Custom Domain" className="hidden dark:block" />
</Frame>

输入你的域名（例如 `docs.yourcompany.com`），并按照页面上的指引在域名服务商处配置 DNS。

<Table>
  | Record Type | Name | Value | TTL |
  |-------------|------|-------|-----|
  | CNAME | docs（或子域名） | cname.mintlify.app | 3600 |
</Table>

<Info>
  DNS 更改可能需要最长 48 小时传播生效，但通常会更快完成。
</Info>

## 故障排查

如果你在设置过程中遇到问题，请查看以下常见解决方案：

<AccordionGroup>
  <Accordion title="本地预览无法运行">
    请确保已安装 Node.js v19+，并在包含 `docs.json` 文件的目录中运行 `mint dev` 命令。
  </Accordion>

  <Accordion title="更改未在线上站点生效">
    部署可能需要几分钟。请检查你的 GitHub Actions（代码工作流）或在 Mintlify 控制台查看部署日志，确认没有构建错误。
  </Accordion>

  <Accordion title="自定义域名无法连接">
    确认你的 DNS 记录配置正确，并预留足够时间等待 DNS 传播（最长 48 小时）。你可以使用 [DNSChecker](https://dnschecker.org) 等工具验证 CNAME 记录。
  </Accordion>
</AccordionGroup>
