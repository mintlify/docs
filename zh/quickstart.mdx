本快速上手指南将演示如何在几分钟内设置并部署你的文档站点。

完成本指南后，你将拥有一个已上线的文档站点，可随时进行自定义和扩展。

<Info>
  **前提条件**：开始之前，请[创建账户](https://mintlify.com/start)并完成初始设置。
</Info>

<div id="getting-started">
  ## 入门
</div>

完成引导流程后，你的文档站点会自动部署到一个唯一的 URL，格式如下：

```
https://<your-project-name>.mintlify.app
```

在你的[仪表盘](https://dashboard.mintlify.com/)的概览页找到你的 URL。

<Frame>
  <img src="/images/quickstart/mintlify-domain-light.png" alt="Mintlify Domain" className="block dark:hidden" />

  <img src="/images/quickstart/mintlify-domain-dark.png" alt="Mintlify Domain" className="hidden dark:block" />
</Frame>

你的网站 URL 会立即可用。在搭建文档站点期间，使用此 URL 进行测试并与团队共享。

<div id="install-the-github-app">
  ### 安装 GitHub 应用
</div>

Mintlify 提供了一个 GitHub 应用，可在你向仓库推送更改时自动部署。

按照引导清单或仪表盘中的说明安装 GitHub 应用。

1. 在 Mintlify 仪表盘前往**设置**。
2. 在侧边栏选择 **GitHub App**。
3. 选择**安装 GitHub 应用**。这会在新标签页打开 GitHub 应用安装页面。
4. 选择你希望安装该应用的组织或用户账户。
5. 选择要连接的代码仓库。

<Frame>
  <img src="/images/quickstart/github-app-installation-light.png" alt="GitHub App Installation" className="block dark:hidden" />

  <img src="/images/quickstart/github-app-installation-dark.png" alt="GitHub App Installation" className="hidden dark:block" />
</Frame>

<Info>
  如果你将文档迁移到其他代码仓库，请更新 GitHub 应用的权限。
</Info>

<div id="authorize-your-github-account">
  ### 授权你的 GitHub 账号
</div>

1. 在 Mintlify 仪表盘前往**设置**。
2. 在侧边栏选择**我的资料**。
3. 选择**授权 GitHub 账号**。这会在新标签页打开 GitHub 授权页面。

<Info>
  根据你的组织设置，GitHub 组织管理员可能需要为你的账号进行授权。
</Info>

<div id="code-based-workflow">
  ## 基于代码的工作流
</div>

基于代码的工作流可与您现有的开发环境和 Git 仓库集成。该工作流最适合希望将文档与代码一并管理的技术团队。

<div id="install-the-cli">
  ### 安装命令行工具
</div>

要在本地维护文档，请在终端运行以下命令安装命令行界面（CLI）[mint](https://www.npmjs.com/package/mint)：

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

<Info>
  需要在本机安装 Node.js。若遇到安装问题，请参阅故障排除指南。
</Info>

<div id="edit-the-documentation">
  ### 编辑文档
</div>

完成环境设置后，您即可开始编辑文档文件。例如，更新介绍页的标题：

1. 打开您在引导创建的仓库。
2. 打开 `index.mdx`，定位到文件顶部：

```mdx index.mdx
---
title: "Introduction"
description: "This is the introduction to the documentation"
---
```

3. 将 `title` 字段更新为 `"Hello World"`。

```mdx index.mdx {2}
---
title: "Hello World"
description: "This is the introduction to the documentation"
---
```

<div id="preview-the-changes">
  ### 预览更改
</div>

要在本地预览更改，运行：

```bash
mint dev
```

预览将可通过 `localhost:3000` 访问。

  <img src="/images/quickstart/mintlify-dev-dark.png" alt="Mintlify Dev" className="hidden dark:block" />

<div id="push-the-changes">
  ### 推送更改
</div>

准备发布时，将更改推送到仓库。

Mintlify 会自动检测更改，构建文档，并将更新部署到您的站点。可在 GitHub 仓库的提交历史或[控制台](https://dashboard.mintlify.com)中查看部署状态。

部署完成后，最新更新将通过 `<your-project-name>.mintlify.app` 访问。

<Card title="跳转到添加自定义域名" icon="arrow-down" href="#adding-a-custom-domain" horizontal>
  您也可以跳过网页编辑器工作流，直接前往添加自定义域名。
</Card>

<div id="web-editor-workflow">
  ## 网页编辑器工作流
</div>

网页编辑器工作流提供所见即所得（WYSIWYG）界面，用于创建和编辑文档。该工作流最适合希望直接在浏览器中工作、无需额外本地开发工具的用户。

<div id="access-the-web-editor">
  ### 访问网页编辑器
</div>

1. 登录你的[控制台](https://dashboard.mintlify.com)。
2. 在左侧边栏选择**Editor**。

<Info>
  如果你尚未安装 GitHub 应用，打开网页编辑器时会提示你进行安装。
</Info>

  <img alt="Mintlify 网页编辑器的可视化编辑模式" src="/images/quickstart/web-editor-light.png" className="block dark:hidden" />

<div id="edit-the-documentation">
  ### 编辑文档
</div>

在网页编辑器中，你可以在侧边栏中浏览文档文件。让我们更新介绍页面：

在文件浏览器中找到并选择 `index.mdx`。

然后在编辑器中，将标题字段更新为“Hello World”。

<Frame>
  <img alt="在网页编辑器中编辑" src="/images/quickstart/web-editor-editing-light.png" className="block dark:hidden" />

  <img alt="在网页编辑器中编辑" src="/images/quickstart/web-editor-editing-dark.png" className="hidden dark:block" />

<Tip>
  编辑器提供丰富的格式化工具和组件。在编辑器中输入 <kbd>/</kbd> 可打开命令菜单并访问这些工具。
</Tip>

<div id="publish-your-changes">
  ### 发布你的更改
</div>

当你对修改满意后，点击右上角的**Publish**按钮。你的更改会立即部署到文档站点。

<Tip>
  使用分支通过拉取请求在部署到线上站点前预览和审核更改。
</Tip>

有关使用网页编辑器的更多详情，包括使用分支和拉取请求进行协作与预览更改，请参阅我们的[网页编辑器文档](/zh/editor)。

<div id="adding-a-custom-domain">
  ## 添加自定义域名
</div>

虽然 `<your-project-name>.mintlify.app` 子域名适合测试和开发，但大多数团队更愿意在生产环境的文档中使用自定义域名。

要添加自定义域名，请在控制台前往 [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain) 页面。

<Frame>
  <img src="/images/quickstart/custom-domain-light.png" alt="Custom Domain" className="block dark:hidden" />

  <img src="/images/quickstart/custom-domain-dark.png" alt="Custom Domain" className="hidden dark:block" />
</Frame>

输入你的域名（例如 `docs.yourcompany.com`），并按页面指引在域名服务商处配置 DNS 设置。

<Table>
  | 记录类型 | 名称 | 值 | TTL |
  |-------------|------|-------|-----|
  | CNAME | docs（或子域名） | cname.mintlify.app | 3600 |
</Table>

<Info>
  DNS 变更最多可能需要 48 小时传播生效，但通常会更快完成。
</Info>

<div id="troubleshooting">
  ## 故障排除
</div>

如果在设置过程中遇到问题，请查看以下常见解决方案：

<AccordionGroup>
  <Accordion title="本地预览无法运行">
    请确保已安装 Node.js v19+，并在包含 `docs.json` 文件的目录下运行 `mint dev` 命令。
  </Accordion>

  <Accordion title="变更未在线上站点生效">
    部署可能需要几分钟。请在 GitHub Actions（代码驱动的工作流）或 Mintlify 控制台的部署日志中检查是否存在构建错误。
  </Accordion>

  <Accordion title="自定义域名无法连接">
    请确认 DNS 记录配置正确，并留出足够时间等待 DNS 传播（最长可达 48 小时）。你可以使用 [DNSChecker](https://dnschecker.org) 等工具验证 CNAME 记录。
  </Accordion>
</AccordionGroup>
