## 介绍

网页版编辑器是在浏览器中直接创建、编辑和审阅文档的可视化界面。

- **可视化编辑**：使用所见即所得（WYSIWYG）编辑器修改文档，实时看到发布后的呈现效果。
- **Git 同步**：所有更改会自动与您的 Git 仓库同步，确保版本控制。
- **实时协作**：多位团队成员可同时编辑文档，变更对所有人可见。
- **零配置**：可从仪表盘立即开始写作。

### 网页版编辑器流程

以下是在网页版编辑器中的典型工作方式：

<Steps>
  <Step title="Choose your branch">
    创建一个分支，或直接在部署分支上进行更改。
  </Step>
  <Step title="Open your file">
    打开现有文件或新建一个文件。
  </Step>
  <Step title="Edit your content">
    在网页版编辑器中使用可视化模式或 Markdown 模式进行编辑。
  </Step>
  <Step title="Preview your changes">
    在可视化模式下预览更改后的呈现效果。
  </Step>
  <Step title="Publish your changes">
    如果您在部署分支上工作，可直接在网页版编辑器中发布更改；在其他分支上，请通过 Pull Request 发布更改。
  </Step>
</Steps>

## Git 基础

网页版编辑器在后台执行 Git 操作。理解这些概念将帮助你更高效地使用网页版编辑器，并与在本地环境中工作的团队成员协作。

<AccordionGroup>
  <Accordion title="Repository">
    存放你文档源文件及其历史的仓库。网页版编辑器会连接到你的仓库以访问和修改内容。
  </Accordion>

  <Accordion title="Commit">
    在特定时间点保存的更改快照。
  </Accordion>

  <Accordion title="Branch">
    用于在不影响线上文档的情况下进行更改的独立工作区。可以将其视作用于试验和较大更新的安全沙盒。
  </Accordion>

  <Accordion title="Deployment branch">
    包含线上文档内容的主分支。对此分支的更改会自动发布到你的文档站点。通常称为 `main`。
  </Accordion>

  <Accordion title="Pull request">
    将你分支中的更改合并到线上文档的一种提议方式。允许在更改上线前进行审查和讨论。通常称为 PR。
  </Accordion>

  <Accordion title="Diff">
    diff（差异）展示同一文件两个版本之间的具体更改。在审查 pull request 时，diff 会突出显示新增、删除或修改的内容。
</Accordion>
</AccordionGroup>

### 网页版编辑器会自动完成的操作

网页版编辑器通过我们的 [GitHub App](/zh/settings/github) 或 [GitLab integration](/zh/settings/gitlab) 连接到你的 Git 仓库，并自动处理 Git。当你：

- **打开文件**：从你的仓库获取最新版本。
- **在文件中进行更改**：将更改跟踪为可转换为 commit 的草稿。
- **保存更改**：创建包含你更改的 commit。
- **创建分支**：在你的仓库中创建一个新分支。
- **在部署分支上发布**：创建一个 commit 并直接推送到部署分支。
- **在其他分支上发布**：创建一个 commit 并打开一个 pull request。

### 何时使用分支

分支让你在不影响线上站点内容的情况下进行更改。当更改准备就绪时，你可以通过 pull request 将其合并到部署分支。

**最佳实践：始终从分支开展工作。** 这能确保你的线上文档保持稳定，并支持规范的审查流程。

## 创建分支

1. 在编辑器工具栏中选择分支名称（通常默认为 `main`）。
2. 选择 **New Branch**。
3. 为分支输入一个具有描述性的名称，例如 `update-getting-started` 或 `fix-installation-steps`。
4. 选择 **Create Branch**。

<Note>
  如果编辑器未自动切换到新分支，可能需要在下拉菜单中手动选择该分支。
</Note>

### 在分支上保存更改

要在分支上保存更改，请点击编辑器右上角的 **Save Changes** 按钮。系统会创建一个包含更改的提交，并将其推送到该分支。

### 切换分支

1. 在编辑器工具栏中选择当前分支名称。
2. 在下拉菜单中选择要切换到的分支。

<Tip>
  切换分支时，任何未保存的更改都会丢失。请在切换前保存或发布你的工作。
</Tip>

## 进行更改

在网页版编辑器中编辑现有内容、创建新页面，并组织你的网站结构。

### 浏览文件

使用侧边栏文件资源管理器浏览文档，或按 <kbd>Command</kbd> + <kbd>P</kbd>（Windows 为 <kbd>Ctrl</kbd> + <kbd>P</kbd>）搜索文件。

1. 在文件资源管理器中定位到你的 `docs.json（站点主配置文件）`。
1. 更新 `navigation` 属性以获得所需的导航结构。更多信息参见 [Navigation](/zh/navigation)。

```json title="Adding a Themes page" highlight="18"
                    "installation"
                ]
            },

                "group": "Profile",
                "pages": [
                    "settings",
                    "account-types",
                    "dashboard",
                    "themes"
                ]
            }
        ]
    }
}
```

## 发布更改

点击 **Publish** 按钮以保存更改并使其生效。

更改的发布方式取决于你当前所在的分支：

* **Deployment branch**：立即更新你的线上站点。
* **Other branches**：创建一个 pull request，便于你在上线前审阅更改。

## 拉取请求与变更审阅

拉取请求允许你从分支发起变更提议，以便在合并到线上文档前由他人进行审阅。这有助于确保变更准确，并为团队内容协作创造机会。

<Tip>
  即使是单人作业，拉取请求也有助于在上线前预览变更，并维护清晰的更新历史。
</Tip>

### 在分支上创建拉取请求 

当你准备从分支发布变更时：

1. 确保所有更改都已保存到该分支。
1. 在编辑器右上角选择 **Publish Pull Request**。
1. 为拉取请求添加标题和描述。好的标题和描述有助于审阅者理解你所做的更改。
1. 选择 **Publish Pull Request**。

网页版编辑器会在你的 Git 仓库中创建一个拉取请求，并提供查看链接。

### 审阅拉取请求

创建拉取请求后：

1. **审阅你的更改**：你和团队成员可以在 GitHub、GitLab 等 Git 提供商中审阅你的拉取请求。
1. **继续修改**：审阅后在网页版编辑器中进行必要更改。保存后会更新该拉取请求。
1. **准备就绪后合并**：当拉取请求准备就绪时，将其合并，以把变更部署到线上文档站点。

## 疑难解答

以下是使用网页版编辑器时可能遇到的常见问题及对应解决方案。

<AccordionGroup>
  <Accordion title="发布后更改未显示">
    **可能原因：**

    - 部署仍在进行
    - 浏览器缓存导致的显示延迟

    **解决方案：**

    1. 在 Mintlify Dashboard 查看部署状态。
    2. 尝试强制刷新浏览器（<kbd>Ctrl</kbd> + <kbd>F5</kbd> 或 <kbd>Cmd</kbd> + <kbd>Shift</kbd> + <kbd>R</kbd>）。
    3. 清除浏览器缓存。
  </Accordion>
  <Accordion title="发布时出现权限错误">
    **可能原因：**

    - 对 Git 仓库的权限不足
    - 与 Git 提供商的身份验证问题

    **解决方案：**

    1. 确认你对该仓库具有正确的访问权限。
    2. 检查你的 Git 集成是否已正确配置。
    3. 查看 [编辑器权限](/zh/advanced/dashboard/permissions) 文档。
  </Accordion>
  <Accordion title="编辑器加载问题">
    **可能原因：**

    - 网络连接异常
    - 文档仓库体量较大

    **解决方案：**

    1. 检查你的网络连接。
    2. 刷新页面后重试。
    3. 若问题仍然存在，请联系支持。
  </Accordion>
</AccordionGroup>
