当你更改 docs 文件夹中文件的路径时，对应页面的 URL 路径也会随之更改。这可能发生在你重构文档或修改侧边栏标题时。

## 失效链接

使用我们的命令行工具检测失效链接。[安装命令行工具](/zh/installation)并运行以下命令：

```bash
mint broken-links
```

命令行工具会识别你文档中所有指向不存在目标的相对链接。

## 重定向

在你的 `docs.json` 文件中添加 `redirects` 字段以设置 301 重定向。

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

这会将 `/source/path` 永久重定向到 `/destination/path`，从而确保不会丢失原页面的既有 SEO 积累。

要匹配通配符路径，请在参数后添加 `*`。在此示例中，`/beta/:slug*` 会匹配 `/beta/introduction` 并将其重定向到 `/v2/introduction`。

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
