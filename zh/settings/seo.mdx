Mintlify 会自动处理众多 SEO 最佳实践，包括：

* 元标签生成
* 站点地图与 `robots.txt` 文件生成
* 语义化 HTML 结构
* 移动端优化

你可以在 `docs.json` 或页面的 frontmatter 中添加 `metatags` 字段，完全自定义站点的元标签。

<div id="global-meta-tags">
  ## 全局元标签
</div>

你可以在 `docs.json` 中添加 `metadata` 字段来设置默认的元标签。

```json
"seo": {
    "metatags": {
        "og:image": "默认元标签图片的链接"
    }
}
```

<div id="page-specific-meta-tags">
  ## 页面级元标签
</div>

将元标签按如下方式添加到页面元数据中：

```mdx
---
title: '示例页面标题'
'og:image': '用于元标签图片的链接'
---
```

<Warning>
  在页面元数据中使用时，包含冒号的 Meta 标签必须用引号括起来。
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## 站点地图与 robots.txt 文件
</div>

Mintlify 会自动生成 `sitemap.xml` 和 `robots.txt` 文件。你可以在文档站点的 URL 末尾追加 `/sitemap.xml` 来查看站点地图。

默认只包含 `docs.json` 中的页面。要包含隐藏链接，请在 `docs.json` 中添加 `seo.indexing`：

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### 自定义站点地图和 robots.txt 文件
</div>

要添加自定义的 `sitemap.xml` 或 `robots.txt` 文件，请在项目根目录创建对应的 `sitemap.xml` 或 `robots.txt`。添加自定义文件会覆盖同名的自动生成文件。删除自定义文件后，将恢复使用默认文件。

<div id="disabling-indexing">
  ## 禁用索引
</div>

如果你想让某个页面不被搜索引擎收录，可以在该页面的[frontmatter](zh/pages)中添加以下内容：

```
---
noindex: true
---
```

你也可以在 `docs.json` 中将 `metadata.robots` 字段设置为 `"noindex"`，以为文档中的所有页面指定 `noindex`：

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## SEO 最佳实践
</div>

<AccordionGroup>
  <Accordion title="撰写清晰且具描述性的标题与摘要">
    * 使用清晰、具描述性的页面标题（50–60 个字符）
    * 撰写有吸引力的页面摘要（150–160 个字符）
    * 包含相关关键词
    * 确保每个页面的标题和摘要唯一
  </Accordion>

  <Accordion title="优化内容结构">
    * 使用正确的标题层级（H1 → H2 → H3）
    * 先面向读者写作，其次考虑搜索引擎
    * 在标题与正文中包含相关关键词
    * 保持 URL 简短、具描述性，并按层级组织
    * 使用小标题和列表拆分长篇内容
  </Accordion>

  <Accordion title="内部链接策略">
    * 在文档中链接到相关页面
    * 使用具描述性的锚文本，而非“点击此处”
    * 通过互链相关概念构建主题集群
    * 使用自动交叉引用功能
  </Accordion>

  <Accordion title="图片 SEO">
    * 为图片使用具描述性的文件名
    * 始终提供用于无障碍与 SEO 的 alt 文本
    * 优化图片文件大小以提升加载速度
    * 使用与内容相关、能起到支撑作用的图片
  </Accordion>
</AccordionGroup>
