Mintlify 会自动处理许多 SEO 最佳实践，包括：

- 元标签生成
- 站点地图和 `robots.txt` 文件生成
- 语义化 HTML 结构
- 移动端优化

你可以在 `docs.json` 或页面的 frontmatter 中添加 `metatags` 字段，来自定义站点的元标签。

## 全局元标签

你可以在 `docs.json` 中添加 `metadata` 字段来设置默认元标签。

```json
"seo": {
    "metatags": {
        "og:image": "指向默认元标签图片的链接"
    }
}
```

## 页面特定的元标签

按如下方式将元标签添加到页面的元数据中：

```mdx
---
title: 'Your Example Page Title'
'og:image': 'link to your meta tag image'
---
```

<Warning>
  将元标签放入页面元数据时，带有冒号的键名需要用引号括起来。
</Warning>

## 站点地图与 robots.txt 文件

Mintlify 会自动生成 `sitemap.xml` 和 `robots.txt` 文件。你可以在文档站点的 URL 后追加 `/sitemap.xml` 来查看站点地图。

默认情况下，仅包含在 `docs.json` 中的页面会被收录。若要收录隐藏链接，请在 `docs.json` 中添加 `seo.indexing`：

```json
"seo": {
    "indexing": all
}
```

### 自定义站点地图与 robots.txt 文件

要添加自定义的 `sitemap.xml` 或 `robots.txt`，请在项目根目录创建同名文件。添加自定义文件将覆盖对应的自动生成文件。删除自定义文件后，将恢复使用默认文件。

## 禁用收录

如果你想让某个页面不被搜索引擎收录，可以在页面的[前置元数据](zh/pages)中加入以下内容：

```
---
noindex: true
---
```

你也可以在 `docs.json` 中将 `metadata.robots` 字段设置为 `"noindex"`，为文档中的所有页面统一指定 `noindex`：

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

## SEO 最佳实践

<AccordionGroup>
  <Accordion title="撰写具有描述性的标题和描述">
    - 使用清晰、具有描述性的页面标题（50–60 个字符）
    - 撰写有吸引力的页面描述（150–160 个字符）
    - 包含相关关键词
    - 确保每个页面的标题和描述唯一
  </Accordion>

  <Accordion title="优化内容结构">
    - 使用正确的标题层级（H1 → H2 → H3）
    - 先为人而写，再考虑搜索引擎
    - 在标题和正文中包含相关关键词
    - 保持 URL 简短、可读，并按层级组织
    - 用小标题和列表拆分长篇内容
  </Accordion>

  <Accordion title="内部链接策略">
    - 在文档内链接到相关页面
    - 使用具有描述性的锚文本，避免使用“点击这里”
    - 通过链接相关概念创建主题集群
    - 利用自动交叉引用功能
  </Accordion>

  <Accordion title="图片 SEO">
    - 为图片使用具有描述性的文件名
    - 始终添加用于无障碍与 SEO 的 alt 文本
    - 优化图片体积以加快加载速度
    - 使用与内容相关、能强化信息的图片
  </Accordion>
</AccordionGroup>
