<Info>
  [Pro 和 Enterprise 方案](https://mintlify.com/pricing?ref=docs-ci)包含对 GitHub 仓库的 CI 检查。
</Info>

使用 CI 检查对文档进行问题排查，并在部署前发出警告。CI 检查会在你配置的部署分支的提交时运行，或在针对该分支的拉取请求中运行。

## 安装

首先，按照 [GitHub](/zh/settings/github) 页面上的步骤进行。

对于 GitHub 应用，你可以选择仅为单个仓库授予权限。我们强烈建议这样做，因为我们只需要访问托管你文档的那个仓库。

## 配置

前往仪表盘的 [Add-ons](https://dashboard.mintlify.com/products/addons) 页面，配置用于某次部署的 CI 检查。启用你希望运行的检查项。

启用检查时，你可以选择将其设置为 `Warning` 或 `Blocking` 级别。

- `Warning` 级别的检查即使出现错误或给出建议，也不会标记为失败。
- `Blocking` 级别的检查如未通过，或提出了更改建议，则会标记为失败。

## 可用的 CI 检查

### 失效链接

类似于你在本地使用的[命令行链接检查器](/zh/settings/broken-links#broken-links)，我们会自动检查文档中的失效链接。

要查看检查结果，请前往 GitHub 上该提交的检查结果页面。

### Vale

[Vale](https://vale.sh/) 是一个开源的、基于规则的文本风格校验工具，支持包括 Markdown 和 MDX 在内的多种文档类型。

Mintlify 支持在 CI 检查中自动运行 Vale，并将结果显示为检查状态。

#### 配置
如果你的部署的根内容目录中包含 `.vale.ini` 文件，我们会自动使用该配置文件。我们也会自动使用你在 `stylesPath` 中指定的任何配置文件。

如果你没有 Vale 配置文件，将会自动加载默认配置。

````mdx Default vale.ini configuration expandable
# Top level styles
StylesPath = /app/styles
MinAlertLevel = suggestion
IgnoredScopes = code, tt, img, url, a
SkippedScopes = script, style, pre, figure, code

# Vocabularies
Vocab = Mintlify

# This is required since Vale doesn't officially support MDX
[formats]
mdx = md

# MDX support
[*.mdx]
BasedOnStyles = Vale
Vale.Terms = NO # Enforces really harsh capitalization rules, keep off

# `import ...`, `export ...`
# `<Component ... />`
# `<Component>...</Component>`
# `{ ... }`
TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

# Exclude:
# `<Component \n ... />`
BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})

CommentDelimiters = {/*, */}
````

<Warning>
请注意，出于安全原因，我们不支持任何绝对路径的 `stylesPath`，或包含 `..` 的 `stylesPath`。请使用相对路径，并将 `stylesPath` 提交到你的代码库中。
</Warning>

#### Packages
Vale 支持一系列[软件包](https://vale.sh/docs/keys/packages)，可用于检查拼写和风格错误。
在代码库中正确的 `stylesPath` 下包含的任何软件包都会被自动安装，并用于你的 Vale 配置。

对于未包含在代码库中的软件包，你可以指定来自 [Vale package registry](https://vale.sh/explorer) 的任意软件包，它们会被自动下载并用于你的 Vale 配置。

<Warning>
请注意，出于安全原因，我们不支持自动下载不来自 [Vale package registry](https://vale.sh/explorer) 的软件包。
</Warning>

#### 借助 `MDX` 使用 Vale
Vale 并不原生支持 `MDX`，但 Vale 的作者提供了一个[自定义扩展](https://github.com/errata-ai/MDX)以实现支持。

如果你不想使用该扩展，建议在 `.vale.ini` 文件中添加以下内容：
```ini
[formats]
mdx = md

[*.mdx]
CommentDelimiters = {/*, */}

TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})
```

要在文档中使用 Vale 的文内注释，请使用 MDX 风格的注释 `{/* ... */}`。如果你在配置中启用 `CommentDelimiters = {/*, */}` 这个[设置](https://vale.sh/docs/keys/commentdelimiters)，Vale 会在进行 lint 时自动识别这些注释。也就是说，你可以轻松使用 Vale 的内置功能，例如跳过某些行或段落。

```mdx
{/* vale off */}

This text will be ignored by Vale

{/* vale on */}
```

如果你不使用 `CommentDelimiters`，但仍想使用 Vale 的注释，则必须把所有 Vale 注释放在 MDX 注释 `{/* ... */}` 中。例如：

```mdx
{/* <!-- vale off --> */}

This text will be ignored by Vale

{/* <!-- vale on --> */}
```
这些注释标签在 Mintlify 组件中不受支持，但可在文档的顶层任意位置使用。
