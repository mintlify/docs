`docs.json` 中的 [navigation](zh/settings#param-navigation) 属性用于控制文档的结构与信息层级。

通过合理配置导航，您可以组织内容，帮助用户精准找到所需信息。

<Info>
  请不要将任何导航元素的标题设为 `api` 或 `mcp`。在生产环境中，`/api` 与 `/mcp` 路径为保留路径；如果页面 URL 包含 `/api` 或 `/mcp`，将返回 404 错误。
</Info>

## Pages

Pages 是最基本的导航组件。Pages 对应于构成文档的 扩展的 Markdown（含 React 组件） 文件。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/pages-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/pages-dark.png" alt="" />

在 `navigation` 对象中，`pages` 是一个数组，其中每个条目都必须引用一个[页面文件](zh/pages)的路径。

```json
{
  "navigation": {
    "pages": [
      "settings",
      "pages",
      "navigation",
      "themes",
      "custom-domain"
    ]
  }
}
```

## 分组

使用分组将侧边栏导航组织为多个部分。分组可以相互嵌套、添加标签，并配合图标展示样式。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/groups-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/groups-dark.png" alt="" />

在 `navigation` 对象中，`groups` 是一个数组，其中每个条目都是一个对象，且必须包含 `group` 字段和 `pages` 字段。`icon` 和 `tag` 字段是可选的。

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Getting started",
        "icon": "play",
        "pages": [
          "quickstart",
          {
            "group": "Editing",
            "icon": "pencil",
            "pages": [
              "installation",
              "editor"
            ]
          }
        ]
      },
      {
        "group": "Writing content",
        "icon": "notebook-text",
        "tag": "NEW",
        "pages": [
          "writing-content/page",
          "writing-content/text"
        ]
      }
    ]
  }
}
```

## 标签页

标签页为文档创建彼此独立的部分，并对应各自的 URL 路径。它们会在文档顶部生成一条水平导航栏，方便用户在各部分之间切换。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/tabs-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/tabs-dark.png" alt="" />

在 `navigation` 对象中，`tabs` 是一个数组，其中每一项都是一个对象，必须包含 `tab` 字段，并可包含其他导航字段，如分组、页面、图标或外部链接。

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "tab": "SDKs",
        "icon": "code",
        "pages": [
          "sdk/fetch",
          "sdk/create",
          "sdk/delete"
        ]
      },
      {
        "tab": "Blog",
        "icon": "newspaper",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

### 菜单

菜单为标签页增加下拉导航项。使用菜单可帮助用户直接访问该标签页内的特定页面。

在 `navigation` 对象中，`menu` 是一个数组，其中每一项都是一个对象，必须包含 `item` 字段，并可包含其他导航字段，如分组、页面、图标或外部链接。

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Developer tools",
        "icon": "square-terminal",
        "menu": [
          {
            "item": "API reference",
            "icon": "rocket",
            "groups": [
              {
                "group": "Core endpoints",
                "icon": "square-terminal",
                "pages": [
                  "api-reference/get",
                  "api-reference/post",
                  "api-reference/delete"
                ]
              }
            ]
          },
          {
            "item": "SDKs",
            "icon": "code",
            "description": "SDKs are used to interact with the API.",
            "pages": [
              "sdk/fetch",
              "sdk/create",
              "sdk/delete"
            ]
          }
        ]
      }
    ]
  }
}
```

## 锚点

锚点会在侧边栏顶部添加常驻的导航项。使用锚点来为内容分区、快速访问外部资源，或创建醒目的行动号召。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/anchors-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/anchors-dark.png" />

在 `navigation` 对象中，`anchors` 是一个数组，每个条目都是一个对象，必须包含 `anchor` 字段，并且可以包含其他导航字段，如分组、页面、图标，或指向外部页面的链接。

```json
{
  "navigation": {
    "anchors": [
      {
        "anchor": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "anchor": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "anchor": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

对于仅跳转到外部链接的锚点，请使用 `global` 关键字。`global` 对象中的锚点必须包含 `href` 字段，且不能指向相对路径。

全局锚点特别适合用于链接那些不属于文档但应便于用户访问的资源，例如博客或支持门户。

```json
{
  "navigation": {
    "global":  {
      "anchors": [
        {
          "anchor": "Community",
          "icon": "house",
          "href": "https://slack.com"
        },
        {
          "anchor": "Blog",
          "icon": "pencil",
          "href": "https://mintlify.com/blog"
        }
      ]
    },
    "tabs": /*...*/
  }
}
```

## 下拉菜单

下拉菜单位于侧边栏导航顶部的可展开菜单中。下拉菜单中的每个项都会指向文档的一个部分。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/dropdowns-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/dropdowns-dark.png" />

在 `navigation` 对象中，`dropdowns` 是一个数组。数组中的每个元素都是一个对象，必须包含 `dropdown` 字段，并且可以包含其他导航字段，例如分组、页面、图标或指向外部页面的链接。

```json
{
  "navigation": {
    "dropdowns": [
      {
        "dropdown": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "dropdown": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "dropdown": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

## OpenAPI

将 OpenAPI（API 规范 v3+）直接集成到你的导航结构中，自动生成 API 文档。你可以创建独立的 API 分区，也可以将端点页面放入其他导航组件中。

可在导航层级的任意层级设置默认的 OpenAPI 规范。子元素将继承该规范，除非它们另行定义。

```json
{
  "navigation": {
    "groups": [
      {
        "group": "API reference",
        "openapi": "/path/to/openapi-v1.json",
        "pages": [
          "overview",
          "authentication",
          "GET /users",
          "POST /users",
          {
            "group": "Products",
            "openapi": "/path/to/openapi-v2.json",
            "pages": [
              "GET /products",
              "POST /products"
            ]
          }
        ]
      }
    ]
  }
}
```

有关在文档中引用 OpenAPI 端点的更多信息，请参见 [OpenAPI 设置](/zh/api-playground/openapi-setup)。

## 版本

将导航按版本分组。可通过下拉菜单选择版本。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/versions-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/versions-dark.png" />

在 `navigation` 对象中，`versions` 是一个数组，其中每个条目是一个对象，必须包含 `version` 字段，并可包含任意其他导航字段。

```json
{
  "navigation": {
    "versions": [
      {
        "version": "1.0.0",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["v1/overview", "v1/quickstart", "v1/development"]
          }
        ]
      },
      {
        "version": "2.0.0",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["v2/overview", "v2/quickstart", "v2/development"]
          }
        ]
      }
    ]
  }
}
```

## 面包屑导航

为页面启用面包屑导航，在页面顶部展示完整的导航路径。

```json
{
  "styling": {
    "eyebrows": "breadcrumbs"
  }
}
```
