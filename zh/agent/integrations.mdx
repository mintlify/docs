代理集成让代理可以访问团队使用的其他工具中的上下文。连接后，代理可以搜索 Slack 线程、阅读 Notion 页面、查询 Linear 工单，并从其他应用中拉取上下文，以便完善其回复和内容更新。

<div id="connection-scope">
  ## 连接范围
</div>

集成的连接方式有两种：

- **共享集成**为整个组织一次性连接。团队中的任何人都可以提示代理使用它们。Slack、Notion、Linear、Jira、Confluence、Salesforce、Intercom 和 Plain 都采用这种方式连接。
- **个人集成**按用户连接。每位团队成员授权自己的账户，代理只能代表连接该集成的用户使用它。Google Calendar 是个人集成。

个人集成将日历事件等用户专属数据限定在连接该集成的用户范围内，而不会在整个组织间共享。

<div id="connect-an-integration">
  ## 连接集成
</div>

管理员和编辑者可以代表组织连接共享集成。

1. 前往控制台中的[代理集成](https://app.mintlify.com/settings/organization/integrations)页面。
2. 找到你想要连接的集成。
3. 点击 **Connect**。
4. 按照 OAuth 提示授权 Mintlify 访问你的账户。

某些集成在连接完成之前会打开额外的配置步骤。Jira、Confluence 和 Salesforce 会在 OAuth 流程开始前要求提供你的工作区 URL 或子域名，而 Plain 等基于 API 密钥的提供方会提示你粘贴 API 密钥。

<div id="how-the-agent-uses-integrations">
  ## 代理如何使用集成
</div>

代理将已连接的集成作为工具使用。当你向代理提出问题或交付任务时，它可以从已连接的应用中搜索并检索内容，以构建上下文。

例如：
- “总结关于 v2 API 迁移的 Slack 线程，并整理成知识库文章。”
- “查看该功能对应的 Linear 工单，并记录其行为。”
- “团队对限流是怎么决定的？去 Slack 里查一下。”

代理仅在与请求相关时才会访问集成数据。它不会主动从已连接的应用中读取内容。

<div id="permissions-and-access">
  ## 权限与访问
</div>

每个集成都会请求其搜索和检索已连接应用内容所需的 OAuth 范围 (或 API 密钥权限)。在提供方支持的情况下，Mintlify 都会请求只读范围。

请在授权连接之前于 OAuth 同意页面中查看具体的权限范围。同意页面会列出代理将获得的所有权限，你也可以随时通过控制台或所连接应用自身的集成设置撤销该连接。

如果你的组织限制了用户可授权的应用，请在连接前请相关管理员 (例如你的 Slack 或 Google Workspace 管理员) 允许 Mintlify 集成。

<div id="what-the-agent-can-read-from-each-app">
  ### 代理可以从每个应用中读取哪些内容
</div>

代理继承授权此连接的账户的访问权限。它只能读取授权用户 (针对个人集成) 或已连接的工作区账户 (针对共享集成) 在来源应用中已有访问权限的内容。

| 集成 | 代理可访问的内容 |
| --- | --- |
| Confluence | 已连接账户可以读取的空间中的页面和附件 |
| Google Calendar | 已连接用户可以查看的日历上的事件 |
| Google Drive | 已连接用户可以查看的文件和文件夹，包括共享云端硬盘 |
| HubSpot | 已连接账户可以读取的 CRM 记录 (联系人、公司、交易、工单) |
| Intercom | 已连接账户可以读取的对话、联系人和帮助中心文章 |
| Jira | 已连接账户可以读取的项目中的问题和评论 |
| Linear | 已连接账户可以读取的团队中的问题、项目和评论 |
| Notion | 显式共享给 Mintlify 集成的页面和数据库 |
| Plain | 使用所提供的 API 密钥可访问的会话和客户记录 |
| Salesforce | 已连接账户可以读取的标准对象和自定义对象 |
| Slack | 工作区授权访问的公开频道，以及已邀请 Mintlify 应用加入的私密频道 |

要缩小代理的可见范围，请在来源应用中限制已连接账户的权限 (例如，仅将 Mintlify Slack 应用邀请到特定频道，或仅将选定的 Notion 页面共享给该集成)。

<div id="data-retention-after-disconnect">
  ## 断开后的数据保留
</div>

当你断开某个集成时，代理会立即失去从该应用读取内容的能力。Mintlify 不会持久保存集成内容的副本 — 代理会在每次请求时通过实时的 OAuth 令牌或 API 密钥获取数据，因此断开连接会切断今后的访问。

出现在过往代理响应或拉取请求中的内容仍会保留 (聊天记录、已发布页面、PR 历史)。如果你也想移除这些内容，请直接编辑或删除相应的页面和对话。

<div id="troubleshoot-a-failed-connection">
  ## 排查连接失败
</div>

如果某个集成停止工作或在控制台中显示错误，通常是因为底层 OAuth 令牌已过期、在已连接应用中被撤销，或其所需的权限范围发生了变化。

要恢复访问：

1. 前往控制台中的[代理集成](https://app.mintlify.com/settings/organization/integrations)页面。
2. 找到受影响的集成，然后点击 **Configure**。
3. 点击 **Disconnect**，再点击 **Connect**，并完成 OAuth 流程。

对于像 Plain 这样基于 API 密钥的提供方，请在来源应用中生成新的 API 密钥，并将其粘贴到该集成的配置中。如果重新连接后代理仍然无法访问数据，请确认进行授权的用户在来源应用中拥有查看相关内容的权限。

<div id="disconnect-an-integration">
  ## 断开集成
</div>

1. 前往控制台中的[代理集成](https://app.mintlify.com/settings/organization/integrations)页面。
2. 找到你想要断开的已连接集成。
3. 点击 **Configure**。
4. 点击 **Disconnect**。

断开共享集成会移除组织中所有人的访问权限，代理会立即失去对该工具的访问权。

断开个人集成只会移除你自己的连接。其他连接了相同集成的团队成员仍保留其访问权，代理仍可代表他们使用该集成。
