每个页面都是一个 MDX 文件，将 Markdown 内容与 React 组件结合，用于创建丰富、交互性强的文档。

## 页面元数据

每个页面都以 frontmatter 开头，即在文件开头由 `---` 包裹的 YAML 元数据。该元数据用于定义页面的展示与行为。

使用元数据可以控制：

- 页面标题和描述
- 侧边栏标题、图标和标签

<ResponseField name="title" type="string" required>
  出现在导航和浏览器标签页中的页面标题。
</ResponseField>

<ResponseField name="description" type="string">
  对本页面内容的简要说明。显示在标题下方，并提升 SEO 效果。
</ResponseField>

<ResponseField name="sidebarTitle" type="string">
  显示在侧边栏导航中的简短标题。
</ResponseField>

<IconsOptional />

<ResponseField name="tag" type="string">
  显示在侧边栏中、页面标题旁的标签。
</ResponseField>

```yaml Example YAML frontmatter wrap
---
title: "About frontmatter"
description: "Frontmatter is the metadata that controls how your page appears and behaves"
sidebarTitle: "Frontmatter"
icon: "book"
tag: "NEW"
---
```

## 页面模式

通过 `mode` 设置控制页面的展示方式。

### 默认

如果未定义 mode，将采用带有侧边导航和目录的标准布局。

```yaml
---
title: "Default page title"
---
```

### 宽屏

宽屏模式会隐藏目录。适用于没有任何标题的页面，或当你希望利用更多横向空间时。

```yaml
---
title: "Wide page title"
mode: "wide"
---
```

### 自定义

自定义模式提供极简布局，移除除顶部导航栏之外的所有元素。它是一块空白画布，可用于创建着陆页或其他只需最少导航元素的独特布局。

```yaml
---
title: "Custom page title"
mode: "custom"
---
```

### 居中

居中模式会移除侧边栏和目录，并将内容居中显示。适用于更新日志或其他需要突出内容的页面。

```yaml
---
title: "Center page title"
mode: "center"
---
```

## API 页面

在 frontmatter 中添加 API 规范（`api` 或 `openapi`），即可生成交互式 API playground。

```yaml
---
openapi: "GET /endpoint"
---
```

深入了解如何构建 [API 文档](/zh/api-playground/overview)。

## 外部链接

在导航中使用 `url` 元数据直接链接到外部站点。

```yaml
---
title: "npm Package"
url: "https://www.npmjs.com/package/mint"
---
```

## 搜索引擎优化

大多数 SEO 元标签会自动生成。你也可以手动设置 SEO 元标签，以提升站点的 SEO、社交分享效果和浏览器兼容性。

<Note>
  含有冒号的元标签必须用引号括起来。
</Note>

```yaml
---
"twitter:image": "/images/social-preview.jpg"
---
```

详见 [SEO](/zh/settings/seo) 以获取完整的 SEO 元数据配置项。

## 内部搜索关键词

在元数据中提供 `keywords`，可提升特定页面在内置搜索中的可发现性。这些关键词不会作为页面内容显示，也不会直接出现在搜索结果列表中，但当用户搜索这些关键词时，该页面仍会作为结果呈现。

```yaml
---
keywords: ['configuration', 'setup', 'getting started']
---
```
