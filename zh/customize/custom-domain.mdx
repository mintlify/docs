若要在自定义域名下托管文档，请在 Mintlify 设置中指定您想要的自定义域名，并在域名服务提供商处配置 DNS（域名系统）设置。

<Info>
  想配置类似 `mintlify.com/docs` 的自定义子目录？请查看[自定义子目录](/zh/deploy/cloudflare)文档。
</Info>

<div id="dashboard-settings">
  ## 控制台设置
</div>

1. 前往你的[控制台](https://dashboard.mintlify.com)。
2. 选择**Settings**。
3. 选择**Domain Setup**。
4. 输入你想使用的自定义域名。例如：`docs.mintlify.com`。

<div id="configure-your-dns">
  ## 配置你的 DNS
</div>

1. 前往域名注册商网站，打开你域名的 DNS 设置。
2. 使用以下值创建一个新的 DNS 记录：

```bash
CNAME | docs | cname.vercel-dns.com.
```

<div id="provider-specific-settings">
  ## 特定提供商设置
</div>

<AccordionGroup>
  <Accordion title="Vercel verification">
    如果 Vercel 是你的 domain 提供商，你需要添加一个用于验证的 `TXT` 记录。提交自定义域名后，这些信息会显示在你的控制台中，并通过电子邮件发送给你。
  </Accordion>

  <Accordion title="Cloudflare security settings">
    如果 Cloudflare 是你的 DNS（域名系统）提供商，你需要在 HTTPS 设置中启用“full strict”安全选项。
  </Accordion>
</AccordionGroup>

<div id="set-a-canonical-url">
  ## 设置规范 URL
</div>

在配置好 DNS 后，设置一个规范 URL，以确保搜索引擎索引你首选的域名。规范 URL 会告诉搜索引擎哪一个版本的文档是主版本。当文档可通过多个 URL 访问时，这有助于提升 SEO（搜索引擎优化），并避免重复内容问题。

在 `docs.json` 中添加 `canonical` 元标签：

```json
"seo": {
    "metatags": {
        "canonical": "https://www.your-custom-domain-here.com"
    }
}
```

将 `https://www.your-custom-domain-here.com` 替换为你的实际自定义 domain。举例来说，如果你的自定义 domain 是 `docs.mintlify.com`，你应使用：

```json
"seo": {
    "metatags": {
        "canonical": "https://docs.mintlify.com"
    }
}
```
