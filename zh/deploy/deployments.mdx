当你将更改推送到已连接的存储库时，你的文档站点会自动部署。这需要正确安装并连接 Mintlify 的 GitHub 应用。

如果你的最新更改没有显示在在线站点上，首先检查 GitHub 应用是否已安装在拥有你文档存储库的账户或组织上。参见 [GitHub 故障排查](/zh/deploy/github#troubleshooting) 以获取更多信息。

如果你已经安装了 GitHub 应用，但更改仍未部署，请在控制台手动触发一次部署。

<div id="manually-trigger-a-deployment">
  ## 手动触发部署
</div>

<Steps>
  <Step title="确认你的最新提交已成功。">
    检查你的最新提交是否已出现在文档存储库中，并确认未出现任何错误。
  </Step>

  <Step title="手动触发部署。">
    前往你的[控制台](https://dashboard.mintlify.com)，点击“部署”按钮。

    <Frame>
      <img src="/images/deployments/manual-update-light.png" alt="用橙色矩形标注的“手动更新”按钮。" className="block dark:hidden" />

      <img src="/images/deployments/manual-update-dark.png" alt="用橙色矩形标注的“手动更新”按钮。" className="hidden dark:block" />
    </Frame>
  </Step>
</Steps>

<div id="delete-a-deployment">
  ## 删除部署
</div>

你可以在控制台设置中的 [Danger zone](https://dashboard.mintlify.com/settings/organization/danger-zone) 永久删除某个部署。该操作不可撤销，将移除所有部署数据，包括任何关联的预览部署。

<Steps>
  <Step title="导航到 Danger zone。">
    前往控制台设置页面中的 [Danger zone](https://dashboard.mintlify.com/settings/organization/danger-zone)。
  </Step>
  <Step title="删除部署。">
    1. 在 **Delete my deployment** 区域中填写删除原因。
    2. 点击删除按钮，并确认你要删除该部署。
  </Step>
</Steps>

<Warning>
  删除部署是永久性的且无法撤销。如果你有有效订阅，你将在当前计费周期内对剩余未使用时间获得按比例折算的抵扣额度。
</Warning>

如果你有多个部署，删除后你会被重定向到另一个部署。如果你删除了唯一的部署，你会被重定向到 Mintlify 首页。当你删除某个部署时，组织管理员会收到一封确认删除的电子邮件通知。
