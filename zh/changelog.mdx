<Update label="8月31日 - 9月13日" tags={["New releases", "Improvements"]} rss={{ title: "每周更新", description: "AI 404 建议、文档助手外部来源网页搜索、安全增强与可靠性修复" }}>
  ## 主要发布

  * **重大增强**：在 404 页面提供 AI 建议页面，[当有人访问死链 → AI 代理读取路径 → 建议语义相似的页面](https://x.com/mintlify/status/1966625627773059495)

    *注意：请联系我们为您的站点启用此功能。*

  ## 文档助手与模型上下文协议

  * 修复了由于时间窗口未正确滑动导致文档助手被错误限流的问题
  * 修复了文档助手工具调用无法正确处理空 `text` 块的问题
  * 修复了将 MCP 服务器名称与工具调用拼接后有时会超过 MCP 客户端强制的 60 字符限制的问题
  * 修复了文档助手菜单高度远大于视口并可无限滚动的问题
  * 修复了仪表板中文档助手花费数值可能显示超过两位小数的问题

  ## 网页编辑器与部署

  * 增强了编辑器安全性：仅具有已连接 Git 托管仓库 `write permissions` 的用户才能进行更改
  * 修复了分支名称包含 `=` 时预览部署无法工作的错误
  * 修复了创建预览部署时长分支名在模态框中溢出的问题
  * 体验优化：注册邀请中的 email 查询参数将自动预填输入框
  * 修复了在 Safari 中从上下文菜单复制页面不起作用的问题

  ## API交互测试台与导航

  * 多个 API交互测试台响应码在聚焦时现在显示为受控样式的选择菜单，而非系统默认的选择菜单
  * 您现在可以在 `docs.json` 中为导航分组使用 [`expanded` 字段以默认展开](https://mintlify.com/docs/navigation#default-expanded-state)

  ## SEO 与界面

  * 修复了站点图标在搜索引擎中不显示的问题：为每个站点从与文档站点相同的 URL 提供
  * 修复了 YouTube 嵌入在加载时闪烁的问题
  * 修复了将反馈菜单展开为包含文字回复时导致与目录发生布局偏移的问题
  * 修复了在 Maple 主题中关闭通知横幅后文本会溢出到顶栏上方的问题
  * 增强了 Maple 与 Willow 主题：在侧边栏新增登录/登出按钮，便于访问

  ## 分析与导出

  * 修复了文档助手分析视图与导出的可靠性问题
  * 文档助手分析导出现改为后台执行，并通过电子邮件发送，体验更可靠

  ## 重要发布：反馈收集能力增强

  * 修复了认证保护文档的预览中无法看到完整导航栏的问题

  ## 组件与样式增强

  * 为 `SidebarNavGroupDivider` 新增可用于自定义样式的 CSS 选择器
  * 为在 MDX 中定义且带有安全配置的 API 页面新增回归测试，进一步提升稳定性

  ## 性能优化

<Update label="August 10 - August 16" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Performance optimizations, LLM integrations, web editor polish, and API playground enhancements" }}>
  ## 性能与构建优化

  * 在未缓存的 NextJS 无服务器环境中，MDX 转译现改为在部署时执行，而非在每次页面加载时执行，从而缩短未缓存页面的首字节时间（TTFB）。
  * 基于内容的哈希在 MDX 未变更时避免重复转译，使页面量较大的客户更新耗时减少约 50%（超过 5 分钟的部署大致能减半）
  * 仪表盘中的预览部署加载更快了——我们在后端新增数据库索引并并行化查询
  * 通过移除每个页面 `rsc` 负载中的重复 `navigation` 数据，降低页面体积——在页面数量多或导航结构复杂时收益更明显
  * 更积极的预取使页面更常达到即刻加载

  ## API交互测试台与 OpenAPI规范 增强

  * 将 OpenAPI 到 MCP 的转换迁移到后端，使托管的 MCP 服务器能够包含工具（更清晰的文档与配置选项即将推出）
  * 为 API交互测试台 增加了 Ruby 支持
  * 新增功能：现在可[仅通过 docs.json 指定 API 页面](/zh/api-playground/openapi-setup#auto-populate-api-pages)，无需创建新的 MDX 文件。
  * 在基于 OpenAPI 规范的文档导航中支持[`webhook` 页面](/zh/api-playground/openapi-setup#webhooks)
  * 通过在跳转至 Anthropic、OpenAI 或其他提供商时，从 Markdown 链接中移除锚点片段，优化了 AI 模型上下文

  ## 网页编辑器 改进

  * 创建/重命名文件时，点击空白处即可保存更改，无需按 Enter 键
  * 修复分支导航问题：当将 URL 改为特定分支时，会被重定向到上次活跃分支而非目标分支
  * 对包含 `/` 的分支标题进行正确的 URL 编码，防止导航异常
  * 修复单体仓库仪表盘编辑器中的 `Ctrl+K` 链接快捷键此前会在前面加上文档仓库路径并导致链接错误的问题

  ## 分析与 LLM 集成

  * 支持自定义 `llms.txt` 与 `llms-full.txt`——将文件放在文档仓库根目录，即可通过 `/llms.txt` 和 `/llms-full.txt` 端点对 LLM 进行定制
  * 添加了 [Hightouch 分析集成](/zh/integrations/analytics/hightouch#hightouch)
  * 增强了上下文菜单的分析追踪（仪表盘视图即将上线）
  * 为 `llms.txt` 与 `llms-full.txt` 增加了端到端测试，确保正确提供

  ## 组件与样式增强

  * `h{1-4}` 标签支持自定义类名，以应用自定义标题样式
  * 修复了自定义页面模式下，`h{1-4}` 标签渲染为带标签条的 `Heading` 组件的问题
  * 为[面包屑](/zh/navigation#breadcrumbs)添加 CSS 选择器，便于自定义 CSS 定位
  * 通过分析尺寸，在 56px 高度下保持比例，修复了被拉伸的 open-graph 图片
  * 在启用时，将上下文菜单中的 `VSCode` 更正为 `VS Code`
  * 修复了自定义组件内的标题会与语义标题一同出现在目录中的问题

  ## Bug 修复与可靠性

  * 通过清理导致生成问题的字符，修复了某些页面标题的 PDF 渲染问题
  * 解决了遇到空的 OpenAPI JSON 文件时 CLI 报错 `Cannot convert undefined or null to object`
  * 修复了自定义 `docs.json` open-graph 元标签被生成标签覆盖的问题
  * 通过对 RSS 链接使用 origin + pathname，修复了落在锚点链接时 RSS 订阅按钮点击失效的问题
  * 通过移除 sourcemap 提升了 CLI 下载速度

  ## 技术改进

  * 在 CI 流水线中加入可视化测试，更早发现回归
  * 增强错误处理与调试能力
  * 为新功能与边界情况提供了更全面的测试覆盖
</Update>

  ## 认证改进

  * 组级公开访问：通过 `docs.json` 将整组页面设为公开，无需在每个页面上设置 `public: true`（[了解更多](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level)）
  * 在 OAuth 配置中支持 [`logoutURL`](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3)，以删除上游 cookies 并完成登出
  * 发生 OAuth 错误时，用户将被重定向到你指定的 `logoutURL` 以重新启动认证流程
  * 修复了在 OAuth/JWT 流程中回调触发前短暂闪现 500 错误的问题
  * 在 OAuth/JWT 认证配置中自动去除 URL 的 `https://` 以防止误配置

  ## API交互测试台增强

  * 新增 [Search API 端点](https://mintlify.com/docs/api-reference/assistant/search)，可基于你的文档构建智能代理和 MCP 服务器
  * 现在会在指定路径提供 `openapi` 和 `asyncapi` 文件（例如 `https://mydocsurl.extension/{openapi-or-file-name}.json`）
  * 现在可以在 openapi 文件中使用 [`x-mint` 字段](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension) 覆盖生成字段、自定义前言内容，或更改代码示例中的端点 URL
  * 在 OpenAPI 配置中，[`x-mcp` 现为 `x-mint.mcp`](https://mintlify.com/docs/api-playground/openapi-setup#mcp)，用于控制哪些路由暴露为 MCP 工具

  ## 文档助手更新

  * 修复了当新消息流式加载时，旧消息的操作菜单（包含复制、点赞等选项）消失的问题
  * 修复了上周[托管 MCP 服务器发布](https://mintlify.com/docs/ai/model-context-protocol#accessing-your-mcp-server)后嵌套的 `/mcp/...` 页面可访问性问题

  ## 性能与可靠性

  * 文档仓库中的所有图片与视频资源现在都会在你的域名下按正确路径提供。例如，如果仓库中有 `/assets/marketing/my-logo.png`，则可通过 `https://mydocsurl.extension/assets/marketing/my-logo.png` 访问。
  * Mintlify 控制台登录页的邮箱输入框现已自动聚焦，可立即输入（便捷性提升）
  * 在 Redis 中优化了自定义域名与子域名的数据，以提升导航加载性能（约 50ms 延迟降低）
  * 为 PDF 导出添加了重试逻辑以提升可靠性
  * 修复了在接受或关闭后 cookie 同意弹窗再次出现的问题——首次选择现已生效
  * 通过在 `navigator.write` 中指定 MIME `type`，修复了在 Safari 上复制页面到剪贴板的问题

  ## 技术改进

  * 修复了 Windows 与 pnpm 下的 CLI 问题，并新增 CI 测试以防回归
  * 改进了错误日志输出——为工程团队调试带来便捷性提升
  * 当缺少 `contentDirectory` 文件时，修复了断链 CI 动作的一些小问题
  * 修复了上周与受保护预览相关修复引发的回归问题，即导航 UI 中活动选项卡未能正确设置
  * 修复了主题的浅色背景未应用于活动选项卡图标的问题
  * 修复了在控制台中更改认证类型会先更新但随后又切回先前保存类型的问题——现在新选择在保存后会被持久化
  * 提升了为企业客户集成自定义 UI 库的内部开发者体验（DX）——我们现在更易在更短周期内纳入你的组件并响应需求

  ## 新的配置模式 `docs.json`

  <Frame>
    ![docs.json 截图](/images/changelog/docsjson.png)
  </Frame>

  我们引入了全新的 `docs.json` 模式以替代 `mint.json`，以支持更灵活的多层级版本管理、更直观的可视化理解，以及更一致的术语体系。了解变更详情，请[查看我们的博客](https://mintlify.com/blog/refactoring-mint-json-into-docs-json)。

  按以下步骤将 `mint.json` 升级到 `docs.json`：

  1. 确保你的 CLI 为最新版本

  ```bash
  npm i mint@latest -g
  ```

  1. 在你的文档仓库中运行

  ```bash
  mint upgrade
  ```

  1. 删除旧的 `mint.json` 文件并推送你的更改
