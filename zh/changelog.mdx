  ## 新功能

  * **Products 导航**：通过产品切换器为多个产品的文档进行组织
  * **无障碍 CLI 命令**：运行 `mint a11y` 测试文档的无障碍问题

  ## 无障碍改进

  * 新增“跳至主要内容”按钮，便于键盘导航
  * 在 Tab 组件中支持方向键导航
  * 增强了工具提示、嵌套列表和代码组的可访问性
  * 改进了整个 UI 的焦点状态与键盘交互模式

  ## AI 助手与 Analytics 增强

  * 为助手 query 分析新增柱状图可视化
  * 在助手分析页统一了日期范围选择器
  * 为助手分析图表添加缩放控件
  * 通过热键与聊天面板改进，优化助手历史记录管理

  ## 缺陷修复与可靠性

  * 修复侧边面板覆盖层的 z-index 问题
  * 修复浅色模式的颜色回退与主题相关样式错误
  * 修复 API 操作台可展开项与锚点链接不兼容的问题
  * 修复面向 ChatGPT 与其他 LLM 代理的 `.md` 链接可访问性
  * 修复按百分比 width 与 height 指定的图片尺寸处理
  * 修复带语言标签与自定义主题的代码块渲染问题
  * 修复折叠面板链接的格式与样式
  * 修复在助手配置为空时页脚间距异常
  * 修复本地存储库图片的卡片 icon 渲染
  * 修复 API 操作台中的音频文件处理（避免 UTF-8 解码）
  * 改进网页版编辑器中的 PR 发布状态管理

<Update label="8月16日" tags={["Improvements"]} rss={{ title: "每周更新", description: "性能优化、LLM 集成、Web 编辑器打磨，以及 API 操作台增强" }}>
  ## 性能与构建优化

  * 在未缓存的 NextJS 无服务器环境中，MDX 转译改为在部署时执行，而非每次页面加载时执行，从而改善未缓存页面的首字节时间（TTFB）。
  * 基于内容的哈希在 MDX 未变化时避免重新转译，将大量页面项目的更新流程时间缩短约 50%（超过 5 分钟的部署大致减半）。
  * 在控制台中预览部署的加载速度更快——我们在后端新增数据库索引并对查询进行并行化。
  * 通过移除每个页面 `rsc` 负载中的重复 `navigation` 数据，减小页面体积——在页面数量多或导航结构复杂时性能提升更明显。
  * 更积极的预取策略，使页面更常实现即刻加载。

  ## API 操作台与 OpenAPI 增强

  * 将 OpenAPI 到 MCP 的转换迁移到后端，使托管的 MCP 服务器能够包含工具（更清晰的文档与配置选项即将推出）。
  * 为 API 操作台新增 Ruby 支持。
  * 新增功能：现在你可以[仅通过 docs.json 指定 API 页面](/zh/api-playground/openapi-setup#auto-populate-api-pages)，无需创建任何新的 mdx 文件。
  * 在来自 OpenAPI 规范的文档导航中支持 [`webhook` 页面](/zh/api-playground/openapi-setup#webhooks)。
  * 优化 AI 模型上下文：在跳转至 Anthropic、OpenAI 或其他提供商时，从 Markdown 链接中去除锚点参数。

  ## Web 编辑器改进

  * 创建/重命名文件时，点击空白处即可保存变更，不再强制要求按 Enter 键。
  * 修复了 branch 导航问题：将 URL 切换到特定 branch 时会被重定向至上次活动的 branch，而非预期目标。
  * 对包含 `/` 的 branch 标题进行正确的 URL 编码，避免导航异常。
  * 修复了 monorepo 控制台编辑器中 `Ctrl+K` 链接快捷键会在前面加上文档仓库路径而导致链接失效的问题。

  ## Analytics 与 LLM 集成

  * 支持自定义 `llms.txt` 与 `llms-full.txt`——将其添加到文档仓库根目录，即可在 `/llms.txt` 与 `/llms-full.txt` 端点提供，用于定制 LLM。
  * 新增 [Hightouch 分析集成](/zh/integrations/analytics/hightouch#hightouch)。
  * 增强了上下文菜单的分析追踪（控制台可视化即将推出）。
  * 为 `llms.txt` 与 `llms-full.txt` 添加端到端（e2e）测试，确保正确提供。

  ## 组件与样式增强

  * `h{1-4}` 标签支持自定义 className，以应用自定义标题样式。
  * 修复了自定义页面模式下 `h{1-4}` 标签被渲染为带徽章的 `Heading` 组件的问题。
  * 为[面包屑](/zh/organize/navigation#breadcrumbs)新增 CSS 选择器，便于自定义 CSS 定位。
  * 通过分析图像尺寸，在 56px height 下保持比例，修复被拉伸的 open-graph 图像。
  * 在上下文菜单中将 `VSCode` 更正为 `VS Code`。
  * 修复了自定义组件内的标题与语义化标题一同出现在目录中的问题。

  ## Bug 修复与稳定性

  * 通过清理导致生成问题的字符，修复了某些页面标题的 PDF 渲染问题。
  * 解决了遇到空的 OpenAPI JSON 文件时出现的命令行界面（CLI）错误 `Cannot convert undefined or null to object`。
  * 修复了自定义 `docs.json` open-graph 元标签被生成标签覆盖的问题。
  * 修复了在访问锚点链接时 RSS 订阅按钮点击失效的问题，现改为使用 origin + pathname 作为 RSS 链接。
  * 通过移除 sourcemaps 提升了命令行界面（CLI）下载速度。

  ## 技术改进

  * 在 CI 流水线中加入可视化测试，以更早发现回归。
  * 增强了错误处理与调试能力。
  * 为新功能与边界情况提供更全面的测试覆盖。
</Update>

<Update label="August 9" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Search API, OAuth polish, asset serving, and MCP tooling" }}>
  ## 认证改进

  * 组级公开访问：通过 `docs.json` 将整个页面组设为公开，无需在每个页面上设置 `public: true`（[了解更多](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level)）
  * 在 OAuth 配置中支持 [`logoutURL`](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3)，用于删除上游 cookie 并完成登出
  * 发生 OAuth 错误时，用户会被重定向到你指定的 `logoutURL`，以重新启动认证流程
  * 修复了在 OAuth/JWT 流程中回调前短暂出现 500 错误的问题
  * 在 OAuth/JWT 认证配置中自动去除 URL 中的 `https://`，以防误配置

  ## API 操作台增强

  * 新增 [Search API 端点](https://mintlify.com/docs/api-reference/assistant/search)，可在你的文档之上构建代理和 MCP 服务器
  * 现在会在指定路径提供 `openapi` 和 `asyncapi` 文件（例如 `https://mydocsurl.extension/{openapi-or-file-name}.json`）
  * 现在可以在 openapi 文件中使用 [`x-mint` 字段](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension) 来覆盖生成的字段、自定义前言内容，或更改代码示例中的端点 URL
  * 在 OpenAPI 配置中，[`x-mcp` 现为 `x-mint.mcp`](https://mintlify.com/docs/api-playground/openapi-setup#mcp)，用于控制哪些路由以 MCP 工具的形式暴露

  ## AI 助手更新

  * 修复了旧消息的操作菜单（包含复制、点赞等选项）在新消息流式加载时消失的问题
  * 修复了上周[托管 MCP 服务器发布](https://mintlify.com/docs/ai/model-context-protocol#accessing-your-mcp-server)后嵌套的 `/mcp/...` 页面可访问性问题

  ## 性能与可靠性

  * 你文档仓库中的所有图片和视频资源现在都会在你的 domain 上按正确路径提供。例如，如果仓库中有 `/assets/marketing/my-logo.png`，它将可通过 `https://mydocsurl.extension/assets/marketing/my-logo.png` 访问。
  * Mintlify 控制台登录页的邮箱输入框现在会自动聚焦，便于立即输入（易用性优化）
  * 在 Redis 中同时优化了自定义域和子域，以提升导航加载性能（约 50ms 延迟降低）
  * 为 PDF 导出添加了重试逻辑以提升稳定性
  * 修复了在接受或关闭后 Cookie 同意弹窗再次出现的问题——现在会尊重首次选择
  * 通过在 `navigator.write` 中指定 MIME `type`，修复了在 Safari 上将页面复制到剪贴板的问题

  ## 技术改进

  * 修复了在 Windows 和 pnpm 上的 CLI 问题，并增加了 CI 测试以防止回归
