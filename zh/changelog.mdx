<Update label="August 10 - August 16" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "性能优化、LLM 集成、Web 编辑器打磨，以及 API 交互演示增强" }}>
  ## 性能与构建优化

  - 在未缓存的 NextJS 无服务器环境中，MDX 转译现已改为在部署时进行，而非每次页面加载时执行，从而改善未缓存页面的首字节时间。
  - 基于内容的哈希可在 MDX 未变更时避免重新转译，将大规模站点（部署超过 5 分钟）更新流程耗时缩短约 50%。
  - 通过在后端新增数据库索引与查询并行化，加速了在仪表盘中查看预览部署的速度。
  - 通过消除每个页面 `rsc` 负载中的重复 `navigation` 数据来减小页面体积——在页面数较多或导航结构复杂时性能提升最为明显。
  - 更积极的预取策略使瞬时页面加载更为常见。

  ## API 交互演示与 OpenAPI（API 规范，v3+）增强

  - OpenAPI 到 MCP（模型上下文协议）转换已移至后端，使托管的 MCP 服务器可包含工具（更清晰的文档与配置选项即将推出）。
  - 为 API 交互演示新增 Ruby 支持。
  - 我们新增了一个功能，现在你可以[仅使用 docs.json（站点主配置文件）指定 API 页面](/zh/api-playground/openapi-setup#auto-populate-api-pages)，无需创建任何新的 MDX 文件。
  - 在基于 OpenAPI 规范的文档导航中支持[`webhook` 页面](/zh/api-playground/openapi-setup#webhooks)。
  - 通过在跳转至 Anthropic、OpenAI 或其他提供商时，从 Markdown 链接中移除锚点参数，优化了 AI 模型上下文。

  ## Web 编辑器（浏览器中的 WYSIWYG/Markdown 编辑器）改进

  - 创建/重命名文件时，点击其他位置即可保存更改，无需按 Enter 键。
  - 修复了分支导航问题：将 URL 更改为特定分支时，会被重定向到上次活跃分支而非预期目的地。
  - 对包含 `/` 的分支标题进行正确的 URL 编码，防止导航异常。
  - 修复了 monorepo 仪表盘 Editor 中 `Ctrl+K` 链接快捷键会在前面加上 docs 仓库路径从而生成错误链接的问题。

  ## 分析与 LLM 集成

  - 支持自定义 `llms.txt` 和 `llms-full.txt`——将其添加到 docs 仓库根目录，即可在 `/llms.txt` 和 `/llms-full.txt` 端点对外提供，便于定制 LLM 行为。
  - 新增 [Hightouch 分析集成](/zh/integrations/analytics/hightouch#hightouch)。
  - 增强了上下文菜单（页面级 AI 操作菜单）的分析追踪（仪表盘可视化即将推出）。
  - 为 `llms.txt` 和 `llms-full.txt` 增加端到端测试，确保正确提供。

  ## 组件与样式增强

  - `h{1-4}` 标签支持自定义 classnames，可应用自定义标题样式。
  - 修复了在自定义页面模式下，`h{1-4}` 标签渲染为带有 chips 的 `Heading` 组件的问题。
  - 为[breadcrumbs](/zh/navigation#breadcrumbs) 添加 CSS 选择器，便于定制化选择与定位。
  - 通过分析尺寸以在 56px 高度下保持比例，修复了被拉伸的 open-graph 图片。
  - 在启用的上下文菜单中，将 `VSCode` 更正为 `VS Code`。
  - 修复了自定义组件内的标题与语义标题一起出现在目录中的问题。

  ## Bug 修复与可靠性

  - 通过清理导致生成问题的字符，修复了某些页面标题的 PDF 渲染问题。
  - 解决了在遇到空的 OpenAPI JSON 文件时，CLI 报错 `Cannot convert undefined or null to object` 的问题。
  - 修复了自定义 `docs.json` open-graph 元标签被生成标签覆盖的问题。
  - 通过对 RSS 链接使用 origin + pathname，修复了落到锚点链接时 RSS 订阅按钮点击的问题。
  - 通过移除 sourcemaps 提升了 CLI 的下载速度。

  ## 技术改进

  - 在 CI 流水线中新增可视化测试，以更早发现回归。
  - 增强了错误处理与调试能力。
  - 为新功能与边界情况提供了更全面的测试覆盖。
</Update>

<Update label="August 3 - August 9" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Search API、OAuth 细化、资源托管与 MCP 工具" }}>
  ## 认证改进

  - 组级公开访问：通过 `docs.json` 将整个页面分组设为公开，无需在每个页面上设置 `public: true`（[了解更多](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level)）
  - 在 OAuth 配置中支持 [`logoutURL`](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3)，以删除上游 cookies 并完成登出
  - 发生 OAuth 错误时，用户将被重定向到你指定的 `logoutURL`，以重新开始认证流程
  - 修复了在回调前的 OAuth/JWT 流程中短暂闪现 500 错误的问题
  - 在 OAuth/JWT 认证配置中自动移除 URL 的 `https://`，以防止错误配置

  ## API 交互演示增强

  - 新增 [Search API 端点](https://mintlify.com/docs/api-reference/assistant/search)，可在你的文档之上构建代理和 MCP 服务器
  - 现已在指定路径提供 `openapi` 和 `asyncapi` 文件（例如 `https://mydocsurl.extension/{openapi-or-file-name}.json`）
  - 你现在可以在 OpenAPI 文件中使用 [`x-mint` 字段](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension) 来覆盖生成字段、自定义前言内容或更改代码示例中的端点 URL
  - 在 OpenAPI 配置中，[`x-mcp` 现为 `x-mint.mcp`](https://mintlify.com/docs/api-playground/openapi-setup#mcp)，用于控制哪些路由以 MCP 工具形式暴露

  ## Assistant 更新

  - 修复了在新消息流入时，旧消息的操作菜单（包含复制、点赞等选项）消失的问题
  - 修复了上周[托管 MCP 服务器发布](https://mintlify.com/docs/ai/model-context-protocol#accessing-your-mcp-server)后嵌套的 `/mcp/...` 页面可访问性问题

  ## 性能与可靠性

  - 你文档仓库中的所有图片和视频资源现在都会在你的域名上以正确路径提供。例如，如果仓库中有 `/assets/marketing/my-logo.png`，将可通过 `https://mydocsurl.extension/assets/marketing/my-logo.png` 访问。
  - Mintlify 控制台登录页的邮箱输入框现在会自动聚焦，你可立即开始输入（体验优化）
  - 在 Redis 中同时处理自定义域名和子域名，以提升导航加载性能（约 50ms 延迟降低）
  - 为 PDF 导出添加重试逻辑以提升稳定性
  - 修复了在接受或关闭后 cookie 同意弹窗仍会再次出现的问题——首次选择现已生效
  - 通过在 `navigator.write` 中指定 MIME `type`，修复了在 Safari 上复制页面到剪贴板的问题

  ## 技术改进

  - 修复了适用于 Windows 和 pnpm 的 CLI 问题，并加入 CI 测试以防回归
  - 改进错误日志输出——为工程团队调试带来体验提升
  - 当缺少 `contentDirectory` 文件时，对断链 CI 动作进行了小幅修复
  - 修复了上周认证保护预览修复引起的回归：导航 UI 中的活动选项卡未能正确设置
  - 修复了主题浅色背景未应用于活动选项卡图标的问题
  - 修复了在控制台中更改认证类型会先更新又回退到先前保存类型的问题——现在保存后新选择会持久化
  - 为使用自定义 UI 库的企业客户带来内部 DX 改进——我们现在更易在更短时间内纳入你的组件并满足需求

<Update label="February 2025" tags={["New releases", "Improvements"]}>
  ## 全新的配置架构 `docs.json`

  <Frame>
    ![docs.json 截图](/images/changelog/docsjson.png)
  </Frame>

  我们推出了新的 `docs.json` 架构以取代 `mint.json`，带来更完善的多级版本管理、更直观的可读性，以及更一致的术语体系。了解具体变更，[请查看我们的博客](https://mintlify.com/blog/refactoring-mint-json-into-docs-json)。

  按以下步骤从 `mint.json` 升级到 `docs.json`：

  1. 确保你的 CLI 为最新版本

  ```bash
  npm i mint@latest -g
  ```

  1. 在你的文档仓库中运行

  ```bash
  mint upgrade
  ```

  1. 删除旧的 `mint.json` 文件并推送你的更改
