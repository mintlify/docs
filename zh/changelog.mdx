<Update label="August 10 - August 16" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "性能优化、LLM 集成、Web 编辑器打磨以及 API 操作台增强" }}>
  ## 性能与构建优化

  * 在未缓存的 Next.js 无服务器环境中，MDX 转译现已改为在部署时进行，而非每次页面加载时执行，从而缩短未缓存页面的首字节时间（TTFB）。
  * 基于内容的哈希可在 MDX 未变更时避免重新转译，将大体量站点的更新流程时间缩短约 50%（超过 5 分钟的部署通常可减半）
  * 通过在后端新增数据库索引并并行化 query，控制台中的预览部署加载更快
  * 通过移除每个页面 `rsc` 负载中重复的 `navigation` 数据来减小页面体积——在页面数量多或导航结构复杂时收益更显著
  * 更激进的预取策略使即时页面加载更常见

  ## API 操作台与 OpenAPI 增强

  * 将 OpenAPI 到 MCP 的转换移至后端，使托管的 MCP 服务器能够包含工具（更清晰的文档与配置选项即将推出）
  * 为 API 操作台新增 Ruby 支持
  * 新增能力：你现在可以[仅通过 docs.json 指定 API 页面](/zh/api-playground/openapi-setup#auto-populate-api-pages)，无需创建新的 mdx 文件。
  * 在基于 OpenAPI 规范的文档导航中支持 [`webhook` 页面](/zh/api-playground/openapi-setup#webhooks)
  * 通过在跳转至 Anthropic、OpenAI 等提供商时，从 Markdown 链接中移除锚点片段，优化了 AI 模型上下文

  ## Web 编辑器改进

  * 创建/重命名文件时，点击空白处即可保存更改，无需按 Enter 键
  * 修复了 branch 导航：将 URL 更改为特定 branch 会被重定向到上次活跃的 branch 的问题
  * 对包含 `/` 的 branch 标题进行正确的 URL 编码，防止导航异常
  * 修复了 monorepo 控制台编辑器中的 `Ctrl+K` 链接快捷键此前会在前面加上文档仓库路径并生成错误链接的问题

  ## Analytics 与 LLM 集成

  * 支持自定义 `llms.txt` 和 `llms-full.txt`——将其添加到文档仓库根目录，可在 `/llms.txt` 与 `/llms-full.txt` 端点提供，用于定制 LLM
  * 新增 [Hightouch 分析集成](/zh/integrations/analytics/hightouch#hightouch)
  * 增强了上下文菜单的分析跟踪（控制台视图即将上线）
  * 为 `llms.txt` 和 `llms-full.txt` 添加端到端测试，确保正确提供

  ## 组件与样式增强

  * 在 `h{1-4}` 标签中支持自定义 className 以应用自定义标题样式
  * 修复了在自定义页面模式下，`h{1-4}` 标签被渲染为带有徽标/标记的 `Heading` 组件的问题
  * 为[面包屑](/zh/navigation#breadcrumbs)添加 CSS 选择器以便执行定向的自定义样式
  * 通过分析图像尺寸，在 56px height 下保持纵横比，修复被拉伸的 Open Graph 图像
  * 在上下文菜单中将 `VSCode` 更正为 `VS Code`
  * 修复自定义组件内的标题与语义化标题同时出现在目录中的问题

  ## Bug 修复与稳定性

  * 通过清理会导致生成问题的字符，修复了某些页面标题的 PDF 渲染问题
  * 在遇到空的 OpenAPI JSON 文件时，修复了命令行界面（CLI）错误 `Cannot convert undefined or null to object`
  * 修复自定义 `docs.json` 的 Open Graph 元标签被自动生成标签覆盖的问题
  * 通过对 RSS 链接使用 origin + pathname，修复了从锚点链接进入页面时 RSS 订阅按钮点击无效的问题
  * 通过移除 source map 提升了命令行界面（CLI）下载速度

  ## 技术改进

  * 在 CI 流水线中添加可视化测试，以更早发现回归
  * 增强了错误处理与调试能力
  * 为新功能与边界情况提供了更全面的测试覆盖
</Update>

<Update label="August 3 - August 9" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Search API, OAuth polish, asset serving, and MCP tooling" }}>
  ## 认证改进

  * 组级公开访问：通过 `docs.json` 将整个页面组设为公开，无需在每个页面上设置 `public: true`（[了解更多](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level)）
  * 在 OAuth 配置中支持 [`logoutURL`](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3)，用于删除上游 cookie 并完成登出
  * 发生 OAuth 错误时，用户会被重定向到你指定的 `logoutURL`，以重新启动认证流程
  * 修复了在 OAuth/JWT 流程中回调前短暂出现 500 错误的问题
  * 在 OAuth/JWT 认证配置中自动去除 URL 中的 `https://`，以防误配置

  ## API 操作台增强

  * 新增 [Search API 端点](https://mintlify.com/docs/api-reference/assistant/search)，可在你的文档之上构建代理和 MCP 服务器
  * 现在会在指定路径提供 `openapi` 和 `asyncapi` 文件（例如 `https://mydocsurl.extension/{openapi-or-file-name}.json`）
  * 现在可以在 openapi 文件中使用 [`x-mint` 字段](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension) 来覆盖生成的字段、自定义前言内容，或更改代码示例中的端点 URL
  * 在 OpenAPI 配置中，[`x-mcp` 现为 `x-mint.mcp`](https://mintlify.com/docs/api-playground/openapi-setup#mcp)，用于控制哪些路由以 MCP 工具的形式暴露

  ## AI 助手更新

  * 修复了旧消息的操作菜单（包含复制、点赞等选项）在新消息流式加载时消失的问题
  * 修复了上周[托管 MCP 服务器发布](https://mintlify.com/docs/ai/model-context-protocol#accessing-your-mcp-server)后嵌套的 `/mcp/...` 页面可访问性问题

  ## 性能与可靠性

  * 你文档仓库中的所有图片和视频资源现在都会在你的 domain 上按正确路径提供。例如，如果仓库中有 `/assets/marketing/my-logo.png`，它将可通过 `https://mydocsurl.extension/assets/marketing/my-logo.png` 访问。
  * Mintlify 控制台登录页的邮箱输入框现在会自动聚焦，便于立即输入（易用性优化）
  * 在 Redis 中同时优化了自定义域和子域，以提升导航加载性能（约 50ms 延迟降低）
  * 为 PDF 导出添加了重试逻辑以提升稳定性
  * 修复了在接受或关闭后 Cookie 同意弹窗再次出现的问题——现在会尊重首次选择
  * 通过在 `navigator.write` 中指定 MIME `type`，修复了在 Safari 上将页面复制到剪贴板的问题

  ## 技术改进

  * 修复了在 Windows 和 pnpm 上的 CLI 问题，并增加了 CI 测试以防止回归
  * 改进了错误日志输出——为工程团队调试带来更佳体验
  * 当缺少 `contentDirectory` 文件时，修复了 broken-link CI action 的一些小问题
  * 修复了上周认证保护预览修复引入的回归问题，导致导航 UI 中未正确设置活动标签页
  * 修复了主题的 light 背景色未应用于活动标签页 icon 的问题
  * 修复了在控制台中更改认证类型后先更新又回退到之前已保存类型的问题——现在新选择在保存后会保持
  * 面向使用自定义 UI 库的企业客户的内部 DX 改进——我们现在更容易在更短周期内纳入你的组件并响应需求
</Update>
