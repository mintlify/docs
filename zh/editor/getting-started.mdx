Web 编辑器是在浏览器中直接用于创建、编辑和审阅文档的可视化界面。

* **可视化编辑**：使用 WYSIWYG（所见即所得）编辑器修改文档，并实时预览发布后的呈现效果。
* **Git 同步**：所有更改会自动与您的 Git 存储库同步，确保版本控制。
* **团队协作**：多人可同时协作编辑文档。
* **组件集成**：使用斜杠命令添加提示、代码块等组件。
* **零配置**：从控制台即可立即开始写作。

<div id="web-editor-workflow">
  ## Web 编辑器工作流
</div>

以下是在 Web 编辑器中的常见工作方式：

<Steps>
  <Step title="Choose your branch">
    创建一个 branch，或直接在你的部署用分支上进行更改。我们建议先创建一个 branch，这样你可以在更改上线前进行预览。
  </Step>

  <Step title="Open your file">
    在侧边栏找到现有文件，或通过文件资源管理器新建一个文件。
  </Step>

  <Step title="Edit your content">
    在 Web 编辑器中进行编辑。试着用右上角的切换按钮在可视化模式和 Markdown 模式之间切换，看看你更偏好哪种模式。
  </Step>

  <Step title="Preview your changes">
    可视化模式会展示你的更改在在线站点上的实际效果。用它来核对是否一切显示正确。
  </Step>

  <Step title="Publish your changes">
    如果你在部署用分支上工作，可直接在 Web 编辑器中发布更改。若在其他分支上，请先创建一个拉取请求（PR；亦称“合并请求”/Merge Request）以供审核，再发布。
  </Step>
</Steps>

<div id="editor-modes">
  ## 编辑模式
</div>

Web 编辑器提供两种模式，以满足不同的编辑偏好和需求。使用编辑器工具栏右上角的切换按钮在两种模式之间切换。

<div id="visual-mode">
  ### 可视化模式
</div>

可视化模式提供 WYSIWYG（所见即所得）体验：你在编辑器中的改动将与发布到文档站点的效果一致。该模式非常适合需要实时查看更改呈现效果的场景。

按 “/” 打开组件菜单，添加样式化内容、提示块、代码片段或其他组件。

<div id="markdown-mode">
  ### Markdown 模式
</div>

Markdown 模式可直接访问文档底层的 `MDX` 代码。此模式非常适合在需要精确控制组件属性或倾向于使用 Markdown 语法写作时使用。
