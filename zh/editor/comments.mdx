评论让你和你的团队可以在无需编辑页面内容的情况下，针对页面中的特定文本留下反馈。使用评论来提出问题、标记问题，或在发布之前讨论变更。

所有[角色](/zh/dashboard/roles)——浏览者、编辑者和管理员——都可以留下和回复评论。只有编辑者和管理员可以将评论线程标记为已解决。

如果你的 branch 在 GitHub 上有一个打开的拉取请求（PR；亦称“合并请求”/Merge Request），Mintlify 会在该拉取请求的 description 中添加未解决评论线程的摘要。该摘要会列出包含未解决评论的文件、每个线程中的第一条评论，以及一个在编辑器中打开该文件并展开评论侧边栏的链接。

当你添加、解决或删除评论时，该摘要会自动更新。

<div id="add-a-comment">
  ## 添加评论
</div>

1. 选中要评论的文本。
2. 点击弹出工具栏中的 **Comment** 按钮。
3. 输入评论内容并按下 <kbd>Enter</kbd> 键提交。

<div id="reply-to-a-comment">
  ## 回复评论
</div>

单击高亮文本，或在评论侧边栏中单击对应线程以打开评论线程。输入回复内容后按 <kbd>Enter</kbd> 键提交。

<div id="mention-a-teammate">
  ## 提及队友
</div>

在评论或回复中输入 `@` 来提及队友，然后在弹出的列表中选择他们的姓名。你只能提及你所在组织的成员。

被提及的队友会收到电子邮件通知。

<div id="resolve-a-comment">
  ## 解决评论
</div>

讨论结束后，编辑和管理员可以将评论线程标记为已解决。

点击对勾 icon，将该线程标记为已解决。

<div id="delete-a-comment">
  ## 删除评论
</div>

点击评论旁的垃圾桶 icon 即可将其删除。删除某个评论线程中的第一条评论会删除整个线程。

你可以删除自己的评论。管理员可以删除任何评论。
