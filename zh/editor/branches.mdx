---
title: "使用 branch 工作"
description: "通过使用 branch 进行更改和评审，而不影响线上文档"
---

branch 是版本控制中的一项功能，用于指向存储库中的特定提交。你的部署用分支，通常称为 `main`，代表用于构建线上文档的内容。除非你选择将它们合并到部署用分支，否则所有其他 branch 都独立于线上文档。

使用 branch 可以为你的文档创建独立实例，以便进行更改、获取评审，并在发布前尝试新方法。你的团队可以在不同的 branch 上同时更新文档的不同部分；在你发布更改之前，不会影响用户在线上站点上看到的内容。

我们建议在更新文档时始终基于 branch 工作，以保持线上站点稳定并支持评审流程。

<Tip>
  - 清晰地为 branch 命名，让队友明白你在做什么。
  - 合并后删除 branch，保持存储库整洁有序。
  - 当你在进行可能影响他人工作的重大更改时，提前告知团队。
</Tip>

<div id="creating-a-branch">
  ## 创建 branch
</div>

1. 在编辑器工具栏中选择 branch 名称（默认通常为 `main`）。
2. 选择 **New Branch**。
3. 为你的 branch 输入一个具有描述性的名称，例如 `update-getting-started` 或 `fix-api-examples`。
4. 选择 **Create Branch**。

<div id="saving-changes-on-a-branch">
  ## 在 branch 上保存更改
</div>

要在 branch 上保存你的更改，请点击编辑器右上角的 **保存更改** 按钮。此操作会创建一个提交，并将你的工作推送到对应的 branch。

<div id="switching-branches">
  ## 切换 branch
</div>

1. 在编辑器工具栏中选择 branch 名称。
2. 从下拉菜单中选择要切换的 branch。

<Tip>
  切换 branch 时，未保存的更改将会丢失。请在切换 branch 之前先保存或发布你的工作。
</Tip>
