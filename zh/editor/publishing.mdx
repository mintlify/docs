你的更改将如何发布取决于你当前所在的 branch：

* **部署用分支**：发布会立即更新你的线上站点。
* **其他分支**：发布会创建一个拉取请求（PR），以便你在部署到生产环境之前审阅更改。

<div id="pull-requests-and-reviewing-changes">
  ## 拉取请求（PR；亦称“合并请求”/Merge Request）与变更审查
</div>

拉取请求（PR）允许你从你的 branch 提交更改，供他人在合并到线上文档之前进行审查。这样可以确保你的更改准确无误，并让你的团队有机会协作完善内容。

<Tip>
  即使你独自工作，拉取请求也有助于预览更改并维护清晰的更新历史。
</Tip>

<div id="creating-a-pull-request">
  ### 创建拉取请求（PR）
</div>

<Steps>
  <Step title="保存你的工作">
    确保你所有的更改已通过 **保存更改** 保存到你的 branch 上。
  </Step>
  <Step title="创建拉取请求">
    在编辑器右上角选择 **Publish Pull Request**。
  </Step>
  <Step title="添加标题和说明">
    撰写清晰的标题和说明，包含：
    - 你做了哪些更改
    - 你为什么进行这些更改
    - 需要重点审核的部分
  </Step>
  <Step title="创建并分享">
    选择 **Publish Pull Request**。编辑器会提供一个链接，用于查看你的拉取请求（PR）。

<div id="reviewing-pull-requests">
  ## 审核拉取请求（PR）
</div>

创建拉取请求后：

1. **审查更改**：你和团队成员可以在 Git 托管平台（如 GitHub 或 GitLab）中审查你的拉取请求。
2. **留下反馈**：添加评论或请求修改。
3. **进一步修改**：在 Web 编辑器中继续修改。保存更改后，编辑器会将其推送到你的拉取请求。
4. **批准**：当你对更改满意时，批准该拉取请求。
5. **合并**：当你准备将更改部署到生产环境时，合并该拉取请求。
