使用导航侧边栏来组织你的文档。你在编辑器中所做的更改会在发布时同步到你的仓库 —— 你无需手动修改 `docs.json`。

<div id="add-navigation-elements">
  ## 添加导航元素
</div>

在导航树底部点击 **Add new**，可在根级别添加一个新元素。要在另一个元素内嵌套添加元素，请点击该元素旁边的 <Icon icon="plus" /> 加号按钮。

创建元素后，通过拖放对其重新排序或嵌套。将光标悬停在元素上并点击齿轮图标即可配置其属性。

右键点击 **页面** 可打开其设置或将其删除。

右键点击 **导航元素** 可打开其设置、将其转换为其他类型的元素、复制它、在其中添加页面或分组、取消分组或将其删除。

<Note>
  某些元素不能嵌套在其他元素中。例如，选项卡不能嵌套在分组中。编辑器会阻止你创建无效的组合。
</Note>

<div id="navigation-elements">
  ## 导航元素
</div>

为你的内容选择合适的结构：

- **Pages** — 独立的文档文件。导航的核心构建块。
- **Groups** — 将相关页面聚合在一起的可折叠区段。可以嵌套在其他分组中。
- **Tabs** — 站点顶部带有水平导航的顶级区段。当你有 API Reference 和 Guides 等不同领域时，请使用 Tabs。
- **Anchors** — 位于侧边栏顶部的常驻项。适合快速访问外部资源或顶级区段。可设置为全局，以在每个页面上显示。
- **Dropdowns** — 位于侧边栏顶部的可展开菜单。相对于 Tabs，是一种更轻量的替代方案，适用于多个相关区段。
- **Menus** — 选项卡内的下拉导航项，可直接链接到特定页面。
- **Products** — 多个不同产品文档集的切换器，每个产品都有自己的导航结构。
- **Versions** — 多个并行文档版本的切换器。
- **Languages** — 多语言文档的切换器。

<div id="configure-elements">
  ## 配置元素
</div>

将光标悬停在任意导航元素上，然后点击齿轮图标即可设置：

- **Title**：在侧边栏中显示的标签。
- **Icon**：标签旁边的视觉标记。
- **Tag**：用于突出显示该项的徽章，如 “NEW” 或 “BETA”。
