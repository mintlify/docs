当你将更改推送到已连接的存储库时，文档站点会自动部署。此功能需要正确安装并连接 Mintlify GitHub 应用。

如果你的最新更改未在生产站点上显示，首先检查拥有文档存储库的账户或组织是否已安装 GitHub 应用。更多信息请参阅 [GitHub 故障排查](/zh/settings/github#troubleshooting)。

如果 GitHub 应用已连接，但更改仍未部署，你可以在控制台中手动触发重新构建。

<div id="manually-triggering-a-deployment">
  ## 手动触发部署
</div>

<Steps>
  <Step title="确认你的最新提交已成功。">
    检查你的文档存储库中是否已出现最新提交，且未出现任何错误。
  </Step>

  <Step title="手动触发部署。">
    前往你的[控制台](https://dashboard.mintlify.com)并点击部署按钮。

      <img src="/images/deployments/manual-update-light.png" alt="用橙色矩形标注的手动更新按钮。" className="block dark:hidden" />

      <img src="/images/deployments/manual-update-dark.png" alt="用橙色矩形标注的手动更新按钮。" className="hidden dark:block" />
    </Frame>
  </Step>
</Steps>
