当你将更改推送到已连接的代码仓库时，文档站点会自动部署。这需要正确安装并连接 Mintlify 的 GitHub 应用。

如果你的最新更改没有出现在线上站点上，首先检查 GitHub 应用是否安装在拥有文档仓库的个人账户或组织中。更多信息请参见 [GitHub 故障排查](/zh/settings/github#troubleshooting)。

如果 GitHub 应用已连接，但更改仍未部署，你可以在控制台中手动触发重建。

<div id="manually-triggering-a-deployment">
  ## 手动触发部署
</div>

<Steps>
  <Step title="确认你的最新提交已成功。">
    检查你的最新提交是否已出现在文档仓库中，并确保没有任何错误。
  </Step>

  <Step title="手动触发部署。">
    前往你的[控制台](https://dashboard.mintlify.com)，然后选择“部署”按钮。

    </Frame>
  </Step>
</Steps>
