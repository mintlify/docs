当你将更改推送到已连接的代码仓库时，文档站点会自动部署。前提是已正确安装并连接 Mintlify 的 GitHub 应用。

如果最新更改没有出现在线上站点上，请先检查 GitHub 应用是否已安装在拥有文档仓库的账户或组织中。更多信息请参见 [GitHub 故障排查](/zh/settings/github#troubleshooting)。

如果 GitHub 应用已连接，但更改仍未部署，你可以在仪表盘中手动触发一次重建。

## 手动触发部署

<Steps>
  <Step title="确认你最新的提交已成功。">
    检查最新提交是否已出现在文档仓库中，并确认没有出现任何错误。
  </Step>

  <Step title="手动触发部署。">
    前往你的[仪表盘](https://dashboard.mintlify.com)，点击部署按钮。

      <img src="/images/deployments/manual-update-light.png" alt="用橙色矩形高亮显示的“手动更新”按钮。" className="block dark:hidden" />

    </Frame>
  </Step>
</Steps>
