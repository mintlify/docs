通过工作区规则和记忆，将 Windsurf 打造成一位熟悉你的风格指南、组件和项目上下文的文档专家。

## 在 Mintlify 中使用 Windsurf

Windsurf 的 Cascade AI 文档助手可以结合 Mintlify 组件，根据你的规范生成文档。Workspace 规则和 Memories 为你的项目提供持久化上下文，帮助 Cascade 输出更一致的建议。

* **Workspace 规则** 存储在你的文档仓库中，并与团队共享。
* **Memories** 提供会随时间累积的个人上下文。

我们建议为共享的文档规范设置 Workspace 规则。你可以在工作过程中逐步积累 Memories，但由于它们不共享，因此团队成员之间的一致性无法保证。

在文档仓库的 `.windsurf/rules` 目录中创建 Workspace 规则。更多信息请参阅 Windsurf 文档中的 [Memories & Rules](https://docs.windsurf.com/windsurf/cascade/memories)。

## 示例工作区规则

此规则为 Cascade 提供关于 Mintlify 组件和通用技术写作最佳实践的上下文。

你可以直接使用此示例规则，或根据你的文档进行自定义：

* **写作规范**：更新语言指南，使其与您的风格指南一致。
* **组件模式**：添加项目特定的组件或修改现有示例。
* **代码示例**：将通用示例替换为与你产品相关的真实 API 调用与响应。
* **风格与语气偏好**：调整术语、格式和其他规则。

将你的规则保存为 `.md` 文件，并放在文档仓库的 `.windsurf/rules` 目录中。

````mdx
# Mintlify 技术写作规则

## 项目背景

- 这是在 Mintlify 平台上的文档项目
- 我们使用带有 YAML frontmatter 的 MDX 文件
- 站点导航在 `docs.json` 中配置
- 我们遵循技术写作最佳实践

## 写作标准

- 说明性内容使用第二人称（“你”）
- 使用主动语态和一般现在时
- 在步骤前列出前提条件
- 为重要步骤说明预期结果
- 使用描述性且包含关键词的标题
- 句子保持简洁但信息充分

## 页面必需结构

每个页面必须以 frontmatter 开头：

```yaml
---
title: "清晰、具体的标题"
description: "用于 SEO 和导航的简明描述"
---
```

## Mintlify 组件

### docs.json

- 构建 docs.json 文件和站点导航时，请参考 [docs.json 架构](https://mintlify.com/docs.json)

### Callouts

- 使用 `<Note>` 提供有用的补充信息
- 使用 `<Warning>` 标注重要注意事项和破坏性变更
- 使用 `<Tip>` 分享最佳实践和专家建议
- 使用 `<Info>` 提供中性背景信息
- 使用 `<Check>` 表示成功校验/确认

### 代码示例

- 适当时，包含完整且可运行的示例
- 多语言示例使用 `<CodeGroup>`
- 在所有代码块上标明语言标签
- 使用真实数据而非占位符
- 在 API 文档中使用 `<RequestExample>` 和 `<ResponseExample>`

### 操作流程

- 顺序性说明使用 `<Steps>` 组件
- 需要时使用 `<Check>` 组件加入校验步骤
- 将复杂流程拆分为更小的步骤

### 内容组织

- 平台特定内容使用 `<Tabs>`
- 渐进披露使用 `<Accordion>`
- 突出内容使用 `<Card>` 和 `<CardGroup>`
- 使用带有描述性替代文本的 `<Frame>` 包裹图片

## API 文档要求

- 使用 `<ParamField>` 记录所有参数
- 使用 `<ResponseField>` 展示响应结构
- 同时包含成功与错误示例
- 嵌套对象属性使用 `<Expandable>`
- 始终包含认证示例

## 质量标准

- 发布前测试所有代码示例
- 站内链接使用相对路径
- 所有图片都包含替代文本（alt）
- 确保正确的标题层级（从 h2 开始）
- 检查既有模式以保持一致性
````

## 使用 Cascade

完成规则设置后，您可以使用 Cascade 来协助处理各类文档任务。更多信息请参阅 Windsurf 文档中的 [Cascade](https://docs.windsurf.com/windsurf/cascade)。

### 示例提示

**撰写新内容**:
```text wrap
创建一个新页面，说明如何对我们的 API 进行身份验证。请包含 JavaScript、Python 和 cURL 的代码示例。
```

**改进现有内容**:
```text wrap
审查此页面并就提升清晰度与组件使用提出改进建议。重点让步骤更易于理解与跟进。
```

**创建代码示例**:
```text wrap
生成一个完整的代码示例，展示该 API 端点的错误处理。使用贴近实际的数据，并包含预期的响应。
```

**保持一致性**:
```text wrap
检查此新页面是否符合我们的文档规范，并提出任何必要的修改建议。
```
