同时为传统搜索引擎与 ChatGPT、Perplexity、Google AI Overviews 等 AI 驱动的回答引擎优化你的文档。

生成式引擎优化（GEO，Generative Engine Optimization）强调通过全面内容与结构化信息获得 AI 系统引用；而传统 SEO（搜索引擎优化）则侧重提升搜索结果排名。

<div id="geo-quickstart">
  ## GEO 快速开始
</div>

<div id="initial-setup">
  ### 初始设置
</div>

1. 在 `docs.json` 设置中**确保你的文档会被索引**
2. **审查当前页面**是否缺少 description 和 title

<div id="content-improvements">
  ### 内容改进
</div>

1. **在合适的页面添加对比表**
2. **审查标题层级**，确保能够回答常见问题
3. **强化相关主题之间的内部链接**
4. **使用 AI 工具进行测试**，以验证准确性

<div id="geo-best-practices">
  ## GEO 最佳实践
</div>

总体而言，撰写得当、结构清晰的文档通常会有出色的 GEO。你仍应把为用户而写放在首位；只要内容能满足他们的需求，你就在为 AI 工具优化的正确道路上。专注于创作真正有用的内容，而非为“优化”而优化，这一点会同时受到传统和 AI 搜索引擎的认可。

重点关注：

- 围绕用户需求的内容，而非仅做关键词匹配
- 结构化、便于快速浏览的信息
- 直接回答问题

<div id="format-for-clarity">
  ### 为清晰表达而设计的格式
</div>

这些格式实践可帮助 AI 工具更好地解析与理解你的内容：

- 不要跳级使用标题（H1 → H2 → H3）
- 使用具体的对象名称，而非“it”或“this”
- 为代码块标注对应的编程语言
- 为图片提供具有描述性的替代文本（alt 文本）
- 链接相关概念以帮助 AI 理解其相互关系

<div id="answer-questions-directly">
  ### 直接回答问题
</div>

撰写能够解决特定用户问题的内容：

- 以核心结论开启每个小节
- 使用与常见问题相匹配的描述性标题
- 将复杂主题拆分为编号步骤

<div id="mintlify-configuration">
  ## Mintlify 配置
</div>

使用这些功能优化 GEO。

<div id="add-descriptive-page-metadata">
  ### 添加具描述性的页面 metadata
</div>

在你的 frontmatter 中包含清晰的 title 和 description：

```mdx
---
title: "API 认证指南"
description: "实现 API 认证的完整指南，包含代码示例"
---
```


<div id="configure-global-indexing-settings">
  ### 配置全局索引编入设置
</div>

将以下内容添加到你的 `docs.json`：

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "您的文档"
    }
  }
}
```


<div id="llmstxt">
  ### LLMs.txt
</div>

LLMs.txt 文件可帮助 AI 系统理解你的文档结构，类似于站点地图之于搜索引擎。Mintlify 会为你的文档自动生成 LLMs.txt 文件，无需进行任何配置。

<div id="testing-your-documentation">
  ## 测试你的文档
</div>

围绕你的产品和文档提出问题，测试各类 AI 工具，评估它们对你的文档引用是否到位。

**向 AI 助手就你的文档提出具体问题：**

- “如何使用此 API 配置认证？”
- “请逐步引导我完成安装流程”

**检查这些工具是否提供：**

- 正确的代码示例
- 准确的分步指引
