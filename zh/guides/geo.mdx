同时针对传统搜索引擎与由 AI 驱动的回答引擎（如 ChatGPT、Perplexity 和 Google AI Overviews）优化你的文档。

生成引擎优化（GEO）通过全面的内容与结构化信息，侧重于被 AI 系统引用；而传统 SEO 则侧重于提升搜索结果排名。

## 生成引擎优化快速上手

### 初始设置

1. 在 `docs.json` 设置中**确保文档已开启索引**
2. **审查现有页面**，补全缺失的标题和描述

### 内容优化

1. **在合适的页面添加对比表**
2. **审查各级标题**，确保能回答常见问题
3. **优化相关主题之间的内部链接**
4. **使用 AI 工具测试**以验证准确性

## 生成引擎优化最佳实践

总体来说，撰写良好且结构清晰的文档会具备出色的生成引擎优化。你仍应优先面向用户写作；如果内容满足用户需求，你就已走在为 AI 工具优化的正确道路上。创造真正有用的内容，而不是为“优化而优化”，既会被传统搜索引擎认可，也会被 AI 搜索引擎认可。

重点关注：

- 面向用户需求的内容，而非单纯的关键词匹配
- 结构化、便于扫读的信息
- 直接回答问题

### 通过格式提升清晰度

以下格式实践有助于 AI 工具解析并理解你的内容：

- 不要跳过标题层级（H1 → H2 → H3）
- 使用具体的对象名称，而不是“it”或“this”
- 为代码块标注所用编程语言
- 为图片提供具描述性的替代文本（alt 文本）
- 链接相关概念，帮助 AI 理解关联关系

### 直接回答问题

撰写能够解决特定用户问题的内容：

- 以核心结论开头展开章节
- 使用与常见查询相匹配的描述性标题
- 将复杂主题拆分为编号步骤

## Mintlify 配置

使用这些功能来提升生成引擎优化。

### 添加具有描述性的页面元数据

在你的 frontmatter 中加入清晰的标题和描述：

```mdx
---
title: "API authentication guide"
description: "Complete guide to implementing API authentication with code examples"
---
```

### 配置全局索引设置

在你的 `docs.json` 中添加：

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "Your docs"
    }
  }
}
```

### LLMs.txt

LLMs.txt 文件可帮助 AI 系统理解你的文档结构，类似于站点地图之于搜索引擎。Mintlify 会为你的文档自动生成 LLMs.txt 文件，无需任何配置。

## 测试你的文档

使用与你的产品和文档相关的问题测试各类AI工具，查看它们对你文档的引用效果。

**向AI文档助手提出关于你文档的具体问题：**
- “如何使用此 API 配置认证？”
- “请按步骤带我完成安装流程”

**检查这些工具是否提供：**
- 正确的代码示例
- 准确的分步说明
