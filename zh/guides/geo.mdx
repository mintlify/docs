同时为传统搜索引擎和由 AI 驱动的回答引擎（如 ChatGPT、Perplexity 和 Google AI Overviews）优化你的文档。

生成引擎优化（GEO）通过完善的内容与结构化信息，侧重提升被 AI 系统引用的概率；而传统 SEO 则以提升搜索结果排名为目标。



<div id="geo-quickstart">
  ## 生成引擎优化快速上手
</div>

<div id="initial-setup">
  ### 初始设置
</div>

1. 在 `docs.json` 配置中**确保文档已开启索引**
2. **审查现有页面**是否缺少描述和标题

<div id="content-improvements">
  ### 内容改进
</div>

1. 在合适的页面**添加对比表**
2. **审查各级标题**以确保它们能回答常见问题
3. **加强相关主题之间的内部链接**
4. **使用 AI 工具测试**并验证准确性



<div id="geo-best-practices">
  ## 生成引擎优化最佳实践
</div>

总体而言，撰写得当、结构清晰的文档会具备出色的生成引擎优化。你仍应以用户为中心撰写内容；只要内容满足他们的需求，你就已迈向为 AI 工具优化的正确方向。产出真正有帮助的内容，而非为“优化而优化”，会同时受到传统搜索引擎和 AI 搜索引擎的青睐。

重点关注：

- 围绕用户需求，而非仅做关键词匹配
- 结构化、可快速扫读的信息
- 对问题作出直接回答

<div id="format-for-clarity">
  ### 为清晰而格式化
</div>

以下格式实践有助于 AI 工具解析并理解你的内容：

- 不要跳过标题层级（H1 → H2 → H3）
- 使用明确的对象名称，避免用“it”或“this”
- 为代码块标注所用编程语言
- 为图片提供具有描述性的替代文本（alt 文本）
- 链接相关概念，帮助 AI 理解其关联

<div id="answer-questions-directly">
  ### 直接回答问题
</div>

撰写能回应具体用户问题的内容：

- 各小节以核心结论开头
- 使用与常见搜索意图相匹配的描述性标题
- 将复杂主题拆解为编号步骤



<div id="mintlify-configuration">
  ## Mintlify 配置
</div>

使用以下功能来提升生成引擎优化。

<div id="add-descriptive-page-metadata">
  ### 添加具描述性的页面元数据
</div>

在 frontmatter 中填写清晰的标题和描述：

```mdx
---
title: "API 认证指南"
description: "实现 API 认证的完整指南，包含代码示例"
---
```

<div id="configure-global-indexing-settings">
  ### 配置全局索引设置
</div>

将以下内容添加到你的 `docs.json`：

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "你的文档"
    }
  }
}
```

<div id="llmstxt">
  ### 语言模型索引文件
</div>

语言模型索引文件可帮助 AI 系统理解你的文档结构，类似于站点地图之于搜索引擎。Mintlify 会为你的文档自动生成语言模型索引文件，无需任何配置。


<div id="testing-your-documentation">
  ## 测试你的文档
</div>

用与你的产品和文档相关的问题在各类 AI 工具中进行测试，评估它们对你的文档的引用质量。

**向文档助手就你的文档提出具体问题：**
- “如何使用此 API 配置认证？”
- “请按步骤带我完成安装流程”

**检查这些工具是否提供：**
- 正确的代码示例
- 准确的分步指引
