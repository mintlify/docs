同时为传统搜索引擎和由 AI 驱动的解答引擎（如 ChatGPT、Perplexity 和 Google AI Overviews）优化你的文档。

生成式引擎优化（GEO）侧重通过全面的内容与结构化信息获得 AI 系统的引用，而传统 SEO（搜索引擎优化）则以提升搜索结果排名为目标。



<div id="geo-quickstart">
  ## GEO 快速入门
</div>

<div id="initial-setup">
  ### 初始设置
</div>

1. 在你的 `docs.json` 设置中**确保文档可被索引**
2. **审查当前页面**，查找缺失的说明和标题

<div id="content-improvements">
  ### 内容优化
</div>

1. **在合适的页面添加对比表**
2. **审查标题**，确保能回答常见问题
3. **改进相关主题之间的内部链接**
4. **使用 AI 工具测试**以验证准确性



<div id="geo-best-practices">
  ## GEO 最佳实践
</div>

总体而言，撰写良好、结构清晰的文档通常会具备出色的 GEO。你仍应以用户为中心；只要内容能满足他们的需求，你就已经在迈向为 AI 工具优化的正确方向。创作真正有帮助的内容，而不是为“优化”而优化，无论是传统还是 AI 搜索引擎都会更青睐。

关注以下方面：

- 以用户需求为导向，而非单纯做关键词匹配
- 结构化、便于快速浏览的信息
- 直接回答问题

<div id="format-for-clarity">
  ### 通过格式提升清晰度
</div>

以下格式实践有助于 AI 工具解析并理解你的内容：

- 不要跳过标题层级（H1 → H2 → H3）
- 使用具体的对象名称，而非“it”或“this”
- 为代码块标注对应的编程语言
- 为图片提供具有描述性的替代文本（alt 文本）
- 链接到相关概念，帮助 AI 理解其关联

<div id="answer-questions-directly">
  ### 直接回答问题
</div>

撰写能回应特定用户问题的内容：

- 以核心结论开篇
- 使用与常见查询相匹配的描述性标题
- 将复杂主题拆解为编号步骤



<div id="mintlify-configuration">
  ## Mintlify 配置
</div>

使用这些功能优化 GEO。

<div id="add-descriptive-page-metadata">
  ### 添加具描述性的页面 metadata
</div>

在你的 frontmatter 中包含清晰的 title 和 description：

```mdx
---
title: "API 认证指南"
description: "实现 API 认证的完整指南，包含代码示例"
---
```

<div id="configure-global-indexing-settings">
  ### 配置全局索引编入设置
</div>

在你的 `docs.json` 中添加：

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "您的文档"
    }
  }
}
```

<div id="llmstxt">
  ### LLMs.txt
</div>

LLMs.txt 文件有助于 AI 系统理解你的文档结构，类似于站点地图之于搜索引擎。Mintlify 会为你的文档自动生成 LLMs.txt 文件，无需配置。


<div id="testing-your-documentation">
  ## 测试你的文档
</div>

使用围绕你的产品和文档的问题来测试各类 AI 工具，查看它们对你的文档引用得是否到位。

**向 AI 助手提出关于文档的具体问题：**
- “如何使用此 API 配置认证？”
- “请按步骤带我完成安装过程”

**检查这些工具是否提供：**
- 正确的代码示例
- 准确的分步指导
