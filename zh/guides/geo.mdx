同时为传统搜索引擎和由 AI 驱动的回答引擎（如 ChatGPT、Perplexity、Google AI Overviews）优化文档。

生成式引擎优化（GEO）侧重通过全面的内容和结构化信息提高被 AI 系统引用的概率，而传统 SEO（搜索引擎优化）则侧重搜索结果排名。



<div id="geo-quickstart">
  ## GEO 快速入门
</div>

<div id="initial-setup">
  ### 初始设置
</div>

1. 在 `docs.json` 中的设置里，**确保文档可被索引**
2. **检查当前页面**是否缺少 description 和 title

<div id="content-improvements">
  ### 内容优化
</div>

1. 在合适的页面**添加对比表**
2. **审查标题层级与措辞**，确保能回答常见问题
3. **优化内部链接**，串联相关主题
4. **使用 AI 工具进行测试**以验证准确性



<div id="geo-best-practices">
  ## GEO 最佳实践
</div>

总体而言，写作得当、结构清晰的文档会具备出色的 GEO。你仍应以用户为先；只要内容满足用户需求，你就已经走在为 AI 工具优化的正确道路上。创作真正有用的内容，而不是为“优化而优化”，会同时受到传统搜索引擎和 AI 搜索引擎的青睐。

重点关注：

- 面向用户需求的内容，而非关键词匹配
- 结构化、便于扫读的信息
- 对问题作出直接回答

<div id="format-for-clarity">
  ### 以清晰为目标的格式
</div>

以下格式实践有助于 AI 工具解析并理解你的内容：

- 不要跳过标题层级（H1 → H2 → H3）
- 使用具体的对象名称，而非“it”或“this”
- 为代码块标注对应的编程语言
- 为图片提供描述性的替代文本（alt 文本）
- 链接到相关概念，帮助 AI 理解关联

<div id="answer-questions-directly">
  ### 直接回答问题
</div>

撰写能解决特定用户问题的内容：

- 各小节以主要结论开头
- 使用与常见检索相匹配的描述性标题
- 将复杂主题拆解为编号步骤



<div id="mintlify-configuration">
  ## Mintlify 配置
</div>

使用这些功能来改进 GEO。

<div id="add-descriptive-page-metadata">
  ### 添加具有描述性的页面 metadata
</div>

在你的 frontmatter 中包含清晰的 title 和 说明：

```mdx
---
title: "API 认证指南"
description: "实现 API 认证的完整指南，包含代码示例"
---
```

<div id="configure-global-indexing-settings">
  ### 配置全局索引编入设置
</div>

将以下内容添加到你的 `docs.json`：

```json
{
  "seo": {
    "indexing": "all",
    "metatags": {
      "og:type": "website",
      "og:site_name": "您的文档"
    }
  }
}
```

<div id="llmstxt">
  ### LLMs.txt
</div>

LLMs.txt 文件可帮助 AI 系统理解你的文档结构，类似于站点地图之于搜索引擎。Mintlify 会为你的文档自动生成 LLMs.txt 文件，无需任何配置。


<div id="testing-your-documentation">
  ## 测试你的文档
</div>

针对你的产品和文档提出问题，测试各类 AI 工具，查看你的文档被引用得是否到位。

**向 AI 助手就你的文档提出具体问题：**
- “如何使用此 API 配置认证？”
- “请按步骤带我完成安装过程”

**检查这些工具是否提供：**
- 正确的代码示例
- 准确的分步指导
