Claude Code 是一款具备代理能力的命令行工具，可帮助你维护文档。它可以撰写新内容、审阅现有页面，并让文档保持最新。

你可以在项目中添加一个 `CLAUDE.md` 文件，并持续迭代完善，以训练 Claude Code 理解你的文档规范和工作流程。

## 入门

**前提条件：**
- 有效的 Claude 订阅（Pro、Max 或 API 访问权限）

**设置：**
1. 安装 Claude Code：
  ```bash
  npm install -g @anthropic-ai/claude-code
  ```
2. 进入文档目录。
3. （可选）将下方的 `CLAUDE.md` 文件添加到项目中。
4. 运行 `claude` 即可开始。

## CLAUDE.md 模板

在文档目录的根目录保存一个 `CLAUDE.md` 文件，帮助 Claude Code 理解你的项目。该文件会让 Claude Code 熟悉你的文档标准、偏好和工作流程。更多信息请参阅 Anthropic 文档中的[管理 Claude 的记忆](https://docs.anthropic.com/en/docs/claude-code/memory)。

复制此示例模板，或根据你的文档规范进行调整：

```mdx
# Mintlify documentation

## Working relationship
- 你可以对想法提出质疑——这有助于产出更好的文档。这样做时请引用来源并解释你的理由
- 一定要先询问以获得澄清，避免主观臆断
- 绝不要撒谎、猜测或捏造信息

## Project context
- 格式：带有 YAML frontmatter 的 MDX 文件
- 配置：docs.json 控制导航、主题与设置
- 组件：Mintlify 组件

## Content strategy
- 文档覆盖以用户成功为准——不多不少
- 优先保证信息的准确性与可用性
- 尽可能让内容保持长期有效
- 在添加新内容前先检索现有信息。除非有策略性理由，否则避免重复
- 检查既有模式并保持一致性
- 从最小可行的改动开始

## docs.json

- 构建 docs.json 文件与站点导航时，请参考 [docs.json 模式](https://mintlify.com/docs.json)

## Frontmatter requirements for pages
- title：清晰、可描述的页面标题
- description：用于 SEO/导航的简明摘要

## Writing standards
- 使用第二人称（“你”）
- 在操作性内容开头列出先决条件
- 发布前测试所有代码示例
- 与现有页面的风格与格式保持一致
- 同时包含基础与高级用例
- 所有代码块添加语言标签
- 所有图片提供替代文本（alt text）
- 内部链接使用相对路径

## Git workflow
- 提交时绝不要使用 --no-verify
- 开始前先确认如何处理未提交的更改
- 如无明确分支可用，请创建新分支
- 在开发过程中频繁提交
- 绝不要跳过或禁用 pre-commit 钩子

## Do not
- 任何 MDX 文件都不要省略 frontmatter
- 内部链接不要使用绝对 URL
- 不要包含未经测试的代码示例
- 不要做未经验证的假设——务必先询问以获得澄清
```

## 示例提示

设置好 Claude Code 后，试试这些提示，看看它如何帮助完成常见的文档任务。你可以直接复制粘贴这些示例，或根据你的具体需求进行调整。

### 将笔记转为完善的文档
将草稿转换为包含组件和前置信息（frontmatter）的规范 Markdown 页面。

**示例提示：**
```text wrap
Convert this text into a properly formatted MDX page: [paste your text here]
```

### 审查文档一致性
获取有关改进风格、格式和组件使用的建议。

**示例提示：**
```text wrap
Review the files in docs/ and suggest improvements for consistency and clarity
```

### 功能变更时更新文档
在产品演进时保持文档同步更新。

**示例提示：**
```text wrap
Our API now requires a version parameter. Update our docs to include version=2024-01 in all examples
```

### 生成完整的代码示例
创建包含错误处理的多语言示例。

**示例提示：**
```text wrap
Create code examples for [your API endpoint] in JavaScript, Python, and cURL with error handling
```

## 扩展 Claude Code

除了手动向 Claude Code 发出提示外，你还可以将其集成到现有工作流中。

### 使用 GitHub Actions 实现自动化
在代码变更时自动运行 Claude Code，以保持文档最新。你可以在拉取请求上触发文档审查，或在检测到 API 变更时更新示例。

### 多实例工作流
为不同任务使用独立的 Claude Code 会话——一个用于撰写新内容，另一个用于评审与质量保障。这样有助于保持一致性，并捕捉单一会话可能遗漏的问题。

### 团队协作
将你优化后的 `CLAUDE.md` 文件与团队共享，确保所有贡献者遵循一致的文档标准。团队通常会制定项目特定的提示词与工作流，作为其文档流程的一部分。

### 自定义命令
在 `.claude/commands/` 中创建可复用的斜杠命令，以处理针对你项目或团队的常用文档任务。
