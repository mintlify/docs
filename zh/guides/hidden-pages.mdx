隐藏页面会从你站点的导航中移除，但任何知道其 URL 的人仍可公开访问。

当你希望内容在站点上可访问，或可作为 AI 工具的上下文引用，但又不希望通过导航被发现时，请使用隐藏页面。

对于需要严格访问控制的内容，必须配置[身份验证](/zh/authentication-personalization/authentication-setup)。

如果你想仅对特定用户群体隐藏页面，请使用个性化来控制[页面可见性](/zh/authentication-personalization/overview#page-visibility)。

<div id="hiding-a-page">
  ## 隐藏页面
</div>

如果某个页面未包含在你的文档配置文件（`docs.json`）导航中，则该页面会被隐藏。要隐藏页面，请将其从导航结构中移除。

隐藏页面会根据其文件路径使用与常规页面相同的 URL 结构。例如，`guides/hidden-page.mdx` 可通过 `docs.yoursite.com/guides/hidden-page` 访问。

查看[隐藏页面示例](/zh/guides/hidden-page-example)。

<Note>
  某些导航元素（如侧边栏、下拉菜单和选项卡）在隐藏页面上可能会显示为空或出现布局偏移。
</Note>

<div id="hiding-a-group-of-pages">
  ## 隐藏一组页面
</div>

如果在 `docs.json`（文档配置文件）中将 `hidden` 属性设置为 `true`，该页面组将被隐藏：

```json highlight={4}
"groups": [
  {
    "group": "快速入门",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  },
  {
    "group": "指南",
    "pages": [
      "guides/hidden-page.mdx",
      "guides/hidden-groups.mdx"
    ]
  }
]
```

在此示例中，`Getting started` 分组被隐藏，而 `Guides` 分组可见。

<div id="hiding-a-tab">
  ### 隐藏选项卡
</div>

你也可以在 `docs.json` 文档配置文件中添加 `hidden` 属性来隐藏选项卡：

```json highlight={4}
"tabs": [
  {
    "tab": "主页",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  }
]
```

<div id="search-seo-and-ai-indexing">
  ## 搜索、SEO 与 AI 索引
</div>

默认情况下，隐藏页面不会被搜索引擎收录、不会出现在文档的内部搜索中，也不会作为文档助手的上下文。若要将隐藏页面纳入搜索结果并作为文档助手的上下文，请在你的 `docs.json` 中添加 `seo` 属性：

```json
"seo": {
    "indexing": "all"
}
```

如需排除某个特定页面，请在其 frontmatter 中添加 `noindex: true`。
