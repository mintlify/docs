title: "隐藏页面"
description: "将页面从导航中排除"

---

隐藏页面会从你的站点导航中移除，但任何知道其 URL 的人仍可公开访问。

当你希望某些内容在站点上可访问，或可作为 AI 工具的 context 进行引用，但又不希望通过导航被发现时，请使用隐藏页面。

对于需要严格访问控制的内容，你必须配置[认证](/zh/authentication-personalization/authentication-setup)。

如果你想仅对特定用户群体隐藏页面，请使用个性化功能来控制[页面可见性](/zh/authentication-personalization/overview#page-visibility)。

<div id="hiding-a-page">
  ## 隐藏页面
</div>

如果某个页面未包含在你的 `docs.json` navigation 中，则该页面将被隐藏。要隐藏页面，请将其从导航结构中移除。

隐藏页面会依据其文件路径使用与常规页面相同的 URL 结构。例如，`guides/hidden-page.mdx` 可通过 `docs.yoursite.com/guides/hidden-page` 访问。

查看[隐藏页面示例](/zh/guides/hidden-page-example)。

<Note>
  某些导航元素（如侧边栏、下拉菜单和选项卡）在隐藏页面上可能会显示为空或导致布局发生变化。
</Note>

<div id="hiding-a-group-of-pages">
  ## 隐藏一组页面
</div>

如果在 `docs.json` 文件中将 `hidden` 属性设置为 `true`，该页面组会被隐藏：

```json highlight={4}
"groups": [
  {
    "group": "快速开始",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  },
  {
    "group": "指南",
    "pages": [
      "guides/hidden-page.mdx",
      "guides/hidden-groups.mdx"
    ]
  }
]
```

在此示例中，`Getting started` 分组被隐藏，而 `Guides` 分组是可见的。

<div id="hiding-a-tab">
  ### 隐藏标签页
</div>

你也可以在 `docs.json` 文件中添加 `hidden` 属性来隐藏某个标签页：

```json highlight={4}
"tabs": [
  {
    "tab": "首页",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  }
]
```

<div id="search-seo-and-ai-indexing">
  ## 搜索、SEO 与 AI 索引编入
</div>

默认情况下，隐藏页面不会被搜索引擎收录、也不会出现在文档内的站内搜索结果中，且不会用于 AI 助手的上下文。若要将隐藏页面包含在搜索结果中并作为 AI 助手的上下文，请在你的 `docs.json` 中添加 `seo` 属性：

```json
"seo": {
    "indexing": "all"
}
```

如需排除某个特定页面，请在其 frontmatter 中添加 `noindex: true`。
