你可以选择使用我们的迁移服务，将文档迁移到 Mintlify。本指南介绍迁移流程、交付时间以及我们的质量保障机制。

<div id="what-to-expect-for-your-migration">
  ## 迁移期间你可以期待什么
</div>

迁移是在特定时间点对你的文档进行的一次性迁移，并且需要代码冻结。我们会迁移你的文档、API 参考页面、品牌元素（logo、颜色和字体）以及导航结构。迁移服务还包括设计或信息架构咨询，以及落地页或自定义集成等自定义组件开发。

你的迁移项目将包含一个用于查看进度的预发布环境链接，并在交付后提供 7 天的反馈期，用于处理与内容、资源或品牌相关的任何问题。

<div id="migration-process">
  ## 迁移流程
</div>

迁移流程是一个高度协作的过程。我们会与你的团队紧密合作，实施适合你的定制解决方案和构建！

<Steps>
  <Step title="提交迁移表单">
    提交[迁移表单](https://mintlify.typeform.com/pro-migrations)。

    提供你的文档来源、品牌素材（Logo、品牌色值、字体）、OpenAPI 规范、上线日期，以及首选交付方式（GitHub PR 或 zip 文件）。
  </Step>

  <Step title="启动会议">
    我们会审阅你的提交内容，并与迁移团队一起主持一次启动会议，详细了解你当前的配置、理解任何限制，并解答你团队可能有的所有问题。
  </Step>

  <Step title="接收范围界定与时间表（1–2 个工作日）">
    我们会基于启动会议的上下文进行审阅和分析，并在 24–48 小时内提供预估交付时间表。我们也会标记任何需要进一步讨论的自定义组件或迁移中的特殊部分。
  </Step>

  <Step title="在迁移过程中审查进度">
    我们开始迁移，并分享一个预发布环境链接，让你可以实时查看进度。
  </Step>

  <Step title="接收最终交付成果">
    我们会根据你的偏好，通过 GitHub PR 或 zip 文件交付迁移结果。
  </Step>

  <Step title="反馈期（七个工作日）">
    就内容、素材或品牌相关内容提交任何反馈。我们会在同一时间范围内处理完所有在约定范围内的问题。
  </Step>

  <Step title="上线支持">
    我们提供文档和指导，帮助你连接代码仓库、设置重定向，并配置自定义域名。
  </Step>
</Steps>

<div id="quality-assurance">
  ## 质量保证
</div>

我们与经过严格筛选的第三方服务商合作，并在交付前进行内部质量审查。我们会检查以下内容：

<Card title="内容完整性" icon="file-check">
  * 已迁移所有源文档中的内容。
  * 内容已使用 Mintlify 组件进行正确排版。
  * 导航结构与您的规范保持一致。
</Card>

<Card title="视觉元素" icon="palette">
  * 图片和视频显示正常。
  * 自定义字体在全站范围内应用。
  * 正确使用明暗模式对应的 Logo。
  * 配色方案在两种模式下都满足对比度要求。
</Card>

<Card title="功能性" icon="rocket">
  * 所有链接均可正常访问（无失效链接）。
  * 目录显示正常。
  * React 组件按预期工作。
  * 无性能问题。
</Card>

<div id="after-your-migration">
  ## 完成迁移之后
</div>

在我们完成迁移并交付给你之后，你需要完成以下步骤来上线你的文档站点：

1. [连接你的 GitHub 仓库并完成安装与配置](/zh/quickstart#install-the-github-app)。
2. [根据需要配置重定向](/zh/create/redirects)。
3. [配置你的自定义域名](/zh/customize/custom-domain)。
4. **上线前测试** - 在从旧文档切换之前：
   * 在预发布环境的链接中检查所有页面。
   * 测试导航和搜索功能。
   * 确认所有图片和视频都能正确加载。
   * 检查代码块是否正确渲染。
   * 分别在移动端和桌面端进行测试。
5. **上线！** 你的文档已经可以正式上线了！

<Note>
  **需要帮助？** 我们的迁移团队可以协助你完成这些步骤。你可以通过专属的 Slack 频道或 [support@mintlify.com](mailto:support@mintlify.com) 联系我们。
</Note>

<div id="frequently-asked-questions">
  ## 常见问题解答
</div>

<AccordionGroup>
  <Accordion title="迁移通常需要多长时间？">
    我们通常会在 2–6 周内完成迁移，具体取决于复杂度。启动会议之后，你会收到一份详细的时间表。
  </Accordion>

  <Accordion title="交付之后我还能提出修改请求吗？">
    可以。交付后的 7 天内，你可以就内容、素材或品牌元素提出修改请求。我们会在同一时间范围内处理所有在范围内的反馈，并与你合作直到你准备好正式上线。
  </Accordion>

  <Accordion title="迁移完成后我需要做什么？">
    迁移会转移你的内容，但你还需要：

    * 将你的文档仓库连接到 Mintlify 仪表盘。
    * 从旧的文档 URL 设置重定向。
    * 配置你的自定义域名。

    对于以上每一步，我们都会提供分步文档和支持。
  </Accordion>

  <Accordion title="我可以自定义导航结构吗？">
    可以。你可以提供自定义的信息架构，我们会为你实现。如果有需要且平台支持，我们也可以与你合作开发自定义组件。
  </Accordion>

  <Accordion title="如果在迁移过程中文档发生变化怎么办？">
    由于迁移需要代码冻结，在提交之后对源文档所做的任何更改都不会被包含在迁移中。我们建议在开始迁移之前完成所有更新。

    如果无法做到这一点，我们的团队会与你合作，安排短迭代来迁移特定部分，或者在第一轮迭代之后预留专门时间来迁移在迁移过程中产生的增量变更。
  </Accordion>
</AccordionGroup>
