您可以选择我们的迁移服务，将文档迁移到 Mintlify。本文将介绍迁移流程、交付时间表以及我们的质量保障方案。

<div id="what-to-expect-for-your-migration">
  ## 预期的迁移内容
</div>

迁移是在特定时间点对你的文档进行的一次性转移，在此期间需要进行代码冻结。我们会迁移你的文档、API 参考页面、品牌元素（标志、颜色和字体）以及导航结构。迁移服务还包括设计或信息架构咨询，以及落地页或自定义 integrations 等自定义组件的开发。

你的迁移服务将包含一个用于查看进度的预发布环境链接，以及交付后 7 天的反馈期，用于处理与文档、资源或品牌相关的任何问题。

<div id="migration-process">
  ## 迁移流程
</div>

迁移流程是一个高度协作的过程。我们会与你的团队紧密合作，实施完全契合你需求的自定义方案和构建方式。

<Steps>
  <Step title="提交迁移表单">
    提交[迁移表单](https://mintlify.typeform.com/pro-migrations)。

    提供你的文档来源、品牌资产（Logo、色彩代码、字体）、OpenAPI 规范、预计上线日期，以及首选交付方式（GitHub PR 或 zip 压缩包）。
  </Step>

  <Step title="启动会议">
    我们会审核你的提交内容，并由迁移团队牵头召开启动会议，逐步梳理你当前的配置情况、了解任何限制条件，并解答你团队可能有的任何问题。
  </Step>

  <Step title="范围与时间预估（1–2 个工作日）">
    我们会根据启动会议中的讨论内容进行审核和分析，并在 24–48 小时内提供预计交付时间表。我们会标注出任何需要进一步讨论的自定义组件或迁移中的关键区域。
  </Step>

  <Step title="迁移过程中的进度查看">
    我们开始执行迁移，并分享预发布环境链接，方便你实时查看进度。
  </Step>

  <Step title="接收最终交付">
    我们会根据你的偏好，通过 GitHub PR 或 zip 文件交付最终迁移结果。
  </Step>

  <Step title="反馈窗口（7 个工作日）">
    就内容、资产或品牌相关部分提交任何反馈。我们会在同一时间范围内解决所有约定范围内的问题。
  </Step>

  <Step title="上线支持">
    我们会提供文档和指导，帮助你连接存储库、设置重定向，并配置自定义域名。
  </Step>
</Steps>

<div id="quality-assurance">
  ## 质量保证
</div>

我们与经过严格筛选的第三方服务商合作，并在交付前进行内部质量审核。我们会重点检查以下内容：

<Card title="内容完整性" icon="file-check">
  * 你提供的所有文档都已完成迁移。
  * 内容已使用 Mintlify 组件进行正确排版。
  * 导航结构与您的规范保持一致。
</Card>

<Card title="视觉元素" icon="palette">
  * 图片和视频显示正常。
  * 自定义字体在全站正确应用。
  * 在浅色和深色模式下正确使用对应的 logo。
  * 配色方案在两种模式下都满足对比度要求。
</Card>

<Card title="功能性" icon="rocket">
  * 所有链接均可正常访问（无失效链接）。
  * 目录渲染正常。
  * React 组件按预期工作。
  * 无性能问题。
</Card>

<div id="after-your-migration">
  ## 迁移完成后
</div>

当我们完成并交付迁移后，你需要完成以下步骤来发布你的文档站点：

1. [连接你的 GitHub 存储库并完成配置](/zh/quickstart)。
2. [根据需要设置重定向](/zh/create/redirects)。
3. [配置你的自定义域名](/zh/customize/custom-domain)。
4. **上线前测试** - 在从旧文档切换之前：
   * 通过预发布环境链接检查所有页面。
   * 测试导航和搜索。
   * 确认所有图片和视频均可正常加载。
   * 检查代码块是否正确渲染。
   * 分别在移动端和桌面端进行测试。
5. **上线。** 你的文档已经准备好对外发布了。

<Note>
  **需要帮助？** 我们的迁移团队可以在整个过程中为你提供指导。请通过你的专属 Slack 频道或 [support@mintlify.com](mailto:support@mintlify.com) 联系我们。
</Note>

<div id="frequently-asked-questions">
  ## 常见问题解答
</div>

<AccordionGroup>
  <Accordion title="迁移通常需要多长时间？">
    我们通常会在 2–6 周内完成迁移，具体取决于复杂程度。启动会议结束后，你会收到一份具体时间表。
  </Accordion>

  <Accordion title="交付后我还能提出修改请求吗？">
    可以。交付后你有 7 天的反馈期，可以对文档内容、资产或品牌相关内容提出修改请求。我们会在同样的时间范围内处理所有在范围内的反馈，并全程与你合作，直到你准备好正式上线。
  </Accordion>

  <Accordion title="迁移完成后我需要做什么？">
    迁移会将你的文档内容迁移过来，但你还需要：

    * 将文档存储库连接到你的 Mintlify 控制台。
    * 为旧文档 URL 设置重定向。
    * 配置自定义 domain。

    我们为以上每一步都提供了详细的分步文档和支持。
  </Accordion>

  <Accordion title="我可以自定义导航结构吗？">
    可以。你可以提供自定义的信息架构，我们会为你实现。如果平台支持且有需要，我们也会与你合作开发自定义组件。
  </Accordion>

  <Accordion title="如果在迁移过程中文档发生变更怎么办？">
    由于迁移需要代码冻结，提交之后对源文档所做的任何更改都不会被包含在迁移中。我们建议在开始迁移之前完成所有更新。

    如果无法做到这一点，我们的团队会与你合作，设定短期冲刺来迁移特定部分，或在第一轮冲刺之后预留专门时间来迁移在迁移期间产生的差异变更。
  </Accordion>
</AccordionGroup>
