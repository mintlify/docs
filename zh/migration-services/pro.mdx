你的 Pro 方案包含迁移服务，用于将你的文档迁移到 Mintlify。本文档介绍迁移流程、交付时间，以及我们的质量保障方案。

<div id="what-to-expect-for-your-migration">
  ## 迁移期间可以期待什么
</div>

一次迁移是在某个时间点对你的文档进行转移的过程，在此期间需要进行代码冻结。我们会迁移你的文档、品牌（标志、颜色和字体）以及导航结构。

你的迁移将包含一个用于查看进度的预发布环境链接，以及交付后的 3 天反馈期，用于解决与内容、资源或品牌相关的任何问题。

<Info>
迁移服务**不**包含：
- 文档的动态同步
- 设计或信息架构咨询
- 自定义组件开发（登录页、自定义集成等）
- 重定向配置
- API playground 配置
- Analytics 配置
</Info>

<div id="migration-process">
  ## 迁移流程
</div>

<Steps>
  <Step title="提交迁移表单">
    提交 [迁移表单](https://mintlify.typeform.com/pro-migrations)。
    
    提供你的文档源、品牌资产（Logo、色值、字体）、OpenAPI 规范、计划上线日期，以及首选的交付方式（GitHub PR 或 zip 文件）。
  </Step>
  
  <Step title="获取范围评估与时间表（1-2 个工作日）">
    我们会审核你的提交，并在 24-48 小时内提供预估的交付时间表。我们会标注任何需要讨论的自定义组件或迁移中需要特别沟通的部分。
  </Step>
  
  <Step title="在迁移过程中查看进度">
    我们开始执行迁移，并共享一个预览环境链接，方便你实时查看进度。
  </Step>
  
  <Step title="接收最终交付结果">
    我们会根据你的偏好，通过 GitHub PR 或 zip 文件交付迁移结果。
  </Step>
  
  <Step title="反馈窗口期（3 个工作日）">
    提交你对内容、资产或品牌相关部分的任何反馈。我们会在同一时间范围内处理所有在范围内的问题。
  </Step>
  
  <Step title="上线支持">
    我们会提供文档和指导，帮助你连接代码仓库、设置重定向以及配置自定义域名。
  </Step>
</Steps>

<div id="quality-assurance">
  ## 质量保证
</div>

我们与经过严格筛选的第三方服务商合作，并在交付前进行内部质量审查。我们会重点检查以下内容：

<Card title="内容完整性" icon="file-check">
  - 已完成迁移您所有来源文档。
  - 内容已正确使用 Mintlify 组件进行排版。
  - 导航结构与您的规范保持一致。
</Card>

<Card title="视觉元素" icon="palette">
  - 图片和视频显示正常。
  - 自定义字体在全站范围内正确应用。
  - 浅色和深色模式下的 Logo 使用得当。
  - 配色在两种模式下都满足对比度要求。
</Card>

<Card title="功能性" icon="rocket">
  - 所有链接均可正常访问（无失效链接）。
  - 目录渲染正常。
  - React 组件按预期运行。
  - 无性能问题。
</Card>

<div id="after-your-migration">
  ## 迁移完成之后
</div>

当我们完成迁移并交付给你后，你需要完成以下步骤来正式上线文档站点：

1. [连接你的 GitHub 仓库并完成设置](https://www.mintlify.com/docs/quickstart#install-the-github-app)。
2. [如有需要，设置重定向](https://www.mintlify.com/docs/create/redirects)。
3. [配置你的自定义域名](https://www.mintlify.com/docs/customize/custom-domain)。
4. **正式上线前测试** - 在从旧文档切换过来之前：
   - 在预发布环境链接上检查所有页面。
   - 测试导航和搜索功能。
   - 确认所有图片和视频都能正常加载。
   - 检查代码块是否正确渲染。
   - 在移动端和桌面端进行测试。
5. **发布上线！** 你的文档已经可以对外开放！

<Note>
  **需要帮助？** 我们的支持团队可以随时协助你完成这些步骤。请通过 [support@mintlify.com](mailto:support@mintlify.com) 联系我们。
</Note>

<div id="frequently-asked-questions">
  ## 常见问题
</div>

<AccordionGroup>
  <Accordion title="一次迁移需要多长时间？">
    通常情况下，我们会在 10 个工作日内完成迁移，具体时间取决于复杂度。在你提交迁移表单后的 24–48 小时内，你会收到一份具体的时间安排。
  </Accordion>
  
  <Accordion title="交付之后我还能提出修改请求吗？">
    可以。交付后你有 3 天的反馈期，可以就内容、资源或品牌样式提出修改请求。我们会在同一时间范围内处理所有在服务范围内的反馈。
  </Accordion>
  
  <Accordion title="迁移完成后我需要做什么？">
    迁移会转移你的内容，但你还需要：
    - 将你的文档代码仓库连接到 Mintlify 控制台
    - 从旧的文档 URL 设置重定向
    - 配置你的自定义域名
    
    对于上述每一步，我们都会提供详细的分步文档和支持。
  </Accordion>
  
  <Accordion title="我可以自定义导航结构吗？">
    可以。你可以在迁移表单中提供自定义的信息架构，我们会为你实现。不过，自定义组件开发（例如自定义落地页）不包含在迁移服务内。
  </Accordion>
  
  <Accordion title="如果我的文档在迁移过程中发生变化会怎样？">
    由于迁移需要代码冻结，提交之后对源文档所做的任何更改都不会被包含在内。我们建议在开始迁移之前先完成所有更新。
  </Accordion>
</AccordionGroup>
