集成将 Mintlify agent 与团队已在使用的工具连接起来。Agent 可以在 Slack 中回复或运行自动化时使用已连接的应用。某些共享集成中的事件还可以启动自定义自动化。

| 使用场景 | 工作方式 |
| --- | --- |
| **Slack agent** | 使用组织的共享集成，以及正在与 agent 对话的团队成员所连接的个人集成。 |
| **自动化工具** | 从自动化 **Tools** 设置中选择的共享集成获取只读上下文。 |
| **自动化触发器** | 当受支持的共享集成中发生所选事件时，启动自定义自动化。 |

<Note>
  [Mintlify Slack 应用](/zh/agent/slack)与本页中的集成相互独立。安装 Slack 应用会将 agent 添加到工作区，并让 agent 和自动化运行访问该应用获准读取的 Slack 内容。你不需要在 **Integrations** 中连接 Slack，也不需要在自动化的 **Tools** 中选择它。
</Note>

<div id="connection-scope">
  ## 连接范围
</div>

- **共享集成**只需为整个组织连接一次。Slack agent 可以使用这些集成；你可以将其添加为自动化工具；如果集成提供事件，还可以用它触发自定义自动化。
- **个人集成**由每位团队成员单独连接。Slack agent 使用正在与其对话的团队成员的个人账户。个人集成不能作为工具或触发器添加到自动化。

例如，如果两位团队成员分别连接不同的 Google Drive 账户，他们在 Slack 中向 agent 提问时，只会获得各自 Drive 中的结果。

可以连接个人账户的团队成员数量没有限制。在有多位参与者的 Slack 线程中，agent 会使用每条消息发送者的个人集成。

<div id="connect-an-integration">
  ## 连接集成
</div>

管理员和编辑者可以为组织连接共享集成。每位团队成员连接自己的个人集成。

### 从控制台连接

1. 前往设置中的 [Integrations](https://app.mintlify.com/settings/organization/integrations) 页面。
2. 找到集成并点击 **Connect**。
3. 完成授权流程。

配置自动化时，你也可以在 **Tools** 中选择尚未连接的应用，并直接完成授权。

### 从 Slack agent 连接

让 agent 连接某个应用，例如“连接我的 Google Drive”。完成 agent 在线程中返回的授权链接，然后在同一线程中再次发送消息，让 agent 确认并使用该连接。

<div id="use-integrations-with-the-slack-agent">
  ## 在 Slack agent 中使用集成
</div>

安装 [Mintlify Slack 应用](/zh/agent/slack)后，agent 可以在回答问题或更新文档时使用集成。你不需要为每次 Slack 对话选择集成。

Agent 可以使用：

- 已为组织连接的共享集成。
- 发送请求的团队成员所连接的个人集成。
- Mintlify 应用获准读取的 Slack 频道和线程。

Agent 仅在集成数据与请求相关时访问这些数据。

<div id="use-integrations-as-automation-tools">
  ## 将集成用作自动化工具
</div>

自动化工具会在自动化启动后为 agent 提供额外上下文。只能选择共享集成。

1. 打开自动化设置。
2. 在 **Tools** 中选择该自动化可以使用的集成。
3. 连接任何尚未连接的已选集成。
4. 保存自动化。

用作触发器的集成会自动添加为该自动化的工具，以便 agent 在收到事件后检索更多详细信息。

<div id="trigger-an-automation-from-an-integration">
  ## 通过集成触发自动化
</div>

1. 创建或打开自定义自动化。
2. 在 **Trigger** 中选择 **Integration**。
3. 选择或连接一个共享集成。
4. 选择 **Trigger event**，并填写出现的其他事件字段。
5. 保存并启用自动化。

每个自动化只能有一个触发器。选择 **Integration** 会替代内容更新、代码变更或自定义计划触发器。

| 集成 | 事件 |
| --- | --- |
| Confluence | 页面创建；向空间添加内容 |
| Jira | Issue 创建；issue 更新；项目创建 |
| Linear | Issue 创建；issue 更新；项目创建；项目更新发布 |
| Notion | 页面创建；页面内容更新 |
| Zendesk | Ticket 创建 |

事件发生时，Mintlify 会启动自动化，并将事件数据作为 agent 的上下文。

<div id="permissions-and-access">
  ## 权限与访问
</div>

Agent 可用的第三方集成工具均为只读。Agent 可以搜索和检索内容，但不能使用这些工具在已连接的应用中创建、编辑或删除数据。

Agent 会继承授权连接的账户所拥有的访问权限。若要缩小访问范围，请限制该账户的权限，或只向 Mintlify 应用开放特定资源。连接应用之前，请在授权页面查看具体权限。

<div id="disconnect-an-integration">
  ## 断开集成
</div>

1. 前往 [Integrations](https://app.mintlify.com/settings/organization/integrations) 页面。
2. 点击该集成的 **Configure**。
3. 点击 **Disable integration**。

断开共享集成会为整个组织移除该集成，并停止将其用作触发器的自动化接收新事件。断开个人集成只会移除你自己的连接。

<div id="troubleshoot-a-connection">
  ## 排查连接问题
</div>

确认授权账户仍可访问相关内容，然后断开并重新连接集成，再次完成授权。如果该连接供 Slack agent 使用，请在原始 Slack 线程中再次发送消息。如果授权被阻止，请让应用或工作区管理员批准 Mintlify。
