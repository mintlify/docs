<Info>
  自动化集成需要 [Enterprise 计划](https://mintlify.com/pricing?ref=automation-integrations)。
</Info>

集成让[自动化](/zh/automations/index)可以访问团队使用的其他工具中的上下文。连接后，自动化可以搜索 Slack 线程、阅读 Notion 页面、查询 Linear 工单，并从其他应用中拉取上下文，以便完善其内容更新。

<div id="connection-scope">
  ## 连接范围
</div>

集成的连接方式有两种：

- **共享集成**为整个组织一次性连接，并可供你的自动化使用。Slack、Notion、Linear、Jira、Confluence、Salesforce、Intercom、HubSpot 和 Plain 都采用这种方式连接。
- **个人集成**按用户连接。每位团队成员授权自己的账户。Google Calendar 和 Google Drive 是个人集成。

个人集成将日历事件等用户专属数据限定在连接该集成的用户范围内，而不会在整个组织间共享。

<div id="connect-an-integration">
  ## 连接集成
</div>

管理员和编辑者可以代表组织连接共享集成。

1. 前往控制台设置中的[集成](https://app.mintlify.com/settings/organization/integrations)页面。
2. 找到你想要连接的集成。
3. 点击 **Connect**。
4. 按照 OAuth 提示授权 Mintlify 访问你的账户。

某些集成在连接完成之前会打开额外的配置步骤。Jira、Confluence 和 Salesforce 会在 OAuth 流程开始前要求提供你的工作区 URL 或子域名。Plain 等基于 API 密钥的提供方会提示你粘贴 API 密钥。

<div id="how-automations-use-integrations">
  ## 自动化如何使用集成
</div>

自动化将连接的集成作为工具使用。当自动化运行时，代理可以从你为该自动化启用的已连接应用中搜索并检索内容，为其更新构建上下文。

若要为自动化启用集成，请打开该自动化的设置，在 **Available integrations** 部分选择你希望代理使用的集成。只有已为你的组织连接的集成会出现在列表中。详情请参见[管理自动化](/zh/automations/manage#integrations)。

代理仅在集成数据与自动化任务相关时才会访问它。它不会主动读取已连接的应用。

<div id="permissions-and-access">
  ## 权限与访问
</div>

每个集成都会请求其搜索和检索连接应用内容所需的 OAuth 范围或 API 密钥权限。只要提供方提供只读范围，Mintlify 都会请求只读范围。

在授权连接之前，请在 OAuth 同意屏幕上查看具体的权限范围。同意屏幕会列出集成获得的每项权限，你可以随时在控制台或已连接应用自身的集成设置中撤销该连接。

如果你的组织限制了用户可授权的应用，请让相关管理员（例如你的 Slack 或 Google Workspace 管理员）先允许 Mintlify 集成，然后再进行连接。

<div id="what-the-agent-can-read-from-each-app">
  ### 代理可从每个应用读取的内容
</div>

代理继承授权连接的账户所拥有的访问权限。它只能读取授权用户（对于个人集成）或已连接的工作区账户（对于共享集成）在源应用中已有访问权限的内容。

| 集成 | 代理可访问的内容 |
| --- | --- |
| Confluence | 已连接账户可读取的空间中的页面和附件 |
| Google Calendar | 已连接用户可查看的日历中的事件 |
| Google Drive | 已连接用户可查看的文件和文件夹，包括共享云端硬盘 |
| HubSpot | 已连接账户可读取的 CRM 记录（联系人、公司、交易、工单） |
| Intercom | 已连接账户可读取的对话、联系人和帮助中心文章 |
| Jira | 已连接账户可读取的项目中的问题和评论 |
| Linear | 已连接账户可读取的团队中的问题、项目和评论 |
| Notion | 明确与 Mintlify 集成共享的页面和数据库 |
| Plain | 使用所提供 API 密钥可访问的线程和客户记录 |
| Salesforce | 已连接账户可读取的标准对象和自定义对象 |
| Slack | 工作区授予访问权限的公共频道，以及你将 Mintlify 应用添加到的私有频道 |

若要缩小代理可见的范围，请在源应用中限制已连接账户的权限。例如，仅将 Mintlify Slack 应用邀请到特定频道，或仅将选定的 Notion 页面与该集成共享。

### 在 Atlassian 中批准 Mintlify 应用（Jira 和 Confluence）

如果你的 Atlassian 组织限制了 Marketplace 或自定义应用，Atlassian 管理员必须先批准 Mintlify 应用，任何人才能连接 Jira 或 Confluence。未经批准就尝试连接的用户会在 OAuth 同意屏幕上看到授权错误。

**应用身份：**

* **App name：** Mintlify
* **App ID：** `872a4b8e-9e83-474d-99f9-dc79112da145`

**管理员步骤：**

1. 前往 [admin.atlassian.com](https://admin.atlassian.com) 并打开你的组织。
2. 前往 **Security** > **Data protection** > **Data security policy**。
3. 检查 **Marketplace and custom app access control** 设置。如果显示为 **Blocked** 或 **Custom**，则需要允许 Mintlify 应用。
4. 编辑该策略，将该控制项设置为 **Allowed**，或使用上述 App ID 为 Mintlify 应用添加覆盖设置。
5. 点击 **Activate** 应用该策略。

在策略生效后，管理员或编辑者即可从[集成](https://app.mintlify.com/settings/organization/integrations)页面完成 Jira 或 Confluence 连接。

#### Mintlify 请求的 OAuth 范围

Mintlify 使用 Atlassian 的 OAuth 2.0 (3LO) 流程，并请求只读范围，以便自动化代理搜索并读取用于支撑其更新的工单或页面。Mintlify 不会请求写入、删除或管理员范围。

**Jira 范围：**

| 范围                          | Mintlify 需要它的原因                                                        |
| --------------------------- | ---------------------------------------------------------------------- |
| `read:jira-user`            | 将问题作者、经办人和评论者解析为可读的用户名                                                 |
| `read:jira-work`            | 读取已连接账户可见的问题、评论、项目以及 JQL 搜索结果                                          |
| `offline_access`            | 刷新 OAuth 访问令牌，使连接无需重新授权即可持续可用                                          |

**Confluence 范围：**

| 范围                               | Mintlify 需要它的原因                                                        |
| -------------------------------- | ---------------------------------------------------------------------- |
| `read:confluence-content.all`    | 读取已连接账户可访问空间中的页面正文和附件                                                  |
| `read:confluence-content.summary`| 在获取完整内容之前搜索并预览页面                                                       |
| `read:confluence-space.summary`  | 列出并解析已连接账户可访问的空间                                                       |
| `read:confluence-user`           | 将页面作者和编辑者解析为可读的用户名                                                     |
| `offline_access`                 | 刷新 OAuth 访问令牌，使连接无需重新授权即可持续可用                                          |

授予该连接的完整范围列表会在用户首次授权集成时显示在 Atlassian OAuth 同意屏幕上，也可以随时在你的 Atlassian 管理员控制台的 **Connected apps** 中查看。

#### Mintlify 如何处理这些数据

* **访问为只读。** Mintlify 绝不会创建、编辑、转换或删除问题、评论、页面或附件。
* **访问继承自授权用户。** 代理只能看到已连接的 Atlassian 账户已有访问权限的项目和空间。若要缩小可见范围，请在 Atlassian 中限制已连接账户的项目和空间权限。
* **访问按需进行。** 只有在自动化运行需要相关上下文时，代理才会从 Atlassian 读取数据。Mintlify 不会抓取 Jira 或 Confluence，也不会将其镜像为持久副本。
* **撤销立即生效。** 在 Mintlify 控制台断开集成，或在 Atlassian 管理员控制台撤销该应用，都会立即切断后续访问。

<div id="data-retention-after-disconnect">
  ## 断开连接后的数据保留
</div>

当你断开集成连接时，自动化会立即失去从该应用读取的能力。Mintlify 不会长期保留集成内容的副本。代理在每次运行时都会通过实时的 OAuth 令牌或 API 密钥获取数据，因此断开连接会切断未来的访问。

过去自动化运行或拉取请求中出现的内容会保留在原位（已发布的页面、PR 历史记录）。如果你也想删除该内容，请直接编辑或删除相关页面。

<div id="troubleshoot-a-failed-connection">
  ## 排查连接失败问题
</div>

如果集成停止工作或在控制台中显示错误，通常是因为底层 OAuth 令牌已过期、已连接的应用撤销了该令牌，或其所需范围发生了更改。

若要恢复访问：

1. 前往控制台设置中的[集成](https://app.mintlify.com/settings/organization/integrations)页面。
2. 找到受影响的集成并点击 **Configure**。
3. 点击 **Disconnect**。
4. 点击 **Connect**。
5. 完成 OAuth 流程。

对于 Plain 等基于 API 密钥的提供方，请在源应用中生成新的 API 密钥，并将其粘贴到集成的配置中。如果重新连接后自动化仍然无法访问数据，请确认授权用户在源应用中有权限查看相关内容。

<div id="disconnect-an-integration">
  ## 断开集成
</div>

1. 前往控制台设置中的[集成](https://app.mintlify.com/settings/organization/integrations)页面。
2. 找到你想要断开的已连接集成。
3. 点击 **Configure**。
4. 点击 **Disconnect**。

断开共享集成会为组织中的所有人移除该集成，自动化会立即失去对该工具的访问权限。

断开个人集成只会移除你自己的连接。已连接同一集成的其他团队成员保留他们的访问权限。
