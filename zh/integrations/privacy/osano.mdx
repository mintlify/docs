在你的 `docs.json` 文件中添加以下内容以接入 [Osano](https://www.osano.com/) 的 Cookie 同意管理器。

<CodeGroup>
  ```json Integration options in docs.json
  "integrations": {
      "osano": "SOURCE"
  }
  ```

  ```json 示例
  "integrations": {
      "osano": "https://cmp.osano.com/2sUB2dqwqdkks/8dqwd-dwd86£-4a9b/osano.js"
  }
  ```
</CodeGroup>

`SOURCE` 可在 Osano 生成的代码片段中作为 `src` 的值找到。它始终以 `https://cmp.osano.com/` 开头。

```html Code snippet from Osano
<script src="https://cmp.osano.com/placeholder/placeholder/osano.js"/>
```

  ## 故障排查
</div>

<Accordion
  title="在 Strict 合规模式下页面无法加载"
>
  如果在使用 Osano 的 **Strict** 合规模式时文档页面无法正常加载，你需要将 Mintlify 的 domain 加入允许列表，以允许图片和其他资源加载。

  <Steps>
    <Step title="前往 Managed Rules">
      在 Osano 控制台中，进入 **Scripts** → **Managed Rules**。
    </Step>

    <Step title="添加 Mintlify domain">
      将 `.mintlify.app/` 添加为一条 managed rule。

      <Frame>
        <img src="/images/integrations/osano-managed-rule.png" alt="Osano managed rule" />
      </Frame>

      <Info>
      这可确保所有由 Mintlify 提供的资产（包括图片、样式表和其他文档资源）被视为必要内容，即使 Osano 阻止未分类的第三方内容时也会加载。
      </Info>
    </Step>
  </Steps>
</Accordion>
