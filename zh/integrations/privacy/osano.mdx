在你的 `docs.json` 文件中添加以下内容，以集成 [Osano](https://www.osano.com/) Cookie 同意管理器。

<CodeGroup>
  ```json Integration options in docs.json
  "integrations": {
      "osano": "SOURCE"
  }
  ```

  ```json Example
  "integrations": {
      "osano": "https://cmp.osano.com/2sUB2dqwqdkks/8dqwd-dwd86£-4a9b/osano.js"
  }
  ```
</CodeGroup>

`SOURCE` 可在 Osano 生成的代码片段中 `src` 的值里找到。它始终以 `https://cmp.osano.com/` 开头。

```html 来自 Osano 的代码片段
<script src="https://cmp.osano.com/placeholder/placeholder/osano.js"/>
```

<Accordion
  title="在严格合规模式下页面无法加载"
>
  如果在使用 Osano 的 **Strict** 合规模式时文档页面无法正常加载，您需要将 Mintlify 的域名加入白名单，以允许图片和其他资源加载。

  <Steps>
    <Step title="前往 Managed Rules">
      在您的 Osano 控制台中，依次进入 **Scripts** → **Managed Rules**。
    </Step>

    <Step title="添加 Mintlify 域名">
      将 `.mintlify.app/` 添加为一条托管规则。

      <Frame>
        <img src="/images/integrations/osano-managed-rule.png" alt="Osano 托管规则" />
      </Frame>

      <Info>
      这样可确保所有由 Mintlify 提供的资源（包括图片、样式表及其他文档资源）被视为必要内容，即使 Osano 阻止未分类的第三方内容，也仍会加载。
      </Info>
    </Step>
  </Steps>
</Accordion>
