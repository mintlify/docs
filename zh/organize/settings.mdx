`docs.json` 文件是 Mintlify 文档站点的核心配置文件。它控制站点的全局设置，包括视觉品牌、导航结构、集成、API 设置等。可以把它看作站点的蓝图。

## 必填字段

你必须定义四个字段才能构建一个可用的站点。

| 字段 | 描述 |
|---|---|
| `name` | 你的项目或组织名称 |
| `theme` | 站点的布局 [主题](/zh/customize/themes) |
| `colors.primary` | 主品牌颜色（十六进制代码） |
| `navigation` | 你的内容结构 |

所有其他字段都是可选的。在自定义和完善站点时逐步添加。

## 最小配置

为获得最佳编辑体验，请在 `docs.json` 顶部包含 `$schema` 引用。这将在大多数编辑器中启用自动补全、验证和内联文档。

```json docs.json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Your project name",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": [
    {
      "group": "Home",
      "pages": ["index"]
    }
  ]
}
```

## 从 `mint.json` 升级

如果你的项目使用已弃用的 `mint.json` 文件，请按照以下步骤升级到 `docs.json`。

<Steps>
  <Step title="安装或更新 CLI">
    如果你还没有安装 [CLI](/zh/installation)，现在安装：

    <CodeGroup>

    ```bash npm
    npm i -g mint
    ```

    ```bash yarn
    yarn global add mint
    ```

    ```bash pnpm
    pnpm add -g mint
    ```

    </CodeGroup>

    如果你已经安装了 CLI，请确保它是最新的：

    ```bash
    mint update
    ```
  </Step>
  <Step title="创建你的 docs.json 文件">
    在你的文档仓库中，运行：

    ```bash
    mint upgrade
    ```

    此命令从你现有的 `mint.json` 创建一个 `docs.json` 文件。检查生成的文件以确保所有设置都正确。
  </Step>
  <Step title="删除你的 mint.json 文件">
    确认 `docs.json` 配置正确后，你可以安全地删除旧的 `mint.json` 文件。
  </Step>
</Steps>
