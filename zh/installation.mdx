<div id="installing-the-cli">
  ## 安装命令行界面（CLI）
</div>

<Info>
  **前置条件**：在继续之前，请先安装 [Node.js](https://nodejs.org/en)。
</Info>

<Steps>
  <Step title="安装 CLI。">
    运行以下命令安装 [CLI](https://www.npmjs.com/package/mint)：

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>
  </Step>

  <Step title="本地预览。">
    进入你的文档目录（`docs.json` 文件所在位置），并执行以下命令：

    ```bash
    mint dev
    ```

    然后在 `http://localhost:3000` 查看文档的本地预览。
  </Step>
</Steps>

或者，如果你不想全局安装 CLI，可以运行一次性脚本：

```bash
npx mint dev
```

<div id="updates">
  ## 更新
</div>

如果本地预览与线上生产环境显示不一致，请更新你的本地命令行界面（CLI）：

```bash
mint update
```

如果你的本地环境中没有可用的 `mint update` 命令，请重新安装最新版本的命令行界面（CLI）：

<CodeGroup>
  ```bash npm
  npm i -g mint@latest
  ```

  ```bash pnpm
  pnpm add -g mint@latest
  ```
</CodeGroup>

<div id="custom-ports">
  ## 自定义端口
</div>

默认情况下，命令行界面（CLI）使用 3000 端口。你可以通过 `--port` 标志来自定义端口。比如，要在 3333 端口运行 CLI，请使用以下命令：

```bash
mint dev --port 3333
```

如果你尝试使用已占用的端口运行，它会改用下一个可用端口：

```mdx
端口 3000 已被占用。正在尝试端口 3001。
```

<div id="previewing-as-a-specific-group">
  ## 以特定用户组进行预览
</div>

如果你使用部分认证来限制文档访问，可以通过 `--group [groupname]` 标志以特定认证用户组的身份进行预览。

例如，如果你有一个名为 `admin` 的用户组，可以使用以下命令以该组成员的身份进行预览：

```bash
mint dev --group admin
```

<div id="additional-commands">
  ## 其他命令
</div>

虽然 `mint dev` 是最常用的命令，但你也可以使用其他命令来管理文档。

<div id="finding-broken-links">
  ### 查找损坏链接
</div>

命令行界面（CLI）可帮助验证文档中的引用链接。要检测任何损坏的链接，请使用以下命令：

```bash
mint broken-links
```

<div id="checking-openapi-spec">
  ### 检查 OpenAPI 规范
</div>

你可以使用命令行界面（CLI）运行以下命令来检查 OpenAPI 文件是否有错误：

```bash
mint openapi-check <openapi文件名或URL>
```

你可以传入文件名（例如 `./openapi.yaml`）或 URL（例如 `https://petstore3.swagger.io/api/v3/openapi.json`）。

<div id="renaming-files">
  ### 重命名文件
</div>

你可以使用以下命令重命名文件，并更新对这些文件的所有引用：

```bash
mint rename <旧文件名> <新文件名>
```

<div id="migrating-mdx-endpoint-pages">
  ### 迁移 MDX 端点页面
</div>

你可以使用以下命令，将 MDX 端点页面基于你的 OpenAPI 规范迁移为自动生成的页面：

```bash
mint migrate-mdx
```

此命令会将单个 MDX 端点页面转换为在 `docs.json` 中定义的自动生成页面，将 MDX 内容移至 OpenAPI 规范中的 `x-mint` 扩展，并更新导航。详见[从 MDX 迁移](/zh/api-playground/migrating-from-mdx)以获取详细信息。

<div id="formatting">
  ## 格式化
</div>

在本地开发时，建议在你的 IDE 中安装扩展以识别并格式化 `MDX` 文件。

如果你使用 Cursor、Windsurf 或 VS Code，我们推荐用于语法高亮的 [MDX VS Code extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx)，以及用于代码格式化的 [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode)。

如果你使用 JetBrains，我们推荐用于语法高亮的 [MDX IntelliJ IDEA plugin](https://plugins.jetbrains.com/plugin/14944-mdx)，并配置 [Prettier](https://prettier.io/docs/webstorm) 进行代码格式化。

<div id="troubleshooting">
  ## 疑难解答
</div>

<AccordionGroup>
  <Accordion title="Error: Could not load the &#x22;sharp&#x22; module using the darwin-arm64 runtime">
    这可能是由于 Node 版本过旧。请尝试以下操作：

    1. 卸载当前安装的 mint 命令行界面（CLI）：`npm uninstall -g mint`
    2. 升级到最新的 Node.js。
    3. 重新安装 mint 命令行界面（CLI）：`npm install -g mint`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
    **解决方案**：进入用户根目录并删除 `~/.mintlify` 文件夹。然后再次运行 `mint dev`。
  </Accordion>

  <Accordion title="Error: permission denied">
    这是由于缺少全局安装 Node 包所需的权限。

    **解决方案**：尝试运行 `sudo npm i -g mint`。系统会提示你输入用于解锁电脑的密码。
  </Accordion>

  <Accordion title="The local preview doesn't look the same as my docs do on the web">
    这很可能是因为 CLI 版本过旧。

    **解决方案：** 运行 `mint update` 获取最新更新。
  </Accordion>

  <Accordion title="mintlify vs. mint package">
    如果你在使用 CLI 包时遇到问题，首先运行 `npm ls -g`。该命令会显示你的机器上全局安装的包。

    如果你不使用 npm，或在 -g 列表中未看到它，请尝试运行 `which mint` 来定位安装位置。

    如果你同时安装了名为 `mint` 和 `mintlify` 的包，应卸载 `mintlify`。

    1. 卸载旧包：

    ```bash
      npm uninstall -g mintlify
    ```

    2. 清理 npm 缓存：

    ```bash
      npm cache clean --force
    ```

    3. 重新安装新包：

    ```bash
    npm i -g mint
    ```
  </Accordion>
</AccordionGroup>
