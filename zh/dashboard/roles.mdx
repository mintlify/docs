Mintlify 提供三种控制台访问级别：Viewer、Editor 和 Admin。

## 各角色权限

下表描述了每个角色的权限：

|                         | Viewer | Editor | Admin |
| ----------------------- | :----: | :----: | :---: |
| 查看文档                |   ✅   |   ✅   |  ✅   |
| 查看分析数据            |   ✅   |   ✅   |  ✅   |
| 编辑内容                |   ❌   |   ✅   |  ✅   |
| 发布更改                |   ❌   |   ✅   |  ✅   |
| 更新用户角色            |   ❌   |   ❌   |  ✅   |
| 删除用户                |   ❌   |   ❌   |  ✅   |
| 邀请用户                |   ❌   |   ✅   |  ✅   |
| 管理并更新账单          |   ❌   |   ❌   |  ✅   |
| 更新自定义 domain       |   ❌   |   ❌   |  ✅   |
| 更新 Git source         |   ❌   |   ❌   |  ✅   |
| 删除组织                |   ❌   |   ❌   |  ✅   |

## 角色说明

**Viewer**：只读访问权限，可查看文档和分析数据。无法对内容或设置进行任何更改。

**Editor**：可以查看和编辑文档内容、发布更改以及邀请其他用户。无法修改组织设置或管理账单。

**Admin**：对所有控制台功能拥有完全访问权限，包括组织设置、账单和用户管理。

## 权限提升规则

启用 RBAC 后，用户无法邀请拥有自己没有的角色的其他用户。例如：

- Viewer 无法邀请任何用户
- Editor 可以邀请 Viewer 和 Editor，但不能邀请 Admin
- Admin 可以邀请任何角色的用户

你可以邀请任意数量的管理员，但建议仅将管理员访问权限授予确有需要的用户。
