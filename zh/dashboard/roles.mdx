</Info>

Mintlify 提供三种控制台访问级别：Viewer、Editor 和 Admin。

## 访问级别

- **Viewer**：对文档的只读访问权限。Viewer 可以查看所有内容，但无法进行更改或访问管理功能。
- **Editor**：可以创建、编辑和发布文档内容。
- **Admin**：对所有控制台功能的完全访问权限，包括用户管理、账单和组织设置。

## 权限

下表描述了每个角色的权限：

|                         | Viewer | Editor | Admin |
| ----------------------- | :----: | :----: | :---: |
| 查看文档                |   ✅   |   ✅   |  ✅   |
| 编辑内容                |   ❌   |   ✅   |  ✅   |
| 更新用户角色            |   ❌   |   ❌   |  ✅   |
| 删除用户                |   ❌   |   ❌   |  ✅   |
| 邀请用户                |   ❌   |   ✅   |  ✅   |
| 管理并更新账单          |   ❌   |   ❌   |  ✅   |
| 更新自定义 domain       |   ❌   |   ❌   |  ✅   |
| 更新 Git source         |   ❌   |   ❌   |  ✅   |
| 删除组织                |   ❌   |   ❌   |  ✅   |

## 权限提升规则

启用 RBAC 后，用户无法邀请拥有他们自己没有的角色的其他用户。例如：

- Viewer 无法邀请任何用户
- Editor 可以邀请 Viewer 和 Editor，但不能邀请 Admin
- Admin 可以邀请任何角色的用户

这可以防止权限提升，并确保组织内的适当访问控制。

你可以邀请任意数量的用户，但建议仅将管理员访问权限授予需要进行组织管理的用户。
