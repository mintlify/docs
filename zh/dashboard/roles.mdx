<Info>
  基于角色的访问控制（RBAC）在 [Enterprise 方案](https://mintlify.com/pricing?ref=rbac) 中提供。
</Info>

Mintlify 提供三种控制台访问级别：查看者、编辑者和管理员。

|                                | 查看者 | 编辑者 | 管理员 |
| ------------------------------ | :----: | :----: | :----: |
| 在编辑器中查看内容             |   ✅   |   ✅   |   ✅   |
| 查看控制台和数据分析           |   ✅   |   ✅   |   ✅   |
| 向 [Slack agent](/zh/agent/slack) 提问 |   ✅   |   ✅   |   ✅   |
| 在网页编辑器中编辑内容         |   ❌   |   ✅   |   ✅   |
| 通过 [Slack agent](/zh/agent/slack) 修改文档 |   ❌   |   ✅   |   ✅   |
| 发布更改                       |   ❌   |   ✅   |   ✅   |
| 创建、重命名和删除页面         |   ❌   |   ✅   |   ✅   |
| 调整导航顺序                   |   ❌   |   ✅   |   ✅   |
| 上传媒体文件                   |   ❌   |   ✅   |   ✅   |
| 触发部署                       |   ❌   |   ✅   |   ✅   |
| 创建和删除 API 密钥            |   ❌   |   ✅   |   ✅   |
| 配置集成                       |   ❌   |   ✅   |   ✅   |
| 配置助手                       |   ❌   |   ✅   |   ✅   |
| 配置附加组件                   |   ❌   |   ✅   |   ✅   |
| 邀请编辑者和查看者             |   ❌   |   ✅   |   ✅   |
| 更新用户角色                   |   ❌   |   ❌   |   ✅   |
| 邀请管理员                     |   ❌   |   ❌   |   ✅   |
| 删除用户                       |   ❌   |   ❌   |   ✅   |
| 管理账单                       |   ❌   |   ❌   |   ✅   |
| 管理 SSO                       |   ❌   |   ❌   |   ✅   |
| 更新自定义域名                 |   ❌   |   ❌   |   ✅   |
| 更新 Git 源                    |   ❌   |   ❌   |   ✅   |
| 删除组织                       |   ❌   |   ❌   |   ✅   |

<Tip>
  仅将管理员访问权限授予需要执行管理员任务的成员。
</Tip>

<div id="add-members-to-your-organization">
  ## 向你的组织添加成员
</div>

默认情况下，创建 Mintlify 组织的人拥有管理员权限。你可以在控制台的 [成员](https://dashboard.mintlify.com/settings/organization/members) 页面添加其他编辑。

你的组织中可以根据需要添加任意数量的成员。成员席位可能会产生额外费用。更多详情请参阅[定价页面](https://mintlify.com/pricing)。

<div id="accept-an-invitation">
  ## 接受邀请
</div>

当管理员邀请你加入他们的组织时，Mintlify 会向他们在 Members 页面中添加的邮箱发送邀请邮件。按照以下步骤加入团队。

<Steps>
  <Step title="打开邀请邮件">
    查找一封来自 Mintlify 的邮件，主题会邀请你加入该组织。如果收件箱中找不到，请检查垃圾邮件文件夹。
  </Step>
  <Step title="创建 Mintlify 账户或登录">
    点击邮件中的链接。如果你还没有 Mintlify 账户，系统会提示你创建一个。**请使用邀请所发送到的同一邮箱地址** — 使用其他邮箱注册会导致邀请无法与你的账户关联。

    新账户会自带一个初始组织。这是正常的 — 你将在下一步加入邀请你的团队。
  </Step>
  <Step title="在控制台中接受邀请">
    登录后，打开 [控制台](https://dashboard.mintlify.com)，使用控制台顶部的 **Accept organization invitations** 切换器接受待处理的邀请。接受后，你将根据管理员为你分配的角色，获得对邀请组织的文档和设置的访问权限。
  </Step>
</Steps>

<div id="switch-between-organizations">
  ### 在组织之间切换
</div>

如果你属于多个组织（例如你自己的初始组织加上你加入的团队），可使用控制台中的组织切换器在它们之间切换。每个组织都有各自的项目、成员和设置。

<div id="troubleshooting">
  ### 故障排查
</div>

- **邀请链接打开的会话属于错误的组织。** 如果你的浏览器已登录到个人或其他 Mintlify 账户，请在无痕或隐私窗口中打开邀请链接，并使用邀请所发送到的邮箱地址登录。
- **看不到 Accept organization invitations 切换器。** 请确认你使用的正是邀请所发送到的邮箱地址登录。如果管理员邀请的是另一个地址（例如工作邮箱而非个人邮箱），请让他们将邀请重新发送到你 Mintlify 账户所用的地址，或使用被邀请的地址创建账户。
- **邀请已过期或你删除了邮件。** 请让邀请你的组织的管理员从 [Members](https://dashboard.mintlify.com/settings/organization/members) 页面重新发送邀请。
