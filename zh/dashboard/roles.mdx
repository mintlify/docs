Mintlify 提供三种控制台访问级别：Viewer、Editor 和 Admin。

## 权限

以下为按角色限制的操作：

|                         | Viewer | Editor | Admin |
| ----------------------- | :----: | :----: | :---: |
| 查看文档                |   ✅   |   ✅   |  ✅   |
| 编辑文档                |   ❌   |   ✅   |  ✅   |
| 更新用户角色            |   ❌   |   ❌   |  ✅   |
| 删除用户                |   ❌   |   ❌   |  ✅   |
| 邀请管理员              |   ❌   |   ❌   |  ✅   |
| 管理并更新账单          |   ❌   |   ❌   |  ✅   |
| 更新自定义 domain       |   ❌   |   ❌   |  ✅   |
| 更新 Git source         |   ❌   |   ❌   |  ✅   |
| 删除组织                |   ❌   |   ❌   |  ✅   |

### Viewer 角色

Viewer 角色提供对控制台的只读访问权限。拥有此角色的用户可以查看文档和分析数据，但无法对内容或设置进行任何更改。

### Editor 角色

Editor 角色允许用户查看和编辑文档内容。编辑者可以对文档进行更改，但无法管理用户、账单或组织设置。

### Admin 角色

Admin 角色拥有对所有控制台功能的完全访问权限，包括用户管理、账单和组织设置。

## 权限提升规则

启用 RBAC 后，用户无法邀请拥有他们自己没有的角色的其他用户。例如：

- Viewer 无法邀请任何用户
- Editor 无法邀请 Admin
- Admin 可以邀请拥有任何角色的用户

你可以邀请任意数量的管理员，但建议仅将管理员访问权限授予确有需要的用户。
