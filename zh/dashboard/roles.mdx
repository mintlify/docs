<Info>
  基于角色的访问控制（RBAC）在 [Enterprise 方案](https://mintlify.com/pricing?ref=rbac) 中提供。
</Info>

Mintlify 提供三种 dashboard 访问级别：查看者、编辑者和管理员。

| 查看 dashboard 和数据分析      |   ✅   |   ✅   |  ✅   |
| 向 [Slack agent](/zh/agent/slack) 提问 |   ✅   |   ✅   |  ✅   |
| 使用 Slack agent 修改内容      |   ❌   |   ✅   |  ✅   |
| 在网页编辑器中编辑内容         |   ❌   |   ✅   |  ✅   |
| 发布更改                       |   ❌   |   ✅   |  ✅   |
| 创建、重命名和删除页面         |   ❌   |   ✅   |  ✅   |
| 调整导航顺序                   |   ❌   |   ✅   |  ✅   |
| 上传媒体文件                   |   ❌   |   ✅   |  ✅   |
| 触发部署                       |   ❌   |   ✅   |  ✅   |
| 创建和删除 API 密钥            |   ❌   |   ✅   |  ✅   |
| 配置集成                       |   ❌   |   ✅   |  ✅   |
| 配置 assistant                 |   ❌   |   ✅   |  ✅   |

| 邀请编辑者和查看者             |   ❌   |   ✅   |  ✅   |
| 更新用户角色                   |   ❌   |   ❌   |  ✅   |

<Tip>
  仅将管理员访问权限授予需要执行管理员任务的成员。
</Tip>

<div id="add-members-to-your-organization">
  ## 向组织添加成员
</div>

默认情况下，创建 Mintlify 组织的人拥有管理员权限。你可以在 dashboard 的 [Members](https://app.mintlify.com/settings/organization/members) 页面添加其他成员。

你可以邀请任意数量的成员加入组织。

<div id="revoke-a-members-credentials">
  ## 撤销成员的凭证
</div>

管理员可以将成员从 dashboard 中登出，并撤销其密钥和令牌，而无需将其从组织中移除。在 offboarding 某人或响应账户被盗用时，可以使用撤销凭证功能。

1. 前往 dashboard 的 [Members](https://app.mintlify.com/settings/organization/members) 页面。
2. 在该成员所在行，点击 <Icon icon="shield-off" /> **Revoke credentials** 按钮。
3. 选择要撤销的凭证。
4. 点击 **Revoke**。

你可以撤销以下凭证的任意组合：

| 凭证 | 撤销后的效果 |
| ---------- | --------------------- |
| Sessions | 将该成员从 dashboard 登出。 |
| CLI tokens | 将该成员从 [CLI](/zh/cli/install) 登出。 |
| API keys | 删除该成员创建的 API 密钥。 |
| Discovery API keys | 删除该成员创建的 discovery API 密钥。 |
| M2M clients | 撤销该成员的机器对机器（M2M）客户端凭证。 |
| Connected apps | 断开该成员通过 OAuth 授权的应用。 |

撤销凭证无法撤消。除非你同时将该成员从组织中移除或阻止其登录方式，否则该成员仍可以重新登录并创建新的密钥和令牌。你无法撤销自己的凭证。
