---
title: "管理页面可见性"
description: "控制你的OpenAPI规范中哪些端点显示在文档导航里"
---

你可以控制哪些OpenAPI操作会生成并发布为文档页面，以及它们在导航中的可见性。这对仅供内部使用的端点、已弃用的操作、测试版功能，或需要通过直接URL访问但不应通过站点导航被发现的端点非常有用。

如果你的页面是从OpenAPI文档自动生成的，你可以使用 `x-hidden` 和 `x-excluded` 扩展来管理页面可见性。



<div id="x-hidden">
  ## `x-hidden`
</div>

`x-hidden` 扩展会为某个端点创建页面，但将其从导航中隐藏。该页面只能通过直接访问其 URL 打开。

`x-hidden` 的常见用例包括：

- 需要编写文档但不希望在导航中推广的端点。
- 仅用于从其他内容中跳转访问的页面。
- 面向特定用户的端点。



<div id="x-excluded">
  ## `x-excluded`
</div>

`x-excluded` 扩展会将某个端点从文档中彻底排除。

`x-excluded` 的常见用例包括：

- 仅供内部使用的端点。
- 不再希望在文档中呈现的已弃用端点。
- 尚未准备公开文档的 Beta 功能。



<div id="implementation">
  ## 实现
</div>

在你的 OpenAPI规范 中，将 `x-hidden` 或 `x-excluded` 扩展添加到对应的 HTTP 方法下。

下面是一些示例，展示了在端点和 webhook 路径的 OpenAPI 规范文档中如何使用每个属性。

```json {11, 19}
"paths": {
  "/plants": {
    "get": {
      "description": "返回商店中的所有植物",
      "parameters": { /*...*/ },
      "responses": { /*...*/ }
    }
  },
  "/hidden_plants": {
    "get": {
      "x-hidden": true,
      "description": "返回商店中部分保密的植物",
      "parameters": { /*...*/ },
      "responses": { /*...*/ }
    }
  },
  "/secret_plants": {
    "get": {
      "x-excluded": true,
      "description": "返回商店中高度机密的植物（不要发布此端点！）",
      "parameters": { /*...*/ },
      "responses": { /*...*/ }
    }
  }
},
```

```json {9, 15}
"webhooks": {
  "/plants_hook": {
    "post": {
      "description": "用于通知商店新增植物信息的 Webhook",
    }
  },
  "/hidden_plants_hook": {
    "post": {
      "x-hidden": true,
      "description": "用于通知商店新增植物的部分保密信息的 Webhook"
    }
  },
  "/secret_plants_hook": {
    "post": {
      "x-excluded": true,
      "description": "用于通知商店新增植物的高度机密信息的 Webhook（不要发布此端点！）"
    }
  }
}
```
