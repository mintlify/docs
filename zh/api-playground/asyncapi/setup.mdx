<div id="add-an-asyncapi-specification-file">
  ## 添加 AsyncAPI 规范文件
</div>

要开始为 WebSocket 创建页面，请确保你有一份有效的 AsyncAPI 架构文档（JSON 或 YAML 格式），并遵循 [AsyncAPI 规范](https://www.asyncapi.com/docs/reference/specification/v3.0.0)。你的架构必须符合 AsyncAPI 规范 3.0+。

<Tip>
  要验证 AsyncAPI 架构是否有效，你可以将其粘贴到
  [AsyncAPI Studio](https://studio.asyncapi.com/)
</Tip>



<div id="auto-populate-websockets-pages">
  ## 自动生成 websockets 页面
</div>

你可以在 `docs.json` 的导航中为任意选项卡或分组添加 `asyncapi` 字段。该字段可以是文档仓库中 AsyncAPI 规范文档的路径、托管的 AsyncAPI 规范文档的 URL，或指向多个 AsyncAPI 规范文档的链接数组。Mintlify 会为每个 AsyncAPI websocket 通道自动生成页面。

**选项卡示例：**

<CodeGroup>

```json Local File {5}
"navigation": {
  "tabs": [
    {
        "tab": "API Reference",
        "asyncapi": "/path/to/asyncapi.json"
    }
  ]
}

```

```json Remote URL {5}
"navigation": {
  "tabs": [
    {
        "tab": "API Reference",
        "asyncapi": "https://github.com/asyncapi/spec/blob/master/examples/simple-asyncapi.yml"
    }
  ]
}
```

</CodeGroup>

**分组示例：**

```json {8-11}
"navigation": {
  "tabs": [
    {
      "tab": "AsyncAPI",
      "groups": [
        {
          "group": "Websockets",
          "asyncapi": {
            "source": "/path/to/asyncapi.json",
            "directory": "api-reference"
          }
        }
      ]
    }
  ]
}
```

<Note>
  directory 字段是可选的。如果未指定，文件将被放在文档仓库的
  **api-reference** 文件夹中。
</Note>



<div id="channel-page">
  ## 通道页面
</div>

如果你需要更精细地控制通道的排序，或仅想引用某个单独的通道，可以创建一个在前言区块（frontmatter）中包含 `asyncapi` 字段的 MDX 文件。

```mdx
---
title: "Websocket Channel"
asyncapi: "/path/to/asyncapi.json channelName"
---
```
