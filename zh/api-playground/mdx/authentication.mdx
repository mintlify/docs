<div id="enabling-authentication">
  ## 启用认证
</div>

你可以在 `docs.json` 中添加一种认证方式，将其全局应用于每个页面；也可以为单个页面分别设置。

如果同时配置了全局方式和页面级方式，则以页面的认证方式为准。

<div id="bearer-token">
  ### Bearer 令牌
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "bearer"
      }
    }
}
```

```mdx Page Metadata
---
title: "页面标题"
authMethod: "bearer"
---
```

</CodeGroup>

<div id="basic-authentication">
  ### 基本认证
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "basic"
      }
    }
}
```

```mdx Page Metadata
---
title: "页面标题"
authMethod: "basic"
---
```

</CodeGroup>

<div id="api-key">
  ### API 密钥
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "key",
        "name": "x-api-key"
      }
    }
}
```

```mdx Page Metadata
---
title: "你的页面标题"
authMethod: "key"
---
```

</CodeGroup>

<div id="none">
  ### None
</div>

在 docs.json 中设置默认认证方式后，若需对某个特定端点停用认证，可使用 “none” 认证方法。

<CodeGroup>
```mdx Page Metadata
---
title: "Your page title"
authMethod: "none"
---
```
</CodeGroup>
