在控制台的 Analytics 页面中查看反馈。

<div id="feedback-types">
  ## 反馈类型
</div>

<Note>
  上下文反馈和代码片段反馈目前处于测试版（Beta）阶段。要为你的文档站点启用它们，请[联系销售团队](mailto:hahnbee@mintlify.com)。
</Note>

反馈标签页会根据你启用的[反馈附加组件](https://dashboard.mintlify.com/products/addons)显示相应信息。

* **点赞评分**：通过简单的点赞/点踩投票来衡量用户对页面的整体满意度。
* **上下文反馈**：针对页面内容的自由填写反馈。可选择[收集用户邮箱地址](#email-collection)。
* **代码片段反馈**：专门针对代码片段的反馈。
* **编辑建议**：允许用户为你的文档提出编辑建议。仅适用于公开的 GitHub 仓库。
* **提交 Issue**：允许用户直接从文档中在 GitHub 上创建 Issue。仅适用于公开的 GitHub 仓库。

<Note>
  如果你在 `docs.json` 文件中禁用了 `telemetry`，文档页面上的反馈功能将无法使用。
</Note>

<div id="email-collection">
  ## 收集邮箱
</div>

启用上下文反馈后，你可以选择允许用户在提交反馈时提供邮箱地址。这样可以方便你就反馈内容与用户进行后续沟通。

要为上下文反馈启用邮箱收集功能，请前往控制台的[附加组件](https://dashboard.mintlify.com/products/addons)页面，启用邮箱字段选项。

你还可以添加自定义免责声明，该声明将显示在邮箱字段旁边。使用此功能来说明你将如何使用邮箱地址，例如用于就反馈进行后续沟通。

<div id="manage-feedback">
  ## 管理反馈
</div>

对于带有上下文和代码片段的反馈，你可以为每条反馈设置处理状态，并添加内部备注，以便跟踪你解决用户反馈的进度。

<div id="change-feedback-status">
  ### 更改反馈状态
</div>

选择反馈项旁边的状态，将其标记为 **待处理 (Pending)**、**进行中 (In Progress)**、**已解决 (Resolved)** 或 **已忽略 (Dismissed)**。

设置反馈状态的最佳实践：

- **待处理 (Pending)**：反馈正在等待审核。
- **进行中 (In Progress)**：反馈已验证且正在处理。
- **已解决 (Resolved)**：反馈已被解决。
- **已忽略 (Dismissed)**：反馈因无法执行、不相关或不准确而被忽略。

<div id="filter-by-status">
  ### 按状态筛选
</div>

使用状态筛选器来控制显示哪些反馈。取消选中某个状态即可隐藏所有具有该状态的反馈。默认情况下，会显示所有反馈。

<div id="add-internal-notes">
  ### 添加内部备注
</div>

点击一条反馈可添加内部备注。只有有权访问你控制台的人才能看到这些备注。

使用备注来补充协作所需的信息、关联相关的支持或工程工单，或记录任何其他有用的信息。

<div id="use-feedback-data">
  ## 使用反馈数据
</div>

查看你的反馈数据，以便：

- **识别表现良好的内容**：正向反馈最多的页面可以帮助你了解文档中哪些部分效果最好。
- **优先改进**：负面反馈最多的页面表明哪些内容可能需要优先优化。
- **采取行动**：根据用户的直接反馈对文档进行更新。
