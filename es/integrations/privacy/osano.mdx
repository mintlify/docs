Añade el consentimiento de cookies de [Osano](https://www.osano.com/) a tu documentación con un [archivo JavaScript personalizado](/es/customize/custom-scripts#custom-javascript) en tu repositorio.

<div id="setup">
  ## Configuración
</div>

Crea un archivo `osano.js` en tu repositorio de documentación con el siguiente contenido:

<CodeGroup>
  ```js Format
  var script = document.createElement("script");
  script.src = "OSANO_SCRIPT_URL";
  document.head.appendChild(script);
  ```

  ```js Ejemplo
  var script = document.createElement("script");
  script.src = "https://cmp.osano.com/your-tenant-id/your-site-id/osano.js";
  document.head.appendChild(script);
  ```
</CodeGroup>

Reemplaza `OSANO_SCRIPT_URL` con la URL de tu script de Osano. La URL de tu script es el valor de `src` en el fragmento de código que genera Osano. La URL de tu script siempre comienza con `https://cmp.osano.com/` y termina con `/osano.js`.

<div id="troubleshooting">
  ## Solución de problemas
</div>

<Accordion title="Las páginas no cargan con el modo de cumplimiento Strict">
  Si tus páginas de documentación no se cargan correctamente al usar el modo de cumplimiento **Strict** de Osano, deberás agregar el domain de Mintlify a tu lista de permitidos para que se carguen las imágenes y otros recursos.

  <Steps>
    <Step title="Ve a Managed Rules">
      En tu dashboard de Osano, ve a **Scripts** → **Managed Rules**.
    </Step>

    <Step title="Agrega el domain de Mintlify">
      Agrega `.mintlify.app/` como una regla gestionada.

      <Frame>
        <img src="/images/integrations/osano-managed-rule.png" alt="Regla gestionada de Osano" />
      </Frame>

      <Info>
        Esto garantiza que todos los recursos servidos por Mintlify (incluidas imágenes, hojas de estilo y otros recursos de documentación) se traten como esenciales y se carguen incluso cuando Osano bloquee contenido de terceros sin categorizar.
      </Info>
    </Step>
  </Steps>
</Accordion>
