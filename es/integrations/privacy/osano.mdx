Añade lo siguiente a tu archivo `docs.json` para incorporar el gestor de consentimiento de cookies [Osano](https://www.osano.com/).

<CodeGroup>
  ```json Integration options in docs.json
  "integrations": {
      "osano": "SOURCE"
  }
  ```

  ```json Ejemplo
  "integrations": {
      "osano": "https://cmp.osano.com/2sUB2dqwqdkks/8dqwd-dwd86£-4a9b/osano.js"
  }
  ```
</CodeGroup>

El `SOURCE` se encuentra como el valor de `src` en el fragmento de código que genera Osano. Siempre empieza por `https://cmp.osano.com/`.

```html Code snippet from Osano
<script src="https://cmp.osano.com/placeholder/placeholder/osano.js"/>
```


<div id="troubleshooting">
  ## Solución de problemas
</div>

<Accordion
  title="Las páginas no cargan con el modo de cumplimiento Strict"
>
  Si tus páginas de documentación no se cargan correctamente al usar el modo de cumplimiento **Strict** de Osano, deberás permitir el domain de Mintlify para que se carguen las imágenes y otros recursos.

  <Steps>
    <Step title="Ve a Managed Rules">
      En tu dashboard de Osano, ve a **Scripts** → **Managed Rules**.
    </Step>

    <Step title="Agrega el domain de Mintlify">
      Agrega `.mintlify.app/` como una regla gestionada.

      <Frame>
        <img src="/images/integrations/osano-managed-rule.png" alt="Regla gestionada de Osano" />
      </Frame>

      <Info>
      Esto garantiza que todos los recursos servidos por Mintlify (incluidas imágenes, hojas de estilo y otros recursos de documentación) se traten como esenciales y se carguen incluso cuando Osano bloquee contenido de terceros sin categorizar.
      </Info>
    </Step>
  </Steps>
</Accordion>
