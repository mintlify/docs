Añade lo siguiente a tu archivo `docs.json` para enviar Analytics a PostHog.

Solo necesitas incluir `apiHost` si alojas PostHog por tu cuenta. De forma predeterminada enviamos eventos a `https://app.posthog.com`.

<CodeGroup>

```json Opciones de Analytics en docs.json
"integrations": {
    "posthog": {
        "apiKey": "YOUR_POSTHOG_PROJECT_API_KEY",
        "apiHost": "optional",
        "sessionRecording": true
    }
}
```

```json Ejemplo
"integrations": {
    "posthog": {
        "apiKey": "phc_TXdpocbYTeZVm5VJmMzHTMrCofBQu3e0kN7HGMNGTVW"
    }
}
```

</CodeGroup>

<div id="configuration-options">
  ### Opciones de configuración
</div>

<ParamField path="apiKey" type="string" required>
  La API key de tu proyecto de PostHog. Debe empezar con `phc_`.
</ParamField>

<ParamField path="apiHost" type="string">
  La URL de tu instancia de PostHog. Solo es Obligatorio si alojas PostHog por tu cuenta. De forma predeterminada es `https://app.posthog.com`.
</ParamField>

<ParamField path="sessionRecording" type="boolean" default="true">
  Activa o desactiva la grabación de sesiones. Establécelo en `false` para desactivar las grabaciones de sesiones y mantener Analytics habilitado.
</ParamField>

<br />

<Warning>
    Al habilitar Analytics de PostHog, se deshabilitará Analytics en el Dashboard de Mintlify.
</Warning>

<div id="session-recordings">
  ## Grabaciones de sesiones
</div>

Las grabaciones de sesiones se habilitan de forma predeterminada al configurar PostHog. Para desactivar las grabaciones de sesiones y mantener Analytics habilitado, establece `sessionRecording` en `false` en tu configuración.

```json Disable session recordings
"integrations": {
    "posthog": {
        "apiKey": "phc_your-key",
        "sessionRecording": false
    }
}
```

Debes agregar la URL de tu sitio de documentación a la sección &quot;Authorized domains for recordings&quot; de PostHog antes de poder recibir grabaciones de sesión. La opción para agregar tu URL se encuentra en la configuración del proyecto de PostHog.
