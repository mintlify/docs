<Update label="3 de octubre" tags={["New releases", "Improvements"]} rss={{ title: "Weekly Updates", description: "Products navigation, accessibility CLI, API playground enhancements, and extensive accessibility improvements" }}>
  ## Nuevas características

  * **Navegación de productos**: Organiza la documentación para múltiples productos con la navegación del selector de productos
  * **Comando CLI de accesibilidad**: Ejecuta `mint a11y` para probar problemas de accesibilidad en tu documentación

  ## Mejoras en el asistente y análisis

<Update label="16 de agosto" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Performance optimizations, LLM integrations, web editor polish, and API playground enhancements" }}>
  ## Optimizaciones de rendimiento y compilación

  * La transpilación de MDX ahora ocurre en el momento del despliegue en lugar de en cada carga de página en entornos sin servidor NextJS no almacenados en caché, mejorando el tiempo hasta el primer byte para páginas no almacenadas en caché.
  * El hash basado en contenido evita la retranspilación cuando MDX no ha cambiado, reduciendo los tiempos de flujo de trabajo de actualización en \~50% para clientes con gran cantidad de páginas (los despliegues de más de 5 minutos deberían reducirse aproximadamente a la mitad)
  * La visualización de despliegues de vista previa en el panel ahora es más rápida con índices de base de datos adicionales y paralelización de consultas en nuestro backend
  * Tamaño de página reducido al eliminar datos `navigation` duplicados en la carga `rsc` de cada página—las ganancias de rendimiento son más notables con recuentos altos de páginas o estructuras de navegación complejas
  * La precarga más agresiva permite cargas de página instantáneas con mayor frecuencia

  ## Mejoras en el playground de API y OpenAPI

  * Conversión de OpenAPI a MCP trasladada al backend, permitiendo que los servidores MCP alojados contengan herramientas (espera pronto documentación más clara y opciones de configuración)
  * Se agregó soporte para Ruby al playground de API
  * Agregamos una característica que te permite ahora [especificar páginas de API usando solo tu docs.json](/es/api-playground/openapi-setup#auto-populate-api-pages) sin crear ningún archivo mdx nuevo.
  * Soporte para [`webhook` páginas](/es/api-playground/openapi-setup#webhooks) en la navegación de documentos desde especificaciones OpenAPI
  * Contexto optimizado del modelo de IA eliminando especificaciones de enlaces de anclaje de los enlaces markdown al navegar a Anthropic, OpenAI u otros proveedores

  ## Mejoras en el editor web

  * La creación/renombrado de archivos ahora guarda los cambios al hacer clic fuera en lugar de requerir presionar la tecla Enter
  * Se solucionó la navegación de ramas donde cambiar la URL a una rama específica redirigía a la última rama activa en lugar del destino previsto
  * Codificación URL adecuada de títulos de rama que contienen `/` para evitar roturas de navegación
  * Se solucionó el atajo de enlace `Ctrl+K` en el Editor del panel de monorepo que estaba anteponiendo la ruta del repositorio de documentos y creando enlaces rotos

  ## Analíticas e integraciones LLM

  * Soporte personalizado para `llms.txt` y `llms-full.txt`—agrégalos a la raíz del repositorio de documentos para servirlos en los endpoints `/llms.txt` y `/llms-full.txt` para personalización de LLM
  * Se agregó [Integración de análisis de Hightouch](/es/integrations/analytics/hightouch#hightouch)
  * Seguimiento mejorado de análisis en el menú contextual (visualización del panel próximamente)
  * Añadidas pruebas e2e para`llms.txt`y`llms-full.txt`para asegurar el correcto funcionamiento

  ## Mejoras de componentes y estilos

  * Soporte para nombres de clase personalizados en`h{1-4}`etiquetas para aplicar estilos de encabezado personalizados
  * Corregido`h{1-4}`etiquetas renderizadas como`Heading`componentes con chips en modo de página personalizada
  * Añadidos selectores CSS a[breadcrumbs](/es/organize/navigation#breadcrumbs)para orientación CSS personalizada
  * Corregidas imágenes open-graph estiradas mediante el análisis de dimensiones para mantener proporciones a 56px de altura
  * Corregido`VSCode`a`VS Code`en el menú contextual cuando está habilitado
  * Corregidos encabezados dentro de componentes personalizados que aparecían en la tabla de contenidos junto con encabezados semánticos

  * Auto-eliminación de`https://`de URLs en configuración de autenticación OAuth/JWT para prevenir configuraciones erróneas
