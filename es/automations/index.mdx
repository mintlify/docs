<Info>
  Las automatizaciones requieren un [plan Pro o Enterprise](https://mintlify.com/pricing?ref=automations).
</Info>

Las automatizaciones ejecutan el agente de forma automática según una programación o en respuesta a cambios en un repositorio. Cada automatización define un prompt para el agente y un activador que indica cuándo ejecutarla. Las automatizaciones son compatibles con repositorios de GitHub y GitLab, con los mismos activadores, automatizaciones predefinidas y modos de actualización. A lo largo de esta guía, "pull request" también se refiere a los merge requests de GitLab, salvo que se indique lo contrario.

Cuando se ejecuta una automatización, el agente lee el contenido de tu proyecto y los repositorios conectados, y luego sigue el prompt para realizar las actualizaciones.

<Tip>
Usa automatizaciones que se ejecutan según una programación para tareas recurrentes, como publicar registros de cambios o auditar metadatos de SEO.

Usa automatizaciones que se activan ante cambios de contenido o de código para tareas de mantenimiento reactivas, como traducciones o actualizar el contenido cuando cambia el código fuente.
</Tip>

<div id="trigger-types">
## Tipos de activadores
</div>

Las automatizaciones admiten tres tipos de activadores.

- **Actualización de contenido**: se ejecuta cuando se hacen pushes con cambios de contenido al repositorio de tu proyecto de Mintlify, incluidas las fusiones de pull requests y los pushes directos.
- **Cambio de código**: se ejecuta cuando se fusiona una pull request en un repositorio de código fuente conectado.
- **Programación personalizada**: se ejecuta de forma recurrente. Las automatizaciones se ponen en cola dentro de los 10 minutos siguientes a la hora programada.

Cada automatización tiene exactamente un activador. No puedes combinar tipos de activadores en una sola automatización.

<div id="pull-request-grouping">
## Agrupación de pull requests
</div>

Las automatizaciones agrupan los cambios relacionados en una sola pull request en lugar de abrir una distinta para cada cambio. Cuando una ejecución posterior produce más cambios que corresponden a una pull request abierta, el agente los añade a esa pull request en lugar de abrir una duplicada. Cómo se aplica la agrupación depende de la automatización. Consulta [Automatizaciones predefinidas](/es/automations/reference) para conocer el comportamiento de cada una.

<Note>
  Las automatizaciones Draft changelog, Draft improvements from assistant conversations, Draft improvements from user feedback y las automatizaciones personalizadas no agrupan ni añaden. Cada ejecución abre su propia pull request con los cambios de esa ejecución.
</Note>

<div id="reviewing-automation-pull-requests">
## Revisar las pull requests de automatizaciones
</div>

Cuando una automatización abre o actualiza una pull request, Mintlify añade contexto directamente en ella para que puedas revisarla sin salir de GitHub o GitLab.

- **Título de la PR** incluye el nombre de la automatización que la generó, para que puedas identificar las PRs de automatizaciones de un vistazo al revisar la lista de PRs abiertas.
- **Descripción de la PR** resume qué cambió y por qué. Cada vez que una ejecución añade nuevos cambios a una PR existente, la descripción se actualiza para mostrar una línea de tiempo continua con lo que aportó cada ejecución.
- **Comentarios en la PR** para cualquier ejecución que añada cambios, explicando los cambios y qué pull requests de código fuente usó el agente como contexto.
- **Revisores** asignados según quién haya escrito los cambios de código fuente que activaron la actualización. Aplica a las automatizaciones que usan repositorios de código fuente como contexto en GitHub. GitLab no admite la asignación automática de revisores, así que asígnalos manualmente en los merge requests.

<div id="usage-limits">
## Límites de uso
</div>

Las ejecuciones de automatizaciones cuentan para el uso de tus créditos. Los créditos que consume una sola ejecución dependen del tamaño de la tarea: cuánto contenido lee el agente, cuántos archivos modifica y cuánto tarda la ejecución. Los repositorios de contexto más grandes y los prompts más amplios consumen más créditos que los que tienen un alcance acotado.

Consulta tu uso de créditos, configura los excedentes y gestiona tu paquete de créditos en la página [Uso](https://app.mintlify.com/settings/organization/usage) de tu dashboard.

Cada automatización puede ejecutarse hasta 500 veces al día. Las ejecuciones que fallan no cuentan para este límite.
