Mintlify incluye automatizaciones predefinidas para las tareas de mantenimiento de documentación más comunes. Cada automatización tiene un prompt base y una configuración predeterminada. Puedes personalizar el activador, el modo de actualización y las instrucciones desde la página [Automatizaciones](https://app.mintlify.com/products/automations) en tu dashboard.

<div id="at-a-glance">
## De un vistazo
</div>

| Automatización | Activador predeterminado | Modo de actualización predeterminado | Repos de contexto | Configuración requerida |
| -------------- | ------------------------ | ------------------------------------ | ----------------- | ----------------------- |
| Update from code changes | Programación personalizada (dos veces por semana) | Pull request | Compatible | Al menos un repositorio de código fuente |
| Draft changelog | Programación personalizada | Pull request | Compatible | Ninguna |
| Draft improvements from assistant conversations | Programación personalizada | Pull request | No compatible | Ninguna |
| Draft improvements from user feedback | Programación personalizada | Pull request | No compatible | Recopilación de comentarios activada |
| Update from support feedback | Programación personalizada | Pull request | Compatible | Integración de Intercom o Plain conectada |
| Translate content | Actualización de contenido | Fusión directa | No compatible | Al menos un idioma de destino |
| Fix broken links | Actualización de contenido | Pull request | No compatible | Ninguna |
| Audit SEO metadata | Actualización de contenido | Pull request | No compatible | Ninguna |
| Fix grammar & typos | Actualización de contenido | Pull request | No compatible | Ninguna |
| Apply style guide | Actualización de contenido | Pull request | No compatible | Ninguna |

Todos los valores predeterminados pueden cambiarse desde el panel de configuración de la automatización. Consulta [Configuraciones](/es/automations/manage#configurations) para ver las opciones disponibles.

<div id="self-updating-content-automations">
## Automatizaciones de contenido autoactualizable
</div>

Las automatizaciones de contenido autoactualizable realizan actualizaciones basadas en el contexto que definas para que tu contenido se mantenga preciso y al día.

<div id="update-from-code-changes">
### Actualizar a partir de cambios de código
</div>

Supervisa tus repositorios de código fuente y abre pull requests para actualizar el contenido ante cambios que afecten a los usuarios. Puedes ejecutar esta automatización según una programación, que generalmente consume menos créditos, o en cada pull request del código fuente. De forma predeterminada, esta automatización se ejecuta dos veces por semana (martes y jueves).

Cuando ejecutas la automatización según una programación, el agente lee todas las pull requests del código fuente fusionadas desde la última vez que se ejecutó. Agrupa los cambios relacionados que requieren actualizaciones de contenido en una pull request en el repositorio de tu proyecto de Mintlify. Si hay pull requests sin fusionar de ejecuciones anteriores, el agente añade los cambios relevantes a las pull requests existentes. El agente cita de dónde provienen los cambios en tus repositorios de código fuente.

Cuando ejecutas la automatización en cada pull request del código fuente, el agente abre una pull request de actualización de contenido cada vez que fusionas un cambio de código que requiere actualizaciones de contenido. Usa la variante basada en push cuando quieras que las actualizaciones de contenido aparezcan tan pronto como se publique el código, en lugar de agruparlas según una programación.

Puedes habilitar las [integraciones](/es/automations/integrations) para esta automatización para darle al agente contexto adicional desde herramientas como Slack o Linear al determinar qué actualizar.

<div id="draft-changelog">
### Redactar el registro de cambios
</div>

Crea una entrada en el registro de cambios con una programación recurrente. Añade repositorios de código fuente como contexto para que el agente tenga más detalles sobre cómo cambió tu producto. Sin repositorios de contexto, el agente trabaja a partir de tus actualizaciones de contenido para escribir una entrada del registro de cambios.

<div id="draft-improvements-from-assistant-conversations">
### Redactar mejoras a partir de conversaciones del asistente
</div>

Revisa las tendencias de preguntas de tu [asistente](/es/assistant/index) para identificar lagunas en tu contenido y luego abre una pull request para abordarlas. Cuantas más conversaciones del asistente tengan tus usuarios, más información tendrá el agente para trabajar. Ejecuta esta automatización de forma recurrente para mantener el contenido alineado con lo que los usuarios quieren saber sobre tu producto.

<div id="draft-improvements-from-user-feedback">
### Redactar mejoras a partir de los comentarios de los usuarios
</div>

Revisa los [comentarios](/es/optimize/feedback) en las páginas —valoraciones con pulgar y comentarios escritos— y actualiza el contenido donde los usuarios han reportado problemas. El agente se centra en las páginas con señales negativas recurrentes en lugar de en comentarios aislados, por lo que los resultados mejoran a medida que se acumulan los comentarios.

Debes habilitar la recopilación de comentarios en la página [Complementos](https://app.mintlify.com/products/addons) antes de poder usar esta automatización.

<div id="update-from-support-feedback">
### Actualizar a partir de comentarios de soporte
</div>

Revisa las conversaciones de soporte resueltas de [Intercom](/es/automations/integrations) o [Plain](/es/automations/integrations) para identificar lagunas recurrentes en tu contenido y luego abre una pull request para abordarlas. El agente se centra en los temas que se repiten en varias conversaciones en lugar de en preguntas aisladas, por lo que los resultados mejoran a medida que crece tu volumen de soporte.

Selecciona una única plataforma de soporte al configurar la automatización y conéctala desde el panel de configuración si aún no está conectada para tu organización. Añade repositorios de código fuente como contexto para ayudar al agente a vincular los temas de soporte con las áreas del producto que describen.

Cada ejecución de la automatización enumera las conversaciones de soporte resueltas que revisó y enlaza de vuelta a la plataforma de origen, para que puedas rastrear cada sugerencia de contenido hasta los comentarios de los clientes subyacentes.

Esta automatización está disponible en los planes con integraciones personalizadas. Contacta con tu equipo de cuenta si no la ves en el dashboard.

<div id="maintenance-automations">
## Automatizaciones de mantenimiento
</div>

Las automatizaciones de mantenimiento automatizan tareas rutinarias para mejorar la calidad y la consistencia de tu contenido. Puedes ejecutarlas con un activador de actualización de contenido o con una programación recurrente.

Ejecutarlas con un activador de actualización de contenido mantiene tu contenido al día con un retraso mínimo. Ejecutarlas según una programación generalmente es más eficiente en créditos porque el agente agrupa el trabajo de varios cambios en una sola ejecución. La contrapartida es un retraso entre el cambio del contenido y el momento en que la automatización actúa sobre él.

Cada automatización de mantenimiento mantiene una sola pull request abierta y le añade nuevas correcciones a medida que cambia el contenido, de modo que el mantenimiento rutinario se acumula en una sola pull request por revisar en lugar de muchas.

Cualquier automatización de mantenimiento habilitada se ejecuta en las pull requests creadas por las automatizaciones de contenido autoactualizable. Si la automatización de mantenimiento identifica cambios necesarios, el agente combina el trabajo en una sola pull request. Por ejemplo, si habilitas la automatización **Translate content**, el agente añade traducciones a cualquier pull request abierta por la automatización **Update from code changes**.

<div id="translate-content">
### Traducir contenido
</div>

Traduce el contenido actualizado a los idiomas de destino que hayas seleccionado cada vez que se envíen cambios al repositorio de tu proyecto. Debes especificar al menos un idioma de destino. Mintlify lee los [idiomas](/es/organize/navigation#languages) definidos en tu `docs.json` y preselecciona los idiomas de destino que ya estén configurados.

Para desactivar las traducciones de un idioma, elimina todos los archivos del directorio de ese idioma. No puedes desactivar las traducciones de un idioma que aún tenga contenido publicado.

<div id="fix-broken-links">
### Corregir enlaces rotos
</div>

Encuentra y corrige los enlaces internos y externos rotos cada vez que se envían cambios de contenido al repositorio de tu proyecto. Útil para ejecutar después de grandes reorganizaciones de contenido o cuando los servicios externos a los que haces referencia cambian sus URLs.

<div id="audit-seo-metadata">
### Auditar metadatos de SEO
</div>

Audita y actualiza títulos, meta descripciones, encabezados y etiquetas canónicas cada vez que cambia el contenido. Añade instrucciones para alinear las sugerencias del agente con tu estrategia de SEO o tus palabras clave objetivo.

<div id="fix-grammar--typos">
### Corregir gramática y erratas
</div>

Encuentra y corrige erratas, errores ortográficos y errores gramaticales cada vez que cambia el contenido. Añade instrucciones para especificar excepciones para términos técnicos, nombres de productos o elecciones estilísticas intencionadas que el agente no debe modificar.

<div id="apply-style-guide">
### Aplicar la guía de estilo
</div>

Aplica reglas de voz, tono y redacción al contenido cada vez que cambia. Añade instrucciones para definir las reglas específicas que deseas que se apliquen o enlaza a tu guía de estilo. Cuanto más precisas sean tus instrucciones, más consistentes serán los resultados.
