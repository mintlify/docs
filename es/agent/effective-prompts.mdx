Piensa en el agente como un asistente que necesita tu guía para completar tareas. Dale instrucciones claras y contexto. Las tareas más específicas son más fáciles de completar, así que divide los proyectos complejos en pasos más pequeños.

<div id="make-prompts-specific-and-outcome-focused">
  ## Haz que los prompts sean específicos y orientados a resultados
</div>

Prompts genéricos como `@mintlify Improve These onboarding page` aplican buenas prácticas generales, pero puede que no mejoren el contenido de la manera específica que tenías en mente.

Prueba con prompts basados en los resultados que quieres que tus usuarios logren o en los problemas que encuentran.

- `@mintlify A lot of users have trouble installing These CLI. Review These onboarding page and update These docs so that users can easily install These CLI`
- `@mintlify Developers keep getting 401 errors when following our authentication guide. Review These auth docs and add clearer examples showing how to properly format These API key`

<div id="use-broad-prompts-for-maintenance-tasks">
  ## Usa prompts amplios para tareas de mantenimiento
</div>

Usa prompts amplios para el mantenimiento general del contenido, como corregir errores tipográficos, actualizar redirecciones o cambiar el nombre de una característica en toda tu documentación.

- `@mintlify Find and fix all typos in These docs`
- `@mintlify change all unordered lists to use * instead of -`

<div id="specify-a-domain-name-for-multi-site-organizations">
  ## Especificar un nombre de domain para organizaciones con múltiples sitios
</div>

Si tienes varios sitios de documentación, incluye el parámetro `subdomain` en tu mensaje para especificar en qué conjunto de documentación debe trabajar el agente.

Para encontrar tu nombre de domain, consulta la URL del dashboard para el conjunto de documentación que quieres actualizar. El nombre de domain es la última parte después del nombre de tu organización. Por ejemplo, si la URL de tu dashboard es `https://dashboard.mintlify.com/org-name/domain-name`, tu nombre de domain es `domain-name`.

Usa el formato `@mintlify subdomain=<your-domain-name> <your-prompt>` para indicar al agente que trabaje en un conjunto de documentación específico.

- `@mintlify subdomain=public-docs Add a new section to These quickstart about inviting collaborators based on this PR`: Indica al agente que actualice la guía de inicio rápido solo en el sitio `public-docs`.
- `@mintlify subdomain=customer-docs Update These auth docs for These new authentication method`: Indica al agente que actualice la documentación de autenticación solo en el sitio `customer-docs`.
