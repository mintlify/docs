El agente ayuda con muchas tareas de documentación diferentes. Estos ejemplos muestran algunas de las formas en que puedes integrar el agente en tu proceso de documentación. Prueba un enfoque que se ajuste a cómo trabaja actualmente tu equipo y adáptalo a tus necesidades específicas.

## Iterar un prompt en un hilo de Slack

Envía un prompt al agente y luego sigue mencionándolo con `@mintlify` en el mismo hilo para refinar e iterar sobre la solicitud de extracción que crea.

Por ejemplo: `@mintlify Our quickstart page needs a new section on inviting collaborators`. Luego `@mintlify The new section should be called "Inviting collaborators"`. Y así con cualquier otra iteración.

## Empieza con el agente y termina manualmente

Pídele al agente que inicie un proyecto, luego cambia a la rama que crea y termina la tarea en tu entorno local o en el editor web. El agente puede ayudarte a empezar y luego tú puedes encargarte de completar la tarea.

Por ejemplo: `@mintlify Update the quickstart page to include information about inviting collaborators` y luego cambia a la rama para hacer cualquier cambio adicional usando tu método preferido.

## Actualizar la documentación al fusionar cambios de funcionalidades

Cuando fusiones una solicitud de extracción de una funcionalidad, comparte el enlace de la PR con el agente para que actualice la documentación relevante.

Por ejemplo: `@mintlify This PR adds a new authentication method. Update the docs to include the new auth flow: [PR link]`.

## Generar notas de la versión a partir de una solicitud de extracción

Proporciona al agente una solicitud de extracción específica para generar notas de la versión o actualizaciones del changelog basadas en el historial de commits.

Por ejemplo: `@mintlify Generate release notes for this PR: [PR link]`.

## Atender comentarios de revisión de solicitudes de extracción

Comparte un enlace de la solicitud de extracción con el agente para atender los comentarios de los revisores y el feedback de la revisión de código en las solicitudes de extracción de documentación.

Por ejemplo: `@mintlify Address the review comments on this PR: [PR link]`.

## Generar ejemplos de código

Pídele al agente que genere ejemplos de código para las funcionalidades en toda tu documentación o en páginas específicas.

Por ejemplo: `@mintlify Generate a code example to make the authentication method easier to understand`.

## Agregar imágenes o archivos a tu documentación

Adjunta archivos o imágenes directamente a tu mensaje de Slack cuando le pidas algo al agente. El agente procesa el archivo adjunto y lo incluye en la documentación.

Por ejemplo: `@mintlify Add this diagram to the architecture overview page` con la imagen adjunta al mensaje.

## Revisar el contenido existente

Indica al agente que revise el contenido existente para comprobar su exactitud técnica, estilo, gramática u otros posibles problemas.

Por ejemplo: `@mintlify Review the API rate limiting section. We changed limits last month`.

## Responder a los comentarios de los usuarios

Proporciona indicaciones al agente con los comentarios de tus usuarios para hacer actualizaciones específicas en tu documentación.

Por ejemplo: `@mintlify Users are getting confused by step 3 in the setup guide. What might be making it unclear?`.

## Automatizar con la API

Integra el agente en tus herramientas de automatización existentes para actualizar la documentación automáticamente cuando se produzcan cambios en el código, desencadenar revisiones de contenido o sincronizar actualizaciones de documentación en múltiples repositorios.

Usa los endpoints del agente para [crear tareas](/es/api-reference/agent/create-agent-job), [obtener una tarea específica](/es/api-reference/agent/get-agent-job) y [obtener todas las tareas](/es/api-reference/agent/get-all-jobs).

Al crear tareas a través de la API, puedes controlar si las solicitudes de extracción se abren en modo borrador usando el parámetro `asDraft` (su valor predeterminado es `true`). Establece `asDraft: false` para crear solicitudes de extracción que no sean borrador y que estén listas para revisión inmediata y fusión en flujos de trabajo automatizados.

Aprende a configurar la automatización de la API en el tutorial [Actualizar la documentación automáticamente cuando se fusiona código](/es/guides/automate-agent).
