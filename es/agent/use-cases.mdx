El agente ayuda con muchas tareas diferentes de documentación. Estos ejemplos muestran algunas de las formas en que puedes integrar el agente en tu proceso de documentación. Prueba un enfoque que encaje con la forma en que tu equipo trabaja actualmente y ajústalo a tus necesidades específicas.

<div id="iterate-on-a-prompt-in-a-slack-thread">
  ## Iterar una instrucción en un hilo de Slack
</div>

Envía una instrucción al agente y luego continúa mencionándolo con `@mintlify` en el mismo hilo para refinar e iterar sobre la solicitud de extracción que crea.

Por ejemplo: `@mintlify Nuestra guía de inicio rápido necesita una nueva sección sobre cómo invitar colaboradores`. Luego: `@mintlify La nueva sección debería llamarse "Inviting collaborators"`. Y después cualquier otra iteración que quieras hacer.

<div id="start-with-the-agent-finish-manually">
  ## Empieza con el agente y termina manualmente
</div>

Indícale al agente que inicie un proyecto, luego cambia a la branch que cree y termina la tarea en tu entorno local o en el editor web. El agente puede ayudarte a empezar y luego tú puedes tomar el control para completar la tarea.

Por ejemplo: `@mintlify Update the quickstart page to include information about inviting collaborators` y luego cambia a la branch para hacer cualquier cambio adicional usando el método que prefieras.

<div id="update-docs-when-merging-feature-changes">
  ## Actualiza la documentación al fusionar cambios de funcionalidades
</div>

Cuando fusiones una solicitud de extracción de una funcionalidad, comparte el enlace del PR con el agente para actualizar la documentación relevante.

Por ejemplo: `@mintlify Este PR agrega un nuevo método de autenticación. Actualiza la documentación para incluir el nuevo flujo de autenticación: [enlace del PR]`.

<div id="generate-release-notes-from-a-pull-request">
  ## Generar notas de lanzamiento a partir de una solicitud de extracción
</div>

Indícale al agente una solicitud de extracción específica para que genere notas de lanzamiento o actualizaciones del changelog basadas en el historial de confirmaciones.

Por ejemplo: `@mintlify Generate release notes for this PR: [PR link]`.

<div id="address-pull-request-review-feedback">
  ## Atender comentarios de revisión de solicitudes de extracción
</div>

Comparte con el agente el enlace de la solicitud de extracción para atender los comentarios de los revisores y los comentarios de revisión de código en las solicitudes de extracción de documentación.

Por ejemplo: `@mintlify Address the review comments on this PR: [PR link]`.

<div id="generate-code-examples">
  ## Generar ejemplos de código
</div>

Pídele al agente que genere ejemplos de código para funcionalidades en toda tu documentación o en páginas específicas.

Por ejemplo: `@mintlify Generate a code example to make the authentication method easier to understand`.

<div id="add-images-or-files-to-your-docs">
  ## Agrega imágenes o archivos a tu documentación
</div>

Adjunta archivos o imágenes directamente a tu mensaje de Slack cuando des instrucciones al agente. El agente procesa el archivo adjunto y lo incluye en la documentación.

Por ejemplo: `@mintlify Añade este diagrama a la página de visión general de la arquitectura` con la imagen adjunta al mensaje.

<div id="review-existing-content">
  ## Revisar el contenido existente
</div>

Pide al agente que revise el contenido existente para comprobar la precisión técnica, el estilo, la gramática u otros posibles problemas.

Por ejemplo: `@mintlify Revisa la sección de limitación de tasa de la API. Cambiamos los límites el mes pasado`.

<div id="respond-to-user-feedback">
  ## Responde a los comentarios de los usuarios
</div>

Proporciona al agente los comentarios de tus usuarios para realizar actualizaciones puntuales en tu documentación.

Por ejemplo: `@mintlify Users are getting confused by step 3 in the setup guide. What might be making it unclear?`.

<div id="automate-with-the-api">
  ## Automatiza con la API
</div>

Integra el agente en tus herramientas de automatización existentes para actualizar la documentación automáticamente cuando se produzcan cambios en el código, iniciar revisiones de contenido o sincronizar actualizaciones de documentación entre múltiples repositorios.

Usa los endpoints del agente para [crear tareas](/es/api-reference/agent/create-agent-job), [obtener una tarea específica](/es/api-reference/agent/get-agent-job) y [obtener todas las tareas](/es/api-reference/agent/get-all-jobs).

Al crear tareas mediante la API, puedes controlar si las solicitudes de extracción se abren en modo borrador usando el parámetro `asDraft` (su valor predeterminado es `true`). Establece `asDraft: false` para crear solicitudes de extracción que no sean borradores y estén listas para revisión inmediata y fusión en flujos de trabajo automatizados.

Aprende cómo configurar la automatización con la API en el tutorial [Actualizar automáticamente la documentación cuando se fusiona el código](/es/guides/automate-agent).
