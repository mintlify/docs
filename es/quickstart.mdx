Esta guía de inicio rápido te muestra cómo configurar e implementar tu sitio de documentación en cuestión de minutos.

Al completar esta guía, tendrás un sitio de documentación en producción listo para personalizar y ampliar.

<Info>
  **Requisitos previos**: Antes de comenzar, [crea una cuenta](https://mintlify.com/start) y completa el onboarding.
</Info>

## Primeros pasos

Después de completar el onboarding, tu sitio de documentación se despliega automáticamente en una URL única con este formato:

```
https://<your-project-name>.mintlify.app
```

Encuentra tu URL en la página Overview de tu [dashboard](https://dashboard.mintlify.com/).

<Frame>
  <img src="/images/quickstart/mintlify-domain-light.png" alt="Mintlify Domain" className="block dark:hidden" />

  <img src="/images/quickstart/mintlify-domain-dark.png" alt="Mintlify Domain" className="hidden dark:block" />
</Frame>

La URL de tu sitio está disponible de inmediato. Úsala para hacer pruebas y compartirla con tu equipo mientras configuras tu sitio de documentación.

### Instalar la GitHub App

Mintlify ofrece una GitHub App que automatiza el despliegue cuando haces push de cambios a tu repositorio.

Instala la GitHub App siguiendo las instrucciones de la checklist de onboarding o desde tu dashboard.

1. Ve a **Settings** en tu dashboard de Mintlify.
2. Selecciona **GitHub App** en la barra lateral.
3. Selecciona **Install GitHub App**. Se abrirá una nueva pestaña con la página de instalación de la GitHub App.
4. Selecciona la organización o la cuenta de usuario donde quieres instalar la app.
5. Selecciona los repositorios que quieres conectar.

<Frame>
  <img src="/images/quickstart/github-app-installation-light.png" alt="GitHub App Installation" />

  <img src="/images/quickstart/github-app-installation-dark.png" alt="GitHub App Installation" />
</Frame>

<Info>
  Actualiza los permisos de la GitHub App si mueves tu documentación a otro repositorio.
</Info>

### Autorizar tu cuenta de GitHub

1. Ve a **Settings** en tu dashboard de Mintlify.
2. Selecciona **My Profile** en la barra lateral.
3. Selecciona **Authorize GitHub account**. Se abrirá una nueva pestaña con la página de autorización de GitHub.

<Info>
  Es posible que un administrador de tu organización en GitHub deba autorizar tu cuenta según la configuración de tu organización.
</Info>

## Flujo de trabajo basado en código

El flujo de trabajo basado en código se integra con tu entorno de desarrollo y repositorios de Git existentes. Este flujo es ideal para equipos técnicos que quieren gestionar la documentación junto con el código.

### Instalar la CLI

Para trabajar localmente con tu documentación, instala la interfaz de línea de comandos (CLI), llamada [mint](https://www.npmjs.com/package/mint), ejecutando este comando en tu terminal:

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

<Info>
  Necesitas tener Node.js instalado en tu equipo. Si encuentras problemas durante la instalación, consulta la guía de solución de problemas.
</Info>

### Editar la documentación

Después de configurar tu entorno, puedes empezar a editar tus archivos de documentación. Por ejemplo, actualiza el título de la página de introducción:

1. Abre el repositorio que creaste durante el onboarding.
2. Abre `index.mdx` y ve al inicio del archivo:

```mdx index.mdx
---
title: "Introduction"
description: "This is the introduction to the documentation"
---
```

3. Actualiza el campo `title` a `"Hello World"`.

```mdx index.mdx {2}
---
title: "Hello World"
description: "This is the introduction to the documentation"
---
```

### Previsualizar los cambios

Para previsualizar los cambios localmente, ejecuta el siguiente comando:

```bash
mint dev
```

La vista previa estará disponible en `localhost:3000`.

<Frame>
  <img src="/images/quickstart/mintlify-dev-light.png" alt="Mintlify Dev" className="block dark:hidden" />

  <img src="/images/quickstart/mintlify-dev-dark.png" alt="Mintlify Dev" className="hidden dark:block" />

### Subir los cambios

Cuando estés listo para publicar tus cambios, súbelos a tu repositorio.

Mintlify detecta automáticamente los cambios, compila tu documentación y despliega las actualizaciones en tu sitio. Supervisa el estado del despliegue en el historial de commits de tu repositorio de GitHub o en el [panel](https://dashboard.mintlify.com).

Una vez que finalice el despliegue, tu actualización más reciente estará disponible en `<your-project-name>.mintlify.app`.

</Card>

## Flujo de trabajo del Web editor

El flujo de trabajo del Web editor proporciona una interfaz WYSIWYG (lo que ves es lo que obtienes) para crear y editar documentación. Este flujo es ideal para quienes desean trabajar en su navegador sin herramientas de desarrollo locales adicionales.

### Accede al Web editor

1. Inicia sesión en tu [panel](https://dashboard.mintlify.com).
2. Selecciona **Editor** en la barra lateral izquierda.

<Info>
  Si no has instalado la app de GitHub, se te pedirá que la instales al abrir el Web editor.
</Info>

<Frame>
  <img alt="El Web editor de Mintlify en modo editor visual" src="/images/quickstart/web-editor-light.png" className="block dark:hidden" />

  <img alt="El Web editor de Mintlify en modo editor visual" src="/images/quickstart/web-editor-dark.png" className="hidden dark:block" />
</Frame>

### Edita la documentación

En el Web editor, puedes navegar por tus archivos de documentación desde la barra lateral. Actualicemos la página de introducción:

Busca y selecciona `index.mdx` en el explorador de archivos.

Luego, en el editor, actualiza el campo de título a "Hello World".

<Frame>
  <img alt="Edición en el Web editor" src="/images/quickstart/web-editor-editing-light.png" className="block dark:hidden" />

  <img alt="Edición en el Web editor" src="/images/quickstart/web-editor-editing-dark.png" className="hidden dark:block" />
</Frame>

<Tip>
  El editor ofrece un conjunto completo de herramientas de formato y componentes. Escribe <kbd>/</kbd> en el editor para abrir el menú de comandos y acceder a estas herramientas.
</Tip>

### Publica tus cambios

Cuando estés conforme con tus ediciones, selecciona el botón **Publish** en la esquina superior derecha. Tus cambios se implementarán de inmediato en tu sitio de documentación.

<Tip>
  Usa ramas para previsualizar y revisar cambios mediante pull requests antes de desplegar en tu sitio en producción.
</Tip>

Para más detalles sobre el uso del Web editor, incluido el uso de ramas y pull requests para colaborar y previsualizar cambios, consulta nuestra [documentación del Web editor](/es/editor).

## Agregar un dominio personalizado

Aunque tu subdominio `<your-project-name>.mintlify.app` funciona bien para pruebas y desarrollo, la mayoría de los equipos prefieren usar un dominio personalizado para la documentación en producción.

Para agregar un dominio personalizado, ve a la página [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain) en tu panel.

<Frame>
  <img src="/images/quickstart/custom-domain-light.png" alt="Dominio personalizado" className="block dark:hidden" />

  <img src="/images/quickstart/custom-domain-dark.png" alt="Dominio personalizado" className="hidden dark:block" />
</Frame>

Ingresa tu dominio (por ejemplo, `docs.yourcompany.com`) y sigue las instrucciones para configurar la DNS con tu proveedor de dominio.

<Table>
  | Tipo de registro | Nombre | Valor | TTL |
  |-------------|------|-------|-----|
  | CNAME | docs (o subdominio) | cname.mintlify.app | 3600 |
</Table>

<Info>
  Los cambios de DNS pueden tardar hasta 48 horas en propagarse, aunque a menudo se completan mucho antes.
</Info>

## Solución de problemas

Si encuentras problemas durante el proceso de configuración, consulta estas soluciones comunes:

<AccordionGroup>
  <Accordion title="La vista previa local no funciona">
    Asegúrate de tener instalado Node.js v19+ y de ejecutar el comando `mint dev` desde el directorio que contiene tu archivo `docs.json`.
  </Accordion>

  <Accordion title="Los cambios no se reflejan en el sitio en producción">
    El despliegue puede tardar varios minutos. Revisa tus GitHub Actions (para flujos basados en código) o los registros de despliegue en el panel de Mintlify para asegurarte de que no haya errores de compilación.
  </Accordion>

  <Accordion title="El dominio personalizado no se conecta">
    Verifica que tus registros DNS estén configurados correctamente y espera el tiempo suficiente para la propagación de DNS (hasta 48 horas). Puedes usar herramientas como [DNSChecker](https://dnschecker.org) para comprobar tu registro CNAME.
  </Accordion>
</AccordionGroup>
