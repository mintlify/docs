</Info>

Mintlify ofrece tres niveles de acceso al dashboard: Viewer, Editor y Admin.

## Niveles de acceso

- **Viewer**: Acceso de solo lectura a la documentación. Los Viewers pueden ver todo el contenido pero no pueden realizar cambios ni acceder a funciones administrativas.
- **Editor**: Puede crear, editar y publicar contenido de documentación.
- **Admin**: Acceso completo a todas las funciones del dashboard, incluyendo gestión de usuarios, facturación y configuración de la organización.

## Permisos

La siguiente tabla describe los permisos para cada rol:

|                         | Viewer | Editor | Admin |
| ----------------------- | :----: | :----: | :---: |
| Ver documentación       |   ✅   |   ✅   |  ✅   |
| Editar contenido        |   ❌   |   ✅   |  ✅   |
| Actualizar roles de usuario |   ❌   |   ❌   |  ✅   |
| Eliminar usuarios       |   ❌   |   ❌   |  ✅   |
| Invitar usuarios        |   ❌   |   ✅   |  ✅   |
| Gestionar y actualizar la facturación |   ❌   |   ❌   |  ✅   |
| Actualizar el domain personalizado |   ❌   |   ❌   |  ✅   |
| Actualizar Git source   |   ❌   |   ❌   |  ✅   |
| Eliminar la organización |   ❌   |   ❌   |  ✅   |

## Reglas de escalada de privilegios

Cuando RBAC está habilitado, los usuarios no pueden invitar a otros con roles que ellos mismos no tienen. Por ejemplo:

- Un Viewer no puede invitar a ningún usuario
- Un Editor puede invitar a Viewers y Editors, pero no a Admins
- Un Admin puede invitar usuarios con cualquier rol

Esto previene la escalada de privilegios y asegura un control de acceso adecuado dentro de tu organización.

Puedes invitar a tantos usuarios como necesites, pero recomendamos limitar el acceso de admin a quienes realmente lo necesiten para la gestión organizacional.
