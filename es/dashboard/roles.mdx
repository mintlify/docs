<Tip>
  Limita el acceso de administrador solo a las personas que necesiten realizar tareas de administración.
</Tip>

<div id="add-members-to-your-organization">
  ## Agregar miembros a tu organización
</div>

Por defecto, la persona que crea tu organización en Mintlify tiene acceso de administrador. Agrega miembros adicionales en la página [Members](https://app.mintlify.com/settings/organization/members) de tu dashboard.

Puedes invitar a cualquier número de miembros a tu organización.

<div id="revoke-a-members-credentials">
  ## Revocar las credenciales de un miembro
</div>

Los administradores pueden cerrar la sesión de un miembro en el dashboard y revocar sus claves y tokens sin eliminarlo de la organización. Utiliza la revocación de credenciales cuando realices el offboarding de alguien o respondas a una cuenta comprometida.

1. Ve a la página [Members](https://app.mintlify.com/settings/organization/members) de tu dashboard.
2. En la fila del miembro, haz clic en el botón <Icon icon="shield-off" /> **Revoke credentials**.
3. Selecciona qué credenciales revocar.
4. Haz clic en **Revoke**.

Puedes revocar cualquier combinación de las siguientes credenciales:

| Credencial | Qué hace la revocación |
| ---------- | --------------------- |
| Sesiones | Cierra la sesión del miembro en el dashboard. |
| Tokens de CLI | Cierra la sesión del miembro en la [CLI](/es/cli/install). |
| API keys | Elimina las API keys que el miembro creó. |
| Discovery API keys | Elimina las discovery API keys que el miembro creó. |
| Clientes M2M | Revoca las credenciales de cliente de máquina a máquina del miembro. |
| Aplicaciones conectadas | Desconecta las aplicaciones que el miembro autorizó a través de OAuth. |

La revocación de credenciales no se puede deshacer. El miembro puede volver a iniciar sesión y crear nuevas claves y tokens a menos que también lo elimines de la organización o bloquees su método de inicio de sesión. No puedes revocar tus propias credenciales.
