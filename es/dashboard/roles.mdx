Mintlify ofrece tres niveles de acceso al dashboard: Visualizador, Editor y Admin.

## Permisos

A continuación se describen las acciones limitadas por rol:

|                         | Visualizador | Editor | Admin |
| ----------------------- | :----------: | :----: | :---: |
| Ver documentación       |      ✅      |   ✅   |  ✅   |
| Editar documentación    |      ❌      |   ✅   |  ✅   |
| Actualizar roles de usuario |   ❌   |   ❌   |  ✅   |
| Eliminar usuarios       |      ❌      |   ❌   |  ✅   |
| Invitar usuarios con rol de admin  |   ❌   |   ❌   |  ✅   |
| Gestionar y actualizar la facturación |   ❌   |   ❌   |  ✅   |
| Actualizar el domain personalizado |   ❌   |   ❌   |  ✅   |
| Actualizar Git source   |      ❌      |   ❌   |  ✅   |
| Eliminar la organización |     ❌      |   ❌   |  ✅   |

### Rol de Visualizador

El rol de Visualizador proporciona acceso de solo lectura al dashboard. Los usuarios con este rol pueden ver la documentación y las analíticas, pero no pueden realizar cambios en el contenido o la configuración.

### Rol de Editor

El rol de Editor permite a los usuarios ver y editar el contenido de la documentación. Los editores pueden realizar cambios en la documentación, pero no pueden gestionar usuarios, facturación o configuración de la organización.

### Rol de Admin

El rol de Admin tiene acceso completo a todas las funciones del dashboard, incluyendo la gestión de usuarios, facturación y configuración de la organización.

## Reglas de escalada de privilegios

Cuando RBAC está habilitado, los usuarios no pueden invitar a otros con roles que ellos mismos no tienen. Por ejemplo:

- Un Visualizador no puede invitar a ningún usuario
- Un Editor no puede invitar Admins
- Un Admin puede invitar usuarios con cualquier rol

Puedes invitar a tantos admins como quieras, pero recomendamos limitar el acceso de admin a quienes realmente lo necesiten.
