Mintlify ofrece tres niveles de acceso al dashboard: Visualizador, Editor y Admin.

## Permisos por rol

La siguiente tabla describe los permisos para cada rol:

| ----------------------- | :----------: | :----: | :---: |
| Ver documentación       |      ✅      |   ✅   |  ✅   |
| Ver analíticas          |      ✅      |   ✅   |  ✅   |
| Editar contenido        |      ❌      |   ✅   |  ✅   |
| Publicar cambios        |      ❌      |   ✅   |  ✅   |
| Actualizar roles de usuario |   ❌      |   ❌   |  ✅   |
| Eliminar usuarios       |      ❌      |   ❌   |  ✅   |
| Invitar usuarios        |      ❌      |   ✅   |  ✅   |
| Gestionar y actualizar la facturación |  ❌  |   ❌   |  ✅   |
| Actualizar el domain personalizado |  ❌  |   ❌   |  ✅   |
| Actualizar Git source   |      ❌      |   ❌   |  ✅   |
| Eliminar la organización |     ❌      |   ❌   |  ✅   |

## Descripción de roles

**Visualizador**: Acceso de solo lectura para ver documentación y analíticas. No puede realizar cambios en el contenido o la configuración.

**Editor**: Puede ver y editar el contenido de la documentación, publicar cambios e invitar a otros usuarios. No puede modificar la configuración de la organización ni gestionar la facturación.

**Admin**: Acceso completo a todas las funciones del dashboard, incluida la configuración de la organización, la facturación y la gestión de usuarios.

## Reglas de escalada de privilegios

Cuando RBAC está habilitado, los usuarios no pueden invitar a otros con roles que ellos mismos no tienen. Por ejemplo:

- Un Visualizador no puede invitar a ningún usuario
- Un Editor puede invitar a Visualizadores y Editores, pero no a Admins
- Un Admin puede invitar usuarios con cualquier rol

Puedes invitar a tantos admins como quieras, pero recomendamos limitar el acceso de admin a quienes realmente lo necesiten.
