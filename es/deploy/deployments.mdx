Tu sitio de documentación se implementa automáticamente cuando haces push de cambios a tu repositorio conectado. Para ello, la Aplicación de GitHub de Mintlify debe estar correctamente instalada y conectada.

Si tus últimos cambios no aparecen en tu sitio en producción, primero verifica que la Aplicación de GitHub esté instalada en la cuenta u organización propietaria de tu repositorio de documentación. Consulta [Solución de problemas de GitHub](/es/deploy/github#troubleshooting) para más información.

Si la Aplicación de GitHub está conectada pero los cambios aún no se implementan, puedes activar manualmente una reconstrucción desde tu dashboard.

<div id="manually-triggering-a-deployment">
  ## Activar una implementación manualmente
</div>

<Steps>
  <Step title="Verifica que tu última confirmación se realizó correctamente.">
    Comprueba que tu confirmación más reciente aparezca en tu repositorio de documentación y que no haya presentado errores.
  </Step>

  <Step title="Activa una implementación manualmente.">
    Ve a tu [dashboard](https://dashboard.mintlify.com) y selecciona el botón de implementación.
