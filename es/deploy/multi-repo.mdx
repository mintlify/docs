<div id="navigation-behavior">
  ## Comportamiento de la navegaciﾃｳn
</div>

Mintlify combina la navegaciﾃｳn de cada repositorio en una ﾃｺnica navegaciﾃｳn del sitio. Cada fuente de repositorio se convierte en una secciﾃｳn de producto de nivel superior bajo la ruta de URL configurada.

El nombre de cada secciﾃｳn de producto proviene del campo `name` en el `docs.json` del repositorio correspondiente. Por ejemplo, si el `docs.json` de un repositorio establece `"name": "API Reference"`, su secciﾃｳn de producto aparece como "API Reference" en la navegaciﾃｳn combinada.

Mantﾃｩn la navegaciﾃｳn de cada fuente acotada a ese repositorio. Por ejemplo, las pﾃ｡ginas en el repositorio de API deben referenciar solo archivos que estﾃｩn en el repositorio de API. De la misma manera, las pﾃ｡ginas en el repositorio de SDK deben referenciar solo archivos que estﾃｩn en el repositorio de SDK.

Las configuraciones anidadas de `navigation.products` no son compatibles dentro de los repositorios de origen individuales.

<div id="link-between-sources">
  ### Enlazar entre fuentes
</div>

Los enlaces de Markdown relativos no se pueden resolver entre repositorios porque cada fuente solo ve sus propios archivos en el momento de la compilaciﾃｳn. Para enlazar desde una fuente a una pﾃ｡gina de otra fuente, usa una ruta URL relativa a la raﾃｭz que incluya la ruta URL de la fuente de destino:

```mdx

```

En este ejemplo, `/api` es la ruta URL configurada para la fuente del repositorio de API. El enlace se resuelve a nivel del sitio publicado como `docs.example.com/api/reference/authentication`.

No uses URLs completas con `https://` para enlaces internos entre fuentes: las rutas relativas a la raﾃｭz mantienen el correcto funcionamiento de las previsualizaciones y los dominios personalizados.

<div id="avoid-path-conflicts-across-sources">
  ### Evitar conflictos de rutas entre fuentes
</div>

Cuando una fuente usa la ruta de URL raﾃｭz, sus pﾃ｡ginas pueden compartir la misma URL final que las pﾃ｡ginas de otra fuente. <div id="reference-navigation-from-another-source">
  ## Referenciar la navegaciﾃｳn desde otra fuente
</div>

Usa `sourceRef` para ubicar la navegaciﾃｳn de otro repositorio en una posiciﾃｳn especﾃｭfica dentro del `docs.json` de la fuente base. Si no usas `sourceRef`, Mintlify agrega cada repositorio como una secciﾃｳn de producto de nivel superior independiente.

<div id="requirements">
  ### Requisitos
</div>

- El repositorio referenciado debe estar configurado como una fuente en la misma implementaciﾃｳn multi-repositorio.
- Los valores de `sourceRef` deben usar el formato `owner/repo`. Las rutas de montaje y los nombres de repositorio por sﾃｭ solos no son compatibles.
- La fuente referenciada debe definir un patrﾃｳn de navegaciﾃｳn similar al de la fuente principal. Por ejemplo, un `sourceRef` dentro de `anchors` requiere que la fuente referenciada defina `navigation.anchors`.
- Las entradas de `sourceRef` no pueden formar un ciclo. Una fuente no puede referenciarse a sﾃｭ misma, y dos fuentes no pueden referenciarse entre sﾃｭ.
- `sourceRef` debe aparecer dentro de un arreglo de navegaciﾃｳn. No es vﾃ｡lido en el nivel superior de `navigation`.

<div id="usage">
  ### Uso
</div>

Agrega una entrada `sourceRef` a un arreglo de navegaciﾃｳn en el `docs.json` de la fuente base. El valor debe ser el identificador `owner/repo` del repositorio.

```json
{ "sourceRef": "acme/api-docs" }
```

Mintlify admite `sourceRef` dentro de `anchors`, `tabs`, `groups`, `pages`, `products` y los arreglos bajo `navigation.global`.

Por ejemplo, para combinar los anchors de varios repositorios en una sola navegaciﾃｳn de anchors:

```json
{
  "navigation": {
    "anchors": [
      {
        "anchor": "Guides",
        "pages": ["quickstart"]
      },
      {
        "sourceRef": "acme/api-docs"
      }
    ]
  }
}
```

Si `acme/api-docs` define su propio `navigation.anchors`, Mintlify reemplaza la entrada `sourceRef` con esos anchors y les antepone la ruta de URL de la fuente referenciada.
