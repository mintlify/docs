Aloja tu documentación en una subruta como `your-domain.com/docs` para mantenerla en tu dominio principal, lo que la hace más fácil de encontrar y mantiene una experiencia de marca coherente para tus usuarios.

<div id="set-up-by-hosting-provider">
  ## Configurar según el proveedor de hosting
</div>

La configuración de una subruta `/docs` varía según tu proveedor de hosting. Elige tu proveedor de hosting y sigue las instrucciones de configuración.

* [Cloudflare](/es/deploy/cloudflare): Configura Cloudflare Workers para tu dominio y sirve tu documentación en la subruta `/docs`.
* [AWS Route 53 and CloudFront](/es/deploy/route53-cloudfront): Despliega tu documentación en la subruta `/docs` usando AWS con Route 53 DNS y el CDN de CloudFront.
* [Vercel](/es/deploy/vercel#host-from-docs) - Usa `rewrites` para desplegar tu documentación en la subruta `/docs`.
* [Custom reverse proxy](/es/deploy/reverse-proxy) - Para otras plataformas de hosting, configura un proxy inverso para desplegar tu documentación en la subruta `/docs`.

<div id="additional-configuration-for-strict-security-policies">
  ### Configuración adicional para políticas de seguridad estrictas
</div>

Si rediriges todo el tráfico a través de tu dominio personalizado, es posible que debas configurar tu proxy o las reglas de tu firewall para añadir las [cabeceras](/es/deploy/csp-configuration) correctas y que tu documentación se muestre correctamente.
