Aloja tu documentación en una subruta como `your-domain.com/docs` para mantenerla en tu dominio principal, lo que la hace más fácil de encontrar y mantiene una experiencia de marca coherente para tus usuarios.

<div id="let-mintlify-manage-your-domain">
  ## Deja que Mintlify gestione tu dominio
</div>

Si tu dominio no aloja nada más, Mintlify puede servir tu documentación en tu subruta sin un proxy. En la página de [configuración de dominio personalizado](https://app.mintlify.com/settings/deployment/custom-domain) en tu dashboard, habilita el interruptor **Host at**, ingresa tu ruta base y añade tu dominio. Luego configura los registros DNS que se muestran en tu dashboard para apuntar tu dominio a Mintlify. Consulta [Dominio personalizado](/es/customize/custom-domain) para obtener instrucciones de configuración.

Utiliza esta opción solo si Mintlify puede recibir todas las rutas del dominio.

<div id="set-up-by-hosting-provider">
  ## Configurar según el proveedor de hosting
</div>

Si otro sitio se ejecuta en tu dominio, configura tu proveedor de hosting para redirigir mediante proxy tu subruta a Mintlify. Elige tu proveedor de hosting y sigue las instrucciones de configuración.

- [Cloudflare](/es/deploy/cloudflare): Configura Cloudflare Workers para tu dominio y sirve tu documentación en tu subruta.
- [AWS Route 53 y CloudFront](/es/deploy/route53-cloudfront): Despliega tu documentación en tu subruta usando AWS con Route 53 DNS y el CDN de CloudFront.
- [Vercel](/es/deploy/vercel): Usa reescrituras para desplegar tu documentación en tu subruta.
- [Proxy inverso personalizado](/es/deploy/reverse-proxy): Para otras plataformas de hosting, configura un proxy inverso para desplegar tu documentación en tu subruta.

<div id="additional-configuration-for-strict-security-policies">
  ### Configuración adicional para políticas de seguridad estrictas
</div>

Si rediriges todo el tráfico a través de tu dominio personalizado, es posible que debas configurar tu proxy o las reglas de tu firewall para añadir las [cabeceras](/es/deploy/csp-configuration) correctas y que tu documentación se muestre correctamente.
