<div id="global-flags">
## Indicadores globales
</div>

Estos indicadores están disponibles en todos los comandos.

| Flag | Description |
| --- | --- |
| `--telemetry`, `-t` | Habilitar o deshabilitar la telemetría de uso anónima. |
| `--help`, `-h` | Mostrar la ayuda del comando. |
| `--version`, `-v` | Mostrar la versión del CLI. Alias de `mint version`. |

<div id="mint-dev">
## `mint dev`
</div>

Iniciar una vista previa local de tu documentación.

```bash
mint dev [flags]
```

| Flag | Description |
| --- | --- |
| `--port` | Puerto en el que ejecutar la vista previa local. Por defecto es `3000`. |
| `--no-open` | No abrir el navegador automáticamente. |
| `--groups` | Lista de grupos de usuarios separados por comas para simular en la vista previa. |
| `--disable-openapi` | Omitir el procesamiento de archivos OpenAPI para mejorar el rendimiento. |
| `--local-schema` | Permitir archivos OpenAPI alojados localmente servidos por HTTP. |

<div id="mint-login">
## `mint login`
</div>

Autenticarse con tu cuenta de Mintlify.

```bash
mint login
```

Abre una ventana del navegador para completar la autenticación. Si el navegador no se abre, el CLI muestra una URL para abrir manualmente y un mensaje para pegar el código de autorización. Las credenciales se guardan en `~/.config/mintlify/config.json`.

<div id="mint-analytics">
## `mint analytics`
</div>

Ver datos de analítica de tu documentación. Requiere autenticación con `mint login`.

```bash
mint analytics <subcommand> [flags]
```

Todos los subcomandos aceptan estos indicadores compartidos:

| Flag | Description |
| --- | --- |
| `--subdomain` | Subdominio de la documentación. Por defecto es el valor establecido con `mint config set subdomain`. |
| `--from` | Fecha de inicio en formato `YYYY-MM-DD`. Por defecto es hace 7 días, o el valor establecido con `mint config set dateFrom`. |
| `--to` | Fecha de fin en formato `YYYY-MM-DD`. Por defecto es hoy, o el valor establecido con `mint config set dateTo`. |
| `--format` | Formato de salida: `plain` (por defecto), `table`, `json` o `graph`. |
| `--agent` | Salida en JSON. Equivalente a `--format json`. También se activa automáticamente cuando la variable de entorno `CLAUDECODE` es igual a `1`. |

<div id="mint-analytics-stats">
### `mint analytics stats`
</div>

Mostrar un resumen de vistas, visitantes, búsquedas, comentarios y uso del asistente.

```bash
mint analytics stats [flags]
```

| Flag | Description |
| --- | --- |
| `--page` | Filtrar por una ruta de página específica. |
| `--humans` | Mostrar solo tráfico humano. |
| `--agents` | Mostrar solo tráfico de agentes de IA. |

<div id="mint-analytics-search">
### `mint analytics search`
</div>

Mostrar consultas de búsqueda con recuentos de resultados y tasas de clics.

```bash
mint analytics search [flags]
```

| Flag | Description |
| --- | --- |
| `--query` | Filtrar por subcadena de consulta de búsqueda. |
| `--page` | Filtrar por página con más clics. |

<div id="mint-analytics-feedback">
### `mint analytics feedback`
</div>

Mostrar comentarios enviados por los usuarios.

```bash
mint analytics feedback [flags]
```

| Flag | Description |
| --- | --- |
| `--type` | Tipo de comentario: `page` (agregar por página) o `code` (comentarios de fragmentos de código). |
| `--page` | Filtrar por una ruta de página específica. |

</div>

Ver datos de conversaciones del asistente.

#### `mint analytics conversation list`
</div>

```bash

```

| Flag | Description |
| --- | --- |


<div id="mint-config">
## `mint config`
</div>

Gestionar valores predeterminados persistentes para los comandos del CLI. La configuración se guarda en `~/.config/mintlify/config.json`.

```bash
mint config <subcommand> <key> [value]
```

| Subcommand | Description |
| --- | --- |
| `set <key> <value>` | Establecer un valor de configuración. |
| `get <key>` | Mostrar un valor de configuración. |
| `clear <key>` | Eliminar un valor de configuración. |

<div id="configuration-keys">
### Claves de configuración
</div>

| Key | Description | Used by |
| --- | --- | --- |
| `subdomain` | Subdominio de documentación predeterminado. | `mint analytics` |
| `dateFrom` | Fecha de inicio predeterminada para consultas de analítica (`YYYY-MM-DD`). | `mint analytics` |
| `dateTo` | Fecha de fin predeterminada para consultas de analítica (`YYYY-MM-DD`). | `mint analytics` |

<div id="mint-broken-links">
## `mint broken-links`
</div>

Verificar enlaces internos rotos en tu documentación.

```bash
mint broken-links [flags]
```

El comando excluye archivos que coinciden con los patrones de [.mintignore](/es/organize/mintignore). Los enlaces que apuntan a archivos ignorados se reportan como rotos.

| Flag | Description |
| --- | --- |
| `--check-anchors` | También validar enlaces de ancla (por ejemplo, `/page#section`) contra los slugs de encabezados. |
| `--check-external` | También verificar URLs externas en busca de enlaces rotos. |
| `--check-snippets` | También verificar enlaces dentro de componentes `<Snippet>`. |

<div id="mint-validate">
## `mint validate`
</div>

Validar la compilación de tu documentación en modo estricto. Sale con un error si hay advertencias o errores. Incluye validación automática de especificaciones OpenAPI referenciadas en tu `docs.json`.

```bash
mint validate [flags]
```

| Flag | Description |
| --- | --- |
| `--groups` | Lista de grupos de usuarios separados por comas para simular durante la validación. |
| `--disable-openapi` | Omitir el procesamiento y la validación de archivos OpenAPI. |
| `--local-schema` | Permitir la validación de archivos OpenAPI alojados localmente servidos por HTTP. Solo admite HTTPS en producción. |

<Note>
  El comando independiente `mint openapi-check` está obsoleto. Usa `mint validate` en su lugar.
</Note>

<div id="telemetry">
## Telemetría
</div>

El CLI recopila telemetría de uso anónima para ayudar a mejorar Mintlify. Los datos de telemetría incluyen el nombre del comando, la versión del CLI, el sistema operativo y la arquitectura. Mintlify **no** recopila información de identificación personal, contenido del proyecto ni rutas de archivos.

Por defecto, el CLI recopila datos de telemetría. Puedes desactivarla en cualquier momento usando el indicador `--telemetry`:

```bash
# Disable telemetry
mint --telemetry false

# Re-enable telemetry
mint --telemetry true
```

También puedes deshabilitar la telemetría configurando una de estas variables de entorno:

| Variable | Value | Description |
| --- | --- | --- |
| `MINTLIFY_TELEMETRY_DISABLED` | `1` | Deshabilitar la telemetría del CLI de Mintlify. |
| `DO_NOT_TRACK` | `1` | Deshabilitar la telemetría usando el estándar [Console Do Not Track](https://consoledonottrack.com/). |

Tu preferencia se guarda en `~/.config/mintlify/config.json` y persiste entre sesiones del CLI.
