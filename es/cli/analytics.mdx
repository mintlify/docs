El comando `mint analytics` te da acceso a la [analítica de tu documentación](/es/optimize/analytics) desde la terminal. Esto es útil para generar informes de analítica mediante scripts, canalizar datos hacia otras herramientas y realizar verificaciones rápidas sin abrir el panel de control.

<div id="prerequisites">
  ## Requisitos previos
</div>

Debes iniciar sesión para usar los comandos de analítica. Ejecuta `mint login` para autenticarte con tu cuenta de Mintlify.

Consulta [Previsualización local](/es/cli/preview#log-in-for-search-and-assistant) para más detalles sobre la autenticación.

<div id="view-key-metrics">
  ## Ver métricas clave
</div>

Muestra un resumen de visitas, visitantes, búsquedas, comentarios y uso del asistente:

```bash
mint analytics stats
```

Usa flags para filtrar por rango de fechas o página:

- `--from`: Fecha de inicio en formato `YYYY-MM-DD`. Por defecto, 7 días atrás.
- `--to`: Fecha de fin en formato `YYYY-MM-DD`. Por defecto, hoy.
- `--page`: Filtrar por una ruta de página específica.

```bash Example analytics stats flags
mint analytics stats --from 2025-01-01 --to 2025-01-31
mint analytics stats --page /quickstart
```

<div id="view-search-analytics">
  ## Ver analítica de búsqueda
</div>

Consulta qué buscan tus usuarios, con recuento de resultados y tasas de clics:

```bash
mint analytics search
```

Usa flags para filtrar por cadena de búsqueda o página:

- `--query`: Filtrar por subcadena de consulta de búsqueda.
- `--page`: Filtrar por una ruta de página específica.

```bash Example analytics search flags
mint analytics search --query "authentication"

mint analytics search --page /api-reference
```

<div id="view-feedback">
  ## Ver comentarios
</div>

Revisa los comentarios enviados por los usuarios en las páginas de tu documentación:

```bash
mint analytics feedback
```

Usa flags para agregar comentarios por página o filtrar por comentarios de fragmentos de código:

- `--type`: Agregar comentarios por página o fragmento de código. Por defecto, página.
- `--page`: Filtrar por una ruta de página específica.

```bash Example analytics feedback flags
mint analytics feedback --type page
mint analytics feedback --type code
```

<div id="view-assistant-conversations">
  ## Ver conversaciones del asistente
</div>

Lista las conversaciones recientes y las categorías de conversaciones del asistente de IA:

```bash
mint analytics conversation list
mint analytics conversation buckets list
```

Cada salida de lista incluye un ID para cada entrada. Pasa ese ID al comando `view` correspondiente para ver los detalles completos:

```bash
mint analytics conversation view <conversation-id>
mint analytics conversation buckets view <bucket-id>
```

<div id="choose-an-output-format">
  ## Elegir un formato de salida
</div>

Todos los comandos de analítica usan por defecto la salida `plain`, que está separada por tabulaciones y es adecuada para canalizar hacia otras herramientas. Usa `--format` para cambiar la salida:

```bash
mint analytics stats --format table    # Tabla con formato y colores
mint analytics stats --format json     # JSON sin formato
mint analytics stats --format graph    # Gráfico de barras horizontal
```

La CLI detecta automáticamente entornos de agentes de IA como Claude Code o terminales no interactivas y usa por defecto la salida en JSON.

<div id="set-defaults-with-config">
  ## Establecer valores predeterminados con config
</div>

Para evitar repetir flags comunes, establece valores predeterminados con `mint config`. La configuración se guarda en `~/.config/mintlify/config.json`.

Cuando ejecutas `mint login`, la CLI establece la clave `subdomain`. Si tienes múltiples despliegues, la CLI te solicita que elijas uno. Puedes cambiar el valor predeterminado en cualquier momento con `mint config set subdomain <subdomain>`.

Claves de configuración disponibles:

| Clave | Descripción |
| --- | --- |
| `subdomain` | Subdominio predeterminado para comandos de analítica. Establecido por el comando `mint login`. |
| `dateFrom` | Fecha de inicio predeterminada para consultas de analítica (`YYYY-MM-DD`). |
| `dateTo` | Fecha de fin predeterminada para consultas de analítica (`YYYY-MM-DD`). |

```bash Example configs
mint config set subdomain my-docs
mint config set dateFrom 2025-01-01
```

Una vez establecidos, los comandos de analítica usan estos valores automáticamente:

```bash
# Usa el subdominio y dateFrom configurados sin flags
mint analytics stats
```

Para ver o borrar un valor, ejecuta:

```bash
mint config get subdomain
mint config clear subdomain
```
