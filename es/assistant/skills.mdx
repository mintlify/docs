Las skills son guías en Markdown escritas por tu equipo que viven junto a tu documentación. El assistant ve un catálogo de tus skills en su prompt de sistema y carga la guía completa solo cuando la pregunta de un usuario coincide claramente con una, para que puedas ofrecer respuestas expertas y curadas para las preguntas que más importan sin sobrecargar cada respuesta.

Usa skills cuando:

- Un tema necesita una respuesta específica y con criterio propio que los resultados de búsqueda genéricos no producen de manera confiable.
- Quieres codificar playbooks internos, flujos de resolución de problemas o guías de migración que no están estructurados como páginas de documentación regulares.
- Quieres definir cómo el assistant guía a los usuarios a través de procedimientos de varios pasos o con matices.

<div id="add-a-skill">
  ## Agregar una skill
</div>

Crea una skill agregando un archivo `skill.md` dentro de un directorio con nombre de slug bajo `.mintlify/assistant/skills/`:

```
.mintlify/
└── assistant/
    └── skills/
        ├── migrate-to-v3/
        │   └── skill.md
        └── debug-webhooks/
            └── skill.md
```

Cada archivo `skill.md` necesita frontmatter YAML con un `name` y una `description`, seguido del contenido completo de la guía:

```markdown .mintlify/assistant/skills/migrate-to-v3/skill.md
---
name: Migrate to SDK v3
description: Step-by-step guide for upgrading from SDK v2 to v3, including breaking changes, code mods, and rollout checklist.
---

# Migrating to SDK v3

Follow these steps to upgrade an existing v2 integration to v3.

## 1. Update dependencies

...
```

El assistant usa `name` y `description` para decidir si carga la skill, así que escribe descripciones que indiquen claramente cuándo aplica la guía. Mantén el cuerpo enfocado: el contenido de la skill se trunca aproximadamente a 6,000 tokens cuando se carga.

<Note>
  Las skills se descubren en tu próxima actualización de documentación. Después de agregar o cambiar un archivo `skill.md`, publica tus docs para que la skill esté disponible para el assistant.
</Note>

<div id="how-the-assistant-uses-skills">
  ## Cómo usa las skills el assistant
</div>

- El assistant lista cada skill disponible (nombre, slug y descripción) en su prompt de sistema.
- Cuando la pregunta de un usuario coincide con una skill, el assistant llama a la herramienta `loadSkill` con el slug de la skill y usa la guía devuelta como contexto autoritativo.
- El assistant puede cargar hasta tres skills por respuesta.
- El contenido de las skills cargadas se trata como orientación escrita por el equipo y tiene precedencia sobre los resultados de búsqueda generales.
- En turnos posteriores, el contenido de las skills cargadas previamente se elimina del historial de la conversación para mantener el contexto pequeño; el assistant recarga una skill si la necesita de nuevo.

Las skills funcionan junto con las [instrucciones personalizadas del assistant](/es/assistant/customize) en `.mintlify/Assistant.md`. Usa `Assistant.md` para el tono, persona y reglas globales; usa skills para orientación profunda y específica de un tema que el assistant debería incorporar solo cuando sea relevante.

<div id="skill-authoring-tips">
  ## Consejos para escribir skills
</div>

- Dale a cada skill una `description` distinta e inequívoca. Las descripciones que se superponen dificultan que el assistant elija la skill correcta.
- Usa un slug único y seguro para URLs en cada nombre de directorio. Los slugs solo pueden contener letras minúsculas, números y guiones. Los slugs duplicados se omiten con una advertencia durante la publicación.
- Estructura las guías con encabezados claros y pasos numerados. Los assistants siguen el contenido procedimental de forma más confiable cuando es explícito.
- Mantén las guías autocontenidas. El assistant carga una skill a la vez, así que no debería requerir otra skill para tener sentido.
- Prefiere prosa y ejemplos de código sobre capturas de pantalla. El assistant solo lee el cuerpo en Markdown.

<div id="disable-skills">
  ## Deshabilitar skills
</div>

Las skills están activadas de forma predeterminada cuando hay al menos un archivo `skill.md` presente. Para desactivar las skills por completo en tu deployment, contacta a [soporte](mailto:support@mintlify.com) para deshabilitar la función de assistant skills. Eliminar el directorio `.mintlify/assistant/skills/` y volver a publicar también limpia el catálogo.
