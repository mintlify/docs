Los usuarios tienen múltiples formas de iniciar una conversación con el assistant. Cada método abre un panel de chat en el lado derecho de tu documentación. Los usuarios pueden hacer cualquier pregunta y el assistant busca en tu documentación una respuesta. Si el assistant no puede recuperar información relevante, responde que no puede contestar la pregunta.

<div id="use-the-assistant-in-local-preview">
  ## Usar el assistant en la previsualización local
</div>

El assistant está disponible en tu previsualización local cuando inicias sesión con la CLI. Esto te permite probar el comportamiento del assistant, las instrucciones personalizadas en `Assistant.md` y las respuestas mientras desarrollas documentación localmente.

1. Si aún no lo has hecho, instala la CLI con `npm i -g mint`.
2. Ejecuta `mint login` para autenticarte.
3. Ejecuta `mint dev` para iniciar la previsualización local.

El assistant en la previsualización local usa el mismo contenido indexado que tu sitio de documentación desplegado.

<div id="keyboard-shortcut">
  ## Atajo de teclado
</div>

Abre el panel de chat del assistant con el atajo de teclado <kbd>Command</kbd> + <kbd>I</kbd> en macOS y <kbd>Ctrl</kbd> + <kbd>I</kbd> en Windows.

<div id="highlight-text">
  ## Resaltar texto
</div>

Resalta texto en una página y haz clic en el botón emergente **Add to assistant** para abrir el panel de chat del assistant y agregar el texto resaltado como contexto. Puedes agregar múltiples fragmentos de texto o bloques de código al contexto del assistant.

<div id="file-attachments">
  ## Archivos adjuntos
</div>

Adjunta archivos a los mensajes del assistant para proporcionar contexto adicional.

Tipos de archivo compatibles:
- **Imágenes**: JPEG, PNG, GIF, WebP, SVG
- **Documentos**: PDF
- **Archivos de código y texto**: JavaScript (`.js`, `.jsx`, `.mjs`, `.cjs`), TypeScript (`.ts`, `.tsx`), Python, HTML, CSS, Markdown, MDX, JSON, YAML, XML, SQL, CSV, texto plano, scripts de shell (`.sh`, `.bash`, `.env`), GraphQL, TOML, Go, Rust, Ruby, Java, Kotlin, Swift, C (`.c`, `.h`), C++ (`.cpp`, `.hpp`), C#, PHP, Lua, R, Scala

Límites:
- Tamaño máximo de archivo: 5 MB por adjunto
- Máximo de adjuntos por mensaje: 10

<div id="urls">
  ## URLs
</div>

Abre el assistant con un parámetro de consulta en la URL para crear enlaces directos que guíen a los usuarios a información específica o compartir conversaciones del assistant con preguntas precargadas.

- **Abrir el assistant**: Agrega `?assistant=open` para abrir el panel de chat del assistant cuando se cargue la página.
  - Ejemplo: [https://mintlify.com/docs?assistant=open](https://mintlify.com/docs?assistant=open)
- **Abrir con una consulta precargada**: Agrega `?assistant=TU_CONSULTA` para abrir el assistant y enviar automáticamente una pregunta.
  - Ejemplo: [https://mintlify.com/docs?assistant=explain webhooks](https://mintlify.com/docs?assistant=explain%20webhooks)

<div id="troubleshooting">
  ## Solución de problemas
</div>

<Accordion title="La barra de chat del assistant no es visible">
  Si la interfaz del assistant no es visible en ciertos navegadores, es posible que necesites enviar un reporte de falso positivo a [EasyList](https://easylist.to). Los navegadores que usan EasyList Cookies List, como Brave y Comet, a veces bloquean el assistant u otros elementos de la interfaz. EasyList Cookies List incluye una regla específica de dominio que oculta elementos fijos en ciertos dominios para bloquear banners de cookies. Esta regla afecta inadvertidamente a componentes legítimos de la interfaz.

  Envía un reporte de falso positivo a [EasyList](https://github.com/easylist/easylist) para solicitar la eliminación de la regla. Esto resuelve el problema para todos los usuarios una vez que se actualice la lista de filtros.

</Accordion>
