Cuando cambias la ruta de un archivo en tu carpeta de documentación, también cambia la ruta de la URL de esa página. Esto puede ocurrir al reestructurar tu documentación o al cambiar el title de la barra lateral.

<div id="broken-links">
  ## Enlaces rotos
</div>

Detecta enlaces rotos con nuestra CLI. [Instala la CLI](/es/installation) y ejecuta el siguiente comando:

```bash
mint enlaces-rotos
```

La CLI identifica los enlaces relativos inexistentes en tu documentación.

<div id="redirects">
  ## Redirecciones
</div>

Configura redirecciones 301 agregando el campo `redirects` a tu archivo `docs.json`.

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

Esto redirige de forma permanente `/source/path` a `/destination/path` para que no pierdas el SEO acumulado de la página original.

Para hacer coincidir una ruta con comodín, usa `*` después de un parámetro. En este ejemplo, `/beta/:slug*` coincide con `/beta/introduction` y la redirige a `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
