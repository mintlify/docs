Mintlify gestiona automáticamente muchas buenas prácticas de SEO, entre ellas:

* Generación de metaetiquetas
* Generación del sitemap y del archivo `robots.txt`
* Estructura HTML semántica
* Optimización para móviles

Puedes personalizar por completo las metaetiquetas de tu sitio añadiendo el campo `metatags` a tu `docs.json` o al frontmatter de una página.

<div id="global-meta-tags">
  ## Metaetiquetas globales
</div>

Puedes configurar metaetiquetas predeterminadas añadiendo el campo `metadata` a tu `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "enlace a tu imagen de meta etiqueta por defecto"
    }
}
```

<div id="page-specific-meta-tags">
  ## Metatags específicos de la página
</div>

Agrega tus metatags a la metadata de la página de la siguiente manera:

```mdx
---
title: 'Título de tu página de ejemplo'
'og:image': 'enlace a la imagen de tu meta etiqueta'
---
```

<Warning>
  Las etiquetas meta con dos puntos deben ir entre comillas cuando las incluyas en la metadata de la página.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps y archivos robots.txt
</div>

Mintlify genera automáticamente un archivo `sitemap.xml` y un archivo `robots.txt`. Puedes ver tu sitemap añadiendo `/sitemap.xml` a la URL de tu sitio de documentación.

De forma predeterminada, solo se incluyen las páginas definidas en tu `docs.json`. Para incluir enlaces ocultos, añade `seo.indexing` a tu `docs.json`:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Sitemaps personalizados y archivos robots.txt
</div>

Para añadir un archivo `sitemap.xml` o `robots.txt` personalizado, crea un archivo `sitemap.xml` o `robots.txt` en la raíz de tu proyecto. Añadir un archivo personalizado reemplazará el archivo generado automáticamente con el mismo nombre. Si eliminas un archivo personalizado, se volverá a usar el archivo predeterminado.

<div id="disabling-indexing">
  ## Desactivar la indexación
</div>

Si quieres impedir que los motores de búsqueda indexen una página, puedes incluir lo siguiente en el [frontmatter](es/pages) de tu página:

```
---
noindex: true
---
```

También puedes especificar `noindex` para todas las páginas de tu documentación configurando el campo `metadata.robots` en `"noindex"` en tu `docs.json`:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## Mejores prácticas de SEO
</div>

<AccordionGroup>
  <Accordion title="Escribe títulos y descripciones claras">
    * Usa títulos de página claros y descriptivos (50-60 caracteres)
    * Redacta descripciones convincentes (150-160 caracteres)
    * Incluye palabras clave relevantes
    * Asegúrate de que cada título y descripción de página sean únicos
  </Accordion>

  <Accordion title="Optimiza la estructura de tu contenido">
    * Usa una jerarquía de encabezados adecuada (H1 → H2 → H3)
    * Escribe primero para las personas y después para los motores de búsqueda
    * Incluye palabras clave relevantes en encabezados y en el contenido
    * Mantén las URL cortas, descriptivas y organizadas jerárquicamente
    * Divide el contenido extenso con subtítulos y listas
  </Accordion>

  <Accordion title="Estrategia de enlaces internos">
    * Enlaza a páginas relacionadas dentro de tu documentación
    * Usa texto de anclaje descriptivo en lugar de &quot;haz clic aquí&quot;
    * Crea clústeres temáticos enlazando conceptos relacionados
    * Usa las funciones automáticas de referencias cruzadas
  </Accordion>

  <Accordion title="SEO de imágenes">
    * Usa nombres de archivo descriptivos para las imágenes
    * Incluye siempre texto alternativo por accesibilidad y SEO
    * Optimiza el tamaño de los archivos de imagen para una carga más rápida
    * Usa imágenes relevantes que respalden tu contenido
  </Accordion>
</AccordionGroup>
