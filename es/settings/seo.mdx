Mintlify gestiona automáticamente muchas de las mejores prácticas de SEO, entre ellas:

* Generación de metaetiquetas
* Generación del sitemap y del archivo `robots.txt`
* Estructura HTML semántica
* Optimización para dispositivos móviles

Puedes personalizar por completo las metaetiquetas de tu sitio añadiendo el campo `metatags` a tu `docs.json` o al frontmatter de una página.

<div id="global-meta-tags">
  ## Metadatos globales
</div>

Puedes configurar metadatos predeterminados añadiendo el campo `metadata` a tu `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "enlace a tu imagen de etiqueta meta por defecto"
    }
}
```

<div id="page-specific-meta-tags">
  ## Metadatos específicos de la página
</div>

Agrega tus meta tags a los metadatos de la página de la siguiente manera:

```mdx
---
title: 'Título de ejemplo de tu página'
'og:image': 'enlace a la imagen de tu etiqueta meta'
---
```

<Warning>
  Las metaetiquetas con dos puntos deben escribirse entre comillas cuando las incluyas en los metadatos de la página.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps y archivos robots.txt
</div>

Mintlify genera automáticamente un archivo `sitemap.xml` y un archivo `robots.txt`. Puedes ver tu sitemap añadiendo `/sitemap.xml` a la URL de tu sitio de documentación.

De forma predeterminada, solo se incluyen las páginas presentes en tu `docs.json`. Para incluir enlaces ocultos, añade `seo.indexing` a tu `docs.json`:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Sitemaps personalizados y archivos robots.txt
</div>

Para añadir un `sitemap.xml` o `robots.txt` personalizado, crea un archivo `sitemap.xml` o `robots.txt` en la raíz de tu proyecto. Añadir un archivo personalizado sustituirá al archivo generado automáticamente con el mismo nombre. Si eliminas el archivo personalizado, se volverá a usar el archivo predeterminado.

<div id="disabling-indexing">
  ## Deshabilitar la indexación
</div>

Si deseas impedir que los motores de búsqueda indexen una página, puedes incluir lo siguiente en el [frontmatter](es/pages) de tu página:

```
---
noindex: true
---
```

También puedes aplicar `noindex` a todas las páginas de tu documentación estableciendo el campo `metadata.robots` en `"noindex"` dentro de tu `docs.json`:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## Mejores prácticas de SEO
</div>

<AccordionGroup>
  <Accordion title="Escribe títulos y descripciones claros">
    * Usa títulos de página claros y descriptivos (50-60 caracteres)
    * Escribe descripciones convincentes (150-160 caracteres)
    * Incluye palabras clave relevantes
    * Haz que el título y la descripción de cada página sean únicos
  </Accordion>

  <Accordion title="Optimiza la estructura de tu contenido">
    * Usa una jerarquía adecuada de encabezados (H1 → H2 → H3)
    * Escribe primero para personas y luego para motores de búsqueda
    * Incluye palabras clave relevantes en encabezados y contenido
    * Mantén las URL cortas, descriptivas y organizadas de forma jerárquica
    * Divide el contenido extenso con subtítulos y listas
  </Accordion>

  <Accordion title="Estrategia de enlaces internos">
    * Enlaza a páginas relacionadas dentro de tu documentación
    * Usa texto de anclaje descriptivo en lugar de &quot;haz clic aquí&quot;
    * Crea grupos temáticos enlazando conceptos relacionados
    * Usa las funciones automáticas de referencias cruzadas
  </Accordion>

  <Accordion title="SEO para imágenes">
    * Usa nombres de archivo descriptivos para las imágenes
    * Incluye siempre texto alternativo para accesibilidad y SEO
    * Optimiza el tamaño de las imágenes para una carga más rápida
    * Usa imágenes relevantes que refuercen tu contenido
  </Accordion>
</AccordionGroup>
