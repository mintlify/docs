---
title: "Página 404 personalizada"
description: "Personaliza el título y la descripción de tu página de error 404."
keywords: ["404 error page", "custom error pages"]
---

Todas las páginas de error 404 sugieren automáticamente páginas relevantes según la URL solicitada. Si los usuarios llegan a una página 404, un agente analiza la ruta y recupera páginas semánticamente relacionadas de tu documentación para ayudarlos a encontrar lo que buscan.

Puedes personalizar el título y la descripción de la página de error 404. Usa la descripción para añadir enlaces útiles o indicaciones que orienten a los usuarios.

<div id="configuration">
  ## Configuración
</div>

Configura la página 404 en la sección `errors.404` de tu archivo `docs.json`:

```json
"errors": {
  "404": {
    "redirect": false,
    "title": "No se puede encontrar",
    "description": "¿Qué **pasó** con esta _página_?"
  }
}
```


<div id="parameters">
  ## Parámetros
</div>

<ResponseField name="redirect" type="boolean">
  Si se debe redirigir automáticamente a la página de inicio cuando no se encuentra una página.
  
  Configúralo en `true` para redirigir a la página de inicio.

  Configúralo en `false` para mostrar la página 404 personalizada.
</ResponseField>

<ResponseField name="title" type="string">
  Título personalizado para la página de error 404. Reemplaza el encabezado predeterminado “Page not found”.
</ResponseField>

<ResponseField name="description" type="string">
  Descripción personalizada para la página de error 404. Admite formato Markdown.
</ResponseField>
