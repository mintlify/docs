---
- title: Solución de problemas
- description: Resuelve problemas comunes con la configuración de páginas de API.
---

Si tus páginas de API no se muestran correctamente, verifica estos problemas comunes de configuración.

<AccordionGroup>
  <Accordion title="Todas mis páginas OpenAPI están completamente en blanco">
    En este escenario, es probable que Mintlify no pueda encontrar tu documento OpenAPI,
    o que tu documento OpenAPI no sea válido.

    Ejecutar `mint dev` localmente debería revelar algunos de estos problemas.

    Para verificar que tu documento OpenAPI pasará la validación:

    1. Visita [este validador](https://editor.swagger.io/)
    2. Cambia a la pestaña "Validate text"
    3. Pega tu documento OpenAPI
    4. Haz clic en "Validate it!"

    Si el cuadro de texto que aparece debajo tiene un borde verde, tu documento ha pasado la validación.
    Este es exactamente el paquete de validación que Mintlify utiliza para validar documentos OpenAPI, así que si tu documento
    pasa la validación aquí, hay una gran probabilidad de que el problema esté en otro lugar.

    Además, Mintlify no admite OpenAPI 2.0. Si tu documento utiliza esta versión de la especificación,
    podrías encontrarte con este problema. Puedes convertir tu documento en [editor.swagger.io](https://editor.swagger.io/) (en Edit > Convert to OpenAPI 3):

    <Frame>
      ![Screenshot of the Swagger Editor with the Edit menu expanded and the "Convert to OpenAPI 3" menu item highlighted.](/images/convert-oas-3.png)
    </Frame>
  </Accordion>

  <Accordion title="Una de mis páginas OpenAPI está completamente en blanco">
    Esto suele ser causado por un campo `openapi` mal escrito en los metadatos de la página. Asegúrate de que
    el método HTTP y la ruta coincidan exactamente con el método HTTP y la ruta en el documento OpenAPI.

    Aquí hay un ejemplo de cómo las cosas pueden salir mal:

    ```mdx get-user.mdx
    ---
    openapi: "GET /users/{id}/"
    ---
    ```

    ```yaml openapi.yaml
    paths:
      "/users/{id}":
        get: ...
    ```

    Observa que la ruta en el campo `openapi` tiene una barra diagonal al final, mientras que la ruta en el documento OpenAPI
    no la tiene.

    Otro problema común es un nombre de archivo mal escrito. Si estás especificando un documento OpenAPI particular
    en el campo `openapi`, asegúrate de que el nombre del archivo sea correcto. Por ejemplo, si tienes dos documentos OpenAPI
    `openapi/v1.json` y `openapi/v2.json`, tus metadatos podrían verse así:

    ```mdx api-reference/v1/users/get-user.mdx
    ---
    openapi: "v1 GET /users/{id}"
    ---
    ```
  </Accordion>

  <Accordion title="Las solicitudes desde el API Playground no funcionan">
    Si tienes configurado un dominio personalizado, esto podría ser un problema con tu proxy inverso. Por
    defecto, las solicitudes realizadas a través del API Playground comienzan con una solicitud `POST` a la ruta

    `/_mintlify/api/request` en el sitio de documentación. Si tu proxy inverso está configurado para permitir solo solicitudes `GET`,
    entonces todas estas solicitudes fallarán. Para solucionarlo, configura tu proxy inverso para
    permitir solicitudes `POST` a la ruta `/_mintlify/api/request`.

    Alternativamente, si tu proxy inverso te impide aceptar solicitudes `POST`, puedes configurar Mintlify para enviar solicitudes directamente a tu backend con la configuración `api.playground.proxy` en el `docs.json`, como se describe en la [settings documentation](/es/organize/settings#param-proxy). Al usar esta configuración, necesitarás configurar CORS en tu servidor ya que las solicitudes vendrán directamente de los navegadores de los usuarios en lugar de a través de tu proxy.
  </Accordion>

  <Accordion title="Las entradas de navegación OpenAPI no están generando páginas">
    Si estás utilizando una configuración de navegación OpenAPI, pero las páginas no se están generando, verifica estos problemas comunes:

    1. **Falta la especificación OpenAPI predeterminada**: Asegúrate de tener un campo `openapi` establecido para el elemento de navegación:

    ```json {5}
    "navigation": {
      "groups": [
        {
          "group": "API reference",
          "openapi": "/path/to/openapi.json",
          "pages": [
            "GET /users",
            "POST /users"
          ]
        }
      ]
    }
    ```

    2. **Herencia de especificación OpenAPI**: Si utilizas navegación anidada, asegúrate de que los grupos secundarios hereden la especificación OpenAPI correcta o especifiquen la suya propia.

    3. **Problemas de validación**: Usa `mint openapi-check <path-to-openapi-file>` para verificar que tu documento OpenAPI sea válido.
  </Accordion>

  <Accordion title="Algunas operaciones OpenAPI aparecen en la navegación pero otras no">
    1. **Operaciones ocultas**: Las operaciones marcadas con `x-hidden: true` en tu especificación OpenAPI no aparecerán en la navegación autogenerada.
    2. **Operaciones inválidas**: Las operaciones con errores de validación en la especificación OpenAPI pueden ser omitidas. Verifica tu documento OpenAPI en busca de errores de sintaxis.
    3. **Inclusión manual vs automática**: Si haces referencia a cualquier endpoint desde una especificación OpenAPI, solo las operaciones referenciadas explícitamente aparecerán en la navegación. No se añadirán automáticamente otras páginas. Esto incluye operaciones que se referencian en elementos de navegación secundarios.
  </Accordion>

  <Accordion title="La navegación mixta (páginas OpenAPI y MDX) no funciona correctamente">
    Al combinar operaciones OpenAPI con páginas de documentación regulares en la navegación:

    1. **Conflictos de archivos**: No puedes tener tanto un archivo `MDX` como una entrada de navegación para la misma operación. Por ejemplo, si tienes `get-users.mdx`, no incluyas también `"GET /users"` en tu navegación. Si necesitas tener un archivo que comparta nombre con una operación, usa la extensión `x-mint` para el endpoint para que el href apunte a una ubicación diferente.
    2. **Resolución de rutas**: Las entradas de navegación que no coincidan con operaciones de OpenAPI serán tratadas como rutas de archivos. Asegúrate de que tus `MDX` archivos existan en las ubicaciones esperadas.
    3. **Sensibilidad a mayúsculas y minúsculas**: La coincidencia de operaciones de OpenAPI distingue entre mayúsculas y minúsculas. Asegúrate de que los métodos HTTP estén en mayúsculas en las entradas de navegación.
  </Accordion>
</AccordionGroup>
