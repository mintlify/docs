## Visión general

El playground de API es un entorno interactivo que permite a los usuarios probar y explorar los endpoints de tu API. Los desarrolladores pueden crear solicitudes de API, enviarlas y ver las respuestas sin salir de tu documentación.

Consulta [Trigger an update](/es/api/update/trigger) para ver un ejemplo del playground de API en acción.

El playground genera páginas interactivas para tus endpoints basadas en tu especificación OpenAPI o esquema AsyncAPI. Si modificas tu API, el playground actualiza automáticamente las páginas relevantes.

Recomendamos generar tu playground de API desde una especificación OpenAPI. Sin embargo, puedes crear manualmente páginas de referencia de API después de definir una URL base y un método de autenticación en tu `docs.json`.

## Comenzar

<Steps>
  <Step title="Añade tu archivo de especificación OpenAPI.">
    <Tip>
      Valida tu archivo de especificación OpenAPI usando el [Swagger Editor](https://editor.swagger.io/) o el comando [Mint CLI](https://www.npmjs.com/package/mint) `mint openapi-check <filename>`.
    </Tip>

    ```bash {3}
    /your-project
      |- docs.json
      |- openapi.json
    ```
  </Step>

  <Step title="Genera páginas de endpoints.">
    Actualiza tu `docs.json` para referenciar tu especificación OpenAPI.

    **Para generar automáticamente páginas para todos los endpoints en tu especificación OpenAPI**, añade una propiedad `openapi` a cualquier elemento de navegación.

    Este ejemplo genera una página para cada endpoint especificado en `openapi.json` y organiza las páginas en el grupo "API reference".

    ```json Generate all endpoint pages
    "navigation": {
      "groups": [
        {
          "group": "API reference",
          "openapi": "openapi.json"
        }
      ]
    }
    ```

    **Para generar páginas solo para endpoints específicos**, enumera los endpoints en la propiedad `pages` del elemento de navegación.

    Este ejemplo genera páginas solo para los endpoints `GET /users` y `POST /users`. Para generar otras páginas de endpoints, añade endpoints adicionales al array `pages`.

    ```json Generate specific endpoint pages
    "navigation": {
      "groups": [
          {
            "group": "API reference",
            "openapi": "openapi.json",
            "pages": [
              "GET /users",
              "POST /users"
            ]
          }
      ]
    }
    ```
  </Step>
</Steps>

## Personaliza tu playground

Personaliza tu playground de API definiendo las siguientes propiedades en tu `docs.json`.

<ResponseField name="playground" type="object">
  Configuraciones para el playground de API.

  <Expandable title="playground" defaultOpen="True">
    <ResponseField name="display" type="&#x22;interactive&#x22; | &#x22;simple&#x22; | &#x22;none&#x22;">
      El modo de visualización del playground de API.

      * `"interactive"`: Muestra el playground interactivo.
      * `"simple"`: Muestra un endpoint copiable sin playground.
      * `"none"`: No muestra nada.

      Por defecto es `interactive`.
    </ResponseField>

    <ResponseField name="proxy" type="boolean" defaultOpen="True">
      Si se deben pasar las solicitudes de API a través de un servidor proxy. Por defecto es `true`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="examples" type="object">
  Configuraciones para los ejemplos de API autogenerados.

  <Expandable title="examples" defaultOpen="True">
    <ResponseField name="languages" type="array of string">
      Lenguajes de ejemplo para los fragmentos de API autogenerados.

      Los lenguajes se muestran en el orden especificado.
    </ResponseField>

    <ResponseField name="defaults" type="&#x22;required&#x22; | &#x22;all&#x22;">
      Si se deben mostrar parámetros opcionales en los ejemplos de API. Por defecto es `all`.
    </ResponseField>

    <ResponseField name="prefill" type="boolean">
      Si se debe precargar el playground de API con datos de ejemplos del esquema. Cuando está habilitado, el playground automáticamente rellena los campos de solicitud con valores de ejemplo de tu especificación OpenAPI. Por defecto es `false`.
    </ResponseField>
  </Expandable>
</ResponseField>

### Configuración de ejemplo

Este ejemplo configura el playground de API para que sea interactivo con fragmentos de código de ejemplo para cURL, Python y JavaScript. Solo se muestran los parámetros requeridos en los fragmentos de código, y el playground precarga el cuerpo de la solicitud con valores de ejemplo.

```json
{
 "api": {
   "playground": {
     "display": "interactive"
   },
   "examples": {
     "languages": ["curl", "python", "javascript"],
     "defaults": "required",
     "prefill": true
   }
 }
}
```

### Páginas de endpoints personalizadas

Cuando necesitas más control sobre tu documentación de API, usa la extensión `x-mint` en tu especificación OpenAPI o crea páginas `MDX` individuales para tus endpoints.

Ambas opciones te permiten:

* Personalizar metadatos de página
* Añadir contenido adicional como ejemplos
* Controlar el comportamiento del playground por página

La extensión `x-mint` es recomendada para que toda tu documentación de API se genere automáticamente desde tu especificación OpenAPI y se mantenga en un solo archivo.

Las páginas `MDX` individuales son recomendadas para APIs pequeñas o cuando quieres experimentar con cambios página por página.
