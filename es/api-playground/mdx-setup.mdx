---
- title: Crear páginas de API manuales
- description: Documenta endpoints de API manualmente con archivos MDX.
---

Puedes definir manualmente los endpoints de API en páginas individuales. Este enfoque es útil para APIs pequeñas o para prototipos.

## Configuración

<Steps>
  <Step title="Configura los ajustes de tu API">
    En tu `docs.json` archivo, define tu URL base y método de autenticación.

    ```json
    "api": {
      "mdx": {
        "server": "https://mintlify.com/api",
        "auth": {
          "method": "key",
          "name": "x-api-key"
        }
      }
    }
    ```

    Si quieres ocultar el playground de la API, establece el campo `display` a `none`. No necesitas incluir un método de autenticación si ocultas el playground.

    ```json
    "api": {
      "playground": {
        "display": "none"
      }
    }
    ```

    Encuentra una lista completa de configuraciones de API en [Settings](/es/organize/settings#api-configurations).
  </Step>

  <Step title="Crea tus páginas de endpoints">
    Crea un archivo `MDX` para cada endpoint. Define el `title` y `api` en el frontmatter:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    ---
    ```

    Especifica los parámetros de ruta envolviéndolos en `{}`:

    ```bash
    https://api.example.com/v1/endpoint/{userId}
    ```

    <Tip>
      Si tienes un campo `server` configurado en `docs.json`, puedes usar rutas relativas como `/v1/endpoint`.
    </Tip>

    Para anular el modo de visualización global del playground para una página específica, añade `playground` al frontmatter:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    playground: 'none'
    ---
    ```

    Options:

    * `playground: 'interactive'` - Muestra el playground interactivo (predeterminado)
    * `playground: 'simple'` - Muestra un endpoint copiable sin playground
    * `playground: 'none'` - Oculta el playground completamente
  </Step>

  <Step title="Añade parámetros y respuestas">
    Usa [campos de parámetros y respuestas](/es/components/fields) para documentar los parámetros y valores de retorno de tu endpoint.

    ```mdx
    <ParamField path="userId" type="string" required>
      Unique identifier for the user
    </ParamField>

    <ParamField body="email" type="string" required>
      User's email address
    </ParamField>

    <ResponseField name="id" type="string" required>
      Unique identifier for the newly created user
    </ResponseField>

    <ResponseField name="email" type="string" required>
      User's email address
    </ResponseField>
    ```
  </Step>

  <Step title="Añade tus endpoints a tu documentación">
    Añade tus páginas de endpoints a la navegación actualizando el campo `pages` en tu `docs.json`. Aprende más sobre cómo estructurar tu documentación en [Navigation](/es/organize/navigation).
  </Step>
</Steps>

## Habilitar autenticación

Puedes establecer la autenticación globalmente en `docs.json` o anularla en páginas individuales usando el campo `authMethod` en el frontmatter. Un método específico de página anula la configuración global.

### Token Bearer

<CodeGroup>
  ```json docs.json
  "api": {
    "mdx": {
      "auth": {
        "method": "bearer"
      }
    }
  }
  ```

  ```mdx Page Metadata
  ---
  title: "Your page title"
  authMethod: "bearer"
  ---
  ```
</CodeGroup>

### Autenticación básica

<CodeGroup>
  ```json docs.json
  "api": {
    "mdx": {
      "auth": {
        "method": "basic"
      }
    }
  }
  ```

  ```mdx Page Metadata
  ---
  title: "Your page title"
  authMethod: "basic"
  ---
  ```
</CodeGroup>

### Clave API

<CodeGroup>
  ```json docs.json
  "api": {
    "mdx": {
      "auth": {
        "method": "key",
        "name": "x-api-key"
      }
    }
  }
  ```

  ```mdx Page Metadata
  ---
  title: "Your page title"
  authMethod: "key"
  ---
  ```
</CodeGroup>

### Ninguna

Para deshabilitar la autenticación en una página específica, establece `authMethod` a `none`:

```mdx Page Metadata
---
title: "Your page title"
authMethod: "none"
---
```
