<div id="enabling-authentication">
  ## Habilitar la autenticación
</div>

Puedes añadir un método de autenticación en tu `docs.json` para habilitarla globalmente en todas las páginas o configurarla de forma individual por página.

El método de autenticación definido en una página reemplazará al global si ambos están establecidos.

<div id="bearer-token">
  ### Token Bearer
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "bearer"
      }
    }
}
```

```mdx Page Metadata
---
title: "Your page title"
authMethod: "bearer"
---
```

</CodeGroup>

```json docs.json
      }
    }
}
```

```mdx Page Metadata
---
title: "Your page title"
```

</CodeGroup>

<div id="api-key">
  ### API key
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "key",
        "name": "x-api-key"
      }
    }
}
```

```mdx Page Metadata
---
title: "Your page title"
authMethod: "key"
---
```

</CodeGroup>

<div id="none">
  ### Ninguno
</div>

El método de autenticación "none" es útil para deshabilitar la autenticación en un endpoint específico después de establecer un valor predeterminado en docs.json.

<CodeGroup>
```mdx Page Metadata
---
title: "Your page title"
authMethod: "none"
---
```
</CodeGroup>
