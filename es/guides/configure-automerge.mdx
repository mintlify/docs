Agrega la aplicación GitHub de Mintlify a la lista de bypass de un ruleset para que la aplicación pueda crear y fusionar pull requests sin ser bloqueada por los requisitos de revisión y verificaciones de estado. Esto te permite configurar [flujos de trabajo](/es/guides/automate-agent) para fusionar automáticamente los cambios a `main`.

<div id="prerequisites">
  ## Requisitos previos
</div>

- Acceso de administrador al repositorio
- [Aplicación GitHub de Mintlify](/es/deploy/github) instalada en el repositorio
- Un ruleset existente que apunte a la rama `main`

<div id="steps">
  ## Pasos
</div>

<Steps>
  <Step title="Navega a los rulesets">
    En GitHub, ve a **Settings** -> **Rules** -> **Rulesets** en tu repositorio.
  </Step>
  <Step title="Selecciona el ruleset">
    Haz clic en el ruleset que apunta a tu rama `main`.

    <Note>
      Si varios rulesets apuntan a `main`, la aplicación debe estar en la lista de bypass de cada ruleset activo.
    </Note>
  </Step>
  <Step title="Agrega la aplicación GitHub de Mintlify a la lista de bypass">
    1. En **Bypass list**, haz clic en **Add bypass**.
    2. Busca y selecciona la aplicación GitHub de Mintlify.
    3. Configura el modo de bypass como **Always**.
  </Step>
  <Step title="Guardar">
    Haz clic en **Save changes**.
  </Step>
</Steps>

<div id="verify-the-setup">
  ## Verifica la configuración
</div>

1. **Para colaboradores**: Abre un PR que apunte a `main`. Confirma que aún requiere aprobación y verificaciones de estado exitosas antes de que se habilite el botón de fusión.
2. **Para flujos de trabajo**: Activa un flujo de trabajo con automerge habilitado. Confirma que el flujo de trabajo se ejecuta y fusiona exitosamente un PR.
