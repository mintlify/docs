Mintlify aloja automáticamente una tarjeta de agente [Agent-to-Agent (A2A)](https://a2aproject.github.io/A2A/latest/) en `/.well-known/agent-card.json`. La tarjeta de agente es un documento JSON estandarizado que ayuda a los agentes de IA a descubrir lo que ofrece tu sitio de documentación y dónde encontrar información detallada sobre sus capacidades.

Piensa en la tarjeta de agente como un mapa para los agentes: un único endpoint de descubrimiento que apunta a los metadatos de tu sitio, la URL de la documentación y las skills disponibles.

Consulta tu tarjeta de agente añadiendo `/.well-known/agent-card.json` a la URL de tu sitio de documentación. Mintlify genera la tarjeta de agente automáticamente y la mantiene actualizada. No es necesaria ninguna configuración.

<div id="when-to-use-the-agent-card">
  ## Cuándo usar la tarjeta de agente
</div>

La tarjeta de agente es más útil cuando:

- Quieres que los agentes compatibles con A2A descubran tu documentación de forma programática.
- Quieres exponer tus archivos `skill.md` a los agentes mediante un endpoint de descubrimiento estándar.
- Estás integrando tu documentación con frameworks de agentes que siguen la especificación A2A.

Si tus usuarios se conectan a tu documentación mediante clientes MCP, consulta [Model Context Protocol (MCP)](/es/ai/model-context-protocol). La tarjeta de agente complementa a MCP al proporcionar una capa de descubrimiento ligera que no requiere establecer una sesión.

<div id="agent-card-structure">
  ## Estructura de la tarjeta de agente
</div>

`GET /.well-known/agent-card.json` devuelve un documento JSON que sigue el esquema 0.2.0 de la tarjeta de agente A2A:

```json
{
  "schemaVersion": "0.2.0",
  "name": "Tu sitio de documentación",
  "description": "Una breve descripción de tu sitio de documentación.",
  "url": "https://your-docs-domain.com/",
  "version": "1.0.0",
  "documentationUrl": "https://your-docs-domain.com/",
  "capabilities": {
    "streaming": false
  },
  "skills": [
    {
      "id": "payments",
      "name": "Payments",
      "description": "Acepta pagos y administra suscripciones.",
      "tags": [],
      "url": "https://your-docs-domain.com/.well-known/agent-skills/payments/skill.md"
    }
  ]
}
```

| Campo              | Descripción                                                                                                  |
| ------------------ | ------------------------------------------------------------------------------------------------------------ |
| `schemaVersion`    | La versión del esquema de la tarjeta de agente A2A.                                                          |
| `name`             | El nombre de tu sitio de documentación, obtenido de tu archivo `llms.txt`.                                   |
| `description`      | Una breve descripción de tu sitio de documentación, obtenida de tu archivo `llms.txt`.                       |
| `url`              | La URL raíz de tu sitio de documentación.                                                                    |
| `version`          | La versión de la tarjeta de agente.                                                                          |
| `documentationUrl` | La URL donde los agentes pueden encontrar documentación legible por personas.                                |
| `capabilities`     | Capacidades del agente admitidas. Actualmente informa `streaming: false`.                                    |
| `skills`           | Una matriz de skills expuestas por tu documentación. Cada entrada enlaza con un archivo `skill.md` alojado.  |

<div id="skill-entries">
  ### Entradas de skills
</div>

Cada entrada de la matriz `skills` describe una sola skill que los agentes pueden obtener:

| Campo         | Descripción                                                                                           |
| ------------- | ----------------------------------------------------------------------------------------------------- |
| `id`          | Un slug seguro para URL derivado del campo `name` del frontmatter de tu `skill.md`.                   |
| `name`        | El nombre visible de la skill.                                                                        |
| `description` | Una breve descripción del frontmatter de tu `skill.md`, truncada a 1024 caracteres.                   |
| `tags`        | Una lista de etiquetas asociadas a la skill. Actualmente siempre está vacía.                          |
| `url`         | La URL para obtener el archivo `skill.md` completo.                                                   |

Las skills provienen de la misma fuente que los [endpoints de descubrimiento de skills](/es/ai/skillmd#skills-discovery-endpoints). Para personalizar las skills que aparecen en tu tarjeta de agente, agrega archivos de skill personalizados a tu proyecto. Consulta [Archivos de skill personalizados](/es/ai/skillmd#custom-skill-files) para más información.

<div id="use-the-agent-card-with-agents">
  ## Usar la tarjeta de agente con agentes
</div>

Los agentes compatibles con A2A obtienen `/.well-known/agent-card.json` para descubrir tu documentación. A partir de la respuesta, los agentes pueden:

- Identificar tu sitio de documentación por su nombre y descripción.
- Seguir `documentationUrl` para recuperar contenido legible por personas.
- Iterar sobre la matriz `skills` y obtener cada archivo `skill.md` para conocer qué pueden lograr con tu producto.

```bash
curl https://your-docs-domain.com/.well-known/agent-card.json
```

<Note>
  Si utilizas un [proxy inverso](/es/deploy/reverse-proxy), configúralo para reenviar la ruta `/.well-known/agent-card.json` a tu subdominio de Mintlify.
</Note>
