<Info>
  El agente está disponible en los [planes Pro y Custom](https://mintlify.com/pricing?ref=agent) para cualquier persona con acceso a tu dashboard.
</Info>

El agente crea solicitudes de extracción con cambios propuestos en tu documentación basados en tus indicaciones. Cuando envías una solicitud al agente, este consulta tu documentación, los repositorios conectados y los mensajes de Slack para crear contenido que siga las mejores prácticas de redacción técnica y se ajuste al esquema de Mintlify. Accede al agente directamente desde tu dashboard, en tu espacio de trabajo de Slack o intégralo en aplicaciones personalizadas con la API.

Usa el agente para:

* Redactar contenido nuevo basado en tus indicaciones, enlaces a solicitudes de extracción o hilos de Slack
* Revisar ejemplos de código desactualizados y referencias de API
* Buscar y actualizar contenido existente
* Responder preguntas sobre tu documentación y temas de redacción técnica

<div id="use-the-agent-in-the-dashboard">
  ## Usa el agente en el dashboard
</div>

Accede al agente desde tu dashboard usando el atajo de teclado <kbd>⌘</kbd>+<kbd>I</kbd> (macOS) o <kbd>Ctrl</kbd>+<kbd>I</kbd> (Windows/Linux), o haciendo clic en el botón **Ask agent**. El panel del agente se puede redimensionar en equipos de escritorio. En dispositivos móviles, el agente se abre a pantalla completa.

El panel del agente tiene tres vistas.

* **Chat**: Envía instrucciones al agente para actualizar tu documentación. El agente crea solicitudes de extracción basadas en tus instrucciones y muestra enlaces para ver las solicitudes de extracción u abrir los cambios en el editor web.
* **History**: Revisa conversaciones anteriores y continúa trabajando en consultas previas. Haz clic en cualquier conversación para cargarla en la vista de chat.
* **Settings**: Configura las integraciones del agente y el acceso al repositorio.

<Tip>
  Inicia una conversación nueva con el agente para cada tarea. Esto mantiene centrado el contexto del agente y te ayuda a asociar cada conversación con proyectos específicos.
</Tip>

<div id="add-the-agent-to-your-slack-workspace">
  ## Añade el agente a tu espacio de trabajo de Slack
</div>

Usa el agente en Slack para colaborar con tu equipo en las actualizaciones de la documentación.

<Note>
  Si el propietario del espacio de trabajo de Slack requiere la aprobación de un administrador para instalar aplicaciones, pídele que apruebe la aplicación Mintlify antes de conectarla.
</Note>

1. Abre el panel del agente en tu dashboard.
2. Haz clic en el botón **Settings**.
   <Frame>
     <img src="/images/agent/dashboard-settings-light.png" alt="El botón de configuración en modo claro." className="block dark:hidden" />

     <img src="/images/agent/dashboard-settings-dark.png" alt="El botón de configuración en modo oscuro." className="hidden dark:block" />
   </Frame>
3. En la sección de integración con Slack, haz clic en **Connect**.
4. Sigue las indicaciones que aparecen en Slack para añadir la aplicación `mintlify` a tu espacio de trabajo.
5. Sigue las indicaciones de Slack para vincular tu cuenta de Mintlify con tu espacio de trabajo de Slack.
6. Comprueba que el agente funciona y responde cuando:
   * Le envías un mensaje directo.
   * Lo mencionas con `@mintlify` en un canal.

<div id="connect-your-github-account">
  ## Conecta tu cuenta de GitHub
</div>

De manera predeterminada, el agente abre solicitudes de extracción atribuidas al bot de Mintlify. Para que las solicitudes de extracción aparezcan atribuidas a ti, conecta tu cuenta de GitHub en la página [Mi perfil](https://dashboard.mintlify.com/settings/account) del dashboard.

<div id="connect-repositories-as-context">
  ## Conecta repositorios como contexto
</div>

El agente solo puede acceder a los repositorios que conectes a través de la Aplicación de GitHub de Mintlify. Configura a qué repositorios puede acceder el agente en el panel del agente en **Settings** o en la [configuración de la Aplicación de GitHub](https://github.com/apps/mintlify/installations/new).

<div id="customize-agent-behavior">
  ## Personalizar el comportamiento del agente
</div>

Crea un archivo `AGENTS.md` en tu repositorio para personalizar el comportamiento del agente (`Agents.md` también es aceptado). El agente lee este archivo y sigue todas las instrucciones que proporciones.

El agente busca archivos `AGENTS.md` en dos ubicaciones. Primero revisa el directorio de documentación y luego la raíz del repositorio. Si tienes archivos `AGENTS.md` en ambas ubicaciones, el agente utiliza el archivo de tu directorio de documentación.

Agrega cualquier instrucción que quieras que el agente siga. El agente añade estas instrucciones a su mensaje de sistema (system prompt), por lo que las instrucciones se aplican a todas las tareas, ya sea que uses el agente en tu dashboard, en Slack o a través de la API.

```markdown Example AGENTS.md file
# Documentation agent instructions

## Code examples
- Use TypeScript for all code examples. Our users are primarily TypeScript developers.
- Always include error handling in API call examples.
- Show both success and error response examples for all endpoints.
- Include import statements at the top of code examples.

## API documentation standards
- Every endpoint must document: authentication requirements, rate limits, and common error codes.
- Use real-world parameter values in examples (not foo/bar placeholders).
- Include a complete request/response cycle for each endpoint.

## Style and formatting
- Write for developers with 2-5 years of experience. Don't oversimplify, but explain non-obvious concepts.
- Use active voice and second person ("you").
- Date format: ISO 8601 (YYYY-MM-DD).
- When referencing UI elements, use bold: **Settings** button.

## What to include
- Add prerequisite sections to guides when users need API keys, environment setup, or dependencies.
- Include "Next steps" sections linking to related documentation.
- Add troubleshooting sections for common issues we see in support tickets.
```

<div id="embed-the-agent-via-api">
  ## Integra el agente mediante la API
</div>

Usa los endpoints del agente para [crear tareas](/es/api-reference/agent/create-agent-job), [obtener una tarea específica](/es/api-reference/agent/get-agent-job) y [obtener todas las tareas](/es/api-reference/agent/get-all-jobs).

Al crear tareas mediante la API, puedes controlar si se crean solicitudes de extracción como borradores mediante el parámetro `asDraft` (predeterminado en `true`). Establece `asDraft: false` para crear solicitudes de extracción que no sean borrador, listas para su revisión inmediata y su fusión en flujos de trabajo automatizados.

<div id="write-effective-prompts">
  ## Escribe prompts efectivos
</div>

Piensa en el agente como un assistant útil que necesita tu guía para completar tareas. Dale instrucciones claras y contexto. Las tareas más específicas son más fáciles de completar, así que divide los proyectos complejos en pasos más pequeños.

Haz que tus prompts sean concretos y orientados a resultados. Prompts genéricos como `@mintlify Improve the onboarding page` aplican buenas prácticas generales, pero puede que no mejoren el contenido exactamente como lo imaginabas.

Prueba prompts basados en los resultados que quieres que tus usuarios alcancen o en los problemas que enfrentan. Por ejemplo:

* `@mintlify A lot of users have trouble installing the CLI. Review the onboarding page and update the docs so that users can easily install the CLI`
* `@mintlify Developers keep getting 401 errors when following our authentication guide. Review the auth docs and add clearer examples showing how to properly format the API key`

Usa prompts amplios para el mantenimiento general del contenido, como corregir errores tipográficos, actualizar redirecciones o renombrar una función en toda tu documentación. Por ejemplo:

* `@mintlify Find and fix all typos in the docs`
* `@mintlify change all unordered lists to use * instead of -`

<div id="specify-a-domain-name">
  ## Especificar un nombre de dominio
</div>

Si tienes varios sitios de documentación, incluye el parámetro `subdomain` en tu mensaje para especificar en qué conjunto de documentación debe trabajar el agente.

Para encontrar tu nombre de dominio, revisa la URL del dashboard para el conjunto de documentación que quieres actualizar. El nombre de dominio es la última parte después del nombre de tu organización. Por ejemplo, si la URL de tu dashboard es `https://dashboard.mintlify.com/org-name/domain-name`, tu nombre de dominio es `domain-name`.

Usa el formato `@mintlify subdomain=<your-domain-name> <your-prompt>` para indicarle al agente que trabaje en un conjunto de documentación específico.

Ejemplos:

* `@mintlify subdomain=public-docs Añade una nueva sección al quickstart sobre cómo invitar colaboradores en función de este PR`: Indica al agente que actualice el quickstart solo en el sitio `public-docs`.
* `@mintlify subdomain=customer-docs Actualiza la documentación de autenticación para el nuevo método de autenticación`: Indica al agente que actualice la documentación de autenticación solo en el sitio `customer-docs`.

<div id="agent-workflows">
  ## Flujos de trabajo del agente
</div>

El agente ayuda con muchas tareas de documentación. Estos flujos de trabajo muestran algunas de las maneras en que puedes integrar al agente en tu proceso de documentación. Prueba un enfoque que se adapte a la forma en que trabaja tu equipo actualmente y ajústalo a tus necesidades específicas.

<div id="iterate-on-a-prompt-in-a-slack-thread">
  ### Iterar un prompt en un hilo de Slack
</div>

Envía un prompt al agente y sigue mencionándolo con `@mintlify` en el mismo hilo para refinar e iterar la solicitud de extracción que cree.

Por ejemplo: `@mintlify Nuestra página de inicio rápido necesita una nueva sección sobre cómo invitar colaboradores`. Luego: `@mintlify La nueva sección debería llamarse "Invitar colaboradores"`. Y continúa con cualquier otra iteración.

<div id="start-with-the-agent-finish-manually">
  ### Empieza con el agente y termina manualmente
</div>

Pídele al agente que inicie un proyecto, luego cambia a la branch que cree y completa la tarea en tu entorno local o en el editor web. El agente puede ayudarte a comenzar; después, puedes retomar para finalizar la tarea.

Por ejemplo: `@mintlify Update the quickstart page to include information about inviting collaborators`, y luego cambia a la branch para realizar cualquier cambio adicional con tu método preferido.

<div id="update-docs-when-merging-feature-changes">
  ### Actualiza la documentación al fusionar cambios de funcionalidades
</div>

Cuando fusiones una solicitud de extracción de una funcionalidad, comparte el enlace del PR con el agente para actualizar la documentación correspondiente.

Por ejemplo: `@mintlify Este PR agrega un nuevo método de autenticación. Actualiza la documentación para incluir el nuevo flujo de autenticación: [enlace al PR]`.

<div id="generate-release-notes-from-a-pull-request">
  ### Generar notas de versión a partir de una solicitud de extracción
</div>

Pídele al agente que use una solicitud de extracción específica para generar notas de versión o actualizaciones del registro de cambios basadas en el historial de confirmaciones.

Por ejemplo: `@mintlify Generate release notes for this PR: [PR link]`.

<div id="generate-code-examples">
  ### Generar ejemplos de código
</div>

Pídele al agente que genere ejemplos de código para funciones en toda tu documentación o en páginas específicas.

Por ejemplo: `@mintlify Genera un ejemplo de código para que el método de Autenticación sea más fácil de entender`.

<div id="review-existing-content">
  ### Revisar el contenido existente
</div>

Pídele al agente que revise el contenido existente para verificar la precisión técnica, el estilo, la gramática u otros aspectos.

Por ejemplo: `@mintlify Revisa la sección de limitación de tasa de la API. Cambiamos los límites el mes pasado`.

<div id="respond-to-user-feedback">
  ### Responder a los comentarios de los usuarios
</div>

Proporciona al agente comentarios de tus usuarios para realizar actualizaciones específicas en tu documentación.

Por ejemplo: `@mintlify Los usuarios se están confundiendo con el paso 3 de la guía de configuración. ¿Qué puede estar haciendo que no quede claro?`.

<div id="automate-with-the-api">
  ### Automatiza con la API
</div>

Integra el agente en tus herramientas de automatización existentes para actualizar la documentación automáticamente cuando se produzcan cambios en el código, activar revisiones de contenido o sincronizar las actualizaciones de la documentación en varios repositorios.

Aprende cómo en el tutorial [Actualizar automáticamente la documentación cuando se hace merge del código](/es/guides/automate-agent).
