<Info>
  El agente está disponible en [planes Pro y Custom](https://mintlify.com/pricing?ref=agent) para cualquier persona con acceso a tu dashboard.
</Info>

El agente crea pull requests con cambios propuestos a tu documentación basados en tus indicaciones. Cuando envías una solicitud al agente, este hace referencia a tu documentación, repositorios conectados y mensajes de Slack para crear contenido que sigue las mejores prácticas de redacción técnica y se adhiere al esquema de Mintlify. Accede al agente en tu espacio de trabajo de Slack o intégralo en aplicaciones personalizadas con la API.

Usa el agente para:

* Escribir nuevo contenido basado en tus indicaciones, enlaces a pull requests o hilos de Slack
* Revisar ejemplos de código desactualizados y referencias de API
* Buscar y actualizar contenido existente
* Responder preguntas sobre tu documentación y temas de redacción técnica

Para comenzar, añade el agente a tu espacio de trabajo de Slack y menciónalo con `@mintlify` en un canal.

## Añadir el agente a tu espacio de trabajo de Slack

<Note>
  Si el Propietario de tu Espacio de Trabajo de Slack requiere aprobación administrativa para instalar aplicaciones, pídele que apruebe la aplicación Mintlify antes de conectarla.
</Note>

1. Navega a la página [agent](https://dashboard.mintlify.com/products/agent) de tu dashboard.
2. Selecciona el botón **Connect**.
3. Sigue las indicaciones de Slack para añadir la aplicación `mintlify` a tu espacio de trabajo.
4. Sigue las indicaciones de Slack para vincular tu cuenta de Mintlify a tu espacio de trabajo de Slack.
5. Comprueba que el agente está funcionando y responde cuando:
   * Le envías un mensaje directo.
   * Lo mencionas con `@mintlify` en un canal.

## Conectar repositorios como contexto

El agente solo puede acceder a los repositorios que conectes a través de la aplicación GitHub de Mintlify. Modifica a qué repositorios puede acceder el agente en la configuración de [GitHub App settings](https://github.com/apps/mintlify/installations/new).

## Escribir indicaciones efectivas

Piensa en el agente como un asistente útil que necesita tu orientación para completar tareas. Dale instrucciones claras y contexto. Las tareas más enfocadas son más fáciles de completar, así que divide los proyectos complejos en pasos más pequeños.

Haz que tus indicaciones sean específicas y enfocadas en resultados. Indicaciones genéricas como `@mintlify Improve the onboarding page` aplican mejores prácticas generales, pero pueden no mejorar el contenido de la manera específica que estabas imaginando.

Prueba indicaciones basadas en los resultados que quieres que tus usuarios logren o problemas que encuentran. Por ejemplo:

* `@mintlify A lot of users have trouble installing the CLI. Review the onboarding page and update the docs so that users can easily install the CLI`
* `@mintlify Developers keep getting 401 errors when following our authentication guide. Review the auth docs and add clearer examples showing how to properly format the API key`

Usa indicaciones amplias para el mantenimiento general del contenido como corregir errores tipográficos, actualizar redirecciones o cambiar el nombre de una característica en toda tu documentación. Por ejemplo:

* `@mintlify Find and fix all typos in the docs`
* `@mintlify change all unordered lists to use * instead of -`

## Especificar un nombre de dominio

Si tienes múltiples sitios de documentación, incluye el parámetro `subdomain` en tu mensaje para especificar en qué conjunto de documentación debe trabajar el agente.

Para encontrar tu nombre de dominio, mira la URL de tu dashboard para el conjunto de documentación que quieres actualizar. El nombre de dominio es la última parte después del nombre de tu organización. Por ejemplo, si la URL de tu dashboard es `https://dashboard.mintlify.com/org-name/domain-name`, tu nombre de dominio es `domain-name`.

Usa el formato `@mintlify subdomain=<your-domain-name> <your-prompt>` para indicar al agente que trabaje en un conjunto de documentación específico.

Examples:

* `@mintlify subdomain=public-docs Add a new section to the quickstart about inviting collaborators based on this PR`: Indica al agente que actualice solo la guía de inicio rápido en el sitio `public-docs`.
* `@mintlify subdomain=customer-docs Update the auth docs for the new authentication method`: Indica al agente que actualice solo los documentos de autenticación en el sitio `customer-docs`.

## Flujos de trabajo del agente

El agente ayuda con muchas tareas diferentes de documentación. Estos flujos de trabajo muestran algunas de las formas en que puedes integrar el agente en tu proceso de documentación. Prueba un enfoque que se adapte a cómo trabaja actualmente tu equipo y adáptalo a tus necesidades específicas.

### Iterar sobre una indicación en un hilo de Slack

Indica al agente, luego continúa mencionándolo con `@mintlify` en el mismo hilo para refinar e iterar sobre el pull request que crea.

Por ejemplo: `@mintlify Our quickstart page needs a new section on inviting collaborators`. Luego `@mintlify The new section should be called "Inviting collaborators"`. Seguido de cualquier otra iteración.

### Comenzar con el agente, terminar manualmente

Indica al agente que comience un proyecto, luego haz checkout de la rama que crea y termina la tarea en tu entorno local o en el editor web. El agente puede ayudarte a comenzar, luego puedes tomar el control para completar la tarea.

Por ejemplo: `@mintlify Update the quickstart page to include information about inviting collaborators` y luego haz checkout de la rama para hacer cualquier cambio adicional usando tu método preferido.

### Actualizar documentos al fusionar cambios de características

Cuando fusionas un pull request de una característica, comparte el enlace del PR con el agente para actualizar la documentación relevante.

Por ejemplo: `@mintlify This PR adds a new authentication method. Update the docs to include the new auth flow: [PR link]`.

### Generar notas de lanzamiento a partir de un pull request

Indica al agente con un pull request específico para generar notas de lanzamiento o actualizaciones de registro de cambios basadas en el historial de commits.

Por ejemplo: `@mintlify Generate release notes for this PR: [PR link]`.

### Generar ejemplos de código

Indica al agente que genere ejemplos de código para características en toda tu documentación o en páginas específicas.

Por ejemplo: `@mintlify Generate a code example to make the authentication method easier to understand`.

### Revisar contenido existente

Indica al agente que revise el contenido existente para verificar la precisión técnica, el estilo, la gramática u otros problemas.

Por ejemplo: `@mintlify Review the API rate limiting section. We changed limits last month`.

### Responder a comentarios de usuarios

Indica al agente con comentarios de tus usuarios para hacer actualizaciones enfocadas a tu documentación.

Por ejemplo: `@mintlify Users are getting confused by step 3 in the setup guide. What might be making it unclear?`.

### Automatizar con la API

Integra el agente en tus herramientas de automatización existentes para actualizar automáticamente la documentación cuando ocurren cambios en el código, activar revisiones de contenido o sincronizar actualizaciones de documentación en múltiples repositorios.

Aprende cómo en el tutorial [Auto-update documentation when code is merged](/es/guides/automate-agent).
