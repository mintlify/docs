<div id="about-the-assistant">
  ## Acerca del assistant
</div>

El assistant responde preguntas sobre tu documentación mediante consultas en lenguaje natural. Los usuarios acceden al assistant en tu sitio de documentación, de modo que puedan encontrar respuestas rápidamente y tener éxito con tu producto incluso si no saben dónde buscar.

El assistant utiliza RAG agéntico (generación aumentada por recuperación) con llamadas a herramientas. Cuando los usuarios hacen preguntas, el assistant:

* **Busca y recupera** contenido relevante de tu documentación para ofrecer respuestas precisas.
* **Cita fuentes** y proporciona enlaces navegables para llevar a los usuarios directamente a las páginas de referencia.
* **Genera ejemplos de código que se pueden copiar** para ayudar a los usuarios a implementar soluciones a partir de tu documentación.

Puedes ver el uso del assistant desde tu dashboard para comprender el comportamiento de los usuarios y la eficacia de la documentación. Exporta y analiza los datos de consultas para ayudar a identificar:

* Preguntas frecuentes que podrían necesitar mejor cobertura.
* Lagunas de contenido donde los usuarios tienen dificultades para encontrar respuestas.
* Temas populares que podrían beneficiarse de contenido adicional.

<div id="how-indexing-works">
  ### Cómo funciona la indexación
</div>

El assistant indexa automáticamente tu documentación publicada para responder a las preguntas con precisión. Cuando publicas cambios, el assistant indexa de inmediato el contenido nuevo, actualizado o eliminado. El assistant no indexa branches de borrador ni implementaciones de vista previa.

De forma predeterminada, el assistant no indexa las páginas ocultas. Para incluir las páginas ocultas en el índice del assistant, establece `seo.indexing: "all"` en tu `docs.json`. Consulta [Páginas ocultas](/es/organize/hidden-pages#search-seo-and-ai-indexing) para obtener más información.

<div id="how-the-assistant-handles-unknown-questions">
  ### Cómo gestiona el assistant las preguntas que no puede responder
</div>

El assistant solo responde preguntas basándose en la información de tu documentación. Si no puede encontrar información relevante después de buscar, responde que no tiene suficiente información para contestar.

Puedes [configurar un correo de derivación](/es/ai/assistant#set-deflection-email) para que el assistant muestre tu correo de soporte a los usuarios cuyas preguntas no puede responder. Esto ofrece a los usuarios una vía a seguir, incluso si la documentación no aborda su pregunta específica.

  ## Configura el asistente
</div>

<div id="search-domains">
  ### Búsqueda en dominios
</div>

En la sección de gestión de respuestas, configura los dominios que el assistant puede usar para búsquedas web y encontrar contexto adicional al responder preguntas.

* Los dominios deben ser de acceso público.
* Los dominios que requieren JavaScript para cargarse no son compatibles.

<Frame>
  <img src="/images/assistant/search-domains-light.png" alt="El panel de búsqueda en dominios del assistant habilitado en el dashboard. El assistant está configurado para buscar en el dominio mintlify.com/pricing." className="block dark:hidden" />

  <img src="/images/assistant/search-domains-dark.png" alt="El panel de búsqueda en dominios del assistant habilitado en el dashboard. El assistant está configurado para buscar en el dominio mintlify.com/pricing." className="hidden dark:block" />
</Frame>

Para tener un control más preciso sobre lo que el assistant puede buscar, usa la sintaxis de filtrado.

* **Filtrado a nivel de dominio**
  * `example.com`: Busca solo en el dominio `example.com`
  * `docs.example.com`: Busca solo en el subdominio `docs.example.com`
  * `*.example.com`: Busca todos los subdominios de `example.com`
* **Filtrado a nivel de ruta**
  * `docs.example.com/api`: Busca todas las páginas bajo la subruta `/api`
* **Múltiples patrones**
  * Agrega varias entradas para cubrir secciones diferentes de los sitios

<div id="customize-assistant-behavior">
  ## Personalizar el comportamiento del assistant
</div>

Agrega un archivo `Assistant.md` (o `ASSISTANT.md`) a tu proyecto para darle al assistant instrucciones personalizadas que definan cómo responde. El assistant usa estas instrucciones como contexto de nivel de sistema para cada respuesta.

Crea el archivo en `.mintlify/Assistant.md` (o `.mintlify/ASSISTANT.md`) en la raíz de tu proyecto.

Las instrucciones en un archivo `Assistant.md` se agregan al mensaje de sistema del assistant. No reemplazan ni anulan las instrucciones predeterminadas del assistant.

Para quitar las instrucciones personalizadas, elimina el archivo `Assistant.md`.

Usa `Assistant.md` para:

* Ajustar la personalidad y el tono del assistant
* Proporcionar contexto específico del producto
* Definir rutas de escalación de soporte
* Especificar preferencias de terminología
* Restringir o delimitar las áreas de enfoque del assistant
* Proporcionar orientación específica según la versión

```markdown Example Assistant.md
Eres un asistente útil para la plataforma de desarrolladores de Acme Corp.

## Tono
- Sé conciso y directo. Ir directo al punto ayuda más a las personas que ser exhaustivo.
- Usa lenguaje técnico apropiado para desarrolladores de software. La mayoría de las personas que hacen preguntas están familiarizadas con los productos de Acme.

## Contexto del producto
- Acme SDK v3 es la versión estable actual. v2 está en desuso. Si preguntan sobre v2, recuérdales que está en desuso pero responde sus preguntas.
- Dirige las preguntas de facturación a support@acme.com.

## Terminología
- Usa "workspace" en lugar de "project" u "organization".
- Usa "access token" en lugar de "API key".
```

<div id="manage-billing">
  ## Administrar facturación
</div>

El assistant utiliza límites de mensajes por niveles. Un mensaje es cualquier interacción del usuario con el assistant que recibe una respuesta correcta. Si tienes mensajes sin usar, hasta la mitad de tu límite de mensajes se puede trasladar al siguiente ciclo de facturación. Por ejemplo, si tienes un límite de 1.000 mensajes y usas 300 mensajes, 500 mensajes se trasladan al siguiente ciclo de facturación, lo que te da un total de 1.500 mensajes para el siguiente ciclo de facturación.

De forma predeterminada, el assistant permite excedentes. Puedes desactivar los excedentes para evitar incurrir en costos adicionales por uso más allá de tu nivel. Si alcanzas tu límite de mensajes con los excedentes desactivados, el assistant no estará disponible hasta que se restablezca tu límite de mensajes. Si permites excedentes, cada mensaje más allá de tu límite genera un cargo por excedente, pero los excedentes ocasionales pueden ser más baratos que actualizar a un nivel superior según tu uso.

<div id="change-your-assistant-tier">
  ### Cambiar tu nivel de assistant
</div>

Los niveles del assistant determinan tu límite mensual de mensajes y el precio.

Consulta y cambia tu nivel actual en la [pestaña Billing](https://dashboard.mintlify.com/products/assistant/settings/billing) de la página del assistant en tu dashboard.

En la sección **Spending Controls**, elige el nivel que prefieras en el menú desplegable.

**Si actualizas tu nivel:**

* Tu nuevo cupo de mensajes estará disponible de inmediato.
* Pagarás la diferencia prorrateada del ciclo de facturación actual.

**Si reduces tu nivel:**

* Tu cupo de mensajes se actualiza de inmediato.
* Los cambios de precio entran en vigor al inicio de tu próximo ciclo de facturación.
* Los mensajes no utilizados de tu nivel actual **no** se transfieren.

<div id="allow-overages">
  ### Permitir excedentes
</div>

Si no quieres permitir excedentes, desactívalos en la sección **Billing Controls** de la [pestaña Billing](https://dashboard.mintlify.com/products/assistant/settings/billing) de la página del assistant en tu dashboard.

<div id="set-usage-alerts">
  ### Configurar alertas de uso
</div>

En la sección Billing Controls, configura alertas de uso para recibir un correo electrónico cuando alcances cierto porcentaje de tu cupo de mensajes.

<div id="make-content-ai-ingestible">
  ## Haz que el contenido sea apto para la IA
</div>

Estructura tu documentación para ayudar al assistant a ofrecer respuestas precisas y relevantes. Una organización clara y un contexto completo benefician tanto a los lectores humanos como a la comprensión de la IA.

<Card title="Estructura y organización">
  * Usa marcado semántico.
  * Escribe encabezados descriptivos para las secciones.
  * Crea una jerarquía de información lógica.
  * Usa un formato coherente en toda tu documentación.
  * Incluye metadata completa en el frontmatter de la página.
  * Divide los bloques largos de texto en párrafos más cortos.
</Card>

<Card title="Contexto">
  * Define términos específicos y siglas la primera vez que se mencionen.
  * Proporciona contenido conceptual suficiente sobre funciones y procedimientos.
  * Incluye ejemplos y casos de uso.
  * Incluye referencias cruzadas a temas relacionados.
  * Agrega [hidden pages](/es/organize/hidden-pages) con contexto adicional que los usuarios no necesitan, pero que el assistant puede usar como referencia.
</Card>

<div id="use-the-assistant">
  ## Usa el assistant
</div>

Los usuarios tienen varias formas de iniciar una conversación con el assistant. Cada método abre un panel de chat en el lado derecho de tu documentación. Los usuarios pueden hacer cualquier pregunta y el assistant busca una respuesta en tu documentación. Si el assistant no puede recuperar información relevante, el assistant indica que no puede responder a la pregunta.

Añade el assistant como bot a tu [espacio de trabajo de Slack](/es/ai/slack-bot) o [servidor de Discord](/es/ai/discord) para que tu comunidad pueda hacer preguntas sin salir de su plataforma preferida.

<div id="urls">
  ### URLs
</div>

Abre el assistant con un parámetro de consulta en la URL para crear enlaces profundos que guíen a los usuarios a información específica o para compartir conversaciones del assistant con preguntas predefinidas.

* **Abrir el assistant**: Añade `?assistant=open` para abrir el panel de chat del assistant cuando la página se cargue.
  * Ejemplo: [https://mintlify.com/docs?assistant=open](https://mintlify.com/docs?assistant=open)
* **Abrir con una búsqueda predefinida**: Añade `?assistant=YOUR_QUERY` para abrir el assistant y enviar automáticamente una pregunta.
  * Ejemplo: [https://mintlify.com/docs?assistant=explain webhooks](https://mintlify.com/docs?assistant=explain%20webhooks)

<div id="troubleshooting">
  ## Solución de problemas
</div>

<Accordion title="La barra de chat del assistant no es visible">
  Si la interfaz del assistant no es visible en determinados navegadores, puede que tengas que enviar un informe de falso positivo a [EasyList](https://easylist.to). Los navegadores que usan la lista EasyList Cookies, como Brave y Comet, a veces bloquean el assistant u otros elementos de la interfaz. La lista EasyList Cookies incluye una regla específica por dominio que oculta elementos fijos en ciertos dominios para bloquear banners de cookies. Esta regla afecta de forma involuntaria a componentes legítimos de la interfaz.

  Envía un informe de falso positivo a [EasyList](https://github.com/easylist/easylist) para solicitar que se elimine la regla. Esto resuelve el problema para todos los usuarios una vez que se actualiza la lista de filtros.
</Accordion>
