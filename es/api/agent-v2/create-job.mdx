The v2 agent API provides asynchronous job execution. Unlike the v1 streaming API, this endpoint returns immediately with job information while the agent executes in the background.

## When to use v2

Use this endpoint when you:

- Need fire-and-forget job execution
- Want to poll for status instead of handling streaming responses
- Plan to send follow-up messages to an ongoing session

## Workflow

1. Create a job with `POST /v2/agent/{projectId}/job`
2. Poll for status using `GET /v2/agent/{projectId}/job/{id}`
3. Optionally send follow-up messages with `POST /v2/agent/{projectId}/job/{id}/message`
4. Check `prLink` in the response once the job completes

## Rate limits

The agent API has the following limits:

- 100 uses per Mintlify project per hour
