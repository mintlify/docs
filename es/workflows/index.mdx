Los workflows ejecutan el agente automáticamente según una programación o en respuesta a cambios en un repositorio. Cada workflow define un prompt para el agente y un trigger que indica cuándo ejecutarlo. Los workflows admiten tanto repositorios de GitHub como de GitLab.

Cuando se ejecuta un workflow, el agente lee el contenido de tu proyecto y los repositorios conectados, y luego sigue el prompt para realizar las actualizaciones.

<Tip>
Usa workflows que se ejecutan según una programación para tareas recurrentes, como publicar changelogs o auditar metadatos de SEO.

Usa workflows que se activan por cambios de contenido o de código para tareas de mantenimiento reactivas, como traducciones o actualizaciones de contenido cuando cambia el código fuente.
</Tip>

<div id="trigger-types">
  ## Tipos de trigger
</div>

Los workflows admiten tres tipos de trigger.

- **Actualización de contenido**—Se ejecuta cuando se hace push de cambios de contenido a tu repositorio de proyecto de Mintlify, incluidos los merges de pull requests y los pushes directos.
- **Cambio de código**—Se ejecuta cuando se mergea una pull request en un repositorio de código fuente conectado.
- **Programación personalizada**—Se ejecuta según una programación recurrente. Los workflows se ponen en cola en un plazo de 10 minutos a partir de la hora programada.

Cada workflow tiene exactamente un trigger. No puedes combinar tipos de trigger en un solo workflow.

<div id="usage-limits">
  ## Límites de uso
</div>

Las ejecuciones de workflows cuentan para tu uso de créditos. Consulta tu uso de créditos, configura excedentes y gestiona tu paquete de créditos en la página [Uso](https://app.mintlify.com/settings/organization/usage) de tu panel.

Cada workflow puede ejecutarse hasta 500 veces al día. Las ejecuciones fallidas no cuentan para este límite.
