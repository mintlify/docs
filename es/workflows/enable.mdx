<div id="prerequisites">
  ## Requisitos previos
</div>

- Un proyecto de Mintlify conectado a un repositorio de GitHub o GitLab
- Para GitHub: la [aplicación de GitHub de Mintlify](/es/deploy/github) instalada en cada repositorio que planees usar en el workflow
- Para GitLab: una cuenta de GitLab conectada (consulta [Configuración de GitLab](#gitlab-setup) a continuación)

<div id="enable-workflows">
  ## Activar workflows
</div>

1. Abre la página [Workflows](https://app.mintlify.com/products/workflows?tab=workflows) en tu panel.
2. Haz clic en el interruptor junto a un workflow para activarlo.
    <Frame>
      <img src="/images/workflows/workflows-dashboard-light.png" alt="El panel de workflows." className="block dark:hidden" />
      <img src="/images/workflows/workflows-dashboard-dark.png" alt="El panel de workflows." className="hidden dark:block" />
    </Frame>

    Si el workflow puede ejecutarse con la configuración predeterminada, se activa de inmediato. De lo contrario, se abre la ventana de configuración para que puedas elegir las opciones necesarias.
    <Frame>
      <img src="/images/workflows/workflow-settings-light.png" alt="Las opciones de configuración del workflow de actualizaciones del código." className="block dark:hidden" />
      <img src="/images/workflows/workflow-settings-dark.png" alt="Las opciones de configuración del workflow de actualizaciones del código." className="hidden dark:block" />
    </Frame>
3. Si se abre la ventana, completa los campos requeridos y haz clic en **Turn on workflow**.

    Para cambiar la configuración de un workflow activo, haz clic en el botón <Icon icon="settings" /> de configuración junto a él. Usa el interruptor en el encabezado de la ventana para activar o desactivar el workflow sin salir de la vista de configuración.

<div id="configurations">
  ## Configuraciones
</div>

<div id="trigger-repositories">
  ### Repositorios activadores
</div>

Para los workflows que se ejecutan en eventos push, especifica los repositorios que se deben monitorear. Puedes especificar hasta 10 repositorios por workflow.

Para los repositorios de GitHub, debes instalar la aplicación de GitHub de Mintlify en cada repositorio. Añade repositorios en la página de configuración de la [aplicación de GitHub](https://app.mintlify.com/settings/organization/github-app).

<div id="update-mode">
  ### Modo de actualización
</div>

Cada workflow tiene un modo predeterminado para realizar las actualizaciones, ya sea actualizando directamente tu repositorio de contenido o abriendo una pull request.

Si quieres requerir revisiones antes de que un workflow actualice tu contenido, haz clic en **Require review**.

<Note>
  Para los repositorios de GitHub, las actualizaciones automáticas requieren que la aplicación de GitHub de Mintlify tenga permisos de omisión en cada conjunto de reglas que apunte a tu rama de despliegue, incluidos los conjuntos de reglas a nivel de organización y repositorio. Consulta [Configurar el merge automático](/es/guides/configure-automerge) para obtener instrucciones de configuración.

  Para los repositorios de GitLab, el merge automático usa la conexión OAuth de GitLab y requiere al menos el rol de Maintainer en cada proyecto.
</Note>

<div id="trigger">
  ### Activador
</div>

Cada workflow tiene un valor predeterminado para cuándo se ejecuta, ya sea en respuesta a cambios en tu repositorio de contenido, en respuesta a cambios en un repositorio activador o según un horario.

Si quieres cambiar el valor predeterminado, selecciona un activador diferente.

<div id="instructions">
  ### Instrucciones
</div>

Añade instrucciones opcionales que se añaden al prompt base del workflow. Usa estas instrucciones para ajustar el estilo, el tono u otros aspectos propios de tu proyecto.

<div id="target-languages">
  ### Idiomas de destino
</div>

Cuando actives el workflow **Translate content**, selecciona uno o más idiomas que quieras mantener sincronizados con tu contenido fuente.

- Mintlify lee los [idiomas](/es/organize/navigation#languages) definidos en tu `docs.json` para identificar tu idioma predeterminado y preselecciona cualquier idioma de destino que ya esté configurado.
- Debes seleccionar al menos un idioma de destino para guardar el workflow.
- No puedes seleccionar el idioma fuente como destino.

Puedes cambiar los idiomas de destino en cualquier momento abriendo la configuración del workflow y editando el campo **Translate to**.

<div id="run-a-translation-immediately">
  #### Ejecutar una traducción de inmediato
</div>

Después de guardar el workflow, haz clic en **Translate now** para ejecutarlo bajo demanda usando los idiomas de destino guardados actualmente. El workflow debe estar activado para activar una ejecución manual. Las ejecuciones manuales no afectan el activador regular del workflow.

<div id="improve-docs-from-assistant-chats">
  ## Mejorar la documentación a partir de conversaciones con el asistente
</div>

El workflow **Improve docs from assistant chats** revisa las tendencias recientes en las preguntas al [asistente](/es/assistant/index) y abre actualizaciones específicas de la documentación que abordan las lagunas con las que se encuentran tus usuarios.

Usa este workflow cuando quieras que el agente clasifique la actividad del asistente por ti y proponga cambios que respondan a las preguntas que los usuarios realmente hacen.

Este workflow se ejecuta según un horario. No requiere repositorios activadores ni configuración de contexto adicional. Actívalo desde la página [Workflows](https://app.mintlify.com/products/workflows?tab=workflows) y elige si las actualizaciones se mergean automáticamente o abren una pull request para su revisión.

<div id="improve-docs-from-feedback">
  ## Mejorar la documentación a partir de los comentarios
</div>

El workflow **Improve docs from feedback** revisa los [comentarios de página](/es/optimize/feedback) recientes y abre actualizaciones específicas de la documentación basadas en los problemas que reportan los usuarios.

Usa este workflow cuando recopiles valoraciones con pulgares o comentarios contextuales y quieras que el agente convierta esa señal en cambios concretos en la documentación.

Cuando se completa una ejecución, los detalles de la ejecución en tu panel incluyen las citas específicas de comentarios que utilizó el agente. Cada cita enlaza al comentario en tu [panel de analítica](https://app.mintlify.com/products/analytics/v2/feedback) para que puedas verificar el contexto que impulsó la actualización.

Este workflow se ejecuta según un horario y requiere que la recopilación de comentarios esté activada en la página [Add-ons](https://app.mintlify.com/products/addons). No requiere repositorios activadores ni configuración de contexto adicional.
