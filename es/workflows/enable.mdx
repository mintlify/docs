<div id="prerequisites">
  ## Requisitos previos
</div>

- Un proyecto Mintlify conectado a un repositorio GitHub o GitLab
- Para GitHub: la [aplicación GitHub de Mintlify](/es/deploy/github) instalada en cada repositorio que planee usar en el flujo de trabajo
- Para GitLab: una cuenta GitLab conectada (consulte [Configuración de GitLab](#gitlab-setup) a continuación)

<div id="enable-workflows">
  ## Activar flujos de trabajo
</div>

1. Abra la página [Workflows](https://dashboard.mintlify.com/products/workflows?tab=workflows) en su panel.
2. Haga clic en el interruptor junto a un flujo de trabajo para activarlo.
    <Frame>
      <img src="/images/workflows/workflows-dashboard-light.png" alt="El panel de flujos de trabajo." className="block dark:hidden" />
      <img src="/images/workflows/workflows-dashboard-dark.png" alt="El panel de flujos de trabajo." className="hidden dark:block" />
    </Frame>
3. Elija las configuraciones requeridas.
    <Frame>
      <img src="/images/workflows/workflow-settings-light.png" alt="Las opciones de configuración para el flujo de trabajo de actualizaciones de la base de código." className="block dark:hidden" />
      <img src="/images/workflows/workflow-settings-dark.png" alt="Las opciones de configuración para el flujo de trabajo de actualizaciones de la base de código." className="hidden dark:block" />
    </Frame>
4. Haga clic en **Activar flujo de trabajo**.

<div id="configurations">
  ## Configuraciones
</div>

<div id="trigger-repositories">
  ### Repositorios disparadores
</div>

Para los flujos de trabajo que se ejecutan en eventos de push, especifique los repositorios a vigilar. Puede especificar hasta 10 repositorios por flujo de trabajo.

Para los repositorios de GitHub, debe instalar la aplicación GitHub de Mintlify en cada repositorio. Añada repositorios en la página de configuración de la [aplicación GitHub](https://dashboard.mintlify.com/settings/organization/github-app).

<div id="update-mode">
  ### Modo de actualización
</div>

Cada flujo de trabajo tiene un modo predeterminado para realizar actualizaciones, ya sea actualizando directamente su repositorio de contenido o abriendo una pull request.

Si desea requerir revisiones antes de que un flujo de trabajo actualice su contenido, seleccione **Requerir revisión**.

<Note>
  Para los repositorios de GitHub, las actualizaciones automáticas requieren que la aplicación GitHub de Mintlify tenga permisos de bypass en cada conjunto de reglas que apunte a su rama de despliegue, incluidos los conjuntos de reglas a nivel de organización y de repositorio. Consulte [Configurar la fusión automática](/es/guides/configure-automerge) para obtener instrucciones de configuración.

  Para los repositorios de GitLab, la fusión automática usa la conexión OAuth de GitLab y requiere al menos el rol de Maintainer en cada proyecto.
</Note>

<div id="trigger">
  ### Disparador
</div>

Cada flujo de trabajo tiene un valor predeterminado para cuándo se ejecuta, ya sea en respuesta a cambios en su repositorio de contenido, en respuesta a cambios en un repositorio disparador, o en un horario.

Si desea cambiar el valor predeterminado, seleccione un disparador diferente.

  ### Instrucciones
</div>

Añada instrucciones opcionales que se añaden al prompt base del flujo de trabajo. Use estas instrucciones para ajustar el estilo, el tono u otros aspectos únicos de su proyecto.
