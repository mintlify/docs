<div id="prerequisites">
  ## Requisitos previos
</div>

- Un proyecto de Mintlify conectado a un repositorio de GitHub o GitLab
- Para GitHub: la [Mintlify GitHub App](/es/deploy/github) instalada en cada repositorio que planees usar en el workflow
- Para GitLab: una cuenta de GitLab conectada (consulta [Configuración de GitLab](#gitlab-setup) a continuación)

<Tip>
  También puedes crear, listar y eliminar workflows desde la terminal con [`mint workflow`](/es/cli/commands#mint-workflow). La CLI es útil para scripting y CI; el panel es la forma más sencilla de configurar y monitorear las ejecuciones de los workflows.
</Tip>

<div id="enable-a-workflow">
  ## Activar un workflow
</div>

1. Abre la página [Workflows](https://app.mintlify.com/products/workflows?tab=workflows) en tu panel.
2. Haz clic en el interruptor junto a un workflow para activarlo.
    <Frame>
      <img src="/images/workflows/workflows-dashboard-light.png" alt="El panel de workflows." className="block dark:hidden" />
      <img src="/images/workflows/workflows-dashboard-dark.png" alt="El panel de workflows." className="hidden dark:block" />
    </Frame>

    Si el workflow puede ejecutarse con la configuración predeterminada, se activa de inmediato. De lo contrario, se abre el panel de configuración para que completes cualquier configuración requerida.
    <Frame>
      <img src="/images/workflows/workflow-settings-light.png" alt="Las opciones de configuración de un workflow." className="block dark:hidden" />
      <img src="/images/workflows/workflow-settings-dark.png" alt="Las opciones de configuración de un workflow." className="hidden dark:block" />
    </Frame>
3. Si se abre el panel de configuración, completa los campos requeridos.
4. Haz clic en **Turn on workflow**.

Para cambiar la configuración de un workflow activo, haz clic en el botón <Icon icon="settings" /> de configuración junto a él. Usa el interruptor en el encabezado del modal para activar o desactivar el workflow sin salir de la vista de configuración.

<div id="configurations">
  ## Configuraciones
</div>

<div id="trigger">
  ### Trigger
</div>

Cada workflow tiene un trigger por defecto que controla cuándo se ejecuta. Para cambiar el trigger, selecciona una opción diferente en el panel de configuración.

- **Actualización de contenido**—Se ejecuta cuando haces push de contenido a tu repositorio de proyecto, incluidos los merges de pull requests y los pushes directos.
- **Cambio de código**—Se ejecuta cuando se mergea una pull request en un repositorio de código fuente conectado. Debes especificar al menos un repositorio de origen.
- **Programación personalizada**—Se ejecuta según una programación recurrente que definas. Los workflows se ponen en cola en un plazo de 10 minutos a partir de la hora programada.

<div id="update-mode">
  ### Modo de actualización
</div>

Cada workflow tiene un modo predeterminado para cómo realiza las actualizaciones. Ya sea mergeando los cambios directamente en tu repositorio de contenido o abriendo una pull request para su revisión.

Para requerir una revisión antes de que el workflow actualice tu contenido, selecciona **Require review** en el panel de configuración.

<Note>
  Para repositorios de GitHub, las actualizaciones automáticas requieren que la Mintlify GitHub App tenga permisos de bypass en cada ruleset que apunte a tu rama de despliegue, incluidos los rulesets a nivel de organización y a nivel de repositorio. Consulta [Configurar automerge](/es/guides/configure-automerge) para las instrucciones de configuración.

  Para repositorios de GitLab, automerge utiliza la conexión OAuth de GitLab y requiere al menos el rol de Maintainer en cada proyecto.
</Note>

<div id="context-repositories">
  ### Repositorios de contexto
</div>

Para workflows personalizados y algunos workflows predefinidos, puedes añadir repositorios de contexto: repositorios de código fuente adicionales que el agente lee cuando se ejecuta el workflow. Esto es útil cuando el prompt de tu workflow hace referencia a código, APIs u otro contenido que vive fuera del repositorio de tu proyecto.

Puedes añadir hasta 10 repositorios de contexto por workflow. Para cada repositorio de GitHub, la Mintlify GitHub App debe estar instalada. Añade repositorios en la página [Configuración de GitHub App](https://app.mintlify.com/settings/organization/github-app).

<div id="integrations">
  ### Integraciones
</div>

Para los workflows personalizados y el workflow **Update from code changes**, puedes habilitar [integraciones del agente](/es/agent/integrations) conectadas para que el agente pueda obtener contexto desde herramientas como Slack, Notion o Linear cuando se ejecute el workflow.

Para habilitar integraciones en un workflow, abre su configuración y selecciona las integraciones que quieres que el agente use en la sección **Available integrations**. Solo aparecerán en la lista las integraciones ya conectadas para tu organización. Consulta [Integraciones del agente](/es/agent/integrations) para más información sobre cómo conectar integraciones.

<div id="slack-notifications">
  ### Notificaciones de Slack
</div>

Envía mensajes de Slack a los canales seleccionados cuando se ejecuten los workflows. Usa las notificaciones de Slack para mantener a tu equipo al tanto de la actividad de los workflows en curso sin tener que vigilar el dashboard.

Para habilitar las notificaciones de Slack, instala la [aplicación de Slack de Mintlify](/es/agent/slack) en tu workspace y, a continuación, añade uno o más canales de notificación en la sección **Notification channels** de la configuración de tu workflow.

Una vez habilitadas, Mintlify publica en los canales configurados cuando:

- Un workflow abre una pull request para revisión.
- Una pull request abierta por un workflow lleva esperando revisión **3 días** sin haberse fusionado. Mintlify envía un recordatorio diario hasta que la pull request se fusione o se cierre. Cuando la pull request se resuelve, el recordatorio original se actualiza con un tachado para que las alertas obsoletas no permanezcan visibles.
- Un workflow fusiona una pull request o no logra completarse.

Los recordatorios de revisiones obsoletas se aplican a cualquier workflow configurado para abrir pull requests para revisión (consulta [Modo de actualización](#update-mode)). Solo se activan cuando hay canales de notificación de Slack configurados para el workflow.

<div id="instructions">
  ### Instrucciones
</div>

Añade instrucciones opcionales que se adjuntan al prompt base del workflow en cada ejecución. Úsalas para ajustar el estilo, el tono u otros comportamientos específicos del proyecto sin cambiar la lógica principal del workflow.

<div id="target-languages">
  ### Idiomas de destino
</div>

Cuando activas el workflow **traducir contenido**, selecciona uno o más idiomas para mantenerlos sincronizados con tu contenido fuente.

- Mintlify lee los [idiomas](/es/organize/navigation#languages) definidos en tu `docs.json` para identificar tu idioma predeterminado y preselecciona los idiomas de destino ya configurados.
- Debes seleccionar al menos un idioma de destino para guardar el workflow.
- No puedes seleccionar el idioma fuente como destino.

Puedes añadir idiomas de destino en cualquier momento abriendo la configuración del workflow y editando el campo **Translate to**.

<div id="disable-a-workflow">
  ## Desactivar un workflow
</div>

1. Ve a la página [Workflows](https://app.mintlify.com/products/workflows?tab=workflows) en tu panel.
2. Haz clic en el interruptor junto a un workflow para desactivarlo.

Cuando reactivas un workflow programado, Mintlify recalcula la próxima hora de ejecución a partir de la hora actual.

<div id="view-run-history">
  ## Ver el historial de ejecuciones
</div>

Cada workflow mantiene un registro de ejecuciones pasadas, incluido el estado y un resumen de los cambios realizados.

1. Ve a la página [Workflow Runs](https://app.mintlify.com/products/workflows) en tu panel.
2. Usa los menús desplegables para filtrar por workflows específicos o por estado.
