Mintlify incluye workflows predefinidos para las tareas de mantenimiento de documentación más comunes. Cada workflow tiene un prompt base y una configuración por defecto. Puedes personalizar el trigger, el modo de actualización y las instrucciones desde la página [Workflows](https://app.mintlify.com/products/workflows?tab=workflows) en tu panel.

<div id="self-updating-content-workflows">
  ## Workflows de contenido autoactualizable
</div>

Los workflows de contenido autoactualizable realizan actualizaciones basadas en el contexto que defines, de modo que tu contenido se mantenga preciso y al día.

<div id="update-from-code-changes">
  ### Actualizar desde cambios de código
</div>

Monitorea tus repositorios de código fuente y abre pull requests para actualizar el contenido cuando se produzcan cambios que afecten a los usuarios. Puedes ejecutar este workflow según una programación, lo que generalmente consume menos créditos, o en cada pull request de código fuente. Por defecto, este workflow se ejecuta dos veces por semana (martes y jueves). Los demás workflows programados siguen ejecutándose con una programación semanal por defecto.

Cuando ejecutas el workflow según una programación, el agente lee todas las pull requests de código fuente mergeadas desde la última vez que se ejecutó el workflow. Agrupa los cambios relacionados que requieren actualizaciones de contenido en una pull request en tu repositorio de proyecto de Mintlify. Si hay pull requests sin mergear de ejecuciones anteriores, el agente añade los cambios relevantes a las pull requests existentes. El agente cita la procedencia de los cambios en tus repositorios de código fuente.

Cuando ejecutas el workflow en cada pull request de código fuente, el agente crea una pull request de actualización de contenido cada vez que mergeas una pull request de código fuente que requiere actualizaciones de contenido. Usa la variante basada en push cuando quieras que las actualizaciones de contenido aparezcan tan pronto como se publique el código, en lugar de agruparlas según una programación.

Puedes habilitar [integraciones del agente](/es/agent/integrations) para este workflow para dar al agente contexto adicional desde herramientas como Slack o Linear al determinar qué actualizar.

<div id="draft-changelog">
  ### Borrador de changelog
</div>

Crea una entrada de changelog según una programación recurrente. Añade repositorios de código fuente como contexto para que el agente tenga más detalles sobre cómo cambió tu producto. Sin repositorios de contexto, el agente trabaja a partir de tus actualizaciones de contenido para escribir una entrada de changelog.

<div id="draft-improvements-from-assistant-conversations">
  ### Borrador de mejoras a partir de conversaciones del asistente
</div>

Revisa las tendencias de preguntas de tu [asistente](/es/assistant/index) para identificar lagunas en tu contenido, y luego abre una pull request para abordarlas. Cuantas más conversaciones del asistente tengan tus usuarios, más señal tendrá el agente para trabajar. Ejecuta este workflow según una programación recurrente para mantener el contenido alineado con lo que los usuarios quieren saber sobre tu producto.

<div id="draft-improvements-from-user-feedback">
  ### Borrador de mejoras a partir de comentarios de usuarios
</div>

Revisa los [comentarios](/es/optimize/feedback) de las páginas — valoraciones con pulgar arriba/abajo y comentarios escritos — y actualiza el contenido en las páginas donde los usuarios han reportado problemas. El agente se centra en las páginas con señales negativas recurrentes en lugar de en comentarios aislados, por lo que los resultados mejoran a medida que se acumulan los comentarios.

Debes habilitar la recopilación de comentarios en la página [Complementos](https://app.mintlify.com/products/addons) antes de poder usar este workflow.

<div id="maintenance-workflows">
  ## Workflows de mantenimiento
</div>

Los workflows de mantenimiento automatizan tareas rutinarias para mejorar la calidad y la consistencia de tu contenido. Puedes ejecutarlos con un trigger de actualización de contenido o según una programación recurrente.

Ejecutarlos con un trigger de actualización de contenido mantiene tu contenido con un retraso mínimo. Ejecutarlos según una programación suele ser más eficiente en créditos, ya que el agente agrupa el trabajo de varios cambios en una sola ejecución, pero hay un retraso entre el momento en que cambia el contenido y el momento en que el workflow de mantenimiento actúa sobre él.

Cualquier workflow de mantenimiento habilitado se ejecuta en las pull requests creadas por workflows de contenido autoactualizable. Si el workflow de mantenimiento requiere cambios, el agente combina el trabajo en una sola pull request. Por ejemplo, si habilitas el workflow **traducir contenido**, el agente añade traducciones a las pull requests abiertas por el workflow **actualizar desde cambios de código**.

<div id="translate-content">
  ### Traducir contenido
</div>

Traduce el contenido actualizado a los idiomas de destino que selecciones cada vez que se hagan push de cambios a tu repositorio de proyecto. Debes especificar al menos un idioma de destino. Mintlify lee los [idiomas](/es/organize/navigation#languages) definidos en tu `docs.json` y preselecciona los idiomas de destino ya configurados.

Para deshabilitar las traducciones de un idioma, elimina todos los archivos en el directorio de ese idioma. No puedes deshabilitar las traducciones de un idioma que todavía tiene contenido publicado.

<div id="fix-broken-links">
  ### Reparar enlaces rotos
</div>

Encuentra y repara enlaces internos y externos rotos cada vez que se hagan push de cambios de contenido a tu repositorio de proyecto. Es útil ejecutarlo después de reorganizaciones importantes de contenido o cuando los servicios externos a los que haces referencia cambian sus URLs.

<div id="audit-seo-metadata">
  ### Auditar metadatos de SEO
</div>

Audita y actualiza los títulos, las meta descripciones, los encabezados y las etiquetas canónicas cada vez que cambie el contenido. Añade instrucciones para alinear las sugerencias del agente con tu estrategia de SEO o con palabras clave objetivo.

<div id="fix-grammar-typos">
  ### Corregir gramática y erratas
</div>

Encuentra y corrige erratas, errores ortográficos y errores gramaticales cada vez que cambie el contenido. Añade instrucciones para especificar excepciones para términos técnicos, nombres de productos o decisiones estilísticas intencionales que el agente no deba cambiar.

<div id="apply-style-guide">
  ### Aplicar guía de estilo
</div>

Aplica reglas de voz, tono y escritura al contenido cada vez que cambia. Añade instrucciones para definir las reglas específicas que quieres que se apliquen o enlaza tu guía de estilo. Cuanto más precisas sean tus instrucciones, más consistentes serán los resultados.
