title: "Visibilidad"
description: "Muestra u oculta contenido según si el lector es un humano o un agente de IA."

---

Usa el componente `Visibility` para mostrar contenido diferente a los lectores humanos y a los agentes de IA. El contenido dentro de `<Visibility for="agents">` solo aparece en las [exportaciones Markdown](/es/ai/markdown-export) y en las respuestas de herramientas de IA, mientras que el contenido dentro de `<Visibility for="humans">` solo se renderiza para los visitantes humanos en el navegador.

<Visibility for="humans">
  Este contenido solo es visible para los humanos que navegan la documentación.
</Visibility>

<Visibility for="agents">
  Este contenido solo es visible para los agentes de IA que consumen la exportación Markdown.
</Visibility>

<div id="example">
  ## Ejemplo
</div>

````mdx Contenido para agentes y humanos
<Visibility for="humans">
  Haz clic en el botón **Comenzar** en la esquina superior derecha para crear tu cuenta.
</Visibility>

<Visibility for="agents">
  Para crear una cuenta, envía una solicitud POST a `/api/v1/accounts` con
  un cuerpo JSON que contenga los campos `email` y `password`.
</Visibility>
````

<div id="how-it-works">
  ## Cómo funciona
</div>

Mintlify detecta si el lector es un humano o un agente de IA:

- **En el navegador**, el componente verifica la cadena de user agent del visitante. Los visitantes humanos ven el contenido `for="humans"`, mientras que los agentes de IA reconocidos ven el contenido `for="agents"`.
- **En las exportaciones Markdown** (URLs `.md`, solicitudes `Accept: text/markdown`, [llms.txt](/es/ai/llmstxt)), solo se incluye el contenido `for="agents"`. Los bloques `for="humans"` se eliminan antes de servir el Markdown.

Esto significa que puedes proporcionar instrucciones adaptadas a las herramientas de IA — como datos estructurados, detalles de API o contexto legible por máquinas — sin saturar la página para los lectores humanos.

<div id="properties">
  ## Propiedades
</div>

<ResponseField name="for" type="string" required>
  El público objetivo del contenido. Usa `"agents"` para dirigirte a los agentes de IA o `"humans"` para dirigirte a los lectores humanos. También se aceptan las formas singulares `"agent"` y `"human"`.
</ResponseField>
