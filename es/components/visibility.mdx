---
title: "Visibility"
description: "Controla qué contenido se muestra en la interfaz web para los lectores humanos y qué contenido se muestra solo en la salida Markdown para agentes de IA."
keywords: ["visibility", "agents", "humans", "markdown", "AI", "conditional content"]
---

Usa el componente `Visibility` para mostrar contenido diferente a los humanos que visitan tu sitio y a los agentes de IA que procesan la salida Markdown. Esto te permite personalizar la información para cada audiencia sin mantener páginas separadas.

En tu sitio publicado, el contenido marcado con `for="humans"` se renderiza normalmente y el contenido marcado con `for="agents"` se oculta. En las [vistas de página en Markdown](/es/ai/markdown-export) (URLs `.md`), ocurre lo contrario: el contenido para agentes se renderiza en la salida Markdown y el contenido exclusivo para humanos no.

<div id="example">
  ## Ejemplo
</div>

```mdx
<Visibility for="humans">
  <Note>
    Estás viendo este contenido **en la web**.

    Este contenido se muestra normalmente en las páginas estándar de tu sitio.

    Úsalo para un paso como: Haz clic en el botón **Get started** en la esquina superior derecha para crear tu cuenta.

    En este ejemplo, las instrucciones orientadas a la interfaz aparecen en la página estándar. Consulta la [versión Markdown de esta página](https://mintlify.com/docs/components/visibility.md) para ver las instrucciones orientadas a la API para agentes.
  </Note>
</Visibility>

<Visibility for="agents">
  > Estás viendo este contenido **en Markdown**.
  >
  > Este contenido solo aparece en la versión Markdown de las páginas.
  >
  > Úsalo para información que solo usan los agentes, como: Para crear una cuenta, llama a `POST /v1/accounts` con una dirección de correo electrónico válida.
  >
  > En este ejemplo, las instrucciones orientadas a la API aparecen en la versión Markdown de la página. Consulta la [página estándar](https://mintlify.com/docs/components/visibility) para ver las instrucciones orientadas a la interfaz para humanos.
</Visibility>
```

<div id="properties">
  ## Propiedades
</div>

<ResponseField name="for" type="string" required>
  La audiencia del contenido envuelto. Debe ser uno de los siguientes:

  - `humans`: Contenido visible en la web y excluido de la salida Markdown
  - `agents`: Contenido oculto en la web e incluido en la salida Markdown
</ResponseField>
