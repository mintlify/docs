Usa el componente `Update` para mostrar entradas del cambios, actualizaciones de versión y notas de la versión con un formato uniforme.

<Update label="Ejemplo" description="v0.1.1">
  ## Ejemplo de actualización

  Puedes añadir cualquier cosa aquí, como una captura de pantalla, un fragmento de código o una lista de cambios.
  
  <Frame>
    <img
      className="block"
      src="https://mintlify-assets.b-cdn.net/hero-light.png"
      alt="Captura de pantalla de la página de introducción de PearsDB como ejemplo de una imagen en un componente Update."
    />
  </Frame>

  ### Novedades
  - Diseño adaptable
  - Ancla para cada actualización
  - Entrada de RSS generada para cada actualización
</Update>

<div id="how-to-use">
  ## Cómo usarlo
</div>

```mdx Update example
<Update label="2024-10-11" description="v0.1.0" tags={["Mintlify"]}>
  Esta es una actualización con una etiqueta, descripción y tag.
</Update>
```

Usa varios componentes `Update` para crear [cambios](/es/create/changelogs).

Los componentes `Update` generan automáticamente entradas en el canal RSS. Las entradas contienen únicamente Markdown puro: no se incluyen componentes, bloques de código, código en línea ni elementos HTML en bruto.

Si tu actualización incluye componentes o HTML, usa la prop `rss` para ofrecer un texto alternativo a los suscriptores de RSS.

```mdx Custom RSS content
<Update
  label="March 2025"
  description="v2.0.0"
  rss={{
    title: "Marzo 2025 - Rediseño importante",
    description: "Renovación completa de la interfaz con nuevos componentes"
  }}
>
  ## New dashboard

  <Frame>
    <img src="/dashboard.png" alt="New dashboard UI displaying updated components." />
  </Frame>

  Check out our redesigned dashboard with improved navigation.
</Update>
```


<div id="props">
  ## Props
</div>

<ResponseField name="label" type="string" required>
  label de la actualización. Aparece a la izquierda de la actualización y crea un enlace ancla. Las labels deben ser únicas.
</ResponseField>

<ResponseField name="tags" type="string[]">
  Etiquetas de la actualización. Se muestran como filtros en el panel lateral derecho.
</ResponseField>

<ResponseField name="description" type="string">
  Descripción de la actualización. Aparece debajo del label y las etiquetas.
</ResponseField>

<ResponseField name="rss" type="object">
  Los componentes de actualización generan automáticamente entradas del feed RSS. Usa la propiedad `rss` para crear una única entrada de feed RSS con un título y una descripción personalizados.
  
  Si no se especifica:

  - Las actualizaciones con encabezados de Markdown crean una entrada de feed RSS separada por cada encabezado.
  - Las actualizaciones sin encabezados de Markdown crean una única entrada de feed RSS usando el `label` como título.

  <Expandable title="propiedades">
    <ResponseField name="title" type="string">
      Título personalizado para la entrada del feed RSS.
    </ResponseField>

    <ResponseField name="description" type="string">
      Descripción personalizada para la entrada del feed RSS.
    </ResponseField>
  </Expandable>
</ResponseField>
