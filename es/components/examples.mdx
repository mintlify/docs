Los componentes `<RequestExample>` y `<ResponseExample>` muestran bloques de código en la barra lateral derecha para crear un diseño de dos columnas que mantiene los ejemplos visibles mientras los usuarios se desplazan por tu contenido. Estos componentes están pensados para la documentación de API, pero funcionan en todas las páginas.

Casos de uso comunes:

* Documentación de endpoints de API con ejemplos de solicitud y respuesta
* Ejemplos de configuración junto con texto explicativo
* Fragmentos de código que los usuarios consultan mientras siguen instrucciones
* Ejemplos de antes y después en tutoriales

En dispositivos móviles, los componentes `<RequestExample>` y `<ResponseExample>` se muestran como bloques de código normales y se pueden omitir al desplazarse.

<RequestExample>
  ```bash Request
    curl --request POST \
      --url https://dog-api.kinduff.com/api/facts
  ```
</RequestExample>

<ResponseExample>
  ```json Response
  { "status": "success" }
  ```
</ResponseExample>

<div id="requestexample">
  ## RequestExample
</div>

Usa `<RequestExample>` para anclar ejemplos de código en la barra lateral derecha. Este componente funciona de forma similar al componente [CodeGroup](/es/components/code-groups), pero muestra el código en la barra lateral en lugar de en línea.

Puedes incluir varios bloques de código dentro de un único `<RequestExample>`. Cada bloque de código debe tener un atributo title.

````mdx EjemploDeSolicitud
<RequestExample>

```bash Solicitud
  curl --request POST \
    --url https://dog-api.kinduff.com/api/facts
```

</RequestExample>
````

<div id="responseexample">
  ## ResponseExample
</div>

El componente `<ResponseExample>` fija ejemplos de código en la barra lateral derecha, debajo de cualquier contenido `<RequestExample>` en la misma página.

````mdx EjemploDeRespuesta
<ResponseExample>

```json Respuesta
{ "status": "success" }
```

</ResponseExample>
````
