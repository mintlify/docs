---
title: 'Campos de respuesta'
description: 'Mostrar valores de respuesta de la API'
---

El componente `<ResponseField>` está diseñado para definir los valores devueltos por una API. Muchos documentos también utilizan `<ResponseField>` en páginas cuando necesitas enumerar los tipos de algo.

<ResponseField name="response" type="string" required>
  Ejemplo de campo de respuesta
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>

```


<div id="props">
  ## Props
</div>

<ResponseField name="name" type="string" required>
  El nombre del valor de la respuesta.
</ResponseField>

<ResponseField name="type" type="string" required>
  Tipo esperado del valor de la respuesta; puede ser cualquier cadena arbitraria.
</ResponseField>

<ResponseField name="default" type="string">
  El valor predeterminado.
</ResponseField>

<ResponseField name="required" type="boolean">
  Mostrar “required” junto al nombre del campo.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Indica si un campo está obsoleto o no.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Etiquetas que se muestran antes del nombre del campo
</ResponseField>

<ResponseField name="post" type="string[]">
  Etiquetas que se muestran después del nombre del campo
</ResponseField>
