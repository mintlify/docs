Utiliza campos para documentar parámetros y respuestas de la API. Hay dos tipos de campos: de parámetro y de respuesta.

<div id="parameter-field">
  ## Campo de parámetro
</div>

El componente `<ParamField>` se utiliza para definir parámetros para tus API o SDK. Al agregar un `ParamField` se añade automáticamente un [Área de pruebas de API](/es/api-playground/overview).

<ParamField path="param" type="string" required>
  Un ejemplo de un campo de parámetro
</ParamField>

```mdx
<ParamField path="param" type="string" required>
  Un ejemplo de un campo de parámetro
</ParamField>
```

<div id="properties">
  ### Propiedades
</div>

<ParamField body="query, path, body, or header" type="string">
  Indica si el parámetro es de tipo query, path, body o header. Luego se especifica el
  nombre del parámetro.
</ParamField>

<ParamField body="type" type="string">
  Tipo esperado del valor del parámetro.

  Admite `number`, `string`, `boolean`, `object`.

  Los arreglos se pueden definir usando el sufijo `[]`. Por ejemplo, `string[]`.
</ParamField>

<ParamField body="required" type="boolean">
  Indica si el parámetro es obligatorio.
</ParamField>

<ParamField body="deprecated" type="boolean">
  Indica si el parámetro está en desuso.
</ParamField>

<ParamField body="default" type="any">
  Valor predeterminado que se establece cuando el valor de la solicitud está vacío
</ParamField>

<ParamField body="placeholder" type="string">
  Texto del placeholder para la entrada en el área de pruebas.
</ParamField>

<ParamField body="children" type="string">
  Descripción del parámetro (con soporte para Markdown).
</ParamField>

<div id="response-field">
  ## Campo de respuesta
</div>

El componente `<ResponseField>` define los valores de retorno de una API.

<ResponseField name="response" type="string" required>
  Un ejemplo de campo de respuesta
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Un ejemplo de campo de respuesta
</ResponseField>
```

<div id="properties">
  ### Propiedades
</div>

<ResponseField name="name" type="string" required>
  El nombre del valor de respuesta.
</ResponseField>

<ResponseField name="type" type="string" required>
  Tipo esperado del valor de respuesta. Puede ser cualquier cadena arbitraria.
</ResponseField>

<ResponseField name="default" type="string">
  El valor predeterminado.
</ResponseField>

<ResponseField name="required" type="boolean">
  Indica si el campo es obligatorio.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Indica si el campo está obsoleto.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Etiquetas que se muestran antes del nombre del campo.
</ResponseField>

<ResponseField name="post" type="string[]">
  Etiquetas que se muestran después del nombre del campo.
</ResponseField>
