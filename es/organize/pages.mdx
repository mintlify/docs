Cada página es un archivo MDX, que combina contenido Markdown con componentes React para permitirte crear documentación rica e interactiva.

## Metadatos de página

Cada página comienza con frontmatter, los metadatos YAML encerrados por `---` al principio de un archivo. Estos metadatos definen cómo aparece y se comporta tu página.

Usa frontmatter para controlar:

* Títulos y descripciones de página
* Títulos de la barra lateral, iconos y etiquetas
* Diseños de página
* Etiquetas meta SEO
* Metadatos personalizados

<ResponseField name="title" type="string" required>
  El título de tu página que aparece en la navegación y en las pestañas del navegador.
</ResponseField>

<ResponseField name="description" type="string">
  Una breve descripción de lo que cubre esta página. Aparece debajo del título y mejora el SEO.
</ResponseField>

<ResponseField name="sidebarTitle" type="string">
  Un título corto que se muestra en la navegación de la barra lateral.
</ResponseField>

<ResponseField name="icon" type="string">
  El icono a mostrar.

  Options:

  * [Icono de Font Awesome](https://fontawesome.com/icons) nombre
  * [Icono de Lucide](https://lucide.dev/icons) nombre
  * URL a un icono alojado externamente
  * Ruta a un archivo de icono en tu proyecto
</ResponseField>

<ResponseField name="iconType" type="string">
  El [Font Awesome](https://fontawesome.com/icons) estilo de icono. Solo se usa con iconos de Font Awesome.

  Options: `regular`, `solid`, `light`, `thin`, `sharp-solid`, `duotone`, `brands`.
</ResponseField>

<ResponseField name="tag" type="string">
  Una etiqueta que aparece junto al título de tu página en la barra lateral.
</ResponseField>

<ResponseField name="<custom>" type="string">
  Cualquier frontmatter YAML válido. Por ejemplo, `product: "API"` o `version: "1.0.0"`.
</ResponseField>

```yaml Example YAML frontmatter wrap
---
title: "About frontmatter"
description: "Frontmatter is the metadata that controls how your page appears and behaves"
sidebarTitle: "Frontmatter"
icon: "book"
tag: "NEW"
---
```

## Modo de página

Controla cómo se muestra tu página con la configuración `mode`.

### Predeterminado

Si no se define ningún modo, se utiliza por defecto un diseño estándar con una barra lateral de navegación y tabla de contenidos.

```yaml
---
title: "Default page title"
---
```

### Ancho

El modo ancho oculta la tabla de contenidos. Esto es útil para páginas que no tienen encabezados o si prefieres usar el espacio horizontal adicional. El modo ancho está disponible para todos los temas.

```yaml
---
title: "Wide page title"
mode: "wide"
---
```

### Personalizado

El modo personalizado proporciona un diseño minimalista que elimina todos los elementos excepto la barra de navegación superior. El modo personalizado es un lienzo en blanco para crear páginas de destino o cualquier otro diseño único que desees tener con elementos de navegación mínimos. El modo personalizado está disponible para todos los temas.

```yaml
---
title: "Custom page title"
mode: "frame"
---
```

### Marco

El modo marco proporciona un diseño similar al modo personalizado, pero conserva la navegación de la barra lateral. Este modo de página permite HTML personalizado y componentes mientras mantiene la experiencia de navegación predeterminada. El modo marco solo está disponible para los temas Aspen y Almond.

```yaml
---
title: "Frame page title"
mode: "frame"
---
```

### Centro

El modo centro elimina la barra lateral y la tabla de contenidos, centrando el contenido. Esto es útil para registros de cambios u otras páginas donde quieres enfatizar el contenido. El modo centro está disponible para los temas Mint y Linden.

```yaml
---
title: "Center page title"
mode: "center"
---
```

## Páginas de API

Crea áreas de prueba interactivas de API añadiendo una especificación de API a tu frontmatter, `api` o `openapi`.

```yaml
---
openapi: "GET /endpoint"
---
```

Aprende más sobre cómo construir [documentación de API](/es/api-playground/overview).

## Enlaces externos

Enlaza a sitios externos directamente desde tu navegación con los metadatos `url`.

```yaml
---
title: "npm Package"
url: "https://www.npmjs.com/package/mint"
---
```

## Optimización para motores de búsqueda

La mayoría de las etiquetas meta SEO se generan automáticamente. Puedes establecer etiquetas meta SEO manualmente para mejorar el SEO de tu sitio, el compartir en redes sociales y la compatibilidad con navegadores.

<Note>
  Las etiquetas meta con dos puntos deben estar entre comillas.
</Note>

```yaml
---
"twitter:image": "/images/social-preview.jpg"
---
```

Consulta [SEO](/es/optimize/seo) para opciones completas de metadatos SEO.

## Palabras clave de búsqueda interna

Mejora la capacidad de descubrimiento de una página específica en la búsqueda integrada proporcionando `keywords` en tus metadatos. Estas palabras clave no aparecerán como parte del contenido de la página o en los resultados de búsqueda, pero a los usuarios que las busquen se les mostrará la página como resultado.

```yaml
---
keywords: ['configuration', 'setup', 'getting started']
---
```
