---

Las páginas ocultas no aparecen en la navegación de tu sitio, pero cualquiera que conozca la URL aún puede acceder a ellas. Por ejemplo, si creas una página oculta como `guides/hidden-page.mdx`, los visitantes pueden acceder a ella en `docs.yoursite.com/guides/hidden-page`.

Usa páginas ocultas para contenido al que quieras que los usuarios accedan o al que hagan referencia como contexto para herramientas de IA, pero que no quieras que aparezca listado en la navegación.

Si tu contenido requiere un control de acceso estricto, debes configurar la [autenticación](/es/deploy/authentication-setup).

Para restringir páginas a grupos de usuarios específicos, configura el [control de acceso basado en grupos](/es/deploy/authentication-setup#control-access-with-groups).

Consulta un [ejemplo de página oculta](/es/organize/hidden-page-example).

<Note>
  Algunos elementos de navegación, como las barras laterales, los menús desplegables y las pestañas, pueden aparecer vacíos o cambiar de diseño en las páginas ocultas.
</Note>

<div id="hide-a-page">
  ## Ocultar una página
</div>

Para ocultar una página, configura `hidden: true` en el [frontmatter](/es/organize/pages) de la página o elimínala de la navigation de tu archivo `docs.json`.

<div id="set-hidden-true-in-frontmatter">
  ### Establecer `hidden: true` en el frontmatter
</div>

Agrega `hidden: true` al frontmatter de una página para eliminarla de la navegación visible y seguir incluyéndola en tu configuración de `docs.json`.

```yaml
---
title: "My hidden page"
hidden: true
---
```

Para hacer visible una página de nuevo, elimina el campo `hidden` por completo. No establezcas `hidden: false` ya que produce un comportamiento indefinido.

<Note>
  De forma predeterminada, `hidden: true` excluye una página de la indexación en motores de búsqueda, los mapas del sitio y el contexto de IA.

  Para incluirla de todos modos, configura `seo.indexing: "all"` en `docs.json`. Consulta [Búsqueda, SEO e indexación de IA](https://www.mintlify.com/docs/#search-seo-and-ai-indexing) para más detalles.

  Nota: `noindex: true` solo afecta a la indexación; no oculta la página de la navegación. Consulta [Desactivar la indexación](https://www.mintlify.com/docs/optimize/seo#disable-indexing) para obtener más información.
</Note>

<div id="remove-the-page-from-navigation">
  ### Quitar la página de la navegación
</div>

Si no incluyes una página en la propiedad `navigation` de tu `docs.json`, esa página quedará oculta. Este método funciona bien para páginas que no quieres que aparezcan en la navegación en absoluto.

<div id="hide-a-group-of-pages">
  ## Ocultar un grupo de páginas
</div>

Para ocultar un grupo de páginas, establece la propiedad `hidden` en `true` para el grupo en tu archivo `docs.json`:

```json highlight={4}
"groups": [
  {
    "group": "Primeros pasos",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  },
  {
    "group": "Guides",
    "pages": [
      "guides/hidden-page.mdx",
      "guides/hidden-groups.mdx"
    ]
  }
]
```

En este ejemplo, el grupo `Getting started` está oculto y el grupo `Guides` está visible.

<div id="hide-a-tab">
  ### Ocultar una Tab
</div>

Para ocultar una Tab, añade la propiedad `hidden` para la Tab en tu archivo `docs.json`:

```json highlight={4}
"tabs": [
  {
    "tab": "Inicio",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  }
]
```

<div id="keep-hidden-pages-discoverable-with-searchable">
  ## Mantener páginas ocultas detectables con `searchable`
</div>

De forma predeterminada, ocultar una tab, un grupo, un anchor o un dropdown también elimina sus páginas descendientes de la búsqueda, el sitemap, `llms.txt`, el assistant de IA y la indexación en motores de búsqueda. Para ocultar un elemento de navegación de la barra lateral sin perder la detectabilidad de sus páginas, agrega `searchable: true` junto con `hidden: true`.

Usa `searchable: true` cuando quieras que una sección sea accesible desde la búsqueda, el SEO o las herramientas de IA, pero sin que aparezca en la navegación renderizada. Casos comunes incluyen productos heredados que aún deben posicionarse en los motores de búsqueda, material de referencia enlazado solo desde otras páginas y contenido que quieras que el assistant de IA use como contexto.

```json highlight={4-5}
"tabs": [
  {
    "tab": "Reference",
    "hidden": true,
    "searchable": true,
    "groups": [
      {
        "group": "Archived APIs",
        "pages": [
          "reference/legacy-api",
          "reference/deprecated-endpoints"
        ]
      }
    ]
  }
]
```

Con esta configuración:

- Las páginas descendientes aparecen en la búsqueda del sitio y en el sitemap.
- Las páginas se incluyen en `llms.txt` y en el contexto del assistant de IA.
- El HTML renderizado no incluye `<meta name="robots" content="noindex">`, por lo que los motores de búsqueda pueden indexar las páginas.
- La tab, el grupo, el anchor o el dropdown sigue sin aparecer en la navegación lateral.

Puedes establecer `searchable: true` en cualquier elemento de navegación que admita `hidden`, incluidos tabs, grupos, anchors y dropdowns. Las páginas habilitadas mediante `searchable` siguen siendo accesibles por URL directa.

<Note>
  `searchable: true` solo tiene efecto cuando también se establece `hidden: true`. En un elemento visible no hace nada.
</Note>

<div id="search-seo-and-ai-indexing">
  ## Búsqueda, SEO e indexación de IA
</div>

De forma predeterminada, las páginas ocultas se excluyen de la indexación para los motores de búsqueda, la búsqueda interna de tu documentación y como contexto para el assistant de IA. Para incluir páginas ocultas en los resultados de búsqueda y como contexto para el assistant, agrega la propiedad `seo` a tu `docs.json`:

```json
"seo": {
    "indexing": "all"
}
```

<div id="understanding-hidden-versus-noindex">
  ### Comprender hidden versus noindex
</div>

La relación entre `hidden` y `noindex` es unidireccional:

* **`hidden: true` → aplica automáticamente `noindex`**: Las páginas ocultas se excluyen automáticamente de los motores de búsqueda, los sitemaps y el contexto de IA.
* **`noindex: true` → NO aplica `hidden`**: Las páginas con `noindex: true` siguen siendo visibles en la navegación. Solo afectan la indexación en motores de búsqueda y el contexto de IA.

Para excluir una página específica de los motores de búsqueda manteniéndola visible en la navegación, agrega `noindex: true` a su frontmatter. Para ocultar una página de la navegación y de los motores de búsqueda, usa `hidden: true`.
