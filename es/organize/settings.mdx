El archivo `docs.json` te permite convertir una colección de archivos Markdown en un sitio de documentación navegable y personalizado. Este archivo de configuración obligatorio controla el estilo, la navegación, las integraciones y más. Piénsalo como el plano de tu documentación.

La configuración en `docs.json` se aplica globalmente a todas las páginas.

<div id="setting-up-your-docsjson">
  ## Configurar tu `docs.json`
</div>

Para empezar, solo necesitas especificar `theme`, `name`, `colors.primary` y `navigation`. Los demás campos son opcionales y puedes añadirlos a medida que crezcan las necesidades de tu documentación.

Para obtener la mejor experiencia de edición, incluye la referencia del esquema en la parte superior de tu archivo `docs.json`. Esto habilita el autocompletado, la validación y sugerencias útiles en la mayoría de los editores de código:

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Tu Documentación",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // Tu estructura de navegación
  }
  // El resto de tu configuración
}
```

<ResponseField name="colors" type="object" required>
  Los colores utilizados en tu documentación. Los colores se aplican de forma diferente según el tema. Si solo proporcionas un color principal, se aplica a todos los elementos de color.

  <Expandable title="Colors">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>

      Debe ser un código hexadecimal que comience con `#`.
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      El color usado para énfasis en el modo oscuro.

      Debe ser un código hexadecimal que comience con `#`.

<ResponseField name="styling" type="object">
  Configuración de estilo visual.

  <Expandable title="Estilo">
    <ResponseField name="eyebrows" type="&#x22;section&#x22; | &#x22;breadcrumbs&#x22;">
      El estilo del eyebrow (encabezado superior) de la página. Elige `section` para mostrar el nombre de la sección o `breadcrumbs` para mostrar la ruta de navegación completa. El valor predeterminado es `section`.
    </ResponseField>

    <ResponseField name="latex" type="boolean">
      Controla si se incluyen las hojas de estilo de LaTeX, anulando la detección automática. De forma predeterminada, Mintlify detecta automáticamente el uso de LaTeX en tu contenido y carga las hojas de estilo necesarias.

      * Establécelo en `true` para forzar la carga de las hojas de estilo de LaTeX cuando la detección automática no reconozca tus expresiones matemáticas.
      * Establécelo en `false` para evitar cargar las hojas de estilo de LaTeX y mejorar el rendimiento si no usas expresiones matemáticas pero tienes contenido que activa una detección de falso positivo.
    </ResponseField>

    <ResponseField name="codeblocks" type="&#x22;system&#x22; | &#x22;dark&#x22; | string | object">
      Configuración del tema de los bloques de código. El valor predeterminado es `"system"`.

      **Configuración simple:**

      * `"system"`: Se ajusta al modo actual del sitio (claro u oscuro)
      * `"dark"`: Usa siempre el modo oscuro

      **Configuración de tema personalizado:**

      * Usa una cadena para especificar un único [tema de Shiki](https://shiki.style/themes) para todos los bloques de código
      * Usa un objeto para especificar [temas de Shiki](https://shiki.style/themes) separados para los modos claro y oscuro

      <ResponseField name="theme" type="string">
        Un único nombre de tema de Shiki que se usará tanto para los modos claro como oscuro.

        ```json
        "styling": {
          "codeblocks": {
            "theme": "dracula"
          }
        }
        ```
      </ResponseField>

      <ResponseField name="theme" type="object">
        Temas separados para los modos claro y oscuro.

        <Expandable title="theme">
          <ResponseField name="light" type="string" required>
            Un nombre de tema de Shiki para el modo claro.
          </ResponseField>

          <ResponseField name="dark" type="string" required>
            Un nombre de tema de Shiki para el modo oscuro.
          </ResponseField>

          ```json
          "styling": {
            "codeblocks": {
              "theme": {
                "light": "github-light",
                "dark": "github-dark"
              }
            }
          }
          ```
        </Expandable>
      </ResponseField>

      <ResponseField name="languages" type="object">
        Configuración personalizada de lenguajes para los bloques de código.

        <Expandable title="languages">
          <ResponseField name="custom" type="array of string">
            Rutas a archivos JSON que describen lenguajes personalizados de Shiki. Usa esto para agregar resaltado de sintaxis para lenguajes que no estén incluidos en el conjunto predeterminado de Shiki.

            El archivo JSON debe seguir el [formato de gramática de TextMate](https://macromates.com/manual/en/language_grammars) utilizado por Shiki.

            ```json
            "styling": {
              "codeblocks": {
                "languages": {
                  "custom": ["/languages/my-custom-language.json"]
                }
              }
            }
            ```
          </ResponseField>
        </Expandable>
      </ResponseField>
    </ResponseField>

      <Expandable title="404">
        <ResponseField name="redirect" type="boolean">

        </ResponseField>

        </ResponseField>

        <ResponseField name="description" type="string">

        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="upgrading-from-mintjson">
  ## Actualizar desde `mint.json`
</div>

Si tu proyecto de documentación usa el archivo en desuso `mint.json`, sigue estos pasos para actualizar a `docs.json`.

<Steps>
  <Step title="Instala o actualiza la CLI">
    Si aún no has instalado la [CLI](/es/installation), instálala ahora:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    Si ya tienes instalada la CLI, asegúrate de que esté actualizada:

    ```bash
    mint update
    ```
  </Step>

  <Step title="Crea tu archivo docs.json">
    En tu repositorio de documentación, ejecuta:

    ```bash
    mint upgrade
    ```

    Este comando crea un archivo `docs.json` a partir de tu `mint.json` existente. Revisa el archivo generado para asegurarte de que toda la configuración sea correcta.
  </Step>

  <Step title="Elimina tu archivo mint.json">
    Después de verificar que tu `docs.json` esté configurado correctamente, puedes eliminar de forma segura tu antiguo archivo `mint.json`.
  </Step>
</Steps>
