## Encabezados

Los encabezados organizan tu contenido y crean anclajes de navegación. Aparecen en la tabla de contenidos y ayudan a los usuarios a recorrer tu documentación.

### Crear encabezados

Usa símbolos `#` para crear encabezados de distintos niveles:

```mdx
## Encabezado de sección principal
### Encabezado de subsección
#### Encabezado de subsubsección
```

<Tip>
  Usa encabezados descriptivos y con palabras clave que indiquen claramente el contenido que sigue. Esto mejora tanto la navegación del usuario como el posicionamiento en buscadores.
</Tip>

## Formato de texto

Compatibilizamos la mayoría del formato Markdown para enfatizar y dar estilo al texto.

### Formato básico

Aplica estos estilos de formato a tu texto:

| Estilo | Sintaxis | Ejemplo | Resultado |
|-------|--------|---------|--------|
| **Negrita** | `**text**` | `**important note**` | **important note** |
| _Cursiva_ | `_text_` | `_emphasis_` | _emphasis_ |
| ~~Tachado~~ | `~text~` | `~deprecated feature~` | ~~deprecated feature~~ |

### Combinación de formatos

Puedes combinar estilos de formato:

```mdx
**_bold and italic_**
**~~bold and strikethrough~~**
*~~italic and strikethrough~~**
```

**_bold and italic_**<br />
**~~bold and strikethrough~~**<br />
_~~italic and strikethrough~~_

### Superíndice y subíndice

Para expresiones matemáticas o notas al pie, usa etiquetas HTML:

| Tipo | Sintaxis | Ejemplo | Resultado |
|------|--------|---------|--------|
| Superíndice | `<sup>text</sup>` | `example<sup>2</sup>` | example<sup>2</sup> |
| Subíndice | `<sub>text</sub>` | `example<sub>n</sub>` | example<sub>n</sub> |

## Enlaces

Los enlaces ayudan a los usuarios a navegar entre páginas y acceder a recursos externos. Usa texto de enlace descriptivo para mejorar la accesibilidad y la experiencia del usuario.

### Enlaces internos

Enlaza a otras páginas de tu documentación usando rutas relativas a la raíz:

```mdx
[Quickstart](/quickstart)
[Steps](/components/steps)
```

[Quickstart](/es/quickstart)<br />
[Steps](/es/components/steps)

<Note>
  Evita los enlaces relativos como `[page](../page)`, ya que cargan más lentamente y no pueden optimizarse tan eficazmente como los enlaces relativos a la raíz.
</Note>

### Enlaces externos

Para recursos externos, incluye la URL completa:

```mdx
[Markdown Guide](https://www.markdownguide.org/)
```

[Markdown Guide](https://www.markdownguide.org/)

### Enlaces rotos

Puedes comprobar si hay enlaces rotos en tu documentación usando la [Mint CLI](/es/installation):

```bash
mint broken-links
```

## Citas en bloque

Las citas en bloque resaltan información importante, citas o ejemplos dentro de tu contenido.

### Citas en bloque de una sola línea

Agrega `>` antes del texto para crear una cita en bloque:

```mdx
> Esta es una cita que destaca del contenido principal.
```

> Esta es una cita que destaca del contenido principal.

### Citas en bloque de varias líneas

Para citas más extensas o con varios párrafos:

```mdx
> Este es el primer párrafo de una cita en bloque de varias líneas.
>
> Este es el segundo párrafo, separado por una línea en blanco con `>`.
```

> Este es el primer párrafo de una cita en bloque de varias líneas.
>
> Este es el segundo párrafo, separado por una línea en blanco con `>`.

<Tip>
  Usa las citas en bloque con moderación para mantener su impacto visual y significado. Considera usar [callouts](/es/components/callouts) para notas, advertencias y otros tipos de información.
</Tip>

## Expresiones matemáticas

Compatibilidad con LaTeX para representar expresiones y ecuaciones matemáticas.

### Matemáticas en línea

Usa signos de dólar simples, `$`, para expresiones matemáticas en línea:

```mdx
The Pythagorean theorem states that $(a^2 + b^2 = c^2)$ in a right triangle.
```

El teorema de Pitágoras establece que $(a^2 + b^2 = c^2)$ en un triángulo rectángulo.

### Ecuaciones en bloque

Usa signos de dólar dobles, `$$`, para ecuaciones independientes:

```mdx
$$
E = mc^2
$$
```

$$
E = mc^2
$$

<Info>
  La compatibilidad con LaTeX requiere una sintaxis matemática adecuada. Consulta la [documentación de LaTeX](https://www.latex-project.org/help/documentation/) para obtener pautas de sintaxis completas.
</Info>

## Saltos de línea y espaciado

Controla el espaciado y los saltos de línea para mejorar la legibilidad del contenido.

### Saltos de párrafo

Separa los párrafos con líneas en blanco:

```mdx
This is the first paragraph.

This is the second paragraph, separated by a blank line.
```

Este es el primer párrafo.

Este es el segundo párrafo, separado por una línea en blanco.

### Saltos de línea manuales

Usa etiquetas HTML `<br />` para forzar saltos de línea dentro de los párrafos:

```mdx
This line ends here.<br />
This line starts on a new line.
```

Esta línea termina aquí.<br />
Esta línea comienza en una línea nueva.

<Tip>
  En la mayoría de los casos, los saltos de párrafo con líneas en blanco aportan mejor legibilidad que los saltos de línea manuales.
</Tip>

## Mejores prácticas

### Organización del contenido

- Usa encabezados para crear una jerarquía de contenido clara
- Sigue una jerarquía de encabezados correcta (no pases de H2 a H4)
- Escribe encabezados descriptivos con palabras clave relevantes

### Formato del texto

- Usa negrita para enfatizar, no para párrafos completos
- Reserva la cursiva para términos, títulos o énfasis sutil
- Evita el exceso de formato que distraiga del contenido

### Enlaces

- Escribe texto de enlace descriptivo en lugar de “haz clic aquí” o “leer más”
- Usa rutas relativas a la raíz para enlaces internos
- Revisa los enlaces con regularidad para evitar referencias rotas
