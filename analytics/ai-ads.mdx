## Event tracking

AI ads send detailed events to your analytics providers:

### Ad performance events

| Event name | Description | Properties |
| :--------- | :---------- | :--------- |
| `docs.ai_ads.impression` | Ad displayed to user | `ad_id`, `placement`, `page_url`, `relevance_score` |
| `docs.ai_ads.click` | User clicked on ad | `ad_id`, `placement`, `click_position`, `revenue` |
| `docs.ai_ads.dismiss` | User dismissed ad | `ad_id`, `placement`, `dismiss_reason` |
| `docs.ai_ads.load_error` | Ad failed to load | `error_type`, `placement`, `page_url` |

### User behavior events

| Event name | Description | Properties |
| :--------- | :---------- | :--------- |
| `docs.ai_ads.page_view_with_ads` | Page view with ads present | `ad_count`, `placements`, `page_category` |
| `docs.ai_ads.user_feedback` | User provided ad feedback | `feedback_type`, `rating`, `ad_id` |
| `docs.ai_ads.scroll_past_ad` | User scrolled past ad | `ad_id`, `time_visible`, `scroll_depth` |

## Custom analytics queries

### Google Analytics 4 queries

Access AI ads data in GA4 with custom dimensions:

```sql GA4 BigQuery example
SELECT
  event_name,
  event_timestamp,
  user_pseudo_id,
  (SELECT value.string_value FROM UNNEST(event_params) WHERE key = 'ad_id') as ad_id,
  (SELECT value.string_value FROM UNNEST(event_params) WHERE key = 'placement') as placement,
  (SELECT value.double_value FROM UNNEST(event_params) WHERE key = 'revenue') as revenue
FROM `your-project.analytics_XXXXXXXXX.events_*`
WHERE event_name LIKE 'docs.ai_ads.%'
  AND _TABLE_SUFFIX BETWEEN '20240101' AND '20240131'
```

### Mixpanel queries

Query AI ads events in Mixpanel:

```javascript Mixpanel JQL example
function main() {
  return Events({
    from_date: '2024-01-01',
    to_date: '2024-01-31',
    event_selectors: [{
      event: 'docs.ai_ads.click'
    }]
  })
  .groupBy(['properties.placement'], mixpanel.reducer.count())
  .groupByUser(['properties.ad_id'], mixpanel.reducer.sum('properties.revenue'));
}
```

- **Developer documentation**: Sidebar ads with technical tools, CTR 2.3%
- **User guides**: In-content ads with productivity tools, CTR 1.8%
- **API references**: Footer ads with developer services, CTR 1.5%

**Underperforming patterns:**

- **Too many ads**: More than 3 ads per page, CTR drops 40%
- **Irrelevant content**: Generic ads on technical pages, CTR below 0.5%
- **Poor placement**: Header ads on mobile, high bounce rate increase

## Troubleshooting analytics

### Common issues

<Accordion title="Analytics data not appearing">
  **Possible causes:**
  - Analytics integration not properly configured
  - Ad blocker preventing tracking
  - Incorrect event tracking setup
  
  **Solutions:**
  - Verify analytics configuration in docs.json
  - Check browser console for tracking errors
  - Test with ad blockers disabled
</Accordion>

<Accordion title="Revenue numbers don't match">
  **Possible causes:**
  - Different time zones in reporting
  - Delayed revenue attribution
  - Currency conversion issues
  
  **Solutions:**
  - Align time zones across all platforms
  - Allow 24-48 hours for revenue reconciliation
  - Check currency settings in analytics
</Accordion>

<Accordion title="Low data quality">
  **Possible causes:**
  - Insufficient traffic volume
  - Bot traffic contamination
  - Tracking implementation errors
  
  **Solutions:**
  - Implement bot filtering
  - Verify tracking code implementation
  - Use data validation tools
</Accordion>
