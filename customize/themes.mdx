## Understanding color configuration

The `colors` object in your `docs.json` controls how your theme applies color throughout your documentation. Each theme interprets these colors differently, so understanding how they work together is essential for creating a cohesive design.

### How color properties work

Mintlify themes use three color properties that work together across light and dark modes:

<ParamField path="primary" type="string" required>
  Your main brand color. Generally used for emphasis in light mode—such as links, active navigation items, and accents. The exact application varies by theme.

  ```json
  "colors": {
    "primary": "#3B82F6"
  }
  ```
</ParamField>

<ParamField path="light" type="string">
  The color used for emphasis in dark mode. When users switch to dark mode, this color replaces `primary` for most accent elements. If not specified, `primary` is used in both modes.

  ```json
  "colors": {
    "primary": "#3B82F6",
    "light": "#60A5FA"
  }
  ```
</ParamField>

<ParamField path="dark" type="string">
  The color used for interactive elements like buttons, hover states, and call-to-action components across both light and dark modes. Some themes also use this for secondary accents.

  ```json
  "colors": {
    "primary": "#3B82F6",
    "light": "#60A5FA",
    "dark": "#1E40AF"
  }
  ```
</ParamField>

### Common color configuration patterns

Here are proven color combinations for different use cases:

<AccordionGroup>
  <Accordion title="Single color (simplest)">
    Use one color for all elements. Best for getting started quickly.

    ```json
    "colors": {
      "primary": "#3B82F6"
    }
    ```

    **Result:** `#3B82F6` applies to all accent elements in both light and dark modes.
  </Accordion>

  <Accordion title="Light and dark mode variants">
    Provide separate colors optimized for each mode. Recommended for better contrast.

    ```json
    "colors": {
      "primary": "#2563EB",
      "light": "#60A5FA"
    }
    ```

    **Result:** Light mode uses `#2563EB`, dark mode uses `#60A5FA`. Both modes use the same color for buttons and interactive elements.
  </Accordion>

  <Accordion title="Full customization">
    Control all color applications independently. Best for precise brand matching.

    ```json
    "colors": {
      "primary": "#2563EB",
      "light": "#60A5FA",
      "dark": "#1E40AF"
    }
    ```

    **Result:** Light mode accents use `#2563EB`, dark mode accents use `#60A5FA`, and buttons/interactive elements use `#1E40AF` in both modes.
  </Accordion>

  <Accordion title="Monochrome theme">
    Use shades of the same color for a cohesive look.

    ```json
    "colors": {
      "primary": "#6366F1",
      "light": "#818CF8",
      "dark": "#4F46E5"
    }
    ```
  </Accordion>

  <Accordion title="High contrast">
    Use bold, saturated colors for maximum visibility.

    ```json
    "colors": {
      "primary": "#DC2626",
      "light": "#F87171",
      "dark": "#991B1B"
    }
    ```
  </Accordion>
</AccordionGroup>

### Testing your color combinations

Follow these steps to ensure your colors work well:

<Steps>
  <Step title="Test both modes">
    Toggle between light and dark modes using the theme switcher. Check that text remains readable and interactive elements are clearly visible in both modes.
  </Step>

  <Step title="Check contrast ratios">
    Use a contrast checker tool to verify your colors meet [WCAG accessibility standards](https://www.w3.org/WAI/WCAG21/Understanding/contrast-minimum.html):
    
    - Normal text: minimum 4.5:1 contrast ratio
    - Large text: minimum 3:1 contrast ratio
    - Interactive elements: minimum 3:1 contrast ratio

    Tools: [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/), [Coolors Contrast Checker](https://coolors.co/contrast-checker)
  </Step>

  <Step title="Test interactive states">
    Hover over buttons, links, and navigation items. Ensure hover states are visible and distinct from default states.
  </Step>

  <Step title="Review across themes">
    If you're considering multiple themes, preview your colors in each one. Themes apply colors differently, so what works in Mint might need adjustment in Maple or Palm.
  </Step>
</Steps>

### Troubleshooting color issues

<AccordionGroup>
  <Accordion title="White text on white background">
    **Problem:** Buttons or links appear invisible with white text on white backgrounds.

    **Common causes:**
    - Using very light colors (e.g., `#FFFFFF`, `#F9FAFB`) as your primary color
    - Not providing a `dark` color for button backgrounds

    **Solutions:**
    ```json
    // ❌ Problematic
    "colors": {
      "primary": "#FFFFFF"
    }

    // ✅ Fixed - use a visible color
    "colors": {
      "primary": "#3B82F6",
      "dark": "#1E40AF"
    }
    ```

    If you want a light aesthetic, use light backgrounds with darker accent colors:
    ```json
    "colors": {
      "primary": "#2563EB",
      "light": "#3B82F6",
      "dark": "#1E40AF"
    },
    "background": {
      "color": {
        "light": "#F9FAFB",
        "dark": "#0F172A"
      }
    }
    ```
  </Accordion>

  <Accordion title="Poor contrast in dark mode">
    **Problem:** Colors that look great in light mode are hard to see in dark mode.

    **Solution:** Provide a lighter variant for dark mode using the `light` property:
    ```json
    "colors": {
      "primary": "#1E40AF",  // Darker blue for light mode
      "light": "#60A5FA",    // Lighter blue for dark mode
      "dark": "#2563EB"
    }
    ```

    **Rule of thumb:** Dark mode colors should be 2-3 shades lighter than light mode colors.
  </Accordion>

  <Accordion title="Buttons blend into background">
    **Problem:** Call-to-action buttons don't stand out.

    **Solution:** Use the `dark` property to set a distinct button color:
    ```json
    "colors": {
      "primary": "#3B82F6",   // Links and accents
      "light": "#60A5FA",     // Dark mode accents
      "dark": "#1E40AF"       // Buttons (darker, more saturated)
    }
    ```

    Choose a `dark` color that's more saturated or darker than your `primary` color.
  </Accordion>

  <Accordion title="Colors look different than expected">
    **Problem:** Your colors appear different from your design mockups.

    **Possible causes:**
    - Theme-specific color application
    - Browser color rendering
    - Display calibration

    **Solutions:**
    1. Check your hex codes are correct (must start with `#`)
    2. Test in multiple browsers (Chrome, Firefox, Safari)
    3. Preview on different devices
    4. Try a different theme if the current one doesn't match your vision
  </Accordion>

  <Accordion title="Inconsistent colors across pages">
    **Problem:** Some pages show different colors than others.

    **Solution:** Colors are defined globally in `docs.json`. If you see inconsistencies:
    1. Clear your browser cache
    2. Check for custom CSS overrides in your MDX files
    3. Verify you're not using multiple `docs.json` files
    4. Redeploy your documentation
  </Accordion>
</AccordionGroup>
