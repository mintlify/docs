## Color customization

Configure your documentation colors using the `colors` object in `docs.json`. Understanding how the `primary`, `light`, and `dark` color properties work together helps you create accessible, visually consistent documentation across both light and dark modes.

### How color properties work

The three color properties serve different purposes depending on the theme mode:

<ResponseField name="primary" type="string" required>
  Used for emphasis elements in light mode, such as links, active navigation items, and interactive elements. This is your brand's main color.
</ResponseField>

<ResponseField name="light" type="string">
  Used for emphasis elements in dark mode. If not specified, the `primary` color is used for both light and dark modes.
</ResponseField>

<ResponseField name="dark" type="string">
  Used for buttons, hover states, and secondary interactive elements across both light and dark modes. Provides visual feedback for user interactions.
</ResponseField>

### Common color patterns

#### Single color configuration

Use only `primary` when you want the same brand color across all modes:

```json docs.json
{
  "colors": {
    "primary": "#3B82F6"
  }
}
```

This applies your primary color to all emphasis elements in both light and dark modes.

#### Light and dark mode optimization

Specify different colors for optimal contrast in each mode:

```json docs.json
{
  "colors": {
    "primary": "#2563EB",
    "light": "#60A5FA",
    "dark": "#1E40AF"
  }
}
```

Use a darker shade for `primary` (light mode emphasis), a lighter shade for `light` (dark mode emphasis), and a medium shade for `dark` (buttons and hover states).

#### Background color customization

Set custom background colors for light and dark modes:

```json docs.json
{
  "colors": {
    "primary": "#3B82F6"
  },
  "background": {
    "color": {
      "light": "#FFFFFF",
      "dark": "#0F172A"
    }
  }
}
```

### Testing color combinations

Test your color configuration in both light and dark modes:

1. **Preview locally** - Run `mint dev` and toggle between light and dark modes using the theme switcher
2. **Check key elements** - Verify colors on links, buttons, navigation items, and code blocks
3. **Test interactions** - Hover over buttons and links to ensure hover states are visible
4. **Review contrast** - Ensure text remains readable against all backgrounds

### Troubleshooting contrast issues

#### White-on-white or invisible text

If buttons or text appear invisible (white on white or dark on dark), check these common causes:

**Problem: Primary button has white background with white text**

```json docs.json
{
  "colors": {
    "primary": "#FFFFFF"  // White primary color causes white buttons
  }
}
```

**Solution: Use a color with sufficient contrast**

```json docs.json
{
  "colors": {
    "primary": "#3B82F6",  // Blue provides contrast
    "dark": "#1E40AF"      // Darker blue for buttons
  }
}
```

**Problem: Dark mode text is invisible**

```json docs.json
{
  "colors": {
    "primary": "#1E293B",  // Too dark for dark mode backgrounds
    "light": "#1E293B"     // Should be lighter
  }
}
```

**Solution: Use lighter colors for dark mode**

```json docs.json
{
  "colors": {
    "primary": "#1E293B",  // Dark color for light mode
    "light": "#60A5FA"     // Light color for dark mode
  }
}
```

#### Checking contrast ratios

Ensure your colors meet WCAG accessibility standards:

- **Normal text**: Minimum 4.5:1 contrast ratio
- **Large text**: Minimum 3:1 contrast ratio
- **Interactive elements**: Minimum 3:1 contrast ratio

Use tools like [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) to verify your color combinations.

### Theme-specific color behavior

Different themes apply colors in unique ways:

- **Mint** - Uses `primary` for links and active states, `dark` for buttons
- **Maple** - Applies `primary` to navigation highlights and interactive elements
- **Palm** - Uses colors for accent borders and subtle backgrounds
- **Willow** - Minimal color usage, primarily for links and buttons
- **Linden** - Terminal-style theme with limited color application
- **Almond** - Card borders and hover states use your color scheme
- **Aspen** - Extensive color usage across navigation and components
- **Sequoia** - Subtle color accents for emphasis elements

Preview your chosen theme with your colors at the theme preview links above to see how colors are applied.
