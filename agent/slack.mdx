## Connect your Slack workspace

When you first open the [Agent](https://dashboard.mintlify.com/products/agent) page, a guided setup walks you through connecting Slack, configuring GitHub, and sending your first message.

1. Open the [Agent](https://dashboard.mintlify.com/products/agent) page in your dashboard.
2. Click **Install Slack app** to begin the setup.
3. Follow the Slack prompts to add the Mintlify app to your workspace and link your account.
4. After connecting Slack, configure your GitHub app to give the agent access to your repositories.
5. Send your first message to the agent. Pick one of the example prompts or write your own — it sends directly to your chosen Slack channel.

After you complete onboarding, the Agent page shows your connection status and settings. You can return to the **Agent settings** section at any time to manage or reconnect your Slack workspace.

### Disconnect Slack

To disconnect Slack from the agent:

1. Open the [Agent](https://dashboard.mintlify.com/products/agent) page.
2. In the **Agent settings** section, click **Disconnect** next to your connected Slack workspace.
3. Confirm the disconnection.

## Use the agent in Slack

Once connected, you can:

- Send direct messages to the agent to use it privately to update your documentation.
- Mention `@mintlify` in a channel to use it publicly and collaboratively.
- Attach files and images directly to your messages for the agent to process and include in your docs.
- Continue conversations in threads to iterate on changes.
- Share pull request links with the agent to update related documentation.

## Send messages from the dashboard

You can also send messages to the agent directly from the dashboard without switching to Slack. On the Agent page, select a Slack channel and type your message. The agent receives it just like a Slack message and creates a pull request with the proposed changes.

## Update documentation

Use the agent to update your documentation with a new request or in an existing thread.

- **New request**: Send a direct message to the agent or mention `@mintlify` in a channel with instructions on what to update.
- **Existing thread**: Reply in the thread and mention `@mintlify` with instructions on what to update.
- **With attachments**: Upload images, diagrams, code files, or other documents with your message. The agent automatically processes and includes them in your documentation.

The agent reads the context of the request or thread and creates a pull request in your connected repository with the updates.

## Best practices

- **Be specific**: Tell the agent exactly what you want documented and where it should go.
- **Add context**: If a thread doesn't contain all the necessary information, include additional details in your message to the agent.
- **Review carefully**: You should always review pull requests that the agent creates before merging them.
