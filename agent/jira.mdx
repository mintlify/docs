Connect the Mintlify agent to your Jira project to trigger documentation updates directly from issues and comments. When you mention the agent in a Jira issue, it reads the issue context and creates a pull request with the proposed changes.

## Connect your Jira project

1. Open the agent panel in your [dashboard](https://dashboard.mintlify.com/products/agent).
2. Click the **Settings** button.
3. In the Jira integration section, click **Connect**.
4. Follow the Atlassian authorization prompts to grant the Mintlify app access to your Jira site.
5. Select the Jira projects you want the agent to access.
6. Test that the agent responds when you mention it in a Jira issue comment.

## Use the agent in Jira

Once connected, you can mention `@mintlify` in Jira issue comments to trigger documentation updates. The agent reads the issue summary, description, and comment thread for context.

You can:

- Mention `@mintlify` in an issue comment with instructions on what to document.
- Reference specific pages or sections of your docs for the agent to update.
- Continue the conversation in follow-up comments to iterate on changes.
- Share pull request URLs for the agent to use as additional context.

## Update documentation from issues

Use the agent to create or update documentation based on issue context.

- **New request**: Comment on any Jira issue and mention `@mintlify` with instructions on what to update.
- **Existing thread**: Reply in the comment thread and mention `@mintlify` to iterate on the changes.
- **Sprint completions**: At the end of a sprint, mention `@mintlify` on a summary issue to draft documentation for completed work.

The agent reads the issue context and creates a pull request in your connected repository with the updates.

## Example workflows

### Document a new feature from a story

When a user story is marked as done, comment:

> `@mintlify Document this new feature based on the issue description and acceptance criteria. Add it to the features section of our docs.`

### Update docs for a bug fix

After resolving a bug that affects documented behavior, comment:

> `@mintlify This bug fix changes how rate limiting works. Update the rate limits page to reflect the new behavior.`

### Draft release notes from a sprint

Mention the agent on an epic or release issue:

> `@mintlify Draft release notes based on the completed stories in this epic. Summarize the user-facing changes.`

## Best practices

- **Be specific**: Tell the agent exactly what you want documented and where it should go.
- **Add context**: If the issue doesn't contain all the necessary information, include additional details in your comment to the agent.
- **Review carefully**: Always review pull requests that the agent creates before merging them.
- **Use issue context**: The more detail in your issue description and acceptance criteria, the better the agent can generate relevant documentation.
