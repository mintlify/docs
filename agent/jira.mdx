Connect the Mintlify agent to Jira so your team can trigger documentation updates directly from Jira issues and comments.

## Connect your Jira workspace

1. Open the agent panel in your [dashboard](https://dashboard.mintlify.com/products/agent).
2. Click the **Settings** button.
3. In the Jira integration section, click **Connect**.
4. Follow the prompts to authorize Mintlify in your Jira workspace.

## Use the agent in Jira

Once connected, mention the agent in a Jira issue comment to trigger a documentation update. The agent reads the issue context, including the summary, description, and comment thread, and creates a pull request with the proposed changes.

You can:

- Mention the agent in an issue comment with instructions on what to document.
- Reference specific pages or sections to update.
- Attach context from related issues or epics for the agent to use.

## Example prompts

Here are some examples of prompts you can use in Jira issue comments:

- `@mintlify Document this new feature in the API reference.`
- `@mintlify Update the quickstart guide to reflect the changes in this issue.`
- `@mintlify Add a troubleshooting section based on the bug described here.`

## Best practices

- **Be specific**: Tell the agent exactly what you want documented and where it should go.
- **Add context**: If the issue doesn't contain all the necessary information, include additional details in your comment to the agent.
- **Review carefully**: Always review pull requests that the agent creates before merging them.
