Connect Linear to the agent so your team can trigger documentation updates directly from Linear issues. When someone mentions the agent in a Linear issue, it reads the issue context and creates a pull request with the requested documentation changes.

## Connect your Linear workspace

1. Open the agent panel in your [dashboard](https://dashboard.mintlify.com/products/agent).
2. Click the **Settings** button.
3. In the Linear integration section, click **Connect**.
4. Follow the prompts to authorize the Mintlify app in your Linear workspace.
5. Select which Linear teams the agent can access.

## Use the agent from Linear

Once connected, mention `@mintlify` in a Linear issue comment to trigger the agent. The agent reads the issue title, description, and comment thread for context, then creates a pull request with the requested changes.

### Trigger a documentation update

Add a comment on any Linear issue and mention `@mintlify` with your instructions:

```
@mintlify Update the API reference to document the new rate limiting behavior described in this issue.
```

The agent uses the full issue context — including the title, description, and previous comments — to understand what documentation changes are needed.

### Iterate in the comment thread

Reply to the agent's comment on the same issue to refine the pull request. Each follow-up mention of `@mintlify` in the thread continues the same conversation, so the agent can iterate on its changes.

```
@mintlify Add a code example showing how to handle rate limit errors.
```

## Example workflows

- **Feature documentation**: When a feature issue moves to done, comment `@mintlify Document this feature based on the issue description and acceptance criteria`.
- **Bug fix updates**: After resolving a bug, comment `@mintlify Update the troubleshooting guide to cover this issue and its resolution`.
- **API changes**: When an API change ships, comment `@mintlify Update the API docs to reflect the changes described in this issue`.

## Best practices

- **Write clear issue descriptions**: The agent uses the issue context to understand what to document. Detailed descriptions produce better results.
- **Be specific in your prompt**: Tell the agent what pages to update and what information to include.
- **Review pull requests**: Always review the pull requests the agent creates before merging them.
- **Use with workflows**: Combine Linear-triggered updates with [workflows](/agent/workflows) to automate documentation tasks when issues reach specific statuses.
