Connect your Linear workspace to delegate issues directly to the Mintlify agent. When you assign an issue to the agent, it reads the issue context, analyzes your documentation, and opens a pull request with the proposed changes.

## Connect your Linear workspace

<Steps>
  <Step title="Open agent settings">
    Open the agent panel in your dashboard and click the **Settings** button.
  </Step>
  <Step title="Connect Linear">
    In the Linear integration section, click **Connect** and follow the prompts to authorize the Mintlify app in your Linear workspace.
  </Step>
  <Step title="Verify the connection">
    After authorizing, the settings panel shows your connected Linear organization name. You can now delegate issues to the agent from Linear.
  </Step>
</Steps>

## Delegate an issue to the agent

To trigger the agent from Linear:

1. Open a Linear issue that describes a documentation change.
2. Delegate the issue to the Mintlify agent app.
3. The agent reads the full issue context — including the description, comments, related issues, and linked pull requests — and creates a documentation PR.

You can also mention the agent in an issue comment to provide specific instructions or additional context.

## Follow up in a thread

After delegating an issue, you can send follow-up messages in the agent session thread to:

- Refine the changes the agent made.
- Request additional updates to the same PR.
- Provide more context or clarification.

The agent retains the context from the original issue and any previous messages, so you can iterate without repeating yourself.

## Stop the agent

If you need to cancel an in-progress run, send a stop signal from the Linear agent session. The agent stops processing and preserves any changes it already made.

## What the agent reads

When you delegate an issue, the agent pulls rich context from Linear to understand the full picture:

- **Issue details**: Title, description, priority, and current status.
- **Related issues**: Parent issues, sub-issues, and linked issues (blockers, dependencies).
- **Linked pull requests**: GitHub PRs and issues attached to the Linear issue.
- **Comments**: Recent discussion on the issue for additional context.

This context helps the agent make informed decisions about what documentation to create or update.

## Best practices

- **Write clear issue descriptions**: The more detail you include in the Linear issue, the better the agent's output. Describe what changed, what documentation needs updating, and where.
- **Link related PRs**: Attach relevant GitHub pull requests to your Linear issue so the agent can reference the code changes.
- **Use follow-ups to iterate**: If the first PR isn't quite right, send a follow-up message rather than creating a new issue.
- **Review before merging**: Always review the agent's pull request before merging it into your documentation.
