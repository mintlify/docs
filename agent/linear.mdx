Connect Linear to the Mintlify agent to automatically create documentation updates when issues are created or updated in Linear. When you link a Linear issue to the agent, it reads the issue context and opens a pull request with the relevant documentation changes.

## Connect your Linear workspace

1. Open the agent panel in your [dashboard](https://dashboard.mintlify.com/products/agent).
2. Click the **Settings** button.
    <Frame>
    <img src="/images/agent/dashboard-settings-light.png" alt="The settings button in light mode." className="block dark:hidden" />
    <img src="/images/agent/dashboard-settings-dark.png" alt="The settings button in dark mode." className="hidden dark:block" />
    </Frame>
3. In the Linear integration section, click **Connect**.
4. Follow the Linear prompts to authorize the Mintlify app in your workspace.

## How it works

Once connected, the agent monitors your Linear workspace for issues that require documentation updates. When the agent picks up an issue:

1. It reads the issue title, description, and any linked pull requests or comments for context.
2. It researches your existing documentation and any connected repositories.
3. It opens a pull request with the proposed documentation changes.
4. The Linear issue is updated with a link to the pull request.

All changes go through pull requests. The agent never commits directly to your main branch.

## Trigger documentation updates from Linear

You can trigger the agent from Linear in the following ways:

- **Assign an issue to the agent**: Assign a Linear issue to the Mintlify agent to trigger a documentation update based on the issue context.
- **Label an issue**: Add a designated label (such as `docs` or `documentation`) to a Linear issue to signal that it needs documentation work.

## Best practices

- **Write clear issue descriptions**: The agent uses the issue title and description to understand what documentation changes are needed. Include specific details about what to document and where.
- **Link related pull requests**: If the issue relates to a code change, link the pull request in the Linear issue. The agent uses this context to write more accurate documentation.
- **Review pull requests carefully**: Always review the pull requests the agent creates before merging them.
- **Use labels consistently**: Establish a consistent labeling convention in your team so the agent picks up the right issues.
