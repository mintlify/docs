Connect Linear to the agent so your team can trigger documentation updates directly from Linear issues. When you assign a documentation task in Linear, the agent picks it up, reads the issue context, and opens a pull request with the proposed changes.

## Connect your Linear workspace

1. Open the agent panel in your [dashboard](https://dashboard.mintlify.com/products/agent).
2. Click the **Settings** button.
3. In the Linear integration section, click **Connect**.
4. Follow the Linear prompts to authorize the Mintlify app in your workspace.

## How it works

Once connected, the agent monitors your Linear workspace for issues that are assigned to it. When the agent receives an issue:

1. It reads the issue title, description, and any linked context.
2. It researches your existing documentation and any connected repositories.
3. It creates a pull request with the proposed documentation changes.
4. It posts a comment on the Linear issue with a link to the pull request.

## Create a documentation task in Linear

Write Linear issues the same way you would prompt the agent in Slack or the dashboard. Include enough context for the agent to understand what needs to change.

A good issue includes:
- **What to update**: Specify which pages or sections need changes.
- **Why it needs updating**: Reference a feature change, user feedback, or new API.
- **Supporting context**: Link to pull requests, design docs, or relevant threads.

### Example issues

**Update API reference for new endpoint**
> Add documentation for the new `/v2/webhooks` endpoint. The PR with the implementation is at `your-org/your-product#452`. Include request/response examples and authentication requirements.

**Document a new feature**
> We shipped a new dashboard analytics feature. Add a page explaining how to view and export usage metrics. The feature code is in `your-org/your-product`.

**Fix outdated content**
> The setup guide still references v1 authentication. Update it to reflect the v2 OAuth flow we shipped last month.

## Best practices

- **Be specific**: Tell the agent exactly what you want documented and where. Vague issues like "update the docs" produce less useful results.
- **Link related resources**: Attach pull requests, design documents, or Slack threads so the agent has full context.
- **Review pull requests**: Always review the pull requests that the agent creates before merging them.
- **One task per issue**: Keep issues focused on a single documentation change. Split large projects into multiple issues.
