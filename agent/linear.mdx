Connect the Mintlify agent to your Linear workspace so your team can trigger documentation updates directly from Linear issues. When you mention the agent in an issue or comment, it reads the context and creates a pull request with proposed documentation changes.

## Connect your Linear workspace

1. Open the agent panel in your [dashboard](https://dashboard.mintlify.com/products/agent).
2. Click the **Settings** button.
3. In the Linear integration section, click **Connect**.
4. Follow the Linear prompts to authorize the Mintlify app in your workspace.

## Use the agent in Linear

Once connected, mention the agent in Linear to trigger documentation updates. The agent reads the issue context — title, description, and comments — to understand what documentation changes are needed.

You can use the agent from:

- **Issue descriptions**: Mention the agent when creating an issue to start a documentation task.
- **Issue comments**: Mention the agent in a comment on an existing issue to request documentation updates based on the issue context.

The agent creates a pull request in your connected documentation repository with the proposed changes.

## Example workflows

### Document a new feature from a Linear issue

When your team tracks a new feature in Linear, mention the agent in the issue to draft documentation for it:

> Update the docs to include the new authentication flow described in this issue.

The agent reads the issue details and any linked context to generate a pull request with the relevant documentation changes.

### Update docs when closing a feature issue

After completing a feature, add a comment mentioning the agent before closing the issue:

> The feature described in this issue shipped. Update the API reference to reflect the new endpoints.

### Capture technical decisions

When your team documents architecture decisions or implementation details in Linear issues, mention the agent to capture that knowledge in your docs:

> Add a section to the architecture overview explaining the caching strategy discussed in this issue.

## Best practices

- **Be specific**: Include details about what you want documented and where it should go in your docs.
- **Add context**: If the issue doesn't contain all the necessary information, include additional details in your message to the agent.
- **Review carefully**: Always review pull requests that the agent creates before merging them.
- **Use issue context**: The agent reads the full issue, so reference specific parts of the issue description or comments when relevant.
