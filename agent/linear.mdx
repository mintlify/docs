Connect Linear to the Mintlify agent to trigger documentation updates directly from your issue tracker. When you mention the agent in a Linear issue or comment, it reads the context and creates a pull request with proposed documentation changes.

## Connect your Linear workspace

1. Open the agent panel in your [dashboard](https://dashboard.mintlify.com/products/agent).
2. Click the **Settings** button.
3. In the Linear integration section, click **Connect**.
4. Follow the Linear prompts to authorize the Mintlify app in your workspace.

## Use the agent in Linear

Once connected, mention `@mintlify` in a Linear issue or comment to trigger the agent. The agent reads the issue context — including the title, description, and thread — and creates a pull request with the proposed documentation changes.

You can use the agent in Linear to:

- Draft documentation for new features described in issues.
- Update existing docs when an issue describes a product change.
- Generate changelog entries from completed issues.
- Capture technical context from issue threads before it gets lost.

## Example prompts

Use these examples as starting points for prompting the agent in Linear issues or comments:

- `@mintlify Document this new feature based on the issue description and linked PRs.`
- `@mintlify Update the API reference to reflect the changes described in this issue.`
- `@mintlify Add a troubleshooting section to the setup guide for the bug described here.`
- `@mintlify Generate a changelog entry for this completed feature.`

## Best practices

- **Include context in the issue**: The agent uses the issue title, description, and comments as context. The more detail you provide, the better the documentation output.
- **Link related pull requests**: When issues reference GitHub pull requests, the agent can use the code changes as additional context for documentation updates.
- **Be specific**: Tell the agent exactly what you want documented and where it should go, just as you would when using it in Slack or the dashboard.
- **Review pull requests**: Always review the pull requests that the agent creates before merging them.
