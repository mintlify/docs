## Connect your Linear workspace

1. Open the agent panel in your dashboard.
2. Click the **Settings** button.
    <Frame>
    <img src="/images/agent/dashboard-settings-light.png" alt="The settings button in light mode." className="block dark:hidden" />
    <img src="/images/agent/dashboard-settings-dark.png" alt="The settings button in dark mode." className="hidden dark:block" />
    </Frame>
3. In the Linear integration section, click **Connect**.
4. Follow the Linear prompts to authorize the Mintlify app in your workspace.

## Use the agent in Linear

Once connected, you can mention `@mintlify` in any Linear issue comment to trigger the agent. The agent reads the issue context, including the title, description, and comment thread, then creates a pull request with the requested documentation changes.

You can:

- Mention `@mintlify` in a comment with instructions on what to document.
- Reply in an existing comment thread to iterate on changes.
- Reference the issue description for context without repeating details.

## Example workflows

<AccordionGroup>
  <Accordion title="Document a new feature from a shipped issue">
    When an engineering issue is marked as done, comment:

    `@mintlify Document this feature based on the issue description. Add it to the guides section.`

    The agent uses the issue title, description, and any linked context to draft the documentation.
  </Accordion>
  <Accordion title="Update docs for an API change">
    On an issue that changes API behavior, comment:

    `@mintlify Update the API reference to reflect the changes described in this issue.`
  </Accordion>
  <Accordion title="Create a changelog entry">
    After shipping a release, comment on the tracking issue:

    `@mintlify Write a changelog entry summarizing the changes in this issue and any sub-issues.`
  </Accordion>
</AccordionGroup>

## Best practices

- **Be specific**: Tell the agent exactly what you want documented and where it should go.
- **Add context**: If the issue description doesn't contain all the necessary information, include additional details in your comment to the agent.
- **Review carefully**: Always review pull requests that the agent creates before merging them.
- **Use issue links**: Reference related issues or PRs in your comment to give the agent more context.
