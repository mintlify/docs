Connect GitHub pull requests to the agent so your team can trigger documentation updates directly from PR comments. When someone mentions the agent in a pull request, it reads the PR context and creates a documentation pull request with the requested changes.

## Connect your GitHub repositories

1. Open the agent panel in your [dashboard](https://dashboard.mintlify.com/products/agent).
2. Click the **Settings** button.
3. In the GitHub integration section, click **Connect**.
4. Follow the prompts to install the Mintlify GitHub App on your organization.
5. Select which repositories the agent can access.

## Use the agent from pull requests

Once connected, mention `@mintlify` in a pull request comment to trigger the agent. The agent reads the PR title, description, diff, and comment thread for context, then creates a separate pull request with the requested documentation changes.

### Trigger a documentation update

Add a comment on any pull request and mention `@mintlify` with your instructions:

```
@mintlify Update the docs to reflect the API changes in this PR.
```

The agent uses the full PR context — including the title, description, diff, and previous comments — to understand what documentation changes are needed.

### Iterate in the comment thread

Reply to the agent's comment on the same pull request to refine the documentation PR. Each follow-up mention of `@mintlify` in the thread continues the same conversation, so the agent can iterate on its changes.

```
@mintlify Add a migration guide for the breaking changes in this PR.
```

## Example workflows

- **Feature documentation**: When a feature PR is ready for review, comment `@mintlify Document the new feature introduced in this PR`.
- **API changes**: When an API change merges, comment `@mintlify Update the API reference to reflect the endpoint changes in this PR`.
- **Migration guides**: For breaking changes, comment `@mintlify Write a migration guide based on the breaking changes in this PR`.
- **Release notes**: Before a release, comment `@mintlify Generate release notes based on the changes in this PR`.

## Best practices

- **Write clear PR descriptions**: The agent uses the PR context to understand what to document. Detailed descriptions and well-structured diffs produce better results.
- **Be specific in your prompt**: Tell the agent what pages to update and what information to include.
- **Review pull requests**: Always review the documentation pull requests the agent creates before merging them.
- **Use with workflows**: Combine PR-triggered updates with [workflows](/agent/workflows) to automatically update docs when code changes merge.
