Trigger an autopilot analysis to check if a pull request requires documentation updates. The autopilot agent will analyze the PR changes and create a suggestion if documentation updates are needed.

## Endpoint

```
POST https://api.mintlify.com/v1/autopilot/:subdomain/suggestions/check
```

## Authentication

This endpoint requires an [admin API key](/api-reference/introduction#admin-api-key).

Include your API key in the `Authorization` header:

```
Authorization: Bearer mint_YOUR_API_KEY
```

## Path parameters

<Fields>
  <Field name="subdomain" type="string" required>
    The subdomain of your documentation site
  </Field>
</Fields>

## Request body

<Fields>
  <Field name="owner" type="string" required>
    The GitHub repository owner (organization or user)
  </Field>
  <Field name="repo" type="string" required>
    The GitHub repository name
  </Field>
  <Field name="pullRequestNumber" type="number" required>
    The pull request number to analyze
  </Field>
</Fields>

## Response

<Fields>
  <Field name="status" type="string">
    Returns `"queued"` when the analysis job is successfully queued
  </Field>
</Fields>

## Example request

```bash
curl -X POST https://api.mintlify.com/v1/autopilot/my-docs/suggestions/check \
  -H "Authorization: Bearer mint_YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "owner": "your-org",
    "repo": "your-repo",
    "pullRequestNumber": 123
  }'
```

## Example response

```json
{
  "status": "queued"
}
```

## How it works

When you trigger a PR check:

1. The autopilot agent fetches the pull request details from GitHub
2. It analyzes the PR changes using an agentic workflow with access to:
   - Your documentation repository files
   - The code repository files
   - The PR diff and metadata
3. If documentation updates are needed, it creates a suggestion in your autopilot dashboard
4. The suggestion includes a prompt describing what should be updated

## Error responses

### 400 Bad Request

Invalid request parameters or missing GitHub installation:

```json
{
  "error": "No GitHub installation found for owner \"your-org\". Make sure the Mintlify GitHub App is installed on this organization."
}
```

### 404 Not Found

Deployment not found for the specified subdomain.

### 500 Internal Server Error

Failed to scan the pull request:

```json
{
  "error": "Failed to scan pull request"
}
```
