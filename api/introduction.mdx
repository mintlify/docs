The Mintlify REST API enables you to programmatically interact with your documentation, trigger updates, embed AI-powered chat experiences, and export analytics data.

## Endpoints

- [Trigger update](/api/update/trigger): Trigger an update of your site when desired.
- [Get update status](/api/update/status): Get the status of an update and other details about your docs.
- [Create Mintie bot job](/api/agent/create-agent-job): Create a Mintie bot job to automatically edit your documentation.
- [Get Mintie bot job](/api/agent/get-agent-job): Retrieve the details and status of a specific Mintie bot job.
- [Get all Mintie bot jobs](/api/agent/get-all-jobs): Retrieve all Mintie bot jobs for a domain.
- [Generate Mintie assistant message](/api/assistant/create-assistant-message): Embed Mintie assistant, trained on your docs, into any application of your choosing.
- [Search documentation](/api/assistant/search): Search through your documentation.
- [Get user feedback](/api/analytics/feedback): Export user feedback from your documentation.
- [Get Mintie assistant conversations](/api/analytics/assistant-conversations): Export Mintie assistant conversation history.

### Common use cases

- **Automated deployments**: Trigger site updates at set intervals or when events occur with [Trigger update](/api/update/trigger) and [Get update status](/api/update/status).
- **CI/CD integration**: Update documentation as part of your deployment pipeline when code changes with [Trigger update](/api/update/trigger).
- **Custom integrations**: Embed Mintie assistant into your product, support portal, or internal tools with [Generate Mintie assistant message](/api/assistant/create-assistant-message) and [Search documentation](/api/assistant/search).
- **Automated editing**: Use Mintie bot jobs to programmatically update documentation at scale with [Create Mintie bot job](/api/agent/create-agent-job), [Get Mintie bot job](/api/agent/get-agent-job), and [Get all Mintie bot jobs](/api/agent/get-all-jobs).
- **Analytics export**: Export feedback and Mintie assistant conversations for external analysis with [Get user feedback](/api/analytics/feedback) and [Get Mintie assistant conversations](/api/analytics/assistant-conversations).

## Authentication

Generate API keys on the [API keys page](https://dashboard.mintlify.com/settings/organization/api-keys) in your dashboard. Each API key belongs to an organization--you can use keys across multiple deployments within the same organization.

You can create up to 10 API keys per hour per organization.

### Admin API key

Use the admin API key to authenticate requests to [Trigger update](/api/update/trigger), [Get update status](/api/update/status), [Create Mintie bot job](/api/agent/create-agent-job), [Get Mintie bot job](/api/agent/get-agent-job), [Get all Mintie bot jobs](/api/agent/get-all-jobs), [Get user feedback](/api/analytics/feedback), and [Get Mintie assistant conversations](/api/analytics/assistant-conversations).

Admin API keys begin with the `mint_` prefix. 

The admin API key is a server-side secret. Do not expose it in client-side code.

### Mintie assistant API key

Use the Mintie assistant API key to authenticate requests to [Generate Mintie assistant message](/api/assistant/create-assistant-message) and [Search documentation](/api/assistant/search) endpoints.

Mintie assistant API keys begin with the `mint_dsc_` prefix.

The Mintie assistant API key is a public token that you can reference in frontend code.

<Note>
  Calls using the Mintie assistant API token can incur costs: either using your Mintie assistant credits or incurring overages.
</Note>
