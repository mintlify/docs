The `docs.json` file is the central configuration file for your Mintlify documentation site. It controls the global settings for your site including visual branding, navigation structure, integrations, API settings, and more. Think of it as the blueprint for your site.

## Required fields

You must define four fields to build a working site.

| Field | Description |
|---|---|
| `name` | Your project or organization name |
| `theme` | The layout [theme](/customize/themes) for your site |
| `colors.primary` | Primary brand color as a hex code |
| `navigation` | Your content structure |

All other fields are optional. Add them as you customize and refine your site.

## Minimal configuration

For the best editing experience, include the `$schema` reference at the top of your `docs.json`. This enables autocomplete, validation, and inline documentation in most editors.

```json docs.json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Your project name",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": [
    {
      "group": "Home",
      "pages": ["index"]
    }
  ]
}
```

## Splitting your configuration with `$ref`

As your `docs.json` grows, you can split it into multiple files using `$ref`. Any value in `docs.json` can reference an external JSON file, and Mintlify resolves all references at build time.

Use `$ref` to:
- Keep navigation groups in separate files so multiple teams can own their own sections
- Reduce merge conflicts by isolating frequently changing sections
- Reuse shared configuration across different parts of your docs

### Basic usage

Set any value to an object with a `$ref` key pointing to a relative JSON file path:

```json docs.json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Your project name",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": [
    { "$ref": "./nav/getting-started.json" },
    { "$ref": "./nav/api-reference.json" }
  ]
}
```

```json nav/getting-started.json
{
  "group": "Getting started",
  "pages": ["introduction", "quickstart", "authentication"]
}
```

```json nav/api-reference.json
{
  "group": "API reference",
  "pages": ["api/list-users", "api/create-user", "api/delete-user"]
}
```

### Nested references

Referenced files can contain their own `$ref` values. Mintlify resolves the full chain automatically. Relative paths in nested files resolve from the directory of the file that contains them.

```json docs.json
{
  "navigation": [{ "$ref": "./nav/guides.json" }]
}
```

```json nav/guides.json
{
  "group": "Guides",
  "pages": { "$ref": "./pages.json" }
}
```

```json nav/pages.json
["guides/setup", "guides/deployment", "guides/monitoring"]
```

### Merging sibling keys

When a `$ref` resolves to an object, you can add sibling keys alongside `$ref` to extend or override the referenced values:

```json docs.json
{
  "navigation": [
    {
      "$ref": "./nav/auth.json",
      "icon": "lock"
    }
  ]
}
```

```json nav/auth.json
{
  "group": "Authentication",
  "pages": ["auth/login", "auth/logout"]
}
```

This produces the equivalent of:

```json
{
  "group": "Authentication",
  "pages": ["auth/login", "auth/logout"],
  "icon": "lock"
}
```

Sibling keys take priority over keys in the referenced file when both define the same key.

<Note>
  Sibling keys are only merged when the referenced file resolves to an object. If it resolves to an array or primitive value, sibling keys are ignored.
</Note>

### Limitations

- Referenced files must contain valid JSON and use the `.json` extension.
- File paths must be relative and cannot reference files outside your project directory.
- Circular references (file A references file B, which references file A) are not allowed and produce a build error.

## Upgrading from `mint.json`

If your project uses the deprecated `mint.json` file, follow these steps to upgrade to `docs.json`.

<Steps>
  <Step title="Install or update the CLI">
    If you haven't installed the [CLI](/installation), install it now:

    <CodeGroup>

    ```bash npm
    npm i -g mint
    ```

    ```bash yarn
    yarn global add mint
    ```

    ```bash pnpm
    pnpm add -g mint
    ```

    </CodeGroup>

    If you already have the CLI installed, make sure it is up to date:

    ```bash
    mint update
    ```
  </Step>
  <Step title="Create your docs.json file">
    In your docs repository, run:

    ```bash
    mint upgrade
    ```

    This command creates a `docs.json` file from your existing `mint.json`. Review the generated file to ensure all settings are correct.
  </Step>
  <Step title="Delete your mint.json file">
    After verifying your `docs.json` is configured correctly, you can safely delete your old `mint.json` file.
  </Step>
</Steps>
